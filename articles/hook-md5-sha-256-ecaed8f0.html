Hook 安全加固：MD5→SHA-256 遷移 + 日誌摘要消毒（2026-03-04）

資訊安全Hook加固SHA-256憑證防護日誌消毒系統優化

Hook 安全加固：MD5→SHA-256 遷移 + 日誌摘要消毒

日期

2026-03-04

概述

對 hooks/behavior_tracker.py 與 hooks/post_tool_logger.py 進行資訊安全加固，修復兩個憑證外洩風險漏洞。

優化項目 1：behavior_tracker.py 安全加固

問題

  1. MD5 雜湊不一致：_compute_signature() 使用 MD5 計算模式簽名，而同專案的 agent_guardian.py 已統一使用 SHA-256。MD5 存在碰撞風險，不符安全最佳實踐。
  2. summary_sample 憑證外洩：track() 函數將原始摘要直接寫入 context/behavior-patterns.json 的 summary_sample 欄位，若 curl 指令包含 Authorization header 或 API Token，敏感資訊會被持久化到磁碟。

修正

  1. MD5→SHA-256：hashlib.md5 改為 hashlib.sha256，與 agent_guardian.py 一致。
  2. 新增 _sanitize_summary() 函數：在寫入 summary_sample 前自動消毒，移除：

- Authorization: Bearer/Basic header 值

- X--Token、X--Key、X-*-Secret header 值

- $TODOIST_API_TOKEN、$env:API_KEY 等環境變數展開值

  1. 正則模式：re.IGNORECASE 確保大小寫不敏感匹配。

測試

新增 tests/hooks/test_behavior_tracker.py：20 個測試（13 消毒 + 7 簽名），全部通過。

優化項目 2：post_tool_logger.py 摘要消毒

問題

classify_bash() 將原始 curl 指令截斷後直接作為 JSONL 日誌的 summary 欄位，可能包含 Authorization header 等敏感資訊，導致憑證外洩至 logs/structured/*.jsonl。

修正

  1. 新增 _sanitize_bash_summary() 函數：使用預編譯模組級正則（_RE_AUTH_HEADER、_RE_TOKEN_HEADER）消毒 Bash 指令摘要。
  2. 修改 classify_bash()：在回傳 summary 前套用消毒，但 tags 分類仍使用原始 command（確保 API 來源偵測不受影響）。
  3. 效能考量：正則在模組載入時預編譯（re.compile），避免每次工具呼叫重複編譯。

測試

更新 tests/hooks/test_post_tool_logger.py：新增 10 個測試（8 消毒 + 2 整合），總計 60 個測試全部通過。

影響範圍

  • hooks/behavior_tracker.py：_compute_signature()（SHA-256）+ _sanitize_summary()（新函數）+ track()（呼叫消毒）
  • hooks/post_tool_logger.py：_sanitize_bash_summary()（新函數）+ classify_bash()（套用消毒）
  • tests/hooks/test_behavior_tracker.py：新建 20 個測試
  • tests/hooks/test_post_tool_logger.py：新增 10 個測試（50→60）

驗證結果

  • 修改檔案測試：80/80 通過
  • 全套件回歸測試：586/586 通過（無回歸）

技術決策

  • SHA-256 統一：所有 hook 模組一致使用 SHA-256，避免混用不同雜湊演算法
  • 消毒不影響分類：tags 分類邏輯使用原始命令判斷，消毒僅影響持久化的 summary
  • 預編譯正則：hot path（每次工具呼叫觸發）使用模組級編譯，降低效能開銷
  • output_chars 始終為 0：Claude Code PostToolUse 協定限制，tool_output 可能不傳入，已於文件中記錄
