一、專案概述

Hindsight（vectorize-io/hindsight）是由 Vectorize 團隊開發的開源 Agent 記憶系統，定位為「Agent Memory That Works Like Human Memory」。專案以仿生學記憶結構為核心設計理念，將記憶視為推理的第一級基底（first-class substrate for reasoning），而非僅是薄層檢索層。

二、核心架構：四網路記憶模型

Hindsight 最大的設計創新是將 Agent 記憶組織為四個層次分明的網路，分離「證據」與「推論」：

2.1 記憶類型階層

| 層級 | 類型 | 功能 | 類比 |

|------|------|------|------|

| L4 | Mental Models | 使用者策展的常用查詢摘要 | 人類的心智模型 |

| L3 | Observations | 系統自動從事實合成的洞察 | 人類的歸納推理 |

| L2 | World Facts | 接收的客觀資訊 | 人類的陳述性記憶 |

| L1 | Experience Facts | Agent 自身的行動與互動紀錄 | 人類的情節性記憶 |

2.2 Observation 自動合成機制

這是 Hindsight 與傳統向量記憶的關鍵差異：

  • 新事實進入時，系統自動分析並合併至既有或新建的 Observation
  • 持續追蹤證據鏈（evidence tracking），隨新資訊到來持續精煉
  • 等同人類的「從經驗中學習」——不只記住片段，而是建立連貫的知識結構

---

三、核心操作：Retain / Recall / Reflect 三柱

3.1 Retain（保留）

將 Agent 互動轉換為結構化、時間感知的記憶：

  • 執行事實萃取（fact extraction）與實體解析（entity resolution）
  • 例如：「Alice」、「alice@company.com」、「帳戶持有人」會解析為同一節點
  • 支援自訂 metadata 豐富記憶上下文

3.2 Recall（回憶）

在 Token 預算限制內檢索最相關的記憶，採用 TEMPR 四策略並行檢索：

| 策略 | 功能 | 適用場景 |

|------|------|----------|

| Semantic | 向量相似度搜尋 | 概念性查詢、同義改述偵測 |

| BM25 Keyword | 關鍵字精確匹配 | 專有名詞、技術術語、精確片語 |

| Graph Traversal | 實體關係圖遍歷 | 間接關聯查詢、多跳推理 |

| Temporal | 時間推理 | 「上個月」、日期範圍、時序查詢 |

四策略同時發動，由 cross-encoder reranker 融合為單一評分排序清單。

3.3 Reflect（反思）

在累積記憶上執行推理、回答問題、更新信念：

  • 支援偏好條件推理（preference-conditioned reasoning）
  • 可配置行為參數：skepticism（懷疑度 1-5）、literalism（字面程度 1-5）、empathy（同理心 1-5）

---

四、Memory Bank 配置系統

每個記憶銀行有三個可配置參數，塑造 Agent 推理行為：

| 參數 | 功能 | 範例 |

|------|------|------|

| Mission | 自然語言身份定義 | 「你是一位系統架構師」 |

| Directives | 硬約束與合規規則 | 「永遠不要推薦未經驗證的方案」 |

| Disposition | 軟性人格特質 | skepticism=4, literalism=2, empathy=3 |

六、MCP 原生整合

6.1 架構

  • API Server：Port 8888（MCP 端點）、Port 9999（管理 UI）
  • 儲存層：PostgreSQL + pgvector
  • 處理管線：事實萃取 → 實體解析 → 多策略檢索 → cross-encoder reranking

6.2 部署


docker run --rm -it --pull always -p 8888:8888 -p 9999:9999 \
  -e HINDSIGHT_API_LLM_API_KEY=YOUR_KEY \
  -v $HOME/.hindsight-docker:/home/hindsight/.pg0 \
  ghcr.io/vectorize-io/hindsight:latest

MCP 端點：http://localhost:8888/mcp/{bank_id}/

6.3 Claude Desktop 配置


{
  "mcpServers": {
    "hindsight": {
      "type": "http",
      "url": "http://localhost:8888/mcp/your_bank_id/"
    }
  }
}

支援 Claude Desktop、Cursor、VS Code、Windsurf 等 MCP 相容客戶端。

八、對 Daily-Digest-Prompt 系統的五大借鑑啟示

啟示 1：四層記憶分離（高價值）

現狀：本系統使用平面 JSON 檔案（digest-memory.json、research-registry.json）存儲記憶

借鑑：可以引入記憶層級概念：

  • L1 Experience：context/continuity/*.json（每日執行紀錄）
  • L2 World Facts：context/research-registry.json（外部研究事實）
  • L3 Observations：新建「系統洞察摘要」，自動從 L1/L2 合成
  • L4 Mental Models：config/*.yaml（策展的系統知識）

優先級：P1 | 工作量：Medium | 目標檔案：context/ 目錄結構重組

啟示 2：TEMPR 多策略並行檢索

現狀：知識庫已有 hybrid search（BM25 + 向量），但缺少圖遍歷和時間推理

借鑑：

  • 在 KB 搜尋中加入時間感知檢索（按日期加權）
  • 研究 entity resolution 可能性（相同主題不同稱呼的統一）

優先級：P2 | 工作量：High | 目標檔案：knowledge-base-search/src/retriever.ts

啟示 3：Observation 自動合成

現狀：system-insight Skill 手動觸發產出系統洞察

借鑑：實作自動合成機制——每次研究匯入 KB 後，自動檢查是否有可合併的相關筆記，產出 Observation 層級的摘要筆記

優先級：P2 | 工作量：Medium | 目標檔案：skills/digest-memory/SKILL.md

啟示 4：Memory Bank 行為配置

現狀：Agent 行為由 CLAUDE.md + SKILL.md 靜態定義

借鑑：Hindsight 的 Mission/Directives/Disposition 三維配置模型可啟發更精細的 Agent 行為控制。可將 config/llm-router.yaml 擴展加入 disposition 參數（如 skepticism 程度影響研究深度）

優先級：P3 | 工作量：Low | 目標檔案：config/llm-router.yaml

啟示 5：MCP Memory Server 整合

現狀：記憶存儲在本地 JSON 檔案，跨工具不共享

借鑑：將 Hindsight 作為 MCP Server 整合至 Claude Code 的 MCP 配置中，讓所有 Agent（daily-digest、todoist、system-audit）共享同一記憶基底

優先級：P1 | 工作量：Low（Docker 一鍵部署）| 目標檔案：.claude/settings.json MCP 配置
