Skill 使用覆盖率诊断与触发机制优化

深度研究洞察简报

日期：2026-04-06

系列：skill-usage-diagnostic (foundation 阶段)

研究动机：系统健康检查显示 Skill 覆盖率仅 53.8%（14/26），低于业界精英标准（90%）

摘要

本研究综合分析业界 AI Agent Skill 路由、触发机制与覆盖率评估的最佳实践，建立本系统低覆盖率的根因分类框架。关键发现：(1) 业界精英覆盖率为 90-100%，但仅 15% 团队达标；(2) 触发机制分三层（Rule-Based、Capability Matching、Intent Detection），当前系统仅实作第一层；(3) Observability 具双重目的（诊断 + 反馈闭环），可支撑渐进式优化。建议采取三阶段策略：短期补充触发词（1-2周）、中期 Metadata 质量工程（1-2月）、长期引入 Hybrid Routing（3-6月），预期可将覆盖率提升至 80% 以上。

核心洞见

1. 覆盖率基准与执行落差

业界仅 15% 团队达到精英覆盖率（90-100%），当前系统 53.8% 有明确优化空间。

2. 触发机制三层架构

Rule-Based（已实作）+ Capability Matching（未实作）+ Intent Detection（未实作）。当前瓶颈在触发词不足。

3. Observability 双重目的

传统诊断 + 持续改进反馈闭环，三层监控（Session/Trace/Span）。

4. 根因分类框架

5 类根因：触发词不足（8-10 Skill）、Metadata 质量低（3-4）、功能重叠（1-2）、业务需求低（1-2）、Capability 不可发现（架构升级）。

5. 渐进式优化路线图

Phase 1（1-2周）补充触发词 → Phase 2（1-2月）Metadata 工程 → Phase 3（3-6月）Hybrid Routing。
