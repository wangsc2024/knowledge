HTML5 Canvas 後處理特效技術研究

CRT 復古濾鏡、Scanline、色差分離與 WebGL Shader 實作最佳實踐

---

技術概述

CRT（Cathode Ray Tube）後處理特效是復古風格遊戲中最具辨識度的視覺效果。透過模擬 CRT 顯示器的物理特性——掃描線、螢光體發光、色差分離、畫面彎曲與暗角效果——為現代 HTML5 Canvas 遊戲增添懷舊氛圍。實作方式分為兩大路線：Canvas 2D API 純 JavaScript 方案（適合輕量級遊戲）與 WebGL Shader 方案（適合效能要求高的場景）。本研究涵蓋兩種路線的核心原理、程式碼範例與效能考量。

---

核心概念與原理

1. 掃描線（Scanlines）

CRT 顯示器以電子槍逐行掃描螢光幕。每行之間存在微小間隙，形成可見的水平暗帶。

Canvas 2D 實作（本專案 Breakout v1.2 使用的方式）：


// 每隔 3px 繪製一條半透明黑線
ctx.fillStyle = 'rgba(0,0,0,0.12)';
for (let y = 0; y < h; y += 3 * scale) {
  ctx.fillRect(0, y, w, 1 * scale);
}
  • 優點：零依賴，CPU 繪製，相容性最高
  • 缺點：大畫布（>1080p）時循環次數多，可用 createPattern 優化

WebGL Shader 實作：


// Fragment shader
float scanline = sin(vUv.y * resolution.y * 3.14159) * 0.5 + 0.5;
scanline = pow(scanline, scanlineIntensity);
gl_FragColor.rgb *= scanline;
  • 正弦波的波峰代表像素中心（亮），波谷代表行間隙（暗）
  • scanlineIntensity 控制強度，0.6 為常見預設值

2. 暗角效果（Vignette）

模擬 CRT 管壁邊緣光線衰減，畫面四角較暗。

Canvas 2D 實作：


const vg = ctx.createRadialGradient(
  w / 2, h / 2, w * 0.35,  // 內圈（全透明）
  w / 2, h / 2, w * 0.75   // 外圈（半透明黑）
);
vg.addColorStop(0, 'rgba(0,0,0,0)');
vg.addColorStop(1, 'rgba(0,0,0,0.4)');
ctx.fillStyle = vg;
ctx.fillRect(0, 0, w, h);

WebGL Shader 實作：


float vignette = smoothstep(0.8, 0.3, length(vUv - 0.5));
gl_FragColor.rgb *= vignette;

3. 色差分離（Chromatic Aberration）

模擬 CRT 中 RGB 三色磷光體的物理偏移。核心原理：對同一張畫面取樣三次，R/G/B 通道各偏移不同方向。

Canvas 2D 簡易實作（本專案使用的方式）：


const shimmer = Math.sin(Date.now() * 0.002) * 0.015;
if (shimmer > 0) {
  ctx.globalCompositeOperation = 'lighter';
  ctx.globalAlpha = shimmer;
  ctx.fillStyle = '#ff0000';
  ctx.fillRect(1, 0, w, h);   // 紅色右偏 1px
  ctx.fillStyle = '#0000ff';
  ctx.fillRect(-1, 0, w, h);  // 藍色左偏 1px
  ctx.globalCompositeOperation = 'source-over';
  ctx.globalAlpha = 1;
}
  • 此方法用「整個畫面覆蓋偏移色塊 + lighter 混合模式」模擬
  • 適合微量（shimmer < 0.02）場景，CPU 開銷極低

WebGL Shader 精確實作：


uniform float chromaticAberration; // 建議 0.0005

float colR = texture2D(uTexture, vec2(uv.x - chromaticAberration, uv.y)).r;
float colG = texture2D(uTexture, uv).g;
float colB = texture2D(uTexture, vec2(uv.x + chromaticAberration, uv.y)).b;
gl_FragColor = vec4(colR, colG, colB, 1.0);
  • 每個通道獨立偏移取樣，效果精確
  • chromaticAberration 建議極小值（0.0005~0.002）

4. 桶形畸變（Barrel Distortion）

模擬 CRT 螢幕的球面彎曲。數學原理：將 UV 座標從中心向外推移。


vec2 barrelDistort(vec2 uv, float amount) {
  vec2 centered = uv - 0.5;
  float dist = dot(centered, centered);
  return uv + centered * dist * amount;
}
  • amount 越大彎曲越明顯，0.001~0.002 為自然值
  • 邊緣像素被推向外側，中央保持不變

5. 螢光點遮罩（Dot Mask / Aperture Grille）

CRT 螢幕以 RGB 三色螢光點或條紋排列，形成亞像素結構。

Trinitron 孔柵式：垂直 RGB 條紋（Sony CRT 特色）

蔭罩式：三角排列的 RGB 螢光點（一般 CRT）


// 簡易 dot mask
vec3 dotMask = vec3(
  step(0.33, fract(gl_FragCoord.x / 3.0)),
  step(0.33, fract((gl_FragCoord.x + 1.0) / 3.0)),
  step(0.33, fract((gl_FragCoord.x + 2.0) / 3.0))
);
gl_FragColor.rgb *= mix(vec3(1.0), dotMask, dotMaskIntensity);

6. 磷光體發光（Phosphor Glow）

模擬 CRT 螢光體被電子束激發後的發光效果，使明亮物體周圍產生光暈。

Canvas 2D 實作（本專案使用）：


if (crtEnabled) {
  ctx.shadowColor = `rgb(${r},${g},${b})`;
  ctx.shadowBlur = 6 * scale;
}
// 繪製磚塊...
if (crtEnabled) {
  ctx.shadowBlur = 0;
}
  • 注意：shadowBlur 在 Canvas 2D 效能極差，僅適合少量物件
  • 大量物件建議改用離屏 Canvas 預渲染或 WebGL bloom

---

最佳實踐與常見陷阱

效能優化

| 技術 | Canvas 2D | WebGL |

|------|-----------|-------|

| Scanlines | createPattern 預建條紋圖案 | Fragment shader 內建 |

| Vignette | createRadialGradient 快取 | Shader 計算（幾乎免費） |

| 色差 | compositeOperation 覆蓋 | 三次 texture2D 取樣 |

| Glow | 避免 shadowBlur（極慢） | Bloom pass（高斯模糊） |

| 全螢幕 | 預渲染離屏 Canvas | GPU 原生支援 |

常見陷阱

  1. shadowBlur 效能陷阱：Canvas 2D 的 shadowBlur 是 CPU 計算的高斯模糊，每個有 shadow 的繪製呼叫都會觸發。解法：僅對少量關鍵物件使用，或改用離屏 Canvas + filter: blur() 預渲染
  1. Scanline 密度與解析度：固定像素間距（如 y += 3）在不同 DPR 裝置上效果不一致。應乘以 scale 或 devicePixelRatio
  1. 色差過度：色差值 > 0.005 會讓文字難以閱讀。遊戲 UI/文字層應排除在色差效果之外
  1. WebGL 上下文切換：同一頁面混用 Canvas 2D 和 WebGL 會導致上下文遺失。選定一種渲染方式後保持一致
  1. 行動裝置相容性：WebGL2 在部分舊裝置不支援，CRTFilter.js 採用 WebGL2 優先 + WebGL1 回退策略

現成函式庫推薦

CRTFilter.js

  • 特色：11 種可調參數（barrel distortion、curvature、chromatic aberration、static noise、glow bloom、scanline intensity 等）
  • 整合方式：new CRTFilterWebGL(canvas); effect.start()
  • 進階功能：VHS tracking jitter、horizontal rolling、signal loss 模擬
  • GitHub：Ichiaka/CRTFilter

RetroZone

  • 特色：零依賴 WebGL 引擎，同時支援 CRT 模式（NTSC 模擬、Trinitron 孔柵、halation）與 Vector 模式（磷光衰減、bloom）
  • 整合方式：new RetroDisplay(canvas); display.setMode('crt')
  • 附加功能：3D 透視投影、向量文字渲染、發光圖形繪製工具
  • 安裝：npm install retrozone

webgl-crt-shader（gingerbeardman）

  • 特色：輕量級、針對低功耗裝置優化（iPhone XS 級別可流暢運行）
  • 整合方式：支援 Three.js 與原生 2D Canvas
  • 來源：從 Love2D shader 移植至 GLSL

與本專案的關聯

現有實作分析（Breakout v1.2）

本專案的 D:\Source\game\breakout\index.html 採用 Canvas 2D 實作 CRT 效果，包含三個核心特效：

  1. Scanlines：每 3px 繪製半透明黑線（rgba(0,0,0,0.12)）
  2. Vignette：createRadialGradient 暗角效果
  3. RGB Shimmer：globalCompositeOperation='lighter' + 時間正弦波色差

此外，磚塊使用 shadowBlur 模擬磷光發光。

優化建議

  1. Scanline 優化：可用 createPattern 預建 3px 高的條紋圖案，單次 fillRect 取代循環
  2. Phosphor Glow 替代：將 shadowBlur 改為離屏 Canvas 預渲染發光磚塊圖案
  3. 進階升級路線：若需更豐富的效果（桶形畸變、dot mask），可引入 CRTFilter.js 將 Canvas 2D 輸出傳入 WebGL 後處理管線
  4. 統一 CRT 系統：D:\Source\game 下 6 款遊戲（Pac-Man、Space Invaders、Breakout、Frogger、Tetris、Pong）可共用一套 CRT 後處理模組
