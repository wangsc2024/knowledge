Unsloth 多模態 VLM 微調與 Vision RL 完整指南

概述

Unsloth 自 2025 年末開始支援 Vision Language Model（VLM）的微調與強化學習，透過 FastVisionModel API 實現多模態模型的高效訓練。相比 Flash Attention 2 + Hugging Face 原生方案，Unsloth 達成 1.5-2x 加速、70-90% VRAM 節省、以及 4-15x 更長的上下文長度，且無精度損失。這使得在消費級 GPU（如 T4 16GB）上微調 7-11B 規模的視覺語言模型成為可能。

---

一、支援模型清單

SFT（監督微調）支援

| 模型 | 參數量 | 最低 VRAM | 備註 |

|------|--------|-----------|------|

| Llama 3.2 Vision | 11B / 90B | 12GB（11B 4bit） | Meta 原生多模態 |

| Qwen2-VL / Qwen2.5-VL | 2B / 7B | 12GB（7B 4bit） | 影像+影片理解 |

| Qwen3-VL | 8B / 32B / 235B | T4（8B 4bit） | 最新，支援影片+物件偵測 |

| Pixtral | 12B | 16GB | Mistral 視覺模型 |

| Gemma 3 | 4B | L4+（需 BF16） | Google 多模態 |

| LLaVA | 多種 | 依模型而定 | 經典 VLM |

| QvQ | 72B | 高端 GPU | 2024/12 新增 |

RL（強化學習）支援

| 模型 | 演算法 | 備註 |

|------|--------|------|

| Qwen2.5-VL-7B | GRPO / GSPO | 可在 T4 上訓練 |

| Qwen3-VL-8B | GRPO / GSPO | 免費 Colab 可跑 |

| Gemma 3-4B | GRPO | 需 L4+（BF16 限制） |

| Llama 3.2 Vision | GRPO | transforms-only，無 vLLM |

from unsloth import FastVisionModel

2.2 LoRA 配接器配置

關鍵特色：Unsloth 允許選擇性微調視覺模型的不同組件：


model = FastVisionModel.get_peft_model(
    model,
    finetune_vision_layers     = True,
    finetune_language_layers   = True,
    finetune_attention_modules = True,
    finetune_mlp_modules       = True,
    r = 16,
    lora_alpha = 16,
    lora_dropout = 0,
    bias = "none",
    random_state = 3407,
    use_rslora = False,
    loftq_config = None,
    target_modules = "all-linear",
    modules_to_save = ["lm_head", "embed_tokens"],
)

選擇性微調策略建議：

  • 僅語言層（vision=False, language=True）：適合影像理解能力已足夠，只需調整文字輸出風格
  • 僅視覺層（vision=True, language=False）：適合特定領域影像（醫學、衛星等）
  • 全部微調（預設）：新領域任務的最佳選擇

三、資料集格式

3.1 對話格式（SFT）


def convert_to_conversation(sample):
    conversation = [
        {
            "role": "user",
            "content": [
                {"type": "text",  "text": "請描述這張圖片的內容"},
                {"type": "image", "image": sample["image"]},
            ],
        },
        {
            "role": "assistant",
            "content": [
                {"type": "text", "text": sample["caption"]},
            ],
        },
    ]
    return {"messages": conversation}

from datasets import load_dataset
dataset = load_dataset("your_dataset", split="train")
converted = [convert_to_conversation(s) for s in dataset]

重要提醒：多圖訓練時使用 list comprehension 而非 .map()，影像建議 300-1000px

四、訓練配置

4.1 SFT 訓練


from unsloth.trainer import UnslothVisionDataCollator
from trl import SFTTrainer, SFTConfig

trainer = SFTTrainer(
    model=model,
    tokenizer=tokenizer,
    data_collator=UnslothVisionDataCollator(model, tokenizer),
    train_dataset=dataset,
    args=SFTConfig(
        per_device_train_batch_size=2,
        gradient_accumulation_steps=4,
        warmup_steps=5,
        max_steps=30,
        learning_rate=2e-4,
        fp16=not torch.cuda.is_bf16_supported(),
        bf16=torch.cuda.is_bf16_supported(),
        optim="adamw_8bit",
        weight_decay=0.01,
        lr_scheduler_type="linear",
        seed=3407,
        output_dir="outputs",
        remove_unused_columns=False,
        dataset_text_field="",
        dataset_kwargs={"skip_prepare_dataset": True},
        max_seq_length=2048,
    ),
)
trainer.train()

4.3 Vision GRPO（強化學習）


from trl import GRPOConfig, GRPOTrainer

def formatting_reward_func(completions, **kwargs):
    import re
    scores = []
    for completion in completions:
        score = 0
        thinking = re.findall(r"<think>.*?</think>", completion, re.DOTALL)
        answer = re.findall(r"<answer>.*?</answer>", completion, re.DOTALL)
        if len(thinking) == 1:
            score += 1.0
        if len(answer) == 1:
            score += 1.0
        removal = completion.replace("addCriterion", "").replace("\n", "")
        if len(completion) > 0 and (len(completion) - len(removal)) / len(completion) >= 0.5:
            score -= 2.0
        scores.append(score)
    return scores

# GSPO 配置
training_args = GRPOConfig(
    per_device_train_batch_size=8,
    gradient_accumulation_steps=4,
    learning_rate=5e-6,
    num_generations=8,
    max_prompt_length=1024,
    max_completion_length=1024,
    importance_sampling_level="sequence",
    loss_type="dr_grpo",
)

GRPO vs GSPO 核心差異：GRPO 逐 token 計算 logprob ratio 再乘 advantage；GSPO 先加總 token logprobs 為序列級 logprob，再指數化後乘以 advantage，更穩定。

|-----------|---------|--------------|----------|

VLM 關鍵數據

  • Qwen3-VL-8B GSPO：免費 Colab T4 可訓練
  • VLM RL：1.5-2x 加速、90% 更少 VRAM、15x 更長上下文

八、常見問題

| 問題 | 解法 |

|------|------|

| 多圖 flat list 錯誤 | 用 list comprehension 而非 .map() |

| addCriterion 亂碼 | 獎勵函數加懲罰項 |

| T4 RL 60步後發散 | Unsloth 2026/02 已修復 F16 精度問題 |

| Gemma 3 無法在 T4 跑 | 需 L4+（vLLM BF16 限制） |

| 影像尺寸不一致 | UnslothVisionDataCollator 的 resize 參數 |
