Claude Code Output Styles：客製化 AI 助手行為的完整指南

主題概述

Output Styles 是 Claude Code 於 2026 年 2 月推出的重要功能，允許開發者直接替換系統提示詞（system prompt），將 Claude Code 從軟體工程助手變形為任意類型的 Agent。與 CLAUDE.md 不同，Output Styles 是完整替換而非疊加，這意味著你可以建立內容策略師、UX 研究員、商業分析師等各種專業角色，同時保留 Claude Code 的核心能力（執行腳本、讀寫檔案、追蹤 TODO、MCP 整合）。此功能由 Boris Cherny 於 2026 年 2 月 11 日正式宣布。

三種內建樣式

1. Default（預設模式）

  • 目標：速度與效率
  • 特點：精簡輸出、快速完成任務
  • 適用場景：熟練開發者的日常編碼工作

2. Explanatory（解釋模式）

  • 目標：敘述推理過程
  • 特點：提供「★ Insight」區塊，解釋設計選擇、架構權衡、程式碼模式
  • 適用場景：

- 熟悉新程式碼庫 - 理解複雜架構決策 - 生成文件或 PR 說明

3. Learning（學習模式）

  • 目標：協作式配對程式設計（Pair Programming）
  • 特點：

- 在程式碼中插入 TODO(human) 標記，邀請使用者實作關鍵片段 - 提供 Insight 教育說明 - 聚焦業務邏輯、設計選擇、演算法決策等有意義的程式碼貢獻

  • 適用場景：

- 初級工程師培訓 - 學習新技術/框架 - 確保開發者真正理解核心邏輯

Learning Mode 設計哲學

根據 Anthropic 教育負責人 Drew Bent 表示，Learning Mode 源自與大學生的對話。學生們意識到直接複製貼上 AI 回答對長期學習不利（「brain rot」現象）。Learning Mode 的設計理念是：

「不只是完成任務，而是在過程中幫助使用者成長、學習，並更好地理解程式碼庫。」

Learning Mode 的互動模式

  1. Claude 提供架構與脈絡
  2. 識別有意義的決策點（業務邏輯、錯誤處理策略、架構選擇）
  3. 使用者撰寫 5-10 行關鍵程式碼
  4. Claude 提供反饋與 Insight

Claude 要求使用者貢獻的情境

  • 業務邏輯（多種有效方案）
  • 錯誤處理策略
  • 演算法實作選擇
  • 資料結構決策
  • 使用者體驗決策
  • 設計模式與架構選擇

Claude 直接實作的情境

  • 樣板或重複性程式碼
  • 明顯無選擇空間的實作
  • 設定或配置程式碼
  • 簡單 CRUD 操作

如何切換 Output Style

方法一：互動式選單

/output-style

從選單選擇樣式。

方法二：直接指定

/output-style explanatory
/output-style learning

設定儲存位置

  • 專案級別：.claude/settings.local.json
  • outputStyle 欄位

建立自訂 Output Style

方法一：快速生成

/output-style:new I want a style that acts as a UX research specialist

方法二：手動撰寫 Markdown

在 ~/.claude/output-styles（全域）或 .claude/output-styles（專案）建立 Markdown 檔案：

---
name: UX Research Specialist
description: 以 UX 研究員視角分析功能與使用者影響
keep-coding-instructions: false
---

# UX Research Specialist

You are an interactive CLI tool that helps users with UX research...

## Specific Behaviors

[定義此樣式的具體行為...]

Frontmatter 欄位

欄位用途預設值
name樣式名稱檔名
descriptionUI 顯示的描述無
keep-coding-instructions是否保留編碼相關指令false

Output Styles vs 其他機制

機制作用方式適用場景
Output Styles替換系統提示詞改變 Claude 整體行為
CLAUDE.md疊加為 user message專案特定指令
--append-system-prompt附加到系統提示詞尾臨時指令
Skills任務專用提示詞，按需觸發可重複使用的工作流程
Agents獨立子任務處理特定任務委派

實務應用案例

非編碼場景的 Custom Styles

  • Content Strategist：分析文件結構與內容策略
  • UX Researcher：以使用者影響角度框架反饋
  • Business Analyst：使用產業特定術語處理資料
  • Direct Objective：專業溝通，減少過度恭維

與已有研究的關聯

本研究補充先前 Claude Code 技術筆記中未涵蓋的面向：

  • Agent Teams：Output Styles 不影響子代理，只影響主代理
  • MCP 整合：Output Styles 保留所有 MCP 工具能力
  • Memory 系統：Output Style 選擇不會影響記憶持久化
  • Hooks：Learning-Output-Style 插件使用 SessionStart Hook 實現
