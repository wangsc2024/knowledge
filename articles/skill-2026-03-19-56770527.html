系統性發現

1. v0.5.0 版本標記一致性問題

pre-flight-check 的 version 是 0.5.0，但 description 缺少警告標記。這違反了 skill-forge 自己定義的版本標記規範（步驟 4）：

v0.5.0：description 末尾加「⚠️ 知識基礎薄弱，建議透過 skill-audit 補強」

建議：執行一次全系統掃描，檢查所有 v0.5.0 或 v0.1.0 的 Skill 是否都有對應的警告標記。

2. 可執行性缺陷模式

pre-flight-check 的風險評分公式用偽代碼描述（步驟 2-3），這是一個常見的 Skill 品質問題：

  • 反模式：用自然語言或偽代碼描述邏輯，期待 Agent 自行實作
  • 正確模式：直接提供可複製貼上的 Bash/Python 腳本

建議：在 prompt-lint Skill 中新增檢測規則，偵測「if ... elif ... else」等偽代碼結構，標記為「缺乏可執行腳本」。

3. 內容重複偵測

skill-forge 的步驟 10b 重複內容問題，顯示需要一個自動化工具偵測 SKILL.md 內的重複段落（≥5 行相同內容）。

建議：在 skill-scanner 或 skill-test-scaffolder 中新增「重複內容偵測」功能。

  2. prompt-lint 增強：新增「偽代碼偵測」規則，要求關鍵邏輯必須有可執行腳本
