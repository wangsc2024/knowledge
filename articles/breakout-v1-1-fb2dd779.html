Breakout v1.1 優化心得

優化概要

將打磚塊遊戲從 v1.0 升級至 v1.1，聚焦 UX 提升，新增三項功能。

新增功能

1. 首次遊玩互動教學系統

  • 實作方式：新增 STATE.TUTORIAL 狀態，3 步驟引導（移動接球板→發射球→接住道具）
  • 動畫效果：每步附帶動態示意（球拍左右移動、球上升軌跡、三種道具浮動）
  • 進度指示：頂部 3 個圓點標示當前步驟
  • 記憶機制：localStorage 記錄教學完成狀態，僅首次顯示
  • 觸發流程：MENU → 點擊 → TUTORIAL(3步) → 自動進入 PLAYING

2. 觸控暫停按鈕

  • 尺寸：按鈕 28px，觸控區域 44×44px（符合 WCAG 觸控目標規範）
  • 位置：右上角，不遮擋 HUD 資訊（生命值左移避開）
  • 雙圖示：暫停時顯示 ▶（播放），遊戲中顯示 ‖（暫停）
  • 互動：滑鼠點擊和觸控皆可用，hit test 使用圓心擴展區域判定

3. Game Over 統計面板

  • 圓角面板：使用 arcTo 手動繪製圓角矩形（避免 roundRect 相容性問題）
  • 5 項統計：磚塊擊破數、最高 Combo、道具收集數、通過關卡數、遊戲時長
  • 配色：每項統計使用對應主題色（綠色=磚塊、彩虹=Combo、粉色=道具、金色=關卡、藍色=時間）

技術要點

狀態機擴展

  • 從 5 狀態擴展為 6 狀態：MENU(0)→TUTORIAL(5)→PLAYING(1)→PAUSED(2)→LEVELCLEAR(4)→GAMEOVER(3)
  • Tutorial 狀態有獨立的 timer 和 step 追蹤

觸控輸入優化

  • 暫停按鈕 hit test 在 touchstart 事件中優先判定（防止穿透到遊戲操作）
  • isInPauseBtn() 函數統一處理滑鼠和觸控判定

修正的 Bug

  1. levelsCleared 雙重遞增：同時在 nextLevel() 和 level clear check 中遞增，移除 nextLevel 中的重複
  2. roundRect 相容性：Canvas roundRect API 在舊瀏覽器不支援，改用 arcTo 手動繪製

品質基線

  • requestAnimationFrame 固定時間步長（1000/60ms）
  • AABB + 圓形碰撞偵測
  • 物件池粒子系統（預分配 120 個）
  • Web Audio API 8-bit 音效合成
  • localStorage 高分和教學狀態持久化
