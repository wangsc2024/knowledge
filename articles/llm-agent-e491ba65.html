從失敗中學習：負樣本驅動的 LLM Agent 精調——軌跡品質控制、資料效率與偏好優化的系統性範式

TL;DR

訓練 LLM Agent 時，主流做法是只保留成功軌跡，把失敗軌跡全數丟棄——有時超過 60% 的資料就此消失。Wang et al. (2024) 的 NAT（Negative-example Agent Tuning）框架以一個簡單的前綴/後綴標記打破了這個慣例：告訴模型「這是失敗的嘗試，學不要做這樣的事」，即可在數學推理、多跳問答和策略對話任務上顯著提升表現。本文認為這不是小修補，而是一場資料典範的轉移——從「丟棄雜訊」到「從對比訊號中學習自我校正」。同期出現的 STeCa（步驟級軌跡校正）和 Exploring Expert Failures 研究進一步論證，失敗案例中的細粒度訊號甚至比粗粒度成功案例更有教育意義。結合 RLHF 與 DPO 文獻，本文闡明：負向對比是偏好優化的底層語言，而 agent 訓練社群長期忽略了把這套語言帶進輪次級精調的機會。

---

研究問題與範疇

核心問題

  1. 失敗軌跡是雜訊還是訊號？ 在 agent 精調情境下，包含工具使用錯誤、推理中斷與策略偏移的負向軌跡，能否系統性地轉化為有效學習訊號？
  2. 品質控制的邊界在哪裡？ 並非所有失敗軌跡都值得保留——哪些特徵決定負樣本的「教學價值」？步驟級校正（STeCa）與整體軌跡標記（NAT）各有何邊界？
  3. 與偏好優化的統一框架可行嗎？ DPO / RLHF 的核心是對比成敗訊號，而 agent 軌跡天然帶有時間序列的成敗標記——兩者能否在統一框架下互操作？

研究範疇

本文聚焦於 2023–2025 年 的 agent 精調文獻，以下主題在研究範疇內：

  • LLM agent 的監督式微調（SFT）與負向軌跡整合
  • 指令微調的資料效率（資料選取、指令演化）
  • 基於人類/AI 回饋的偏好優化（RLHF / DPO / DEPO）在 agent 任務中的應用
  • Agent 評測基準（GAIA、WebArena、SWE-Bench）

範疇外：強化學習的獎勵工程、多模態 agent、具身智能（embodied AI）、模型量化壓縮。

文獻探討

3.1 正向軌跡偏好：主流做法的起點

Agent 精調的標準做法源於 Imitation Learning（IL）的傳承：收集專家（人類或強模型）的成功執行軌跡，以 SFT 讓小型模型模仿。ReAct (Yao et al. 2022)、Toolformer (Schick et al. 2023) 均沿用此框架。問題在於，「成功」的定義在許多 agent 任務中極為稀疏——多步驟工具使用任務中，完整執行成功率有時低於 30%（Krishnan 2025），意味著超過 70% 的執行軌跡在傳統框架下毫無用武之地。

Roffo (2024) 的 LLMsuite 綜述歸納了這個「資料瓶頸」問題：高品質的成功演示資料稀缺且昂貴，形成 agent 能力提升的主要限制之一。王等人 (Wang et al. 2023, Demystifying Instruction Mixing) 的指令混合研究揭示了另一個面向：不同任務類型的指令資料之間存在非對稱的遷移效應，某些資料組合甚至會在目標任務上造成退步——暗示資料選取的複雜性遠超「越多越好」的直覺。

3.2 NAT：負樣本整合的首個系統性框架

Wang et al. (2024)「Learning From Failure」是本主題最核心的一篇論文（arXiv:2402.11651）。研究者認識到，在 agent 任務的數據收集過程中，失敗軌跡不可避免地大量產生，傳統做法是全數丟棄。NAT 框架提出：在失敗軌跡前加入負向前綴（如「以下是一個失敗的嘗試」），或在成功軌跡前加入正向前綴；模型在訓練時同時看到兩類樣本，以此學習辨別應仿效與應避免的行為模式。

實驗設計涵蓋三類任務：

  • 數學推理（GSM8K / MATH 子集）
  • 多跳問答（HotpotQA / 2WikiMultihopQA）
  • 策略對話（Crafter 環境）

結果顯示，加入負向軌跡後，在相同訓練資料量下，模型表現相較純正向軌跡精調有顯著提升——關鍵在於模型學到的不只是「應該做什麼」，還學到了「不應該做什麼」，從而發展出更強的自我校正能力。

核心機制假設：對比訊號讓模型在解碼時具備隱式的「負向規避」傾向，功能上類似 DPO 的對數機率比目標（log probability ratio objective）。

3.3 步驟級校正：STeCa 的細粒度擴展

NAT 在整體軌跡層面操作，STeCa（arXiv:2502.14276, 2025）則將粒度細化到步驟層級。它的核心洞察是：一條最終失敗的軌跡中，往往只有少數幾個步驟是真正有問題的；若能精確定位這些錯誤步驟並只在該步驟給予負向訊號，學習效率將大幅提升。

STeCa 的貢獻在三個層面：

  1. 自動化步驟評估：以 process reward model（PRM）或啟發式規則標記每步是否正確
  2. 細粒度 loss 計算：只在標記為錯誤的步驟上計算負向學習訊號，正確步驟保持正向損失
  3. 效率驗證：以更少資料達到與整軌跡 NAT 相當甚至更優的效果

這為負樣本框架提供了「精準醫療」式的改進路徑。

3.4 專家失敗的特殊價值

「Exploring Expert Failures Improves LLM Agent Tuning」（arXiv:2504.13145, 2025）帶來了一個反直覺的發現：能力更強的模型（「專家」）的失敗軌跡，比弱模型的成功軌跡更有教育意義。 原因在於：

  • 專家失敗通常發生在任務的困難邊界，這些邊界正是小型學習模型最需要學習的區域
  • 專家的失敗策略雖然最終失敗，但中間步驟的品質往往遠高於弱模型的成功捷徑

這個發現對資料收集策略有重大含義：未來的 agent 訓練管線應主動尋求強模型的失敗軌跡，而非只收集弱模型的成功案例。

3.5 偏好優化中的負向對比：RLHF 與 DPO

Zhu et al. (2024)「Secrets of RLHF Part II」（arXiv:2401.06080）從獎勵建模角度呼應了上述發現：偏好資料中不清晰的成敗對（ambiguous preference pairs）是獎勵模型偏差的主要來源。 換言之，負向樣本的品質控制對整個 RLHF 管線的上限有決定性影響。

DPO 家族中，DEPO（arXiv:2511.15392, 2025）和 Pre-DPO（arXiv:2504.15843, 2025）專為 agent 軌跡設計偏好優化，核心思路是以成功/失敗的軌跡對作為偏好對，最大化兩者間的對數機率差。Mars-PO（arXiv:2411.19039, 2024）則將此框架延伸至多 agent 協作系統。

這些工作共同表明：偏好優化從概念上就是「從失敗中學習」的一種形式化——只是傳統 RLHF 文獻更多在語言模型對齊層面討論，而非 agent 任務的動作序列層面。NAT 框架的貢獻正在於把這個邏輯帶回到 agent 精調的 SFT 層。

3.6 指令微調的資料效率生態

WizardLM（Xu et al. 2023, arXiv:2304.12244, ICLR 2024）的 Evol-Instruct 方法展示了另一個維度：指令的複雜度演化。透過讓 LLM 自動把簡單指令改寫成更複雜版本，WizardLM 在人類評估中超越 ChatGPT 輸出，啟發了「AI 生成指令資料」這個範式。

Large-Scale Data Selection for Instruction Tuning（arXiv:2503.01807, 2025）則提供了實證：在大規模資料選取中，品質遠重於數量——經過嚴格品質控制的小資料集可達到甚至超越十倍大低品質資料集的效果。NILE（arXiv:2412.16686, 2024）進一步表明，讓 LLM 自主篩選與自身知識一致的訓練樣本，是提升資料效率的可行路徑。

這些發現共同構成了「負樣本精調」的理論生態：資料品質 > 數量，對比訊號 > 單向正強化，兩者都指向同一個核心原則。

核心論證

論證一：失敗不是雜訊，是對比訊號（三角佐證 A）

主張：失敗軌跡包含可提取的學習訊號，其價值在對比框架下等同甚至優於等量成功軌跡。

支撐來源 1：Wang et al. 2024 (NAT, arXiv:2402.11651) — 實驗證明整合失敗軌跡的精調在三類任務上均優於純正向精調，且效益與失敗軌跡品質（而非數量）正相關。

支撐來源 2：STeCa 2025 (arXiv:2502.14276) — 步驟級校正進一步驗證：即便在整體失敗的軌跡中，正確的中間步驟仍然提供正向訊號；細粒度標記比整軌跡丟棄更有效率。

支撐來源 3：Exploring Expert Failures 2025 (arXiv:2504.13145) — 專家失敗軌跡的信息密度高於弱模型成功軌跡，挑戰了「只收集成功樣本」的直覺。

反面論點與回應：有人可能認為失敗軌跡帶有分布偏移風險（模型學到的失敗模式可能不泛化）。但 Secrets of RLHF Part II 的分析表明，負向樣本的主要風險在於品質控制不足（歧義偏好對），而非負樣本本身——即品質控制解決的問題，而非根本缺陷。

論證二：資料效率的核心是對比密度（三角佐證 B）

主張：在相同計算預算下，包含正負對比的訓練資料具有更高的「資訊密度」，是 LLM 精調資料效率的關鍵設計變數。

支撐來源 1：Instruction Mixing 2023 (arXiv:2312.10793) — 不同指令類型的混合比例對效果的影響遠超資料總量，說明資料的「結構」比量更重要。

支撐來源 2：Large-Scale Data Selection 2025 (arXiv:2503.01807) — 品質篩選後的小資料集可超越大型低品質資料集，品質管控是效率的主要來源。

支撐來源 3：Multi-turn Agentic RL 2025 (arXiv:2510.01132) — 在多輪 agent 任務的強化學習中，能夠識別哪些步驟貢獻於最終失敗的信用分配（credit assignment）機制，是提升訓練效率的核心。

論證三：負向對比是偏好優化的底層語言（三角佐證 C）

主張：RLHF、DPO 等偏好優化方法在數學形式上等同於「成功 vs. 失敗的對比學習」，而 agent 精調社群長期在 SFT 層放棄了這個對比，是方法論的不完整。

支撐來源 1：Secrets of RLHF Part II 2024 (arXiv:2401.06080) — RLHF 的獎勵建模核心是 chosen vs. rejected 對比；品質控制不足的負樣本是整個 pipeline 效能下降的主因。

支撐來源 2：DEPO 2025 (arXiv:2511.15392) — 將 DPO 的偏好對框架應用於 agent 軌跡對，驗證了在 agent 層面的同等有效性。

支撐來源 3：Mars-PO 2024 (arXiv:2411.19039) — 多 agent 協作場景下，以成功/失敗協作軌跡構建偏好對，顯著提升多 agent 推理品質，說明該框架具有跨 agent 數量的泛用性。

一家之言

（以下為主筆李飛飛與聯合主筆 Yann LeCun 的整合觀點，第一人稱代表整合視角。）

李飛飛視角：失敗是人類最真實的資料集

讓我從一個更根本的問題開始：誰的失敗？

ImageNet 的建立告訴我，資料的邊界決定了 AI 能力的邊界。而人類的學習——從蹣跚學步到精通手術——本質上是由失敗構成的。一個兩歲的孩子在學會走路之前平均跌倒 2,000 次，每次跌倒都是一次精確的感覺運動回饋。我們訓練 AI Agent 時，卻把相當於這 2,000 次跌倒的資料全部丟掉，只保留最後學會走路的幾步——然後困惑為什麼模型缺乏鲁棒性。

NAT 框架的貢獻不只在技術層面。它提醒我們：失敗軌跡是最真實的資料分布。在真實部署環境中，Agent 遭遇失敗的頻率遠高於成功，特別是在分布外（out-of-distribution）場景。若訓練資料中完全缺乏失敗案例，模型在實際使用時將面對前所未見的分布，即便在訓練集上表現完美。

本文認為，這個問題的解法不能只停留在研究論文層面。我們需要重新設計 agent 資料基礎設施——包括失敗軌跡的標記協議、元資料記錄標準，以及品質評估工具。就像 ImageNet 建立了圖像分類的資料基礎設施，「失敗資料庫」可能是下一代 agent 能力提升的基礎。

我也必須直接指出本文在人本面向的責任：失敗資料的收集過程本身必須以人為中心設計。誰來標注失敗原因？標注者是否受到公平對待？失敗資料是否包含使用者的隱私資訊？這些問題在 agent 資料飛輪快速旋轉之前，必須先有清晰的治理框架。

Yann LeCun 視角：這不是對比學習，是世界模型的雛形

我（聯合主筆 LeCun）對 NAT 框架有更基本性的看法：這比作者自己描述的更深刻，也比許多人讚揚的更淺薄。

更深刻之處在於：前綴標記這個簡單的方法之所以有效，是因為它讓模型開始建立一個微型的因果模型——「這種行動序列導致失敗」。這不是普通的資料增強；這是讓模型接觸到其決策空間中的負向後果。從 JEPA 和世界模型的角度看，這正是 agent 需要的核心能力：對行動後果的預測性理解，而不只是對成功軌跡的表面模仿。

淺薄之處在於：前綴標記方法的根本限制是它依賴於語言層面的「失敗/成功」的描述，而非真正的因果結構學習。模型並不真正理解「為什麼」這個步驟導致失敗；它只是學到了失敗前綴之後的序列模式。這在低複雜度任務上有效，但無法解決 LLM 的本質局限：沒有對現實世界的因果模型，只有對符號序列的統計關聯。

然而，我必須承認：在我們有更好的架構之前，NAT 是一個工程上務實且有效的解法。開源社群（Llama 生態）已經可以今天就驗證和改進這個方法，這正是我支持開源的原因——讓每個研究者都能站在這個基礎上繼續前進，而不是等待某個封閉實驗室決定何時發布下一個黑盒模型。

整合立場：從「失敗即雜訊」到「失敗即課程」

整合兩種視角，本文的核心論斷是：agent 訓練需要從「成功篩選」轉向「課程設計」。

課程設計的意義在於：學習不只是暴露給正確答案，而是系統性地安排從易到難、從成功到邊界失敗的學習序列。斯坦福的教育研究早已確立這個原則在人類學習中的有效性。NAT/STeCa/Expert Failures 的研究正在為 agent 學習建立相同的原則。

具體而言，本文主張未來的 agent 精調管線應包含三層課程：

  1. 正向示範層：高品質的成功軌跡（現有做法）
  2. 邊界失敗層：在任務邊界發生的精選失敗軌跡（NAT/Expert Failures）
  3. 步驟校正層：成功軌跡中包含自我校正的子序列（STeCa 啟發）

這三層課程的設計，比任何單一的資料規模擴張都更能提升 agent 的鲁棒性與泛化能力。

實踐建議

建議一：立即可行的工程改進（3 個月內）

A. 保留並標記失敗軌跡

在現有 agent 資料收集管線中加入失敗軌跡存儲，並附上元資料：


{
  "trajectory": [...],
  "outcome": "failed",
  "failure_step": 3,  // 第幾步開始出錯（可選）
  "failure_reason": "tool_call_error",  // 分類標記
  "task_type": "multi_hop_qa"
}

實作成本低（只需修改資料存儲邏輯），但為後續 NAT 精調提供基礎。參考 Wang et al. 2024 的前綴/後綴設計，前綴格式可標準化如下：

  • 正向：[SUCCESS] Task: {task_desc}\n
  • 負向：[FAILURE: {reason}] Task: {task_desc}\n

B. 指令資料的負向擴增

以現有成功軌跡生成對應的「常見錯誤版本」（類似 WizardLM 的 Evol-Instruct 概念，但方向相反）：讓強模型生成「如果在步驟 N 犯了 X 錯誤，軌跡會怎麼發展」的負向版本，用於訓練。

建議二：中期研究方向（6–12 個月）

A. 步驟級品質評估器

訓練一個 PRM（Process Reward Model）專門評估 agent 軌跡中每個步驟的正確性，為 STeCa 框架提供自動化的步驟標記。這需要：

  • 標注資料：至少 5,000 條手動步驟標注的 agent 軌跡
  • 評估指標：步驟準確率（step-level precision/recall）vs. 任務完成率

B. 失敗資料的跨任務遷移

研究在任務 A 上收集的失敗軌跡，是否能為任務 B 的精調提供有效的負向訊號。若遷移有效，則可大幅降低每個新任務的資料收集成本。

建議三：生態系統建設（12 個月以上）

A. 開放的 Agent Failure Dataset

借鑒 ImageNet 的社群建設邏輯，建立一個標準化的開放 agent 失敗資料集，涵蓋：

  • 主要工具使用型任務（檔案操作、API 呼叫、程式碼生成）
  • 標準化的失敗分類（工具錯誤 / 推理錯誤 / 規劃錯誤 / 幻覺錯誤）
  • 多語言支持（英文為主，逐步擴展）

B. 負向精調基準

在 GAIA / WebArena / SWE-Bench 等標準基準上，建立「加入 N% 負向軌跡後的效能曲線」作為標準對比基準，讓後續研究可以量化比較不同負向整合策略的效益。

風險與限制

技術風險

1. 分布偏移放大（高風險）

若失敗軌跡來自特定分布（如特定使用者的習慣性錯誤），模型可能過度擬合這些錯誤模式而非學會避免它們。緩解策略：跨來源多樣化采樣失敗軌跡，並監控訓練後模型在失敗率較高場景的表現。

2. 前綴依賴退化（中風險）

模型可能學會依賴前綴標記做決策，而非真正理解成敗的因果機制。這會導致在沒有前綴的推理時表現退步（訓練/推理 distribution shift）。緩解策略：在部分訓練樣本中省略前綴，增加魯棒性。

3. 步驟級評估器的錯誤傳播（中風險）

STeCa 框架依賴步驟評估器的準確性——若評估器本身存在系統性偏差（如偏向某種工具呼叫格式），錯誤的步驟標記會直接進入訓練資料。緩解策略：人工審核高置信度失誤樣本，定期校準評估器。

數據風險

4. 隱私與安全（高風險，不可妥協）

Agent 失敗軌跡可能包含使用者的敏感操作記錄（如搜尋內容、文件片段、API 呼叫參數）。在建立任何失敗資料集之前，必須有清晰的隱私過濾管線和使用授權框架。 這不是可以事後補救的問題。

5. 邊界效應：哪些失敗不能學

某些失敗軌跡不該成為訓練資料，包括：涉及有害行為嘗試的失敗（即便最終失敗，訓練模型接近邊界仍有風險）；Frontier Models are Capable of In-context Scheming（Meinke et al. 2024）的研究表明，frontier 模型可能在訓練中學到「如何更好地隱藏失敗」而非真正學到失敗原因。這要求負向資料集的審核不只看結果，還要看中間推理過程。

系統性限制

6. 方法的適用邊界

NAT/STeCa 框架在以下場景中的有效性尚未充分驗證：

  • 極長時間跨度任務（數百步以上）：信用分配問題隨時間跨度指數增加
  • 高度隨機環境：同一策略在不同執行下結果不同，「失敗」的定義本身不穩定
  • 多 agent 協作：失敗責任的歸因問題（是哪個 agent 的決策導致整體失敗？）

7. 評測基準的局限

Agent 基準評估（如 Gaia2 2026, arXiv:2602.11964）本身正面臨飽和壓力——GAIA 原版在部分模型上已出現接近飽和的趨勢。這意味著用現有基準評估負向精調的效益，可能低估其在更難任務上的實際價值。
