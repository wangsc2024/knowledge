workflow-forge: Hook 規則更新驗證 workflow（2026-03-28）

執行摘要

  • 產出類型：validation_workflow（驗證工作流 YAML）
  • 產出路徑：workflows/hook-rules-update-validation-workflow.yaml
  • 優先級：P1
  • 總分：90/100

缺口分析

問題描述

config/hook-rules.yaml 更新或 hooks/*.py 新增/修改規則時，缺少標準化驗證流程確保：

  1. YAML 語法正確：修改後可被 YAML parser 正確解析
  2. 規則 pattern 可編譯：所有正規表達式 pattern 可被 re.compile() 編譯
  3. 現有測試仍通過：529 個 Hook 測試仍全部通過
  4. Schema 一致性：YAML 結構與 hook_utils.py 的 load_hook_rules() 期望一致
  5. Preset 一致性：presets 引用的 rule_id 都存在於 rules[] 中

影響範圍

  • 5 個 Hooks：pre_bash_guard, pre_write_guard, pre_read_guard, post_tool_logger, on_stop_alert
  • 所有 Agent 執行：Hook 規則錯誤會導致系統性攔截失敗或誤攔
  • 529 個測試：修改規則可能破壞現有測試

信號來源

  1. config/hook-rules.yaml 存在（v3.0.0），20 條規則，3 個 preset
  2. tests/hooks/ 有 529 個測試，但缺少「規則更新後執行測試」的 workflow 定義
  3. workflows/index.yaml 中無 Hook 相關 workflow（缺口）
  4. docs/OPERATIONS.md 提及 Hook 規則但未定義驗證 workflow

Workflow 規格

觸發條件

  • 檔案：config/hook-rules.yaml 或 hooks/*.py 修改/建立
  • 模式：手動執行 或 整合至 git pre-commit hook / CI pipeline

驗證步驟（6 步驟）

  1. YAML 語法驗證（critical）

- 命令：uv run python -c "import yaml; yaml.safe_load(open('config/hook-rules.yaml'))"

- 成功條件：Exit code 0 且無 SyntaxError

- 失敗處理：中止流程，報告語法錯誤行號

  1. Pattern 編譯驗證（critical）

- 命令：用 Python 讀取所有 rules[].pattern，用 re.compile() 驗證

- 成功條件：所有 pattern 可編譯為正規表達式

- 失敗處理：報告無法編譯的 pattern，中止流程

  1. Hook 測試套件執行（critical）

- 命令：uv run pytest tests/hooks/ -v --tb=short

- 成功條件：所有測試通過（529 passed）

- 失敗處理：報告失敗測試名稱與數量，中止流程

- Fallback：若 tests/hooks/ 不存在，跳過此步驟並記錄警告

  1. 規則 Schema 一致性檢查（critical）

- 方法：確認 YAML 的 rules[] 每項含 id, pattern, action, message, severity

- 成功條件：Schema 符合 hook_utils.py 期望

- 失敗處理：報告 schema 不匹配欄位，中止流程

  1. Preset 一致性檢查（warning）

- 方法：比對 presets[].enabled_rules 與 rules[].id

- 成功條件：所有 enabled_rules 引用的 rule_id 都存在

- 失敗處理：報告孤立 rule_id，警告但不中止

  1. 更新文件（info，可選）

- 方法：提示更新 docs/OPERATIONS.md 的 Hook 規則段落

- 成功條件：文件含新規則說明或標記「文件待更新」

- 失敗處理：記錄警告，不阻止流程

整合選項

Git Pre-commit Hook

將步驟 1-3 加入 .git/hooks/pre-commit，自動驗證：


#!/bin/bash
uv run python -c "import yaml; yaml.safe_load(open('config/hook-rules.yaml'))"
uv run pytest tests/hooks/ -q

check-health.ps1 整合

新增 [Hook 規則健康度] 區塊：


Write-Host "[Hook 規則健康度]" -ForegroundColor Cyan
pwsh -Command "& { uv run python -c 'import yaml; yaml.safe_load(open("config/hook-rules.yaml"))' }"
pwsh -Command "& { uv run pytest tests/hooks/ -q --tb=no }"

CI Pipeline 整合

建立 .github/workflows/validate-hooks.yml：


name: Validate Hook Rules
on:
  pull_request:
    paths:
      - 'config/hook-rules.yaml'
      - 'hooks/*.py'
jobs:
  validate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Run validation
        run: |
          uv run python -c "import yaml; yaml.safe_load(open('config/hook-rules.yaml'))"
          uv run pytest tests/hooks/ -v
