Daily-Digest-Prompt 系統熵增優化 Phase 2（2026-03-09）

系統優化熵增治理品質量化pytestState管理架構決策

系統熵增優化 Phase 2 — State 膨脹防護與品質量化

優化背景

深度分析發現專案面臨三大熵增問題：

  1. State 檔案膨脹：166 個 loop-state + 13 個 stop-alert 殭屍檔案（612KB），無自動清理
  2. Context 膨脹：behavior-patterns.json 佔 context/ 目錄 67%（248KB，500 patterns）
  3. 測試覆蓋率無量化：682 個測試但無覆蓋率報告，無法追蹤品質趨勢

實施項目

1. State 自動清理機制

修改檔案：run-agent-team.ps1、run-todoist-agent-team.ps1

在兩個 team 腳本的清理區段新增三項清理邏輯：

  • loop-state-*.json：>48 小時自動刪除（每 session 產生一個，~83 個/天）
  • stop-alert-*.json：>7 天自動刪除（session 結束告警去重用）
  • results/*（排除 spans）：>7 天自動刪除（防 40MB/4天 → 300MB/月膨脹）

預期效果：state/ 目錄從 612KB 控制在 ~100KB 以內

2. behavior-patterns.json 瘦身

修改檔案：hooks/behavior_tracker.py

  • MAX_PATTERNS：500 → 200（每個 pattern ~500 bytes，200 個 ≈ 100KB）
  • DECAY_DAYS：30 → 14（加速淘汰低頻模式）
  • 一次性清理：248KB → 98KB（-60%，移除 300 個低信心 patterns）

設計決策：200 個 patterns 足以追蹤日常 Agent 行為模式（每日新增 ~10-20 個新 signature），14 天衰減確保資料新鮮度

3. pytest 覆蓋率量化

修改檔案：pyproject.toml

pytest addopts 新增：

  • --cov=hooks --cov=skills：量測 hooks 和 skills 模組覆蓋率
  • --cov-report=term-missing：終端顯示未覆蓋行號
  • --cov-fail-under=40：基線門檻（當前 44.86%），防止覆蓋率回退

覆蓋率現狀（682 tests, 3183 statements）：

  • agent_guardian: 83% | behavior_tracker: 83%
  • pre_read_guard: 80% | pre_write_guard: 79%
  • hook_utils: 75% | query_logs: 77%
  • pre_bash_guard: 62% | cjk_guard: 35%
  • on_stop_alert: 45% | post_tool_logger: 42%
  • validate_config: 12%（最大改進空間）

驗證結果

  • 682 個測試全部通過（0 failures）
  • 覆蓋率 44.86% > 40% 門檻（通過）
  • behavior-patterns.json：248KB → 98KB
  • 無非預期的檔案變更

後續建議

  1. validate_config.py 測試補強（12% → 50%+）
  2. 覆蓋率門檻逐步提升（40 → 50 → 60 → 70）
  3. Tools 模組測試覆蓋（目前 0%，10 個 Python 檔案）
  4. GitHub Actions CI 整合覆蓋率檢查
