一、專案概述

OpenClaw（openclaw/openclaw）是一款開源個人 AI 助手，定位為「在你自己的裝置上運行的 AI 助手」。專案於 2025 年 11 月 24 日在 GitHub 建立，採用 MIT 授權，截至 2026 年 3 月已累積 245,576 Stars、47,436 Forks，是 2025-2026 年間增長最快的開源專案之一，在個人 AI 助手類別中 Stars 數遙遙領先（第二名 Khoj 僅 32,759 Stars）。

OpenClaw 經歷多次更名：Warelay -> Clawdbot -> Moltbot -> OpenClaw。最初是作者（steipete）的個人實驗專案，目標是「建造一個能在真實電腦上執行真實任務的 AI 助手」。

二、核心架構

2.1 Gateway 控制平面

OpenClaw 的核心是一個 WebSocket 控制平面（Gateway），運行在本地 ws://127.0.0.1:18789，統一管理所有 sessions、channels、tools 和 events。


WhatsApp / Telegram / Slack / Discord / Signal / iMessage / ...
               |
               v
+-------------------------------+
|          Gateway              |
|     (control plane)           |
|   ws://127.0.0.1:18789       |
+-------------------------------+
               |
               +-- Pi agent (RPC)
               +-- CLI (openclaw ...)
               +-- WebChat UI
               +-- macOS app
               +-- iOS / Android nodes

設計特點：

  • 單一控制平面：所有通訊頻道、工具、事件都經由 Gateway 統一路由
  • RPC 模式：Pi agent 以 RPC 方式運行，支援 tool streaming 和 block streaming
  • Session 模型：main session 用於直接聊天，群組隔離，支援 activation modes 和 queue modes
  • Media Pipeline：統一的圖片/音訊/影片處理管線，含轉錄 hooks 和大小上限

2.2 多頻道收件匣（22+ 頻道）

OpenClaw 支援的通訊頻道數量令人印象深刻：

核心頻道（內建）：WhatsApp（Baileys）、Telegram（grammY）、Slack（Bolt）、Discord（discord.js）、Google Chat（Chat API）、Signal（signal-cli）、BlueBubbles（iMessage）、IRC

擴充頻道（Extension）：Microsoft Teams、Matrix、Feishu（飛書）、LINE、Mattermost、Nextcloud Talk、Nostr、Synology Chat、Tlon、Twitch、Zalo、WebChat

原生 App：macOS 選單列 app、iOS node、Android node

2.3 Multi-Agent 路由

Gateway 支援將不同的 inbound channels/accounts/peers 路由到隔離的 agents：

  • 每個 agent 有獨立的 workspace 和 per-agent sessions
  • Agent-to-Agent 通訊透過 sessions_list、sessions_history、sessions_send 工具
  • 支援 reply-back ping-pong 和 announce 步驟

三、核心特色

3.1 Local-First 隱私設計

  • 本地優先：Gateway 預設綁定 loopback，所有資料留在使用者裝置上
  • DM 配對安全：預設 dmPolicy="pairing"，未知發送者需輸入配對碼才能與 bot 互動
  • Tailscale 整合：支援 Serve（tailnet-only）和 Funnel（public）模式，Gateway 仍綁定 loopback
  • 機密管理：v2026.2.26 引入完整的 openclaw secrets 工作流（audit/configure/apply/reload）

3.2 Voice Wake + Talk Mode

  • Wake Words：macOS/iOS 支援語音喚醒
  • Talk Mode：Android 支援連續語音對話
  • TTS：ElevenLabs + 系統 TTS fallback
  • 媒體理解：圖片/音訊/影片自動轉錄和理解

3.3 Live Canvas + A2UI

  • Canvas：Agent 驅動的視覺化工作空間
  • A2UI（Agent-to-UI）：Agent 可直接推送/重置 Canvas 內容，支援 eval 和 snapshot
  • 跨平台：macOS、iOS、Android 都支援 Canvas

3.4 Skills 平台 + ClawHub

  • 三層 Skills：bundled（內建）、managed（託管）、workspace（工作區）
  • ClawHub：Skills 登錄平台（clawhub.com），Agent 可自動搜尋和安裝新 Skills
  • Plugin API：完整的插件 API，npm 套件分發 + 本地擴充載入

3.5 瀏覽器控制

  • 專用的 OpenClaw Chrome/Chromium 實例
  • CDP（Chrome DevTools Protocol）控制
  • Snapshots、Actions、Uploads、Profiles 管理

3.6 自動化能力

  • Cron Jobs：定時排程任務
  • Webhooks：外部事件觸發
  • Gmail Pub/Sub：郵件事件監聽
  • Auto-Reply：智慧自動回覆（v2026.2.24 擴充 stop phrases）

四、模型支援與 Failover

  • 多模型提供者：OpenAI、Anthropic、Google、AWS Bedrock
  • 推薦模型：Anthropic Pro/Max（100/200）+ Opus 4.6，因長上下文能力和 prompt injection 抵抗力較佳
  • Auth Profile 輪替：支援 OAuth vs API keys 切換
  • Model Failover：自動降級到備用模型
  • Thinking Level：off/minimal/low/medium/high/xhigh（v2026.3.1 預設 Claude 4.6 為 adaptive）

五、開發者體驗

5.1 CLI-First 設計


# 安裝
npm install -g openclaw@latest

# 引導設定精靈
openclaw onboard --install-daemon

# 啟動 Gateway
openclaw gateway --port 18789 --verbose

# 發送訊息
openclaw message send --to +1234567890 --message "Hello"

# 與助手對話
openclaw agent --message "Ship checklist" --thinking high

# 健康檢查
openclaw doctor

5.2 開發通道

  • stable：tagged releases（vYYYY.M.D），npm dist-tag latest
  • beta：prerelease（vYYYY.M.D-beta.N），npm dist-tag beta
  • dev：main 分支最新，npm dist-tag dev

5.3 測試基礎設施

  • Vitest 框架，V8 coverage
  • 門檻：70% lines/branches/functions/statements
  • 測試分類：unit（.test.ts）、e2e（.e2e.test.ts）、live（真實 API keys）、Docker live
  • 多 Agent 安全：git stash/worktree/branch 操作均有保護機制

七、對 Daily-Digest-Prompt 的借鑑意義

  1. Gateway 模式：OpenClaw 的 WebSocket 控制平面概念可啟發 daily-digest 的通知管道統一化（目前僅 ntfy，未來可擴充）
  2. Skills 平台：ClawHub 的 Skills 分發模式與本專案的 SKILL_INDEX.md 異曲同工，但 ClawHub 更進一步實現了 Skills 的自動發現與安裝
  3. Multi-Agent 路由：OpenClaw 的 session 隔離與 Agent-to-Agent 通訊，與本專案的 Team 並行模式有相似設計理念
  4. 版本命名：vYYYY.M.D 的日曆版本號是值得借鑑的簡潔方案
  5. Doctor 診斷工具：openclaw doctor 的自我診斷概念與本專案的 check-health.ps1 相似
  6. MCP 橋接策略：透過 mcporter 保持 MCP 整合靈活且解耦，避免核心膨脹

八、關鍵洞察

為何 OpenClaw 能在 3 個月達到 245K Stars？

  1. 解決真實痛點：將 AI 助手從 Web 界面帶入日常通訊工具（WhatsApp/Telegram/Slack），降低使用門檻
  2. Local-First 信任模型：數據不離開使用者裝置，在隱私議題日益重要的時代具有強大號召力
  3. 開發者友善：TypeScript 生態、清晰的 CLI、豐富的文件、每日更新
  4. 平台完整性：不只是後端，還有 macOS/iOS/Android 原生 app、Voice Wake、Canvas
  5. 商業贊助：OpenAI、Vercel 等大廠背書增加可信度

潛在挑戰

  1. Open Issues 近萬：9,990 個 open issues 暗示維護壓力巨大
  2. 單人主導：steipete 貢獻 10,885 commits，佔壓倒性比例，bus factor 風險
  3. MCP 策略保守：選擇橋接而非原生支援，可能限制 MCP 生態整合度
  4. 頻道維護成本：22+ 頻道各有 API 變更風險，長期維護是考驗
