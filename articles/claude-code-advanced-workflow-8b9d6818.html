核心工作流

  1. Skills 專案初始化：建立可重用 Agent Skill 定義技術棧，用 npx skills 安裝至新專案，Agent 協助設定相依套件與客製化。比傳統模板更靈活（Agent 可互動問答移除不需要的功能）。
  1. Design System 先行：專案骨架完成後先建設計系統（用 Claude Design / Stitch / Dribbble 截圖），存入 design.md，後續所有元件一致遵循。避免後期後端複雜度高時再改 UI。
  1. Spec 驅動實作：用 Claude 規劃模式生成計畫，再透過 create-spec skill 拆分為多 Phase 的 tasks 清單存入 specs/ 資料夾，每 Phase 有獨立依賴與任務項。
  1. Goal + Agent View 並行實作：Agent View 啟動多 session，主 session 用 /goal 命令迴圈執行 spec，sub-agent 並行處理各 wave 任務。額外開 /loop 定時任務：每 10 分鐘安全掃描、每 15 分鐘 UI 設計審查 + 漸進改善。
  1. Checkpoint Skill：自訂 commit skill，一鍵產生 TLDR commit message。

關鍵觀點

  • 不要用單一 session 實作整個 app（token 浪費且失敗率高）
  • CLAUDE.md 只留精簡規則（~37 行），細節靠 AGENTS.md
  • Sub-agent 並行（Wave 1/2）比逐一修改效率高數倍
  • 設計系統 + specs + goal 指令三件套是生產級開發核心模式
