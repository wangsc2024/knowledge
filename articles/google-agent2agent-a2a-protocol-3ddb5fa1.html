Google Agent2Agent (A2A) Protocol

概述

Agent2Agent (A2A) 是 Google 於 2025 年 4 月推出、現由 Linux Foundation 維護的開放通訊協定，旨在實現不同框架、不同廠商建構的 AI Agent 之間的互操作性。截至 2026 年 4 月，A2A 已獲得超過 50 家企業合作夥伴支持（包括 Salesforce、SAP、LangChain、PayPal、ServiceNow 等），成為與 Anthropic MCP 並列的兩大 Agentic AI 基礎設施標準。

核心概念與資料模型

六大核心物件

| 物件 | 說明 |

|------|------|

| Agent Card | JSON 格式的能力宣告，描述 Agent 的技能、認證需求與連線資訊，供 Client 端探索與匹配 |

| Task | 工作單元，具唯一 ID 與定義明確的生命週期狀態（submitted → working → completed/failed），支援即時與長時間執行 |

| Message | 基本通訊單元，承載 Client 與 Remote Agent 間的上下文、指令與回覆 |

| Part | 訊息中的內容片段，可為文字、檔案、結構化資料，支援內容類型協商 |

| Artifact | 任務完成後的交付物（文件、圖片、試算表等） |

| A2A Client/Server | Client 發起請求，Server 處理任務並回傳結果 |

三步工作流程

  1. Discovery（探索）：Client Agent 定位並讀取 Remote Agent Card，找到能力匹配的 Agent
  2. Authentication（認證）：使用 API Keys、OAuth 2.0、OpenID Connect 等企業級安全方案
  3. Communication（通訊）：透過 HTTPS + JSON-RPC 2.0 交換 Task 物件

通訊模式

| 模式 | 技術 | 適用場景 |

|------|------|----------|

| 同步請求/回應 | JSON-RPC 2.0 over HTTPS | 即時完成的簡短任務 |

| 串流 | Server-Sent Events (SSE) | 需要即時進度回饋的任務 |

| 非同步推播 | Webhook 推播通知 | 長時間執行任務（數小時至數天） |

A2A 與 MCP 的互補定位（關鍵區分）

| 面向 | A2A | MCP |

|------|-----|-----|

| 核心焦點 | Agent 對 Agent 的協作通訊 | 應用程式對服務/工具的存取 |

| 使用場景 | 多個 Agent 跨系統協調工作 | LLM 存取外部工具、API、資料來源 |

| Agent 透明度 | 不透明（Opaque）：不暴露內部邏輯 | 透明：工具介面公開 |

| 關係 | 互補協定，可同時使用 | 互補協定，可同時使用 |

實際範例：庫存 Agent 使用 MCP 查詢資料庫取得庫存資料，然後訂單 Agent 使用 A2A 與供應商 Agent 協調補貨 — 兩個協定各司其職。

安全模型

  • 設計為「預設安全（Secure by Default）」
  • 認證方案與 OpenAPI 認證 schemes 對齊
  • 支援 API Keys、OAuth 2.0、OpenID Connect
  • Agent 作為不透明實體運作，不暴露內部邏輯或專有系統
  • v0.3 新增安全卡片簽署（Security Card Signing）機制

v0.3 版本更新

  • 新增 gRPC 傳輸支援（除原有 HTTP/SSE 外）
  • 安全卡片簽署機制
  • Python SDK 擴展客戶端支援
  • 更穩定的介面定義

企業級應用場景

  1. 跨平台多 Agent 工作流：HR 招聘流程中，履歷篩選 Agent、面試排程 Agent、背景調查 Agent 透過 A2A 協調
  2. 長時間研究任務：研究 Agent 花費數小時收集資料，透過非同步推播通知委託方 Agent
  3. 供應鏈協作：不同企業的 Agent 透過 A2A 交換訂單、庫存、物流資訊，各自保持內部邏輯不透明

對 Daily-Digest Agent 系統的啟示

  1. Agent Team 互操作：目前系統的 Team 模式（run-todoist-agent-team.ps1）採用檔案交接（results/*.json），未來可考慮 A2A 式的 Task 生命週期管理
  2. 能力宣告：A2A 的 Agent Card 概念類似 SKILL_INDEX.md 的 Skill 能力宣告，可借鏡其標準化 JSON Schema
  3. MCP + A2A 雙協定：本系統已使用 MCP（屏東新聞 MCP），A2A 可作為未來跨系統 Agent 協作的補充
  4. 不透明 Agent：A2A 的 Opaque Agent 設計與本系統的子 Agent 獨立執行模式理念一致
