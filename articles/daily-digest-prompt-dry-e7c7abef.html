Daily-Digest-Prompt 安全合規優化：DRY消毒、CWD修復、斷路器簡化

安全合規HooksDRY程式碼品質daily-digest-prompt重構

優化摘要

針對 daily-digest-prompt 專案的 Hooks 安全層進行 5 項優化，提升安全合規性與程式碼品質。

優化項目

P0: Gmail 依賴宣告

  • pyproject.toml 新增 [project.optional-dependencies] gmail 群組
  • 包含 google-auth-oauthlib、google-auth-httplib2、google-api-python-client
  • 消除隱性依賴，明確化安裝需求

P1-1: 共用敏感資料消毒函數（DRY）

  • 從 post_tool_logger.py 和 behavior_tracker.py 抽取重複的正則消毒邏輯
  • 統一至 hook_utils.py 的 sanitize_sensitive_data() 函數
  • 3 個預編譯正則：Authorization header、X-Token/Key/Secret header、環境變數
  • 兩個消費端改為委派呼叫，減少 ~40 行重複碼

P1-2: behavior_tracker.py CWD 漂移修復

  • 原始碼使用 os.path.join("context", ...) 相對路徑
  • 團隊模式下 CWD 可能非專案根目錄，導致檔案寫入錯誤位置
  • 改用 os.path.dirname(os.path.abspath(__file__)) 腳本相對路徑
  • 與 post_tool_logger.py 的路徑策略一致

P1-3: CircuitBreaker fallback 簡化

  • agent_guardian.py 的 _atomic_update() fallback 從 30 行 msvcrt/fcntl 重新實作
  • 簡化為 3 行直接讀寫（無鎖模式）
  • hook_utils 已提供完整的 file_lock，fallback 只需最簡實作

P2: 消毒函數測試覆蓋

  • tests/hooks/test_hook_utils.py 新增 TestSanitizeSensitiveData 類別
  • 10 個測試案例：Bearer/Basic token、X-Api-Token/Key、環境變數、PowerShell env、大小寫不敏感、URL 結構保留

驗證結果

  • 652 個測試通過（+10 新增），7 個失敗（既有 Gmail import 問題，未變更）
  • Git diff: 6 檔案，+112/-79 行，範圍可控

技術決策

  • 消毒正則在模組層級預編譯（避免 hot path 重複編譯）
  • ImportError fallback 保持向後相容（hook_utils 不可用時原樣返回）
  • 選擇 SHA-256 而非 MD5（與 agent_guardian.py 一致，避免碰撞風險）
