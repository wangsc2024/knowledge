Test-Time Compute Scaling 與 Long Chain-of-Thought 推理

概述

2026 年，AI 推理能力的提升已從「訓練時擴展」（更大模型、更多數據）轉向「推理時擴展」（Test-Time Compute Scaling, TTS）。核心洞見是：在推理階段投入更多計算資源，可能比增加模型參數更有效。這一範式轉移催生了 Long Chain-of-Thought（Long-CoT）推理模型家族，包括 OpenAI o3/o4-mini、DeepSeek-R1、Claude Opus 4（混合推理模式）、Gemini 2.5 Pro 等。

---

一、Test-Time Compute Scaling 四維分類框架

根據 2025-2026 年的系統性綜述（testtimescaling.github.io），TTS 研究可從四個維度理解：

1.1 What to Scale（擴展什麼）

| 策略 | 機制 | 代表方法 |

|------|------|----------|

| Parallel Scaling | 平行生成多個輸出，再聚合選優 | Self-Consistency、Best-of-N |

| Sequential Scaling | 根據中間步驟導引後續計算 | Chain-of-Thought、Tree-of-Thought |

| Hybrid Scaling | 結合平行與序列的優勢 | MCTS + Self-Consistency |

| Internal Scaling | 模型自主決定推理深度 | Implicit Thought Transformer (ITT) |

1.2 How to Scale（如何擴展）

訓練階段方法：

  • SFT（監督微調）：用合成推理範例訓練，如 OpenThoughts、NaturalThoughts 資料集
  • RL（強化學習）：用獎勵信號引導模型生成更長、更正確的推理鏈，如 DeepSeek-R1 的 GRPO 演算法

推理階段方法：

  • Stimulation（刺激）：促使 LLM 生成更多、更長的推理樣本
  • Verification（驗證）：用 Process Reward Models (PRM) 或 Outcome Reward Models (ORM) 評估推理品質
  • Search（搜尋）：系統性探索推理空間（Monte Carlo Tree Search、Beam Search）
  • Aggregation（聚合）：整合多個推理路徑為最終答案

1.3 Where to Scale（應用領域）

  • 數學推理（rStar-Math、Math-Shepherd）
  • 程式碼生成（OpenCodeReasoning）
  • 科學問題（GPQA 基準測試）
  • 通用 Agent 任務（多步驟規劃與執行）

1.4 How Well to Scale（效果評估）

四個評估維度：正確性（答案準確率）、效率（成本效益比）、可控性（資源約束下的表現）、可擴展性（追加計算的邊際收益）。

二、Long Chain-of-Thought 推理三大能力軸

根據 Awesome-Long-Chain-of-Thought-Reasoning 綜述，Long-CoT 能力分為三大軸：

2.1 Deep Reasoning（深度推理）

格式化方法：

  • 自然語言推理：Natural Program、Code I/O
  • 結構化語言推理：Program-of-Thought、DeepSeek-Prover（用程式碼作為推理載體）
  • 潛在空間推理：Coconut、Heima、Implicit Thought Transformer — 在模型內部表徵空間進行推理，不需生成可見 token

學習方法：

  • 模仿學習：從人類、進階 RLLM 或大模型蒸餾推理軌跡
  • 自我學習：STaR、ReST、ReST-EM — 模型自我生成推理範例並迭代改進
  • 樹搜尋優化：ReST-MCTS*、FastMCTS — 結合蒙特卡洛樹搜尋與自我改進

2.2 Extensive Exploration（廣度探索）

  • 序列探索：延長單一推理鏈的深度
  • 平行探索：Self-Consistency、ECM — 同時探索多條推理路徑
  • RL 引導探索：PPO、DivPO、GRPO — 強化學習策略引導模型探索更多推理分支
  • 外部搜尋：PPO-MCTS、MindStar、rStar-Math — 結合外部搜尋演算法

2.3 Feasible Reflection（可行反思）

模型自我檢查與修正能力：

回饋機制：

  • 結果層級：Outcome Reward Models、LLM-as-Judge
  • 過程層級：Process Reward Models（PRM800K、Math-Shepherd）— 逐步驟評分
  • 混合機制：Consensus Filtering、Step-KTO

修正方法：

  • Prompt-based：ReAct、Reflexion、Self-Backtracking
  • SFT-based：rStar、Journey Learning
  • RL-based：SCoRe、DeepSeek-R1、S²R、ReVISE

四、關鍵技術突破與前沿方向

4.1 蒸餾 vs 強化學習的根本辯論

2026 年最重要的發現之一：在固定計算/資料預算下，基於驗證器的 RL/搜尋方法遠優於純蒸餾推理軌跡。這意味著僅複製大模型的推理過程不夠，模型需要學會「自己驗證」。

4.2 最佳推理長度的領域差異

研究發現推理鏈並非越長越好：不同領域存在不同的最佳推理長度分布。過長的 CoT 在某些領域反而會損害推理表現（overthinking problem）。

4.3 前沿方法

  • CoT-Valve：根據任務難度動態調整推理深度 — 簡單問題用短鏈，困難問題用長鏈
  • UnCert-CoT：基於熵感知的不確定性排程 — 在模型不確定的步驟分配更多計算
  • MAKER 框架：透過極端任務分解 + 多 Agent 投票，實現超過 100 萬 LLM 步驟零錯誤
  • Implicit Thought Transformer (ITT)：token 級別的思考路由，模型在每個 token 決定是否需要額外思考
  • Forest-of-Thought：將 Tree-of-Thought 擴展為森林結構，平行探索多棵推理樹

4.4 潛在空間推理（Latent Reasoning）

最激進的前沿方向：模型在內部表徵空間進行推理，不生成可見的中間 token。代表方法：Coconut（Continuous CoT）、Heima。優勢是推理不佔用上下文長度，但可解釋性較低。

---

五、對實務的啟示

  1. 選擇推理模型時：考慮任務類型（數學用 DeepSeek-R1、程式碼用 Claude Opus 4、通用用 o3）
  2. 推理預算管理：使用 CoT-Valve 或類似機制動態分配推理計算
  3. 開源部署：DeepSeek-R1 + vLLM 可在本地實現高品質推理
  4. 品質保證：Process Reward Models 比 Outcome Reward Models 更適合長鏈推理的品質控制
  5. Agent 系統：混合推理模式（如 Claude Extended Thinking）最適合複雜 Agent 工作流
