核心要點

Anthropic 新增 Claude Code 巢狀子 Agent 功能（最多 5 層），讓子 Agent 可再生成子 Agent：

原理：子 Agent 隔離雜訊工具呼叫（搜尋、程式碼探索、驗證），避免主 context window 被污染而品質下降。

巢狀架構：

  • Layer 0：主 context（重要決策）
  • Layer 1：一級子 Agent（獨立任務）
  • Layer 2+：更深層子 Agent（搜尋驗證、交叉核查）

實用場景：

  1. 文章驗證：每篇文章一個 L1 子 Agent，重要聲明由 L2 搜尋驗證
  2. 法律合同分析：10 份合同並行，L2 分析衝突範圍，L3 查閱相關郵件
  3. Debug 根因分析：L1 載入事故報告，L2 並行搜尋 codebase，L3 交叉核查 server log
  4. Skill 比較優化：讓多個 L1 子 Agent 分別運行不同 Skill 後比較結果

Fable-5 加持：新模型能自動判斷何時需要巢狀子 Agent，不必手動 prompt 觸發。

Codex 對應：config 加 max_depth 參數（預設 1）可開啟類似功能。

適用情境

需要大量並行驗證、二階/三階效應分析、或保持主 context window 精簡的 Claude Code 工作流。
