專案優化：query_logs.py 測試覆蓋 + BENIGN_PATTERNS 精確匹配修正

專案優化報告 (2026-02-25)

背景

透過四路並行分析（hooks 程式碼、PowerShell 腳本、配置一致性、測試覆蓋率），識別出以下優化項目。

優化項目

#### 1. query_logs.py 測試覆蓋（從 0 到 20 個測試）

  • 問題：query_logs.py 是唯一無測試覆蓋的 hooks 工具
  • 修正：新增 test_query_logs.py，覆蓋 load_entries、load_session_summaries、print_summary、print_cache_audit、print_sessions
  • 亮點：包含跨模組驗證測試，確保 cache audit 來源清單與 hook_utils.API_SOURCE_PATTERNS 一致

#### 2. BENIGN_PATTERNS 精確匹配修正（bug fix）

  • 問題：post_tool_logger.py 的 BENIGN_PATTERNS 中 exit code 0 會子字串匹配 exit code 0xFF，導致真正的錯誤被誤判為良性
  • 修正：替換為三個精確變體（冒號、空格終止、換行終止）
  • 驗證：新增 4 個測試覆蓋 exit code 0/1/0xFF 場景

四路分析主要發現（供後續參考）

#### hooks 程式碼品質（17 個問題）

  • Critical(3): on_stop_alert 競態條件、CircuitBreaker 並發寫入、validate_config 無深度限制
  • High(7): exit_code 推斷不可靠、路徑正規化不完整、YAML 載入缺 Schema 驗證

#### PowerShell 腳本品質（12 個問題）

  • Critical(3): Token 洩漏風險、scheduler-state 並發寫入、stderr 資源洩漏

#### 配置一致性（11 個問題）

  • High: todoist-auto-creative_game_optimize.md fail-safe 檔名不一致

測試結果

  • 測試總數增加 24 個（20 個 query_logs + 4 個 BENIGN_PATTERNS）
  • 全部通過：532/532
