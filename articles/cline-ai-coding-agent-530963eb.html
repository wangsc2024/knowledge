Cline — 開源 AI Coding Agent 深度洞察報告

專案概述

Cline 是一個在 VS Code 內執行的開源自主編程 Agent，能夠建立/編輯檔案、執行終端命令、操控瀏覽器，並透過 Model Context Protocol（MCP）無限擴展能力——每一步都需要開發者明確授權。

|------|------|

| GitHub Stars | 58,100+ |

| Forks | 5,800+ |

| 安裝量 | 500 萬+（VS Code Marketplace） |

| 授權 | Apache 2.0 |

| 主要語言 | TypeScript |

| 建立時間 | 2024 年（原名 Claude Dev） |

| 最近活躍 | 2026-02-19（持續活躍） |

一、核心架構分析

1.1 Plan-Act-Verify 迴圈

Cline 的核心工作流程是一個三階段循環：

  1. Plan（規劃）：分析檔案結構、執行 regex 搜尋、讀取相關檔案以理解現有專案
  2. Act（執行）：建立/編輯檔案、執行終端命令、操控瀏覽器
  3. Verify（驗證）：人機協作審查，透過 diff view 確認每一步變更

這種分離設計讓開發者能在「資訊蒐集」與「實際行動」之間建立清晰的邊界，顯著降低 AI 自主操作的風險。

1.2 Human-in-the-Loop（人機協作）

Cline 的安全模型建立在「反轉風險模型」之上：

「AI 不是安全風險，缺乏架構邊界才是。」

具體實踐：

  • 每個檔案變更需經開發者在 diff view 中審批
  • 每個終端命令需經明確核准
  • Context 組裝可見：開發者在推理前能看到送給模型的完整上下文
  • Checkpoint 快照：每步自動建立工作區快照，可回滾到任意歷史狀態

1.3 模型抽象層

Cline 實現了徹底的「模型無關」架構：

| 供應商類型 | 支援 |

|-----------|------|

| 商業 API | OpenRouter, Anthropic, OpenAI, Google Gemini |

| 雲端平台 | AWS Bedrock, Azure, GCP Vertex |

| 高效推理 | Cerebras, Groq |

| 本地模型 | LM Studio, Ollama |

| 通用相容 | 任何 OpenAI 相容 API |

這種設計消除了廠商鎖定（vendor lock-in），讓團隊能依據成本、隱私和效能需求自由切換模型。Addy Osmani 推薦「DeepSeek-R1（規劃）+ Claude Sonnet（執行）」的混合策略，宣稱可降低 97% 成本。

1.4 MCP 擴展生態系

MCP（Model Context Protocol）是 Cline 最具前瞻性的架構決策：

  • MCP Marketplace：類似「App Store」的一鍵安裝市場，數百個預建工具伺服器
  • 自動建立工具：用自然語言描述需求（「add a tool that...」），Cline 自動建立 MCP 伺服器並安裝
  • 智能路由：MCP Rules 根據對話上下文自動選擇工具，開發者無需手動指定
  • 分類觸發：將相關伺服器分組並定義觸發關鍵字，實現語境感知的工具選擇

二、Memory Bank — 跨 Session 記憶系統

Cline 社群發展出 Memory Bank 框架，用 markdown 文件建立持久化記憶：

檔案層次


memory-bank/
├── projectbrief.md      <- 基礎文件（真理源頭）
├── productContext.md     <- 產品/用戶視角
├── systemPatterns.md     <- 技術架構決策
├── techContext.md        <- 開發環境配置
├── activeContext.md      <- 當前開發狀態
└── progress.md           <- 進度追蹤

運作機制

  1. 初始化：建立 memory-bank/ 目錄，收集必要資訊
  2. 開發期間：每次 session 開始時讀取所有 Memory Bank 文件、驗證上下文完整性、遵循既定模式
  3. 維護：重大變更後更新文件保持一致性

創新之處：Mermaid 圖表作為規格語言

Memory Bank 在 Custom Instructions 中使用 Mermaid 流程圖作為「形式化規格語言」，比自然語言描述更明確——每個節點和決策點都被精確映射，提升 AI 的流程理解能力。

四、可借鏡技術項目（對 Daily-Digest-Prompt 專案的啟示）

4.1 MCP Rules 自動路由 -> 強化 routing.yaml

Cline 的做法：MCP Rules 根據對話關鍵字自動選擇工具伺服器，無需手動指定。

可借鏡：本專案的 config/routing.yaml 三層路由可進一步引入「語境感知」機制——不僅匹配標籤，還根據任務描述內容智能推薦 Skill，類似 MCP Rules 的觸發關鍵字分組。

4.2 Memory Bank 檔案層次 -> 強化 digest-memory.json

Cline 的做法：6 個語義分離的 markdown 文件，各有明確職責，層次化依賴。

可借鏡：本專案的 context/digest-memory.json 是單一扁平 JSON，可考慮分離為：

  • context/project-state.md（系統架構決策）
  • context/active-context.md（當前工作狀態）
  • 保留 digest-memory.json 作為統計數據層

4.3 Checkpoint 快照機制 -> 強化工作區復原能力

Cline 的做法：每步操作自動建立工作區快照，可 diff 對比並選擇性回滾。

可借鏡：本專案的 agent_guardian.py 已有迴圈偵測，但缺乏「狀態快照」能力。可在團隊模式 Phase 2 執行前自動建立 Git stash 或 commit 作為 checkpoint，失敗時自動回滾。

4.4 Plan/Act 模式分離 -> 強化 Todoist 任務執行

Cline 的做法：明確分離「資訊蒐集」與「實際行動」兩個模式。

可借鏡：本專案的 Todoist 團隊模式已有 Phase 1/Phase 2 的分離，但 Phase 2 內部的子 Agent 沒有明確的 Plan/Act 邊界。可在 sub-agent 模板中加入明確的「先讀後寫」約束。

4.5 模型混合策略 -> 降低 API 成本

Cline 的做法：用便宜模型做規劃（DeepSeek-R1），用強力模型做執行（Claude Sonnet），降低 97% 成本。

可借鏡：本專案所有 Agent 都使用同一模型。可新增 config/model-policy.yaml，定義不同任務類型的模型偏好：

  • 研究/規劃任務 -> haiku（快速且便宜）
  • 程式碼修改 -> sonnet 或 opus（精準）
  • 日常摘要組裝 -> haiku

4.6 Mermaid 圖表作為規格語言 -> 強化 Skill 定義

Cline 的做法：用 Mermaid 流程圖而非自然語言定義工作流程。

可借鏡：本專案的 SKILL.md 可在複雜 Skill 中實驗加入 Mermaid 流程圖，測試是否能提升 Agent 的步驟理解準確度。

實施優先級建議

| 優先級 | 項目 | 複雜度 | 預期效益 |

|--------|------|--------|----------|

| P0 | 4.5 模型混合策略 | 低 | 直接降低 API 成本 |

| P1 | 4.4 Plan/Act 分離 | 低 | 減少子 Agent 衝突寫入 |

| P1 | 4.3 Checkpoint 快照 | 中 | 提升失敗復原能力 |

| P2 | 4.1 語境感知路由 | 中 | 提升 Skill 匹配精準度 |

| P2 | 4.6 Mermaid 規格 | 低 | 待驗證 |

| P3 | 4.2 Memory 分離 | 高 | 長期可維護性 |

五、關鍵洞見與趨勢

5.1 「本地優先」成為 2026 年 AI 工具主流範式

Cline 的成功證明：開發者偏好能在本地環境中運行、不需將程式碼上傳雲端的 AI 工具。58K stars 和 500 萬安裝量的背後是對「資料主權」的強烈需求。這對企業環境（合規、資安）尤其重要。

5.2 MCP 正在成為 AI 工具的「USB-C」

Cline 的 MCP Marketplace 展示了協議標準化的威力：一旦工具遵循統一協議，生態系統就能爆發式增長。Anthropic 推出 MCP、Cline 率先大規模採用、社群貢獻數百個 MCP 伺服器——這個正向飛輪已經啟動。

5.3 Human-in-the-Loop 不是限制，是競爭優勢

Cline 刻意選擇「每步審批」而非「完全自主」，結果反而獲得企業客戶信任。這暗示：在 AI Agent 領域，「可控性」比「自主性」更重要——至少在 2026 年是如此。

5.4 開源 Fork 生態的挑戰

Cline 2026 年面臨 Kilo Code（$8M 種子輪）和 Roo Code（「穩定性優先」fork）的競爭。開源專案一旦成功就會被 fork，原始團隊需要在社群治理和商業化之間找到平衡——Cline 選擇推出 Enterprise 版本（SSO、審計日誌、私有網路）作為答案。
