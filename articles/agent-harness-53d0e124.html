Agent Harness 自主管理優化基準與通知治理實作指南

研究問題

如何把 agent harness 的自主管理、系統優化與通知閉環做成可量測、可驗證、可回退的實作體系？

核心概念（外部文獻）

  • Anthropic (2025-11)：long-running agent 需 initializer + coding agent 雙階段、feature list、incremental progress、clean state
  • inference.sh：durable execution = checkpoint + resumability + retry + idempotency
  • LangChain：harness 優化可顯著提升表現（52.8%→66.5%）
  • Kubernetes AI Operator：預測異常、根因分析、自校正；降低 87% 停機

本專案已落地

  • 控制面：autonomous_harness.py 聚合 run-fsm、scheduler-state、failure-stats、api-health、fairness、token budget、heartbeat、resource snapshot
  • 恢復佇列：autonomous-recovery-queue.json + autonomous_recovery_worker.py
  • Runtime override：autonomous-runtime-overrides.json
  • 排程：run-autonomous-harness.ps1 每 5 分鐘（HEARTBEAT）

量化指標（待建立）

  • success_rate、MTTR、queue_backlog、budget_burn
  • 通知成效：ntfy 發送成功率、噪音抑制、升級策略

延伸方向

  • Phase 2 snapshot/resume（ADR-20260318-028）
  • 通知與 SLO、approval gate、incident triage 統一治理
  • 長期趨勢檔：resource_snapshot、done_cert、recovery 結果
