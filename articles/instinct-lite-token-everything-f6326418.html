Instinct Lite + Token 經濟學：基於 everything-claude-code 的系統強化實施

背景

基於知識庫文章「everything-claude-code 深度研究」的四大啟示，選擇兩個最高價值項目落實到 daily-digest-prompt 專案。

強化項目

1. Instinct Lite 行為模式採集器

靈感來源：everything-claude-code 的 Continuous Learning v2 Instinct 系統

實作：hooks/behavior_tracker.py（獨立模組，~180 行）

核心機制：

  • 原子級模式：tool + 簽名摘要作為模式 key（MD5 hash）
  • 信心分數：0.1（初始）→ 每次重複 +0.05 → 最高 1.0
  • 正規化簽名：移除 UUID、日期、時間戳等動態部分，保留結構
  • 滾動視窗：30 天未觀察的模式自動衰減移除
  • Token 統計：累積每個模式的 input_len + output_len

信心分數 >= 0.5 的模式可考慮演化為正式 Skill（未來 /evolve 指令）。

資料存儲：context/behavior-patterns.json（最多 500 個模式）

2. Token 經濟學追蹤

靈感來源：everything-claude-code 的 Token Economy 章節

實作：在 post_tool_logger.py JSONL entry 新增 input_len 欄位

  • input_len：tool_input JSON 序列化後的字元長度（近似 input tokens）
  • output_len：原有欄位，tool_output 長度
  • 兩者結合可量化每次工具呼叫的 Token 消耗
  • behavior_tracker 自動累積每個模式的 total_input + total_output

可用 python hooks/behavior_tracker.py report 查看 Token 經濟概要。

3. Benchmark 升級（v1 → v2）

config/benchmark.yaml 新增三個指標：

| 指標 | 目標 | 權重 | 來源 |

|------|------|------|------|

| avg_io_per_call | <= 5000 chars | 10 | behavior-patterns.json |

| behavior_pattern_count | >= 20 | 5 | behavior-patterns.json |

| high_confidence_patterns | >= 5 | 5 | behavior-patterns.json |

同時在 comparison.reference_projects 新增 everything-claude-code。

JSON Schema（benchmark.schema.json）已同步更新，version 從 const: 1 改為 minimum: 1, maximum: 10。

整合方式


post_tool_logger.py
  ├── 計算 input_len（Token 經濟）
  ├── 寫入 JSONL（含 input_len 欄位）
  └── 呼叫 behavior_tracker.track()（Instinct Lite）
        └── 更新 context/behavior-patterns.json

測試驗證

  • 505 個 hooks 測試全通過（含 post_tool_logger 50 + validate_config 37）
  • behavior_tracker 採用 try/except ImportError fallback，不影響現有功能
  • 所有改動向下相容

未來方向

  1. Skill 自動演化：高信心模式 >= 0.7 自動建議新 Skill
  2. Token 預算控制：依歷史 I/O 數據設定每次執行的 Token 上限
  3. 紅隊安全層：在 system-audit 加入 Agent 對抗推理（需評估成本）
  4. Instinct 匯出/匯入：跨專案共享行為模式
