二、新發現的優化建議（與昨日報告差異化）

P0（緊急）：Hook 系統強化

| 建議項目 | 說明 | 預估工時 |

|---------|------|----------|

| Hook 單元測試 | 4 個 Python hooks 目前無測試覆蓋，pre_bash_guard 的正則表達式攔截邏輯容易出錯 | 2-3h |

| Hook 效能監控 | post_tool_logger.py 每次工具呼叫都寫入 JSONL，高頻執行時可能成為瓶頸 | 1-2h |

| Offset 追蹤容錯 | on_stop_alert.py 的 .last_analyzed_offset 若損壞會導致重複告警 | 1h |

P1（高優先級）：可觀測性提升

| 建議項目 | 說明 | 預估工時 |

|---------|------|----------|

| 結構化日誌 Dashboard | 目前只有 CLI 查詢 (query_logs.py)，缺少可視化面板 | 4-6h |

| Session 健康度指標 | 增加 cache_hit_rate、skill_usage_count、avg_tool_latency 等指標 | 2-3h |

| 告警分級細化 | on_stop_alert.py 目前只有 critical/warning，可增加 info/debug 級別 | 1h |

| 跨日趨勢分析 | query_logs.py --trend 只統計 14 天，可增加月度/季度報告 | 2h |

P2（中優先級）：開發體驗優化

| 建議項目 | 說明 | 預估工時 |

|---------|------|----------|

| 熱重載機制 | 目前修改 config/*.yaml 需等下次排程，可加入 watch 模式 | 3-4h |

| 本地測試模式 | 增加 --dry-run 參數，不實際呼叫外部 API | 2-3h |

| 配置驗證工具 | 新增 validate-config.ps1，在排程執行前檢查 YAML 語法和必填欄位 | 2h |

| Skill 腳手架生成器 | 自動生成 SKILL.md 模板 + 目錄結構 | 1-2h |

P3（低優先級）：架構精練

| 建議項目 | 說明 | 預估工時 |

|---------|------|----------|

| Template 懶載入優化 | templates/ 按需載入但缺乏載入計數，無法得知哪些模板從未使用 | 1h |

| Config 層 Schema 驗證 | 為 pipeline.yaml、routing.yaml 等增加 JSON Schema | 2-3h |

| 六層依賴可視化 | 自動生成架構依賴圖（哪個 Prompt 引用哪些 Config/Template/Skill） | 3-4h |

| 增量備份機制 | context/ 和 state/ 目前無版本控制，可加入 git-based 備份 | 2h |
