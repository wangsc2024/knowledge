Workflow Forge：Prompt Fail-Safe Write 合規性驗證清單（2026-06-17）

workflow-forge流程標準化consistencyfail-saferesult_file_missing

本次鑄造報告

gap_type：consistency（P1）

產物路徑：workflows/prompt-fail-safe-write-compliance-checklist.md

問題背景

2026-06-16 共 7 個 auto-task（shurangama/jiaoguangzong/ai_news_daily/insight_briefing/kb_insight_evaluation/self_heal/podcast_jiaoguangzong）集中出現 result_file_missing 失敗。根因為 prompt 未嚴格落實 preamble 規定的「Fail-Safe 佔坑」協議——雖引用了 preamble，但無明確 Write 指令，導致 LLM 略過佔坑直接執行主要步驟，agent crash 時 result file 不存在。

驗證清單內容

6 項必填要素（F1-F6）：F1 明確讀取 preamble.md；F2 Write 佔坑在主要步驟之前；F3 佔坑包含 status:failed；F4 summary ≥30 字含 ISO8601；F5 路徑 results/todoist-auto-{task_key}.json；F6 佔坑後驗證指令。包含 Grep 批量掃描指令、7 個 P1 不合規任務列表（shurangama/jiaoguangzong/ai_news_daily/insight_briefing/kb_insight_evaluation/self_heal/podcast_jiaoguangzong）、修補範本與回歸驗證步驟。
