搜尋日期：2026-06-20

主題：Agent 架構（runtime orchestration / state model / human-in-the-loop）

專案觀測摘要

1. langchain-ai/langgraph（35,200★）

  • runtime_model: graph-based execution with resumable checkpoints
  • state_strategy: 雙層記憶（short-term working + long-term persistent）
  • human_review_model: interrupt-point 模式 — 圖執行遇 breakpoint 暫停，等待人工 approve/edit
  • 與本系統關聯: loop-contracts.yaml v3 per-task mode=advise 模式具備等待人工審核語意，可借鑑 interrupt() breakpoint 宣告方式

2. crewAIInc/crewAI（54,000★）

  • runtime_model: crews (role-based teams) + flows (event-driven DAG @router)
  • state_strategy: pydantic-based structured state management
  • human_review_model: seamless human-in-the-loop collaboration（callback 抽象）
  • 與本系統關聯: @router DAG 條件路由可作為 planner_executor 非線性 phase 跳轉借鑑

3. openai/openai-agents-python（27,300★）

  • runtime_model: agent handoff delegation + sandbox agents
  • state_strategy: session-based conversation history（Redis 支援）
  • human_review_model: handoff-based — explicit handoff 轉移控制權，HITL 為高層抽象 approval callback
  • 與本系統關聯: Phase 1a dispatch 語義設計參考

| 優先級 | 專案 | 提案 | 風險 |

審查結論

  • crewAI P1 提案（DAG 路由計畫）：可行性 7/10，穩定性 8/10（高風險需計畫文件）
  • openai P2 提案（文件評估）：可行性 9/10，穩定性 10/10
