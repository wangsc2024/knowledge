PentAGI — 全自主 AI 滲透測試 Agent 系統深度研究

專案概述

PentAGI 是由 VXControl 開發的開源全自主 AI 滲透測試系統，於 2025 年初在 GitHub 發布，2026 年 2 月成為 GitHub Trending 熱門專案。它將 AI Agent 架構應用於資安領域，整合 20+ 專業安全工具，實現從偵察到漏洞利用的全自動化流程。

  • GitHub：github.com/vxcontrol/pentagi
  • Stars：7,600+（截至 2026-02-23）
  • 授權：MIT License（完全開源、商業可用）
  • 技術棧：Go（後端）+ React TypeScript（前端）+ PostgreSQL/pgvector + Neo4j + Docker
  • 系統需求：2+ vCPU、4GB+ RAM、20GB 磁碟空間
  • 支援平台：Linux（amd64/arm64）、Windows（amd64）、macOS（Intel/M-series）

核心創新：為什麼 PentAGI 值得關注？

1. 全自主多 Agent 架構

PentAGI 採用四角色分工的多 Agent 系統，模擬真實滲透測試團隊的協作模式：

Agent 角色職責類比
Orchestrator初始化流程、查詢歷史任務、載入相關知識專案經理
Researcher分析目標、搜尋模式、查詢漏洞資料庫情報分析師
Developer規劃攻擊路徑、載入漏洞利用技術攻擊策略師
Executor實施計畫、儲存結果、回報狀態執行操作員

每個 Agent 在整個執行過程中都能存取向量資料庫和知識圖譜，實現資訊共享與經驗累積。

2. 三層記憶架構

這是 PentAGI 最具創新性的設計之一，解決了 LLM Agent 常見的「健忘症」問題：

  • 長期記憶（Long-term）：向量嵌入資料庫存儲領域專業知識、工具使用模式
  • 工作記憶（Working）：當前任務上下文、活躍目標、系統狀態
  • 情境記憶（Episodic）：歷史動作、結果、成功模式

搭配 Chain Summarization 機制，選擇性壓縮舊對話訊息，避免 token 溢出同時保留關鍵上下文。

3. Graphiti 知識圖譜

使用 Neo4j 圖資料庫建構知識圖譜，由 Graphiti 驅動：

  • 自動捕捉 Agent 回應和工具執行結果
  • 建立實體間語義關聯（如：漏洞 → 影響服務 → 可用 exploit）
  • 支援進階上下文理解和跨任務知識遷移

技術架構分析

微服務架構

PentAGI 採用完整的微服務架構，包含以下組件：

應用層：

  • React TypeScript 前端（使用者介面）
  • Go 後端（REST + GraphQL API，Bearer Token 認證）

資料層：

  • PostgreSQL + pgvector（語義搜尋和記憶儲存）
  • Neo4j（知識圖譜）
  • Redis（快取）
  • ClickHouse（分析倉儲）
  • MinIO（S3 相容物件儲存）

可觀測性層：

  • OpenTelemetry（統一資料收集）
  • Grafana + VictoriaMetrics（儀表板和時序資料）
  • Jaeger（分散式追蹤）
  • Loki（日誌聚合）
  • Langfuse（LLM 效能監控）

整合的 20+ 安全工具

包括但不限於：

  • 偵察：Nmap（網路掃描）、Whois、DNS 工具
  • 漏洞掃描：Nikto、WPScan、Nuclei
  • 漏洞利用：Metasploit、SQLMap
  • Web 測試：Burp Suite 整合、OWASP ZAP
  • 密碼測試：Hydra、John the Ripper
  • 網路分析：Wireshark/tshark

部署方式

採用 Docker Compose 一鍵部署，透過環境變數配置：

# 基本部署步驟
git clone https://github.com/vxcontrol/pentagi.git
cd pentagi
cp .env.example .env  # 配置 LLM API Key 等
docker compose up -d

所有安全測試操作在沙盒化 Docker 容器中執行，完全隔離不影響宿主系統。系統支援智慧容器選擇——根據任務需求自動選擇適當的工具容器。

對 Daily Digest 系統的啟發

PentAGI 的架構設計有幾個值得借鑑的模式：

  1. 三層記憶架構：與我們的 digest-memory.json + research-registry.json + cache/ 三層設計理念類似，但 PentAGI 更精緻地區分了 episodic memory
  2. Chain Summarization：類似我們的 context window 保護策略，但他們用選擇性壓縮而非截斷
  3. 知識圖譜：Neo4j 的語義關聯追蹤，可作為未來 KB 進化方向的參考
  4. 多 Agent 分工：Orchestrator → Researcher → Developer → Executor 的四角色模式，與我們的 Todoist 團隊模式（Phase 1 查詢 → Phase 2 並行執行 → Phase 3 組裝）有異曲同工之妙
  5. 完整可觀測性：OpenTelemetry + Grafana + Jaeger 的監控堆疊遠超我們目前的 JSONL 日誌系統
