OpenClaw（openclaw/openclaw）— 2026 年最快增長的個人 AI 助理：Local-First 架構、50+ 平台整合與改名風波全紀錄

執行摘要

OpenClaw 是一個開源個人 AI 助理，於 2025 年 11 月以 Clawdbot 之名首次發布，後因 Anthropic 商標爭議兩度改名（Clawdbot→Moltbot→OpenClaw），並在 2026 年 1 月病毒式爆紅，數天內從 9,000 stars 暴增至 60,000+，截至 2026 年 3 月已累積 247,000 stars，成為 GitHub 史上增長最快的開源專案之一 [來源：ByteByteGo Blog, 2026; Wikipedia, 2026]。核心價值在於 local-first 架構——完全運行於使用者自有設備，並透過「閘道（Gateway）」整合 50+ 訊息平台（WhatsApp、Telegram、Slack、Discord 等），讓使用者可在任何慣用的通訊工具中與 AI 互動，而非被鎖定在特定應用程式 [來源：OpenClaw GitHub README, 2026]。技術棧採用 TypeScript + Node.js，支援語音喚醒、瀏覽器自動化、Shell 執行、智慧家居控制等進階功能，並透過 Docker/SSH 沙箱確保多使用者環境下的安全性 [來源：OpenClaw GitHub README, 2026]。

---

專案概述

解決的核心問題

當前市場上的 AI 助理（如 ChatGPT、Claude、Gemini）大多綁定專屬介面（網頁/App），使用者需要在工作流程中「切換平台」才能與 AI 互動。OpenClaw 則提出不同方案：「不改變你的工作方式，而是讓 AI 進入你已經在用的平台」[來源：OpenClaw GitHub README, 2026]。

核心價值主張

  1. 平台中立：支援 WhatsApp、Telegram、Slack、Discord、Google Chat、Signal、iMessage、BlueBubbles、IRC、Microsoft Teams、Matrix、Feishu、LINE、Mattermost、Nextcloud Talk、Nostr、Synology Chat、Tlon、Twitch、Zalo、WeChat、QQ、WebChat 等 50+ 平台 [來源：OpenClaw GitHub README, 2026; LumaDock Tutorial, 2026]。
  2. 資料主權：所有資料儲存於使用者自己的伺服器，無需擔心隱私外洩或服務商停用 [來源：ByteByteGo Blog, 2026]。
  3. 跨平台連續性：在 WhatsApp 開始的對話可在 Telegram 繼續，前提是綁定相同使用者身份 [來源：YingTu AI Blog, 2026]。

技術架構

核心元件

Gateway（閘道）：擔任中央控制平面，負責 session 管理、channel 路由、tool 呼叫與 event 處理 [來源：OpenClaw GitHub README, 2026]。

Multi-Channel Integration：透過各平台的 Bot API 或非官方協定（如 Baileys for WhatsApp、grammY for Telegram）連接訊息通道 [來源：OpenClaw Telegram Docs, 2026; Blink.new Blog, 2026]。

Sandbox Backends：支援 Docker、SSH、OpenShell 三種沙箱後端，隔離危險操作（瀏覽器、shell、canvas）於受控環境 [來源：OpenClaw GitHub README, 2026]。

Companion Apps：可選的 macOS menu bar app、iOS node、Android node 提供語音喚醒（Voice Wake）與 TTS（ElevenLabs 整合）[來源：OpenClaw GitHub README, 2026]。

設計決策

Why Local-First?

與 OpenAI Assistants API 或 Claude API 等雲端方案相比，local-first 讓使用者擁有完整控制權，可在自有 VPS（$5/月）或家用 NAS 上運行，僅需支付 LLM API 呼叫費用（典型任務約 $0.30）[來源：ByteByteGo Blog, 2026]。

Why Multi-Channel over Single App?

根據 RamNode 的使用者調查，開發者平均同時使用 3-5 個訊息平台（工作用 Slack、個人用 Telegram、團隊用 Discord），單一 App 會增加認知負擔 [來源：RamNode OpenClaw Guide, 2026]。

使用方式

快速上手（5 分鐘部署）


# 1. 安裝（需 Node.js 22.16+）
npm install -g openclaw@latest

# 2. 初始化配置（會建立 ~/.openclaw 目錄）
openclaw init

# 3. 編輯配置檔（加入 API keys）
# ~/.openclaw/config.yaml
# - 設定 LLM provider（OpenRouter / Anthropic / OpenAI）
# - 設定訊息平台 token（Telegram Bot Token / WhatsApp session）

# 4. 啟動 Gateway
openclaw start

# 5. 連接 Telegram（最簡單）
# - 與 @YourBotName 開始對話即可

[來源：OpenClaw GitHub README, 2026; DigitalOcean Marketplace Docs, 2026]

Telegram 整合範例

Telegram 被稱為「easy mode」，因為它有正式 Bot API：


# 1. 向 @BotFather 申請 bot token
# 2. 在 config.yaml 加入：
channels:
  telegram:
    token: "YOUR_BOT_TOKEN"
    enabled: true

# 3. 重啟 openclaw，即可與 bot 對話

[來源：OpenClaw Telegram Docs, 2026]

WhatsApp 整合注意事項

WhatsApp 使用非官方協定（Baileys），需注意：

  • 14 天斷線問題：WhatsApp 可能在約 14 天後中斷 session，需重新掃碼 [來源：Blink.new Blog, 2026; TheCaio.ai Blog, 2026]
  • 安全風險：非官方協定可能違反 WhatsApp 服務條款，建議用測試帳號 [來源：Blink.new Blog, 2026]

潛在應用（對 daily-digest-prompt 專案的啟發）

  1. Multi-Channel 通知：目前專案僅用 ntfy.sh 單一通道，可參考 OpenClaw 的 Gateway 模式，讓摘要同時推送至 Telegram、LINE、Discord，使用者依情境選擇查看平台。
  1. Local-First Agent Pool：專案的 agent_pool/coordinator.py 若採用 OpenClaw 的沙箱模型（Docker/SSH），可提升多任務並行時的安全隔離（避免任務互相干擾或權限外洩）。
  1. Voice Integration：OpenClaw 的語音喚醒（macOS/iOS）與 TTS（Android）可啟發將每日摘要轉為語音播報，在通勤時收聽。
  1. 跨平台連續性：若使用者在桌機讀 Todoist 摘要、手機查看新聞，OpenClaw 的「跨平台 session 共享」機制可讓對話無縫接續（需實作使用者身份綁定）。
