Daily-Digest-Prompt Hooks 子系統優化（2026-03-07）

專案優化Hooks安全修復TDD測試覆蓋daily-digest-prompt

優化摘要

針對 daily-digest-prompt 專案的 Hooks 子系統進行 5 項品質優化，測試從 632 增至 675（+43），全部通過。

優化項目

1. 路徑遍歷安全漏洞修復（pre_read_guard.py）

  • 問題：_is_within_project 使用 startswith 字串比對，導致 daily-backup 被誤判為 daily 的子目錄
  • 修復：改用 resolved.startswith(norm_root + os.sep) 確保路徑分隔符安全比對
  • 新增測試：3 個（prefix collision attack、suffix collision、exact root path）

2. Schema 驗證同步（frequency-limits.schema.json）

  • 問題：frequency-limits.yaml 的 model 欄位在 JSON Schema 中缺失，導致驗證失敗
  • 修復：新增 model、default_model、max_auto_per_run 等欄位定義；template_params 改為 additionalProperties: true
  • 教訓：additionalProperties: false 雖防 typo，但每次新增欄位都需同步更新 schema

3. LoopDetector 防禦性反序列化（agent_guardian.py）

  • 問題：跨進程 JSON 狀態損壞時（如 content_hash_window=42），deque(42) 拋 TypeError
  • 修復：所有 initial_state 欄位加 isinstance() 型別檢查，非法值回退預設值
  • 新增測試：3 個（corrupted non-list window、non-int count、missing keys）

4. 通用 YAML 載入函式（hook_utils.py）

  • 新增：load_yaml_file(filename, fallback) 含模組層級快取 _yaml_file_cache
  • 重構：agent_guardian._load_loop_thresholds 改用共用函式，消除重複 YAML 載入邏輯
  • 新增測試：6 個（載入、fallback、快取、不同檔案、絕對路徑、清除快取）

5. on_stop_alert 整合測試補充

  • 新增：4 個測試覆蓋 _compute_error_budget 和 _check_slow_session 端對端場景
  • 修復：monkeypatch os.path.dirname 遞迴問題 — 必須先捕獲原始函式引用再 patch

技術洞察

  1. 路徑安全：startswith 做路徑比對是常見安全隱患，必須加 os.sep 確保目錄邊界
  2. 跨進程狀態：JSON 反序列化後的資料不可信，所有欄位都需 isinstance() 防禦
  3. monkeypatch 陷阱：patch 函式的 lambda fallback 中呼叫同名函式會無限遞迴，需先保存原始引用
  4. Schema 演化：additionalProperties: false 需搭配嚴格的 schema-config 同步流程

數據

  • 測試總數：675（hooks 648 + skills 27）
  • 新增測試：16 個
  • 修改檔案：8 個（4 source + 4 test）
  • 全部通過：675/675
