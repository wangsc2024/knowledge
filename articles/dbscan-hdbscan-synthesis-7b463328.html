一、演算法定位回顧（synthesis 視角）

1.1 根本設計差異

DBSCAN 與 HDBSCAN 的核心分歧不在「哪個更好」，而在問題定義本身：

| 維度 | DBSCAN | HDBSCAN |

|------|--------|----------|

| 設計哲學 | 固定密度閾值切割 | 多尺度密度層次，提取穩定核心 |

| 必要參數 | eps（半徑）+ min_samples | min_cluster_size + min_samples |

| 最難參數 | eps（高度敏感、需 k-NN 圖定位「膝點」） | min_samples（較不敏感） |

| 輸出 | 硬標籤（-1 為雜訊） | 硬標籤 + 軟歸屬機率（需 prediction_data=True） |

| 變密度能力 | 弱（同一 eps 無法跨密度） | 強（透過 MST condensation 自適應） |

| 新點預測 | 不支援（transductive only） | approximate_predict()（近似，需定期重訓） |

1.2 共同優勢（與 K-Means 等比較）

兩者均屬密度型演算法，共享的優勢：

  • 不需預先指定群數
  • 能辨識任意形狀的群（非凸）
  • 自動標記雜訊點（異常偵測天然支持）
  • 對尺度縮放不敏感

二、可擴展性實測基準

2.1 時間複雜度現實

| 時間預算 | DBSCAN 上限 | HDBSCAN 上限 | 備註 |

|----------|------------|-------------|------|

| 互動（30 秒） | ~75,000 點 | ~100,000 點 | 探索性分析 |

| 咖啡時間（5 分鐘） | ~250,000 點 | ~500,000 點 | 批次作業 |

| 午休（1 小時） | ~1,000,000 點 | ~1,000,000 點 | 日常排程 |

| 隔夜（8 小時） | ~2,500,000 點 | ~5,000,000 點 | 大型批次 |

結論：HDBSCAN 在相同時間預算下可處理的資料量約為 DBSCAN 的 1.3～2 倍。

2.2 記憶體複雜度危機

  • 44,000 點成對距離矩陣 ≈ 7.2 GB RAM（float64）
  • 生產緩解方案（三選一）：

1. 預計算稀疏鄰域圖（radius_neighbors_graph → metric='precomputed'）

2. 改用 OPTICS（O(n) 記憶體）

3. BIRCH 前置降維 + DBSCAN

2.3 GPU 加速效益（RAPIDS cuML）

| 資料集 | CPU 時間 | GPU 時間 | 加速倍數 |

|--------|---------|---------|--------|

| 70k 肺細胞 RNA（50 PC） | ~29 秒 | ~1 秒 | 29x |

| 400k 新聞標題（軟聚類） | ~17 小時 | ~1.3 秒 | ~47,000x |

| 3M×300 詞嵌入 | >24 小時（停止） | ~22.8 分鐘 | >60x |

cuML API drop-in replacement：


# CPU
from hdbscan import HDBSCAN
# GPU（cuML 26.04+）
from cuml.cluster import HDBSCAN

三、異常偵測應用框架

3.1 算法選擇矩陣

| 場景 | 推薦算法 | 原因 |

|------|---------|------|

| 已知正常密度，快速去噪 | DBSCAN | eps 可直接設定 |

| 多密度混合資料（如 KB 搜尋結果） | HDBSCAN | 自動處理稀疏/密集並存 |

| 需要異常分數排序 | HDBSCAN outlier_scores_ | 可接入告警優先級 |

| 高維嵌入向量（dim > 50） | HDBSCAN + UMAP 前置降維 | 保留拓樸 |

| 超大規模（>1M 點）實時偵測 | cuML HDBSCAN on GPU | CPU 路徑時間不可接受 |

3.2 HDBSCAN 異常分數範例


import hdbscan, numpy as np

clusterer = hdbscan.HDBSCAN(min_cluster_size=10, prediction_data=True)
clusterer.fit(X)
scores = clusterer.outlier_scores_
threshold = np.percentile(scores, 95)
anomalies = X[scores > threshold]
soft = hdbscan.all_points_membership_vectors(clusterer)
boundary_mask = soft.max(axis=1) < 0.3  # 低信心點

四、生產部署決策樹


資料規模？
├─ < 10k：任何算法均可；優先 HDBSCAN
├─ 10k–100k：密度均勻→DBSCAN；密度不均→HDBSCAN
├─ 100k–1M：有GPU→cuML HDBSCAN；純CPU記憶體充足→HDBSCAN；記憶體受限→稀疏+DBSCAN
└─ > 1M：GPU叢集(Dask+cuML)→HDBSCAN多GPU；純CPU→子採樣+approximate_predict

五、線上推理策略

HDBSCAN approximate_predict


clusterer = hdbscan.HDBSCAN(min_cluster_size=15, prediction_data=True)
clusterer.fit(X_train)
new_labels, strengths = hdbscan.approximate_predict(clusterer, X_new)

重要：資料漂移後預測偏差累積 → 每週或每 10% 資料量更新後重訓。

DBSCAN 新點（無原生支援，手工實作）


from sklearn.neighbors import NearestNeighbors
core_mask = np.zeros(len(X_train), dtype=bool)
core_mask[db.core_sample_indices_] = True
nn = NearestNeighbors(n_neighbors=1, radius=eps).fit(X_train[core_mask])
distances, indices = nn.kneighbors(X_new)
new_labels = np.where(distances.ravel() <= eps,
                      db.labels_[core_mask][indices.ravel()], -1)

六、daily-digest-prompt 專案適用場景


# KB 搜尋結果分群（向量嵌入天然不均勻密度→HDBSCAN）
def cluster_kb_results(embeddings, min_cluster_size=5):
    clusterer = hdbscan.HDBSCAN(
        min_cluster_size=min_cluster_size, metric='euclidean',
        cluster_selection_method='eom', prediction_data=True
    )
    return clusterer.fit_predict(embeddings), clusterer.outlier_scores_

# Token 使用異常偵測（時序特徵維度低→DBSCAN）
def detect_token_anomalies(features, eps=0.5):
    db = DBSCAN(eps=eps, min_samples=3, algorithm='ball_tree')
    return np.where(db.fit_predict(features) == -1)[0]

七、參數調優速查

DBSCAN

| 參數 | 起始值 | 調大效果 | 調小效果 |

|------|--------|---------|----------|

| eps | k-NN 膝點 | 群變大，雜訊減少 | 群破碎，雜訊增加 |

| min_samples | 2 * dimensions | 更嚴格，雜訊增加 | 更鬆散 |

HDBSCAN

| 參數 | 起始值 | 說明 |

|------|--------|------|

| min_cluster_size | 5-15（小）/50-100（大） | 最重要參數 |

| cluster_selection_method | 'eom'（預設） | leaf 更細碎；eom 穩健 |

| metric | 'euclidean'/'cosine' | 依資料性質 |

八、生產清單

  • [ ] eps 選擇：k-NN 距離圖膝點分析
  • [ ] 記憶體預算：估算稀疏圖而非 n² 矩陣
  • [ ] n > 50k 且 CPU：使用 precomputed metric
  • [ ] n > 100k 且有 GPU：評估 cuML HDBSCAN
  • [ ] HDBSCAN 線上推理：fit 時設 prediction_data=True
  • [ ] 重訓觸發：新點 label=-1 比例 > 20% 時重訓
  • [ ] 異常閾值：outlier_scores_ 百分位數（p95）而非固定值
  • [ ] 軟聚類不確定性：max(membership) < 0.3 標記「低信心」
  • [ ] 監控：每次執行記錄群數、雜訊比例、最大群大小
