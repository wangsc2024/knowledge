技術概述

ReDoS（Regular Expression Denial of Service）是利用回溯式正規表示式引擎的最壞情況複雜度，讓原本看似正常的驗證、過濾或路由規則在特製輸入下耗盡 CPU 的攻擊。OWASP 指出，典型成因是「重複中的重複」或「重疊選項」造成路徑爆炸；CodeQL 進一步說明，這類模式的時間複雜度可能呈多項式甚至指數級。對 daily-digest-prompt 而言，任何 Hook、防護規則、輸入驗證與日誌清理 regex 若缺乏結構約束，都可能在本地自動化流程中形成效能與可用性風險。

核心概念與原理

1. 災難性回溯如何發生

OWASP 將 Evil Regex 歸納為兩種典型結構：

  • 量詞包住另一個可重複子式，例如 (a+)+$
  • 在重複區塊內放入可重疊的 alternation，例如 (a|aa)+$

其根因不是 regex 本身「功能太多」，而是回溯引擎會在失敗時嘗試大量路徑。CodeQL 的說法更工程化：若 r* 或 r+ 的子式 r 能以多種方式匹配同一段字串，就會形成歧義，當尾段又刻意不匹配時，回溯成本會暴增。

2. Python 3.11 的直接防禦能力

Python 3.11 在標準 re 模組加入兩個重要語法：

  • possessive quantifiers：*+、++、?+、{m,n}+
  • atomic grouping：(?>...)

官方文件說明，possessive quantifier 會盡量匹配，但在後續失敗時「不允許回退」；atomic group 則在離開群組後丟棄內部堆疊點。這兩者都能直接切斷某些本來會爆炸的回溯路徑，是 Python 端最值得立刻採用的新能力。

3. 只改語法還不夠，還要補治理層

Semgrep 文件顯示，現在可用 metavariable-analysis 的 redos analyzer 直接偵測可疑 regex 反模式；CodeQL 也已有 ReDoS query，建議優先重寫歧義模式，若無法重寫，則改用保證線性時間的替代引擎。若專案需要 timeout，第三方 regex 模組已支援在 matching API 上設置 timeout=，可作為高風險場景的執行期護欄。

最佳實踐與常見陷阱

最佳實踐

  • 優先避免「巢狀量詞」與「重疊 alternation」，先改寫結構，再談優化。
  • 可以明確界定字元集合時，不要用過寬的 .；像 CodeQL 範例把 (__|.)+ 改成 (__|[^_])+，就是典型去歧義。
  • Python 3.11 以上，對高風險區段優先使用 possessive quantifier 或 atomic grouping。
  • 針對外部輸入參與的 regex，區分兩類風險：pattern 本身可爆炸，或使用者可注入惡意 pattern。後者要搭配 re.escape() 或禁止動態拼接。
  • 在 CI 增加 Semgrep / CodeQL 規則，避免修掉一處又在別處重新引入。
  • 對不可完全信任的複雜匹配，考慮改用支援 timeout 的 regex 模組，或使用 RE2 類線性時間引擎。

常見陷阱

  • 只測正常輸入，不測「幾乎匹配但最後失敗」的惡意樣本。
  • 以為非同步或多執行緒可以掩蓋問題；ReDoS 本質上仍是 CPU 耗盡。
  • 把 regex 當作萬用解析器，導致 pattern 不斷疊加例外分支，歧義越來越高。
  • 誤以為 Python 3.11 新語法會自動修好舊 pattern；實際上仍需人工重寫 pattern。

與本專案的關聯

今日專案優化已修復 hooks/pre_bash_guard.py 的 ReDoS 問題，這說明 daily-digest-prompt 的 Hook 層確實存在 regex 風險面。可直接採用的做法：

  • 盤點 hooks/、輸入驗證、路徑判定與日誌處理中的所有 regex。
  • 為每條安全相關 regex 建立一組「正常輸入 + 惡意近似輸入」基準測試。
  • 對高風險 pattern 優先改寫為較窄字元類別，必要時用 (?>...) 或 *+ 阻斷回溯。
  • 在 skill_audit 或 qa_optimize 流程中加入 Semgrep ReDoS 規則，將此風險前移到 PR / CI。

程式碼範例


import re

# 壞例子：巢狀量詞，容易在接近匹配但最後失敗時產生災難性回溯
VULN = re.compile(r"^(a+)+$")

# 較好的思路 1：直接改寫需求，避免歧義
SAFE_LITERAL = re.compile(r"^a+$")

# 較好的思路 2：Python 3.11+ 使用 atomic grouping / possessive quantifier
ATOMIC = re.compile(r"^(?>a+)+$")
POSSESSIVE = re.compile(r"^a++$")

# 動態拼接外部輸入時，至少先 escape
user_text = "a+b(c)"
SAFE_DYNAMIC = re.compile(rf"^{re.escape(user_text)}$")

若必須處理不可完全信任的複雜 pattern，可改用第三方 regex 模組：


import regex

pattern = regex.compile(r"(a+)+$")
try:
    pattern.fullmatch("a" * 5000 + "!", timeout=0.05)
except TimeoutError:
    print("疑似 ReDoS，已中止匹配")
