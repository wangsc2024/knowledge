一、專案概述

Gemini CLI（google-gemini/gemini-cli）是 Google 推出的開源終端 AI Agent，將 Gemini 模型的能力直接帶入開發者的命令列環境。採用 Apache 2.0 授權，以 TypeScript 開發，截至 2026 年 4 月已累積 100,466 Stars、12,934 Forks，是 GitHub 上增長最快的終端 AI Agent 專案之一。

核心數據

| 指標 | 數值 |

|------|------|

| Stars | 100,466 |

| Forks | 12,934 |

| 授權 | Apache 2.0 |

| 語言 | TypeScript |

| 主要模型 | Gemini 2.5 Pro（1M token context） |

| 免費額度 | OAuth: 60 req/min, 1000 req/day |

| 系統需求 | Node.js 20+（macOS/Linux/Windows） |

二、架構設計

Gemini CLI 採用前後端分離的模組化架構，分為三個核心套件：

2.1 CLI 套件（packages/cli）

負責使用者面向的操作：輸入處理、歷史管理、顯示渲染與主題自訂。這層專注於終端互動體驗，與後端邏輯完全解耦。

2.2 Core 套件（packages/core）

後端引擎，負責：

  • Gemini API 互動編排：prompt 建構、對話歷史管理
  • 工具註冊與執行：統一的工具調用介面
  • Session 狀態管理：支援對話檢查點與恢復
  • 作為 CLI 前端與外部 API 的中介層

2.3 工具系統（packages/core/src/tools/）

獨立模組擴展模型能力：

  • 檔案系統操作：讀寫、搜尋、目錄遍歷
  • Shell 命令執行：直接在終端執行命令
  • Web 擷取：抓取網頁內容
  • Google Search Grounding：即時搜尋整合
  • MCP Server 支援：透過 Model Context Protocol 擴展自訂工具

2.4 ReAct 迴圈（Reason-Act Loop）

Gemini CLI 的核心執行機制採用 ReAct 模式：

  1. 使用者輸入 → CLI 轉發至 Core
  2. Core 建構 prompt（含對話歷史 + 工具定義）→ 送往 Gemini API
  3. API 回應：直接回答 或 工具調用請求
  4. 工具執行：破壞性操作需使用者確認；唯讀操作自動執行
  5. 工具結果回傳 API → 生成最終回應
  6. Core 將結果送回 CLI 渲染

這與 Claude Code 的工具調用迴圈類似，但 Gemini CLI 更強調 Google Search Grounding 的即時資訊能力。

三、關鍵特性分析

3.1 MCP（Model Context Protocol）整合

Gemini CLI 是 MCP 生態的積極參與者，支援：

  • 本地 MCP Server：透過 ~/.gemini/settings.json 配置
  • 遠端 MCP Server：連接外部服務（GitHub、Slack、資料庫等）
  • FastMCP 整合：快速安裝與管理 MCP Server
  • 工具篩選：子 Agent 可選擇性使用部分工具（multi-registry 架構）

3.2 GEMINI.md 專案配置

類似 Claude Code 的 CLAUDE.md，Gemini CLI 使用 GEMINI.md 檔案：

  • 專案層級的行為自訂
  • 持久化配置，團隊共享
  • 定義專案特定的上下文與規則

3.3 免費存取策略

Google 的策略性定價極具競爭力：

  • OAuth（Google 帳號）：60 req/min、1,000 req/day — 對個人開發者幾乎無限制
  • API Key：100 req/day
  • 預設使用 Gemini 2.5 Pro（1M context），可切換至 Flash 等模型

3.4 2026 年 4 月最新更新

  • Multi-registry 架構：子 Agent 工具篩選
  • 動態模型解析：ModelConfigService
  • Session 恢復改善：對話檢查點更穩定
  • 沙箱能力增強：更安全的命令執行

四、終端 AI Agent 生態定位比較

| 特性 | Gemini CLI | Claude Code | Codex CLI |

|------|-----------|-------------|----------|

| 開源授權 | Apache 2.0 | 非開源 | 開源 |

| 上下文窗口 | 1M tokens | 200K tokens | 128K tokens |

| 免費額度 | 1000 req/day | 付費 | 付費 |

| MCP 支援 | ✅ | ✅ | ❌ |

| Google Search | ✅ 原生 | 需 MCP | ❌ |

| 語言 | TypeScript | TypeScript | Python |

| 專案配置 | GEMINI.md | CLAUDE.md | — |

Gemini CLI 的差異化優勢：

  1. 免費策略：1M context + 1000 req/day 免費，大幅降低入門門檻
  2. Google Search Grounding：原生即時搜尋能力，其他工具需額外配置
  3. 完全開源：Apache 2.0 允許商業使用與修改
  4. 1M Token Context：處理大型 codebase 的能力遠超競品

五、對 Daily-Digest-Prompt 系統的借鑑

  1. GEMINI.md 模式：驗證了「專案配置檔引導 Agent 行為」的設計方向（本系統的 CLAUDE.md）
  2. 前後端分離架構：CLI/Core 解耦模式可參考應用於 Agent 模組化
  3. Multi-registry 工具篩選：子 Agent 僅載入需要的工具，降低 token 消耗 — 與本系統的 allowed-tools 最小權限原則一致
  4. Session 檢查點：對話恢復機制可啟發連續記憶（continuity）的改進
  5. Google Search Grounding：即時搜尋整合值得在研究任務中善用

六、技術洞見

6.1 開源終端 Agent 的收斂趨勢

2026 年的終端 AI Agent 呈現明顯收斂：ReAct 迴圈 + MCP 擴展 + 專案配置檔（CLAUDE.md / GEMINI.md）已成為事實標準。Gemini CLI 的加入進一步驗證了這一架構模式的成熟度。

6.2 免費策略的戰略意義

Google 以 1M context + 免費額度切入，目標是快速擴大開發者生態。這對 Claude Code 構成直接競爭壓力，但也推動了整體市場的品質提升。

6.3 MCP 作為互操作標準

Gemini CLI 對 MCP 的全面支援（包含 FastMCP 整合）表明 MCP 正成為 AI Agent 工具擴展的通用標準。這與 Google A2A 協議形成互補：MCP 管工具調用，A2A 管 Agent 間通訊。
