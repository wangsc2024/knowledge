專案概述

LangGraph 是 LangChain Inc 開發的低層次編排框架，專為建構長期運行、有狀態的 AI Agent 設計。其核心設計理念來自 Google Pregel 和 Apache Beam，將 Agent 建模為有限狀態機（Finite State Machine），每個節點代表推理或工具使用步驟，轉換由輸出決定。已被 Klarna、Replit、Elastic 等知名企業採用。

---

核心特色與技術亮點

1. 持久執行（Persistent Execution）

  • Agent 可在故障中持續運行，自動從中斷處恢復
  • 適合需要高可靠性的企業場景

2. 人機協作（Human-in-the-Loop）

  • 任何執行階段皆可檢查和修改 Agent 狀態
  • 支援審批流程、人工干預等企業需求

3. 記憶管理（Memory Management）

  • 短期記憶：工作記憶，單次會話內有效
  • 長期記憶：跨會話持久化，支援用戶偏好等場景

4. 圖形化狀態機架構

  • 使用 StateGraph 定義節點和邊
  • 清晰的控制流，易於除錯和維護
  • 根據 2026 年基準測試，延遲和 token 使用量最低

5. 生產級可觀測性

  • 與 LangSmith 深度整合
  • 執行路徑追蹤、狀態轉換可視化
  • 支援針對有狀態長期工作流的可擴展基礎設施

安裝與快速上手

安裝


pip install -U langgraph

基本使用範例


from langgraph.graph import START, StateGraph
from typing_extensions import TypedDict

class State(TypedDict):
    text: str

def node_a(state: State) -> dict:
    return {"text": state["text"] + "a"}

def node_b(state: State) -> dict:
    return {"text": state["text"] + "b"}

# 建立圖形
graph = StateGraph(State)
graph.add_node("node_a", node_a)
graph.add_node("node_b", node_b)
graph.add_edge(START, "node_a")
graph.add_edge("node_a", "node_b")

# 編譯並執行
result = graph.compile().invoke({"text": ""})
print(result)  # {'text': 'ab'}

進階：帶條件分支的 Agent


from langgraph.graph import END

def should_continue(state: State) -> str:
    if len(state["text"]) > 3:
        return "end"
    return "continue"

graph.add_conditional_edges(
    "node_b",
    should_continue,
    {"continue": "node_a", "end": END}
)
