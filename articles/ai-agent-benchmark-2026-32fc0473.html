AI Agent 記憶評估 Benchmark 深度研究報告 2026

執行摘要

2026 年，AI Agent 記憶評估 benchmark 進入關鍵轉型期。LongMemEval、LoCoMo 和 AMA-Bench 三大評估框架各具側重：LongMemEval（ICLR 2025）專注五類長期記憶任務，SOTA 達 OMEGA 95.4%；LoCoMo（arXiv:2402.17753）測試超長跨 session 對話，全上下文基準線 ~73% 已反超多數專用系統；AMA-Bench（arXiv:2602.22769）走向真實 Agentic 場景，軌跡長度高達 57,506 tokens，揭示當前 SOTA 仍在 57-72% 區間。核心共識：記憶架構選擇比模型規模更重要（效能改善幅度 0.45 vs 0.038）；選擇性遺忘是所有系統最薄弱環節；業界正向 MemoryArena 式嵌入式 Agentic 評估遷移。生產環境結論：全上下文方案不可接受（9.87 秒延遲、14 倍 token 成本）；Episodic Memory（MemMachine 0.8487 on LoCoMo）與 Knowledge Graph（Zep/Graphiti 71.2% on LongMemEval）各有擅長。

三大 Benchmark 比較

LongMemEval（ICLR 2025）

  • 500 道問題，五類任務：資訊提取、多 session 推理、時序推理、知識更新、棄答
  • 商業助手長期記憶準確率下降 30%
  • 三階段框架：索引→檢索→閱讀（時序感知 query expansion）
  • 2026-04 SOTA：OMEGA 95.4%、Mastra 94.87%、Emergence AI 86%、Zep/Graphiti 71.2%

LoCoMo（arXiv:2402.17753）

  • 300 輪對話、9K tokens avg、最多 35 sessions
  • 根本局限：對話長度落在現代 LLM 上下文窗口內，全上下文基準線 ~73% 可超越多數專用系統
  • 2025-09 SOTA：MemMachine 0.8487 > Memobase 0.7578 > Zep 0.7514 > Mem0 Graph 0.6688 > LangMem 0.5810 > OpenAI 0.5290
  • Zep vs Mem0 爭議：75.14% vs 65.99%，差距來自實作缺陷（user model/timestamp/搜尋策略）

AMA-Bench（arXiv:2602.22769）

  • 2,496 真實 + 1,200 合成 QA，6 個 Agentic 域
  • 軌跡長度高達 57,506 tokens（真實）至 128K（合成）
  • SOTA：AMA-Agent 57.22%、GPT-5.2 72.26%
  • 記憶架構貢獻 0.45 效能改善，模型規模僅 0.038

關鍵洞見（三角佐證）

  1. LoCoMo 正走向過時：業界評估重心向 MemoryArena 式嵌入式 Agentic 評估遷移（3 來源驗證 ✅）
  2. 記憶架構比模型規模重要：0.45 vs 0.038 效能差距（3 來源驗證 ✅）
  3. 選擇性遺忘是最弱能力：無系統能掌握全部四項認知能力（3 來源驗證 ✅）
  4. 被動召回 ≠ 主動決策記憶：LoCoMo 高分模型在 Agentic 任務降至 40-60%（3 來源驗證 ✅）
  5. Benchmark 五大盲點：單 session 偏差、遺忘缺失、效率指標缺席、記憶類型未分離、對抗性測試缺失（3 來源驗證 ✅）

分歧點

  • Zep vs Mem0 LoCoMo 分數爭議：65.99% vs 75.14%（差距來自三個實作缺陷）
  • 專用記憶系統是否優於全上下文：LoCoMo 存疑，LongMemEval/AMA-Bench 明顯優勢

生產選型建議

  • 多 session 對話持久化 → Knowledge Graph（Zep/Graphiti）
  • 高頻 Agentic 任務 → Episodic Memory（MemMachine 範式，單跳 93.3%）
  • 成本敏感場景 → Mem0 vector（66.9%、0.71s latency）
  • 拒絕全上下文方案（9.87s、14x token 成本）
  • 必須實施選擇性遺忘機制（所有系統最弱環節）
  • 不以 LoCoMo 單一分數作選型依據
