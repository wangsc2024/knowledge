一、專案概述

Crawl4AI 是一個完全開源的 Python 網頁爬蟲引擎，專為 LLM（大型語言模型）、AI Agent 和 RAG（Retrieval-Augmented Generation）管線設計。其核心使命是：將混亂複雜的網頁內容轉化為乾淨、結構化的 Markdown 格式，讓 AI 應用能直接消費。

  • GitHub：github.com/unclecode/crawl4ai
  • Stars：60,300+（2026-02-18）
  • Forks：6,100+
  • 提交數：1,216+
  • 授權：Apache-2.0（寬鬆商業友善）
  • 最新版本：v0.8.0
  • 語言：Python

核心價值

  1. LLM 優先設計：輸出的 Markdown 經過最佳化，比原始 HTML 減少約 67% 的 token 消耗
  2. 完全自託管：不需要外部 API 金鑰或帳戶，資料永遠在你的基礎設施內
  3. 高效能：異步架構（asyncio + Playwright），支援並行爬取
  4. 零成本入門：開源免費，僅當使用 LLM 擷取功能時才有外部 LLM 成本

二、技術架構

核心技術棧

| 層級 | 技術 | 用途 |

|------|------|------|

| 瀏覽器引擎 | Playwright | 自動化瀏覽器控制（支援 Chromium） |

| 內容處理 | 自訂 Markdown 生成器 | FitMarkdown 雜訊過濾 |

| 資料擷取 | CSS/XPath + LLM 驅動 | 結構化資料提取 |

架構設計亮點

  1. 管線式處理：URL → 瀏覽器渲染 → JavaScript 執行 → 內容清理 → Markdown 生成 → 結構化擷取
  2. 會話管理：支援有狀態的多步驟爬取（登入、導航、資料收集）
  3. 瀏覽器池：企業級部署支援多個瀏覽器實例的池化管理
  4. 故障恢復：v0.8.0 新增 resilience 機制，自動重試失敗的爬取
  5. 預取模式：v0.8.0 新增頁面預取，大幅提升批量爬取效能

三、功能特色

3.1 Markdown 生成（核心賣點）

  • FitMarkdown：智慧過濾網頁雜訊（廣告、導航列、頁尾），只保留主要內容
  • PruningContentFilter：可調整過濾閾值（threshold），控制內容精度
  • 引文保留：自動將超連結轉為 Markdown 格式 text
  • 程式碼區塊：自動偵測並保留 和 
     區塊

3.2 結構化資料擷取

三種擷取策略：

  1. CSS/XPath 模式匹配：定義 JSON Schema，指定 selector 和 field type
  2. LLM 驅動擷取：將內容餵給 LLM（如 GPT-4o），取得結構化 JSON
  3. 混合模式：先用 CSS 縮小範圍，再用 LLM 精煉

# JSON Schema 定義結構化擷取
schema = {
    "name": "Products",
    "baseSelector": ".product-item",
    "fields": [
        {"name": "title", "selector": "h3", "type": "text"},
        {"name": "price", "selector": ".price", "type": "text"}
    ]
}

3.3 進階爬取能力

  • 深度爬取（BFS）：從起始 URL 按廣度優先搜尋遍歷整個網站
  • 無限滾動處理：模擬捲動以載入動態內容（適用 SPA 和懶載入頁面）
  • iframe 內容擷取：深入 iframe 取得嵌入內容
  • 截圖與 PDF：爬取同時保存頁面快照
  • 代理（Proxy）支援：整合代理伺服器以繞過地域限制

3.4 企業級功能（v0.7.7+）

  • 即時監控儀表板：Web UI 顯示爬取進度、成功率、效能指標
  • 瀏覽器池管理：動態擴縮瀏覽器實例
  • session 持久化：跨請求保持登入狀態

四、使用方式

4.1 安裝


# 基礎安裝
pip install -U crawl4ai
crawl4ai-setup  # 下載 Playwright 瀏覽器

# Docker 部署
docker pull unclecode/crawl4ai:latest
docker run -d -p 11235:11235 --shm-size=1g unclecode/crawl4ai:latest

4.2 基礎用法


import asyncio
from crawl4ai import AsyncWebCrawler

async def main():
    async with AsyncWebCrawler() as crawler:
        result = await crawler.arun(url="https://example.com")
        print(result.markdown)  # 乾淨的 Markdown

asyncio.run(main())

4.3 CLI 快速爬取


# 單頁爬取輸出 Markdown
crwl https://docs.example.com -o markdown

# 深度爬取（BFS 模式，最多 10 頁）
crwl https://docs.crawl4ai.com --deep-crawl bfs --max-pages 10

4.4 帶 JavaScript 執行的爬取


config = CrawlerRunConfig(
    js_code="window.scrollTo(0, document.body.scrollHeight);",
    wait_for="css:.lazy-loaded-content",
    page_timeout=30000
)
result = await crawler.arun(url=url, config=config)

五、與同類工具比較

Crawl4AI vs Firecrawl（72K stars）

| 面向 | Crawl4AI | Firecrawl |

|------|----------|----------|

| 架構 | Python Library（自託管） | API-first（雲端優先） |

| 授權 | Apache-2.0（寬鬆） | AGPL-3.0（限制性） |

| 定價 | 完全免費（LLM 成本外部） | 免費層 + 付費訂閱 |

| 上手難度 | 需 Python 經驗 | 圖形介面 + 自然語言查詢 |

| 擴展性 | 自行管理水平擴展 | 內建 serverless 擴展 |

| 輸出品質 | 需調校 PruningContentFilter | Extract 版即用格式 |

| MCP 整合 | 社群 MCP Server | 官方 MCP Server |

| 適合場景 | 深度客製化、自建基礎設施 | 快速原型、企業團隊 |

Crawl4AI vs Jina Reader

  • Crawl4AI 提供完整的爬蟲工作流（爬取 + 渲染 + 擷取 + 結構化）
  • Jina Reader 專注於單頁 Markdown 轉換（r.jina.ai/URL），更輕量但功能有限

效能實測數據

  • 基本 HTML 爬取（無 LLM）：約 2 秒/頁
  • LLM 結構化擷取（GPT-4o）：約 25 秒/頁
  • 準確率：配合 LLM 時達 100%（測試集 20 項全數正確提取）

九、優缺點評估

優點

  1. Apache-2.0 授權：商業使用無限制，比 Firecrawl 的 AGPL-3.0 更友善
  2. 完全自託管：資料安全性有保障，無需第三方 API
  3. LLM 原生設計：Markdown 輸出直接可用，token 效率高
  4. 深度客製化：從瀏覽器控制到內容過濾都可調校
  5. Docker 一鍵部署：企業級部署簡便
  6. MCP 生態系：與 Claude Code 等 AI 工具整合良好
  7. 活躍社群：GitHub 增長迅猛，持續維護

缺點

  1. 需 Python 技能：非開發者難以上手（無圖形介面）
  2. 水平擴展需自理：大規模爬取需要自行管理瀏覽器叢集
  3. LLM 成本隱藏：結構化擷取依賴外部 LLM，成本不可忽視
  4. 反爬措施應對：缺乏內建的 CAPTCHA 破解和指紋規避
  5. Windows 支援：主要針對 Linux/Mac，Windows 需要 Docker 或 WSL
  6. 文件待完善：部分進階功能文件不足
