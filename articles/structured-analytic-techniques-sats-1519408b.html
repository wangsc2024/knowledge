結構化分析技術（Structured Analytic Techniques, SATs）

主題概述

結構化分析技術（SATs）源自美國情報界（CIA/DIA），由 Richards J. Heuer Jr. 與 Randolph H. Pherson 系統化整理為八大類別、六十六種技術的完整分類體系。這些技術的核心目的是：透過外部化的結構化流程，矯正人類固有的認知偏誤（cognitive biases），減少分析錯誤的頻率與嚴重度。2026 年，隨著 AI 決策系統普及，SATs 從情報分析擴展至企業策略、競爭情報、網路安全與個人決策領域，成為邏輯思維方法論中最具系統性的實踐框架。

與知識庫既有框架的定位差異：

  • Paul-Elder 框架 → 思維品質的「審查標準」（What makes good thinking?）
  • 心智模型（Munger/Farnam Street）→ 跨學科「問題解決工具箱」
  • SATs → 團隊/組織層級的「偏誤矯正程序」（How to prevent bad thinking?）

SATs 填補了既有框架在「組織層級認知偏誤系統性矯正」上的空白。

---

一、Heuer-Pherson 八大類別分類體系

| 類別 | 目的 | 代表技術 |

|------|------|----------|

| 1. 分解與視覺化 | 將複雜問題拆解為可管理的組件 | 時間軸分析、網路分析、心智圖 |

| 2. 構想生成 | 突破思維定勢產生新想法 | 腦力激盪、名義群體技術、形態分析 |

| 3. 情境與指標 | 預測未來可能的發展路徑 | 替代未來分析、情境樹、指標驗證 |

| 4. 假設生成與測試 | 系統性評估多個競爭假設 | ACH、多假設生成器、診斷推理 |

| 5. 因果分析 | 探究事件的根本原因 | 時間軸因果圖、關鍵驅動因素分析 |

| 6. 挑戰分析 | 主動質疑既有結論 | 魔鬼代言人、紅隊分析、結構化自我批判 |

| 7. 衝突管理 | 在分歧中找到共識路徑 | 利害關係人分析、SWOT 矩陣 |

| 8. 決策支援 | 提供結構化的決策框架 | 決策矩陣、力場分析、專家引導 |

第三版（2019）擴展至 66 種技術，涵蓋情報、執法、國土安全與商業分析的最佳實踐。

---

二、六大核心技術詳解

技術 1：關鍵假設檢查（Key Assumptions Check, KAC）

類型：診斷技術

目的：識別並挑戰支撐判斷的隱性假設，防止假設崩潰時結論瓦解。

三步驟流程：

  1. 識別假設：將結論語言化，用 W 問題（Who/What/When/Where/Why/How）拆解為組成部分
  2. 挑戰有效性：對每個假設評級 — Supported（S，有證據支撐）、Caveat（C，有條件成立）、Unsupported（U，無證據支撐）
  3. 精煉清單：移除不受支撐的假設，持續評估新浮現的假設

實務應用：任何重大決策前的第一步 — 在投入 ACH 或紅隊分析之前，先確保基礎假設經過檢驗。

矯正的偏誤：確認偏誤（confirmation bias）、錨定效應（anchoring）、第一印象過度依賴。

技術 2：競爭假設分析（Analysis of Competing Hypotheses, ACH）

類型：假設測試技術

開發者：Richards J. Heuer Jr.（CIA 資深分析師）

核心原理：優先尋找反證（disconfirming evidence），而非確認證據。一條強力反證可淘汰整個假設。

七步驟流程：

| 步驟 | 動作 | 關鍵要點 |

|------|------|----------|

| 1 | 定義問題 | 中性措辭，不預設因果 |

| 2 | 列出所有合理假設 | 包含令人不舒服的可能性，防止盲區 |

| 3 | 蒐集證據與論點 | 多源交叉驗證，評估來源可靠度 |

| 4 | 分析一致性 | 建立「假設×證據」矩陣，標記 C（一致）/ I（不一致）/ N（中性） |

| 5 | 精煉矩陣 | 聚焦「診斷價值」最高的證據（能區分假設的關鍵證據） |

| 6 | 得出暫定結論 | 選擇「不一致證據最少」的假設，而非「支持證據最多」的 |

| 7 | 設定監控指標 | 定義後續觀察事件，防止分析漂移 |

ACH 矩陣範例：

| 證據 | 假設 A | 假設 B | 假設 C |

|------|--------|--------|--------|

| 證據 1 | C | I | N |

| 證據 2 | C | C | I |

| 證據 3 | I | C | C |

| 證據 4 | N | I | C |

| 不一致數 | 1 | 2 | 1 |

→ 假設 B 因不一致證據最多而被降低可信度；假設 A 與 C 需進一步區分。

矯正的偏誤：確認偏誤、過早封閉（premature closure）、群體迷思。

局限性：

  • 對全新情境（無可比較數據）效果較差
  • 成效取決於初始假設的全面性
  • 研究顯示 ACH 可能增加判斷不一致性（Dhami, 2019），建議搭配其他技術使用

技術 3：魔鬼代言人（Devil's Advocacy）

類型：挑戰/對抗技術

歷史：源自 16 世紀天主教封聖程序中的「Advocatus Diaboli」，專門反駁候選人聖德的官方角色。

三步驟流程：

  1. 概述現行觀點：識別有缺陷的假設、有問題的邏輯、可疑的證據、遺漏的替代假設
  2. 選擇與審查：挑選最容易被挑戰的面向 — 驗證證據效力、識別遺漏資訊、評估欺騙可能性
  3. 提出替代方案：為競爭假設列出證據，清楚區分對抗立場與主流觀點

矯正的偏誤：群體迷思（groupthink）、虛假共識效應、社會從眾壓力。

與紅隊分析的區別：魔鬼代言人從內部質疑自身判斷；紅隊則模擬外部對手的視角。

---

技術 4：紅隊分析（Red Team Analysis）

類型：挑戰/對抗技術

起源：美軍與情報機構，用於克服認知偏誤與群體迷思。

核心方法：指定一組人員採用對手（adversary）的視角，模擬對手可能的反應、欺騙手段或攻擊路徑。

應用場景：

  • 軍事：模擬敵方戰術
  • 網路安全：滲透測試（penetration testing）
  • 企業：模擬競爭對手策略
  • 政策：預測反對方的論點

與 AI 紅隊（2026 趨勢）：紅隊概念已延伸至 AI 安全測試 — 對 LLM 進行對抗性提示測試，識別有害輸出、偏見與安全漏洞。16 世紀的「魔鬼代言人」概念正在被 AI 紅隊直接繼承。

矯正的偏誤：想像力失敗（failure of imagination）、鏡像思維（mirror imaging，假設對手和自己想法一樣）。

---

技術 5：結構化自我批判（Structured Self-Critique）

類型：挑戰技術（預驗屍法 Premortem 的進化版）

核心差異：傳統預驗屍法問「如果失敗了，原因是什麼？」；結構化自我批判提供四維度審查框架。

四維度審查：

  1. 不確定性來源：是否存在多個正確答案？情境是否仍在演變？有無替代假設？
  2. 關鍵假設：哪些假設如果被證偽，對結果影響最大？
  3. 證據評估：支持決策的證據有多充分？存在哪些資訊缺口？
  4. 環境因素：哪些外部變數（社會、政治、法律、倫理）超出控制？

矯正的偏誤：過度自信（overconfidence）、後見之明偏誤（hindsight bias）。

---

技術 6：欺騙偵測（Deception Detection）

類型：診斷技術

核心：四組助記詞構成的檢查清單，系統性識別蓄意誤導。

四組檢查框架（MOM-POP-MOSES-EVE）：

| 框架 | 全稱 | 檢查要點 |

|------|------|----------|

| MOM | Motive, Opportunity, Means | 潛在欺騙者是否有動機、機會與手段？ |

| POP | Past Opposition Practices | 對方過去是否有欺騙前例？ |

| MOSES | Manipulability of Sources | 資訊來源是否可能被操控？ |

| EVE | Evaluation of Evidence | 整個資訊鏈（採集→分析）是否有可疑之處？ |

2026 應用場景：深偽影片辨識、AI 生成假新聞檢測、社交工程攻擊防禦。

三、SATs 的認知偏誤矯正對照表

| 認知偏誤 | 定義 | 對應 SAT 矯正技術 |

|---------|------|-------------------|

| 確認偏誤 | 只尋找支持現有信念的證據 | ACH、魔鬼代言人 |

| 錨定效應 | 過度依賴第一條資訊 | 關鍵假設檢查 |

| 群體迷思 | 團隊追求一致而壓制異議 | 紅隊分析、魔鬼代言人 |

| 過早封閉 | 過快得出結論、停止搜集資訊 | ACH（強制保持多假設） |

| 可得性偏誤 | 過度依賴容易想起的資訊 | 替代未來分析 |

| 鏡像思維 | 假設對手的想法與自己相同 | 紅隊分析 |

| 過度自信 | 高估自身判斷的準確度 | 結構化自我批判 |

| 後見之明偏誤 | 事後認為結果是可預見的 | 預驗屍法、指標追蹤 |

四、SATs 在商業與個人決策中的應用（2026 趨勢）

4.1 企業策略應用

  • 競爭情報：用 ACH 評估競爭對手的下一步動作
  • 投資決策：用關鍵假設檢查審查投資論點的隱性假設
  • 產品上市：用替代未來分析預測市場反應的多種情境

4.2 AI 整合趨勢

  • AI 紅隊：用結構化方法測試 LLM 的安全性與偏見
  • 人機協作：AI 負責資訊蒐集與模式辨識，人類運用 SATs 做最終判斷
  • 自動化 ACH：工具如 Palantir、i2 Analyst's Notebook 已整合 ACH 矩陣自動化

4.3 個人決策應用

  • 職涯決策：用 ACH 矩陣比較多個職涯路徑的證據
  • 重大購買：用關鍵假設檢查審查自己的購買動機
  • 人際衝突：用魔鬼代言人法強迫自己考慮對方立場

五、SATs 實踐指南：何時選用哪種技術

| 情境 | 推薦技術 | 理由 |

|------|---------|------|

| 初步分析、假設尚未形成 | 關鍵假設檢查 | 先確保基礎穩固 |

| 多個解釋並存、需要系統比較 | ACH | 矩陣化比較最有效 |

| 團隊已形成共識、擔心盲區 | 魔鬼代言人 | 從內部質疑共識 |

| 面對對手或競爭者 | 紅隊分析 | 模擬對手視角 |

| 高風險決策前的最後審查 | 結構化自我批判 | 四維度全面審查 |

| 懷疑資訊來源的可靠性 | 欺騙偵測 | MOM-POP-MOSES-EVE 清單 |

| 未來高度不確定 | 替代未來分析 | 情境矩陣探索多種可能 |

六、與知識庫既有思維框架的整合地圖

| 既有框架 | SATs 的互補角色 |

|---------|----------------|

| 第一性原理思維 | 第一性原理拆解問題，SATs 的 KAC 確保拆解的假設有效 |

| 逆向思維/預驗屍法 | 預驗屍法是 SATs 的子集，結構化自我批判是其進化版 |

| 貝葉斯推理 | 貝葉斯更新先驗機率，ACH 提供系統性的「反證搜索」結構 |

| 系統思考 | 系統思考看反饋迴路，SATs 的因果分析提供具體操作工具 |

| 後設認知 | 後設認知監控「是否在正確思考」，SATs 提供「如何確保正確思考」的程序 |

| 類比推理 | 類比推理跨域遷移，紅隊分析可檢驗類比映射是否真正成立 |

| 決策疲勞 | 決策疲勞說明「何時」判斷力下降，SATs 提供「無論何時」都能執行的結構化流程作為防護網 |
