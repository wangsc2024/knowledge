二、核心洞見：四維「專家路由」同構模型

洞見 1：MoE 專家路由 ≅ Skill-First 路由精簡

Flash-MoE 的突破在於：397B 參數模型中，每次推理只需載入 4 個專家（~3.9MB each），其餘 expert 保留在 SSD 上按需串流 [來源：Flash-MoE GitHub, 2026]。這與本專案 Skill-First 策略的精簡方向完全同構——從 62 個 Skill 中，依任務類型路由到「必用」子集（約 10 個核心 Skill），其餘按需載入 [來源：Daily-Digest-Prompt Skill 驅動架構優化策略, 2026-03-23]。

深層結構：兩者共享同一個設計原理——稀疏激活（Sparse Activation）。Flash-MoE 用 2-bit 量化專家 + 全精度路由矩陣；Skill-First 用「必用/積極用/搭配用」三層路由。核心都是：路由器本身必須輕量且精確，被路由的專家可以重量但按需載入。

洞見 2：十乘觀法的「觀境路由」

天台圓教十乘觀法的修證次第，本質上也是一種「專家路由」：以「觀不思議境」為入口（相當於路由器的首次分類），然後依修行者根機（input context），動態調度後續九乘觀法 [來源：十乘觀法 — 天台圓教止觀修證次第, 2026-03-23]。

利根者：第一乘即悟，不需後續觀法（≈ MoE 中 top-1 expert 即足夠）

中根者：需 2-3 乘觀法配合（≈ top-4 experts）

鈍根者：十乘全用（≈ 全量載入，但效率最低）

這個同構揭示了一個跨域原則：最優系統不是功能最多的系統，而是能根據「輸入根機」動態調整「激活深度」的系統。

洞見 3：Agency-Agents 的組織路由

Agency-Agents 將 147 個 AI Agent 角色組織為 12 個部門（Engineering 23、Marketing 26、Game Dev 26 等），用 Markdown + YAML frontmatter 定義每個角色的專業邊界 [來源：Agency-Agents 深度研究, 2026-03-23]。這是「專家路由」在組織設計層面的實現：

部門 = MoE 的 expert group

角色 frontmatter = expert 的 routing weights

convert.sh 四策略轉換 = 跨平台的 inference backend 適配

洞見 4：快取驅逐即「專家淘汰」

快取命中率優化研究中的 ARC（Adaptive Replacement Cache）演算法，本質是在有限記憶體中決定「哪些資料值得常駐、哪些該被驅逐」[來源：快取命中率優化洞察簡報, 2026-03-23]。這與 MoE 的 expert selection、Skill-First 的路由精簡、十乘觀法的「利根者只需第一乘」，都指向同一個優化問題：在有限頻寬/記憶體/注意力下，最大化命中率。

三、次要洞見：「mechanism 階段」的知識累積效應

2026-03-23 有 4 篇筆記明確標記為 mechanism 階段：

十乘觀法（天台圓教止觀的內部原理）

OpenClaw Gateway（WebSocket 路由與並發控制實作）

skill-forge 核心機制（評分算法與去重機制）

快取命中率優化（TTL 動態調整與 ARC 驅逐策略）

這代表知識庫正從「是什麼」（foundation）系統性地推進到「怎麼運作」（mechanism）。依 kb-research-strategist 的五階段模型（foundation→mechanism→application→optimization→synthesis），多個研究系列同步進入第二階段，預示接下來將迎來 application 階段的爆發期。

四、外部視角補充

擴散語言模型的產業衝擊

外部搜尋驗證了知識庫中擴散語言模型研究的前沿性。Mercury Coder 達到 1,109 tokens/sec（H100），比自迴歸模型快 10 倍 [來源：Inception Labs Blog, 2026]。LLaDA 在 Winograd Schema 挑戰中提升 22%，並解決了逆向詛咒（reversal curse）問題 [來源：arxiv 2502.09992, 2026]。dLLM 統一工具包（2026-03 發布）整合 Fast-dLLM 實現 2-4 倍加速 [來源：BrightCoding, 2026-03-19]。

MCP 生態系進入企業級採用

知識庫的 MCP 全景分析與外部趨勢一致：2026 年 MCP 從實驗階段轉入企業級採用，1,000+ 社群 MCP Server，Salesforce/ServiceNow/Workday 等 50+ 合作夥伴 [來源：CData Blog, 2026]。Autodesk 主導了 MCP 的企業安全規範 [來源：Autodesk News, 2026]。MCP v1.27 加入 transport scalability 與 agent communication [來源：Context Studios Blog, 2026]。

Flash-MoE 的邊緣推理革命

Flash-MoE 在 MacBook Pro（48GB RAM）上以 5.5 tokens/sec 運行 Qwen3.5-397B，只用 5.5GB 常駐記憶體 [來源：AI for Automation, 2026-03-19]。技術基礎來自 Apple 2023 年論文「LLM in a Flash」，整個引擎由 Claude Code 在 24 小時內自主建構 [來源：Simon Willison Blog, 2026-03-18]。這驗證了「稀疏激活 + 快速儲存串流」的範式可行性，與本專案的 Skill 按需載入哲學高度呼應。

五、跨領域啟示與建議行動

5.1 對本專案的啟示

Skill 路由器應學習 MoE 的 top-k 策略：目前 routing.yaml 是靜態規則，可考慮引入輕量評分機制（類似 MoE 的 gating network），根據任務描述動態選擇 top-3 Skill

mechanism 階段飽和預警：4 個系列同時進入 mechanism 階段，建議下週優先推進 application 階段研究，避免 mechanism 分析過度集中

快取策略可借鏡 ARC 的自適應性：目前 cache-policy.yaml 使用固定 TTL，可考慮類似 ARC 的頻率/時近雙因子動態調整

5.2 對知識體系的啟示

「觀境路由」可作為跨域思維工具：十乘觀法的「根機→激活深度」模型，可推廣為通用的「資源受限系統設計原則」

稀疏激活是 2026 年的核心設計範式：從 MoE 到 Skill 到佛學觀法，「不全量載入、按需精準路由」已成為跨領域共識

類比推理 + 第一性原理 = 本報告的方法論基礎：昨日新增的兩篇認知科學筆記（類比推理的結構映射、第一性原理的三步框架）恰好是產出本洞見報告的元方法
