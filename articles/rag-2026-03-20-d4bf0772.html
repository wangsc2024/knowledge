一、現況診斷：兩大結構性問題

1.1 日誌路徑嵌套（P2 — 資料完整性）

症狀：logs/structured/ 目錄下產生多達 8 層嵌套結構：


logs/structured/logs/structured/logs/structured/.../2026-03-10.jsonl

根因分析：

  • post_tool_logger.py（line 554）使用相對路徑：os.path.join("logs", "structured")
  • on_stop_alert.py 多處（line 66, 112, 124, 367, 421）同樣使用相對路徑
  • hook_utils.py（line 266）的 log_blocked_event() 也使用相對路徑
  • 當 CWD 漂移至 logs/structured/ 時，os.makedirs 會在當前目錄下建立嵌套結構
  • 嵌套目錄總計 38KB（6 個 JSONL 檔案），對日誌搜尋造成干擾

1.2 State 檔案膨脹（P3 — 磁碟衛生）

症狀：219 個 loop-state-.json + 48 個 stop-alert-.json，state/ 目錄 1.1MB

根因：每次 Agent session 產生新 loop-state，清理機制保留 7 天但產生速度更快。

---

二、修復方案與實施

2.1 日誌路徑絕對化（核心修復，3 檔 8 處）

#### hooks/post_tool_logger.py（1 處）


from hook_utils import get_project_root
_log_root = get_project_root()
log_dir = os.path.join(_log_root, "logs", "structured")

#### hooks/on_stop_alert.py（6 處）

新增模組級常數 _PROJ_ROOT，替換所有相對路徑引用。

#### hooks/hook_utils.py（1 處）

log_blocked_event() 改用 get_project_root() 構建絕對路徑。

2.2 測試強化（2 檔案）

  • test_on_stop_alert.py：新增 autouse fixture 自動 monkeypatch _PROJ_ROOT
  • test_hook_utils.py：TestLogBlockedEvent 新增 fixture monkeypatch get_project_root

2.3 現場清理

  • 刪除嵌套 logs/structured/logs/ 目錄（38KB）
  • 清理 10 個超過 48 小時的 state 過期檔案

四、設計決策

  • 模組級常數：_PROJ_ROOT 載入時計算一次，Hooks 每次 tool call 觸發，效能敏感
  • 根本預防：所有 hooks 中寫入路徑必須使用絕對路徑
