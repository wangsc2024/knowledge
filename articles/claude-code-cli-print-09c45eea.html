一、技術概述

Claude Code CLI 是 Anthropic 提供的命令列介面工具，讓開發者能在終端機環境中與 Claude AI 互動。核心特性包括：

  • Print Mode (-p)：非互動式執行模式，執行單一指令後自動退出
  • Agent SDK：提供 CLI、Python、TypeScript 三種介面的程式化控制
  • 工具自動批准：透過 --allowedTools 參數實現無人值守自動化
  • 結構化輸出：支援 JSON、Stream-JSON 等機器可讀格式
  • Session 管理：跨次對話的狀態保持與 Context 延續

Claude Code v2.1+ 已將原稱「headless mode」統一為「print mode」，所有 CLI 選項保持向後相容。

二、核心概念與執行模式

2.1 Print Mode 工作原理

Print mode 透過 -p 或 --print 旗標啟動，執行單一查詢並輸出結果後立即退出：


claude -p "Generate a Python quicksort function"

關鍵特性：

  • 退出碼：成功為 0，錯誤為 1（可用於 shell 腳本判斷）
  • 標準輸出：預設輸出至 stdout，可重導向至檔案
  • 非互動式：無需啟動 REPL session，適合腳本化

2.2 輸出格式控制

透過 --output-format 參數控制回應格式：

| 格式 | 用途 | 範例 |

|------|------|------|

| text | 預設純文字輸出 | Shell 腳本直接讀取 |

| json | 結構化 JSON（含 metadata） | 機器可解析，包含 session ID、成本、用量 |

| stream-json | 逐行 NDJSON | 即時串流監控，搭配 jq 過濾 |

JSON 輸出範例：


claude -p "Summarize this project" --output-format json

回應結構：


{
  "result": "專案摘要內容...",
  "session_id": "abc123...",
  "cost_usd": 0.03,
  "duration_ms": 2345,
  "num_turns": 1
}

Structured Output（JSON Schema 驗證）：


claude -p "Extract function names from auth.py" \
  --output-format json \
  --json-schema '{"type":"object","properties":{"functions":{"type":"array","items":{"type":"string"}}},"required":["functions"]}'

回應包含 structured_output 欄位（而非 result），內容符合指定 schema。

2.3 串流輸出（Real-Time Streaming）

結合 --output-format stream-json、--verbose、--include-partial-messages 實現 token 級串流：


claude -p "Write a poem" --output-format stream-json --verbose --include-partial-messages | \
  jq -rj 'select(.type == "stream_event" and .event.delta.type? == "text_delta") | .event.delta.text'

應用場景：

  • CI/CD 管線的即時進度顯示
  • 長時間運算的中間結果監控
  • WebSocket 推送的後端整合

三、常見使用模式與範例

3.1 Pipe 管線整合

Claude CLI 支援標準輸入，可與 Unix 工具鏈無縫整合：


# Pipe file content
cat src/utils.ts | claude -p "Add missing TypeScript types"

# Pipe git diff for code review
git diff HEAD~1 | claude -p "Review this diff, list potential issues"

# Pipe logs for analysis
tail -n 100 /var/log/app.log | claude -p "Identify error patterns"

3.2 工具自動批准（Auto-Approve Tools）

透過 --allowedTools 參數實現無人值守執行：


# 允許 Bash、Read、Edit 工具
claude -p "Run the test suite and fix any failures" \
  --allowedTools "Bash,Read,Edit"

# 使用權限規則語法（Permission Rule Syntax）
claude -p "Create a commit for staged changes" \
  --allowedTools "Bash(git diff *),Bash(git log *),Bash(git commit *)"

權限規則語法：

  • Bash(git diff ) — 允許所有以 git diff 開頭的命令（注意空格 + ）
  • Read(.env) — 允許讀取特定檔案
  • Write(src/**) — 允許寫入 src 目錄下所有檔案

安全注意事項：

  • 避免 Bash(rm *) 等危險操作
  • 生產環境建議使用白名單（僅允許必要工具）
  • 定期審查 allowed tools 設定

3.3 System Prompt 客製化

追加 System Prompt（保留預設行為）：


gh pr diff "$1" | claude -p \
  --append-system-prompt "You are a security engineer. Review for vulnerabilities." \
  --output-format json

完全替換 System Prompt：


claude -p "Analyze code" \
  --system-prompt "You are a code quality expert. Focus on maintainability." \
  --output-format json

System Prompt 檔案（僅 -p 模式支援）：


claude -p "Review this PR" \
  --system-prompt-file .claude/security-review.md \
  --output-format json

3.4 Session 延續（Multi-Turn Conversations）

繼續最近對話：


claude -p "Review this codebase for performance issues"
claude -p "Now focus on the database queries" --continue
claude -p "Generate a summary of all issues found" --continue

Resume 特定 Session：


session_id=$(claude -p "Start a review" --output-format json | jq -r '.session_id')
claude -p "Continue that review" --resume "$session_id"

Session 管理優勢：

  • Token 節省：相較於獨立請求，可減少約 40% token 消耗（無需重複傳遞 context）
  • Context 保留：跨次呼叫時保持專案理解、變數狀態
  • Cost 優化：減少重複的 context window 佔用

四、CI/CD 整合策略

4.1 GitHub Actions 整合

#### 快速設定（官方 GitHub App）

最快方式是透過 Claude Code 終端執行：


claude
/install-github-app

此指令會引導完成：

  1. GitHub App 安裝（需 repo admin 權限）
  2. 設定 ANTHROPIC_API_KEY secret
  3. 自動產生 .github/workflows/claude.yml

#### 手動設定

步驟 1：安裝 Claude GitHub App

  • 訪問：https://github.com/apps/claude
  • 授予權限：Contents、Issues、Pull requests（Read & Write）

步驟 2：設定 Repository Secrets

  • ANTHROPIC_API_KEY — Claude API 金鑰（從 console.anthropic.com 取得）

步驟 3：建立 Workflow 檔案

基礎 Workflow（.github/workflows/claude.yml）：


name: Claude Code
on:
  issue_comment:
    types: [created]
  pull_request_review_comment:
    types: [created]
jobs:
  claude:
    runs-on: ubuntu-latest
    steps:
      - uses: anthropics/claude-code-action@v1
        with:
          anthropic_api_key: ${{ secrets.ANTHROPIC_API_KEY }}
          # 自動回應 @claude 提及

Code Review Workflow（自動觸發）：


name: Code Review
on:
  pull_request:
    types: [opened, synchronize]
jobs:
  review:
    runs-on: ubuntu-latest
    steps:
      - uses: anthropics/claude-code-action@v1
        with:
          anthropic_api_key: ${{ secrets.ANTHROPIC_API_KEY }}
          prompt: "Review this pull request for code quality, correctness, and security."
          claude_args: "--max-turns 5"

排程任務（Daily Report）：


name: Daily Report
on:
  schedule:
    - cron: "0 9 * * *"
jobs:
  report:
    runs-on: ubuntu-latest
    steps:
      - uses: anthropics/claude-code-action@v1
        with:
          anthropic_api_key: ${{ secrets.ANTHROPIC_API_KEY }}
          prompt: "Generate a summary of yesterday's commits and open issues"
          claude_args: "--model claude-opus-4-6"

4.2 安全最佳實踐

API Key 管理：

  • ✅ 使用 GitHub Secrets：${{ secrets.ANTHROPIC_API_KEY }}
  • ❌ 禁止硬編碼 API Key 至 workflow 檔案
  • ✅ Organization-level secrets 優於 repository-level（集中管理）
  • ❌ 禁止將 API Key 寫入日誌或作為 workflow input

權限最小化：


permissions:
  contents: write    # 僅在需要修改程式碼時啟用
  issues: write      # 僅在需要回應 issue 時啟用
  pull-requests: write
  id-token: write    # 僅 OIDC 驗證時需要

Fork PR 安全：

  • GitHub 預設不將 secrets 傳遞給 fork PR
  • 防止未經授權的貢獻者消耗 API credits
  • 若需支援 fork PR，使用 pull_request_target + 手動審查機制

4.3 CLAUDE.md 專案配置

在專案根目錄建立 CLAUDE.md，Claude 會自動載入此檔案至 context：


# Claude 配置指引

## 程式碼風格
- 使用 TypeScript strict mode
- 函式命名採用 camelCase
- 檔案命名採用 kebab-case

## 測試要求
- 所有新功能必須包含單元測試
- 測試覆蓋率需達 80% 以上

## Code Review 重點
1. 安全性：檢查 SQL injection、XSS 漏洞
2. 效能：避免 N+1 查詢
3. 可讀性：複雜邏輯需加註解

CLAUDE.md 優勢：

  • 無需在每次 prompt 重複說明規則
  • 節省 token 成本（context 自動載入）
  • 整個團隊共用一致的 AI 互動標準

4.4 Cost 與效能監控

成本指標收集：


claude -p "Review PR" --output-format json | jq '{
  cost: .cost_usd,
  duration: .duration_ms,
  turns: .num_turns
}' >> metrics.jsonl

典型成本範圍（2026 年數據）：

  • 一般 Code Review：$0.02-0.05
  • 複雜重構建議：$0.10-0.15
  • 完整 codebase 分析：$0.20-0.50

效能優化技巧：

  • 使用 --max-turns 限制迭代次數（推薦 10 次以內）
  • 啟用 Session 延續減少 context 重複傳遞
  • 針對簡單任務使用 Haiku 模型（成本降低 70%）
  • 設定 workflow timeout（避免 runaway jobs）

告警閾值建議：

  • 單次執行 >120 秒 → 觸發告警
  • 單次成本 >$0.20 → 觸發告警
  • 每日總成本 >$5 → 觸發告警

五、進階功能與配置

5.1 CLI 參數完整參考

| 參數 | 用途 | 範例 |

|------|------|------|

| --model | 指定模型 | --model claude-opus-4-6 |

| --max-turns | 限制對話輪數 | --max-turns 5 |

| --allowedTools | 批准工具清單 | --allowedTools "Read,Write" |

| --disallowedTools | 禁用工具清單 | --disallowedTools "Bash(rm *)" |

| --add-dir | 新增工作目錄 | --add-dir ../apps ../lib |

| --verbose | 啟用詳細日誌 | --verbose |

| --output-format | 輸出格式 | --output-format json |

| --json-schema | 結構化輸出 schema | --json-schema '{...}' |

| --continue | 繼續最近對話 | --continue |

| --resume | Resume 特定 session | --resume "session_id" |

5.2 設定檔階層

Claude Code 支援多層級設定檔，優先順序如下（高 → 低）：

  1. CLI 參數 — 直接傳入的 --model、--max-turns 等
  2. .claude/settings.local.json — 個人覆寫（不納入 git）
  3. .claude/settings.json — 專案共用（版本控制）
  4. ~/.claude/settings.json — 全域預設

專案設定範例（.claude/settings.json）：


{
  "model": "claude-sonnet-4-6",
  "maxTokens": 4096,
  "permissions": {
    "allowedTools": ["Read", "Write", "Bash(git *)"],
    "deny": ["Read(./.env)", "Bash(rm *)"]
  }
}

5.3 Hooks 自動化

透過 hooks 實現工具執行後的自動化流程：


{
  "hooks": {
    "PostToolUse": [{
      "matcher": "Write(*.py)",
      "hooks": [{
        "type": "command",
        "command": "python -m black \"$file\""
      }]
    }]
  }
}

常見 Hook 應用場景：

  • Python 檔案寫入後自動執行 black 格式化
  • Markdown 檔案變更後自動執行 markdownlint
  • 敏感檔案寫入前觸發告警

5.4 子 Agent（Subagents）

透過 /agents 指令建立專責 sub-agent：


# Interactive mode 範例
claude
/agents create security-reviewer
/agents assign security-reviewer "Review auth.py for vulnerabilities"

子 Agent 應用場景：

  • Code Review Agent：專注於程式碼品質檢查
  • Debugging Agent：專注於錯誤排查
  • Architecture Agent：專注於系統設計建議

六、與本專案（Daily-Digest-Prompt）的關聯

6.1 本專案當前使用模式

Daily-Digest-Prompt 專案大量使用 claude -p 模式：


# run-agent-team.ps1 中的典型呼叫
cat task_prompt.md | claude -p

當前實踐：

  • 透過 cat prompt.md | claude -p 傳遞多行指令
  • 使用 PowerShell Start-Job 並行執行多個 Claude CLI 任務
  • 透過 stdout 擷取結果並寫入 JSON 檔案
  • 設定 timeout 機制防止任務 hang

6.2 可優化方向

#### 優化 1：使用 --output-format json 標準化輸出

現況問題：

  • 目前直接擷取 stdout，未使用結構化格式
  • 無法取得 session ID、成本、執行時間等 metadata

建議改進：


$result = cat task_prompt.md | claude -p --output-format json | ConvertFrom-Json
$resultText = $result.result
$sessionId = $result.session_id
$cost = $result.cost_usd

效益：

  • 精確的成本追蹤（寫入 state/token-usage.json）
  • Session ID 記錄（可用於多輪對話）
  • 執行時間監控（用於效能優化）

#### 優化 2：啟用 Session 延續減少 token 消耗

應用場景：

  • Phase 1 查詢 Todoist → Phase 2 執行任務 → Phase 3 組裝結果
  • 三個 Phase 可共用同一 session，避免重複傳遞 context

實作範例：


# Phase 1: 查詢
$p1Result = cat todoist-query.md | claude -p --output-format json | ConvertFrom-Json
$sessionId = $p1Result.session_id

# Phase 2: 執行（繼續 Phase 1 的 session）
$p2Result = cat todoist-execute.md | claude -p --resume $sessionId --output-format json | ConvertFrom-Json

# Phase 3: 組裝（繼續同一 session）
$p3Result = cat todoist-assemble.md | claude -p --resume $sessionId --output-format json | ConvertFrom-Json

預期效益：

  • Token 消耗減少約 30-40%
  • 成本降低相應比例
  • 執行速度提升（減少 context 傳遞時間）

#### 優化 3：設定檔取代 Prompt 內重複規則

現況問題：

  • 所有 prompt 都需引用 templates/shared/preamble.md
  • Skill-First 規則、nul 禁令等內容重複出現

建議改進：

在專案根目錄建立 CLAUDE.md：


# Daily-Digest-Prompt 專案指引

## Skill-First 規則
1. 執行前必須先讀取 skills/SKILL_INDEX.md
2. 先讀 SKILL.md 再動手
3. 能用 Skill 就用 Skill
4. Skill 鏈式組合
5. 所有外部 API 必經 api-cache

## 禁止行為
- 禁止產生 nul 檔案（Windows 環境）
- 禁止使用 > nul 重導向
- 使用 > /dev/null 2>&1 替代

## Shell 執行規則
- 任何 Shell 命令都必須用 Bash tool 實際執行
- 不得只輸出命令文字

效益：

  • 所有 prompt 無需重複引用 preamble.md
  • 節省 token（CLAUDE.md 自動載入，不佔用 prompt token）
  • 更易維護（集中管理專案規則）

#### 優化 4：使用 --allowedTools 自動批准工具

現況問題：

  • 目前每次 Bash、Write 工具執行都需要人工批准
  • 自動化任務需要無人值守執行

建議改進：


cat task_prompt.md | claude -p \
  --allowedTools "Read,Write,Bash(git *),Bash(curl *),Bash(python *)" \
  --disallowedTools "Bash(rm *),Write(.env)" \
  --output-format json

效益：

  • 完全無人值守自動化
  • 安全性不降低（精確控制允許的工具與命令）
  • 執行速度提升（無需等待人工批准）

七、實戰最佳實踐總結

7.1 開發階段最佳實踐

  1. 本地測試先用 interactive mode

- 快速迭代、即時反饋

- 理解 Claude 的推理過程

- 確認 prompt 有效性

  1. 確認可行後轉為 print mode

- 加上 --output-format json 取得 metadata

- 使用 --max-turns 限制迭代

- 設定 --allowedTools 白名單

  1. 整合至自動化腳本

- 使用 session 延續減少成本

- 錯誤處理（檢查 exit code）

- 日誌記錄（成本、時間、turns）

7.2 生產環境最佳實踐

  1. 安全性優先

- API Key 使用環境變數或 secrets

- 限制工具權限（Read-only for review）

- 定期審查 allowed tools 清單

  1. 成本控制

- 設定 --max-turns 上限

- 使用適當模型（Haiku for simple tasks）

- 監控每日總成本

  1. 可靠性保證

- 設定 timeout 機制

- 重試邏輯（exponential backoff）

- 降級策略（API 失敗時的 fallback）

  1. 可觀測性

- 收集 JSON 輸出至 metrics.jsonl

- 定期分析成本趨勢

- 告警閾值設定
