DBSCAN/HDBSCAN 演算法內部實作細節與時間複雜度分析（mechanism 階段）

本文為 python-clustering-anomaly-detection 系列第 mechanism 階段，承接 foundation 階段的基本原理知識，深入探討演算法內部實作細節、時間複雜度分析與參數自動化選擇方法。

執行摘要

DBSCAN 與 HDBSCAN 是兩種重要的密度聚類演算法。DBSCAN 的核心在於區域查詢（region query）與鄰域搜尋機制，時間複雜度在最壞情況下為 O(n²)，但透過空間索引（KD-tree/Ball tree）可優化至平均 O(n log n) [Wikipedia, 2026; scikit-learn, 2026]。HDBSCAN 改進了 DBSCAN 的固定密度限制，使用最小生成樹（MST）建構層次聚類結構，時間複雜度為 O(n log n) [HDBSCAN docs, 2026]。參數選擇方面，k-distance plot 的 knee 方法可半自動化選擇 epsilon，而 MinPts 通常設為 2*dim（維度）[Sefidian, 2022; DataCamp, 2026]。

一、DBSCAN 內部實作機制

1.1 區域查詢（Region Query）與鄰域搜尋

DBSCAN 的核心運算是區域查詢（region query）：對資料集中每個點，計算其 ε-鄰域內的所有鄰近點 [ScienceDirect, 2019]。

實作細節：

  • 對包含 n 個點的資料集，需執行 n 次區域查詢
  • 每次區域查詢計算一個點的局部鄰居
  • 演算法效能主要由區域查詢的總執行次數決定 [ScienceDirect, 2019]

點分類機制（承接 foundation 階段知識）：

  • 核心點（Core Point）：ε-鄰域內至少有 MinPts 個點
  • 邊界點（Border Point）：在核心點鄰域內，但自身鄰域點數不足
  • 噪音點（Noise Point）：既非核心點也非邊界點

1.2 空間索引優化（KD-tree vs Ball tree）

無索引情況（brute force）：

  • 時間複雜度：O(n²)
  • 每次區域查詢需比對所有點
  • 記憶體複雜度：O(n) [Wikipedia, 2026]

KD-tree 索引：

  • 適用場景：低維資料（< 20 維） [GeeksforGeeks, 2026]
  • 單次查詢複雜度：O(log n)
  • 總時間複雜度：O(n log n)（平均情況）
  • 機制：遞迴劃分空間，沿交替維度切割
  • 限制：高維資料時退化至 O(n²) [GeeksforGeeks, 2026]

Ball tree 索引：

  • 適用場景：高維資料 [GeeksforGeeks, 2026]
  • 機制：使用超球體（hyper-spheres）劃分空間
  • 優勢：更好處理高維空間的密集區域
  • 支援非歐氏距離度量 [GeeksforGeeks, 2026]

scikit-learn 實作選項：


from sklearn.cluster import DBSCAN

# algorithm 參數選項
dbscan = DBSCAN(
    algorithm='ball_tree',  # 或 'kd_tree', 'brute'
    leaf_size=30,           # 影響建樹與查詢速度
    metric='euclidean'
)

[scikit-learn, 2026]

1.3 記憶體複雜度權衡

現代實作策略：

  • 批量計算所有鄰域查詢（bulk-compute）
  • 記憶體複雜度：O(n·d)，其中 d 為平均鄰居數
  • 原始 DBSCAN：O(n)
  • 完整鄰域圖最壞情況：O(n²) [scikit-learn, 2026]

二、HDBSCAN 最小生成樹（MST）構建機制

2.1 演算法概述

HDBSCAN 透過建構最小生成樹（Minimum Spanning Tree, MST）來識別多密度層級的穩定聚類 [HDBSCAN docs, 2026]。

2.2 Prim's Algorithm 實作

核心步驟：

  1. 從任意頂點開始
  2. 每次加入權重最低的邊，連接當前樹與未加入的頂點
  3. 逐邊建樹，直到所有頂點加入 [HDBSCAN docs, 2026]

優化方法：

  • 對於度量空間（metric space）資料，可使用 Dual Tree Boruvka 演算法加速
  • 比單純 Prim's algorithm 更快 [HDBSCAN docs, 2026]

2.3 Mutual Reachability Distance

關鍵差異：HDBSCAN 建構的是基於 mutual reachability distance 的 MST，而非純粹距離圖 [HDBSCAN docs, 2026]。

定義：


mutual_reachability(a, b) = max(
    core_distance(a, k),
    core_distance(b, k),
    distance(a, b)
)

目的：

  • 平滑化不同密度區域
  • 讓低密度點也能參與聚類層次結構
  • 允許識別變密度聚類 [Arize AI, 2026]

2.4 層次聚類建構

邊排序與合併：

  1. 將 MST 的邊按距離遞增排序
  2. 依序迭代每條邊，建立新合併的聚類
  3. 形成層次聚類結構基礎 [HDBSCAN docs, 2026]

三、時間複雜度詳細分析

3.1 DBSCAN 時間複雜度

| 情境 | 時間複雜度 | 條件 |

|------|-----------|------|

| 最壞情況 | O(n²) | 無加速索引，或資料退化（所有點距離 < ε） [Wikipedia, 2026] |

| 平均情況 | O(n log n) | 使用空間索引（KD-tree/R-tree），且 ε 選擇合理（平均返回 O(log n) 個鄰居）[Wikipedia, 2026] |

| 形式正確性 | 無理論保證 | 平均情況無嚴格理論證明，取決於資料分布與索引結構 [DBSCAN Revisited, 2026] |

實務考量：

  • 效能主要由 regionQuery 調用次數決定
  • DBSCAN 對每個點恰好執行一次區域查詢 [DBSCAN Revisited, 2026]

3.2 HDBSCAN 時間複雜度

整體複雜度：O(n log n) [Arize AI, 2026]

適用場景：

  • 小到中型資料集：高效
  • 大型資料集：可能成為瓶頸 [Arize AI, 2026]

比較：

  • HDBSCAN 比 DBSCAN 多了 MST 建構步驟，但整體仍保持 O(n log n)
  • 代價是更高的常數因子

3.3 空間複雜度

| 演算法 | 空間複雜度 | 說明 |

|--------|-----------|------|

| DBSCAN（原始） | O(n) | 僅儲存標籤與核心點資訊 |

| DBSCAN（現代實作） | O(n·d) | 批量計算鄰域圖，d 為平均鄰居數 |

| HDBSCAN | O(n²) 最壞 | MST 構建需儲存邊資訊 |

四、k-distance 圖與參數自動化選擇

4.1 k-distance Plot 方法

原理：

  1. 對每個點計算其第 k 個最近鄰的距離（k-distance）
  2. 將所有 k-distance 值遞增排序
  3. 繪製 k-distance 曲線 [Sefidian, 2022]

epsilon 選擇：

  • 找曲線的「膝點」（knee）或「肘點」（elbow）
  • 對應曲線斜率最大變化處（最大曲率點）[DataCamp, 2026; Sefidian, 2022]

實作注意事項：

  • 使用 k = MinPts - 1 計算 k-NN 距離
  • 原因：MinPts 包含點本身，而 k-NN 距離排除查詢點 [Sefidian, 2022]

4.2 MinPts 參數選擇

經驗法則：

  • 2 維資料：MinPts = 4（DBSCAN 預設值）[Sefidian, 2022; DataCamp, 2026]
  • d > 2 維資料：MinPts = 2 × dim [Sefidian, 2022; DataCamp, 2026; MATLAB, 2026]

限制：

  • 無完全自動化方法
  • 最終需依賴領域知識與資料特性判斷 [Sefidian, 2022]

4.3 多密度情境的處理

問題：

  • 當 k-distance plot 顯示多個膝點時，難以選擇單一 epsilon
  • 表示資料存在多個密度尺度 [ResearchGate, 2026]

解決方案：

  • 使用層次化方法（如 HDBSCAN）
  • 消除固定 epsilon 的需求
  • 自動處理變密度聚類 [ResearchGate, 2026]

五、實務應用建議

5.1 何時使用 DBSCAN

適用情境：

  • 資料維度 < 20
  • 資料集中等大小（< 10萬點）
  • 需要明確的噪音點標記
  • 可接受固定密度假設

最佳實踐：

  • 使用 algorithm='kd_tree'（低維）或 'ball_tree'（高維）
  • 調整 leaf_size 參數優化效能（預設 30）
  • 用 k-distance plot 輔助選擇 epsilon

5.2 何時使用 HDBSCAN

適用情境：

  • 資料含多密度層級聚類
  • 難以選擇單一 epsilon 值
  • 需要層次聚類結構
  • 可接受較高計算成本

優勢：

  • 自動處理變密度
  • 僅需調整一個參數（min_cluster_size）
  • 提供聚類穩定性評分
