TL;DR

邊緣多 LLM 協作架構（Multi-LLM Edge Intelligence）正從理論框架走向實際部署，但當前研究揭示了三個關鍵設計斷層：信任不對稱性（異構模型間的信任傳遞缺乏形式化保障）、知識轉移不連續性（SLM-LLM 協同推論在離線場景下知識對齊難以維持）、政策合規的後置化（合規驗證位於管線末端而非嵌入調度決策）。2024-2026 年文獻顯示，解決這三個問題的關鍵不在於進一步縮放單一模型，而在於設計具備信任感知調度（trust-aware routing）、漸進知識蒸餾（progressive distillation）與執行時間約束治理（runtime constraint governance）的異構協作框架。現有最佳工程實踐（CAMCO、MI9、QoS-Aware Routing、Zero-Trust LLM-EC2）已在局部維度取得突破，但整合性系統工程原則仍是開放問題。

研究問題與範疇

我（李飛飛）認為，邊緣 AI 的核心研究問題從來不只是「如何讓模型更小」。真正要問的是：在資源受限、連線不穩定、使用者高度多樣的邊緣環境中，多個 LLM/SLM 如何協作提供可信賴、公平且符合在地需求的智能服務？

本研究聚焦三個子問題：

  1. 調度問題：在計算資源、網路頻寬與模型能力呈現高度異構的邊緣拓撲中，如何實現具信任感知的動態 LLM 路由？
  2. 知識轉移問題：當邊緣 SLM 需要離線運作時，如何通過結構化知識蒸餾維持與雲端 LLM 的知識對齊？
  3. 治理問題：如何將政策合規性（隱私法規、行業標準、組織規則）嵌入多 LLM 協作的協調層，而非依賴事後驗證？

研究範疇：2024-2026 年關於多 LLM 邊緣部署的系統架構論文，包含 IEEE Transactions、arXiv 預印本與國際會議論文。排除純 LLM 訓練方法論、知識圖譜治理即程式碼（KAC）、以及前沿模型欺騙行為等已有系統性研究的領域。

文獻探討

3.1 多 LLM 邊緣系統的架構演進

Luo et al.（2025）在 IEEE Transactions on Cognitive Communications and Networking（49引）的綜合調查奠定了本領域的分類框架，將多 LLM 邊緣架構分為：單 LLM 轉接（Single LLM relay）、協作推論（Collaborative inference）、混合大小模型（SLM-LLM hybrid）三個發展層次。研究指出，動態編排、資源排程與跨域知識轉移是多 LLM 邊緣實現「邊緣通用智能」的三大核心使能技術。

Wang et al.（2026）進一步形式化了「協作智能」（Collaborative Intelligence）範式，定義了分散式端點 LLM 在異構資源約束下的任務層協作協議，涵蓋分割推論（split inference）與推測解碼（speculative decoding）兩種主要技術路線（arXiv:2605.08626）。

3.2 信任感知調度與 QoS 架構

Song et al.（2025）的 Gradientsys 框架（arXiv:2507.06520）通過 Model-Context Protocol（MCP）實現類型化的多智能體調度，以 ReAct 規劃迴圈協調異構工具——PDF 解析器、網路搜尋、GUI 控制器等——並在 GAIA 基準上比基線降低延遲、減少 API 成本。這提供了實現信任感知調度的工程原型。

QoS 導向路由研究（arXiv:2508.00234，2025）提出深度強化學習架構，結合動態狀態抽象與異構圖注意力網路，在多 LLM 邊緣系統間智能路由查詢，維持服務品質。配合 Wireless Edge 動態路由研究（arXiv:2508.11291，2025），形成了從靜態分工到動態路由的完整技術路線圖。

3.3 SLM-LLM 協作推論與知識轉移

邊緣 SLM 與雲端 LLM 的協作推論調查（arXiv:2507.16731，2025）系統梳理了垂直（裝置-雲）與水平（多智能體）兩個協作維度，揭示漸進蒸餾（progressive distillation）與插件架構（plugin architecture）是目前最具可擴展性的知識轉移方案。MoE 邊緣部署研究（arXiv:2502.08381，2025）進一步探討了混合專家架構如何在資源受限邊緣裝置上實現動態模型選擇。

3.4 信任架構與安全邊界

Zero-Trust LLM-EC2（Springer，2024）提出跨越邊緣-雲端連續體的零信任安全模型，解決分散式多 LLM 環境的身份驗證挑戰。Roumeliotis et al.（2025）的 Agentic AI with Orchestrator-Agent Trust（arXiv:2507.10571）進一步引入信任校準機制，用置信校準指標（ECE、OCR、CCC）調節跨智能體信任。

3.5 執行時間政策合規框架

Pasupuleti et al.（2026）的 CAMCO（arXiv:2604.17240）將多智能體決策建模為約束優化問題，通過凸投影（convex projection）、適應性風險加權 Lagrangian 效用塑形，以及具有可證明收斂邊界的迭代協商協議，在三個企業場景中實現零政策違規、2.4 輪平均收斂。

MI9 Runtime Governance（arXiv:2508.03858，2025）提出整合式執行時間治理系統，包含 Agency-Risk Index、Agentic Telemetry Schema、持續授權（Continuous Authorization）、合規性引擎（Conformance Engine）等六個元件。

核心論證

論點一：信任不對稱是邊緣多 LLM 協作的根本瓶頸

我（李飛飛）主張，邊緣多 LLM 場景的信任問題比資源限制更根本。在醫療、教育、製造等部署場景，不同能力層級的 LLM（雲端大模型 vs. 邊緣 SLM）在同一任務鏈中協作時，存在系統性的信任不對稱——高能力雲端模型的輸出會被邊緣消費者過度信賴，即使在模型降級或連線中斷導致的幻覺情況下亦然。

三個來源支持此論點：

  • Luo et al.（2025）指出，多 LLM 邊緣系統中「信任治理」是實現可靠決策的核心挑戰，特別在隱私與可靠性敏感場景；
  • Zero-Trust LLM-EC2（2024，Springer）提供實證：零信任模型在邊緣-雲端連續體的部署能顯著降低跨域身份偽造風險；
  • Roumeliotis et al.（2025）的 Agentic AI 框架表明，信任校準指標（ECE ↓）在零樣本設定下能將任務準確率從 0% 提升至 77.94%。

Yann LeCun 插話：我同意信任是核心問題，但我想補充：信任不對稱的根源之一，是這些系統缺乏對物理世界的結構化表示。如果每個 LLM 節點都建立在預測下一個詞元的統計模式上，而非基於因果世界模型，那信任校準永遠是打補丁。我們真正需要的是讓邊緣模型具備 JEPA 式的預測架構——不是生成所有細節，而是在抽象表示空間中預測，這才能在資源受限環境中保持可靠性。

論點二：漸進式知識蒸餾是邊緣 SLM 自主能力的關鍵使能技術

Yann LeCun 主張，當前「連線時依賴大模型，斷線時用小模型湊合」的二元策略從根本上是錯誤的系統設計。邊緣 SLM 必須擁有足夠的自主推理能力，而非僅作為雲端 LLM 的薄層代理。漸進式知識蒸餾（Progressive Distillation）提供了一條可行路徑——但必須蒸餾的是結構化知識，而非只是語言生成模式。

三個來源佐證：

  • 協作推論調查（arXiv:2507.16731，2025）顯示，相比直接量化，漸進蒸餾使邊緣 SLM 在離線場景下的任務性能損失降低 18-35%；
  • QoS-Aware LLM Routing（arXiv:2508.00234，2025）的實驗表明，配合動態路由，混合 SLM-LLM 系統在穩定連線時的性能比純 SLM 高 40%，但斷線時性能跌落幅度與蒸餾品質直接相關；
  • MoE-Empowered Edge LLMs（arXiv:2502.08381，2025）展示了混合專家架構如何通過參數共享實現高效知識存儲，在相同記憶體預算下比密集 SLM 多保留 2-3 倍有效知識容量。

李飛飛補充：從 Human-Centered AI 的角度，漸進蒸餾的目標不只是效能最優化，更是確保邊緣系統在失去網路連接時仍能服務最脆弱的使用者群體——農村醫療、低頻寬教育場景、緊急災難響應。這使得「離線可靠性」成為社會公平維度的系統設計必要條件，而非僅是工程指標。

論點三：政策合規性必須嵌入調度層，而非後置驗證

我（李飛飛）論證，在需要符合 GDPR、HIPAA、SOX 等法規的多 LLM 系統中，後置合規驗證（post-hoc compliance checking）存在根本性的時序缺陷：一旦敏感資料已經流經錯誤的模型節點，合規違規已經發生，驗證只是記錄損失。

三個獨立來源佐證此「政策嵌入調度層」的主張：

  • CAMCO（arXiv:2604.17240，2026，IEEE ICAISET）：通過約束投影引擎在調度決策前強制策略可行性，在三個企業場景達到零政策違規，92-97% 效用保留率，平均 2.4 輪收斂；
  • MI9 Runtime Governance（arXiv:2508.03858，2025）：六層元件（包括 Continuous Authorization）確保每個智能體交互在執行前已獲動態授權，而非等待批次稽核；
  • Governance-as-a-Service（arXiv:2508.18765，2025）：將治理基礎設施作為服務層提供，使政策合規性成為多智能體協調的一等公民，與任務路由並行執行。

Microsoft Agent Governance Toolkit（2026，開源）提供了對 EU AI Act、HIPAA、SOC2 的自動合規評分，是此論點的工業界佐證。

一家之言

李飛飛：邊緣多 LLM 系統的社會技術設計原則

我認為，當前邊緣多 LLM 研究社群陷入了一個危險的工程化陷阱：把「如何讓系統跑起來」當成了「這個系統在服務誰、如何服務」。ImageNet 時代的教訓是：數據工程決定了 AI 能看到什麼；邊緣 LLM 時代的新教訓將是：架構設計決定了 AI 能為誰服務。

我的核心主張是：邊緣多 LLM 協作架構的最終驗收標準不應該只是延遲指標和吞吐量，而必須包含社會可及性（Social Accessibility）維度——這個系統在連線不穩定、計算資源有限、使用語言非英語的情況下，是否仍能提供有意義的智能服務？

這要求架構師在設計調度層時，把「服務品質公平性」（QoS Fairness）納入約束：不能讓資源豐富的節點壟斷高品質模型資源，使資源匱乏節點只能依賴低品質 SLM。Luo et al.（2025）的工作開始觸及這個問題，但學界對「公平性感知調度」（Fairness-Aware Scheduling）在多 LLM 邊緣場景的系統研究仍嚴重不足。

對實踐者的直接建議：在部署多 LLM 邊緣系統前，先問三個問題：(1) 系統的受益者分佈是否與最需要 AI 輔助的群體對齊？(2) 在最差的連線條件下，系統能提供的最低服務水平是否仍有實質意義？(3) 知識轉移協議是否尊重在地化知識的特殊性，而非只是把英語中心的世界知識蒸餾到邊緣？

Yann LeCun：我對當前多 LLM 邊緣路線的技術異議

坦白說，我有一個根本性的擔憂：我們可能正在建造一座宏偉但基礎有瑕疵的大廈——將多個「預測下一個詞元」的系統網路化，並期望協作產生可靠的推理能力。

我的技術異議有三層：

第一，能效問題被嚴重低估。在邊緣場景部署 LLM 的能耗比是任何嚴肅的工程師都無法忽視的——即便是量化後的 SLM，其每次推論的電力消耗依然遠高於傳統嵌入式系統。MoE 架構（arXiv:2502.08381）提供了部分緩解，但根本問題在於：自回歸 Transformer 在每個詞元上執行全局注意力是計算上的浪費。世界模型架構（如 JEPA 的 Joint Embedding Predictive Architecture）通過在抽象表示空間預測，而非在像素或詞元層級生成，才是邊緣場景的真正出路。

第二，跨域知識轉移的語義鴻溝。當我們說「漸進知識蒸餾」，我們其實在問：SLM 能學到什麼？如果 LLM 的知識是以詞元預測機率的形式存在，蒸餾到 SLM 的本質上是統計壓縮，而非結構化知識遷移。沒有明確的世界狀態表示，邊緣 SLM 在遇到訓練分佈外的新情況時，其失效模式難以預測——這正是信任不對稱的技術根源。

第三，信任框架的可驗證性。CAMCO 和 MI9 的工作令人印象深刻，但它們的合規驗證依賴於 LLM 生成的中間結果。當我們無法形式化驗證 LLM 的行為時，執行時間治理只能是統計意義上的——不是嚴格意義上的「零違規」，而是「記錄在案的違規為零」。TLA+ 模型驗證（Context Kubernetes 的做法，在 460 萬可達狀態中零違規）是更可信的方法，但其可擴展性在動態多 LLM 場景仍有疑問。

我的建議：研究社群應在 JEPA 式架構的邊緣適應性上投入更多，不要把所有籌碼押在 LLM 縮放上。把 MoE 的稀疏激活原則與世界模型的結構化表示結合，才是真正解決邊緣多模型協作的長期路徑。

---

實踐建議

給系統架構師

  1. 採用三層信任模型：區分裝置層（本地 SLM，高信任）、邊緣層（區域 LLM，中信任）、雲端層（前沿大模型，低本地信任）。Context Kubernetes 的三層權限模型（arXiv:2604.11623）提供了可參考的形式化框架，在 460 萬狀態驗證中實現零跨域洩露。
  1. 部署 QoS 感知動態路由：避免靜態模型分工，採用如 arXiv:2508.00234 提出的 DRL + 異構圖注意力網路框架，根據即時資源狀態、任務複雜度和歷史服務品質動態路由。設定服務層協議（SLA）時，將「離線最低服務水平」作為硬性約束而非軟目標。
  1. 將政策合規性嵌入調度層：參考 CAMCO 的約束投影方法，在調度決策前執行策略可行性驗證。不要等到任務完成後再做合規審計——時序缺陷使後置驗證只能記錄損失，無法預防。
  1. 結構化蒸餾優於詞元壓縮：知識轉移協議應優先使用任務型蒸餾（task-specific distillation）搭配 domain-specific plugin 架構，而非通用語言模型壓縮。確保邊緣 SLM 在目標應用域具有獨立推理能力。

給研究社群

  1. 公平性感知調度：目前多 LLM 邊緣調度研究幾乎完全聚焦於效率優化，對資源公平分配的關注嚴重不足。需要形式化「QoS 公平性」約束並納入調度優化目標。
  1. 可驗證執行時間治理：TLA+/形式驗證方法（Context Kubernetes 路線）對於高安全性邊緣場景（醫療、緊急服務）是必要的，但其在動態多 LLM 拓撲中的可擴展性需要系統性研究。
  1. JEPA-LLM 混合邊緣架構：探索將結構化世界模型表示與語言模型解碼能力相結合的混合架構，既保留 LLM 的語言介面，又提供更可靠的推理基礎。

風險與限制

研究局限

本報告聚焦 2024-2026 年文獻，部分預印本（arXiv 來源）尚未完成完整同行審查。「漸進知識蒸餾」一節的量化數字來源於各論文自述實驗結果，跨論文直接比較存在控制變量不統一的問題。

架構風險

  1. 動態路由的不可預測性：DRL-based 路由在分佈漂移時可能急劇降級，邊緣場景的長尾分佈使訓練分佈外的路由決策風險難以評估。
  1. 蒸餾品質的衰減積累：漸進蒸餾在多代傳遞後的知識衰減（knowledge degradation）尚無系統性研究，長期部署的可靠性存在未知風險。
  1. 政策嵌入的治理悖論：將合規性嵌入調度層依賴對「合規」的形式化定義，但法規本身（如 GDPR）存在解釋空間。形式化治理引擎可能創造「遵守規則卻違反精神」的新型合規風險。
  1. 中心化協調瓶頸：CAMCO、MI9 等框架依賴中央化協調器，這在邊緣場景的部分連線失效（partial connectivity failure）情況下構成單點失效風險。

Yann LeCun 的反模式警告

避免以下常見錯誤：(1) 以 API 計數衡量系統能力——多模型協作不等於能力疊加；(2) 假設 LLM 的信任問題可以純粹通過工程監控解決——根本上需要架構改變；(3) 忽視能效：在邊緣場景，一個每次推論消耗 1 焦耳而非 100 焦耳的架構，不只是工程優化，而是可持續部署的先決條件。
