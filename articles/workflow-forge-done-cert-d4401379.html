DONE_CERT 品質認證驗證工作流

產出時間：2026-04-11

類型：workflow_yaml

優先級：P0

產物路徑：workflows/done-cert-validation-workflow.yaml

問題識別

當前狀態：

  • 39 個自動任務結果檔中，僅 20 個（51.3%）包含 done_cert 欄位
  • 19 個任務未執行 Draft-Verify 品質閘門
  • 違反 ADR-380 與 templates/shared/preamble.md（第 195-206 行）強制要求

影響：

  • 品質門檻不統一 → 輸出不穩定
  • 系統成功率僅 53.5%（遠低於 90% 門檻，critical alert）
  • state/result-file-diagnostics.json 顯示多個任務 diagnostics_missing

總分：91/100

  • 影響範圍（38/40）：影響 49% 自動任務
  • 可執行性（33/35）：僅需新增 YAML workflow
  • 信號強度（20/25）：ADR-380 強制要求 + 實際數據

工作流設計

驗證規則（3 個步驟）

步驟 1：必填欄位檢查

  • done_cert 欄位存在且為物件
  • 必須包含：status（DONE|PARTIAL|FAILED）、quality_score（1-5 整數）、draft_verify（物件）、remaining_issues（陣列）

步驟 2：draft_verify 子欄位檢查

  • issues_found（整數 ≥ 0）
  • issues_fixed（整數 ≥ 0，且 ≤ issues_found）
  • skipped_reason（null 或 task_failed / timeout_pressure）

步驟 3：語義一致性檢查

  • status=FAILED → skipped_reason 應為 task_failed
  • issues_found > issues_fixed → remaining_issues 陣列應非空
  • quality_score 應反映 status（FAILED=1-2，PARTIAL=2-3，DONE=3-5）

失敗處理（3 種情境）

  1. missing_done_cert（warning）：

- 記錄 warning 日誌

- 更新 state/result-file-diagnostics.json

- 不阻擋流程

  1. invalid_structure（warning）：

- 記錄具體缺失欄位

- 提供修復範本 JSON 片段

- 更新診斷追蹤

  1. semantic_inconsistency（info）：

- info 級別日誌

- 供後續審查（ntfy_review、system_audit）

整合檢查清單（3 個整合點）

  1. Phase 2 驗證（run-todoist-agent-team.ps1）：

- Start-CursorCliJob 完成後呼叫 tools/validate_done_cert.py（若存在）

- 驗證失敗時記錄 warning，不中斷流程

  1. 健康檢查覆蓋率（check-health.ps1）：

- 新增 [DONE_CERT 覆蓋率] 檢查區塊

- 掃描結果檔，計算包含 done_cert 的比例

- 目標 ≥ 95%

  1. Prompt 模板指引：

- 所有新 prompt 必須包含 Draft-Verify 段落

- 引用 templates/shared/draft-verify.md

- 結果 JSON 範例必須包含完整 done_cert 欄位

度量追蹤

基準線（2026-04-11）：

  • 覆蓋率：20/39 = 51.3%
  • 來源：grep -l '"done_cert"' results/todoist-auto-*.json

目標：

  • 覆蓋率 ≥ 95%（37+/39 任務）
  • 時間表：2 週內達成

度量方式：

  • 頻率：每日（check-health.ps1 執行時）
  • 指標：done_cert_coverage_ratio
  • 告警閾值：< 90%（warning），< 80%（critical）

成功標準：

  1. 覆蓋率達 95%+
  2. 所有 done_cert 欄位符合 3 個驗證步驟規則
  3. 系統成功率提升至 ≥ 75%（中期目標）
