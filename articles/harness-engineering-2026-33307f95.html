Harness Engineering 深度優化方案

基於 2026 年最新理論，針對 Daily Digest Prompt 系統提出 7 項深度優化方案。

核心理論

三階段演化（Epsilla）：

  • Prompt Engineering（2022-24）：單次優化
  • Context Engineering（2025）：單 Window Token 策展
  • Harness Engineering（2026）：跨 Window 持久狀態編排

三層架構（Augment Code）：

  1. Constraint Harnesses（前饋約束）
  2. Feedback Loops（修正迴路）
  3. Quality Gates（執行閘門）

自演化 Harness：Live-SWE-agent 達 77.4% Solve Rate，腳手架從失敗信號自適應，無需人工調參。

現有實施（B0-B3）

✅ B0 地基：backlog_writer、timing-policy、ooda-workflow、autonomous-harness

✅ B1 約束管線：constraint_evolve、constraint-registry、post_tool_logger

✅ B2 回饋迴路：quality_gate、scoring.yaml

✅ B3 可觀測性：constraint_coverage、entropy_score、check-health

七項優化方案

P0（即刻啟動）

1. 約束精細化管線

  • 目標：proposed→accepted 轉換率 20%→60%
  • 方案：constraint_pipeline.py 改 marker-based、validate_config.py --check-constraint-coverage、tools/constraint_auto_disposition.py
  • 參考：arXiv:2602.10133、arXiv:2509.23994

2. IO 壓縮三步驟

  • 目標：avg_io_per_call 20141→8000（降 50%）
  • 方案：Phase 0 預計算 generate_config_snapshot.py、子 Agent 最小包、preamble.md 配置快照優先
  • 參考：Anthropic Context Engineering、arXiv:2510.00615v1

3. Runaway Session 自動阻斷

  • 目標：cache_read>50M 跳過下次 schedule
  • 方案：state/session-blocklist.json + run-todoist-agent-team.ps1 Phase 0 pre-check + 24h 自動解封

P1（2 週內）

4. ADR Fitness 批量補齊

  • 目標：adr_fitness_coverage 35.4%→70%
  • 方案：tools/adr_fitness_suggest.py（suggest/apply/status）、LLM 推斷（Haiku/Groq）、Pre-commit hook
  • 參考：Dev.to、arXiv:2602.07609

5. 可操作 Lint 改造

  • 目標：錯誤信號從「檢測到違規」→「精確補救措施」
  • 方案：validate_results.py enhancement、constraint-registry.yaml 新增 remediation_template、validate_config.py 精確提示
  • 參考：Augment Code: Actionable Lint

P2（1 個月）

6. 主動 Context 壓縮檢查點

  • 目標：從被動（token limit）改主動（Phase Gate）
  • 方案：Phase Gate 轉換時主動檢查、Agent 自主呼叫 compress_context.py、config/context-policy.yaml 配置化
  • 參考：Awesome Harness Engineering

7. 週期性修復 Agent

  • 目標：自動化文檔園藝和架構漂移清理
  • 方案：todoist-auto-doc_gardening（每週 Saturday 03:00）、檢查 ADR-code 不一致/過期 TODO/冗餘配置、輸出 doc-drift-report.json
  • 參考：Self-Evolving Harness Pattern

結論

Daily Digest Prompt 系統 B0-B3 已具堅實基礎。優化重點：

  1. 自演化能力：失敗→約束閉環提升轉換率（20%→60%）
  2. 可操作性：錯誤信號提供精確補救
  3. Context 經濟：20KB→8KB，改主動壓縮
  4. 自主園藝：週期性修復 Agent 防漂移

核心原則：Harness, not model — 架構優化可讓 Agent 從 rank 30→top 5，無需換模型。
