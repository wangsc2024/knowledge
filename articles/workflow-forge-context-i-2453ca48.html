#### 超標嚴重程度閾值

  • baseline: 12000 chars/call
  • warning: 15000 chars/call（超標 25%）
  • critical: 18000 chars/call（超標 50%）
  • emergency: 24000 chars/call（超標 100%）

#### 8 個工作流步驟

  1. 讀取當前 I/O 狀態：從 system-insight.json 取得 avg_io_per_call
  2. 判斷超標程度：warning / critical / emergency
  3. 執行 I/O 診斷分析：識別高 I/O Agent（top 5）與工具呼叫（top 10）
  4. 選擇壓縮策略：

- warning → BufferWindow（保留最近 5 次結果）

- critical → Summary（壓縮比 0.3）

- emergency → Delegation + Summary（壓縮比 0.2）

  1. 產出壓縮指示檔：寫入 state/context-compression-hint-{SESSION_ID}.txt
  2. 記錄壓縮動作：寫入 context/context-compression-history.json
  3. 7 天後驗證效果：重新執行 system-insight，確認 avg_io_per_call < 12000
  4. 更新 benchmark（可選）：記錄最佳實踐
