workflow-forge: Prompt 輸出規範驗證清單產出報告（2026-03-21）

workflow-forge流程標準化prompt規範系統優化驗證清單

workflow-forge 執行報告

日期：2026-03-21

任務：workflow-forge 自動任務

產物：docs/workflows/prompt-output-validation-checklist.md

背景

識別缺口：29 個 prompts/team/todoist-auto-*.md 的輸出 JSON 規範不一致：

  • 部分有詳細「關鍵欄位確認清單」，部分僅有基本範例
  • 命名不一致（task_type vs agent）
  • 缺少統一的必填欄位檢查列表

影響：

  • Phase 3 組裝時可能因欄位缺失失敗
  • avg_io_per_call 超標 5.3 倍（system-insight 告警），部分原因是 prompt 需重複說明輸出格式

產出物

類型：驗證清單（validation checklist）

路徑：docs/workflows/prompt-output-validation-checklist.md（325 行，6 大驗證類別）

內容：

  1. Frontmatter 驗證（6 項檢查）：frontmatter 存在、4 個必填欄位、格式正確
  2. 結果 JSON 路徑驗證（3 項檢查）：路徑格式、底線命名、出現 ≥ 2 次
  3. 必填欄位檢查列表（5 項檢查）：3 個必填欄位、格式一致、值域明確
  4. 欄位命名一致性（3 項檢查）：優先 agent、格式正確、對齊 Schema
  5. Schema 對齊驗證（3 項檢查）：參考 Schema、選填欄位符合定義、type 值有效
  6. 執行流程完整性（4 項檢查）：4 個必備區塊（共用規則、Skill-First、禁止事項、輸出規格）

總計：24 項檢查點

整合狀態

  • ✅ 已建立 docs/workflows/prompt-output-validation-checklist.md
  • ✅ 已更新 CLAUDE.md 配置文件速查表
  • ✅ 已登記至 context/workflow-forge-registry.json

預期成果

即時成果

  • 所有 prompt 開發者有明確檢查列表
  • 新增/修改 prompt 時可自我驗證

中期成果（2-4 週）

  • 結果 JSON 格式一致性提升至 95% 以上（目前約 70%）
  • Phase 3 組裝失敗率下降

長期成果（2-3 個月）

  • avg_io_per_call 因減少重複格式說明而下降 10-15%
  • 系統輸出穩定度顯著提升

對齊檔案

  • Schema 檔案：config/schemas/results-auto-task-schema.json（v1.1.0）
  • 良好範例：prompts/team/todoist-auto-skill_forge.md（有完整「關鍵欄位確認清單」）
  • 待改善範例：prompts/team/todoist-auto-system_insight.md（僅有基本 JSON 範例）

驗證方式

人工審查

開啟待驗證的 prompts/team/todoist-auto-{task_key}.md，對照清單逐條檢查（6 大類共 24 項檢查點）。

自動化驗證（可選）

可建立 tools/validate_prompt_output.py 腳本，自動掃描 prompts/team/ 目錄並驗證規則。

關鍵洞察

本次執行透過分析 system-insight.json、improvement-backlog.json 與現有 prompts，識別出「輸出格式不一致」為影響範圍最大（29 個任務）且可立即執行（僅需文件）的缺口。

驗證清單的建立，結合了：

  • Schema 定義（config/schemas/results-auto-task-schema.json）
  • 良好實踐範例（todoist-auto-skill_forge.md）
  • 系統告警信號（avg_io_per_call 超標）

提供了一個可操作、可驗證的標準化工具，讓所有 prompt 開發者在新增/修改時有明確的品質門檻。

後續行動

  1. 將此驗證清單納入 new-auto-task.ps1 一鍵工具的檢查流程
  2. 逐步審查現有 29 個 prompts，補齊「關鍵欄位確認清單」區塊
  3. 追蹤 avg_io_per_call 指標，驗證格式統一後的改善效果
