1. 技術概述

DBSCAN 與 HDBSCAN 在前三階段（foundation / mechanism / application）已建立完整的概念與 API 基礎。進入 optimization 階段，核心挑戰從「正確使用」轉為「在生產環境中高效運作」。

標準 DBSCAN 的時間複雜度為 O(n log n)（搭配空間索引）至 O(n²)（暴力法），記憶體複雜度在 scikit-learn 的實作中更高達 O(n·d)，最壞情況達 O(n²)。GitHub issue #17650 的實測記錄顯示：745k 筆 / 411 維特徵的資料集跑了超過 6 天；約 1M 筆資料集在 750GB RAM 機器上直接 OOM。

optimization 階段重點：

  1. 近似最近鄰（ANN）加速：以近似替代精確計算
  2. 記憶體效率：稀疏矩陣、預計算圖
  3. 超參數自動選擇：k-distance elbow method
  4. 分散式 / GPU 加速：Spark、RAPIDS cuML

2. 核心優化技術

2.1 近似最近鄰（ANN）加速

| 方法 | 機制 | 適用場景 |

|------|------|----------|

| KD-Tree | 遞迴二分空間 | 低維（<20D）資料 |

| Ball-Tree | 超球分割 | 任意距離度量 |

| LSH（局部敏感雜湊） | 碰撞機率近似距離 | 高維資料 |

| 隨機投影（sDBSCAN） | NeurIPS 2024 新方法 | 百萬量級 |

scikit-learn DBSCAN 的 algorithm 參數預設為 'auto'，低維時自動選 KD-Tree；高維時退化為暴力法。

NeurIPS 2024 — sDBSCAN [來源：NeurIPS 2024]：基於隨機投影的可擴展 DBSCAN 變體，支援平行化，可處理數百萬筆資料。


from sklearn.cluster import DBSCAN

db = DBSCAN(
    eps=0.5,
    min_samples=5,
    algorithm='kd_tree',   # 明確指定，避免高維退化
    leaf_size=30,
    n_jobs=-1
).fit(X_low_dim)

2.2 記憶體效率優化（稀疏矩陣 + 預計算圖）

策略一：預計算稀疏鄰居圖 [來源：scikit-learn docs]


from sklearn.neighbors import NearestNeighbors
from sklearn.cluster import DBSCAN

nn = NearestNeighbors(radius=eps, algorithm='ball_tree', n_jobs=-1)
nn.fit(X)
graph = nn.radius_neighbors_graph(X, mode='distance')  # 稀疏 CSR 格式

# 傳入預計算圖，記憶體從 O(n²) 降至 O(n·d_avg)
db = DBSCAN(metric='precomputed', min_samples=5).fit(graph)

策略二：子採樣加速 [來源：GitHub issue #17650]：子採樣法可獲得 200x 加速 和 250x 記憶體節省。

leaf_size 建議：大資料集設為 40-50（節省記憶體）。

2.3 超參數自動選擇

#### DBSCAN：k-distance Elbow Method [來源：Sefidian, 2022]


import numpy as np
from sklearn.neighbors import NearestNeighbors
from kneed import KneeLocator

def auto_epsilon(X, min_samples=5):
    nn = NearestNeighbors(n_neighbors=min_samples)
    nn.fit(X)
    distances, _ = nn.kneighbors(X)
    k_distances = np.sort(distances[:, -1])[::-1]
    kneedle = KneeLocator(range(len(k_distances)), k_distances,
                          S=1.0, curve='convex', direction='decreasing')
    return k_distances[kneedle.knee]

minPts 規則：d 維資料 → MinPts = 2 × d（業界經驗法則）。

#### HDBSCAN：EOM 自動選群 [來源：hdbscan docs]

  1. 主要參數只有一個：min_cluster_size
  2. EOM 自動選群：在 epsilon 整個範圍內找最穩定群集
  3. 文件明言：「不應需要調整超過 3 個參數」

import hdbscan

clusterer = hdbscan.HDBSCAN(
    min_cluster_size=50,
    min_samples=10,
    cluster_selection_method='eom',
    approx_min_span_tree=True,    # 近似 MST 加速
    core_dist_n_jobs=-1,
    prediction_data=True
)
clusterer.fit(X)

3. 最佳實踐與常見陷阱

最佳實踐

  1. 先降維再聚類：高維（>50D）務必先 PCA/UMAP
  2. 使用預計算稀疏圖：資料量 >50k 時建立稀疏 CSR 圖
  3. HDBSCAN 優先：記憶體效率更好，自動化程度更高
  4. approx_min_span_tree=True：對準確度影響極小但速度顯著提升
  5. 資料標準化：StandardScaler 是必要前處理
  6. kneed 函式庫自動化 ε 選擇

常見陷阱

  1. 高維直接跑暴力 DBSCAN：OOM 或數天不完成（有真實案例）
  2. 忽略 n_jobs=-1：多核心機器浪費 CPU
  3. 用 DBSCAN 處理串流資料：應改用 DenStream 或 BIRCH
  4. 參數掃描不設預算：2M 資料集每次測試需 45 分鐘

5. 生產級完整管線


import numpy as np
from sklearn.preprocessing import StandardScaler
from sklearn.decomposition import PCA
from sklearn.neighbors import NearestNeighbors
from sklearn.cluster import DBSCAN
from kneed import KneeLocator
import hdbscan

def optimized_dbscan_pipeline(X, min_samples=5):
    X_scaled = StandardScaler().fit_transform(X)
    if X_scaled.shape[1] > 50:
        X_scaled = PCA(n_components=50, random_state=42).fit_transform(X_scaled)
    
    nn = NearestNeighbors(n_neighbors=min_samples, n_jobs=-1)
    nn.fit(X_scaled)
    distances, _ = nn.kneighbors(X_scaled)
    k_dist = np.sort(distances[:, -1])[::-1]
    kneedle = KneeLocator(range(len(k_dist)), k_dist, S=1.0, curve='convex', direction='decreasing')
    eps = k_dist[kneedle.knee]
    
    nn_eps = NearestNeighbors(radius=eps, algorithm='ball_tree', leaf_size=40, n_jobs=-1)
    nn_eps.fit(X_scaled)
    graph = nn_eps.radius_neighbors_graph(X_scaled, mode='distance')
    return DBSCAN(metric='precomputed', min_samples=min_samples, n_jobs=-1).fit_predict(graph)

def optimized_hdbscan_pipeline(X, min_cluster_size=50):
    X_scaled = StandardScaler().fit_transform(X)
    if X_scaled.shape[1] > 50:
        from umap import UMAP
        X_scaled = UMAP(n_components=10, random_state=42).fit_transform(X_scaled)
    return hdbscan.HDBSCAN(
        min_cluster_size=min_cluster_size,
        cluster_selection_method='eom',
        approx_min_span_tree=True,
        core_dist_n_jobs=-1
    ).fit_predict(X_scaled)
