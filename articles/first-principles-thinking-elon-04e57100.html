第一性原理思考（First Principles Thinking）

主題概述

第一性原理思考（First Principles Thinking）是一種將複雜問題分解為最基本、不可再化約的真理，然後從這些基礎真理重新建構解決方案的系統性思維方法。其核心理念是：拒絕類比推理的慣性，回歸事物本質，從根基重新思考問題。

2026 年，隨著 AI Agent 架構、創業方法論與認知科學的交叉發展，第一性原理思考已從哲學概念演變為跨領域的實用決策工具，被廣泛應用於工程設計、商業創新、產品開發與系統架構等領域。

一、哲學根源：從柏拉圖到笛卡爾

1.1 亞里斯多德的定義

亞里斯多德在《形而上學》中將第一性原理定義為「認識事物的最初基礎」（the first basis from which a thing is known）。他區分了兩種知識獲取方式：

  • 從第一性原理推導（episteme）：從不證自明的真理出發，透過演繹推理得出結論
  • 從經驗歸納（empeiria）：從觀察到的現象歸納出一般規律

1.2 笛卡爾的系統性懷疑

笛卡爾的「方法論懷疑」（Methodological Doubt）是第一性原理思考的哲學前身：懷疑一切可以懷疑的，直到找到不可懷疑的基礎真理（「我思故我在」）。

1.3 柏拉圖的理型論

柏拉圖認為存在超越感官經驗的「恆常基礎知識」，這為第一性原理提供了形而上學的根基——真正的知識必須建立在不變的基礎之上。

二、核心方法論：四步驟框架

步驟 1：定義問題（Define the Problem）

區分「真正的問題」與「症狀」：

  • 自問：「這是問題本身，還是問題的症狀？」
  • 用最簡潔、最精確的語句陳述問題
  • 避免在問題定義中預設解決方案

步驟 2：解構分解（Break It Down）

將問題拆解為組成部分：

  • 輸入（inputs）是什麼？
  • 需要什麼行動（actions）？
  • 有哪些約束條件（constraints）？
  • 每個組成部分的真實成本或價值是什麼？

步驟 3：挑戰假設（Challenge Assumptions）

使用蘇格拉底式提問法（Socratic Questioning）系統性質疑：

  1. 釐清思維：「為什麼我這樣想？我的推論依據是什麼？」
  2. 挑戰假設：「我怎麼知道這是真的？如果我想的是錯的呢？」
  3. 尋求證據：「有什麼證據支持這個觀點？」
  4. 考慮替代觀點：「如果換一個角度看，會怎樣？」
  5. 檢驗後果：「如果我錯了，最壞的結果是什麼？」
  6. 質疑問題本身：「為什麼我認為這個問題重要？」

另一種技術是「五個為什麼」（Five Whys）：持續追問「為什麼？」直到觸及根本原因，模仿兒童自然的追問方式。

步驟 4：重建方案（Rebuild the Solution）

僅從已確認的事實出發，重新建構解決方案：

  • 自問：「如果沒有先前的假設，我會怎麼做？」
  • 探索跨領域的組合創新可能性
  • 優先考慮功能（function）而非形式（form）

三、Elon Musk 的三步法與工程實踐

3.1 Musk 的物理學框架

Musk 明確表示：「我傾向用物理學框架思考問題。物理學教你從第一性原理推理，而不是從類比推理。」

3.2 SpaceX 案例：火箭成本革命

傳統思維（類比推理）：火箭發射成本約 6,500 萬美元，這是行業標準價格。

第一性原理分析：

  1. 解構：火箭由什麼材料組成？航太級鋁合金、鈦、銅、碳纖維
  2. 查證基礎事實：這些材料在大宗商品市場上的價格是多少？約為典型火箭售價的 2%
  3. 重建：自行採購原材料、自建製造產線
  4. 結果：SpaceX 將發射成本降低了近 90%

3.3 Tesla 案例：電池成本

傳統觀點：電池組成本每千瓦時 600 美元，太貴了。

第一性原理：電池的基本材料（鈷、鎳、鋰、碳、聚合物隔膜）在倫敦金屬交易所的價格遠低於成品電池。透過重新設計製造流程和供應鏈，大幅降低成本。

四、第一性原理 vs 類比推理

4.1 類比推理的陷阱

| 面向 | 類比推理 | 第一性原理 |

|------|---------|----------|

| 起點 | 「別人怎麼做」 | 「事物的本質是什麼」 |

| 風險 | 繼承既有假設和限制 | 需要更多認知努力 |

| 適用 | 成熟領域的漸進改進 | 突破性創新與根本變革 |

| 速度 | 快速但可能方向錯誤 | 慢但更可能找到根本解 |

| 常見錯誤 | 優化形式而非功能 | 過度簡化忽略實際約束 |

4.2 「飛行汽車」的思維陷阱

當人們問「飛行汽車在哪裡？」時，他們犯了類比推理的錯誤——將「交通工具」的形式（汽車 + 飛行）等同於「快速移動」的功能。實際上，飛機早已實現了這個功能。第一性原理思考聚焦功能（function），而非繼承形式（form）。

五、跨領域應用案例

5.1 John Boyd 的雪橇摩托車思想實驗

將不同載具解構為基本部件，然後重新組合：

  • 自行車的車把和座椅
  • 坦克的金屬履帶
  • 摩托艇的引擎

→ 組合成全新的雪橇摩托車（snowmobile）

這展示了第一性原理的核心：解構 → 跨域重組 → 創新。

5.2 古騰堡的印刷術

Johannes Gutenberg 將葡萄酒壓榨機的壓力機制與活字鑄造技術結合，發明了印刷機——這是跨領域第一性原理重組的經典案例。

5.3 Nick Kokonas 的餐廳預約革命

傳統假設：餐廳訂位的 no-show 是不可避免的。

第一性原理重構：為什麼餐廳不能像劇院一樣售票？Kokonas 創建了 Tock 平台，將餐廳預約改為預付票制，徹底消除了 no-show 問題。

5.4 BuzzFeed 的內容策略

傳統假設：好的媒體 = 好的內容。

第一性原理：媒體的第一性原理是「分發」，不是「內容品質」。BuzzFeed 將社交分享性（shareability）置於內容品質之上，成為數位媒體巨頭。

5.5 Eli Lilly 製藥工程

在藥品設計與開發中，Eli Lilly 運用第一性原理預測和解釋「為什麼在特定條件下會出現某種行為而非另一種」，超越了試錯法的經驗主義。

六、認知科學視角

6.1 與認知偏誤的關係

第一性原理思考直接對抗以下認知偏誤：

  • 錨定效應（Anchoring）：拒絕以既有數字或方案為起點
  • 從眾效應（Bandwagon Effect）：不因「大家都這樣做」而接受
  • 現狀偏誤（Status Quo Bias）：質疑「一直都是這樣」的慣性
  • 功能固著（Functional Fixedness）：重新定義物件或概念的用途
  • 確認偏誤（Confirmation Bias）：系統性質疑支持自身觀點的證據

6.2 認知成本

第一性原理思考需要大量的認知能量（System 2 思維），這解釋了為什麼多數人偏好類比推理（System 1 思維）。根據認知負荷理論，建議：

  • 不必對所有問題都使用第一性原理——僅在高影響、高不確定性的決策中使用
  • 建立「第一性原理清單」作為外部認知支架，降低工作記憶負擔
  • 團隊協作分擔認知負荷：不同成員負責不同假設的質疑

七、實踐指南：何時使用、何時不用

7.1 適用場景

  • 面對行業長期未解決的結構性問題
  • 臨時修補反覆失敗，需要根本性重新思考
  • 進入全新領域，無既有經驗可類比
  • 成本結構異常但「一直都是這樣」
  • 設計全新系統架構（如 AI Agent 管線設計）

7.2 不適用場景

  • 時間緊迫需要立即行動的戰術決策
  • 已有成熟最佳實踐的標準化流程
  • 問題規模小、影響低的日常決策
  • 領域知識不足以判斷「什麼是基本真理」

7.3 與其他框架的組合使用

| 組合 | 效果 |

|------|------|

| 第一性原理 + 系統思考 | 解構系統邊界假設，重新定義槓桿點 |

| 第一性原理 + 貝氏推理 | 從根基重新審視先驗假設的合理性 |

| 第一性原理 + SATs（ACH） | 用第一性原理生成被忽視的競爭假設 |

| 第一性原理 + 認知負荷理論 | 簡化問題至最小認知負擔的核心要素 |

| 第一性原理 + 心智模型格柵 | 用第一性原理選擇最適合的心智模型 |

八、常見陷阱與防範

  1. 過度簡化：忽略實際約束（法規、文化、物理限制），導致理論上正確但實踐上不可行
  2. 無限回歸：不斷追問「為什麼」卻找不到停止點——需要設定實用的「基岩層」
  3. 忽略脈絡：第一性原理是脈絡相依的，不同領域的「基本真理」不同
  4. 獨斷主義：誤以為自己找到的第一性原理就是「唯一真理」
  5. 分析癱瘓：花太多時間解構而未行動——需要設定時間框架
