Harness Engineering 深化優化方案（v5.0）

執行摘要

整合 2026 年 4 月最新突破（Harvey Legal Agents、Meta-Harness、Shaped.ai Context 排名），提出三大核心洞見與八項深化優化。

最新突破數據：

  • Harvey：40.8% → 87.7%（+116% 成功率）
  • Meta-Harness：TerminalBench-2 Opus 排名 #2（76.4%）、Haiku 排名 #1（37.6%）
  • Context 排名：10x 成本降低（$48k → $4.5k/月）+ 品質 +20%
  • 物理約束：Attention O(n²) — 2x tokens = 4x cost

---

一、三大核心洞見

洞見 1：Autoresearch + Harness = Hill-Climbing Engine

來源：Harvey Legal Agents 實驗

機制：Agent 嘗試 → LLM 評分（依 rubric）→ Coding agent 分析失敗叢聚 → 形成假設 → 修改組件 → 重新執行 → 循環

實證（12 個法律任務）：

  • 初始：5 個任務成功率 2-7%
  • 最終：7 個任務 >90%（1 個 100%）
  • 平均：40.8% → 87.7%（+115%）
"When the rubric is high quality, the agent can hill-climb surprisingly far."

對本專案：

  • 當前：手動改進（improvement-backlog.json）
  • 缺口：無自動化 rubric 評分 + 假設形成
  • 機會：Task 成功率 75% → 90%+

實施：

  1. 建立高品質 Rubrics（35 個任務 × 5-10 維度）
  2. Autoresearch Engine（失敗叢聚分析 + 假設形成）
  3. 每週六自動執行，P0 假設自動通知

---

洞見 2：Meta-Harness — 自動化優於手工

來源：Meta-Harness 論文（Stanford）

性能：TerminalBench-2 Opus 排名 #2（76.4%），Haiku 排名 #1（37.6%），診斷資訊量 10M tokens/iteration（比傳統多 1000 倍）

對本專案：用 Meta-Harness 自動發現更優 Harness 版本，預期 ≥3 個低成功率任務大幅改進

---

洞見 3：Context 排名比填充重要 10 倍

來源：Shaped.ai Context Window 優化

量化對比：Naive RAG（50K tokens → $48k/月）vs. Shaped 排名（2.5K tokens → $4.5k/月），成本降 91%、品質升 20%

關鍵：多檢索器管線（Vector + Lexical + Collaborative + Trending + ML + Diversity）
