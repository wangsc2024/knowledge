Evals for Taste: 幻燈片生成 Agent 爬坡 — 品質驅動的 Agent 迭代方法（Code with Claude London）

youtuberesearchautoanthropicevalagentslide-generationopus

Anthropic 工程師在 Code with Claude London 下午場分享「以 Eval 為品味導向」的 Agent 迭代方法，以幻燈片生成 Agent 為實例演示完整 Hill-Climbing 流程。

Eval 的必要性：缺乏 Eval 時只能靠直覺反應，無法分辨真實問題與雜訊，無法驗證迭代是否有效。Eval 迫使工程師明確定義「成功」的樣子。

評分器三類：

  • 程式碼評分器（確定性）：表情符號計數、幻燈片張數、雜亂度、小字比例，速度快、成本低，但脆弱易碎
  • 模型評分器（LLM-as-judge）：文字連貫性、色彩搭配、布局評估，靈活但需校準。最佳實踐：先要理由再給分（避免錨定效應）；多評審共識（三局兩勝）可增加確定性
  • 人工審查：成本最高，用於抽樣與 A/B 測試

Hill-Climbing 流程：初始 Agent（Sonnet 4.7 極簡提示）→ 識別失效模式（表情符號氾濫、小字、布局雜亂）→ 更新系統提示（排版規範、禁用 AI 套語）→ 加入 QA 循環（批評 Agent 審查輸出並回饋）→ 升級為 Opus 4.7 → 所有評審分提升至 4.2-4.8/5。

關鍵洞察：Opus 4.7 具備業務幻燈片的天然「品味」（不用表情符號、布局感知更佳），基礎提示即可超越精細調教的 Sonnet 4.7。Eval 需隨時間演進，避免 Eval Saturation（評分器不再提供有效資訊）。
