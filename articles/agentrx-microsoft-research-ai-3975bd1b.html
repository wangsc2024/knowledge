AgentRx — AI Agent 自動除錯框架

概述

AgentRx 是 Microsoft Research 於 2026 年 3 月 12 日發布的開源框架，專門用於自動診斷 AI Agent 在複雜任務中失敗的根因。隨著 AI Agent 從簡單聊天機器人演進為能管理雲端事件、導航複雜網頁介面、執行多步驟 API 工作流的自主系統，AgentRx 解決了其中最關鍵的可觀察性與透明度問題。

論文：AgentRx: Diagnosing AI Agent Failures from Execution Trajectories（arXiv: 2602.02475）

---

核心架構：六階段診斷管線

AgentRx 將 Agent 執行視為需要驗證的系統軌跡（trajectory），採用結構化的六階段管線進行診斷：

1. Trajectory IR（軌跡中間表示）

將來自不同領域的異質性執行日誌正規化為通用中間表示格式，消除跨平台差異。

2. Static Invariants（靜態不變量）

從任務規格中識別預期的行為模式，建立「應該發生什麼」的約束條件。

3. Dynamic Invariants（動態不變量）

從實際 Agent 執行過程中推導運行時約束，捕捉「實際發生了什麼」。

4. Checker（檢查器）

交叉比對靜態預期與觀察到的實際行為，逐步評估約束條件，僅在守衛條件（guard conditions）適用時執行，產出可審計的違規證據日誌。

5. Judge（LLM 判定器）

LLM 分析驗證日誌與九類失敗分類法，判定偏差是否構成真正的失敗，識別關鍵失敗步驟（Critical Failure Step）——軌跡變得不可恢復的第一個錯誤點。

6. Report（報告生成）

綜合診斷結果，輸出定位到具體步驟的失敗報告與根因分析。

---

九類失敗分類法（Failure Taxonomy）

AgentRx 定義了九個失敗類別，幫助開發者區分 Agent 出錯的細微差異：

  1. 計畫遵守失敗（Plan Adherence Failure）：Agent 偏離既定計畫
  2. 資訊虛構（Invention of New Information）：Agent 幻覺產生不存在的資訊
  3. 工具調用錯誤（Tool Invocation Error）：API 呼叫參數錯誤或選錯工具
  4. 規劃崩潰（Planning Breakdown）：多步驟推理鏈斷裂
  5. 幻覺能力（Hallucinated Capabilities）：Agent 呼叫不存在的工具或功能
  6. 上下文遺失（Context Loss）：長對話中遺忘關鍵資訊
  7. 約束違反（Constraint Violation）：違反領域策略或安全規則
  8. 迴圈行為（Looping Behavior）：重複執行相同失敗動作
  9. 提前終止（Premature Termination）：未完成任務即停止

基準測試與效能

AgentRx Benchmark

  • 115 個人工標注的失敗軌跡，跨三個領域：

- tau-bench：結構化 API 工作流

- Flash：真實世界事件管理

- Magentic-One：多 Agent 網頁任務

效能提升（vs. prompting baselines）

  • 失敗定位準確率 +23.6%
  • 根因歸因準確率 +22.9%

關鍵創新

核心突破在於約束合成（Constraint Synthesis）——自動從工具 schema 和領域策略生成可執行的驗證規則，而非依賴單一語言模型做錯誤偵測。

已知限制與改進方向

無跨 Session 記憶

AgentRx 最大的限制是每次分析都從零開始，不會跨部署累積失敗模式知識：

  • 反覆出現的失敗模式無法跨部署識別
  • 九類分類法保持靜態，無法從經驗中擴展
  • 團隊重複推導相同的根因
  • 無法累積關於失敗預防的制度知識

潛在整合方向

  • 搭配 Agent Memory 系統（如 MemU）將診斷結果轉為可查詢的知識圖譜
  • 跨部署趨勢偵測，識別系統性問題
  • 解決方案記憶，將過去的修復方案連結到反覆出現的失敗

對本專案的啟示

  1. Guardian 系統對齊：AgentRx 的九類分類法可與本專案 agent_guardian.py 的 ErrorClassifier 對照，考慮擴展分類維度
  2. 軌跡日誌標準化：hooks/post_tool_logger.py 的 JSONL 日誌可作為 AgentRx Trajectory IR 的輸入格式
  3. 約束合成思路：從 tool schema 自動生成驗證規則的思路，可應用於 Hook 規則自動化生成
  4. 關鍵失敗步驟：trace_analyzer.py 的根因分析可借鑑 AgentRx 的「不可恢復點」概念
