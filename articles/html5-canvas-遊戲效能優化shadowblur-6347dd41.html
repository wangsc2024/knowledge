HTML5 Canvas 遊戲效能優化筆記

問題：shadowBlur 是 Canvas 2D 最大效能殺手

在「輪迴塔」(Tower of Samsara) 遊戲優化中，發現 ctx.shadowBlur 是造成 FPS 下降的首要原因。

根因分析：

  • Canvas 2D 的 shadowBlur 底層使用高斯模糊，每次繪製都需要額外的像素運算
  • 若每幀有 60 個平台各設 shadowBlur = 10，等同每幀 60 次高斯模糊
  • 在低端裝置上可導致 FPS 從 60 降至 20-30

替代方案：多層半透明繪製

平台發光效果：

// 用較大的半透明矩形模擬光暈
ctx.globalAlpha = 0.12;
ctx.fillStyle = accent;
const pad = 4 * scale;
ctx.fillRect(x - pad, y - pad, w + pad * 2, h + pad * 2);
ctx.globalAlpha = 1;

角色發光效果：

// 用半透明圓形模擬身體光暈
ctx.globalAlpha = 0.15;
ctx.fillStyle = accent;
ctx.beginPath();
ctx.arc(x, y + h/2, 18 * scale, 0, Math.PI * 2);
ctx.fill();
ctx.globalAlpha = 1;

文字發光效果：

// 先畫較大的半透明文字作底，再畫正常文字
ctx.globalAlpha = 0.25;
ctx.font = `bold ${bigSize}px serif`;
ctx.fillText(text, x, y);
ctx.globalAlpha = 1;
ctx.font = `bold ${normalSize}px serif`;
ctx.fillText(text, x, y);

其他效能最佳實踐

  1. crumble 動畫避免 Math.random()：用確定性函數 Math.sin(timer * 0.5 + index * 2.1) 取代，消除閃爍
  2. 物件池：粒子(200)、平台(60)、浮動文字(30) 使用預建池
  3. Fixed timestep：accumulator += dt; while (accumulator >= FIXED_DT) { update(); }
  4. 離開視窗判斷：if (sy > H + 20 || sy < -40) return; 跳過螢幕外繪製

觸控優化要點

  • 必須提供可見的虛擬按鍵，隱形分區操作體驗極差
  • touchcancel 事件必須處理，否則 iOS 來電等中斷會卡住移動
  • touchmove 中重新評估所有觸點，支援平滑滑動換方向

成就系統設計

  • 使用 localStorage 持久化，與 highScore 一同存儲
  • 佛學主題命名增加文化特色（五蘊皆空、十善業道、百丈竿頭等）
  • 成就在 Game Over 時檢查，新解鎖的成就直接在結束畫面顯示
