Hermes Agent（NousResearch/hermes-agent）GitHub 熱門自主 AI Agent 研究（2026-04-10）

概述

Hermes Agent 是由 Nous Research 維護的開源自主 AI Agent 專案（GitHub：NousResearch/hermes-agent，授權 MIT）。官方定位強調「會隨使用時間變得更強」：內建學習迴路（從經驗建立／改進 Skills）、持久化記憶與跨工作階段檢索（README 描述含 FTS5 工作階段搜尋與 LLM 摘要、以及 Honcho 使用者建模等機制）。與單一 IDE 外掛型助理不同，Hermes 設計為可部署在 VPS、GPU 叢集或近乎閒置時零成本的 serverless 形態，並可透過 Telegram、Discord、Slack、WhatsApp、Signal 等通道與 CLI 交談（同一 gateway 進程）。

安裝邊界（重要）：官方 README 明確指出 原生 Windows 不支援，需 WSL2 後再執行安裝腳本。

重點（功能模組）

  • 多模型供應商切換：可用 Nous Portal、OpenRouter、OpenAI、Anthropic 相容端點或自架端點；以 hermes model 切換，降低供應商鎖定。
  • 真實終端機體驗：互動式 TUI、slash command、對話歷史、工具輸出串流等。
  • Skills 系統：Skills 以 SKILL.md 形式管理，採 progressive disclosure 以降低 token；文件說明與 agentskills.io 開放標準相容，並支援從社群／目錄安裝與本機 /.hermes/skills/ 作為主要來源（詳見官方 Skills 文件）。
  • 工具與延伸：文件導覽包含 Tools & Toolsets、MCP 整合、Cron 排程（自然語言建立定期任務並可送到各平台）、Context files（專案脈絡）。
  • 執行後端（終端／沙箱）：README 列舉多種 terminal backends（如 local、Docker、SSH、Daytona、Singularity、Modal），用於在不同隔離與運維條件下執行工具鏈。
  • 從 OpenClaw 遷移：提供 hermes claw migrate 與精靈偵測 /.openclaw，可匯入 persona、記憶、Skills、訊息平台設定與部分金鑰（仍需人為審核安全邊界）。

分析（對本專案 daily-digest-prompt 的啟發）

  1. Skill-First 與可攜 Skills：Hermes 將 Skills 視為「程序性記憶」並與 agentskills.io 對齊，與本專案 skills/*/SKILL.md 的文件驅動策略高度共鳴；可借鏡的是 hub 安裝、版本信任與外部目錄掃描（官方文件提到 xternal_dirs）在多工具並存時的治理。
  2. Gateway + 多通道：與本專案 ntfy／排程 Agent 的「非同步交付」類似，Hermes 把 cron 與 多平台訊息納入同一套執行邊界，可參考其 安全章節（命令核准、DM pairing、容器隔離）作為工具權限模型的對照閱讀。
  3. 運維與成本：後端可選 Modal／Daytona 等「休眠喚醒」形態，對長時背景任務的成本控管有參考價值；但本專案仍應以既有 PowerShell 管線與 ADR 邊界為準，不應直接混用未經評估的遠端執行器。
  4. 研究倫理與來源鑑別：GitHub 上可能存在名稱相近或抄襲 README 的倉庫；請以 NousResearch/hermes-agent 與 hermes-agent.nousresearch.com 網域為準，避免引用可疑分叉內容。
