Claude Code CI/CD 整合與 Headless 自動化完整指南

主題概述

Claude Code 的 CI/CD 整合與 Headless 自動化是將 AI 編程助手從互動式開發工具擴展為自動化基礎設施的核心能力。透過 -p（print）旗標、Agent SDK 程式化介面、GitHub Actions 官方整合與 GitLab CI/CD 支援，開發團隊可以在持續整合管線中嵌入 AI 驅動的程式碼審查、自動修復、測試生成與文件產出。2026 年的資料顯示，超過 60% 採用 Claude Code 的團隊使用非互動式模式自動化重複任務，平均減少 lint 錯誤解決時間 85%、程式碼審查時間 40%。

一、Headless 模式（CLI `-p` 旗標）

1.1 基本概念

Headless 模式（現已整合入 Agent SDK）是 Claude Code 的非互動式執行模式，透過 -p 旗標將提示作為單一命令發送，取得回應後立即退出，無需互動介面。


claude -p "Find and fix the bug in auth.py" --allowedTools "Read,Edit,Bash"

1.2 三種輸出格式

| 格式 | 用途 | 解析方式 |

|------|------|----------|

| text（預設） | CI 日誌、終端顯示 | 直接輸出 |

| json | 腳本整合、API 串接 | jq 或 JSON 解析器 |

| stream-json | 即時儀表板、串流處理 | NDJSON 逐行解析 |

JSON 回應包含：type、result、cost_usd、duration_ms、num_turns、session_id。

1.3 結構化輸出（JSON Schema）

可透過 --json-schema 指定輸出格式，Claude 會嚴格遵守 Schema 定義：


claude -p "Extract the main function names from auth.py" \
  --output-format json \
  --json-schema '{"type":"object","properties":{"functions":{"type":"array","items":{"type":"string"}}},"required":["functions"]}'

結構化輸出存放在回應的 structured_output 欄位。

1.4 串流輸出

使用 --output-format stream-json --verbose --include-partial-messages 可接收逐 token 生成的 NDJSON 事件：


claude -p "Write a poem" --output-format stream-json --verbose --include-partial-messages | \
  jq -rj 'select(.type == "stream_event" and .event.delta.type? == "text_delta") | .event.delta.text'

1.5 工具權限控制

--allowedTools 使用權限規則語法（Permission Rule Syntax），支援前綴匹配：


# 允許 git 相關指令（注意 * 前的空格）
--allowedTools "Bash(git diff *),Bash(git log *),Bash(git status *),Bash(git commit *)"

安全最佳實踐：CI/CD 環境中應始終限制工具為最小必要集合。

1.6 系統提示自訂

  • --append-system-prompt：追加指令，保留預設行為
  • --system-prompt：完全替換預設系統提示

gh pr diff "$1" | claude -p \
  --append-system-prompt "You are a security engineer. Review for vulnerabilities." \
  --output-format json

1.7 多輪對話（Session 管理）


# 首次請求
claude -p "Review this codebase for performance issues"

# 繼續最近的對話
claude -p "Now focus on the database queries" --continue

# 恢復特定 session
session_id=$(claude -p "Start a review" --output-format json | jq -r '.session_id')
claude -p "Continue that review" --resume "$session_id"

每個 session 保留約 200,000 tokens 的上下文。

二、GitHub Actions 整合

2.1 核心功能

Claude Code GitHub Actions（anthropics/claude-code-action@v1）是官方維護的 GitHub Action，支援：

  • @claude 提及觸發：在 PR 或 Issue 留言中 @claude 即可觸發
  • 自動 PR 建立：描述需求後 Claude 自動建立完整的 PR
  • 程式碼實作：將 Issue 轉化為可運作的程式碼
  • 技能整合：支援 /review、/fix 等預建技能
  • CLAUDE.md 感知：遵循專案中的 CLAUDE.md 編碼標準

2.2 基本工作流配置


name: Claude Code
on:
  issue_comment:
    types: [created]
  pull_request_review_comment:
    types: [created]
jobs:
  claude:
    runs-on: ubuntu-latest
    steps:
      - uses: anthropics/claude-code-action@v1
        with:
          anthropic_api_key: ${{ secrets.ANTHROPIC_API_KEY }}

2.3 自動化排程工作流


name: Daily Report
on:
  schedule:
    - cron: "0 9 * * *"
jobs:
  report:
    runs-on: ubuntu-latest
    steps:
      - uses: anthropics/claude-code-action@v1
        with:
          anthropic_api_key: ${{ secrets.ANTHROPIC_API_KEY }}
          prompt: "Generate a summary of yesterday's commits and open issues"
          claude_args: "--model opus"

2.4 v1.0 GA 版本參數

| 參數 | 說明 | 必填 |

|------|------|------|

| prompt | Claude 的指令（文字或技能如 /review） | 否 |

| claude_args | 傳遞給 Claude CLI 的參數 | 否 |

| anthropic_api_key | API 金鑰 | 是（直連）|

| github_token | GitHub 存取權杖 | 否 |

| trigger_phrase | 觸發詞（預設 @claude） | 否 |

| use_bedrock | 使用 AWS Bedrock | 否 |

| use_vertex | 使用 Google Vertex AI | 否 |

2.5 企業雲端整合

支援三種認證方式：

  • Claude API 直連：ANTHROPIC_API_KEY
  • AWS Bedrock：透過 OIDC 角色假設（AWS_ROLE_TO_ASSUME）
  • Google Vertex AI：透過 Workload Identity Federation（GCP_WORKLOAD_IDENTITY_PROVIDER）

三、GitLab CI/CD 整合（Beta）

3.1 核心架構

GitLab 整合由 GitLab 官方維護（非 Anthropic），採用三層架構：

  1. 事件驅動編排：監聽 @claude 提及、MR 事件、Web pipeline
  2. 供應商抽象：Claude API / AWS Bedrock / Google Vertex AI
  3. 沙箱執行：每個任務在隔離容器中執行，寫入受限

3.2 基本 .gitlab-ci.yml


stages:
  - ai

claude:
  stage: ai
  image: node:24-alpine3.21
  rules:
    - if: '$CI_PIPELINE_SOURCE == "web"'
    - if: '$CI_PIPELINE_SOURCE == "merge_request_event"'
  variables:
    GIT_STRATEGY: fetch
  before_script:
    - apk update
    - apk add --no-cache git curl bash
    - curl -fsSL https://claude.ai/install.sh | bash
  script:
    - /bin/gitlab-mcp-server || true
    - >
      claude
      -p "${AI_FLOW_INPUT:-'Review this MR and implement changes'}"
      --permission-mode acceptEdits
      --allowedTools "Bash Read Edit Write mcp__gitlab"
      --debug

3.3 安全治理

  • 每個 job 在隔離容器中執行
  • 所有變更通過 MR 流程，審閱者可見完整 diff
  • 分支保護與核准規則適用於 AI 生成的程式碼
  • Claude Code 使用工作區範圍權限限制寫入

四、Claude Agent SDK 程式化控制

4.1 SDK 概述

Claude Agent SDK（原 Claude Code SDK）提供與 Claude Code 相同的工具、agent 迴圈和上下文管理，可在 Python 和 TypeScript 中程式化使用。

版本（2026-02）：Python v0.1.34 / TypeScript v0.2.37

4.2 核心能力矩陣

| 能力 | 說明 |

|------|------|

| 內建工具 | Read、Write、Edit、Bash、Glob、Grep、WebSearch、WebFetch |

| Hooks 回呼 | PreToolUse、PostToolUse、Stop、SessionStart、SessionEnd |

| 子 Agent | 透過 AgentDefinition 定義專責子 Agent 委派任務 |

| MCP 整合 | 連接外部系統（資料庫、瀏覽器、API） |

| 權限模型 | allowedTools + permissionMode（bypassPermissions、acceptEdits） |

| Session 管理 | 跨多輪對話保持上下文，支援恢復與分支 |

| 結構化輸出 | JSON Schema 約束輸出格式 |

4.3 Python SDK 基本用法


import asyncio
from claude_agent_sdk import query, ClaudeAgentOptions

async def main():
    async for message in query(
        prompt="Find and fix the bug in auth.py",
        options=ClaudeAgentOptions(allowed_tools=["Read", "Edit", "Bash"]),
    ):
        if hasattr(message, "result"):
            print(message.result)

asyncio.run(main())

4.4 TypeScript SDK 基本用法


import { query } from "@anthropic-ai/claude-agent-sdk";

for await (const message of query({
  prompt: "Find and fix the bug in auth.py",
  options: { allowedTools: ["Read", "Edit", "Bash"] }
})) {
  if ("result" in message) console.log(message.result);
}

4.5 SDK vs CLI 選型指南

| 使用場景 | 最佳選擇 |

|----------|----------|

| 互動式開發 | CLI |

| CI/CD 管線 | SDK |

| 自訂應用程式 | SDK |

| 一次性任務 | CLI |

| 生產自動化 | SDK |

五、常見 CI/CD 自動化場景

5.1 AI 驅動的程式碼審查


steps:
  - uses: anthropics/claude-code-action@v1
    with:
      anthropic_api_key: ${{ secrets.ANTHROPIC_API_KEY }}
      prompt: "/review"
      claude_args: "--max-turns 5"

效益：平均減少審查時間 40%。

5.2 自動測試生成


claude -p "Generate unit tests for the auth module" \
  --allowedTools "Read,Write" \
  --max-turns 5

效益：平均覆蓋 72% 的未測試程式碼分支。

5.3 Lint 錯誤自動修復


claude -p "Fix all ESLint errors in src/" \
  --allowedTools "Read,Edit,Bash(npx eslint *)" \
  --output-format json

效益：平均減少解決時間 85%。

5.4 自動提交訊息


claude -p "Look at my staged changes and create an appropriate commit" \
  --allowedTools "Bash(git diff *),Bash(git log *),Bash(git status *),Bash(git commit *)"

六、安全最佳實踐

6.1 五大安全規則

  1. 限制工具權限：--allowedTools Read,Write（CI 中避免 Bash）
  2. 密鑰管理：使用 GitHub Secrets / GitLab CI Variables / Vault
  3. 限制迭代次數：--max-turns 3-10 防止無限迴圈
  4. 驗證輸出：對 JSON 回應進行 Schema 驗證
  5. 審計日誌：--verbose 2>claude-audit.log

6.2 成本控制

  • 典型 CI 執行成本：$0.03-$0.20 / 次
  • 設定適當的 --max-turns 防止過度迭代
  • 使用 GitHub/GitLab 的並行控制限制同時執行數
  • 設定工作流層級的 timeout 避免失控

6.3 環境變數

| 變數 | 用途 | 預設值 |

|------|------|--------|

| ANTHROPIC_API_KEY | API 認證 | 必填 |

| CLAUDE_CODE_MAX_TURNS | 迭代上限 | 無限制 |

| CLAUDE_CODE_MODEL | 模型選擇 | claude-sonnet-4-6 |

| CLAUDE_CODE_OUTPUT_FORMAT | 輸出格式 | text |

| CLAUDE_CODE_VERBOSE | 除錯日誌 | false |

七、與本專案（daily-digest-prompt）的關聯

本專案廣泛使用 Claude Code 的 Headless 模式進行排程自動化：

| 本專案實踐 | 對應 CI/CD 技術 |

|-----------|----------------|

| claude -p --allowedTools | Headless 工具權限控制 |

| PowerShell Start-Job 並行 | 類似 GitHub Actions 矩陣策略 |

| --output-format json 結果解析 | JSON 結構化輸出管線 |

| --resume session 恢復 | 多輪對話上下文維持 |

| CLAUDE.md 專案指引 | GitHub Actions 中的 CLAUDE.md 感知 |

潛在改進方向

  1. Agent SDK 遷移：從 CLI -p 遷移至 Python Agent SDK，獲得更細粒度的控制（hooks 回呼、結構化輸出、tool approval）
  2. JSON Schema 約束：為自動任務結果定義 JSON Schema，確保輸出一致性
  3. 串流監控：使用 stream-json 格式實現即時執行監控儀表板
  4. GitHub Actions 整合：將部分自動任務遷移至 GitHub Actions 以利用雲端計算資源
