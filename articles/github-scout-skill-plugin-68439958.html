一、專案概覽

1. obra/superpowers（106K stars）

專案: obra/superpowers

相關性: ⭐⭐⭐ 高

最後更新: 2026-03-17

核心特色:

  • 強制工作流: brainstorming → planning → TDD，防止 Agent 直接跳到編碼
  • Skills 自動觸發: 根據專案上下文自動啟用對應 Skills（非可選建議）
  • Subagent 並行: 任務拆解後派發專屬 sub-agents 並行執行，兩階段審查
  • Git Worktrees 隔離: 風險操作在臨時 worktree 執行，驗證通過再 merge
  • 跨平台支援: Claude Code / Cursor / Codex / Gemini CLI

技術棧: Shell (57.8%) / JavaScript (30.1%) / Python (3.9%)

關鍵模式:

  1. Skills 是 mandatory workflows，非 optional suggestions
  2. TDD-first 強制，包含 anti-pattern 指導
  3. Systematic debugging：4-phase 根因分析框架

2. sickn33/antigravity-awesome-skills（27K stars）

專案: antigravity-awesome-skills

相關性: ⭐⭐⭐ 高

最後更新: 2026-03-23

核心特色:

  • 1,306+ Skills 庫: 涵蓋 Architecture / Security / Data & AI 等領域
  • NPX 一鍵安裝: npx antigravity-awesome-skills，支援多平台路徑（--claude / --cursor / --gemini）
  • Bundle 策略: 按角色/目標分組（Full-Stack Developer / Security Developer / OSS Maintainer）
  • 機器可讀索引: skills_index.json（機器解析）+ CATALOG.md（人類瀏覽）
  • Workflow Playbooks: Step-by-step 執行劇本（Ship a SaaS MVP / Security Audit / Build AI Agent Systems）
  • 安全掃描: GitHub Actions 自動審查高風險命令

技術棧: SKILL.md 標準格式，跨平台一致

關鍵模式:

  1. Bundle ≠ Separate Installations: 一次安裝，按需引用 bundle 中的 skills
  2. 機器可讀 Registry: 觸發詞衝突偵測、依賴圖驗證
  3. 社區驅動: 官方 + 社區貢獻，自動化審查降低風險

專案: NocoBase

最後更新: 2026-03

二、改進建議（6 個，含 4 個 P0/P1）

P0 建議（2 個）

#### 1. 機器可讀 Skill Registry（skills_index.json）

來源: antigravity-awesome-skills

目標檔案: skills/skill-registry.json, skills/SKILL_INDEX.md

優先級: P0

工作量: low

描述:

建立 skills/skill-registry.json（機器可讀），包含每個 Skill 的 name / version / triggers / depends-on / allowed-tools，由 skill-registry-sync Skill 自動掃描 frontmatter 生成。

效益:

  • ✅ 觸發詞衝突偵測（避免多個 Skill 搶同一關鍵字）
  • ✅ 依賴圖驗證（檢查 depends-on 的 Skill 是否存在）
  • ✅ LLM 快速查詢（直接讀 JSON 而非遍歷 61 個 SKILL.md）

---

P1 建議（2 個）

#### 2. Bundle 策略（角色/目標導向的 Skill 組合）

來源: antigravity-awesome-skills

目標檔案: config/skill-bundles.yaml, skills/SKILL_INDEX.md

優先級: P1

工作量: medium

描述:

建立 config/skill-bundles.yaml，定義 bundles（如：daily-digest-core, todoist-agent-core, research-bundle），每個 bundle 含 skills 陣列。

效益:

  • ✅ 簡化 Agent 啟動（讀取 bundle 而非逐一讀 SKILL.md）
  • ✅ 角色明確（todoist-agent 自動載入 todoist + routing + scoring bundles）
  • ✅ 降低 I/O（減少 Read 呼叫次數）

---

#### 3. Skills 自動觸發機制（context-based activation）

來源: superpowers

目標檔案: hooks/skill_trigger_guard.py, config/skill-triggers.yaml

優先級: P1

工作量: high

描述:

新增 PostToolUse Hook，監控 prompt 內容與 Skill triggers 關鍵字匹配，自動提示 Agent「建議使用 XXX Skill」（非強制，但高優先級提示）。

效益:

  • ✅ 強化 Skill-First 執行率（防止 Agent 遺漏已有 Skill）
  • ✅ 主動提醒（Agent 看到關鍵字就收到建議）
  • ✅ 降低 Skill 被繞過的機率

---

#### 4. Git Worktrees 隔離機制

來源: superpowers

目標檔案: tools/worktree_manager.py, config/worktree-policy.yaml

優先級: P1

工作量: medium

描述:

新增 worktree_manager.py，高風險操作（如 github-scout 落實中風險方案）在臨時 worktree 執行，驗證通過後 merge，失敗直接刪除 worktree。

效益:

  • ✅ 安全實驗（不污染主分支）
  • ✅ 零成本 rollback（worktree 刪除即回到原點）
  • ✅ 保護主分支穩定性

---

P2 建議（2 個）

#### 5. Workflow Playbooks（step-by-step 執行劇本）

來源: antigravity-awesome-skills

目標檔案: workflows/playbooks/, workflows/index.yaml

優先級: P2

工作量: medium

描述:

擴充 workflows/ 為 Playbooks 目錄，將現有 validation-checklist 改為可執行 playbook（含 steps / verification / rollback）。

效益:

  • ✅ 標準化操作（每次執行一致）
  • ✅ 可重複（CI/CD 可自動化執行）
  • ✅ 降低人工錯誤

---

#### 6. Skill 安全掃描（GitHub Actions 自動化審查）

來源: antigravity-awesome-skills

目標檔案: .github/workflows/skill-security-scan.yml, tools/skill_security_scanner.py

優先級: P2

工作量: low

描述:

新增 GitHub Actions workflow，PR 觸發時掃描新增/修改的 SKILL.md，檢查高風險命令（rm -rf / / curl | bash / eval）、敏感路徑、硬編碼端點。

效益:

  • ✅ 社區貢獻安全審查（若開源）
  • ✅ 降低惡意 Skill 風險
  • ✅ CI/CD 整合（自動化檢查）

三、與本系統的對照分析

| 面向 | 本系統 | superpowers | antigravity |

|------|--------|-------------|-------------|

| Skill 數量 | 61 | ~20 核心 | 1,306+ |

| 觸發機制 | 手動引用 | 自動觸發（context-based） | 手動引用 |

| 安裝方式 | 內建（D:\Source\skills） | 跨平台 marketplace | NPX 一鍵安裝 |

| Registry | SKILL_INDEX.md（人類可讀） | 無獨立 registry | skills_index.json（機器可讀） |

| Bundle 策略 | 無（依 task_type 選擇） | 無 | Bundle 分組 |

| Worktrees | 無 | 有（EnterWorktree 工具） | 無 |

| 安全掃描 | Hook 攔截（runtime） | 無 | GitHub Actions（pre-merge） |

四、優先落實路徑

Phase 1（低成本高效益）

  1. P0-1: 機器可讀 Skill Registry（1-2 小時）

- 新建 skills/skill-registry.json

- 用 Python 腳本掃描 61 個 SKILL.md frontmatter

- 寫入 JSON（含 triggers 衝突檢查）

  1. P2-6: Skill 安全掃描（2-3 小時）

- 新增 .github/workflows/skill-security-scan.yml

- 掃描高風險關鍵字（預定義 blacklist）

Phase 2（中等投入）

  1. P1-2: Bundle 策略（3-4 小時）

- 設計 config/skill-bundles.yaml schema

- 定義 3-5 個核心 bundles

- 更新 Agent 啟動 prompt（先讀 bundle）

  1. P1-4: Git Worktrees 隔離（4-5 小時）

- 實作 tools/worktree_manager.py（create / verify / merge / cleanup）

- 整合到 github-scout 步驟 9

Phase 3（高投入長期收益）

  1. P1-3: Skills 自動觸發（6-8 小時）

- 新增 hooks/skill_trigger_guard.py（PostToolUse）

- 匹配 triggers 關鍵字，動態提示

- 測試覆蓋率（避免誤觸發）

  1. P2-5: Workflow Playbooks（5-6 小時）

- 設計 Playbook schema（steps / verification / rollback）

- 遷移現有 4 個 validation-checklist
