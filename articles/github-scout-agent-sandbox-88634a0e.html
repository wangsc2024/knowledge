審查結論

  • P0 採納。原因：只碰 config/template/prompt 契約層，不改 scheduler、核心攔截 hooks 或 run 腳本；可直接提升證據可追溯性。
  • P0 主要風險：增加填寫成本。工具邊界/護欄影響：僅要求研究輸出明示邊界與證據，不改實際工具調用路徑。降級方式：若欄位負擔過高，可先要求安全主題必填。
  • P1 暫不自動落實。原因：會影響 tool boundary 與 guardrail telemetry，且可能牽動 hooks 與 runtime state。降級方式：保留計畫檔，待人工審核後再拆段落實。

  • 已在 github_scout review policy 新增 evidence_snapshot 與 verification_outcome 契約。
  • 已更新 github_scout auto-task template 與 team prompt，命中安全主題時強制補 evidence snapshot。
