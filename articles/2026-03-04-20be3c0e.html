優化摘要

本次優化聚焦於「並行安全」與「文件一致性」兩大主軸，共完成 7 項優化。

優化項目

1. behavior_tracker.py 原子寫入（Critical）

  • 問題：_save_patterns() 使用 json.dump() 直接寫入，團隊模式多 Agent 並行時可能導致 JSON 損壞
  • 修正：改用 hook_utils.atomic_write_json()（write-to-temp + os.replace()），含 ImportError 退回機制

2. behavior_tracker.py file_lock 保護（High）

  • 問題：track() 的 read-modify-write 序列無互斥保護，多進程並行時可能資料遺失
  • 修正：加入 hook_utils.file_lock 包裹整個 read-modify-write 序列，含 ImportError 退回（nullcontext）

3. on_stop_alert.py Gmail OAuth state 原子寫入（High）

  • 問題：三處 Gmail OAuth state 寫入使用 json.dump() 直接寫入
  • 修正：改用 atomic_write_json()，含 ImportError 退回機制

4. CLAUDE.md 自動任務數量修正（Medium）

  • 問題：三處標註「47 次/日」，但實際活躍上限為 17 次/日
  • 修正：更新為「17 次/日活躍」，群組數字對齊 daily_limit 實際值

5. MEMORY.md 測試數量修正（Medium）

  • 問題：標註「544 個測試」，實際為 556 個
  • 修正：更新測試數量與分類明細

6. frequency-limits.yaml 群組註釋修正（Medium）

  • 問題：8 個群組的註釋數字仍為舊值（未反映 daily_limit: 0 的暫停狀態）
  • 修正：全部 8 個群組註釋更新為實際 daily_limit 總和

7. on_stop_alert.py kept 變數防禦性修正（Low）

  • 問題：_rotate_logs 中 kept 變數在 try 區塊內宣告，若 readlines 失敗會導致 NameError
  • 修正：kept = [] 提前到 try 之外宣告

技術細節

並行安全模式

本專案的團隊模式（run-agent-team.ps1、run-todoist-agent-team.ps1）會同時啟動多個 claude -p 進程。共享狀態檔案的讀寫必須使用：

  1. file_lock：互斥鎖保護 read-modify-write 序列
  2. atomic_write_json：write-to-temp + os.replace() 防半寫入損壞

已保護的狀態檔案

  • state/token-usage.json（post_tool_logger.py）— 已有 file_lock
  • state/loop-state-*.json（post_tool_logger.py）— 已有 file_lock
  • context/behavior-patterns.json（behavior_tracker.py）— 本次新增 file_lock + atomic_write
  • state/gmail-oauth-state.json（on_stop_alert.py）— 本次新增 atomic_write
  • logs/structured/session-summary.jsonl（on_stop_alert.py）— 已有 atomic_write_lines

驗證結果

  • 556/556 測試全部通過
  • 所有修改含 ImportError 退回機制，確保獨立可用
  • YAML 配置驗證通過（19 個任務解析正確）
