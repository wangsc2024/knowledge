Gemini 3.1 多模態記憶整合：Embedding 2、Deep Research Max 與長任務 Agent 記憶架構

研究日期：2026-04-22

---

執行摘要

本研究揭示 2026 年 AI 長任務代理記憶架構的三大匯流：Google 的 Gemini Embedding 2 統一多模態向量空間（2026-03）成為 Deep Research Max（2026-04）的記憶基座，透過 MCP（Model Context Protocol）整合專業資料流與工具定義，將研究代理從 web searcher 升級為可導航知識庫的自主系統。Anthropic 的 context editing + memory tool 雙層架構實證 39% 複雜任務效能提升，透過 client-side file-based memory 與 server-side context editing 實現 just-in-time retrieval，保持 active context 專注於當前任務。OpenAI 的 Agents SDK sandbox harness（2026-04）引入 模型原生 harness 與 憑證-執行環境分離，為長任務 agent 建立安全邊界。三大廠商路線在「統一向量空間作為記憶層」達成共識（3 獨立來源佐證），但在持久化策略（client-side vs. embedding space vs. workspace manifest）與安全模式（ZDR vs. sandbox vs. MCP）呈現設計分歧。本研究對照 daily-digest-prompt 的 continuity/KB/RAG 流程，萃取可落地的多模態記憶模式，並量化效能基線：latency -70%、複雜任務 +39%、DeepSearchQA 93.3%。

---

核心發現

1. 統一多模態向量空間：記憶基座的共識 ✅

Gemini Embedding 2 是業界首個原生多模態 embedding 模型，將 Text（8192 tokens）、Image（6 張/請求）、Video（120 秒）、Audio、PDF 映射至單一 3072 維向量空間。IBM 技術文章將此機制歸類為「Multimodal RAG 最先進路徑」：文字查詢可直接檢索相關多模態資料。Nature Communications 論文的 Reminisce 系統實證此架構在資源受限行動裝置上的可行性。

關鍵機制：Matryoshka Representation Learning（MRL）讓 Gemini Embedding 2 動態縮放維度（預設 3072，可降維平衡效能與儲存成本），為部分客戶降低 70% latency。

2. Just-in-time Context Retrieval：長任務記憶核心模式 ✅

Anthropic Memory tool 官方文件將此模式定義為「按需提取記憶而非前置載入全部資訊，保持 active context 專注於當前相關內容」。Context editing 文件補充理論基礎：「Context 是有限資源，無關內容會降低模型專注力」。Anthropic 實測 memory tool + context editing 組合達 39% 效能提升（複雜多步驟任務）。

3. 三大廠商記憶持久化策略分歧

  • Anthropic 路線：Client-side file-based（開發者自控儲存，/memories 目錄），強調隱私與 ZDR
  • Google 路線：Unified embedding space（隱式持久化），強調無縫整合
  • OpenAI 路線：Workspace manifest（檔案掛載 + 雲端儲存整合），強調靈活性

實務應用建議

  1. Just-in-time Retrieval 模式：將 daily-digest-prompt 的 continuity 改為按需載入，Phase 0 只載最近 1 筆
  2. Coarse-to-Fine 兩階段 KB 檢索：先 semantic topK=20（低成本）→ retrieve topK=5（高精度）
  3. MCP-like Tool Registry：統一外部資料源定義（todoist/pingtung-news/KB）
  4. Context Editing 機制：Phase 3 assemble 前清除過時 results（>10 → 保留 5）
  5. 多模態 KB 擴展：整合 Gemini Embedding 2，支援螢幕截圖/podcast 音訊檢索

量化效能基線

  • Gemini Embedding 2：latency -70%
  • Anthropic memory tool + context editing：複雜任務 +39%
  • Deep Research Max：DeepSearchQA 93.3%（原 66.1%）、HLE 54.6%（原 46.4%）
