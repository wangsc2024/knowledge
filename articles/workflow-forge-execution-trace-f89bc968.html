Execution Trace Schema 統一

缺口識別

類型：output_schema

優先級：P1

總分：95/100

問題描述：

trace_id、span_id、parent_span_id、phase、tool_or_skill、status、duration_ms、artifacts 等欄位分散在多個輸出檔案（logs/structured/.jsonl、results/.json、state/run-fsm.json）中，缺乏統一 schema，導致追蹤與回溯成本高。

來源：improvement-backlog.json Rank 10 (P1)

影響範圍：

所有 Agent 執行追蹤（daily-digest、todoist-agent-team、system-audit-team、gmail-agent）

---

產出物

檔案路徑：config/schemas/execution-trace-schema.json

Schema 版本：v1.0.0

核心欄位

| 欄位 | 類型 | 必填 | 說明 |

|------|------|------|------|

| trace_id | string | ✅ | 追蹤 ID，串接同一次執行的所有 spans |

| span_id | string | ✅ | Span ID，唯一識別當前操作 |

| parent_span_id | string/null | ❌ | 父 Span ID，建立因果關聯 |

| task_key | string | ✅ | 任務鍵或 agent 名稱 |

| phase | enum | ✅ | 執行階段（init/phase0/phase1/phase2/phase3/finalize/cleanup） |

| tool_or_skill | string | ❌ | 使用的工具或 Skill 名稱 |

| status | enum | ✅ | 執行狀態（started/running/success/failed/timeout/skipped） |

| duration_ms | number | ❌ | 執行耗時（毫秒） |

| artifacts | array | ❌ | 產出的 artifact 列表 |

| cause_chain | object | ❌ | 因果鏈（task_selection/cache_status/retry_attempts/failure_point/final_artifact） |

| timestamp | datetime | ✅ | 記錄時間戳（ISO 8601） |

設計原則

  1. 借鑑 OpenTelemetry：trace_id、span_id、parent_span_id、attributes、events 欄位設計參考 OpenTelemetry Traces 規範
  2. 本專案特有設計：cause_chain 欄位用於串接任務選擇至最終產出的完整因果鏈
  3. 向後相容：允許 additionalProperties: true，支援未來擴充
  4. 必填最小化：僅 6 個核心欄位為 required（trace_id、span_id、task_key、phase、status、timestamp）
