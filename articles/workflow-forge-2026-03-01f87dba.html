workflow-forge 2026-03-18：自動任務結果格式統一 Schema（v1.1.0）

workflow-forge流程標準化JSON Schema系統一致性ADR-003

workflow-forge 執行報告

日期: 2026-03-18

缺口: results/*.json 無統一 Schema 定義

產物: config/schemas/results-auto-task-schema.json (v1.1.0)

影響範圍: 29 個自動任務（todoist-auto-*.json）

---

1. 缺口分析

目前 29 個自動任務 prompt 各自用文字說明結果 JSON 格式，導致：

  • 欄位名稱不一致：有的用 agent，有的用 task_type/task_key
  • status 語義不一致：有的禁止 partial，有的允許
  • 必填欄位不一致：done_cert、artifact、stage 等欄位無統一規範

證據：

  • todoist-auto-workflow_forge.md 要求 agent + task_key
  • todoist-auto-ai_deep_research.md 只有 agent，無 task_key
  • todoist-auto-system_insight.md 使用 task_type + task_key
  • 現有結果檔案 results/todoist-auto-kb_insight_evaluation.json 缺少必填欄位 task_key

影響：

  • system-insight.json 顯示 avg_io_per_call 超標 398%（24875 chars），部分原因是 LLM 需在 prompt 內自行理解格式說明，導致 I/O 膨脹
  • Phase 3 assemble 解析結果檔案時容易因格式漂移導致錯誤

---

2. 產出 Schema 規格

Schema 版本: 1.1.0（基於 1.0.0 擴充）

必填欄位:

  • task_key: 任務鍵（對應 frequency-limits.yaml）
  • status: 執行狀態（success / partial / failed / format_failed）
  • agent 或 task_type：至少一個（agent 推薦）

新增欄位（v1.1.0）:

  • done_cert: 完成證明（Agent Pool 使用，ADR-004 P5-A）
  • stage: 階段編號（多階段任務，如 ai_deep_research 的 1-4 階段）
  • topic: 研究主題名稱
  • task_id: Todoist 任務 ID
  • duration_seconds: 執行耗時（與 execution_time_seconds 相容）

條件必填:

  • status=failed 或 format_failed 時，error 欄位必填

Pattern 約束:

  • agent: ^todoist-auto-[a-z_]+$（底線格式）
  • task_key: ^[a-z_]+$（底線格式）

---

3. 驗證結果

Schema 格式驗證: ✅ 通過（可解析，required 欄位定義清晰）

現有檔案抽樣驗證: ❌ 發現問題

  • results/todoist-auto-kb_insight_evaluation.json 缺少必填欄位 task_key
  • prompts/team/todoist-auto-ai_deep_research.md 的結果 JSON 示例無 task_key 欄位

結論: Schema 本身正確，但現有 prompt 格式不一致，需逐步更新。
