What Legal Agents Inherit from Coding Agents: Lessons from Legora

youtuberesearchClaudelegal AIauto

摘要

Legora 工程師 Jacob 分享從 AI 編碼 Agent 經驗遷移到法律 Agent 的關鍵教訓。Legora 是協作式法律 AI 平台，已有 1000+ 客戶含全球最大法律事務所，估值超 50 億美元，12 個月內從 1 至 1 億 ARR。核心洞察：法律工作與軟體工程有結構性相似——都需要理解複雜文件、跨多源推理、產出結構化輸出。他們採用 Claude Managed Agents 重建架構，從「多步驟 pipeline」轉向「Agent 自主決策」模式。關鍵學習：(1) 給 Agent 專家級工具而非通用工具 (2) 使用 Memory 讓 Agent 學習法律事務所的特定慣例 (3) Human-in-the-loop 在法律領域是必要的，不是可選的 (4) Dreaming 幫助優化跨 session 的法律推理品質。

關鍵發現

  • 法律 AI 與 coding AI 的結構性相似：複雜文件理解 + 多源推理 + 結構化輸出
  • 從 pipeline → agent 轉型的核心挑戰：可預測性 vs 靈活性的權衡
  • Memory 在法律領域的價值：學習事務所特定慣例與風格
  • Harvey benchmark：Dreaming 使完成率提升 6 倍

適用情境

Agent 架構設計、領域遷移策略、法律 AI 應用
