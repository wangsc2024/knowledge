AI 驅動的城市空氣品質預測與環境感測網路技術

領域概述

空氣品質監測是智慧城市最具公共衛生影響力的應用領域之一。傳統空品監測仰賴昂貴的國家級測站（每站造價數百萬美元），覆蓋密度低、維護成本高。隨著低成本 IoT 感測器與 AI 預測模型的成熟，城市得以建構高密度的環境感測網路，結合即時監測與 AI 預報，實現精細化空品管理、公共衛生早期預警及智慧決策支援。

AI 技術應用方式

1. 預測模型架構

空品預測已從單一模型演進至多模態融合架構，主要技術路線包括：

傳統機器學習（仍為基線首選）：

  • Random Forest（RF）：特徵重要性可解釋，R² 可達 0.83-0.87（O3 預測）
  • Gradient Boosting / XGBoost：台灣環保署 77 站 PM2.5 預測的首選方案，結合 580 個氣象站逐時數據
  • SVR（Support Vector Regression）：處理非線性關係，適用小樣本場景
  • KNN / AdaBoost：作為 ensemble 成員補強

深度學習（時空特徵捕捉）：

  • LSTM（Long Short-Term Memory）：捕捉污染物濃度的長期時序依賴，是空品預測最廣泛使用的 DL 模型
  • CNN（Convolutional Neural Network）：提取空間分佈特徵，結合衛星影像與地面測站數據
  • ConvLSTM + Attention：整合空間（Conv）與時間（LSTM）特徵，Attention 機制聚焦關鍵時間步驟
  • Transformer 架構：2024-2025 年新興，自注意力機制處理長距離時空依賴

混合模型（Hybrid，最佳泛化性能）：

  • RF + Prophet 混合：Prophet 生成時間特徵（週期性、趨勢）餵入 RF。Graz 奧地利案例中，NO 預測 R² 提升 26%，NO2 提升 18%
  • Physics-Informed Neural Network（PINN）：結合大氣擴散物理模型與數據驅動學習
  • Ensemble Stacking：多模型預測結果作為元特徵，再由元學習器組合

多模態深度學習（前沿方向）：

  • Vision-AQ：結合城市環境照片（ResNet50 CNN 特徵提取）與污染物數據，分類 AQI 等級
  • 交通 + 空品多模態：整合 CCTV 交通密度與空品感測器即時數據

3. 關鍵技術挑戰

  • 低成本感測器校準：微型感測器精度不如國家級測站，需 ML 校準演算法（如 spatiotemporal calibration）修正偏差
  • 缺值處理：感測器故障或維護時的數據填補，AI 可預測「缺少感測器的區域」的空品
  • 時空不均勻性：感測器密度不一、地形複雜、氣象變化快，需要圖神經網路（GNN）等空間建模能力
  • 即時性要求：Edge AI 在感測器端直接推論，減少雲端延遲（Edge Computing + TinyML）

國際案例分析

秘魯利馬：低成本感測器 + AI 預報系統

利馬市（Frontiers in Sustainable Cities, 2022）建構了結合低成本感測器與 AI 的空品監測預報系統：

  • 部署低成本 PM2.5 / PM10 感測器網路，覆蓋傳統測站無法觸及的社區
  • 使用 ML 模型進行 24-48 小時空品預報
  • 透過 Web Dashboard 與手機即時提供市民空品資訊
  • 驗證低成本方案在發展中國家智慧城市的可行性

奧地利 Graz：Hybrid RF-Prophet 長期預測

（Aerosol and Air Quality Research, 2024）：

  • 使用 5 個監測站 2014-2020 年逐時數據（PM10、NO、NO2、O3）
  • RF + Prophet-generated features 混合模型，整合 ERA5 再分析氣象數據
  • 全年測試證明「模型長期穩定」，夜間預測一致優於日間
  • 特徵重要性分析：Prophet 時間預測和風速是最關鍵預測因子

全球趨勢：Vision-AQ 多模態

（MDPI Mathematics, 2025）：

  • 創新性地將城市環境照片作為輸入特徵，使用 ResNet50 預訓練 CNN
  • 結合影像特徵與污染物數值數據，分類 AQI 等級
  • 開創「可見」空品評估的新範式，適用於缺乏感測器的區域

台灣現況與機會

民生公共物聯網（Civil IoT Taiwan）

2017 年啟動的跨部會國家級計畫，四大領域：空品、地震、防災、水資源。

空品感測基礎設施：

  • 環保署 77 座標準監測站 + 580 座氣象局氣象站，提供逐時數據
  • 微型空品感測器：工研院（ITRI）研發、維新汽車電子製造、綠點環境科技部署的端到端台灣供應鏈
  • 數據品質滿意度達 97%，證明國產感測器的可靠性
  • 開放 API 供各界介接，促進加值應用

AI 預測應用：

  • Gradient Boosting 模型：利用 77 站空品 + 580 站氣象逐時數據，預測 PM2.5（Scientific Reports, 2020）
  • 區域預報：環保署空氣品質監測網提供 72 小時區域預報與潛勢分析
  • 訊舟科技（Edimax）AirBox：全球知名的社群空品感測器，PM2.5 測試能力接近美國聯邦參考設備

屏東縣相關機會：

  • 屏東縣位於南台灣空品區，受境外傳輸（秋冬東北季風挾帶中國沿岸污染物）與在地排放（農廢燃燒、工業）雙重影響
  • 農業大縣特有需求：農廢燃燒監測、畜牧業氨排放監測、農藥噴灑空品影響
  • 智慧農業整合：空品預測 + 作物生長模型 + 精準農業，提升農產品質與農民健康
  • 觀光區空品管理：墾丁國家公園等景區空品即時資訊服務

五大空品區劃分

台灣環保署依地理與氣候特性分為：北部（台北、新北、基隆）、竹苗（桃園、新竹、苗栗）、中部（台中、彰化、南投）、雲嘉南（雲林、嘉義、台南）、高屏（高雄、屏東）。高屏空品區因地形與氣候因素，PM2.5 年均濃度通常高於其他區域。

技術挑戰與倫理考量

技術挑戰

  1. 低成本感測器精度瓶頸：微型感測器受溫濕度影響大，需定期校準與 ML 修正
  2. 數據時空不連續：感測器故障、通訊中斷導致資料缺漏，影響模型訓練品質
  3. 模型可解釋性：深度學習模型的「黑箱」特性，難以說服政策制定者採信預測結果
  4. Edge vs Cloud 取捨：即時告警需要 Edge Computing，但複雜模型推論仍依賴雲端
  5. 跨域數據整合：交通、氣象、衛星、地面感測器等多源數據的標準化與融合

倫理考量

  1. 環境正義：感測器部署密度往往與社經地位相關，弱勢社區可能成為「監測盲區」
  2. 數據隱私：結合交通 CCTV 影像的多模態空品預測涉及個人行蹤追蹤
  3. 預測責任：AI 預報失準導致公眾未做防護（或過度反應）的責任歸屬
  4. 數據主權：跨國感測平台（如 IQAir、AirVisual）的數據存取與使用規範

實施建議（從概念驗證到規模化）

第一階段：概念驗證（6 個月）

  • 選定一個區域（如屏東市區 3-5 平方公里），部署 20-30 個低成本 PM2.5/PM10 感測器
  • 建立 LoRaWAN/NB-IoT 通訊架構，數據回傳至雲端
  • 訓練 RF / XGBoost 基線模型，整合氣象站數據
  • 建立 Web Dashboard 即時呈現空品地圖

第二階段：模型優化（6 個月）

  • 引入 LSTM / ConvLSTM 深度學習模型，提升時序預測精度
  • 開發感測器自動校準演算法，比對鄰近國家級測站
  • 增加衛星遙測（如 Sentinel-5P NO2、MODIS AOD）作為輔助特徵
  • 開發 72 小時預報 API

第三階段：規模化部署（1 年）

  • 擴展至全縣範圍，感測器數量 100+
  • 開發手機 APP 推播空品預警通知
  • 整合農業管理系統（農廢燃燒時段管制、畜牧業排放監測）
  • 建立公民科學計畫，鼓勵居民貢獻感測數據

第四階段：智慧決策（持續）

  • AI 輔助政策模擬：評估不同減排方案的空品改善效果
  • 數位孿生整合：空品圖層疊加交通、人口、地形模型
  • 跨縣市聯防：與高雄市空品數據共享，建立區域性預警機制
