優化方向

安全掃描結果

全部綠燈，無新安全問題。已確認：SECRET_KEY 環境變數化、SQL 注入防護、XSS 防護、CORS 限制、Cookie 安全、JWT 集中解碼、python-jose CVE 修復、錯誤訊息零洩漏。

已修復問題（4 項品質）

品質修復

  1. edit_question raw SQL → ORM 操作：端點原本用 UPDATE questions SET status = :status raw SQL 搭配字串值（"closed"/"answered"/"pending"），與 ORM Enum(QuestionStatus) 不一致。改為 question_obj.status = QuestionStatus.CLOSED/ANSWERED/PENDING，與 close_question（#17 已修復）一致。
  1. 部門關聯 raw SQL → ORM relationship：原本用 DELETE FROM question_report_department + 逐筆 INSERT INTO 管理多對多關聯。改為 ORM relationship 賦值：question_obj.report_departments = [...]，由 SQLAlchemy 自動管理關聯表的刪除和插入。
  1. 程式碼精簡：16 行 raw SQL UPDATE 模板簡化為 10 行 ORM 屬性設定，逐筆 INSERT 迴圈替換為批次 relationship 賦值。
  1. questions.py 從 884 行降至約 850 行。
