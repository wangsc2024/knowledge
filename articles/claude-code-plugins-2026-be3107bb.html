Claude Code Plugins 插件系統完整指南（2026）

主題概述

Claude Code Plugins 是 Anthropic 於 2026 年 1 月 30 日正式發布的擴展系統，讓開發者可以自訂 slash commands、specialized agents、hooks 和 MCP servers，並透過 marketplace 分享給團隊或社群。插件系統將 Claude Code 從單一 CLI 工具轉型為可擴展的開發平台，目前已有超過 9,000 個插件，涵蓋程式碼審查、瀏覽器自動化、安全掃描、設計轉碼等領域。

---

插件目錄結構

標準的 Claude Code Plugin 目錄結構：


plugin-name/
├── .claude-plugin/
│   └── plugin.json          # 插件元資料（必要）
├── commands/                 # Slash commands（可選）
├── agents/                   # Specialized agents（可選）
├── skills/                   # Agent Skills（可選）
├── hooks/                    # Event handlers（可選）
├── .mcp.json                # MCP server 配置（可選）
├── .lsp.json                # LSP server 配置（可選）
└── README.md                # 插件文檔

重要提醒：commands/、agents/、skills/、hooks/ 目錄必須放在插件根目錄，不要放進 .claude-plugin/ 內。

---

plugin.json 格式

插件清單檔（manifest）定義插件身份：


{
  "name": "my-plugin",
  "description": "插件描述",
  "version": "1.0.0",
  "author": {
    "name": "Your Name"
  },
  "homepage": "https://github.com/...",
  "repository": "https://github.com/...",
  "license": "MIT"
}

| 欄位 | 用途 |

|------|------|

| name | 唯一識別碼，也是 skill 命名空間前綴（如 /my-plugin:hello） |

| description | 顯示在插件管理器中的描述 |

| version | 語意化版本號（Semantic Versioning） |

| author | 作者資訊（選填） |

核心元件詳解

1. Skills（技能）

技能是模型自動調用的可重用工作流。每個 Skill 是一個資料夾，內含 SKILL.md 檔案。

建立 Skill：


plugin-name/
└── skills/
    └── code-review/
        └── SKILL.md

SKILL.md 格式：


---
name: code-review
description: Reviews code for best practices and potential issues.
---

When reviewing code, check for:
1. Code organization and structure
2. Error handling
3. Security concerns

自動觸發機制：Claude Code 會比對使用者請求與已註冊的 skill descriptions，符合條件時自動調用，無需使用者明確指定。

2. Commands（命令）

使用者透過 /command-name 手動觸發的工作流。支援 $ARGUMENTS 佔位符接收使用者輸入。

範例：


---
description: Greet the user with a personalized message
---

Greet the user named "$ARGUMENTS" warmly.

3. Agents（代理）

專門化的 AI agents，具備特定能力，可並行運行多個 agents 處理複雜任務。

4. Hooks（鉤子）

事件處理器，攔截特定交互：

| Hook 類型 | 觸發時機 |

|-----------|----------|

| SessionStart | Session 開始時 |

| PreToolUse | 工具使用前（可攔截阻止） |

| PostToolUse | 工具使用後 |

| Stop | Session 結束時 |

hooks.json 範例：


{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [{ "type": "command", "command": "npm run lint:fix" }]
      }
    ]
  }
}

5. MCP（Model Context Protocol）

連接外部服務和 API，讓 Claude 存取原本無法取得的資料。配置於 .mcp.json。

6. LSP（Language Server Protocol）

提供即時程式碼智慧功能：跳轉定義、查找引用、類型錯誤提示。


{
  "go": {
    "command": "gopls",
    "args": ["serve"],
    "extensionToLanguage": {
      ".go": "go"
    }
  }
}

安裝與測試

開發測試

使用 --plugin-dir 載入本地插件（不需安裝）：


claude --plugin-dir ./my-plugin

正式安裝

從 marketplace 安裝：


claude plugin install <plugin-name>

或新增 marketplace 來源：


claude plugin marketplace add owner/repo

Marketplace 生態系

  1. Official Anthropic Marketplace：Anthropic 官方維護的插件目錄（plugins.claude.ai）
  2. ClaudePluginHub：社群主導的大型 marketplace
  3. Claude-Plugins.dev：社群登記表與 CLI 工具
  4. 自架 Git-based Marketplace：任何 Git hosting（GitHub、GitLab、Bitbucket）都可作為 marketplace 來源

---

Standalone vs Plugin 比較

| 面向 | Standalone (.claude/) | Plugin |

|------|-------------------------|--------|

| 技能名稱 | /hello | /plugin-name:hello |

| 適用情境 | 個人工作流、專案特定 | 團隊分享、跨專案重用 |

| 更新機制 | 手動複製 | 版本控制 + 自動更新 |

建議流程：先在 .claude/ 快速迭代，穩定後再轉換為 plugin 分享。
