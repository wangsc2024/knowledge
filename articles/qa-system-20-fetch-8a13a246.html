QA System 安全優化報告 #20

日期：2026-02-28 | 品質分數：5/5 | 第二十次掃描

優化方向

_fetch_question_with_departments + list_questions raw SQL SELECT → ORM query + selectinload（消除 N+1 查詢）

安全掃描結果

全部綠燈，無新安全問題。已確認：SECRET_KEY 環境變數化、SQL 注入防護、XSS 防護、CORS 限制、Cookie 安全、JWT 集中解碼、python-jose CVE 修復、錯誤訊息零洩漏。

已修復問題（4 項品質）

品質修復

  1. _fetch_question_with_departments raw SQL → ORM：原本用 3 次 raw SQL（SELECT * FROM questions + 2 次 JOIN departments），改為 ORM query + selectinload 預載入，1 次查詢搞定。
  1. list_questions N+1 查詢消除：原本先 SELECT 所有 questions，再逐筆循環取部門（N+1 問題），改為 ORM + selectinload 一次載入所有部門關聯。
  1. 新增 _question_to_dict() helper：統一 ORM Question → dict 轉換邏輯，保持 4 處呼叫者和所有 Jinja2 模板介面完全不變。
  1. 移除 _parse_date_fields dead code：ORM 物件的日期欄位已是 datetime 型別，不需要字串解析函式。

數據

  • questions.py：770 → 723 行（-47 行，-6%）
  • 累計：1064 → 723 行（-32%）
  • 103/103 測試全通過
  • questions.py CRUD 全部完成 ORM 遷移（close→edit→create→fetch/list）
