優化方向

create_question（POST /）raw SQL INSERT → ORM 操作 + 脆弱 regex 清理邏輯移除

安全掃描結果

全部綠燈，無新安全問題。已確認：SECRET_KEY 環境變數化、SQL 注入防護、XSS 防護、CORS 限制、Cookie 安全、JWT 集中解碼、python-jose CVE 修復、錯誤訊息零洩漏。

已修復問題（3 項品質）

品質修復

  1. create_question raw SQL INSERT → ORM 操作：端點原本用 INSERT INTO questions ... RETURNING id raw SQL 搭配字串 "pending"，改為 Question() + db.add() + QuestionStatus.PENDING，消除 raw SQL/Enum 不一致風險。
  1. 部門關聯逐筆 raw SQL INSERT → ORM relationship：原本用 for 迴圈逐筆 INSERT INTO question_report_department + 個別 try/except，改為一次 query 取出所有部門 + new_question.report_departments = [...] / answer_departments = [...]，由 SQLAlchemy 自動管理關聯表。
  1. 脆弱 regex 清理邏輯移除：原本錯誤處理中用 re.search(r'question_id, department_id...) 解析異常訊息取 ID 做手動 DELETE 清理——極度脆弱且依賴錯誤訊息格式。ORM 模式下 db.rollback() 自動回復所有未提交變更，整段 regex 清理邏輯已移除。

ORM 遷移里程碑
