Unsloth GRPO 強化學習實戰 — 從零訓練推理模型的完整指南

概述

GRPO（Group Relative Policy Optimization）是由 DeepSeek 開發的強化學習演算法，用於訓練其 R1 系列推理模型。相比傳統 PPO，GRPO 移除了 Value Model 和 Reward Model，以自訂獎勵函數取代，大幅降低記憶體需求。Unsloth 將 GRPO 訓練帶入消費級 GPU，最低僅需 5GB VRAM 即可在本地訓練推理模型——這意味著你可以用一張 RTX 3060 複製 DeepSeek R1 的核心訓練方法論。本指南涵蓋 GRPO 原理、Unsloth 支援的 RL 方法、完整程式碼、獎勵函數設計、長上下文最佳化與效能基準。

---

一、GRPO 核心原理

1.1 與 PPO 的關鍵差異

| 特徵 | PPO | GRPO |

|------|-----|------|

| Value Model | 需要（額外 VRAM） | 移除 |

| Reward Model | 需要獨立 LLM | 以自訂函數取代 |

| 記憶體效率 | 低（需 3 個模型） | 高（僅需 1 個模型 + 獎勵函數） |

| 訓練穩定性 | 依賴 Value 估計 | 依賴群組統計量 |

| 適用場景 | 通用 RLHF | 可驗證推理任務（數學、程式、邏輯） |

1.2 演算法流程

  1. 產生群組回應：對每個問題，模型產生 N 個候選回應（如 8 個）
  2. 獎勵評分：每個回應透過獎勵函數取得分數
  3. 計算群組平均：以群組內所有回應的平均分數作為基線
  4. 相對優勢計算：高於平均的回應獲得正向強化，低於平均的被抑制
  5. 策略更新：模型權重朝「好回應」方向調整

核心洞察：GRPO 不需要預先收集 chain-of-thought 資料，它能讓模型自主發展推理能力——DeepSeek 稱此為「aha moment」。

二、Unsloth 支援的 RL 方法

Unsloth 不僅支援 GRPO，還整合了多種最新 RL 方法：

| 方法 | 特色 | 適用場景 |

|------|------|----------|

| GRPO | DeepSeek 原版，群組相對優化 | 通用推理訓練 |

| GSPO | 可設定 epsilon/delta 的增強版 | 需精細控制探索的場景 |

| DrGRPO | 替代損失函數公式 | 更穩定的梯度更新 |

| DAPO | 額外優化策略 | 特定分佈偏好調整 |

| RFT | OpenAI 風格強化微調 | 簡單的 SFT + RL 混合 |

配置切換非常簡單：


from trl import GRPOConfig

training_args = GRPOConfig(
    loss_type='grpo',        # 選項: 'grpo', 'gspo', 'dr_grpo'
    epsilon=0.2,             # GRPO 預設
    epsilon_high=0.28,       # GSPO 單側參數
    delta=1.5,               # GSPO 雙側參數
    mask_truncated_completions=True,
)

3.4 定義獎勵函數（關鍵）


import re

def correctness_reward_func(prompts, completions, answer, **kwargs):
    """檢查最終答案是否正確"""
    responses = [c[0]["content"] for c in completions]
    rewards = []
    for response, correct in zip(responses, answer):
        # 從 <answer>...</answer> 標籤中提取答案
        match = re.search(r'<answer>(.*?)</answer>', response)
        if match:
            extracted = match.group(1).strip()
            rewards.append(3.0 if extracted == correct else -3.0)
        else:
            rewards.append(-1.0)  # 格式不符也扣分
    return rewards

def int_reward_func(completions, **kwargs):
    """鼓勵輸出整數答案"""
    responses = [c[0]["content"] for c in completions]
    rewards = []
    for response in responses:
        match = re.search(r'<answer>(.*?)</answer>', response)
        if match and match.group(1).strip().isdigit():
            rewards.append(1.0)
        else:
            rewards.append(0.0)
    return rewards

def strict_format_reward_func(completions, **kwargs):
    """嚴格要求 <reasoning>...</reasoning><answer>...</answer> 格式"""
    pattern = r'^<reasoning>\n.*?\n</reasoning>\n<answer>\n.*?\n</answer>$'
    responses = [c[0]["content"] for c in completions]
    return [1.0 if re.match(pattern, r, re.DOTALL) else 0.0 for r in responses]

def xmlcount_reward_func(completions, **kwargs):
    """根據 XML 標籤數量給分（漸進式獎勵）"""
    responses = [c[0]["content"] for c in completions]
    rewards = []
    for response in responses:
        score = 0.0
        score += 0.25 if '<reasoning>' in response else 0.0
        score += 0.25 if '</reasoning>' in response else 0.0
        score += 0.25 if '<answer>' in response else 0.0
        score += 0.25 if '</answer>' in response else 0.0
        rewards.append(score)
    return rewards

3.5 配置 GRPO 訓練


from trl import GRPOConfig, GRPOTrainer

training_args = GRPOConfig(
    output_dir="outputs",
    learning_rate=5e-6,
    num_train_epochs=1,
    per_device_train_batch_size=1,
    gradient_accumulation_steps=4,
    num_generations=8,           # 每題產生 8 個候選回應
    max_completion_length=512,   # 最大回應長度
    max_prompt_length=256,       # 最大提示長度
    logging_steps=1,
    save_steps=100,
    use_vllm=True,               # 使用 vLLM 加速推論
    loss_type='grpo',
    mask_truncated_completions=True,
)

3.6 準備資料集與啟動訓練


from datasets import load_dataset

# GSM8K 數學推理資料集（最熱門的 R1 風格訓練選擇）
dataset = load_dataset('openai/gsm8k', 'main', split='train')

# 訓練
trainer = GRPOTrainer(
    model=model,
    processing_class=tokenizer,
    reward_funcs=[
        xmlcount_reward_func,       # 格式獎勵
        strict_format_reward_func,  # 嚴格格式獎勵
        int_reward_func,            # 整數獎勵
        correctness_reward_func,    # 正確性獎勵（最重要）
    ],
    args=training_args,
    train_dataset=dataset,
)

trainer.train()

四、獎勵函數設計最佳實踐

4.1 設計原則

  • 可驗證性：獎勵函數必須能精確判斷對錯，不用 LLM 評分
  • 漸進式獎勵：使用多個小獎勵函數組合，而非單一大函數
  • 格式 + 正確性分離：格式獎勵確保可解析，正確性獎勵確保答案品質
  • 避免獎勵駭客：過於寬鬆的獎勵會讓模型學到捷徑

4.2 適用場景範例

| 場景 | 獎勵設計策略 |

|------|------------|

| 數學推理 | 正確答案 +3 / 錯誤 -3 / 格式不符 -1 |

| 程式碼生成 | 通過測試 +2 / 語法正確 +1 / 輸出格式 +0.5 |

| 郵件撰寫 | 含必要關鍵字 +1 / 篇幅適中 +1 / 含簽名 +1 |

| 邏輯推理 | 最終答案正確 +3 / 步驟合理 +1 / 含推理標籤 +1 |

4.3 進階：proximity-based 獎勵

Unsloth 提供接近度獎勵——預測 9 而正確答案為 10，比預測 3 得到更高獎勵。這對數值類任務特別有效。

五、長上下文 GRPO 訓練（2026 年 1 月新功能）

5.1 技術創新：Flattened Sequence Chunking

Unsloth 引入創新的批次演算法，實現 ~7 倍更長的上下文 RL 訓練：

| 方法 | Logit 記憶體（batch=4, ctx=8192, vocab=128K） |

|------|------|

| 傳統方法 | 3.3 GB |

| Fused Loss | 0.83 GB |

| Unsloth Flattened Chunking | ~0.2 GB |

核心技術：

  1. Flattened Sequence Chunking：不一次實體化整個 logit tensor，而是跨 batch + sequence 兩個維度分塊
  2. Hidden States Chunking：log-probability 計算時的隱藏狀態也分塊，batch=8 + ctx=64K 從 2GB 降至 0.24GB
  3. Activation Offloading：自訂 Unsloth_Offloaded_Log_Softmax 類，將前向傳播外的激活值卸載至 CPU

5.2 硬體與上下文對照表

| 硬體 | 模型 | 最大上下文長度 |

|------|------|---------------|

| NVIDIA B200 192GB | gpt-oss QLoRA | 380K |

| H100 80GB | Qwen3-8B GRPO + vLLM | 110K |

| 24GB VRAM | gpt-oss QLoRA | 20K |

| 24GB VRAM | Qwen3-VL-8B QLoRA | 32K |

5.3 配置參數


training_args = GRPOConfig(
    unsloth_grpo_mini_batch=3,              # Hidden States 分塊大小
    unsloth_logit_chunk_multiplier=2,       # Logit 分塊乘數
    # 預設: max(4, context_length // 4096)
)

六、VRAM 需求與最佳化對比

6.1 QLoRA 4-bit 簡易估算

經驗法則：模型參數量（十億）≈ 所需 VRAM（GB）

  • 1.5B 模型 → ~5GB VRAM
  • 8B 模型 → ~15GB VRAM
  • 14B 模型 → ~15GB VRAM

6.2 Unsloth vs 標準方案記憶體對比（Qwen3-8B 範例）

| 組件 | Unsloth | 標準 + FA2 |

|------|---------|------------|

| 訓練 | 42 GB | 414 GB |

| GRPO Loss | 9.8 GB | 78.3 GB |

| 推論 | 0 GB（共享） | 16 GB |

| KV Cache (20K ctx) | 2.5 GB | 2.5 GB |

| 合計 | 54.3 GB | 510.8 GB |

記憶體節省約 90%，關鍵技術：

  • Memory-efficient 線性核心（8 倍縮減）
  • 智慧梯度檢查點（節省 52GB）
  • GPU/CUDA 記憶體與推論引擎共享（節省 16GB）

七、訓練注意事項與常見問題

7.1 關鍵訓練指引

| 項目 | 建議值 |

|------|--------|

| 最低訓練步數 | 300 步以上才會看到獎勵上升 |

| 最佳資料量 | 500+ 行（最少 10 行可運作） |

| 訓練時長 | 12+ 小時為佳 |

| 模型大小 | ≥1.5B 參數才能穩定產生思考 token |

| 建議起點 | 指令微調模型（非 base model） |

7.2 常見問題排除

| 症狀 | 解法 |

|------|------|

| 模型不產生推理過程 | 增加訓練步數，確認 ≥1.5B 參數 |

| 獎勵不上升 | 檢查獎勵函數邏輯，確保可驗證 |

| OOM 記憶體不足 | 減少 context length 或 batch size |

| 獎勵駭客（reward hacking） | 重新設計獎勵函數，加入多維度評分 |

7.3 「耐心就是你需要的」原則

RL 的本質是透過取樣直到正向結果出現。只要正確答案的機率不為零，最終一定會遇到——RL 只是透過負向信號「剪枝」輸出分佈，加速這個過程。
