DeerFlow 2.0（bytedance/deer-flow）— ByteDance 開源 SuperAgent Harness

概述

DeerFlow 2.0 是 ByteDance 開源的 SuperAgent 運行時框架，基於 LangGraph + LangChain 構建，於 2026 年 2 月 28 日登上 GitHub Trending 榜首。截至 2026 年 3 月已累積超過 41,000 Stars。DeerFlow 2.0 是從 v1 完全重寫的版本，從深度研究工具轉型為通用型 Agent 運行時，核心設計理念是透過 Lead Agent 編排子 Agent、記憶系統與沙箱環境來處理從數分鐘到數小時的複雜多步驟任務。

GitHub: https://github.com/bytedance/deer-flow

授權: MIT

Stars: 41,300+（2026-03）

五大核心組件

1. Lead Agent 與任務分解

  • 角色：編排器（Orchestrator），接收複雜 prompt 後分解為結構化子任務
  • 機制：判斷哪些子任務可並行執行，為每個子 Agent 分配專屬 context、工具集與終止條件
  • 合成：子 Agent 完成後，Lead Agent 將結果合成為連貫輸出

2. 沙箱環境（Sandbox）

  • 三種模式：本地執行、Docker 容器（推薦）、Kubernetes Pod
  • 能力：持久化檔案系統、Bash 終端存取、程式碼建立/修改/執行
  • 隔離：每個任務在獨立容器中運行，防止跨任務干擾

3. Skills 系統

  • 格式：Markdown 為基礎的可擴展能力模組
  • 漸進式載入：「only when the task needs them」，減少 context 開銷
  • 內建 Skills：研究報告、簡報製作、網頁生成、圖片/影片生成
  • 自訂 Skills：支援 .skill 封裝格式（含 version、author、compatibility 元資料）
  • Claude Code 整合：claude-to-deerflow Skill 可從 Claude Code 終端互動

4. 長期記憶層（Memory Layer）

  • 追蹤：使用者偏好、寫作風格、專案上下文
  • 持久化：跨 session 保留，非揮發性
  • 更新機制：非同步 debounced queue，避免頻繁寫入
  • 企業級：TIAMAT 雲端後端支援大規模部署

5. 模型無關性（Model Agnosticism）

  • 支援模型：GPT-4/5、Claude 系列、Gemini、DeepSeek、Doubao、Ollama
  • 介面：任何 OpenAI 相容 API
  • 推薦模型：Doubao-Seed-2.0-Code、DeepSeek v3.2（強指令遵循能力）
  • 配置方式：YAML 配置 + .env 環境變數

架構特色分析

與其他框架的差異化定位

| 維度 | DeerFlow 2.0 | LangGraph/CrewAI/AutoGen |

|------|-------------|-------------------------|

| 定位 | 可直接運行的完整系統 | 基礎建構元件 |

| 預設值 | 內建執行模型、Skills、沙箱、記憶 | 需自行組裝 |

| 部署速度 | 快速（make docker-start） | 需要更多配置 |

| 擴展性 | 有約束的擴展 | 高度自由 |

技術棧

  • 後端：Python 3.12+
  • 前端：Node.js 22+
  • 基礎設施：Docker、Kubernetes、nginx
  • Agent 引擎：LangGraph（langgraph dev CLI server）

IM 通道支援

  • Telegram（Bot API, long-polling）
  • Slack（Socket Mode）
  • 飛書/Lark（WebSocket）

快速啟動


git clone https://github.com/bytedance/deer-flow.git
cd deer-flow
make config        # 生成配置文件
make docker-start  # Docker 啟動（推薦）
# 訪問 http://localhost:2026

實際應用場景

  1. 研究報告生成：自動搜尋、整理、生成帶引用的格式化報告
  2. 資料管線自動化：在隔離沙箱中執行腳本
  3. 簡報製作：從 prompt 生成含圖片的投影片
  4. 全端應用鷹架：從需求描述到專案結構
  5. 競品分析：多個子 Agent 並行蒐集不同競品資訊

---

對 Daily-Digest-Prompt 專案的借鏡價值

  1. Skills 漸進式載入：與本專案 Skill-First 策略相呼應，DeerFlow 的「按需載入」設計驗證了此模式的可行性
  2. Lead Agent 編排模式：類似本專案的 Team 並行模式，但更細粒度的子 Agent context 隔離值得參考
  3. 沙箱執行：Docker/K8s 隔離模式對長時間自動任務的穩定性有啟發
  4. 記憶層設計：debounced queue 非同步更新機制可借鏡於 digest-memory.json 的寫入優化
  5. IM 通道整合：Telegram/Slack/飛書支援與本專案的 LINE Bot 通知管線可比較
