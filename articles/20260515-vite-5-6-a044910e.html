[研究 20260515] Vite 5→6 官方遷移要點

vitemigrationfrontendresearchvite6

來源

  • https://v6.vite.dev/guide/migration （官方 Migration from v5）
  • 同日 WebSearch/Vite Blog 概要佐證

Environment API／Runtime API

  • Vite 6 內含實驗性 Environment API 等大型內部重構，但仍盡量避免破壞性變更；低階相依框架若遇到退步可回報 issue。
  • 先前實驗性 Vite Runtime API（5.1） 已由 Module Runner API 取代，需改用對應寫法。

一般設定調整（常見陷阱）

  • resolve.conditions 預設改為 ['module','browser','development|production']；ssr.resolve.conditions 預設改為 ['module','node','development|production']，且不再沿用 resolve.conditions。客製時需保留官方預設條件（可用 defaultClientConditions/defaultServerConditions 展開）。
  • json：json.stringify 預設改為 'auto'（大檔較易被 stringify）；json.stringify:true 時 namedExports 仍會被尊重，若想回舊行為設 namedExports:false。
  • 資產/HTML：支援更多 HTML 元素上的資源引用；可用 vite-ignore 停用處理。
  • postcss-load-config 升到 v6；TypeScript postcss 設定需 tsx/jiti（非 ts-node），YAML postcss 設定需 yaml 套件。
  • Sass 預設改 modern API；可暫設 legacy，但 Vite 7 將移除。
  • library 模式 CSS 檔名：預設改向 package.json 之 name 對齊；必要時用 build.lib.cssFileName 強制。

應用在知識庫靜態站

  • knowledge 網站（Vite）升級或大版本相依 bump 時，優先對照上文檢查 resolve/JSON/Sass/CommonJS／glob 細節，再跑一次 npm run build。
