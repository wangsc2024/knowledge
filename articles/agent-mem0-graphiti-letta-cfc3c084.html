Agent 長期記憶系統架構模式（mem0 / graphiti / Letta）

蒐集日期：2026-04-26 | 來源：GitHub Scout

概述

三套主流 Agent 長期記憶框架，各有不同的架構哲學：

  • mem0：混合存儲三層，多範圍隔離
  • graphiti：時態知識圖，fact 有有效期窗口
  • Letta（前身 MemGPT）：OS 式分頁記憶，Agent 主動管理

---

mem0（mem0ai/mem0，~53,500★）

架構哲學

Universal Memory Layer — 對 AI Agent 來說像「通用記憶層」，不同框架都能接入。

Memory Architecture

混合存儲三層：

  • Vector DB（Qdrant/Pinecone）：語義相似度查詢
  • Key-Value DB（Redis）：精確查找
  • Graph DB（Neo4j）：關係遍歷

多範圍隔離：user / session / agent，各自獨立命名空間。

衝突處理

Mem0g 變體加入衝突偵測：新 fact 寫入前先比對現有圖節點，有矛盾時先標記再寫入。

整合廣度

支援 21 個 AI 框架整合（LangChain、LlamaIndex、CrewAI 等）。

---

graphiti（getzep/graphiti，~20,000★）

架構哲學

Temporally-aware Knowledge Graph — 每個 fact 都有時間有效窗口，舊 fact 失效而非刪除。

Memory Architecture

時態知識圖（Temporal KG）：

  • 每個 fact 節點含 valid_from / valid_to 有效窗口
  • 過期 fact 標記失效，保留歷史可回溯
  • 支援時點查詢（「T 時刻有效的 facts」）

Storage Backend

Neo4j 圖存儲，增量更新不需重算整圖。

效能基準

LongMemEval 得分 63.8%（vs mem0 的 49.0%，差距 +14.8%）。

本專案啟發

→ 在 memory-policy.yaml continuity 層新增 finding_expires_days 欄位，讓 Agent 記憶的發現有時效性。

---

Letta（letta-ai/letta，前身 MemGPT，~13,000★）

架構哲學

OS-inspired Stateful Agent — 把 Agent 記憶比擬為作業系統分頁：RAM（in-context）vs Disk（external storage）。

Memory Architecture

三層分頁：

  • core memory：永遠在 context（如 RAM），含 persona + human 描述
  • archival memory：外部向量庫（如 Disk），容量無限
  • recall memory：對話歷史，可搜尋

Agent 主動管理

與 mem0/graphiti 不同，Letta 的記憶管理是 Agent 主動呼叫工具函式，而非被動注入：

  • memory_search() — 搜尋 archival
  • memory_append() — 新增到 core/archival

多 Agent 協作

支援 multi-agent 共享記憶，跨 Agent 傳遞知識。

---

三套框架比較

| 維度 | mem0 | graphiti | Letta |

|------|------|---------|-------|

| 存儲模型 | 混合三層 | 時態知識圖 | OS 式分頁 |

| 時效處理 | 衝突偵測標記 | valid_from/valid_to | 無內建 TTL |

| Agent 角色 | 被動注入 | 被動注入 | 主動呼叫工具 |

| 最強項 | 整合廣度（21框架） | 時態準確性（LongMemEval +15%） | 有狀態長對話 |

| Storage | Qdrant+Redis+Neo4j | Neo4j | 向量DB+in-context |
