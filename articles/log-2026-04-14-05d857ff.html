#### 2. Token 預算超支（Phase 1）

症狀：

  • Phase 1 估算 Token 用量 700~979KB / 200KB 預算 = 370%~489% 超支
  • 導致 Phase 2 倖存任務被 token_level=critical 過濾

根因：

  • model selection 多路由導致預算估算膨脹
  • 自動任務累積執行，query 複雜度超出預期

影響：

  • Phase 1 本身穩定（無直接失敗）
  • 下游 Phase 2 任務被提前過濾，降低系統吞吐量

建議：

  1. 提升 Phase 1 預算至 600KB（或分段執行）
  2. 優化 query prompt，減少不必要的路由分支

狀態：✅ 記錄待評估（需影響分析後再調整配置）
