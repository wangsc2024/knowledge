Hook 系統優化：YAML 快取共用 + JSONL 原子寫入

優化概述

針對 daily-digest-prompt 專案的 Hook 機器強制層進行兩項資安與可靠性優化。

優化項目 1：post_tool_logger YAML 快取共用

問題

post_tool_logger._load_benign_patterns_from_yaml() 獨立開檔讀取 hook-rules.yaml，繞過 hook_utils 模組的 _yaml_config_cache。在同一進程中若多個 hook 依序觸發，會造成重複 I/O。

解法

在 hook_utils.py 新增 load_yaml_section() 通用函數，支援載入任意型別（list/dict/str）的 YAML 頂層區段，共用既有的模組層級快取。post_tool_logger 改用此函數，附帶 ImportError fallback 確保向下相容。

效益

  • 消除每日 44 次 Agent 執行中的重複 YAML 開檔
  • 與 load_yaml_rules 共享快取，降低 I/O 開銷
  • 6 個新測試驗證快取共用行為

優化項目 2：on_stop_alert session-summary 原子寫入

問題

on_stop_alert._rotate_logs() 使用 open('w') 直接覆寫 session-summary.jsonl。在團隊並行模式（5 個 Agent 同時結束）下，多進程同時覆寫可能導致資料損壞或遺失。

解法

在 hook_utils.py 新增 atomic_write_lines() 函數，採用 write-to-temp + os.replace() 原子替換策略（POSIX 與 Windows NTFS 均為原子操作）。on_stop_alert._rotate_logs() 改用此函數。

效益

  • 消除並行場景的競態條件風險
  • 寫入失敗時不會留下半寫入的損壞檔案
  • 暫存檔失敗時自動清理
  • 6 個新測試覆蓋正常寫入、覆寫、目錄建立、空檔案、Unicode、暫存檔清理

變更檔案

  • hooks/hook_utils.py：+53 行（load_yaml_section + atomic_write_lines）
  • hooks/post_tool_logger.py：+33/-8 行（改用 load_yaml_section）
  • hooks/on_stop_alert.py：+28/-8 行（改用 atomic_write_lines）
  • tests/hooks/test_hook_utils.py：+136 行（12 個新測試）

測試結果

529 個測試全數通過（517 既有 + 12 新增），0 失敗。
