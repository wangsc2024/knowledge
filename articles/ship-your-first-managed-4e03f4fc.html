影片摘要

來源：https://www.youtube.com/watch?v=19HDQ9HppOA

頻道：Claude（Anthropic 官方）｜時長：約 37 分鐘｜發布：2026-05-26

核心架構：Claude Managed Agents

演進路徑：Messages API（2023）→ Agent SDK → Claude Managed Agents（CMA）。CMA 提供託管基礎設施，開發者專注任務配置與自訂工具。

三大資源：

  1. Agent：定義模型（Opus 4.7）、系統 prompt、MCP server、skills — 即 Agent 的「大腦」
  2. Environment：Agent 執行環境（Anthropic 雲 / 自帶容器）— 即 Agent 的「雙手」
  3. Session：綁定 Agent + Environment，以 Event 串流（非傳統 request/response）

關鍵設計：大腦與雙手解耦 → 安全性（沙箱隔離憑證）+ 效能（TTFT P95 降低 >90%）+ 可靠性（容器重啟不中斷 Agent loop）。

工作坊實作：SRE 事故響應 Agent

用 Streamlit 前端 + CMA 後端建構 Site Reliability Agent：

  • 定義本地工具（get_metrics / get_recent_deploys / get_diff）
  • 上傳日誌檔作為 context
  • Agent 自動分析指標、查看部署紀錄、讀取 diff → 定位 P99 延遲 10x 飆升根因（資料庫連接池耗盡）
  • Session 持久化：重整頁面不遺失對話，可刪除/恢復 session

進階功能預覽

  • Sub-agents：多 Agent 並行 + 各自 context window
  • Memory + Dreaming：Agent 自主審視記憶日誌，學習使用者偏好與修正（自改善 Agent）
  • Outcomes：定義 rubric → Agent 朝預期結果行動而非盲目執行工具
  • Vaults：加密憑證存儲，per-user per-session 安全隔離
  • Bring Your Own Compute：剛於 Code with Claude London 發布，可在自有容器內執行工具
  • MCP Tunnels：私有網路內運行 MCP server

關鍵觀點

  • Context Engineering 是開發者花最多時間的部分（檔案上傳、工具設計、prompt 調整）
  • 從原型到生產的速度：CMA 使用者比傳統方式快 10-15 倍
  • SRE Agent 可擴展為全自動修復流程（接入 Claude Code + PR 提交能力）
