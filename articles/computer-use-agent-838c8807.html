Computer-Use Agent 安全與評測機制

核心架構

Computer-use agent 由「視覺感知（截圖）→ 推理規劃（CoT）→ 動作執行（滑鼠/鍵盤）」三層迴圈構成。Anthropic Claude Computer Use 與 OpenAI CUA 皆遵循此模式。

效能現況（2026 Q1）

  • OSWorld 50-step 最佳：38.1%（OpenAI CUA），人類基準 72.4%
  • WebArena：58.1%（OpenAI CUA）
  • CUB（真實企業工作流）：最佳僅 10.4%
  • GAIA Level 3：61%（Writer Action Agent）

安全威脅 — 致命三角（Lethal Trifecta）

當系統同時具備：(1) 私有資料存取 (2) 不受信任輸入 (3) 外洩向量，prompt injection 攻擊幾乎不可避免。

真實案例：EchoLeak (CVE-2025-32711) 零點擊攻擊 M365 Copilot、GeminiJack 攻擊 Google Gemini、GitHub Copilot CVE-2025-53773 RCE。

七大關鍵洞見

  1. 致命三角是最實用的風險評估框架
  2. 間接攻擊已超越直接 prompt injection 成為主流
  3. Test-time scaling 是能力提升的關鍵槓桿
  4. 結構化輸出驗證是生產化的第一步
  5. Agent 需要獨立的非人類身分體系
  6. CUB benchmark 揭示真實企業場景能力差距（學術 30-38% → 企業 10%）
  7. 治理是自主能力的賦能者，非限制者

防護五層架構

模型層（injection 分類器）→ 基礎設施層（沙箱隔離）→ 存取控制層（最小權限）→ 運行時層（結構化驗證）→ 人工層（HITL 確認）

四大失敗模式

格式漂移、計畫偏離、歧義迴圈、靜默錯誤

對 daily-digest-prompt 的啟示

  • Hook 機器強制層正是結構化 guardrails 的實踐
  • results/ JSON middleware 驗證應強制啟用
  • 專案已具備致命三角的三個條件，需定期審查防護
