一、專案概述

MiroFish（666ghj/MiroFish）是一套開源的群體智慧預測引擎，定位為「簡潔通用的群體智能引擎，預測萬物」。專案由北京郵電大學學生郭杭江（Baifu）開發，於 2026 年 3 月 7 日登上 GitHub 全球 Trending 榜首，短短數日累積 18,000 Stars、1,900 Forks，截至 2026 年 3 月已超過 33,000+ Stars [GitHub, 2026]。

| 社交動作 | 23 種（關注、評論、轉發、按讚、搜尋等） |

核心理念

MiroFish 的核心理念是「在數位沙盒中預演未來，在無數次模擬後贏得決策」。與傳統統計模型不同，它不分析歷史資料，而是建構高保真的平行數位世界，填充數千個具有獨立人格、長期記憶與行為邏輯的 AI 代理，讓它們自由互動並觀察湧現結果 [emelia.io, 2026; DEV Community, 2026]。

二、五階段管線架構（核心技術）

Stage 1：圖譜建構（Graph Building）

用戶上傳原始資料（新聞、財報、政策草案、小說文本），系統使用 GraphRAG（針對結構化資料優化的 RAG 技術）提取實體與關係，建構知識圖譜作為模擬基礎 [GitHub README, 2026; DEV Community, 2026]。

Stage 2：世界建構（World Building）

環境配置代理讀取知識圖譜，自動生成模擬參數，包括反映源材料人口統計與文化背景的代理人格。每個 Agent 擁有獨立的性格特質（樂觀/悲觀、風險偏好、專業背景）[blocmates, 2026; Medium, 2026]。

Stage 3：模擬執行（Simulation）

在雙平台架構上並行運行——類 Twitter 與類 Reddit 環境同時執行。代理自主互動：發文、辯論、形成聯盟。系統提供「上帝視角」介面，允許用戶在模擬過程中注入新變量（如突發新聞），觀察群體反應變化 [DEV Community, 2026; emelia.io, 2026]。

Stage 4：報告生成（Report Generation）

專用 ReportAgent 分析模擬後環境，使用完整工具集產出結構化預測報告，包含湧現模式、意見路徑、事件序列與置信度評估 [GitHub README, 2026]。

Stage 5：深度互動（Deep Interaction）

模擬世界在運行結束後持續可訪問。用戶可直接查詢個別代理，或與 ReportAgent 繼續對話以獲取額外分析 [DEV Community, 2026]。

三、關鍵技術組件

3.1 OASIS 模擬引擎

MiroFish 的模擬核心是 OASIS（Open Agent Social Interaction Simulations），由 CAMEL-AI 團隊開發。OASIS 可擴展至百萬代理規模，支援 23 種社交動作 [GitHub, 2026; Medium, 2026]。

3.2 GraphRAG 知識圖譜

不同於傳統 RAG 的平面向量檢索，GraphRAG 將原始資料結構化為實體-關係圖譜，讓代理的信念基於結構化資料而非原始文本，提高模擬的語境準確度 [DEV Community, 2026]。

3.3 Zep Cloud 記憶管理

代理維持跨輪次的持久記憶，包括既定的人格檔案、每輪累積的記憶更新、當前世界狀態觀察。Zep Cloud 處理持久化儲存，免費層可供初始實驗，Pro 層（$50-$200/月）支援多租戶企業部署 [DEV Community, 2026]。

3.4 技術棧

四、應用場景

4.1 輿情預測

武漢大學案例：將 BettaFish 產出的輿情報告作為種子材料，模擬公眾情緒演化軌跡。這是 MiroFish 最成熟的應用場景 [emelia.io, 2026]。

4.2 金融情緒模擬

模擬投資者對財報意外的反應、追蹤敘事傳播路徑、驗證投資者溝通策略 [DEV Community, 2026]。

4.3 創意寫作

團隊將《紅樓夢》前 80 回作為種子材料，讓系統模擬故事可能的結局——這展示了群體智慧在非預測性任務上的創造性應用 [emelia.io, 2026; blocmates, 2026]。

4.4 危機模擬

公關危機情境預演、政策發布前的社會反應預測、組織變革接受度分析 [DEV Community, 2026]。

六、開發背景與生態

開發者郭杭江的前作 BettaFish（多代理情緒分析工具）曾在 2024 年底登上 GitHub 全球 Trending 榜首，一週內收穫 20,000 Stars。MiroFish 延續了其群體智慧的研究方向，並在架構上進行了質的飛躍 [emelia.io, 2026]。

MiroFish-Offline 是社群維護的英文離線版本，使用 Neo4j + Ollama 本地堆疊替代雲端依賴 [GitHub nikmcfly/MiroFish-Offline, 2026]。

七、優勢與局限

優勢

  • 質性敘事驅動預測的獨特能力，傳統預測工具無法實現
  • 自動化人格生成與模擬後查詢功能
  • 百萬代理擴展能力（OASIS 引擎）
  • 相容任何 OpenAI API 後端，部署靈活

局限

  • 缺乏與真實結果的正式驗證及公開準確率基準
  • 相同輸入的多次運行結果不同，存在可重現性挑戰
  • 定量精度不及專用計量經濟學模型
  • 文件主要為中文，英文資源仍在發展中
  • v0.1.2 階段，定位為策略研究工具而非生產就緒平台

---

八、與本專案（Daily-Digest-Prompt）的借鑑啟示

  1. 群體智慧 vs 單一代理：MiroFish 證明多代理湧現行為能產生超越單一 LLM 的洞見，值得探索在自動任務中引入簡單的多視角模擬
  2. GraphRAG 結構化知識：知識圖譜優於平面向量檢索的理念可應用於知識庫進階檢索
  3. 動態變量注入：「上帝視角」中途注入新事件的設計，可借鑑於 Todoist 任務執行中的動態優先級調整
  4. Zep 記憶管理：持久化代理記憶的實作模式值得參考，強化 digest-memory.json 的跨次記憶機制
