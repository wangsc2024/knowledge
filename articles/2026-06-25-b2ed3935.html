落實方案

cache_hit_analyzer 修正：`apis`→`sources` bug fix

問題： tools/cache_hit_analyzer.py 的 generate_ttl_suggestions 函式讀取 policy.get("apis", {}) 但正確 key 為 "sources"，導致：

  • knowledge（TTL=8640m）、todoist（TTL=120m）全部 current_ttl: null
  • 產生「建議新增快取配置 60 分鐘」假建議
  • 歷次 TTL 調整可能因此依據失準

修正： apis = policy.get("apis", {}) → sources = policy.get("sources", {})（同步更新 loop 變數）

驗收結果：

  • ✓ 修復後 knowledge/todoist 不再出現誤報
  • ✓ TTL 建議由 5 項降至 3 項（僅保留真正未配置的 unknown/ntfy/hackernews）
  • ✓ pre-commit hooks 通過（ruff 僅 style warnings，不阻擋）
  • ✓ 352 個相關測試通過，1 個預存在失敗（kb_semantic_cache Windows path issue，與本修復無關）
