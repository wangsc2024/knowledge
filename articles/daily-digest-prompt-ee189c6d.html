Daily-Digest-Prompt 使用統計機制解讀

系統設計使用統計架構決策運維

使用統計系統架構

資料來源雙軌制

  1. PostToolUse Hook 自動記錄（hooks/post_tool_logger.py）

- Skills Read: 當 Agent 用 Read 工具讀取 skills/*/SKILL.md

- Workflows Read: 讀取 workflows/.yaml 或 .md

- Tools Invoked: Agent 透過 Bash 工具執行 uv run tools/*.py

  1. PowerShell 主動登記（tools/record_tool_invocation.py）

- run-todoist-agent-team.ps1 等排程腳本直接呼叫 tools 時，透過此工具登記

冷資產比例高屬正常現象

觀察：首次檢討顯示 81 個 Skills 中 78 個（96%）未讀取，96 個 tools 中 86 個（90%）未調用。

原因解讀（非問題）：

  1. Preamble 自動注入機制：templates/shared/preamble.md 自動要求讀取 SKILL_INDEX.md（limit=30 取得速查表），大部分 Skill 內容無需再次 Read
  1. PowerShell 直呼叫：run-todoist-agent-team.ps1、check-health.ps1 等腳本直接 uv run tools/*.py，不經過 Agent Bash 工具，除非主動登記否則不計入統計
  1. Workflows 按需載入：validation checklist 僅在特定情境觸發（如修改 config、新增自動任務），非每日必用
  1. 資料窗不足：7 天窗若僅含 1-2 日數據，無法反映完整使用模式

優化建議應對照實際路徑

誤區：僅因統計為 0 就建議刪除資產。

正確做法：

  • 對照 config/routing.yaml 的 Skill 映射
  • 對照 config/frequency-limits.yaml 的自動任務定義
  • 對照 PowerShell 腳本的 uv run 呼叫
  • 檢查 workflows/index.yaml 的 task_types 篩選條件

優化方向應聚焦於：

  1. 補齊統計覆蓋：高頻 PowerShell 腳本加入登記呼叫
  2. 建立基準線：累積 7+ 天數據後才進行趨勢分析
  3. 監控異常：某 Skill/Tool 使用量突降 50% 可能代表路由失效
  4. 健康儀表：將高頻 tools 納入 check-health.ps1 監控
