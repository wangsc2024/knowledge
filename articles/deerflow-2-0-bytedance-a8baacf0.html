一、專案概述

DeerFlow（Deep Exploration and Efficient Research Flow，bytedance/deer-flow）是 ByteDance 開源的 SuperAgent Harness 框架，定位為「能研究、寫程式、創作的長程超級代理人工具」。2026 年 2 月 28 日發布 2.0 版後即登上 GitHub Trending 第一名，截至 2026 年 4 月已累積 45,600+ Stars、3,800+ Forks，採用開源授權。

二、核心架構

2.1 Lead Agent 任務分解

DeerFlow 的核心架構圍繞一個 Lead Agent（主代理） 進行任務分解與協調：

  1. 結構化拆分：將使用者 prompt 分解為多個子任務
  2. 並行識別：自動識別可並行執行的工作
  3. 專責派發：為每個子任務生成具有限定上下文（scoped context）的子代理
  4. 結果綜合：收集所有子代理產出，合成連貫的最終輸出

此設計使 DeerFlow 能處理長達數小時的任務而不會觸及上下文窗口限制。

2.2 沙箱隔離執行

每個任務在隔離的 Docker 容器中執行：

  • 持久化檔案系統：skills、workspace、uploads、outputs 目錄
  • Bash 終端存取：可執行 shell 命令
  • Python 腳本執行：支援直接運行 Python
  • 真實檔案操作：不是模擬，而是實際的建立、修改、刪除

三種部署模式：

  • 本地執行：直接在主機上運行
  • Docker 隔離容器：標準部署方式
  • Kubernetes Pod：企業級 Provisioner 服務

2.3 長期記憶系統

  • 持久化儲存：記錄使用者偏好、寫作風格、專案上下文
  • 非同步防抖更新：non-blocking 設計，不影響主流程效能
  • TIAMAT 雲端後端：企業部署選項
  • 注意事項：Agent 記憶在生產環境仍需充分測試驗證

---

三、Skill 系統（Markdown-Based）

DeerFlow 的 Skill 系統是其最大特色之一：

  • 格式：以 Markdown 檔案定義工作流程
  • 內容：描述最佳實踐、參考資源、執行步驟
  • 載入方式：漸進式上下文載入（progressive context loading）

內建 Skills

| Skill 類別 | 功能 |

|-----------|------|

| 深度研究 | 網路搜尋、交叉驗證、結構化報告 |

| 程式碼生成 | 隔離環境中寫碼、測試、除錯 |

| 簡報製作 | 從研究成果生成投影片 |

| 網頁建置 | 生成靜態網頁 |

| 媒體創作 | 圖片與影片生成 |

自訂 Skill

使用者可依循相同的 Markdown 格式建立自訂 Skill，框架自動發現並載入。

六、與其他框架比較

| 框架 | 定位 | 差異 |

|------|------|------|

| LangGraph | Agent 建構積木 | DeerFlow 是完整可用系統，非積木 |

| CrewAI | 多代理協作框架 | DeerFlow 包含沙箱、記憶、Skill 全套 |

| AutoGen | 對話式代理 | DeerFlow 強調長程任務與實際執行 |

| Claude Code | 終端 AI 助手 | DeerFlow 是 Web-first 的協調平台 |

關鍵差異：DeerFlow 不是提供建構積木，而是交付一個有主見的（opinionated）完整可運作系統。

七、對 Daily-Digest-Prompt 系統的借鑑啟示

7.1 Skill-as-Markdown 驗證

本系統的 SKILL.md 模式與 DeerFlow 的 Markdown Skill 不謀而合，驗證了以 Markdown 作為 Agent Skill 定義格式的普遍趨勢。

7.2 沙箱分級參考

本系統的 Harness Engineering B0-E（risk_tiers）可參考 DeerFlow 的三級沙箱模式（本地→Docker→K8s），在不同風險等級下選用不同隔離層級。

7.3 記憶非同步化

DeerFlow 的「非同步防抖更新」機制值得本系統的 digest-memory 模組參考，避免記憶寫入阻塞主流程。

7.4 模型任務配對

本系統的 llm-router.yaml 路由機制已實現類似功能，DeerFlow 的 per-task model assignment 進一步驗證了此設計的正確性。

7.5 可觀測性整合

DeerFlow 內建 LangSmith/Langfuse 整合，本系統可考慮將 post_tool_logger 的 JSONL 輸出對接 OpenTelemetry，提升可觀測性。

八、ByteDance 的策略意圖

「價值不在 Agent 框架本身，而在其上構建的數據和服務。」

ByteDance 的開源策略旨在建立生態價值而非直接商業化核心技術，類似 Meta 的 Llama 策略。
