一、專案概述（What & Why）

OpenCut（OpenCut-app/OpenCut）是一款免費、開源的影片編輯器，定位為 CapCut（剪映國際版）的開源替代品。專案於 2025 年 6 月 22 日在 GitHub 上建立，採用 MIT 授權，截至 2026 年 3 月已累積 46,168 Stars、4,731 Forks，平均每日增長約 183 Stars，是 2025-2026 年間增長最快的非 AI 開源專案之一。

為什麼需要 OpenCut？

  1. 隱私保護：影片完全在本地裝置上處理，不上傳雲端
  2. 免費功能回歸：CapCut 越來越多基礎功能改為付費牆（paywall），OpenCut 提供免費替代
  3. 簡單易用：延續 CapCut 證明的「簡單即正義」設計哲學，降低影片編輯門檻
  4. 跨平台願景：目標覆蓋 Web、桌面、行動裝置三大平台

二、核心功能與技術亮點

目前已實現功能

  • 時間軸編輯（Timeline-based editing）：專業級多軌道時間軸
  • 多軌道支援（Multi-track support）：影片、音訊、字幕分軌管理
  • 即時預覽（Real-time preview）：編輯同步預覽效果
  • 無浮水印：與 CapCut 免費版加浮水印不同，OpenCut 完全無浮水印
  • 無訂閱制：所有功能免費使用，無隱藏付費

| 層級 | 技術選型 | 說明 |

架構重大轉型：從 DOM 渲染到二進位渲染

目前 OpenCut 的預覽系統使用 HTML DOM 元素渲染，這是原型階段的做法。團隊正在進行重大重構：

  • 現有架構：HTML/CSS 渲染 → 預覽與匯出結果不一致
  • 目標架構：二進位渲染（Binary rendering），類似 CapCut 原生渲染方式
  • 預期效益：預覽與匯出完全一致、效能大幅提升、支援更複雜的特效
  • WebCodecs API：Issue #460 追蹤中，利用瀏覽器原生 WebCodecs 實現核心影片匯出

這個架構決策意味著 OpenCut 不只是「網頁版 CapCut」，而是要成為真正的專業級編輯器。

三、使用場景與範例

適用場景

  1. 個人創作者：YouTube/TikTok 短影片剪輯，無需安裝桌面軟體
  2. 教育用途：學校電腦無法安裝專業軟體時，OpenCut 的 Web 版即開即用
  3. 隱私敏感場景：處理機密或個人影片，不希望上傳至第三方雲端
  4. 企業自架：透過 Docker Compose 一鍵自架部署（docker compose up -d），資料完全自控
  5. 開發者學習：TypeScript + Next.js + WebAssembly FFmpeg 的最佳實踐參考

快速上手


# 方法一：Docker 全包部署
git clone https://github.com/OpenCut-app/OpenCut.git
cd OpenCut
docker compose up -d
# 開啟 http://localhost:3100

# 方法二：開發模式
bun install
docker compose up -d db redis serverless-redis-http
bun dev:web
# 開啟 http://localhost:3000

六、評估結論

優勢

  1. 填補市場空白：Web 原生影片編輯器幾乎沒有競爭者
  2. 技術棧現代化：React 19 + Next.js 16 + Bun + WebAssembly FFmpeg，吸引前端開發者
  3. 增長驚人：252 天 46K Stars，社群動能強勁
  4. 自架能力：Docker 一鍵部署，適合企業內部使用
  5. MIT 授權：最寬鬆的開源授權，商業友善

風險與挑戰

  1. 單人核心風險：71% commits 來自一人，bus factor = 1
  2. 功能成熟度低：v0.1.0 階段，距離生產級使用有距離
  3. 架構重構中：DOM → Binary rendering 的遷移是高風險技術債
  4. 瀏覽器限制：WebAssembly FFmpeg 效能仍不及原生，大檔案處理受限
  5. 匯出功能未完成：核心功能之一（WebCodecs 匯出）仍在開發中

技術啟示

  • WebAssembly 成熟度：FFmpeg 編譯為 WASM 在瀏覽器運行，展示了 WebAssembly 的實用價值
  • Bun 生態崛起：OpenCut 選用 Bun 而非 npm/yarn，反映前端工具鏈的典範轉移
  • Zustand 取代 Redux：輕量狀態管理在中大型專案中的可行性
  • 瀏覽器端 AI：@huggingface/transformers 在瀏覽器中運行推論，無需後端 GPU

總評

OpenCut 是 2025-2026 年最值得關注的非 AI 開源專案之一。它不只是「另一個影片編輯器」，而是對「瀏覽器能力邊界」的一次大膽探索。雖然目前功能尚不成熟，但其技術願景（Binary rendering + WebCodecs + 瀏覽器端 AI）和社群動能使其具有成為「Web 版 DaVinci Resolve」的潛力。
