Pac-Man v1.1 優化心得：水果獎勵道具系統與螢幕震動視覺回饋

優化概述

本次為 HTML5 Canvas Pac-Man 遊戲新增兩大創意改良：

  1. 水果獎勵道具系統 — 忠於原作的 Bonus Item 機制
  2. 螢幕震動效果 — 碰撞視覺回饋增強打擊感

水果獎勵道具系統

設計要點

  • 每關吃 70 顆和 170 顆豆子時各觸發一次水果出現
  • 水果類型隨關卡遞增：Cherry(100) → Strawberry(300) → Orange(500) → Apple(700) → Grape(1000) → Key(2000) → Crown(5000)
  • 水果出現在迷宮中央（ghost house 下方 y=17），持續 10 秒後消失
  • 最後 3 秒有閃爍預警效果

技術實作

  • 使用 Canvas fillText 繪製 emoji 水果（避免圖片資源依賴）
  • 分層圓形光暈效果（取代 shadowBlur 以維持效能）
  • 上下浮動動畫（Math.sin 驅動的 bob 效果）
  • 專屬音效：出現時用 sine 波上行 blip，吃到時用 triangle 波四音階慶祝
  • HUD 顯示最近收集的 5 個水果 emoji，未收集時以半透明顯示當前關卡水果

狀態管理

  • fruitSpawned[0,1] 追蹤每關兩次觸發是否已用
  • fruitsCollected[] 記錄整場遊戲收集的水果歷史
  • restartGame() 和 nextLevel() 正確重置狀態

螢幕震動效果

設計要點

  • 吃到幽靈：中等震動（intensity=4）
  • 吃到高分水果：中等震動（intensity=3，≥1000分則 5）
  • Pac-Man 死亡：強烈震動（intensity=8）

技術實作

  • 使用 ctx.save() → ctx.translate(shakeX, shakeY) → 繪製全場景 → ctx.restore()
  • 震動衰減用指數衰減：shakeIntensity *= Math.pow(0.01, dt)
  • 每幀隨機偏移量 = (Math.random() - 0.5) intensity 2
  • 死亡狀態（STATE_DYING）也持續更新震動，確保動畫連貫

效能考量

  • 水果光暈用分層圓形而非 shadowBlur（shadowBlur 在 Canvas 2D 極慢）
  • 震動用 translate 而非重新計算每個物件座標
  • 水果過期時發射少量粒子（8 個）避免過度特效

品質基線

  • 原有功能全部保留：4 隻 Ghost AI、粒子系統、8-bit 音效、CRT 濾鏡、觸控支援
  • Brace 平衡檢查通過，2030 行程式碼
  • 版本：v1.0 → v1.1（小改良，非核心機制變更）
