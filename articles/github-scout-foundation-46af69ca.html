  • 核心機制： - 分析 code diff → 映射至歷史測試結果 → ML 模型預測「最可能失敗的測試」 - 動態優先排序：最易失敗的測試先跑（fail-fast），可並行的測試同步執行 - 效果：50-80% 測試時間減少，pipeline feedback 從 30 分鐘縮短到 6-10 分鐘
  • Launchable 工作流： ```yaml # .github/workflows/smart-test.yml - name: Record test results uses: launchableinc/record-build@v1 with: build-name: ${{ github.run_id }}
  • - name: Get subset (TIA) run: | launchable subset --build ${{ github.run_id }} \ --target 20% pytest > test_subset.txt
  • - name: Run only impacted tests run: uv run pytest $(cat test_subset.txt) ```
  • SAPAL = Sense-Analyze-Predict-Act-Learn（專為 CI/CD 設計的 MAPE-K 改良版）：
  • | 層次 | 功能 | |------|------| | Sense | 收集 code changes、build logs、test results、部署指標 | | Analyze | 模式識別、異常偵測（flaky test 識別）| | Predict | 預測建構失敗機率、測試穩定性評分 | | Act | 動態決策：哪些測試跑、哪些 skip、是否觸發 rollback | | Learn | 每次執行後更新模型，cross-stage 因果依賴學習 |
  • 五層架構： 1. Data Collection（可觀測性） 2. Reliability Intelligence（模式識別） 3. Predictive Modeling（失敗預測） 4. Adaptive Execution（動態決策） 5. Human-AI Collaboration（開發者回饋閉環）
  • 量化效果： - Flaky test 導致的建構失敗：減少 60% - ML 測試選擇的 feedback 時間：縮短 50-80% - Pipeline noise（非程式碼因素建構失敗率）：從 5-16% 降至 <3%
