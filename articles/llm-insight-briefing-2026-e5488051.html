批判性思維與 AI 系統：認知偏誤如何影響 LLM 輸出品質及工程緩解策略

產出日期：2026-05-31 | 任務：insight_briefing | 系列：cognitive-bias-llm-output-quality（foundation）

摘要

大型語言模型（LLM）的輸出品質問題，根本上源於人類認知偏誤的三層傳播機制：訓練語料中的統計捷徑（預訓練層）、提示語言的隱性偏見（提示層）、以及人類標注的主觀偏差（RLHF 層）。2025 年最新研究顯示，錨定偏誤導致 46% 的推理任務錯誤，確認偏誤在 GPT-5 中達 31%。好消息是：以「軸邏輯推理線索（Axiomatic Reasoning Cues）」為代表的工程策略可將偏誤降低 51–73%，且實施成本低。本報告梳理偏誤分類體系、傳播路徑與五層工程緩解框架，直接可用於 Agent 系統設計與 Prompt 工程實踐。

核心洞見

洞見 1：LLM 幻覺的四類型分類體系

來源：Frontiers in Artificial Intelligence, 2025（A 級）

LLM 幻覺依「輸入與輸出關係」分為四類：

| 類型 | 定義 | 工程特徵 |

|------|------|---------|

| 內在型（Intrinsic） | 輸出與輸入來源矛盾 | 高 PS（提示敏感度），低 MV（模型變異性） |

| 外在型（Extrinsic） | 輸出資訊不在輸入來源中 | 混合 PS/MV |

| 事實型（Factual） | 輸出陳述與已知事實不符 | 低 PS，高 MV（模型主導） |

| 邏輯型（Logical） | 語法正確但推理不一致 | 最難偵測，CoT 可部分暴露 |

關鍵發現：模型優化「流暢度/一致性」而非「真實性」，token 機率最大化本身就是偏誤的工程根源。

---

洞見 2：認知偏誤的三層傳播機制

來源：arXiv:2510.06265（A 級），arXiv:2505.19240（A 級）


預訓練層 → 提示層 → RLHF 對齊層
    ↓           ↓           ↓
統計捷徑   隱性引導    標注偏差傳遞

預訓練層：

  • Transformer 從語料庫習得統計「捷徑」，頻率相關性取代因果推理
  • 錨定偏誤：46% 的推理任務錯誤源於初始提示詞的過度依賴
  • 確認偏誤放大率：GPT-5 達 31%，Claude 3.5 達 19%（更小更優）

提示層：

  • 模糊提示導致 65% 更多事實錯誤（allaboutai.com 基準測試）
  • 提示語言的情感傾向、問題框架方式，直接決定偏誤程度

RLHF 對齊層：

  • 標注者池同質性 → 中立性喪失
  • 惡意/偏斜標注可污染全域獎勵模型
  • 政治傾向偏差隨標注規模累積

洞見 3：軸邏輯推理線索是當前最有效的提示層緩解策略

來源：arXiv:2604.16756v2（A 級），2025 年

核心機制：將隱性最佳實踐轉化為顯性推理規則，嵌入提示中作為「公理」。


傳統提示：「請分析這個需求」
→ 偏誤風險：錨定在初始措辭

軸邏輯提示：「分析此需求時，安全性影響 > 效能優化」
→ 偏誤降低：51–73%

對比其他策略的效能：

  • Chain-of-Thought：18.1% 偏誤率 vs. 基準 38.3%（↓53%，但仍有偏誤放大風險）
  • 自我去偏（Self-debiasing）：幾乎無效
  • 模糊去偏指令（「請避免偏誤」）：幾乎無效

反直覺發現：CoT 有時會惡化偏誤敏感度，因為推理鏈可能強化初始錨定。

洞見 4：RLHF 層的兩大工程修補方向

來源：arXiv:2502.13417（RLTHF），arXiv:2501.05790（Influence Functions），Nature Scientific Reports（A 級）

方向 A：針對性人類反饋（RLTHF）

  • 傳統 RLHF 隨機採樣 → RLTHF 聚焦特定失敗模式
  • 對關鍵弱點的批評性反饋比均勻分佈更有效
  • 降低標注成本同時提升對齊品質

方向 B：影響函數監測

  • 追蹤每個訓練樣本對模型輸出的影響力
  • 識別「毒性反饋」的系統性模式
  • 三步驟修補：① 樣本重加權 → ② 迭代精修 → ③ 高影響樣本人工複審

方向 C：共識獎勵模型

  • 防止單一惡意標注者污染全域模型（Nature, 2025）
  • 多標注者共識機制 + 異常值偵測

本專案 RCA 記錄中多次出現「模型幻覺」「不當修復建議」「路徑幻覺（如 /var/log...）」，對照本次研究框架的診斷：

| 本專案症狀 | 偏誤類型 | 對應緩解 |

|-----------|---------|---------|

| Gemma-E4B 對 timeout 場景給出不適用建議 | 能力邊界幻覺 | 模型路由 + 分類閾值（已修：claude_sonnet46） |

| 連續 RCA 誤判（silent_degradation → resource_exhaustion） | 確認偏誤（既有分類框架錨定） | 軸邏輯提示：明示分類邊界規則 |

| doc_generation 讀步驟 6 停止（gpt-oss-120b） | 模型能力 × 指令複雜度幻覺 | 模型分配對齊任務複雜度 |

---

建議行動

  1. 立即可行（低成本）：在 Agent prompt 中引入軸邏輯推理線索

- 對 RCA 分類 prompt 加入明確規則：「span duration ≥ timeout × 0.95 → RESOURCE_EXHAUSTION，非 SILENT_DEGRADATION」

- 對任務路由 prompt 加入：「模型能力等級 < 任務複雜度 → 拒絕並回傳 quality_gate_blocked」

  1. 中期（中成本）：建立 RLTHF 回饋閉環

- 識別本系統中最常出錯的任務類型（已有 RCA 數據）

- 對這些類型增加針對性反饋標注，而非均勻採樣

  1. 長期（高價值）：影響函數監測 + 共識驗證

- 多模型交叉驗證關鍵決策（已有部分實踐：claude_sonnet46 ← gemma_e4b）

- 建立「偏誤偵測」層作為關鍵任務的第二閘門
