GitHub Scout: LLM 可觀測性/trace（Langfuse/Phoenix/OpenLLMetry + OTel GenAI）

觀測日期: 2026-04-26

來源: WebSearch 快照（stars 值待 gh api 複核）

專案觀測摘要

1. langfuse/langfuse（~25k★）

開源 LLM 工程平台。v4 Python SDK（March 2026）完整重寫，全面 OTel-native。涵蓋 tracing、LLM-as-judge eval、prompt 管理、dataset 管理。YC W23，2300+ 公司使用，每月處理 billions observations。

2. Arize-ai/phoenix（~9.1k★）

AI Observability & Evaluation。OTLP-native，支援 LLM-as-judge、human annotation workflow。為 agents/tools/guardrails/handoffs 各生成獨立 OTel span，對齊 GenAI Agentic Conventions。

3. traceloop/openllmetry（~6.7k★）

OTel extensions for LLM。含 opentelemetry-instrumentation-ollama/langchain/openai/mcp 等儀器包。新增 MCP server for querying traces。採用 gen_ai. semconv 命名（daily_digest. 已在本系統 review policy 中引用）。

改進提案

P1-a: ollama span alignment（已落實）

在 config/github-scout-review-policy.yaml audit_trace_crosswalk 補充 gen_ai.system=ollama / gen_ai.operation.name=chat 標準值，對齊 OpenLLMetry opentelemetry-instrumentation-ollama 慣例。讓 tools/ollama_local.py 呼叫有業界標準對齊點。

P1-b: SKILL.md triggers 擴充（已落實）

github-scout SKILL.md 新增 10 個 LLM 觀測觸發詞（Langfuse/arize-phoenix/openllmetry/OTel GenAI 等），版本升至 v2.6.0。

P2: OTel GenAI Agentic semconv note（已落實）

在 crosswalk 補充 gen_ai.agent.name / gen_ai.agent.task 屬性參考（issue #2664 實驗草案），為 trace_analyzer.py 未來對齊 agentic span 命名提供方向。

審查結論

全部 3 個提案通過（feasibility ≥ 9, stability ≥ 9），均為純 YAML note/trigger 追加，無功能衝擊，已直接落實。
