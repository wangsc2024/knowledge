  1. []{#__RefHeading___Toc387310327 .anchor}作業說明

1. 存取控制政策

1. 資訊資產之存取範圍應與本身業務需求相關，未經授權不得存取業務範圍外之資訊資產。

2. 應正確地使用資訊資產，以維護資訊資產之可用性、完整性與機密性。

3. 非因業務需求不得將系統存取帳號提供給非本府員工，若因業務需要開放帳號予外部人員，應有適當安全控管措施，該安全控管措施應考量業務需求範圍及資訊資產機密性，並授與適當存取權限及有效日期。

4. 被賦予系統最高管理權限之人員、掌理重要技術及作業控制之特定人員，應經過審慎之授權評估。

5. 可攜式電腦或儲存媒體，例如：筆記型電腦、平板電腦、隨身碟、外接式硬碟、光碟、磁帶等，應採取適當之控管措施，以防止未經授權之資料、系統或網路存取。

6. 資料、資訊之存取，必須符合「個人資料保護法」、「電子簽章法」及「著作權法」等相關法規之規定，或契約對資料保護及資料存取控管之規定。

7. 內部共享資料交換空間之存取權限應有適當控管，以避免未經授權存取或誤刪、誤用。

8. 桌上型電腦、可攜式電腦、伺服器應設定於一定時間閒置時登出或自動鎖定系統並以密碼保護，以避免未經授權之存取。

2. 帳號與通行密碼管理

1. 新建置之應用軟體或系統，安裝完成後應立即更新預設之通行密碼，並刪除或關閉不必要之帳號。

2. 不得開放匿名或共享之帳號。

3. 不得任意新增、刪除或修改帳號登入軌跡資料，以避免資訊安全事件發生時無法追蹤查詢。

4. 使用者帳號管理

1. 使用者申請公文管理系統、差勤系統、OA系統、OA2系統、AD或府外單一簽入系統帳號時，需透過線上申請程序，且權責主管線上審核同意後，由行政暨研考處資訊管理科辦理。

2. 除上述系統外，申請人應填寫「帳號申請異動單」，經權責主管審核同意後，交由系統管理單位辦理。

3. 使用者於業務異動時，應比照帳號申請程序辦理帳號之異動／停用／刪除。

4. 系統使用者應以個人專屬之帳號及密碼進行登入作業。

5. 系統管理者帳號管理

1. 系統管理人員應避免共用系統管理者帳號，且應妥善保管帳號與通行密碼。

2. 系統管理人員需經權責主管審慎挑選指派，並填寫「帳號申請異動單」，經權責主管核准後，授予具管理權限之帳號及通行密碼。

3. 系統管理人員因職務調動，需異動其系統帳號時亦需填寫「帳號申請異動單」。

4. 系統管理人員應建立備用管理帳號供職務代理人使用，並製作帳號及通行密碼表，彌封放置於指定場所，於緊急狀況時使用。使用後需填寫紀錄，並由系統管理人員變更密碼後重新製表及彌封。

5. 委外廠商如必須使用系統特殊權限帳號，得向府內權責人員提出申請，並由該人員填寫「帳號申請異動單」經權責主管核准後，由系統管理單位授予特殊權限之帳號及通行密碼。

6. 通行密碼管理

1. 使用者帳號密碼安全設定原則：

1. 密碼長度：長度應為8個字元以上。

2. 密碼複雜度：宜具一定之複雜度(如：英文數字符號混合)，不得為空白或與帳號相同，且應避免使用容易猜測之密碼。

3. 密碼變更週期：至少每6個月變更密碼一次。

4. 系統預設密碼之變更：於使用預設密碼第一次登入時，須立即變更系統預設之密碼。

5. 帳號鎖定：密碼連續5次以下輸入錯誤，應鎖住該帳號，需等待3分鐘以上由系統自動解鎖，以防密碼遭到破解。

2. 資訊系統管理者帳號密碼安全設定原則：

1. 密碼長度：長度應為8個字元以上。

2. 密碼複雜度：宜具一定之複雜度(如：英文數字符號混合)，不得為空白或與帳號相同，且應避免使用容易猜測之密碼。

3. 密碼變更週期：至少每6個月變更密碼一次。

4. 重複使用限制：至少不得與前3代之密碼重複。

5. 系統預設密碼之變更：於使用預設密碼第一次登入時，須立即變更系統預設之密碼。

6. 帳號鎖定：密碼連續5次以下輸入錯誤，應鎖住該帳號，需等待3分鐘以上由系統自動解鎖，以防密碼遭到破解。

3. 資料庫帳號密碼安全設定原則

1. 密碼長度：長度應為8個字元以上。

2. 密碼複雜度：宜具一定之複雜度(如：英文數字符號混合)，不得為空白或與帳號相同，且應避免使用容易猜測之密碼。

3. 密碼變更週期：應用系統內建使用之帳號（已寫在應用系統程式內之帳號、密碼，如公文系統AP程式內建使用之資料庫帳號及密碼）原則不變更密碼，其他如管理者帳號、一般使用者帳號至少每6個月變更密碼一次。

4. 如系統無內建密碼安全管控機制，則以人工方式管控。

5. 使用者須遵守下述密碼管理規定：

1. 密碼保密原則：密碼必須保密，不得借予他人使用。

2. 密碼儲存方式：避免以明文且未受保護之形式儲存密碼，如:

以書面記載且放置於明顯處。

3. 密碼洩露處理：懷疑密碼洩露或忘記時，須立即更換密碼。

4. 限制使用自動儲存密碼之功能：使用非本府個人專用電腦存取本府網路或資訊系統時，不得使用自動儲存密碼之功能。

6. 系統登入介面應避免於登入程序中以明碼方式顯示通行密碼。

7. 系統登入介面宜在完成所有的登入資料輸入後，才開始查驗登入資訊的正確性；如果登入發生錯誤，系統不應顯示那一部分資料是正確的，那一部分資料是錯誤的。

8. 系統如經評估須建立較通行密碼更高等級的安全機制，可利用電子簽章等安全等級更高的存取控制技術。

3. 作業系統存取控制

1. 使用者帳號避免顯示任何足以辨識使用者特別權限的訊息，例如：admin顯示其可能為管理者。

2. 系統管理人員結束系統維護作業後，應關閉作業系統管理畫面，清除螢幕上的資訊，登出系統，並鎖定主控台螢幕。

3. 應留存作業系統之軌跡紀錄。

4. 資料庫之存取控制

1. 資料庫使用者之帳號密碼設定與使用必須符合本程序書之帳號密碼管理要求。

2. 資料庫最高權限帳號之存取授權應僅限於資料庫管理員。

3. 安裝資料庫時需改變或移除系統的公用存取帳號。

4. 資料庫預設帳號應變更密碼，或是關閉使用。

5. 應留存資料庫之軌跡紀錄。

6. 資料庫不得以明文方式儲存、傳送或顯示密碼。

5. 應用系統之存取控制

1. 資訊存取之限制

1. 應用系統資訊之使用，僅限業務相關之授權使用者，並應適當控制。例如：新增、刪除或執行等權限控管。

2. 應用系統之資訊依機密等級「密」(含)以上資料，應與一般資訊作適當區隔，並加強權限控管措施。

2. 重要系統程式資源之存取控制

1. 應用程式原始碼，應指定專人管理程式之增修作業。

2. 開發中之原始程式碼，應與線上程式碼分開放置與控管。

3. 舊版的原始程式應妥慎保管，並詳細記錄使用的明確時間，以備新版失敗回復使用。

3. 應用系統宜設定系統登入程序之時間限制，如果超出閒置時間限制，系統將自動中斷登入。

6. 網路存取控制

1. 網路系統應依重要性之不同加以保護，以降低可能的安全風險。

2. 網路管理人員應定期檢視重要網路設備存取活動，並留存日誌備查，該日誌需以適當之權限加以管制。

3. 對於開放外部客戶或廠商存取之本府網路，必須限制使用者可用之網路服務類型，以確保網路安全。

4. 網路路由之規劃必須確保任何網路連線或資訊傳輸符合網路存取之安全需求。

5. 網路服務存取申請原則依本府「通信管理程序書」辦理。

7. 帳號權限清查

1. 為確保一般使用者存取權限之適當性，重要系統之管理人員應每年定期或視需要辦理帳號權限清查作業。

2. 應建立重要系統最高權限及特殊權限使用者「系統帳號權限清單」，系統管理人員每年至少實施帳號權限審查1次，不必要之權限與帳號應即時修訂刪除，審查結果送權責主管審查核定。
