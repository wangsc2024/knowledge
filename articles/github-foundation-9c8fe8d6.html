  • - Pydantic v2 是 Python 生態中最廣泛使用的資料驗證與序列化函式庫。2023 年 6 月發布的 v2 版本以 Rust 重寫核心驗證引擎（pydantic-core），帶來 4～50 倍效能提升。[來源：Pydantic 官方文件，2026；GitHub Discussion #6748] 它解決三個関鍵問題：
  • - 問題 1：型別安全邊界。Python 的動態型別系統在系統邊界（HTTP API 輸入、LLM 輸出、檔案讀取）極易引入 AttributeError 與 KeyError。Pydantic 在邊界強制執行結構化驗證，通過即成為已知結構的 Python 物件。
  • - 問題 2：LLM 輸出解析。LLM 回應是非結構化文字或 JSON，Pydantic v2 是 Instructor、PydanticAI、LangChain、LlamaIndex 等 2026 年主流 Agent 框架的共同解析骨幹。[來源：jangwook.net，2026]
  • - 問題 3：設定管理。pydantic-settings 提供統一的多來源設定載入（環境變數 / .env 檔 / Kubernetes secrets / CLI），含型別驗證與優先順序管理。[來源：Pydantic Settings 官方文件，2026]
  • - 與替代方案的差異：marshmallow 效能差距 5-20 倍且語法繁瑣；attrs 輕量但無 JSON Schema 生成；Python dataclasses 無驗證語義。2026 年 AI 生態中 Pydantic v2 是結構化輸出的事實標準，PyPI 週下載量超過 2,000 萬次。[來源：pypi.org/project/pydantic，2026]
  • - Pydantic v2 的核心是 pydantic-core，用 Rust 實作驗證邏輯並透過 PyO3 暴露 Python API。驗證流程：Python 呼叫 BaseModel(**data) → Rust 端執行型別強制與驗證 → 回傳已驗證 Python 物件。[來源：Pydantic 官方文件 models，2026]
