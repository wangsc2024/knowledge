Spotify 如何用 Agent 處理 2000 萬行程式碼：Niklas Gustavsson 訪談（9DHZLw5653E）

【YouTube 摘要 | 來源：Anthropic YouTube | 2026-06-29】

Spotify 首席架構師 Niklas Gustavsson（分子生物學博士出身，轉工程 30 年）分享 Spotify 在 AI 代理上的前沿實踐。

個人突破點：

  • 早期曾用 LLM 自動化程式碼變更（GPT 初期），成效有限但看見潛力
  • 個人真正突破：Claude Opus 4/5（2025 年 11-12 月），從「聰明自動補全」進化為「可以丟真實問題」的工具
  • 最大改變：不再需要進 IDE 做最後 20-30% 的編輯，直接接受 model 輸出

Spotify 工作方式：

  • 以 tmux 多視窗並行跑多個 Claude Code agent
  • 前年底預測「年底沒人用 IDE」，2 個月後自己親身驗證
  • 面對 2000 萬行以上的程式庫，Agent 協作是唯一可行的規模化路徑

適用情境：大型工程團隊 AI 採用、Claude Code 工作流設計、Agent 並行策略。
