AI Agent Hook/Guard 工具護欄 2026

技術概述

AI Agent 的工具護欄（Guardrail）是在工具呼叫執行前攔截並評估請求的安全機制，2026 年已形成三個主流架構：BodAIGuard（YAML 驅動全通道防護）、anywhere-agents guard.py（PreToolUse 鉤子四閘門）與動態授權模型（ABAC 取代 RBAC）。這三者可組合部署，形成縱深防禦。

核心工具：BodAIGuard

BodAIGuard（AxonLabsDev/BodAIGuard）是通用 AI Agent 護欄框架，特點：

  • 規則規模：45 條封鎖規則 + 45 條確認規則，涵蓋 12 個類別
  • 12 大類別：filesystem、database、network、email、git、process、secrets、packages、permissions、containers、deployment、prompt injection
  • 五種執行通道：

1. Claude Code hooks（PreToolUse/PostToolUse）

2. HTTP proxy（攔截 API 呼叫，強制 stream: false）

3. MCP stdio proxy（包裝 Model Context Protocol 伺服器）

4. System prompt injection（嵌入護欄至 Agent 指令）

5. REST API（獨立評估服務）

SSRF 防護


# ~/.bodaiguard/rules.yaml 範例
actions:
  network:
    block:
      - pattern: "absolute_url_in_proxy"
        reason: "防止 SSRF：代理請求禁止絕對 URL"
paths:
  block:
    - "~/sensitive-dir/**"
  • 拒絕代理請求中的絕對 URL（大小寫不敏感，所有 HTTP 方法）
  • 移除 proxy-authorization 標頭，防止代理驗證 token 洩漏至目標 API
  • 符號連結解析 + 多路徑評估，防止路徑遍歷繞過

反提示注入（三層偵測）

| 層次 | 方法 | 延遲 |

|------|------|------|

| L1 regex | 模式比對 | ~0.1ms |

| L2 heuristic | Base64 + 零寬字符解碼 | ~1ms |

| L3 structural | 隱藏 HTML、注釋注入、不可見文字 | ~5ms |

安裝


bodaiguard install hooks
# → 修改 ~/.claude/settings.json，注入 PreToolUse + PostToolUse

Fail-closed 原則：任何錯誤預設封鎖，不靜默放行。

anywhere-agents guard.py（PreToolUse 四閘門）

anywhere-agents（yzhao062/anywhere-agents）提供可攜式 PreToolUse hook，部署至 ~/.claude/hooks/guard.py：

四閘門架構


PreToolUse hook (guard.py)
├── Gate 1: Destructive Git/GitHub (Ask Gates)
│   ├── 需確認：git push, git commit, git merge, git rebase
│   ├── 需確認：git reset --hard, gh pr merge, gh pr create
│   └── 靜默通過：git status, git diff, git log（唯讀操作）
│
├── Gate 2: Compound Command Shape (Deny Gates)
│   ├── 攔截：cd /path && destructive_command
│   └── 無條件拒絕（無例外）
│
├── Gate 3: Writing-Style Enforcement (Deny Gates)
│   ├── 目標：.md / .tex / .rst / .txt 檔案
│   ├── 封鎖：~45 個 AI 語氣詞（delve、nuanced、pivotal 等）
│   └── Write/Edit 含禁用詞 → 拒絕並列出違規詞
│
└── Gate 4: Banner/Status Gates (Deny Gates)
    └── 防止靜默執行配置敏感操作

配置與繞過


// ~/.claude/settings.json
{
  "env": {
    "AGENT_CONFIG_GATES": "off"  // 停用 Gate 3+4，Gate 1+2 不受影響
  }
}

不對稱設計：破壞性/複合命令守衛始終啟動，風格/橫幅守衛可關閉，優先保護倉庫安全。

通用 Pre-Action Authorization 模式


[Agent] → 嘗試工具呼叫
    ↓
[Hook/Interceptor] → 攔截請求
    ↓
[Policy Engine] → 評估規則（allow / deny / require_approval）
    ↓
[Action / Block]  ~40ms 額外延遲

40+ 防護模式分類：

  • filesystem：系統路徑、遞迴刪除
  • network：非白名單域名
  • data exports：批量讀取、PII 偵測
  • code execution：eval 模式、子進程生成
  • messaging：外部收件人、Agent 生成附件
重要：此層評估動作，不評估內容，與輸入/輸出過濾互補，非替代關係。

為何 RBAC 不夠：轉向 ABAC 動態授權

RBAC 失敗的兩種情境：

  1. Confused Deputy 問題：Agent 繼承用戶的所有 scope（email + storage + GitHub + Slack），即使任務只需要其中一個
  2. 角色動態漂移：單次對話中 Agent 可能從「唯讀研究」轉為「可寫入程式碼執行」，靜態角色無法應對

混合架構（RBAC + ABAC）實踐：


RBAC（結構邊界）：定義允許的最大權限集
    +
ABAC（動態約束）：評估 who + what resource + context
    ↓
Authorization Fabric → ALLOW / DENY / REQUIRE_APPROVAL

實施建議：


# Agent 授權設計原則
agent_identity:
  - 獨立 Agent 身份（分離 OAuth 憑證，非繼承用戶 token）
  - 操作特定工具（narrow scope，非通用端點）
  - 短效憑證（分鐘級，非天級）
  - MCP 資源綁定 token（RFC 草案規範）

audit_trail:
  - 動作 → 授權規則的完整連結
  - 策略引用稽核軌跡（不只記錄動作）

與本專案的應用場景

| 場景 | 建議工具 | 整合點 |

|------|---------|--------|

| Claude Code hooks 強化 | BodAIGuard bodaiguard install hooks | ~/.claude/settings.json |

| 破壞性 git 操作防護 | anywhere-agents guard.py Gate 1+2 | hooks/pre_bash_guard.py 可吸收 Gate 2 邏輯 |

| SSRF 防護（curl / fetch） | BodAIGuard network 規則 | hooks/pre_bash_guard.py 新增絕對 URL 偵測 |

| Agent 動態授權 | ABAC 策略評估 | tools/budget_guard.py 可擴充任務範圍約束 |

| 提示注入偵測 | BodAIGuard L1-L3 三層 | hooks/hook_pipeline.py 插入 prompt_injection_detector |

風險與注意事項

  1. 延遲成本：Pre-action hook 約增加 40ms/呼叫，高頻工具（Bash）累積明顯
  2. False Positive：writing-style gate 45 個封鎖詞可能誤封正當技術文件
  3. 繞過風險：AGENT_CONFIG_GATES=off 應限制為開發環境，不得進入排程腳本
  4. Fail-open 陷阱：看門人邏輯出錯時，必須預設封鎖（fail-closed），而非靜默放行
  5. 規則維護：YAML 驅動規則需定期審查，舊規則可能與新工具版本衝突
