搜尋主題

Agent 架構 + Skill/Plugin 系統

重點專案分析

1. openai/openai-agents-python (18,986 Stars)

  • 核心架構：Agent = LLM + Instructions + Tools + Guardrails + Handoffs
  • Guardrails 機制：輸入驗證（tripwire 模式攔截危險輸入）+ 輸出驗證（結構化型別檢查），是安全驗證的一等公民
  • Handoff 控制轉移：特殊工具呼叫，Agent 可宣告式地將控制權轉移給其他 Agent，附帶上下文傳遞
  • Sessions 跨次記憶：內建 SQLiteSession，自動維護對話歷史
  • Tracing 內建追蹤：支援 Logfire、AgentOps、Braintrust 等外部追蹤
  • 借鑑價值：Guardrails 可強化本專案的 quality-gate，Handoff 可改善子 Agent 委派語義

2. pydantic/pydantic-ai (14,942 Stars)

  • 核心架構：完全型別安全的 Agent 框架，FastAPI 設計哲學
  • 依賴注入：RunContext + deps_type 模式，工具函數透過型別安全的 DI 取得資源
  • Evals 評估系統：系統性測試 Agent 效能與準確度，持續監控
  • MCP/A2A 整合：同時支援 Model Context Protocol 和 Agent2Agent 協議
  • Human-in-the-Loop：工具呼叫可標記需人工審批
  • Durable Execution：持久化執行，跨 API 故障保留進度
  • 借鑑價值：依賴注入可解決 Skill 硬編碼端點問題，Evals 可建立自動品質回歸偵測

3. lastmile-ai/mcp-agent (8,035 Stars)

  • 核心願景：MCP is all you need — 簡單模式比複雜架構更穩健
  • 完整 MCP 支援：Tools/Resources/Prompts/Notifications + OAuth/Sampling/Elicitation
  • 工作流模式：實作 Anthropic 論文所有模式（map-reduce、orchestrator、evaluator-optimizer、router），且可組合串接
  • Temporal 持久化：從簡單 Agent 到生產級工作流，無需改 API
  • Agent as MCP Server：Agent 可暴露為 MCP Server 供其他系統呼叫
  • 借鑑價值：可組合工作流模式可升級本專案的固定三階段管線

改進建議（按優先級）

P0 — 依賴注入模式

來源：pydantic-ai 的 RunContext 建議：建立 config/dependencies.yaml 集中管理 API 端點與可注入依賴，Skill 模板從中讀取而非硬編碼

P1 — Guardrails 輸出驗證

來源：openai-agents-python 的 Guardrails 建議：在 quality-gate 引入輸出 schema 驗證，DONE_CERT 增加結構化格式檢查

P1 — Agent Handoff 語義化

來源：openai-agents-python 的 Handoff 建議：在 routing.yaml 定義 handoff 規則，標準化子 Agent 上下文傳遞格式

P1 — Evals 評估系統

來源：pydantic-ai 的 Evals 建議：建立 config/evals.yaml 定義品質指標，自動偵測品質回歸

P1 — 可組合工作流

來源：mcp-agent 的 Workflow Patterns 建議：pipeline.yaml 支援工作流模式宣告，引入 evaluator-optimizer 自動精修

P2 — Agent as MCP Server

來源：mcp-agent 建議：長期願景，將核心 Skill 包裝為 MCP Server 實現跨專案共用

P2 — Human-in-the-Loop 審批

來源：pydantic-ai 建議：hook-rules.yaml 新增 action: prompt 選項，高風險操作透過 ntfy 發送審批請求
