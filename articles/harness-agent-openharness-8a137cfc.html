Harness Agent 深度研究報告

本筆記摘要自 docs/research/harness-agent-architecture-workflow_20260320.md。

核心結論

  • Harness 是以單機程序為主的 coding agent runtime，而非典型分散式 control plane。
  • 核心路徑是 CLI/SDK -> core engine -> loop -> provider/tools/sub-agents -> session/context persistence。
  • 已知模組包含 core、providers、tools、agents、hooks、mcp、skills、memory、permissions、ui、eval、cli。
  • 外部通訊主要是 provider API、MCP server、shell 與本機檔案系統；未見 gRPC 或 message queue 證據。
  • 部署型態目前以單機 CLI/SDK/CI runner 為主，官方未提供成熟 Docker/Kubernetes 工件。

本機驗證限制

  • git clone https://github.com/AgentBoardTT/openharness.git 失敗，原因是本機對 GitHub 443 連線受限。
  • agent -p 實測回傳 [internal]。
  • 本機未安裝 harness、plantuml、java，且 Docker daemon 未啟動。
