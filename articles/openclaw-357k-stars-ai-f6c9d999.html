專案概述

OpenClaw 是 2026 年 GitHub 上成長最快的開源專案，從 2026 年 1 月底的 9,000 stars 爆發至目前的 357,000+ stars，成為 GitHub 史上增長速度最快的開源專案之一。它是一個 MIT 授權的自託管個人 AI 助手平臺，核心理念是「在你自己的設備上運行 AI，透過你已經在用的通訊軟體互動」。

OpenClaw 解決的核心問題是：使用者需要一個統一的 AI 入口，能跨越 WhatsApp、Telegram、Slack、Discord、iMessage 等 24+ 個通訊平臺，同時保持資料在本地、不依賴雲端服務 GitHub README。

技術架構

Gateway 控制面

OpenClaw 的核心抽象是 Gateway — 一個持久化的 WebSocket 控制面，運行在 ws://127.0.0.1:18789，負責 GitHub README DigitalOcean Guide：

  • Session 管理與路由：每個對話是獨立 session，支援多 agent 路由
  • Channel 連接與事件串流：統一抽象 24+ 通訊平臺的訊息收發
  • Tool 執行與可用性管理：Skills 系統的工具調度
  • 配置與 Presence 追蹤：即時狀態同步

分散式架構

  • Gateway：管理控制邏輯與工具執行
  • Device Nodes（iOS/Android/macOS）：透過 node.invoke RPC 處理裝置本地動作
  • Clients：透過 WebSocket 連接進行聊天、控制與監控

Model Router（BYOK）

統一的 Bring Your Own Key 介面，支援 OpenAI、Anthropic、Google、DeepSeek 及本地模型（Ollama）。使用者自帶 API key，OpenClaw 不經手任何模型呼叫費用 OneClaw Guide。

功能特色（與同類工具差異化）

1. 多通道整合（24+ 平臺）

這是 OpenClaw 最核心的差異化優勢。透過原生整合（非 webhook 橋接），支援：

  • WhatsApp（Baileys）、Telegram（grammY）、Slack（Bolt）、Discord（discord.js）
  • iMessage（BlueBubbles）、Signal、Matrix、Microsoft Teams
  • Google Chat、LINE、WeChat、IRC 等

相比之下，Hermes 專注推理演化，NemoClaw 專注企業安全，ZeroClaw 追求零開銷 — 都沒有 OpenClaw 的通道廣度 Vellum Blog Cognio。

2. Skills 三層系統

  • Bundled Skills：隨 OpenClaw 安裝的核心技能
  • Managed Skills：官方維護，首次使用時自動下載
  • Workspace Skills：使用者自定義，存放在工作區目錄
  • ClawHub：Skills 市集，15,000+ 社群貢獻的 Skills，agent 推理時可自動搜尋安裝

每個 Skill 以目錄形式存在，內含 SKILL.md（元資料與使用說明）— 這與 daily-digest-prompt 專案的 Skill 架構高度相似 GitHub README。

3. SOUL.md 個性化配置

每次 agent 啟動時先讀取 SOUL.md — 一個 Markdown 配置檔定義 AI 的個性、價值觀和溝通風格。這是一種「配置即人格」的設計哲學 WenHaoFree Blog。

4. Companion Apps

  • macOS：Menu bar 控制、語音喚醒、Talk Mode
  • iOS：Canvas 渲染、語音觸發、Bonjour 配對
  • Android：連續語音、螢幕錄製、SMS/聯絡人存取

使用方式（快速上手）

安裝


npm install -g openclaw@latest
openclaw onboard --install-daemon

常用指令


# 啟動 Gateway
openclaw gateway --port 18789 --verbose

# 發送訊息
openclaw message send --to +1234567890 --message "Hello"

# Agent 模式
openclaw agent --message "研究 AI 趨勢" --thinking high

聊天指令

在任何支援的通道中可用：

  • /status — Session 指標（模型、token、成本）
  • /new — 清除 session 上下文
  • /think — 調整推理深度（minimal/low/medium/high/xhigh）

潛在應用（與 daily-digest-prompt 的連結）

  1. Skills 架構借鏡：OpenClaw 的三層 Skill 系統（bundled/managed/workspace）+ ClawHub 市集概念，可啟發本專案的 Skill 分發機制
  2. SOUL.md vs CLAUDE.md：兩者都用 Markdown 配置 agent 行為，但 SOUL.md 更偏向「人格定義」，CLAUDE.md 偏向「規則約束」— 可考慮融合
  3. Gateway 模式：OpenClaw 的 WebSocket 控制面可作為未來 daily-digest-prompt 多 agent 協調的參考架構
  4. 多通道通知：目前本專案僅用 ntfy.sh，OpenClaw 的 channel adapter 模式可啟發支援 Telegram/Discord 等通道
  5. Device Node 概念：手機端的 RPC 呼叫（拍照、讀取聯絡人等）是目前本專案未涉及的能力擴展方向

優缺點評估

優點

  • 通道覆蓋最廣：24+ 平臺原生整合，無需第三方橋接
  • 完全自託管：資料不離開使用者設備，隱私保護強
  • BYOK 模式：不綁定特定 AI 供應商，成本透明
  • Skills 生態系豐富：ClawHub 15,000+ 社群 Skills
  • MIT 授權：無商業限制
  • 跨平臺 Companion Apps：macOS/iOS/Android 原生應用

缺點

  • 安全模型是操作者信任制：AI 模型對工具和服務有廣泛存取權，無程序級隔離 — 這是架構性的取捨 Vellum Blog FountainCity
  • Prompt Injection 被排除在安全修復範圍外：SECURITY.md 明確列出，截至 2026-04 有 469 個開放的安全 issue
  • 無內建長期記憶架構：Context 僅存在 session state，沒有自動建構使用者模型的機制
  • Node.js 24 依賴：對部分環境來說版本要求較高
  • Windows 僅支援 WSL2：非原生支援，對 Windows 使用者不友善
