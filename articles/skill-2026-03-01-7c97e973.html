chatroom-query 審查結果

Frontmatter 規範性（2/5 -> 修正後 5/5）

  • name 與目錄名一致
  • description 含 "Use when:" 觸發條件
  • 修正 1：version: 1.1.0 -> version: "1.1.0"（加引號，統一字串格式）
  • 修正 2：allowed-tools: [Read, Bash, Write] -> allowed-tools: Bash, Read, Write（統一逗號分隔格式與順序）
  • 修正 3：depends-on: api-cache -> depends-on:\n - api-cache（統一 YAML 陣列格式）
  • cache-ttl: 15min 合理（與 cache-policy.yaml 一致）

內容品質（4/5 -> 修正後 5/5）

  • 端點清單完整（7 個端點，含健康檢查、CRUD、排程查詢）
  • 每個端點附帶可執行的 curl 範例
  • 快取策略段落完整，與 cache-policy.yaml 配置一致
  • 錯誤處理表格覆蓋 5 種失敗情境
  • Gun.js 資料結構說明提供必要背景
  • 修正 4：標題 v1.0.0 -> v1.1.0（與 frontmatter 版本號一致）

路由一致性（5/5）

  • SKILL_INDEX.md 觸發關鍵字 9 個，完整覆蓋
  • routing.yaml 有 ^Chat系統 標籤映射，template 指向 chatroom-task.md
  • 子 Agent 模板 templates/sub-agent/chatroom-task.md 存在且正確引用此 Skill
  • frequency-limits.yaml 有 chatroom_optimize 自動任務（daily_limit: 2）
  • cache-policy.yaml 有 chatroom 條目（TTL 15min，降級 120min）

修正摘要

| 項目 | 修正前 | 修正後 |

|------|--------|--------|

| version 格式 | 1.1.0（無引號） | "1.1.0"（加引號） |

| allowed-tools 格式 | [Read, Bash, Write] | Bash, Read, Write |

| depends-on 格式 | api-cache（純字串） | - api-cache（陣列） |

| 標題版本 | v1.0.0 | v1.1.0 |
