workflow-forge：自動任務結果 JSON Schema 統一化

執行日期

2026-03-17

背景問題

28 個自動任務（todoist-auto-*.md）各自定義結果 JSON 格式，缺少統一 Schema，易產生格式漂移與不一致。

缺口分析：

  • 影響範圍：所有 prompts/team/todoist-auto-*.md（28 個）
  • 優先級：P0（影響範圍 40 分 + 可執行性 35 分 + 信號強度 20 分 = 95 分）
  • 缺口類型：output_schema（輸出格式規範）

產出物

檔案路徑：config/schemas/results-auto-task-schema.json

Schema 特性：

  • 基於 JSON Schema Draft 7
  • 必填欄位：task_key、status
  • 二選一規則：agent（推薦）或 task_type（舊格式相容）
  • 支援多種選填欄位：artifact、artifacts、summary、kb_imported 等
  • 包容現有 prompt 格式變化（agent vs task_type、artifact vs artifacts）

主要欄位：

  • agent：格式 todoist-auto-{task_key}（推薦）
  • task_type：固定 "auto"（舊格式相容）
  • task_key：對應 frequency-limits.yaml 鍵名（必須底線，禁連字號）
  • status：success | partial | failed | format_failed
  • artifact：單一產出物（物件）
  • artifacts：多個產出物（陣列）
  • summary：一句話摘要
  • 任務專用欄位：repairs_attempted（self-heal）、alerts_count（system-insight）

整合狀態

✅ 已整合：

  • 寫入 config/schemas/results-auto-task-schema.json
  • 更新 config/README.md（配置文件速查 + 配置階層）
  • 登記至 context/workflow-forge-registry.json

驗證結果

✅ JSON Schema 格式合法（Draft 7）

✅ 與現有 3 個 prompt 欄位對齊（workflow_forge、self_heal、system_insight）

✅ 包容 agent / task_type 雙格式

✅ 包容 artifact / artifacts 雙格式

價值與影響

  1. 格式一致性：28 個自動任務結果 JSON 有統一規範可遵循
  2. 格式漂移偵測：可用此 Schema 驗證 results/*.json 是否符合規範
  3. 文件化：新增 prompt 時可參考此 Schema 定義必填欄位
  4. 向下相容：保留 task_type 欄位，既有 prompt 無需立即修改

後續建議

  1. P1：在 run-todoist-agent-team.ps1 Phase 3 加入 Schema 驗證（用 jsonschema 驗證 results/*.json）
  2. P2：逐步統一所有 prompt 改用 agent 欄位（淘汰 task_type）
  3. P2：擴充 Schema 支援其他 Agent 類型（如 daily-digest、system-audit）
