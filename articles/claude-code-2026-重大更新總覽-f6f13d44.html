Claude Code 2026 重大更新總覽

Claude Code 在 2026 年迎來多項重大更新，從 Agent SDK 重新命名、自訂狀態列、MCP Tool Search 到 Agent Teams 等功能，大幅提升開發者體驗與 Agent 開發能力。

核心更新

1. Claude Agent SDK（原 Claude Code SDK）

SDK 正式更名為 Claude Agent SDK，提供 Python 與 TypeScript 雙語言支援。核心設計哲學是「給 Agent 一台電腦」，讓 AI 能像人類一樣操作檔案系統、執行命令、搜尋網路。

安裝方式：

# Python
pip install claude-agent-sdk

# TypeScript
npm install @anthropic-ai/claude-agent-sdk

基本使用：

from claude_agent_sdk import query, ClaudeAgentOptions

async for message in query(
    prompt="Find and fix the bug in auth.py",
    options=ClaudeAgentOptions(allowed_tools=["Read", "Edit", "Bash"]),
):
    print(message)

2. 內建工具集

工具功能
Read讀取工作目錄內任意檔案
Write建立新檔案
Edit精確編輯現有檔案
Bash執行終端命令、腳本、git 操作
Glob以模式尋找檔案
Grep以正則搜尋檔案內容
WebSearch搜尋網路取得即時資訊
WebFetch擷取並解析網頁內容
AskUserQuestion向使用者提問澄清問題

3. Hooks 機制

在 Agent 生命週期的關鍵點執行自訂程式碼，可用於驗證、記錄、阻擋或轉換 Agent 行為。

可用 Hooks：

  • PreToolUse / PostToolUse
  • Stop
  • SessionStart / SessionEnd
  • UserPromptSubmit

4. Subagents 子代理

可生成專門化的子代理處理聚焦的子任務，主代理委派工作，子代理回報結果。支援平行執行與隔離的 context 管理。

5. MCP 整合

透過 Model Context Protocol 連接外部系統：資料庫、瀏覽器、API 等數百種整合。

2026 新功能亮點

Agent Teams（實驗性）

多代理協作的研究預覽功能，需設定環境變數：

CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1

自訂狀態列（2026/02/11）

透過 /statusline 指令自訂顯示當前模型、目錄、剩餘 context、成本估算等資訊。

Output Styles（2026/02/11）

透過 /config 設定預定義風格（explanatory、learning）或自訂風格。

MCP Tool Search（2026/01）

「Lazy Loading」工具定義，僅在需要時動態載入。內部測試 token 消耗從 ~134k 降至 ~5k。

LSP 工具

新增 Language Server Protocol 工具，支援 go-to-definition、find references、hover documentation 等程式碼智能功能。

PDF 處理改進

  • Read 工具新增 pages 參數，可指定頁面範圍
  • 大型 PDF（>10 頁）@ 提及時回傳輕量參照而非內聯 context

Session Teleportation

透過 /teleport 將 session 無縫移轉到 claude.ai/code。

Apple Xcode 整合（2026/02/03）

Xcode 26.3 原生整合 Claude Agent SDK，開發者可直接在 IDE 內使用 Claude 的完整能力。

認證方式

支援多種雲端供應商：

  • Anthropic API：ANTHROPIC_API_KEY
  • Amazon Bedrock：CLAUDE_CODE_USE_BEDROCK=1
  • Google Vertex AI：CLAUDE_CODE_USE_VERTEX=1
  • Microsoft Azure：CLAUDE_CODE_USE_FOUNDRY=1
