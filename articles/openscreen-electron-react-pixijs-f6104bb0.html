一、專案概述

OpenScreen 是一款免費、開源的桌面螢幕錄製與展示影片製作工具，定位為 Screen Studio（付費訂閱制，年費約 $89）的開源替代品。由開發者 Siddharth Vaddem 創建，採用 MIT 授權，允許個人與商業使用，無浮水印、無訂閱費。

核心定位：「如果 Screen Studio 是豪華轎車，OpenScreen 就是可靠的掀背車——帶你到達目的地，不用付租金。」它瞄準 80% 的使用場景，以實用性取代高級感。

二、技術架構分析

2.1 技術棧

| 層級 | 技術 | 用途 |

|------|------|------|

| 桌面框架 | Electron | 跨平台桌面應用包裝 |

| 前端框架 | React + TypeScript | UI 介面與狀態管理 |

| 建置工具 | Vite | 快速開發與建置 |

| 圖形引擎 | PixiJS | 視覺效果渲染（縮放、模糊、動畫） |

| 時間軸 | dnd-timeline | 拖拽式時間軸編輯 |

| 樣式 | Tailwind CSS + PostCSS | 響應式 UI 樣式 |

| 測試 | Playwright + Vitest | E2E 與單元測試 |

2.2 架構亮點

  1. PixiJS 圖形引擎：選用 PixiJS 而非 Canvas 2D 或 WebGL 裸寫，兼顧效能與開發效率。PixiJS 的 GPU 加速渲染使縮放動畫、模糊效果等視覺處理流暢運行。
  1. Electron 跨平台策略：一套程式碼支援 macOS/Windows/Linux 三平台。macOS 需 13+ 版本方可擷取系統音訊（利用 ScreenCaptureKit），Linux 依賴 PipeWire，Windows 原生支援。
  1. Vite 建置優化：相較於傳統 Webpack，Vite 的 HMR（熱模組替換）大幅提升開發體驗，特別適合 Electron + React 的組合。

三、核心功能

3.1 錄製功能

  • 全螢幕或視窗錄製：靈活選擇錄製範圍
  • 雙音訊擷取：麥克風 + 系統音訊同步錄製
  • 網路攝影機支援：v1.3.0 新增，支援垂直模式與畫中畫（PiP）

3.2 編輯功能

  • 自動/手動縮放效果：可自訂深度層級，是與 Screen Studio 最相似的「招牌功能」
  • 影片裁切與修剪：隱藏敏感區域、精確剪輯
  • 分段速度調整：不同片段可設不同播放速度
  • 標註工具：文字、箭頭、圖片疊加
  • 動態模糊：過場平滑效果

3.3 匯出功能

  • 多種長寬比：16:9、4:3、1:1、9:16 等
  • 多解析度：自訂輸出解析度
  • 背景客製：桌布、純色、漸層、自訂圖片

八、對 Daily-Digest-Prompt 系統的啟發

  1. PixiJS + Electron 組合模式：若未來需要視覺化儀表板或動態報表，此架構提供參考。GPU 加速的 Canvas 渲染適合即時資料視覺化。
  1. 開源替代商業工具的策略：OpenScreen 的成功印證「瞄準 80% 需求、MIT 授權、跨平台」的開源策略可快速獲得社群認可。
  1. Vite + React + TypeScript 標準化組合：此技術棧已成為 2026 年前端開發的事實標準，值得在工具開發中採用。
  1. 版本管理與多語系：v1.3.0 的 i18n 支援（中/英/西）展示了開源專案國際化的最佳實踐。

---

九、潛在風險與限制

  1. Electron 效能開銷：相較於原生 Swift（Screen Studio），Electron 的記憶體佔用與啟動速度仍有差距
  2. Beta 階段：開發者自述可能存在 Bug，尚未達生產級穩定
  3. macOS 音訊限制：macOS 12 以下不支援系統音訊錄製
  4. 單人維護風險：目前主要由一位開發者維護，長期可持續性待觀察
