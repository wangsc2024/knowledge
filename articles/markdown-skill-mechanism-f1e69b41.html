CommonMark AST 解析管線

Markdown 解析三階段：

  1. 區塊解析（Block Parsing）：逐行掃描，辨識容器區塊（引用、清單可嵌套）和葉子區塊（段落、標題、程式碼區塊不可嵌套）
  2. 行內解析（Inline Parsing）：在葉子區塊內，使用定界符堆疊演算法配對 */_ 等標記。優先級：行內程式碼 > 自動連結 > HTML > 連結 > 強調 > 換行
  3. AST 輸出：Document → Heading/Paragraph/List → Text/Strong/Emphasis 等節點樹

關鍵機制洞察

定界符堆疊（Delimiter Stack）

  • 掃描到 * 或 _ 時推入堆疊
  • 遇到匹配的關閉定界符時彈出配對
  • 單組=斜體，雙組=粗體，三組=粗斜體
  • __ 有 intraword 側位規則限制（foo__bar__baz 不加粗），** 無此限制

區塊辨識規則

  • ATX 標題：行首 # + 空格（#標題 無空格不算）
  • 縮排程式碼：4 空格縮排（易與清單衝突，建議用圍欄式）
  • 分隔線：3+ 個 -/*/_，與 Setext 標題和 YAML front matter 的 --- 需靠空行區分

33 種語法分類

CommonMark 標準（20 種）

ATX 標題、Setext 標題、段落、換行、粗體、斜體、粗斜體、引用區塊、無序清單、有序清單、行內程式碼、程式碼區塊、連結、圖片、分隔線、跳脫字元、HTML 內嵌、自動連結、參考式連結、縮排程式碼

GFM 擴展（7 種）

刪除線、任務清單、表格、裸 URL 自動連結、Emoji :name:、告警區塊 [!NOTE]、YAML Front Matter

第三方擴展（6 種）

腳註、定義清單、上標/下標、高亮 ==、數學公式 $...$、Mermaid 圖表

編輯 Skill 要點

  1. 一行一句（Semantic Line Breaks）：減少 Git 合併衝突
  2. 表格不對齊 |：改動最小化
  3. 批次編輯用正則：如連結路徑替換 \]\(https://.../(.*?)\) → ](.\/\$1)
  4. VS Code 必裝擴充：Markdown All in One（TOC）、markdownlint、Markdown Table Formatter

總結 Skill 要點

  1. TOC 生成：VS Code 「Markdown All in One」一鍵生成，或 Python 腳本 slugify + 遍歷標題
  2. 大綱擷取：遍歷標題行，跳過程式碼區塊中的 #
  3. 摘要標記： 註釋框定摘要範圍
  4. GitHub 錨點規則：小寫 → 移除標點 → 空格變 - → 重複加 -1 後綴
