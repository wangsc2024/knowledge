執行摘要

近三天知識庫新增的非佛學內容呈現出一個核心趨勢：AI Agent 正從「能力展示」進入「工程化自治」階段。三條獨立的知識脈絡——自治度評測框架、Harness 約束工程、互通協定標準化——正在收斂為一套完整的系統成熟度模型。本報告提出「AHIP 成熟度模型」（Autonomy-Harness-Interoperability-Production），將散落的知識點整合為可操作的工程框架，並針對 daily-digest-prompt 系統提出六項具體優化建議。

一、三大收斂模式分析

模式 1：自治度評測 — 從 Demo 指標到生產 KPI

Anthopic 最新發表的自治度量化研究揭示了評測思維的根本轉變：

| 維度 | Demo 時代 | 生產時代（2026） |

|------|----------|------------------|

| 核心指標 | 任務完成率 | 自治度分數（1-10）x 風險等級矩陣 |

| 時間觀測 | 單次延遲 | 轉向持續時間分布（P50=45s, P99.9=42min） |

| 人類介入 | 視為失敗 | 視為自校準信號（複雜任務 Agent 主動停止 > 人為中斷 2 倍） |

| 風險量化 | 二元（安全/不安全） | 連續光譜（0.8% 不可逆操作的精確追蹤） |

關鍵洞見：「自治度是模型、使用者與產品三方共構的」，靜態 pre-deployment 測試無法預測真實行為，必須搭配 post-deployment 觀測基礎設施。

模式 2：Harness 約束工程 — 約束即加速器

LangChain Deep Agents Terminal Bench 2.0 從 52.8% 提升至 66.5%，純靠系統改善無需模型升級，驗證了反直覺命題：約束收窄解空間，Agent 反而更快收斂到正確解。

Harness 四大組件：

  1. 架構約束：模組邊界、穩定資料結構、領域規則
  2. 回饋迴路：錯誤追蹤、模式聚類、修正回注
  3. 驗證守衛：確定性工具 + AI 交叉審查
  4. 生命週期管理：標準化腳手架、CI/CD、可觀測性

OpenAI 2026 案例研究：Harness 導入後 Agent 工作流可靠性提升 2-5 倍。

模式 3：互通協定 — 從孤島到可組合架構

  • MCP（Model Context Protocol）：Agent 到工具的 USB-C，三原語（tools/resources/prompts）
  • A2A（Agent-to-Agent）：Agent 間通訊匯流排，Agent Card 描述能力元資料
  • ACP 已併入 A2A（2025-08），Linux Foundation AAIF 成為共同家園

分層模型：A2A 處理 Agent 間調度（who does what），MCP 處理能力發現（how to do it）。Gartner 預測 2026 年 40% 企業應用將包含任務專用 AI Agent。

二、AHIP 成熟度模型 — 跨模式整合框架

| 等級 | 名稱 | 自治度 | Harness | 互通性 | 典型特徵 |

|------|------|--------|---------|--------|----------|

| L1 | Scripted | 1-3 | 無 | 無 | 硬編碼流程，人工監督每步 |

| L2 | Guarded | 4-5 | 基礎守衛 | 單協定 | Hook 攔截、靜態約束，MCP 工具串接 |

| L3 | Adaptive | 6-7 | 閉環回饋 | 雙協定 | OODA 自修正、動態約束演化，A2A+MCP |

| L4 | Autonomous | 8-10 | 自演化 | 完整互通 | 自主發現瓶頸並修正，跨系統 Agent 協作 |

daily-digest-prompt 系統現狀評估：L2.5 到 L3 過渡中

  • Hook 機器強制層（L2 達成）
  • OODA 閉環已建立（L3 部分達成）
  • 約束演化仍需人工觸發
  • 互通性僅限 MCP，無 A2A 級 Agent 發現
  • 自治度量化尚未建立

三、認知科學視角的補充啟發

  • 決策疲勞模型（Baumeister）：Token 預算是認知資源耗竭的數位類比，Harness 約束等效於減少認知負荷
  • 論證圖示法：可作為 Agent 決策軌跡可視化的理論基礎
  • 認知負荷理論：max_tasks_per_run=3 暗合 Miller's Law，Agent 有效工作記憶受 context window 限制

四、系統優化建議（六項）

建議 1：建立 Agent 自治度量化儀表板

在 benchmark.yaml 新增 autonomy_metrics — human_interrupt_rate、self_stop_rate、irreversible_action_pct、risk_autonomy_score。

建議 2：約束演化管線自動化

引入約束置信度：連續 5 次人工審查無回退自動提升授權（Level 0 建議 到 Level 1 自動+通知 到 Level 2 靜默+週報）。

建議 3：Agent Card 能力描述標準化

為每個 Skill 生成 A2A 相容的 Agent Card JSON，存於 context/agent-cards/。

建議 4：決策軌跡可視化

在 trace_analyzer 輸出增加 Mermaid 決策圖（claim-reason-evidence 三層結構）。

建議 5：Token 預算與認知負荷對齊

依任務類型設定差異化 Token 預算，預算接近閾值時自動縮減 max_tasks_per_run。

建議 6：Harness 回饋迴路閉環速度提升

引入 micro-heal 快速回饋通道——連續 3 次相同 error pattern 直接觸發修正。
