  • - 通教是天台「化法四教」中承上啟下之一環：就所詮，說即空無生之四真諦（無生四諦），使三乘同觀無生而共學；就所被，以菩薩為正機、二乘為傍機，故名「通」——通前同於藏教所證偏真，通後可接入別教但中與圓教即具。《教觀綱宗》全書以「教觀相資」為骨，通教正對應「體假即空」的觀門（與藏教「析假入空」相對），在漸教中多屬般若方等段之共般若層次。五時上屬般若時（亦與方等帶通相涉），化儀上則隨頓漸秘密不定之施設而轉，不可將「通教」誤等同於「漸教」或「不定教」。

  • ### 1. Tool RAG 化 — Progressive Disclosure 三層載入 - Discovery：僅名稱+一句話描述（100-300 tokens） - Activation：任務相関工具完整指令（2,000-5,000 tokens） - Execution：完整 schema 與範例 - 90+ 工具 >50,000 tokens，Progressive Disclosure 可大幅降低前置成本 - 本系統 Gap：Agent Pool coordinator.py 全量注入，routing.yaml 靜態路由未達 RAG 化
  • ### 2. Context Routing 混合策略 - Hybrid = rule-based（O(1)）+ LLM classifier（語義） - Pre-Route 輕量 metadata 分析 → SkewRoute 檢索分數分佈代理 query hardness - 本系統：llm_classifier.py + llm_router.yaml 已是 hybrid，但研究子類型粒度可細化
  • ### 3. Context Compressor 三段觸發（LangChain Deep Agents） - Tool result > 20,000 tokens → filesystem offload + 10 行 preview - Context > 85% capacity → older args offload - Offload 仍不足 → LLM 結構化摘要（session intent + artifacts + next steps） - 壓力測試建議從 10-20% capacity 觸發
  • ### 4. Active Context Compression（arxiv 2601.07190） - 22.7% token 節省（14.9M → 11.5M tokens），精確度維持（60%） - Focus Agent：slime mold 啟發，agent 主動決定壓縮時機 - Knowledge block 保留學習成果，raw history 逐步清除
  • ### 5. Context Drift 優先於 Token 耗盡 - 65% 企業 AI 失敗源於 drift/memory loss - 防 Drift：保留 error traces + structured summaries（三部分）+ needle-in-haystack 驗證
