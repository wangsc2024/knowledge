GitHub靈感蒐集系統改進Hook/Guardguardrails

GitHub Scout：Hook/Guard 研究筆記（2026-07-02）

專案觀測摘要

  • NVIDIA-NeMo/Guardrails（6587★，REST 觀測 2026-07-02，pushed_at=2026-07-01）：LLM 對話系統的可程式化 guardrails 工具，YAML+Colang 宣告式設定，分 input/dialog/execution/output 四類 rails；default_mode=deny，設定層即可 rollback。
  • crewAIInc/crewAI（54719★，REST 觀測 2026-07-02，pushed_at=2026-07-01）：issue #4877 提出 GuardrailProvider 介面，BeforeToolCallHook 可回傳 False 阻擋工具呼叫，ToolCallHookContext 含 tool_name/tool_input/agent/task/crew，fail_closed 預設 True。
  • pre-commit/pre-commit-hooks（6604★，REST 觀測 2026-07-02，pushed_at=2026-05-19）：commit-time 同步檔案內容檢查（detect-private-key、check-added-large-files 等），與 agent runtime 無關但攔截時機（commit-time）值得借鑑。
  • 另觀測 AxonLabsDev/BodAIGuard（stars=0，REST 觀測 2026-07-02）：YAML-driven fail-closed universal agent guardrail，但未達篩選門檻（>500 stars），僅供模式參考，不計入證據鏈。

改進提案

  1. P1｜commit-time staged-diff secret 同步攔截（借鑑 pre-commit-hooks detect-private-key）：現有 scripts/auto-task-secret_scan_daily.ps1 為每日 01:30 排程事後掃描（最長 24h 曝險窗口）。提案在 scripts/git-hooks/pre-commit 新增 staged-diff secret pattern 同步掃描，commit 前即阻擋，曝險窗口收斂為 0。risk_level=high（目標檔案不在本任務步驟9允許直接修改範圍），已輸出高風險計畫。
  2. P2｜review-policy 新增 hook_guard_agent_guardrails 結構化欄位（借鑑 NeMo rails 分類 + crewAI GuardrailProvider 結構化 context）：config/github-scout-review-policy.yaml 新增 topic_overrides.hook_guard_agent_guardrails（enforcement_point/fail_mode/rail_categories 三面向），供未來同類研究對齊結構化欄位，降低漂移。risk_level=low，本輪已直接落實。

審查結論

  • 提案 1：feasibility_score=8、stability_score=6（regex 邏輯完全複用既有 pattern，技術可行；但目標檔案屬 commit-blocking 關鍵路徑且超出本任務允許直接修改範圍，需人工審核）→ status=plan_ready
  • 提案 2：feasibility_score=9、stability_score=9（純新增 YAML 欄位，無邏輯變更，風險極低）→ status=approved，已落實

已落實項目

  • config/github-scout-review-policy.yaml 新增 topic_overrides.hook_guard_agent_guardrails（enforcement_point/fail_mode/rail_categories 結構化欄位定義）
