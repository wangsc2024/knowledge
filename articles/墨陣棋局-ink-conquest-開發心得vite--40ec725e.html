墨陣棋局 Ink Conquest 開發心得

專案概述

使用 Vite + 純 Canvas 2D 開發的水墨風格即時策略領地控制遊戲。部署於 Cloudflare Pages。

核心技術

1. Vite 構建

  • 使用 npm create vite@latest --template vanilla
  • build 輸出僅 26.82KB（gzip 9.07KB）
  • vite.config.js 設定 base: './' 確保相對路徑部署
  • sync-games.ps1 自動偵測 dist/ 目錄使用建置輸出

2. 回合制策略玩法

  • 9x9 格盤，程序生成山脈障礙+河流地形+禪字特殊格
  • 墨力資源管理：落子消耗 20，蔓延消耗 5，每回合恢復 15
  • 6 種禪字能力：空(清除)、定(凍結)、慧(墨力)、悲(轉化)、施(擴展)、忍(護盾)
  • 25 回合制，佔領最多領地者獲勝

3. AI 對手設計

  • 三級難度：初心(高隨機)、修行(平衡)、覺悟(深度評估)
  • 位置評估函數：禪字價值 + 搶奪加分 + 連片效益 + 中心位置 + 蔓延潛力
  • 高難度額外考慮威脅防守

4. 模組架構（6 個 JS 模組）

  • constants.js：常數定義
  • board.js：棋盤邏輯（程序生成 + 墨跡蔓延 + 禪字效果）
  • ai.js：AI 對手（多策略評估）
  • renderer.js：Canvas 2D 渲染（水墨風格 + HUD + 動畫）
  • particles.js：粒子系統（物件池 200 上限）
  • audio.js：Web Audio API 8-bit 音效合成
  • game.js：主控制器（狀態機 + 遊戲循環 + 存檔）

5. 品質標準達成

  • requestAnimationFrame 遊戲循環（非 setInterval）
  • 鍵盤 + 滑鼠 + 觸控三種輸入（含防雙觸發機制）
  • 完整狀態機：MENU → PLAYING → AI_TURN → SPREADING → ZEN_EFFECT → GAME_OVER / PAUSED
  • 粒子物件池管理，無記憶體洩漏
  • LocalStorage 存檔/讀檔
  • DPR 適配響應式設計

6. 知識庫筆記活用

  • 參考 Canvas 2D 效能優化（alpha:false 建議）
  • 參考 Game & Watch 極簡設計哲學：成熟技術做全新體驗
  • 參考存檔系統設計：LocalStorage 狀態序列化
  • 參考 Pac-Man 開發經驗：Grid-based 移動 + 粒子系統

創意亮點

  • 「墨跡蔓延」機制：落子後墨跡自動向四方擴散，河流格蔓延機率減半
  • 禪字特殊能力系統：6 種能力改變戰局
  • 程序生成棋盤：每局山脈、河流、禪字位置不同
  • 墨跡衰退機制：墨跡隨時間減弱，長期不補強會失去領地

部署流程

  1. npm run build 產生 dist/
  2. sync-games.ps1 -Full 自動偵測 dist/ 並部署
  3. gameMetadata.js 補充遊戲條目
  4. Cloudflare Pages 自動從 GitHub 部署
