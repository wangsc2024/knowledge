Claude Code Plugin 系統與第三方擴展生態完整指南

主題概述

Claude Code Plugin 系統是 Anthropic 推出的擴展機制（2026 年進入 Public Beta），允許開發者將 Skills、Agents、Hooks、MCP Servers 和 LSP Servers 打包成可安裝、可分享的插件。截至 2026 年 2 月，生態系已擁有 9,000+ 插件、43+ 社群 Marketplace、4,961 GitHub 儲存庫，安裝量最高的 Frontend Design 插件突破 96K 安裝次數。Plugin 系統代表 Anthropic 對 Claude Code 擴展性的核心策略方向。

一、Plugin 架構與目錄結構

1.1 核心組件

一個 Plugin 是一個獨立目錄，包含以下結構：


my-plugin/
├── .claude-plugin/
│   └── plugin.json          # 插件清單（必須）
├── commands/                 # Slash 指令（Markdown 檔案）
├── agents/                   # 自定義 Agent 定義
├── skills/                   # Agent Skills（含 SKILL.md）
├── hooks/
│   └── hooks.json            # 事件處理器
├── .mcp.json                 # MCP Server 配置
├── .lsp.json                 # LSP Server 配置
├── settings.json             # 預設設定
└── README.md                 # 文件說明

重要注意：commands/、agents/、skills/、hooks/ 必須放在 Plugin 根目錄，不可放在 .claude-plugin/ 內。只有 plugin.json 放在 .claude-plugin/ 內。

1.2 plugin.json 清單結構


{
  "name": "my-plugin",
  "description": "插件描述",
  "version": "1.0.0",
  "author": { "name": "Author Name" },
  "homepage": "https://example.com",
  "repository": "https://github.com/user/repo",
  "license": "MIT"
}

| 欄位 | 用途 |

|------|------|

| name | 唯一識別碼，同時作為 Skill 命名空間前綴（如 /my-plugin:hello） |

| description | 顯示於 Plugin Manager 的描述 |

| version | 遵循語意化版本（Semantic Versioning） |

| author | 選填，用於署名 |

二、五大擴展組件詳解

2.1 Skills（模型調用型）

Skills 是 Claude 自動辨識並使用的能力擴展。每個 Skill 是一個目錄，包含 SKILL.md 檔案：


---
name: code-review
description: Reviews code for best practices and potential issues.
---
When reviewing code, check for:
1. Code organization and structure
2. Error handling
3. Security concerns
  • 用 /plugin-name:skill-name 手動調用，或 Claude 根據上下文自動觸發
  • 支援 $ARGUMENTS 佔位符接收使用者輸入
  • 支援 disable-model-invocation: true 禁止自動觸發（僅手動）

2.2 Agents（專責子代理）

自定義 Agent 放在 agents/ 目錄，具有獨立的系統提示詞、工具限制和模型選擇。可透過 settings.json 的 agent 鍵將某個 Agent 設為主執行緒預設。

2.3 Hooks（系統事件處理器）

Hooks 是系統側的事件處理器，在 PreToolUse、PostToolUse、Stop 等事件觸發時執行。Plugin 的 hooks 設定放在 hooks/hooks.json（而非 .claude/settings.json）：


{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [{ "type": "command", "command": "jq -r '.tool_input.file_path' | xargs npm run lint:fix" }]
      }
    ]
  }
}

2.4 MCP Servers（外部工具整合）

透過 .mcp.json 配置 MCP Server，連接外部 API、資料庫、檔案系統等。搭配 MCP Tool Search 功能，可實現延遲載入，減少 95% 的 context 消耗。

2.5 LSP Servers（程式碼智慧分析）

新增的 LSP 支援讓 Claude 獲得即時程式碼智慧能力。透過 .lsp.json 配置語言伺服器：


{
  "go": {
    "command": "gopls",
    "args": ["serve"],
    "extensionToLanguage": { ".go": "go" }
  }
}

四、Plugin 開發最佳實踐

4.1 開發流程

  1. 建立目錄結構 → mkdir -p my-plugin/.claude-plugin
  2. 撰寫 plugin.json → 定義名稱、版本、描述
  3. 新增組件 → skills/、agents/、hooks/ 依需要添加
  4. 本地測試 → claude --plugin-dir ./my-plugin
  5. 版本管理 → 遵循 Semantic Versioning
  6. 發佈 → 推送到 GitHub Marketplace 儲存庫

4.2 獨立配置 vs Plugin 選擇指南

| 場景 | 建議 |

|------|------|

| 單一專案個人化 | 獨立配置（.claude/），Skill 名稱簡短如 /hello |

| 團隊共享工作流 | Plugin，Skill 名稱帶命名空間如 /my-plugin:hello |

| 跨專案復用 | Plugin |

| 快速實驗 | 先獨立配置，成熟後遷移為 Plugin |

| 社群分發 | Plugin + Marketplace |

4.3 從 .claude/ 遷移到 Plugin

| 獨立配置（.claude/） | Plugin 等效 |

|---|---|

| .claude/commands/ | plugin-name/commands/ |

| .claude/skills/ | plugin-name/skills/ |

| .claude/agents/ | plugin-name/agents/ |

| settings.json 中的 hooks | hooks/hooks.json |

| 手動複製分享 | /plugin install 一鍵安裝 |

4.4 settings.json 預設配置

Plugin 可附帶 settings.json 設定預設 Agent：


{ "agent": "security-reviewer" }

啟用此 Plugin 時，自動以 security-reviewer Agent 作為主執行緒。

4.5 Context 管理策略

Plugin 支援啟用/停用切換，讓使用者按需管理 context 使用量。搭配 MCP Tool Search 的延遲載入，有效控制 token 消耗。

五、生態系統分析與趨勢

5.1 爆發性成長

  • 推出不到一年即達 9,000+ 插件
  • 去中心化分發模型：無需審核流程、無需 build 步驟，直接從 GitHub 安裝
  • 社群 Marketplace 從 0 → 43+ 個，涵蓋 DevOps、前端、安全、文件等類別

5.2 Plugin 類別分佈

  • UI/UX 設計：Frontend Design（96K）、Figma MCP（18K）
  • 開發品質：Code Review（50K）、Security Guidance（25K）
  • 測試自動化：Playwright（28K）、Chrome DevTools（20K）
  • 文件與知識：Context7（71K）
  • 迭代工作流：Ralph Loop（57K）
  • 專案管理：Linear（9.5K）

5.3 技術趨勢

  1. LSP 整合：Plugin 可打包語言伺服器，提供即時程式碼分析
  2. MCP Tool Search：延遲載入 MCP 工具，解決 context 膨脹問題
  3. Agent 預設化：Plugin 可透過 settings.json 改變 Claude Code 預設行為
  4. Hooks 生態：自動化 lint、test、部署等工作流
