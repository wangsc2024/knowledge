Zvec — 向量資料庫的 SQLite

研究日期：2026-02-18 GitHub：https://github.com/alibaba/zvec Stars：4,400+ | Forks：232 | Contributors：11 | 最新版本：v0.2.0（2026-02-13） 授權：Apache-2.0

一、專案概述

Zvec 是阿里巴巴於 2026 年 2 月開源的嵌入式向量資料庫，定位為「向量資料庫領域的 SQLite」。它基於阿里巴巴久經考驗的 Proxima 向量搜尋引擎構建，以進程內函式庫（in-process library）的形式直接嵌入應用程式，無需獨立的伺服器或 RPC 層。

核心定位：為 RAG（檢索增強生成）、多模態搜尋、邊緣裝置 AI 提供生產級、低延遲、可擴展的相似性搜尋能力。

二、核心特性

2.1 嵌入式零依賴

  • 無需外部服務、無需網路通訊
  • 直接在 Python/Node.js 進程中運行
  • 適合：筆記型電腦、伺服器、CLI 工具、邊緣裝置、車載系統

2.2 向量型別支援

  • 稠密向量（Dense Vectors）：語義嵌入搜尋
  • 稀疏向量（Sparse Vectors）：關鍵字級匹配
  • 多向量查詢（Multi-Vector）：支援多通道語義檢索與語義+關鍵字混合檢索

2.3 索引策略

索引類型適用場景
Flat Index精確搜尋（黃金標準）
HNSW高效能近似最近鄰
IVF反向檔案索引，適合大規模資料集
Quantizationint8 量化壓縮，適合資源受限環境

2.4 混合搜尋

  • 標量-向量混合搜尋：標量篩選條件可下推到向量索引執行層，避免全表掃描
  • 內建重排器：支援加權融合（Weighted Fusion）與 RRF（Reciprocal Rank Fusion），自動融合排序，無需手動合併

2.5 完整 CRUD 與 Schema 演進

  • 完整的增刪改查操作
  • 支援 Schema 演進：索引策略可隨 metadata 與查詢模式變化而調整
  • 即時更新知識庫

與競品定位比較

類別代表Zvec 優勢
純索引函式庫FaissZvec 有完整資料庫功能（標量儲存、混合查詢、CRUD）
嵌入式方案DuckDB-VSSZvec 索引選擇更多、有量化壓縮、資源控制更強

嵌入式ChromaDBZvec 基於 C++ 高效能引擎，QPS 大幅領先

四、三層記憶體管理

  1. 串流寫入：以 64MB 區塊處理，避免一次載入所有資料
  2. 記憶體映射模式：enable_mmap=true 啟用按需分頁
  3. 硬上限：memory_limit_mb 明確限制記憶體預算

併發控制：可配置建索引與查詢的執行緒上限，防止 UI 降級。

五、安裝與使用

安裝

# Python (3.10-3.12)
pip install zvec

# Node.js
npm install @zvec/zvec

Python 使用範例

import zvec

# 開啟集合
collection = zvec.open_collection("my_docs", schema={
    "embedding": zvec.DenseVector(dim=768)
})

# 插入文件
collection.insert({"id": "doc1", "embedding": [0.1, 0.2, ...]})

# 向量查詢
results = collection.query(
    zvec.VectorQuery("embedding", vector=[0.1, 0.2, ...]),
    topk=10
)

支援平台

  • Linux x86_64
  • Linux ARM64
  • macOS ARM64（Apple Silicon）

六、應用場景

6.1 本地 RAG 系統

  • 在 PC 或手機上建立本地 RAG 助手，無需網路連線
  • 適合隱私敏感場景

6.2 邊緣 AI

  • 車載系統、IoT 裝置上的即時語義搜尋
  • 低延遲、低資源消耗

6.3 開發者工具

  • CLI 工具中嵌入向量搜尋
  • 快速原型開發，無需部署外部服務

6.4 多模態搜尋

  • 結合文字、圖片、音訊等多模態嵌入的統一檢索

七、技術架構洞察

底層技術

  • 語言：C++ 81.5% + SWIG 8.4% + Python 7.6%
  • 核心引擎：Proxima（阿里巴巴內部向量搜尋引擎）
  • 加速技術：多執行緒併發、記憶體佈局優化、SIMD 加速、CPU 預取

與 SQLite 類比的深層意義

  • SQLite 是嵌入式關聯資料庫的標竿 → Zvec 要成為嵌入式向量資料庫的標竿
  • 零設定、零依賴、進程內運行 → 降低向量搜尋的採用門檻
  • 適合從原型到生產的全生命週期

八、對 Daily-Digest-Prompt 專案的啟發

  1. 知識庫升級方向：目前使用 Qdrant 作為向量搜尋後端，Zvec 提供了輕量級替代方案
  2. 離線 RAG 可能性：Zvec 的嵌入式特性意味著可在無網路環境下運行 RAG
  3. 效能基準參考：8000+ QPS 的基準可作為向量搜尋效能評估的標竿
  4. 多向量檢索：稠密+稀疏向量混合搜尋可提升知識庫搜尋品質
