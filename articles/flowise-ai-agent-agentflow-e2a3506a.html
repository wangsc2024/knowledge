概述

Flowise 是一個以 TypeScript 開發的開源 AI Agent 視覺化建構平台。核心理念是「Build AI Agents, Visually」，讓技術與非技術人員都能透過拖放畫布，把向量資料庫、LLM、工具節點串接成可部署的 RAG Pipeline 或 Agent Workflow，而無需撰寫應用程式碼。

相較 n8n 的「通用自動化 + AI 疊加」定位，Flowise 從一開始就是 AI-first：每個節點都是 LangChain 生態系的直接視覺對應，最短 15 分鐘可跑起一個附有 RAG 的聊天機器人原型。2026 年 4 月發布的 v3.1.2 持續迭代 AgentFlow V2 架構，已成為 No-code AI 工作流領域的代表性標竿之一。

---

技術架構

Monorepo 結構（5 個 Package）

Flowise 採用 PNPM Workspace + Turborepo 管理的 Monorepo，按照 components → server → ui 的建構順序串接：

| Package | 技術 | 職責 |

|---------|------|------|

| packages/server | Express.js | REST API + 執行引擎 |

| packages/ui | React + ReactFlow | 拖放畫布前端 |

| packages/components | TypeScript | 1000+ AI 提供者整合 |

| packages/agentflow | React（模組化） | AgentFlow V2 有狀態工作流 UI |

| packages/api-documentation | Swagger | API 文件 |

語言組成：TypeScript 59.5%、JavaScript 28.6%，Runtime 需求 Node.js ≥18.15.0。

核心執行機制：NodesPool 動態插件系統

NodesPool 類別在伺服器啟動時，自動掃描並載入 flowise-components Package 中的所有節點實作（每個節點實作 INode 介面）。執行時依工作流圖的依賴順序，逐節點初始化、注入憑證、呼叫 AI 提供者，最終透過 SSE（Server-Sent Events）串流結果。這種設計讓新增自訂節點等同於新增一個 TypeScript 類別，無需修改執行引擎。

兩種工作流類型

Chatflow（DAG 型）：使用廣度優先遍歷（buildChatflow()），無狀態、單輪或多輪對話，適合 RAG Q&A、文件問答、客服機器人。

Agentflow（有狀態序列型）：使用 executeAgentFlow() 序列執行，支援共享狀態（$flow.state）、分支、迴圈，適合多步驟 Agent 任務、Human-in-the-Loop 流程。

---

工作流核心能力

AgentFlow V2（2026 核心新架構）

V2 是 Flowise 最重要的架構演進，放棄依賴外部框架（LangChain Agent），改為原生開發 14 種專用節點：

  • Start：工作流入口，初始化 $flow.state
  • LLM：呼叫語言模型
  • Agent：具備工具呼叫能力的 AI 代理
  • Tool：外部工具（API/函數/MCP）
  • Retriever：知識庫檢索
  • HTTP：REST API 呼叫
  • Condition：條件分支
  • Loop / Iteration：迴圈與迭代
  • Human Input：暫停等待人類輸入（HITL）
  • Direct Reply：直接回覆不經 LLM
  • Custom Function：JavaScript 自訂邏輯
  • Execute Flow：呼叫子 Chatflow/AgentFlow（子工作流）

$flow.state 共享狀態機制：工作流執行期間的 Key-Value 儲存庫，必須在 Start 節點初始化。所有節點可跨分支讀寫，實現跨步驟資料持久化——類似本系統的 state/*.json 模式，但在視覺層直接操作。

多 Agent 協作（Supervisor/Worker 模式）：Supervisor Agent 可分解任務並委派給多個 Worker Agent，每個 Agent 可存取完整對話歷史，是視覺化工具中少數支援原生多 Agent 協調的實作。

子工作流嵌入（Execute Flow Node）：可在當前工作流內呼叫另一個完整 Chatflow 或 AgentFlow，實現模組化重用。

---

AI 整合機制

MCP（Model Context Protocol）原生支援

AgentFlow V2 將 MCP 工具作為工作流的一等公民（Tool Node），而非只作為 Agent 的附屬工具。官方提供 GitHub MCP、Atlassian Jira MCP、Brave Search MCP 等整合。自訂 MCP Server 可直接接入 Tool Node，無需修改 Flowise 核心。

RAG Pipeline 設計

標準 RAG 建構路徑：Document Loader → Text Splitter → Embeddings → Vector Store → Conversational Retrieval QA Chain

支援的 Vector Store：Pinecone、Qdrant、Weaviate、Chroma 等。Document Loader 支援：PDF、DOCX、CSV、網頁、Notion Pages。

LLM 支援廣度

1000+ 節點覆蓋所有主流 LLM 提供者（OpenAI、Anthropic、Groq、Mistral、Ollama 本地模型等），通過憑證管理統一注入。

部署與維運

部署選項：本機 NPX（npx flowise start）、Docker（最乾淨生產部署，10 分鐘完成）、Docker Compose + PostgreSQL（生產級認證與資料持久化）、Queue Mode（Redis + BullMQ 水平擴展）、Flowise Cloud（官方托管）、Railway/Render/HuggingFace Spaces（第三方平台快速部署）。

資料持久化：TypeORM 支援 SQLite/PostgreSQL/MySQL；Blob 儲存支援 Local/S3/GCS/Azure Blob Storage。

安全注意：預設無登入保護，需設定 FLOWISE_USERNAME / FLOWISE_PASSWORD 環境變數。

---

與 daily-digest-prompt 的連結

  1. Phase 管線視覺化：Execute Flow Node 可把 Phase 0→1→2→3 的 Agent 呼叫鏈可視化，方便除錯特定 Phase 斷點。
  1. RAG 知識庫橋接：Vector Store 節點可連接 localhost:3000 知識庫後端，讓 Flowise 工作流直接查詢/寫入 KB。
  1. *$flow.state 對應 state/.json**：本系統的手工 JSON 快照模式，對應 Flowise 的 $flow.state，提供有狀態 Agent 工作流的視覺化設計驗證。
  1. *tools/.py 透過 MCP 接入**：本系統 CLI 工具若包裝為 MCP Server，可直接插入 AgentFlow 作為 Tool Node，無需重寫。
  1. Human Input Node ≈ human-review-queue.json：圖形化替代方案，適合 research_task 或 self_heal 的人工審核流程。
  1. 快速 POC 工具：新 auto-task 構想先在 Flowise 驗證 Node 串接邏輯，確認可行後再轉化為正式 Prompt + PowerShell 腳本。

優缺點評估

優點：零程式碼快速原型（RAG 聊天機器人 15 分鐘可跑起）；AgentFlow V2 原生多 Agent（Supervisor/Worker、HITL、子工作流嵌套）；MCP 一等公民支援；1000+ 節點整合；Apache 2.0 授權（商業友好，無 Fair-Code 限制）；Docker Queue Mode 企業級水平擴展。

缺點 / 限制：非通用自動化工具（缺乏 n8n 的 400+ 業務系統整合）；TypeScript/Node.js 限制（Python 重度使用者有學習曲線）；視覺畫布複雜度上限（超過 30-50 個節點可讀性急降）；無 DAG 複雜依賴管理；預設無認證（生產環境需手動配置）；無原生排程機制（需搭配外部排程器）。
