概述

superpowers 是由 Jesse Vincent 與 Prime Radiant 團隊開發的開源 Agentic Skills 框架，截至 2026 年 3 月已累積超過 118,000 顆 GitHub Stars、9,500+ Forks，是目前 GitHub 上增長最快的 Agent Harness 專案之一（單週增長 +19,621 Stars）。

專案核心理念：「不要讓 AI Agent 直接寫程式碼，先讓它規格化問題、建立計畫、再以 TDD 循環實作」。這是一套完整的軟體開發工作流方法論，而非單純的程式碼生成工具。

工作流程（七步驟）

  1. 釐清提問：Agent 先問問題，不直接寫程式碼
  2. 設計驗證：以可讀區塊呈現設計方案供人類確認
  3. 計畫建立：產出粒度細緻、可測試的任務清單
  4. Subagent 派發：每個任務交給全新的 Agent 執行
  5. TDD 品質門檻：強制測試先行
  6. 任務間審查：Spec 合規性 + 程式碼品質雙重審查
  7. 分支完成：提供合併/PR 決策選項

方法論核心：Spec-First 開發

superpowers 的哲學可以用四個關鍵字概括：

  • Test-Driven Development：測試先行，非測試後補
  • Systematic over Ad-hoc：系統化取代臨時拼湊
  • Complexity Reduction：降低複雜度為首要目標
  • Evidence over Claims：用證據說話，非口頭宣稱

這與傳統 AI 程式碼生成工具的「直接產出程式碼」模式形成鮮明對比。superpowers 將 AI Agent 視為「需要詳細規格的初級工程師」，透過結構化流程確保輸出品質。

對 Daily-Digest-Prompt 系統的啟示

  1. Spec-First 與本系統高度契合：superpowers 的 Brainstorming → Plan → TDD 流程，與本專案 CLAUDE.md 中的「規格→計畫→任務→TDD」指令鏈幾乎同構
  2. Subagent 派發模式：每任務獨立 Agent + 兩階段審查，可借鏡強化本系統的 Team 並行模式
  3. Skill 可組合性：superpowers 的 Skills 設計為可獨立使用、可組合，與本系統的 Skill-First 策略理念一致
  4. Configuration Sprawl 警示：社群已注意到配置膨脹問題，與本系統的熵增治理方向吻合
  5. Git Worktrees 實務：隔離式工作區管理值得本系統在複雜多 Agent 場景中採用
