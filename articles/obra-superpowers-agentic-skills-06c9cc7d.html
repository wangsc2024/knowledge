obra/superpowers — Agentic Skills 框架與軟體開發方法論

概述

Superpowers 是由 Jesse Vincent（Keyboardio 創辦人、Best Practical 創辦人）開發的開源 Agentic Skills 框架，為 AI 編碼代理提供結構化的軟體開發工作流程。核心理念是「給 Agent 紀律，而非更多工具」——透過 Markdown 技能檔（SKILL.md）編碼領域知識與流程規範，讓 Agent 自動遵循 brainstorm → plan → implement (TDD) → review → finish 的開發週期。

二、核心架構：Skill 驅動的強制工作流

2.1 Skill 系統設計

Superpowers 的 Skill 是可重用的 Markdown 指令檔（SKILL.md），具備以下特性：

  • 自動發現：Agent 啟動時掃描 skills/ 目錄，根據任務上下文自動觸發對應 Skill
  • 強制性：Skill 不是可選建議，而是必須遵循的流程
  • 可組合：多個 Skill 可以串聯組合
  • 自我擴展：Agent 可以透過撰寫新的 SKILL.md 建立新技能

2.2 14 個核心 Skill 分類

| 類別 | Skills | 用途 |

|------|--------|------|

| 核心工作流 | brainstorming, planning, subagent-driven-development | 需求釐清→方案規劃→任務分派 |

| 開發實踐 | TDD, systematic-debugging, git-worktrees | 紅綠燈測試、根因分析、並行開發 |

| 協作工具 | parallel-agent-dispatching, requesting-code-review, receiving-code-review | 多 Agent 並行、雙階段審查 |

| 元技能 | using-superpowers, verification-before-completion | 框架使用指引、完成前驗證 |

2.3 工作流程（必須按序執行）

  1. Brainstorm（腦力激盪）：Agent 不直接寫程式碼，先提問釐清需求，探索替代方案
  2. Plan（規劃）：將工作拆解為 2-5 分鐘的小任務，每個附帶精確檔案路徑與驗證步驟
  3. Implement（實作）：子 Agent 驅動開發，每個任務派遣獨立子 Agent，雙階段審查
  4. TDD（測試驅動）：嚴格 RED-GREEN-REFACTOR 循環
  5. Review and Finish（審查與完成）：verification-before-completion 確保修復有效

三、關鍵創新

3.1 子 Agent 驅動開發（Subagent-Driven Development）

  • 上下文隔離：每個任務由全新子 Agent 執行，防止認知汙染（cognitive pollution）
  • 雙階段審查：第一階段檢查規格合規性，第二階段檢查程式碼品質
  • 人類檢查點：批次執行時在關鍵節點暫停等待人類確認

3.2 說服力原則驅動 LLM 行為

Vincent 發現 Robert Cialdini 的六大說服力原則可以有效影響 LLM 行為：

| 原則 | 在 Skill 中的應用 |

|------|------------------|

| 權威（Authority） | 引用領域專家的最佳實踐 |

| 承諾（Commitment） | 讓 Agent 先宣告遵循的原則 |

| 互惠（Reciprocity） | 提供上下文換取 Agent 遵守規範 |

| 稀缺（Scarcity） | 強調時間壓力下的優先決策 |

| 社會證明（Social Proof） | 展示成功案例 |

| 統一（Unity） | 建立「我們是同一團隊」的框架 |

Dan Shapiro 的同儕審查研究驗證了這種方法的科學基礎。

3.3 TDD for Skills（技能測試驅動開發）

Vincent 開創了「對 Skill 本身進行 TDD」的方法：

  • 壓力情境測試：模擬時間壓力、沉沒成本謬誤、競爭優先級等場景
  • 子 Agent 測驗：主 Agent 評估子 Agent 是否理解並遵循 Skill 指令
  • 迭代改進：根據測試反饋調整 Skill 內容
  • 成果：2,249 個 Markdown 檔從對話記憶中萃取

四、與 Daily-Digest-Prompt 架構比較

| 面向 | Superpowers | Daily-Digest-Prompt |

|------|-------------|--------------------|

| Skill 格式 | SKILL.md（純 Markdown） | SKILL.md（YAML frontmatter + Markdown） |

| 觸發機制 | 自動掃描 + 上下文匹配 | SKILL_INDEX.md 索引 + 關鍵字觸發 |

| 子 Agent | 每任務獨立派遣 | Team 模式 + 子 Agent 並行 |

| 工作流 | brainstorm→plan→implement→review | Phase 0→1→2→3 管線式 |

| 品質保證 | 雙階段審查 | Hook 機器強制層 + done_cert |

| 配置管理 | 純 Markdown | YAML 外部配置 + Markdown 模板 |

| 記憶系統 | 對話記憶萃取（2,249 檔） | context/*.json 結構化記憶 |

可借鏡的設計模式

  1. TDD for Skills：可考慮對本專案的 Skill 建立壓力測試
  2. 說服力原則：在 preamble.md 中加入 Cialdini 原則強化 Agent 遵循度
  3. 雙階段審查：done_cert 可擴展為規格合規 + 品質的雙重驗證
  4. 上下文隔離：每個自動任務已用獨立 prompt，可進一步加強隔離

五、社群生態

六、限制

  • 依賴 Agent 對說服力原則的響應，大規模驗證不足
  • 缺乏量化的失敗模式文檔
  • 智慧財產問題：從技術書籍萃取的 Skill 版權歸屬尚未明確
  • 僅 10 位貢獻者，社群貢獻規模有限
