kb-research-strategist Skill 設計文件

設計動機（三個問題同時解決）

  1. KB 運用能力不足：現有去重只讀標題，從未真正讀取 KB 文件內容，無法利用既有知識
  2. 規劃深度不夠：只提示「換個方向」，沒有基於 KB 現況的結構化研究計畫
  3. 長期研究缺失：research-registry.json 7 天 TTL，系列研究進度無法跨週追蹤

核心設計：雙層追蹤機制

| 文件 | 用途 | TTL |

|------|------|-----|

| context/research-registry.json | 短期去重（防止重複） | 7 天滾動 |

| context/research-series.json | 長期系列追蹤（累積深度） | 永久保留 |

五階段知識深化模型

每個研究系列依知識累積深度分為五個階段：

| 階段 | 英文鍵 | 研究目標 |

|------|--------|----------|

| 1 基礎建立 | foundation | 概念、術語、整體架構 |

| 2 機制深化 | mechanism | 內部原理、演算法、實作細節 |

| 3 應用實踐 | application | 程式碼範例、實際案例、部署方法 |

| 4 優化精進 | optimization | 效能、最佳實踐、邊緣案例 |

| 5 整合昇華 | synthesis | 與其他主題連結、未來展望 |

Skill 七步驟執行流程

  • 步驟 0：讀取 research-series.json，偵測現有系列（按 domain + tags 比對）
  • 步驟 1：KB hybrid search（topK=10）
  • 步驟 2：讀取相關筆記全文（score 0.4-0.85，避免高重複的 >0.85）
  • 步驟 3：合成現有知識狀態（depth_level + covered/partial/missing concepts）
  • 步驟 4：系列狀態判定（情境 A 命中/B 有基礎/C 全新）→ 決定本次研究目標
  • 步驟 5：輸出 kb-research-brief.json（含系列 context + 結構化研究計畫）
  • 步驟 7：降級處理（KB 不可用時仍依系列計畫執行）

kb-research-brief.json 關鍵欄位


{
  "series": { "series_id": "rag-advanced", "current_stage": "application", "completion_pct": 40 },
  "kb_foundation": { "depth_level": "mechanism", "synthesis": "...", "missing_concepts": [...] },
  "knowledge_gaps": [{ "gap": "Reranking 實作", "stage": "application" }],
  "research_plan": { "primary_question": "...", "steps": [{"search_keywords": [...]}] },
  "recommendation": "deepen",
  "series_update": { "series_id": "...", "stage_to_update": "application", "new_status": "in_progress" }
}

共用模板：kb-depth-check.md（4 Phase 流程）

  • Phase A：執行 kb-research-strategist Skill
  • Phase B：讀取 brief，依 recommendation 決定研究深度
  • Phase C：研究完成後更新 research-series.json（Python 腳本處理系列狀態推進）
  • Phase D：清理 kb-research-brief.json

Bot Server 整合（Chatroom 研究任務）

新流程：用戶研究請求 → Groq 提取研究關鍵詞（---RESEARCH_KEYWORDS--- 分隔符）→ Worker 偵測 is_research=true → kb-research-strategist 分析 KB 現況 → 注入系列上下文到 prompt → claude -p 執行（含深度指引）→ 更新 research-series.json。

修改文件：bot/lib/classifier.js（SYSTEM_PROMPT + optimizeTask 結構化回傳）、bot/lib/routes.js（回應新增 research_keywords + is_research）、bot/skills/task-optimizer.md（研究意圖識別段落）、bot/process_messages.ps1（pre/post KB 深化步驟）。

完整研究流程


① registry 去重（3 天冷卻）
② KB 高閾值去重（score > 0.85 換主題）
③ kb-research-strategist → kb-research-brief.json
④ 研究代理讀取 brief，聚焦對應階段問題
⑤ WebSearch + WebFetch
⑥ 新筆記匯入 KB（含交叉參考）
⑦ 更新 research-series.json
⑧ 清理 kb-research-brief.json
⑨ 更新 research-registry.json
