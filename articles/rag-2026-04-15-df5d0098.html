一、核心发现：五大跨领域洞见

洞见 1：三层治理架构的收敛趋势

观察：多个先进框架不约而同采用三层治理模式：

  1. Claude Code Hooks（2026-02）：

- Layer 1：Command Handler（即时同步拦截）

- Layer 2：Prompt Handler（动态注入上下文）

- Layer 3：Agent Handler（子 Agent 分支决策）

- 支持 17 个生命周期事件，PreToolUse 可修改输入（v2.0.10+）

  1. Claude Code Plugin 系统（2026-03）：

- 7 组件架构：Commands、Agents、Skills、Hooks、MCP Servers、Settings、Manifest

- Hookify 自动化规则引擎：从配置生成 Hook 逻辑

- Plugin-Dev 8 阶段构建工作流

  1. DeepEval LLM 评测框架：

- 六层架构：Entry Points → Evaluation Engine → Data Management → Metrics → Model Abstraction → Observability

- 50+ 评测指标、Pytest 风格、CI/CD 集成

同构性识别：三者共享的设计原则

  • 分离关注点：守卫（Guard）、调度（Orchestration）、执行（Execution）各层独立
  • 可组合性：每层可独立替换、测试、扩展
  • 声明式配置：规则外部化，减少硬编码

对 daily-digest-prompt 的启示：

  • ✅ 已有基础：Phase 0/1/2/3 分离、Hook 机器强制、Skill-First 路由
  • ⚠️ 待强化：Hook 规则分散在多个 Python 脚本（ADR-031 pending），缺乏统一 registry
  • 📋 优化方向：引入 宣告式 Hook Registry（参考 Hookify），统一管理 pre_bash_guard、pre_write_guard 等规则

洞见 2：Observational Memory — 从被动记录到主动反思

观察：Mastra 框架引入创新的 Observational Memory 模式（23K Stars, 2026）：

  1. Observer（观察者）：

- 被动记录 Agent 行为（tool calls、errors、user feedback）

- 结构化存储：JSON Schema 定义的事件流

- 类比：daily-digest-prompt 的 logs/structured/*.jsonl

  1. Reflector（反思者）：

- 主动分析历史行为模式

- 生成 insights：「过去 7 天，每次 API 失败后的重试间隔过短」

- 调整策略：动态修改 Agent 参数

  1. Memory Store：

- 向量化存储 observations

- 检索相似历史场景：「上次遇到这个错误时，解决方案是...」

同构性识别：与佛學「观照」的映射

  • Observer = 能观之智（觉察当下）
  • Reflector = 所观之境（反思历史）
  • Memory Store = 识藏（储存种子）

对 daily-digest-prompt 的启示：

  • ✅ 已有基础：digest-memory.json（schema_version=2）、logs/structured/*.jsonl
  • ⚠️ 当前限制：digest-memory.json 为纯记录型，无主动反思能力
  • 📋 优化方向：

1. 升级为 Observational Memory 模式：

- digest-memory.json → digest-observations.json（被动记录）

- 新增 digest-insights.json（主动反思）

2. 每日生成 insights：

- 「近 7 天 Todoist 任务完成率趋势」

- 「最常触发的 Hook 规则 Top 5」

- 「Token 消耗异常时段分析」

3. 反馈回路：insights 自动触发 config/*.yaml 调整建议

观察：多个框架同时支持 MCP（Model Context Protocol）：

二、系统优化建议（可执行的改进方案）

优化方案 1：宣告式 Hook Registry（优先级：P0）

当前痛点：

  • Hook 规则分散在 13 个 Python 脚本（hooks/*.py）
  • 新增规则需修改代码，缺乏配置化管理
  • ADR-031 pending：宣告式 hook registry 与 stage 化 guard 链

优化方案：

  1. 创建 config/hook-registry.yaml：

```yaml

version: 1

guards:

pre_bash_nul_guard:

stage: pre_bash

pattern: "> nul"

action: block

message: "禁止在 Bash 使用 > nul（使用 > /dev/null 2>&1）"

priority: 1

pre_write_scheduler_state:

stage: pre_write

path_pattern: "state/scheduler-state.json"

allowed_callers: ["pwsh.exe", "run-agent-team.ps1"]

action: warn

message: "scheduler-state.json 写入权限限定于 PowerShell"

priority: 2

```

  1. 实现 hooks/hook_registry.py：

- 读取 YAML，构建 guard 链

- 按 priority 排序执行

- 支持 action: block|warn|log

  1. 重构现有 Hooks：

- pre_bash_guard.py → 调用 hook_registry.load("pre_bash")

- 减少硬编码规则，80% 规则迁移至 YAML

预期收益：

  • ✅ 新增 Hook 规则无需修改代码
  • ✅ 规则优先级可视化、可调整
  • ✅ 降低 Hook 维护成本

实施计划：

  • Week 1：创建 config/hook-registry.yaml，迁移 5 个高频规则
  • Week 2：实现 hooks/hook_registry.py
  • Week 3：重构 pre_bash_guard.py、pre_write_guard.py
  • Week 4：测试套件更新（tests/hooks/test_hook_registry.py）

优化方案 2：Observational Memory 反思系统（优先级：P1）

  • 系统异常需人工查阅日志，缺乏自动 insights

优化方案：

  1. 拆分记忆系统：

- digest-observations.json：被动记录（保留现有结构）

- digest-insights.json：主动反思（新增）

  1. insights 生成逻辑（每日执行）：

```python

# tools/generate_insights.py

import json

from datetime import datetime, timedelta

def analyze_observations():

# 读取近 7 天的 observations

obs = load_observations(days=7)

insights = {

"generated_at": datetime.now().isoformat(),

"insights": []

}

# Insight 1: Todoist 完成率趋势

completion_rates = [day['todoist_completed'] / day['todoist_total']

for day in obs if day['todoist_total'] > 0]

if len(completion_rates) >= 3:

trend = "上升" if completion_rates[-1] > completion_rates[0] else "下降"

insights['insights'].append({

"type": "performance_trend",

"title": f"Todoist 完成率呈{trend}趋势",

"data": {"rates": completion_rates},

"recommendation": "考虑调整任务复杂度" if trend == "下降" else "继续保持"

})

# Insight 2: Token 消耗异常时段

high_usage_hours = [h for h, usage in enumerate(obs['hourly_tokens'])

if usage > obs['avg_tokens_per_hour'] * 2]

if high_usage_hours:

insights['insights'].append({

"type": "resource_anomaly",

"title": "Token 消耗异常时段",

"data": {"hours": high_usage_hours},

"recommendation": f"检查 {high_usage_hours[0]}:00 时段的任务类型"

})

return insights

```

预期收益：

  • ✅ 减少人工日志分析时间

优化方案 3：Skill Discovery Manifest（优先级：P1）

当前痛点：

  • Skill 触发依赖手动配置 triggers 关键字
  • skills/SKILL_INDEX.md 需手动维护，容易过时
  • 无法动态发现新 Skill

优化方案：

  1. 每个 Skill 生成 JSON manifest：

```bash

# tools/generate_skill_manifests.ps1

Get-ChildItem skills/*/SKILL.md | ForEach-Object {

$skillDir = $_.Directory

$manifest = "$skillDir/manifest.json"

# 从 SKILL.md frontmatter 生成 JSON

$content = Get-Content $_ -Raw

$yaml = ($content -split '---')[1]

$data = ConvertFrom-Yaml $yaml

@{

name = $data.name

version = $data.version

triggers = $data.triggers

allowed_tools = $data.'allowed-tools'

dependencies = $data.'depends-on'

schema = @{

input = $null # TODO: 从 SKILL.md 提取 input schema

output = $null

}

} | ConvertTo-Json | Set-Content $manifest

}

```

  1. 实现 Skill Discovery Engine：

```python

# tools/skill_discovery.py

import json

from pathlib import Path

class SkillRegistry:

def __init__(self):

self.skills = {}

self.trigger_map = {} # O(1) lookup

self._load_manifests()

def _load_manifests(self):

for manifest_path in Path("skills").rglob("manifest.json"):

with open(manifest_path) as f:

skill = json.load(f)

self.skills[skill['name']] = skill

for trigger in skill['triggers']:

self.trigger_map[trigger.lower()] = skill['name']

def find_skill(self, query: str) -> str | None:

"""O(1) 触发词查找"""

return self.trigger_map.get(query.lower())

```

  1. 集成到 Agent 启动流程：

- run-agent-team.ps1 Phase 0：执行 skill_discovery.py

- 生成 state/skill-registry.json（运行时缓存）

预期收益：

  • ✅ Skill 触发词查找从 O(n) → O(1)
  • ✅ 新增 Skill 自动发现，无需手动更新 INDEX
  • ✅ 为未来 MCP 整合铺路

实施计划：

  • Week 1：实现 tools/generate_skill_manifests.ps1
  • Week 2：实现 tools/skill_discovery.py
  • Week 3：集成到 Phase 0
  • Week 4：扩展：支持 input/output schema 定义

  1. 实现评测脚本：

```python

# tools/content_quality_eval.py

import re

from typing import Literal

class ContentQualityEvaluator:

def eval_digest_density(self, digest_text: str) -> float:

# 提取关键信息：日期、数字、专有名词

key_info = len(re.findall(r'\d{4}-\d{2}-\d{2}|\d+(?:\.\d+)?%?|[A-Z][a-z]+(?:[A-Z][a-z]+)+', digest_text))

total_chars = len(digest_text)

return key_info / total_chars if total_chars > 0 else 0

def eval_task_executability(self, task_desc: str) -> float:

action_verbs = ['做', '完成', '整理', '研究', '分析', '实现', '测试']

has_verb = any(verb in task_desc for verb in action_verbs)

has_time = bool(re.search(r'\d{1,2}:\d{2}|今天|明天|本週', task_desc))

return (int(has_verb) + int(has_time)) / 2

def eval_research_depth(self, note_content: str) -> Literal['foundation', 'mechanism', 'application', 'optimization', 'synthesis']:

# 关键词匹配

if '演算法' in note_content or '原理' in note_content:

return 'mechanism'

elif '```' in note_content: # 包含代码块

return 'application'

elif 'benchmark' in note_content or '效能' in note_content:

return 'optimization'

else:

return 'foundation'

```

```powershell

```

  • ✅ 与 Harness B0-B5 形成完整评测體系
