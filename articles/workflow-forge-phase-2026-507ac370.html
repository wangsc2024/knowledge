workflow-forge：Phase 間交接格式驗證清單（2026-03-23）

workflow-forge流程標準化Phase 交接系統優化

Phase 間交接格式驗證清單

產出背景

運用 workflow-forge Skill 識別系統缺口，針對 run-todoist-agent-team.ps1 的 Phase 1→2→3 交接格式不一致問題，產出標準化驗證清單。

問題識別

  • 現況：system-insight 顯示執行成功率僅 87.5%（目標 95%）
  • 根因：Phase 間交接格式缺乏明確定義，導致 Phase 3 組裝失敗
  • 缺口：現有 3 個 workflow（prompt-output、results、config-consistency）未涵蓋執行流程中的資料交接

產出內容

主要產物：workflows/phase-handoff-validation-checklist.md

驗證範圍：

  1. Phase 1 輸出驗證（results/todoist-query.json）

- 必填欄位：processable_tasks, skipped_tasks, routing_summary, timestamp

- processable_tasks[] 結構完整性

- routing_summary 計數一致性

  1. Phase 2 輸出驗證（引用 results-validation-checklist.md）

- 必填欄位：agent, task_key, status

- 檔案命名一致性

  1. Phase 1→2 交接驗證

- 任務數量一致性

- 來源追溯性

- template 對應性

  1. Phase 2→3 交接驗證

- 檔案存在性

- JSON 可解析性

- total_tasks 一致性

  1. 錯誤處理規範

- Phase 1 失敗 → 跳過 Phase 2

- Phase 2 部分失敗 → 記錄後繼續組裝

- Phase 3 組裝失敗 → 標記 format_failed

  1. 端到端驗證

- 全流程數量一致性檢查

整合方式

  • workflows/index.yaml：新增 entry wf-20260323-phase-handoff
  • context/workflow-forge-registry.json：登記產物資訊
  • task_types：設為 all（適用所有任務）

預期成效

  • 即時：開發者有明確 Phase 交接規範
  • 中期：Phase 3 組裝失敗率下降 50%+，成功率提升至 92%+
  • 長期：執行成功率穩定在 95% 以上

技術細節

評分模型：

  • 影響範圍：40/40（影響每日 ~44 次 Todoist 團隊執行）
  • 可執行性：35/35（僅需定義 workflow，不改 PowerShell）
  • 信號強度：25/25（來自 system-insight critical + improvement-backlog）
  • 總分：100/100

開創性：符合（現有 3 個 workflow 未涵蓋 Phase 間交接）

參考資料

  • workflows/results-validation-checklist.md
  • config/schemas/results-auto-task-schema.json
  • run-todoist-agent-team.ps1
