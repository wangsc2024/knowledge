二、重點發現（七大主題深度萃取）

2.1 多模型路由新範式（AI Agent 架構）

核心洞見：2026 年企業平均部署 3-7 個模型，「單模型回應所有查詢」範式已被「動態路由 + 多模型協作」取代。

  • Attention-MoA：以「語義注意力」（自然語言批評）取代標量權重，搭配層間殘差合成，小模型組合可超越單一大型模型
  • 六大路由範式：難度感知、人類偏好對齊、聚類路由、強化學習路由、不確定性量化、級聯策略
  • GreenServ 以多臂老虎機實現上下文感知路由，準確率提升 22%
  • 本專案啟示：llm-router.yaml 可借鏡分層合成，Groq 生成初稿 → Claude 精煉合成

2.2 認知決策四重奏（認知科學）

近三天密集產出四篇認知科學筆記，形成一套完整的「思維工具箱」：

| 框架 | 核心能力 | 應用場景 |

|------|---------|----------|

| 第一性原理 | 拆解假設、從零重建 | 架構決策、成本分析 |

| 超級預測 | 校準式機率估計 | 風險評估、優先級排序 |

| 選擇架構 | 環境設計影響行為 | UI/UX、通知策略 |

| 蘇格拉底式提問 | 六類問題引導深度思考 | Prompt 設計、需求釐清 |

交叉洞見：這四者形成互補閉環 — 第一性原理提供拆解工具、蘇格拉底式提問確保不遺漏假設、超級預測量化不確定性、選擇架構將結論轉化為行動環境。可直接應用於 Todoist 任務優先級評分（scoring.yaml）的權重設計。

2.3 系統治理三大缺口（系統運維）

RAG 品質度量日報揭示三個嚴峻缺口：

  1. Constraint 嵌入率 0%：11 個 proposed 約束 / 0 個 accepted，約束演化管線（constraint-evolve Skill）未形成閉環
  2. 配置膨脹 +116%：YAML 行數、Skill 數量（+160%）、配置檔數量（+161%）持續膨脹
  3. RAG 排序品質不足：Precision@3=66.7%（接近 70% 目標）但 Precision@5=40%（低於 50% 目標），「找得到但排序不佳」

2.5 測試時計算擴展（前沿技術）

2026 年最重要的 AI 範式轉移：從「訓練時擴展」轉向「推理時擴展」。

  • OpenAI o1 將 AIME 數學從 GPT-4 的 9% → 79-80%，o3 達 96.7%
  • 六大核心技術：CoT、Tree of Thought/MCTS、Best-of-N、Process Reward Models、自我精煉、ReTool
  • 計算最優策略可使小模型在 FLOPs 匹配下超越大模型
  • 啟示：本專案的 llm-router.yaml 可考慮為複雜任務啟用 extended thinking 模式

2.6 Harness Engineering 統一框架（synthesis 完結）

三維度模型（3C）正式定稿：

  • Context：Agent 感知環境（CLAUDE.md、SKILL.md、config YAML）
  • Constraint：機器可執行約束（Hook 守衛、Linter、結構測試）
  • Convergence：約束評估 → 缺口辨識 → 規則精煉的迭代迴路

2.7 AI for Science 跨域突破

  • 蛋白質設計：IsoDDE 準確度超 AlphaFold 3 兩倍
  • 材料科學：GNoME 預測 240 萬種穩定材料
  • 自駕實驗室：「設計→合成→表徵→優化」全閉環自主運作

三、跨領域湧現模式

模式 1：「分層合成」設計模式

多模型路由（MoA 層間殘差合成）、Dify（DAG 節點變數綁定）、Harness 3C 框架（Context→Constraint→Convergence）— 三篇獨立研究不約而同指向同一架構原則：複雜系統應以「分層 + 合成」取代「單體 + 串行」。

模式 2：「校準回饋」閉環

超級預測的 Brier Score 校準、RAG 品質的 Precision@K/MRR 度量、SRE Postmortem 的證據包限定 — 共同指向：沒有量化回饋的系統必然漂移。

模式 3：「環境設計 > 意志力」

選擇架構的助推理論、習得性無助研究的微行動啟動、第一性原理的「形式 vs 功能」辨析 — 啟示：改變環境比改變人有效，系統設計應降低正確行為的摩擦力。

四、系統優化建議（五項可落地方案）

建議 1：Constraint 嵌入率修復（P0，預計 30 分鐘）

  • 問題：11 proposed / 0 accepted，管線斷裂
  • 方案：審核 context/constraint-metrics.json 中 pending 約束，將成熟且有測試覆蓋的約束批次升級為 accepted
  • 驗證：嵌入率 > 0%

建議 2：RAG 排序品質提升（P1，預計 1 小時）

  • 問題：Precision@5=40%，排序品質不足
  • 方案：調整 config/cache-policy.yaml 的 hybrid search 權重，或在 tools/rag_quality_check.py 中增加 reranking 邏輯
  • 驗證：Precision@5 ≥ 50%

建議 3：配置膨脹治理（P1，持續）

  • 問題：YAML +116%、Skill +160%、配置檔 +161%
  • 方案：執行 analyze-config.ps1 識別冗餘配置；考慮合併低使用率 Skill；設定膨脹率月度基準線
  • 驗證：下次審查膨脹率增速放緩

建議 4：LLM Router 分層合成（P2，研究階段）

  • 問題：所有任務使用同一路由策略
  • 方案：借鏡 MoA 分層合成，為複雜研究任務啟用 Groq 初稿 + Claude 精煉的雙階段路由
  • 驗證：研究任務品質分 ≥ 4.0

建議 5：認知決策框架融入 scoring.yaml（P2）

  • 問題：任務優先級評分缺乏認知科學理論支撐
  • 方案：將選擇架構的「預設效應」與超級預測的「基準率錨定」融入 config/scoring.yaml 的權重計算
  • 驗證：使用者主觀滿意度提升
