Claude Code 2026 年 2 月最新功能研究

概述

Claude Code 在 2026 年 2 月迎來重大更新，核心亮點包括 Claude Opus 4.6 模型發布、Agent Teams 多代理協作功能（研究預覽版）、以及首次在 Opus 級別實現的 1M token context window。這些更新大幅提升了 Claude Code 處理大型程式碼庫和複雜開發任務的能力。

關鍵發現

1. Claude Opus 4.6 模型發布（2026-02-05）

核心改進：

  • 更謹慎的規劃能力，能維持更長時間的 agentic 任務
  • 在大型程式碼庫中運作更可靠
  • 程式碼審查與除錯能力顯著提升
  • 在 Terminal-Bench 2.0 agentic coding 評估中獲得最高分
  • 在 Humanity's Last Exam 中領先所有其他前沿模型

1M Token Context Window（Beta）：

  • 首次在 Opus 級別模型推出
  • MRCR v2 的 8-needle 1M 測試中達到 76%（相比 Sonnet 4.5 的 18.5%）
  • 對處理大型程式碼庫至關重要

新 API 功能：

  • Adaptive Thinking：模型自動判斷何時需要深度推理
  • Effort 控制：低/中/高（預設）/最大四個等級
  • Context Compaction：自動總結舊內容，突破上下文限制

2. Agent Teams 多代理協作（研究預覽）

啟用方式：

export CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1

功能特點：

  • 可同時啟動多個代理並行工作
  • 代理間能自主協調分工
  • 適合可拆分為獨立、讀取密集型的任務
  • 可使用 Shift+Up/Down 或 tmux 直接接管任何子代理
  • 支援通過 agent "tools" frontmatter 中的 Task(agent_type) 語法限制可生成的子代理類型

Hook 事件支援：

  • TeammateIdle：團隊成員閒置時觸發
  • TaskCompleted：任務完成時觸發

3. 自動記憶功能

功能說明：

  • Claude 現在會自動記錄和回憶工作記憶
  • 新增 memory frontmatter 欄位支援
  • 可配置範圍：user（使用者級）、project（專案級）、local（本地級）
  • 跨會話保持工作上下文

4. Fast Mode 快速模式

版本支援： 2.1.36 起 Opus 4.6 支援

特點：

  • 輸出 token 生成速度最高提升 2.5 倍
  • 採用 premium pricing
  • 文件：https://code.claude.com/docs/en/fast-mode

5. PDF 處理改進

新增功能：

  • pages 參數支援指定頁面範圍，例如 pages: "1-5"
  • 大型 PDF（>10 頁）在 @ 提及時返回輕量級引用而非完整內容
  • 錯誤訊息顯示實際限制（100 頁、20MB）

技術細節

最新版本：2.1.39

重要修復：

  • Bedrock/Vertex/Foundry 用戶模型識別符問題
  • MCP 工具串流時返回圖像內容導致的崩潰
  • /resume 會話預覽顯示問題
  • VS Code 終端滾動到頂部的回歸問題
  • Tab 鍵自動完成行為
  • Heredoc 分隔符解析防止命令注入
  • 沙箱模式阻止對 .claude/skills 目錄的寫入

新增功能：

  • --from-pr 標誌：恢復與特定 GitHub PR 關聯的會話
  • 會話自動連結至 gh pr create 建立的 PR
  • 「從這裡總結」功能：部分會話總結
  • MCP OAuth 客戶端憑證支援（Slack 等）
  • VSCode 遠端會話支援，OAuth 用戶可瀏覽和恢復 claude.ai 會話
  • 會話選擇器顯示 git 分支和訊息計數

其他整合

  • Xcode 26.3（2026-02-03）：整合 Claude Agent SDK
  • Chrome 擴充功能（Beta）：從 Claude Code 控制瀏覽器
  • Claude in Excel：升級處理長期複雜任務
  • Claude in PowerPoint（研究預覽）：品牌一致性幻燈片生成

定價

API 定價維持不變：$5/$25 per million tokens（超過 200k tokens 輸入採用溢價定價）
