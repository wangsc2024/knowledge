X Algorithm — X/Twitter For You 推薦演算法開源深度研究

專案概述

xai-org/x-algorithm 是 xAI（Elon Musk 旗下 AI 公司）於 2026 年 1 月開源的 X（前 Twitter）"For You" 動態推薦演算法完整實作。以 Apache 2.0 授權釋出，目前已累積 15,559 顆星與 2,722 個 Fork。主要語言為 Rust（63%）與 Python（37%），採用 Grok-1 Transformer 架構作為推薦排序核心模型。

這是繼 2023 年 Twitter 開源舊版推薦演算法後的第二次開源，但本次版本徹底採用了基於 Grok 的 Transformer 架構，消除了所有手工特徵工程（hand-engineered features），完全依賴深度學習模型自動學習內容相關性。

---

系統架構（四大核心模組）

1. Home Mixer（編排層）

角色：gRPC 服務端，接收使用者請求，編排整個推薦管線。

管線階段：

| 階段 | 說明 |

|------|------|

| Query Hydrators | 取得使用者上下文（互動歷史、追蹤清單） |

| Sources | 從 Thunder + Phoenix 取得候選貼文 |

| Hydrators | 補充貼文元資料（作者、媒體、影片時長） |

| Filters | 移除不合格候選（重複、封鎖、靜音等） |

| Scorers | ML 預測 + 加權計分 + 多樣性調整 |

| Selector | 依分數排序，選出 Top K |

| Post-Selection | 最終可見性檢查（刪除/垃圾/暴力內容） |

| Side Effects | 快取請求資訊供未來使用 |

2. Thunder（即時貼文儲存）

角色：In-Network 候選源。

技術特點：

  • 從 Kafka 消費貼文建立/刪除事件
  • 維護每位使用者的記憶體貼文儲存（原創、回覆/轉發、影片分類）
  • 亞毫秒級查詢（sub-millisecond lookups），無需外部資料庫
  • 自動修剪超過保留期限的舊貼文

3. Phoenix（ML 模型層）

角色：同時負責檢索（Retrieval）與排序（Ranking）兩大任務。

#### 3a. 檢索：Two-Tower 模型

  • User Tower：將使用者特徵與互動歷史編碼為向量 [B, D]
  • Candidate Tower：將所有貼文編碼為向量 [N, D]
  • 相似度搜尋：透過 dot product 取回 Top-K 候選（從數百萬篇 → 數千篇）

#### 3b. 排序：Grok-based Transformer + 候選隔離

  • 基於 Grok-1 開源模型適配的 Transformer 架構
  • 候選隔離（Candidate Isolation）：推論時候選貼文之間無法相互注意（attention mask），確保每篇貼文的分數獨立於同批次其他貼文，使分數可一致且可快取
  • 預測 15 種互動類型的機率（P(like)、P(reply)、P(repost)、P(click)、P(block)、P(report) 等）

#### 加權計分公式


Final Score = SUM(weight_i * P(action_i))
  • 正面動作（按讚、轉發、分享）→ 正權重
  • 負面動作（封鎖、靜音、檢舉）→ 負權重，壓低使用者可能不喜歡的內容

4. Candidate Pipeline（可組合管線框架）

角色：通用推薦管線框架，提供可重用的 Trait：

| Trait | 用途 |

|-------|------|

| Source | 從資料來源取得候選 |

| Hydrator | 補充候選特徵 |

| Filter | 移除不應顯示的候選 |

| Scorer | 計算排序分數 |

| Selector | 排序並選出 Top 候選 |

| SideEffect | 執行非同步副作用（快取、日誌） |

支援平行執行獨立階段，具可配置的錯誤處理與日誌記錄。

---

五大關鍵設計決策

  1. 零手工特徵：完全依賴 Grok Transformer 從使用者互動序列學習，大幅降低資料管線與服務基礎設施複雜度
  2. 候選隔離：Transformer 推論時候選之間互不可見，分數獨立且可快取
  3. Hash-Based Embeddings：檢索與排序均使用多重雜湊函數進行嵌入查找
  4. 多動作預測：非單一「相關性」分數，而是預測 15 種動作機率的加權組合
  5. 可組合管線架構：業務邏輯與管線執行/監控分離，易於新增來源、過濾器、計分器

與 2023 年舊版 Twitter 演算法比較

| 面向 | 2023 版（twitter/the-algorithm） | 2026 版（xai-org/x-algorithm） |

|------|----------------------------------|-------------------------------|

| 語言 | Scala/Java 為主 | Rust + Python（JAX） |

| ML 架構 | 混合模型 + 大量手工特徵 | 純 Grok Transformer，零手工特徵 |

| 檢索 | SimClusters + TwHIN + Graph | Two-Tower（User/Candidate）ANN |

| 排序 | 多模型級聯 | 單一 Transformer + 候選隔離 |

| 星數 | 62K（已歸檔） | 15.5K（持續成長中） |

---

對開發者的啟示

  1. Rust 在推薦系統的崛起：亞毫秒級效能需求驅動從 JVM 生態遷移到 Rust，Thunder 的 in-memory 儲存展現 Rust 在高效能系統的優勢
  2. Transformer 通吃推薦系統：從搜尋、NLP 到推薦，Transformer 架構正統一所有序列建模任務
  3. 候選隔離模式：確保批次推論中分數一致性的精妙設計，值得所有推薦系統借鏡
  4. 可組合管線設計：Trait-based 的管線框架設計模式，適用於任何多階段資料處理系統
