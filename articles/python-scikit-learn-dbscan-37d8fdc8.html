執行摘要

本文提供 Python 環境下 DBSCAN 和 HDBSCAN 聚類演算法的完整實作指南。DBSCAN（Density-Based Spatial Clustering of Applications with Noise）是一種密度基聚類演算法，能自動識別任意形狀的聚類並標記噪音點，HDBSCAN 則是其階層式改良版本，可處理變化密度的資料集。本文涵蓋環境準備、sklearn.cluster.DBSCAN 與 hdbscan.HDBSCAN 的 API 使用、參數調整技巧（eps 選擇、min_samples 經驗法則）、特徵縮放的必要性、Iris 資料集實作範例、聚類評估指標（Silhouette Score、DBCV）、異常檢測應用、以及 DBSCAN vs HDBSCAN 選擇指南。所有程式碼範例均可直接執行，並附完整註解說明。

二、DBSCAN 實作指南

2.1 基本用法

sklearn.cluster.DBSCAN 提供標準化的聚類介面，核心 API 使用如下 [來源：scikit-learn DBSCAN API 文件, 2024]：


from sklearn.cluster import DBSCAN

# 建立 DBSCAN 聚類器
dbscan = DBSCAN(
    eps=0.5,          # 鄰域半徑（epsilon）
    min_samples=5,    # 最小樣本數
    metric='euclidean',  # 距離度量（預設歐式距離）
    algorithm='auto'  # 演算法選擇（auto, ball_tree, kd_tree, brute）
)

# 訓練模型並預測聚類標籤
labels = dbscan.fit_predict(X)

# 取得核心樣本索引
core_samples_mask = np.zeros_like(labels, dtype=bool)
core_samples_mask[dbscan.core_sample_indices_] = True

DBSCAN 將資料點分為三類：

  1. 核心點（Core Points）：在 eps 鄰域內至少有 min_samples 個點
  2. 邊界點（Border Points）：在核心點的鄰域內，但本身不是核心點
  3. 噪音點（Noise Points）：既不是核心點也不是邊界點，label = -1

2.2 參數調整

DBSCAN 的效能高度依賴兩個參數的選擇 [來源：DataCamp DBSCAN 指南, 2026]：

eps（epsilon, 鄰域半徑）選擇方法：

最常用的方法是繪製 k-distance 圖表找出「膝點（knee point）」[來源：Mastering DBSCAN with Scikit-learn, 2024]：


from sklearn.neighbors import NearestNeighbors

# 計算每個點到第 k 個最近鄰的距離
k = 4  # min_samples 值
neighbors = NearestNeighbors(n_neighbors=k)
neighbors_fit = neighbors.fit(X)
distances, indices = neighbors_fit.kneighbors(X)

# 排序距離並繪圖
distances = np.sort(distances[:, k-1], axis=0)
plt.plot(distances)
plt.ylabel('k-NN Distance')
plt.xlabel('Data Points (sorted by distance)')
plt.title('k-Distance Graph')
plt.axhline(y=0.5, color='r', linestyle='--', label='Suggested eps=0.5')
plt.legend()
plt.show()

膝點位置對應的距離值即為建議的 eps 值。

min_samples 經驗法則：

根據實務經驗 [來源：Mastering DBSCAN with Scikit-learn, 2024]：

  • 一般建議：min_samples ≥ 資料維度 × 2
  • 低維資料（2D-3D）：min_samples = 4 或 5
  • 高維資料：可增加至 10-20
  • 資料較多噪音時：增加 min_samples 提高聚類穩健性

2.3 特徵縮放

特徵縮放對 DBSCAN 結果至關重要，因為演算法依賴距離計算 [來源：scikit-learn DBSCAN 文件, 2024]。不同量級的特徵會導致某些維度主導距離計算，產生錯誤的聚類結果。

StandardScaler 使用範例：


from sklearn.preprocessing import StandardScaler

# 建立縮放器
scaler = StandardScaler()

# 訓練並轉換資料（訓練集）
X_scaled = scaler.fit_transform(X)

# 僅轉換資料（測試集）
X_test_scaled = scaler.transform(X_test)

# 檢查縮放結果
print(f"原始資料範圍: {X.min(axis=0)} ~ {X.max(axis=0)}")
print(f"縮放後範圍: {X_scaled.min(axis=0)} ~ {X_scaled.max(axis=0)}")
print(f"縮放後均值: {X_scaled.mean(axis=0)}")  # 應接近 0
print(f"縮放後標準差: {X_scaled.std(axis=0)}")  # 應接近 1

常用的縮放方法 [來源：DBSCAN Python 教學, 2024]：

  • StandardScaler：標準化至均值=0、標準差=1（適用於常態分佈資料）
  • MinMaxScaler：縮放至 [0, 1] 範圍（適用於有明確界限的資料）
  • RobustScaler：使用中位數和四分位距（適用於有極端值的資料）

2.4 Iris 資料集實作範例

以下是完整的端到端實作範例，使用經典的 Iris 資料集 [來源：DBSCAN Clustering with HDBSCAN: Python Tutorial with Iris Dataset, 2024]：


import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
import seaborn as sns
from sklearn.datasets import load_iris
from sklearn.preprocessing import StandardScaler
from sklearn.cluster import DBSCAN
from sklearn.metrics import silhouette_score

# 1. 載入資料
iris = load_iris()
X = iris.data
y_true = iris.target  # 真實標籤（僅用於驗證）

# 2. 特徵縮放
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# 3. 建立並訓練 DBSCAN
dbscan = DBSCAN(eps=0.6, min_samples=4)
labels = dbscan.fit_predict(X_scaled)

# 4. 分析結果
n_clusters = len(set(labels)) - (1 if -1 in labels else 0)
n_noise = list(labels).count(-1)

print(f"聚類數量: {n_clusters}")
print(f"噪音點數量: {n_noise}")
print(f"核心樣本數量: {len(dbscan.core_sample_indices_)}")

# 5. 評估聚類品質（排除噪音點）
if n_clusters > 1:
    mask = labels != -1
    score = silhouette_score(X_scaled[mask], labels[mask])
    print(f"Silhouette Score: {score:.3f}")

# 6. 視覺化（使用前兩個主成分）
plt.figure(figsize=(12, 5))

# 子圖 1：DBSCAN 聚類結果
plt.subplot(1, 2, 1)
unique_labels = set(labels)
colors = [plt.cm.Spectral(each) for each in np.linspace(0, 1, len(unique_labels))]

for k, col in zip(unique_labels, colors):
    if k == -1:
        col = [0, 0, 0, 1]  # 黑色標記噪音點

    class_member_mask = (labels == k)
    xy = X_scaled[class_member_mask]
    plt.plot(xy[:, 0], xy[:, 1], 'o', markerfacecolor=tuple(col),
             markeredgecolor='k', markersize=8, label=f'Cluster {k}')

plt.title(f'DBSCAN Clustering (eps={dbscan.eps}, min_samples={dbscan.min_samples})')
plt.xlabel('Sepal Length (scaled)')
plt.ylabel('Sepal Width (scaled)')
plt.legend()

# 子圖 2：真實標籤（對照用）
plt.subplot(1, 2, 2)
scatter = plt.scatter(X_scaled[:, 0], X_scaled[:, 1], c=y_true, cmap='viridis')
plt.colorbar(scatter)
plt.title('Ground Truth Labels')
plt.xlabel('Sepal Length (scaled)')
plt.ylabel('Sepal Width (scaled)')

plt.tight_layout()
plt.show()

# 7. 建立結果 DataFrame
results_df = pd.DataFrame({
    'sepal_length': X[:, 0],
    'sepal_width': X[:, 1],
    'petal_length': X[:, 2],
    'petal_width': X[:, 3],
    'cluster': labels,
    'true_label': y_true
})

print("\n前 10 筆聚類結果:")
print(results_df.head(10))

執行結果說明：

使用 Iris 資料集時，eps=0.6 和 min_samples=4 通常能產生 2-3 個主要聚類 [來源：DBSCAN Python Tutorial with Iris Dataset, 2024]。由於 Iris 資料集的 setosa 類別與其他兩類在特徵空間中有明顯分離，DBSCAN 能有效識別出這個聚類。

三、HDBSCAN 實作指南

3.1 基本用法

HDBSCAN（Hierarchical DBSCAN）提供與 scikit-learn 相容的介面，但參數更簡化 [來源：HDBSCAN 官方文件基本用法, 2024]：


import hdbscan

# 建立 HDBSCAN 聚類器
clusterer = hdbscan.HDBSCAN(
    min_cluster_size=5,       # 最小聚類大小（唯一必要參數）
    min_samples=None,         # 最小樣本數（預設與 min_cluster_size 相同）
    cluster_selection_epsilon=0.0,  # 額外的 epsilon 閾值（預設 0）
    metric='euclidean',       # 距離度量
    cluster_selection_method='eom'  # 聚類選擇方法（eom 或 leaf）
)

# 訓練模型
clusterer.fit(X_scaled)

# 取得聚類標籤
labels = clusterer.labels_

# 取得機率分數（soft clustering）
probabilities = clusterer.probabilities_

HDBSCAN 的關鍵優勢在於簡單的 API：只需設定 min_cluster_size 參數，無需手動調整 eps [來源：HDBSCAN 官方文件, 2024]。

3.2 參數調整

min_cluster_size（最小聚類大小）：

這是 HDBSCAN 最重要的參數，決定形成聚類所需的最小點數。選擇建議：

  • 小值（5-10）：產生更多、更小的聚類
  • 大值（15-30）：產生較少、較大的聚類
  • 經驗法則：觀察資料分佈，估計最小有意義聚類的大小

soft clustering（軟聚類）：

HDBSCAN 為每個資料點提供 0.0-1.0 的成員分數，表示該點屬於聚類的信心程度 [來源：HDBSCAN 基本用法文件, 2024]：


# 取得成員分數
probabilities = clusterer.probabilities_

# 識別低信心點
low_confidence_mask = probabilities < 0.5
low_confidence_points = X[low_confidence_mask]

print(f"低信心點數量: {low_confidence_points.shape[0]}")

# 視覺化成員分數
plt.scatter(X_scaled[:, 0], X_scaled[:, 1], c=probabilities, cmap='viridis')
plt.colorbar(label='Membership Probability')
plt.title('HDBSCAN Membership Scores')
plt.show()

3.3 與 DBSCAN 比較

HDBSCAN 相較於 DBSCAN 的三大優勢 [來源：HDBSCAN vs DBSCAN - Daily Dose of Data Science, 2024]：

1. 處理變化密度的聚類：

DBSCAN 假設所有聚類具有相似的密度，使用單一 eps 值。HDBSCAN 則在不同 epsilon 值下執行 DBSCAN 並整合結果，能自動適應密度變化的資料集 [來源：Understanding HDBSCAN - Arize, 2024]。


# 示範：變化密度資料集
from sklearn.datasets import make_blobs

# 建立兩個密度不同的聚類
X1, _ = make_blobs(n_samples=200, centers=1, cluster_std=0.3, random_state=42)
X2, _ = make_blobs(n_samples=200, centers=1, cluster_std=1.5, random_state=42)
X2 = X2 + 5  # 移動至不同位置
X_varied = np.vstack([X1, X2])

# DBSCAN 結果（單一 eps）
dbscan = DBSCAN(eps=0.5, min_samples=5)
dbscan_labels = dbscan.fit_predict(X_varied)

# HDBSCAN 結果（自適應 eps）
hdbscan_clusterer = hdbscan.HDBSCAN(min_cluster_size=10)
hdbscan_clusterer.fit(X_varied)
hdbscan_labels = hdbscan_clusterer.labels_

# 比較結果（HDBSCAN 能同時識別兩個密度不同的聚類）

2. 無需手動調整 eps 參數：

DBSCAN 的 eps 選擇需要 k-distance 圖表分析，容易受主觀判斷影響。HDBSCAN 僅需 min_cluster_size，參數調整更直觀 [來源：HDBSCAN vs DBSCAN, 2024]。

3. 可預測新資料點：

HDBSCAN 支援 approximate_predict() 方法，可為新資料點分配聚類標籤，DBSCAN 則不支援此功能 [來源：HDBSCAN vs DBSCAN, 2024]：


# 訓練 HDBSCAN
clusterer = hdbscan.HDBSCAN(min_cluster_size=5)
clusterer.fit(X_train)

# 預測新資料點
new_labels, new_probabilities = hdbscan.approximate_predict(clusterer, X_new)

計算複雜度差異：

HDBSCAN 的時間複雜度略高於 DBSCAN（O(n log n) vs O(n log n) with optimized implementation），但對於中小型資料集（< 100,000 點）差異不顯著。

3.4 實作範例

完整的 HDBSCAN 實作流程：


import hdbscan
import matplotlib.pyplot as plt
import seaborn as sns
from sklearn.datasets import load_iris
from sklearn.preprocessing import StandardScaler

# 1. 載入與前處理
iris = load_iris()
X = iris.data
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# 2. 建立並訓練 HDBSCAN
clusterer = hdbscan.HDBSCAN(
    min_cluster_size=5,
    min_samples=3,
    cluster_selection_method='eom'
)
clusterer.fit(X_scaled)

# 3. 取得結果
labels = clusterer.labels_
probabilities = clusterer.probabilities_

# 4. 分析
n_clusters = len(set(labels)) - (1 if -1 in labels else 0)
n_noise = list(labels).count(-1)

print(f"聚類數量: {n_clusters}")
print(f"噪音點數量: {n_noise}")
print(f"平均成員分數: {probabilities.mean():.3f}")

# 5. 視覺化 condensed tree
clusterer.condensed_tree_.plot(select_clusters=True, selection_palette=sns.color_palette())
plt.title('HDBSCAN Condensed Tree')
plt.show()

# 6. 視覺化聚類結果
plt.figure(figsize=(12, 5))

# 子圖 1：聚類標籤
plt.subplot(1, 2, 1)
plt.scatter(X_scaled[:, 0], X_scaled[:, 1], c=labels, cmap='viridis')
plt.colorbar(label='Cluster Label')
plt.title('HDBSCAN Clustering Results')
plt.xlabel('Feature 1 (scaled)')
plt.ylabel('Feature 2 (scaled)')

# 子圖 2：成員分數
plt.subplot(1, 2, 2)
plt.scatter(X_scaled[:, 0], X_scaled[:, 1], c=probabilities, cmap='plasma')
plt.colorbar(label='Membership Probability')
plt.title('HDBSCAN Membership Scores')
plt.xlabel('Feature 1 (scaled)')
plt.ylabel('Feature 2 (scaled)')

plt.tight_layout()
plt.show()

condensed tree 視覺化說明：

condensed tree 是 HDBSCAN 特有的視覺化工具，顯示聚類在不同 lambda（相當於 1/epsilon）值下的階層結構 [來源：How HDBSCAN Works 官方文件, 2024]。圖中水平軸代表 lambda 值，垂直軸代表樣本數量，分支點顯示聚類的分裂與合併過程。

四、聚類結果評估

4.1 Silhouette Score

Silhouette Score（輪廓係數）是最常用的聚類評估指標，範圍為 -1 到 +1 [來源：Intrinsic Measures for Clustering Evaluation, 2024]：

  • +1：樣本遠離鄰近聚類，與自身聚類非常匹配
  • 0：樣本在兩個聚類的邊界上
  • -1：樣本可能被分配到錯誤的聚類

計算公式：

對於樣本 i，Silhouette Score 定義為：


s(i) = (b(i) - a(i)) / max(a(i), b(i))

其中：

  • a(i)：樣本 i 到同聚類其他點的平均距離
  • b(i)：樣本 i 到最近聚類所有點的平均距離

Python 程式碼：


from sklearn.metrics import silhouette_score, silhouette_samples

# 整體 Silhouette Score（排除噪音點）
mask = labels != -1
overall_score = silhouette_score(X_scaled[mask], labels[mask])
print(f"整體 Silhouette Score: {overall_score:.3f}")

# 每個樣本的 Silhouette Score
sample_scores = silhouette_samples(X_scaled[mask], labels[mask])

# 視覺化每個聚類的 Silhouette 分佈
import matplotlib.pyplot as plt
import numpy as np

fig, ax = plt.subplots()
y_lower = 10

unique_labels = np.unique(labels[mask])
for i, label in enumerate(unique_labels):
    cluster_scores = sample_scores[labels[mask] == label]
    cluster_scores.sort()

    size = cluster_scores.shape[0]
    y_upper = y_lower + size

    color = plt.cm.Spectral(float(i) / len(unique_labels))
    ax.fill_betweenx(np.arange(y_lower, y_upper), 0, cluster_scores,
                      facecolor=color, edgecolor=color, alpha=0.7,
                      label=f'Cluster {label}')

    y_lower = y_upper + 10

ax.axvline(x=overall_score, color="red", linestyle="--", label='Average Score')
ax.set_xlabel("Silhouette Coefficient")
ax.set_ylabel("Cluster")
ax.set_title("Silhouette Analysis")
ax.legend()
plt.show()

適用場景：

Silhouette Score 適用於凸形（convex）聚類，如 K-Means 產生的結果。對於任意形狀的密度基聚類，Silhouette 可能低估聚類品質 [來源：Intrinsic Measures for Clustering, 2024]。

4.2 DBCV 指標

DBCV（Density-Based Clustering Validation）專為密度基聚類設計，考慮聚類的密度變化和噪音點 [來源：Intrinsic Measures for Clustering Evaluation, 2024]。DBCV 評估聚類內部密度連接性和聚類間分離度，範圍為 -1 到 +1，值越高表示聚類品質越好。

為何 DBCV 優於 Silhouette：

  • Silhouette 假設聚類為凸形，無法正確評估任意形狀的聚類
  • Silhouette 依賴質心概念，對密度聚類不適用
  • DBCV 考慮局部密度變化，更適合評估 DBSCAN/HDBSCAN 結果

Python 程式碼：


from validclust import dunn
from sklearn.metrics.pairwise import euclidean_distances

def compute_dbcv(X, labels):
    """
    計算 DBCV 指標（簡化實作）
    實務上建議使用 validclust 或 hdbscan 套件提供的實作
    """
    try:
        from hdbscan import validity_index
        return validity_index(X, labels)
    except ImportError:
        print("請安裝 hdbscan 套件以使用 DBCV: pip install hdbscan")
        return None

# 計算 DBCV
dbcv_score = compute_dbcv(X_scaled, labels)
if dbcv_score is not None:
    print(f"DBCV Score: {dbcv_score:.3f}")

注意事項：

DBCV 計算較為耗時，對於大型資料集（> 10,000 點）可能需要數秒至數分鐘。若僅需快速評估，Silhouette Score 仍是實務上的常用選擇。

4.3 視覺化方法

五、異常檢測應用

5.1 原理

DBSCAN 和 HDBSCAN 將無法歸入任何聚類的資料點標記為噪音（label = -1），這些噪音點可視為異常（outliers）[來源：How to do Anomaly Detection using Machine Learning in Python, 2024]。相較於孤立森林（Isolation Forest）或 One-Class SVM，密度聚類的優勢在於：

  • 無需事先設定異常比例
  • 能同時識別正常資料的群組結構
  • 對任意形狀的資料分佈有較強適應性

5.2 實際案例

以下是工業感測器資料異常檢測的完整範例 [來源：Anomaly Detection using Machine Learning in Python, 2024]：


import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
from sklearn.cluster import DBSCAN
from sklearn.preprocessing import StandardScaler

# 1. 模擬工業感測器資料（溫度、壓力、振動）
np.random.seed(42)

# 正常運作資料（3 個感測器，1000 筆）
normal_data = np.random.randn(1000, 3) * [10, 5, 2] + [75, 100, 1.5]

# 4. 識別異常點
is_anomaly = (labels == -1)
anomaly_indices = np.where(is_anomaly)[0]

print(f"偵測到異常數量: {is_anomaly.sum()}")
print(f"異常比例: {is_anomaly.sum() / len(sensor_data) * 100:.2f}%")

# 5. 建立結果 DataFrame
results_df = pd.DataFrame(sensor_data, columns=['Temperature', 'Pressure', 'Vibration'])
results_df['Cluster'] = labels
results_df['Is_Anomaly'] = is_anomaly

# 6. 視覺化
fig = plt.figure(figsize=(15, 5))

            c='red', s=100, marker='x', label='Anomaly')

ax2.scatter(sensor_data[is_anomaly, 1], sensor_data[is_anomaly, 2],
            c='red', s=100, marker='x', label='Anomaly')
ax2.set_xlabel('Pressure (kPa)')
ax2.set_ylabel('Vibration (mm/s)')
ax2.set_title('Pressure vs Vibration')

# 子圖 3：異常檢測時序圖

實務應用建議：

  1. 參數調整：eps 和 min_samples 需根據資料特性調整，過小的 eps 可能將正常變異標記為異常
  2. 閾值設定：若需控制異常比例，可調整 HDBSCAN 的 outlier_score_ 並設定閾值
  3. 時間序列處理：對於時間序列資料，可使用滑動窗口特徵（如均值、標準差、變化率）作為聚類輸入
  4. 結合領域知識：異常檢測結果需與設備規格、運作條件結合判斷，避免誤報

六、實務最佳實踐

6.1 參數調整技巧

k-NN 距離圖找 eps 值：

k-distance 圖表是選擇 eps 的標準方法 [來源：DataCamp DBSCAN 指南, 2026]。繪製每個點到第 k 個最近鄰的距離（k = min_samples），並排序。圖中出現明顯彎曲的「膝點」位置即為建議的 eps 值：


from sklearn.neighbors import NearestNeighbors
import matplotlib.pyplot as plt
import numpy as np

# 計算 k-NN 距離
k = 5  # 與 min_samples 一致
neighbors = NearestNeighbors(n_neighbors=k)
neighbors_fit = neighbors.fit(X_scaled)
distances, indices = neighbors_fit.kneighbors(X_scaled)

# 排序第 k 個鄰居的距離
distances = np.sort(distances[:, k-1], axis=0)

# 繪製 k-distance 圖
plt.figure(figsize=(10, 6))
plt.plot(distances)
plt.ylabel(f'{k}-NN Distance')
plt.xlabel('Data Points (sorted by distance)')
plt.title(f'{k}-Distance Graph for DBSCAN eps Selection')
plt.grid(True)

# 手動標記膝點（示範）
knee_point = 150  # 實務上需目視判斷或使用 kneed 套件自動偵測
plt.axhline(y=distances[knee_point], color='r', linestyle='--',
            label=f'Suggested eps={distances[knee_point]:.3f}')
plt.axvline(x=knee_point, color='r', linestyle='--')
plt.legend()
plt.show()

print(f"建議 eps 值: {distances[knee_point]:.3f}")

自動膝點偵測（使用 kneed 套件）：


from kneed import KneeLocator

kl = KneeLocator(range(len(distances)), distances, curve='convex', direction='increasing')
knee_index = kl.knee

if knee_index:
    suggested_eps = distances[knee_index]
    print(f"自動偵測建議 eps: {suggested_eps:.3f}")
else:
    print("未偵測到明顯膝點，建議手動調整")

min_samples 經驗法則：

根據實務經驗 [來源：Mastering DBSCAN with Scikit-learn, 2024]：

  • 一般規則：min_samples ≥ 資料維度 × 2
  • 低維資料（2D-3D）：min_samples = 4-5 即可
  • 高維資料（10D+）：增加至 10-20，避免「維度詛咒」
  • 噪音較多：增加 min_samples 提高聚類穩健性
  • 小型資料集（< 1000 點）：min_samples = 3-5，避免過度分割

試驗策略（Grid Search）：


from sklearn.metrics import silhouette_score

# 定義參數範圍
eps_values = np.linspace(0.3, 1.0, 10)
min_samples_values = [3, 5, 7, 10]

best_score = -1
best_params = {}

for eps in eps_values:
    for min_samples in min_samples_values:
        dbscan = DBSCAN(eps=eps, min_samples=min_samples)
        labels = dbscan.fit_predict(X_scaled)

        # 排除噪音點計算分數
        mask = labels != -1
        if len(set(labels[mask])) > 1:  # 至少 2 個聚類
            score = silhouette_score(X_scaled[mask], labels[mask])

            if score > best_score:
                best_score = score
                best_params = {'eps': eps, 'min_samples': min_samples}

print(f"最佳參數: eps={best_params['eps']:.3f}, min_samples={best_params['min_samples']}")
print(f"最佳 Silhouette Score: {best_score:.3f}")

6.2 DBSCAN vs HDBSCAN 選擇指南

根據資料特性選擇合適演算法 [來源：HDBSCAN vs DBSCAN - Daily Dose of Data Science, 2024]：

選擇 DBSCAN 的情境：

  • 資料集具有均勻密度分佈
  • 需要明確控制聚類密度（透過 eps）
  • 資料集較小（< 10,000 點），執行速度優先
  • 需要與其他工具整合（sklearn 生態系完全相容）

選擇 HDBSCAN 的情境：

  • 資料集具有變化的密度分佈
  • 不確定合適的 eps 值，希望演算法自動決定
  • 需要預測新資料點的聚類標籤
  • 需要 soft clustering（成員分數）
  • 需要視覺化聚類階層結構（condensed tree）

效能比較：

| 特性 | DBSCAN | HDBSCAN |

|------|--------|---------|

| 時間複雜度 | O(n log n) | O(n log n) |

| 空間複雜度 | O(n) | O(n²) (condensed tree) |

| 參數數量 | 2（eps, min_samples） | 1（min_cluster_size） |

| 處理變化密度 | ❌ | ✅ |

| 預測新點 | ❌ | ✅ |

| 階層視覺化 | ❌ | ✅ |

6.3 常見陷阱

陷阱 1：未縮放特徵

DBSCAN 依賴距離計算，不同量級的特徵會導致某些維度主導聚類結果 [來源：scikit-learn DBSCAN 文件, 2024]。


# ❌ 錯誤示範：直接使用原始資料
dbscan = DBSCAN(eps=0.5, min_samples=5)
labels = dbscan.fit_predict(X)  # X 未縮放

# ✅ 正確做法：特徵縮放
from sklearn.preprocessing import StandardScaler
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)
labels = dbscan.fit_predict(X_scaled)

陷阱 2：參數選擇不當

過小的 eps 或過大的 min_samples 會將大部分點標記為噪音：


# 檢查噪音比例
n_noise = list(labels).count(-1)
noise_ratio = n_noise / len(labels)

if noise_ratio > 0.3:
    print(f"警告：噪音比例過高 ({noise_ratio:.1%})，建議調整參數")
    print("建議：增加 eps 或減少 min_samples")

陷阱 3：忽略噪音點

在評估聚類品質時，若不排除噪音點會導致錯誤的分數計算：


# ❌ 錯誤：包含噪音點
score = silhouette_score(X_scaled, labels)

# ✅ 正確：排除噪音點
mask = labels != -1
if len(set(labels[mask])) > 1:
    score = silhouette_score(X_scaled[mask], labels[mask])

陷阱 4：高維資料的「維度詛咒」

在高維空間中，所有點之間的距離趨於相似，DBSCAN 效能下降。建議使用降維技術（PCA、t-SNE）：


from sklearn.decomposition import PCA

# 降維至 10 個主成分
pca = PCA(n_components=10)
X_reduced = pca.fit_transform(X_scaled)

# 在降維後的資料執行 DBSCAN
dbscan = DBSCAN(eps=0.5, min_samples=5)
labels = dbscan.fit_predict(X_reduced)

陷阱 5：資料不平衡

若資料集包含大小差異極大的聚類，單一 eps 值可能無法適應所有聚類。此情境建議使用 HDBSCAN：


# 資料不平衡時優先使用 HDBSCAN
clusterer = hdbscan.HDBSCAN(min_cluster_size=5)
clusterer.fit(X_scaled)
labels = clusterer.labels_

    ax4.plot(xy[:, 2], xy[:, 3], 'o', markerfacecolor=tuple(col),
             markeredgecolor='k', markersize=6)
ax4.set_title('DBSCAN (Features 2-3)')
ax4.set_xlabel('Petal Length (scaled)')
ax4.set_ylabel('Petal Width (scaled)')

# 子圖 5：HDBSCAN 成員分數
ax5 = fig.add_subplot(235)
scatter = ax5.scatter(X_scaled[:, 2], X_scaled[:, 3], c=hdbscan_probs,
                      cmap='plasma', s=50, edgecolors='k', linewidths=0.5)
plt.colorbar(scatter, ax=ax5, label='Membership Probability')
ax5.set_title('HDBSCAN Membership Scores')
ax5.set_xlabel('Petal Length (scaled)')
ax5.set_ylabel('Petal Width (scaled)')

# 子圖 6：聚類數量比較
ax6 = fig.add_subplot(236)
algorithms = ['DBSCAN', 'HDBSCAN', 'Ground Truth']
cluster_counts = [n_clusters_dbscan, n_clusters_hdbscan, len(set(y_true))]
noise_counts = [n_noise_dbscan, n_noise_hdbscan, 0]

x = np.arange(len(algorithms))
width = 0.35

bars1 = ax6.bar(x - width/2, cluster_counts, width, label='Clusters')
bars2 = ax6.bar(x + width/2, noise_counts, width, label='Noise Points')

ax6.set_ylabel('Count')
ax6.set_title('Cluster Count Comparison')
ax6.set_xticks(x)
ax6.set_xticklabels(algorithms)
ax6.legend()

plt.tight_layout()
plt.savefig('clustering_comparison.png', dpi=150, bbox_inches='tight')
plt.show()

# ========================================
# 8. HDBSCAN Condensed Tree 視覺化
# ========================================
print("\nStep 8: HDBSCAN Condensed Tree")
hdbscan_clusterer.condensed_tree_.plot(select_clusters=True,
                                        selection_palette=sns.color_palette('Set2'))
plt.title('HDBSCAN Condensed Tree with Cluster Selection')

print("\nStep 9: 結果匯出")

# 儲存結果
results_df.to_csv('clustering_results.csv', index=False)
