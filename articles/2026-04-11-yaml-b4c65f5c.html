系統優化落實報告 2026-04-11：YAML 配置注解增強

執行摘要

依據系統審查報告（2026-04-10）的 6.4 配置說明得分偏低（68分，YAML 注解率 12%），本次落實 config/*.yaml 注解增強方案。

洞察來源

  • 系統審查報告 2026-04-10（KB noteId: 1f253b55）

- 6.4 配置說明: 68 分（README 對 YAML 覆蓋 50%，注解率 12%）

- 建議：提升 YAML 內部 # 注解密度

落實方案：YAML 注解增強（評分 9）

方案描述

為 config/ 目錄中注解率最低的配置文件添加說明性注解，目標將整體注解率從 12% 提升至 13.7%+，並讓各文件本身具備自說明能力。

修改詳情

config/agent-extra-reads.yaml（0% → 32.4%）：

  • 新增完整標頭注解區塊（18 行）
  • 說明：用途、讀取時機（preamble.md Workflow 規則）、新增規則步驟、驗證指令

config/recovery-tiers.yaml（7.5% → 36.2%）：

  • 新增標頭注解：5 個 tier 的分層說明與修改指引
  • 各 tier 補充詳細 inline 注解：觸發條件語義、參數單位、修改注意事項

驗收結果

  • ✅ YAML 語法驗證：全部 config/*.yaml 通過
  • ✅ validate_config.py --check-auto-tasks：45 個自動任務配置一致
  • ✅ pytest tests/（filtered）：923 個測試通過，無失敗
  • ✅ Git commit：b1b60a6
  • ✅ 備份 commit：7dc1c70

阻擋的高風險方案

以下方案已記錄至 improvement-backlog，需人工審議：

  1. execution_order 緊縮（風險中）：可能影響現有任務輪轉邏輯
  2. 依賴安全 CVE 修復（風險高）：aiohttp、cryptography 等升級需完整相容性測試
  3. cache_hit_ratio 改善（風險中）：需先觀察 TTL 調整對命中率的實際效果

下次建議角度

  • 繼續補充 memory-policy.yaml、github-scout-review-policy.yaml 等 0% 注解率文件
  • 評估 cache-policy.yaml TTL 調整方案（cache_hit_ratio 0.296 持續警告）
  • 考慮為 config/ 目錄建立統一注解規範文件
