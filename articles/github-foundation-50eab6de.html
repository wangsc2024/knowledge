核心內容摘錄：

  • 本研究以多模態大語言模型在異質資料上的跨域推理為核心，重新檢視現有研究計畫後，改以 arXiv、ACL Anthology、CVPR、Google Research、OpenReview 與 NVIDIA 官方技術資料補強證據鏈。整體結論很清楚：跨域推理的主要瓶頸，已不只是模型參數量，而是系統能否把文字、表格、圖表、圖像與版面結構一起納入檢索、壓縮與推理流程 [來源：Ask in Any Modality: A Comprehensive Survey on Multimodal Retrieval-Augmented Generation, 2025] [來源：RAG-Anything: All-in-One RAG Framework, 2025]。MuRAG、VisDoMRAG、RagVL 等工作共同顯示，只要外部多模態證據被正確檢索與融合，知識密集任務的正確率會明顯優於單純文字 RAG 或長上下文硬塞法 [來源：MuRAG: Multimodal Retrieval-Augmented Generator, 2022] [來源：VisDoM: Multi-Document QA with Visually Rich Elements Using Multimodal Retrieval-Augmented Generation, 2025] [來源：VLM Is a Strong Reranker, 2025]。另一方面，Qwen2-VL、Qwen2.5-VL 與 LaCo 說明效率優化的主軸已轉向動態解析度與視覺 token 壓縮，而不是固定縮圖或盲目增大 context [來源：Qwen2-VL, 2024] [來源：Qwen2.5-VL Technical Report, 2025] [來源：LaCo, 2025]。但 MMMU 仍證明，即使是強模型，在跨學科多模態推理上仍遠未達專家水準，因此 production 系統必須依賴檢索、reranking、query decomposition 與可驗證引用，而不能只信任模型裸推理 [來源：MMMU, 2024]。
  • 多模態模型近兩年進步很快，但真實世界的跨域推理任務比 benchmark 更複雜。企業與研究資料通常不是純文字，而是混合了段落、表格、圖表、公式、掃描頁與示意圖；若系統只檢索正文文字，答案往往會忽略真正関鍵的證據 [來源：RAG-Anything: All-in-One RAG Framework, 2025] [來源：Build AI-Ready Knowledge Systems Using 5 Essential Multimodal RAG Capabilities, 2026]。ACL 2025 的 Multimodal RAG 綜述進一步指出，RAG 雖能緩解靜態訓練資料造成的過時知識與幻覺，但一旦進入多模態場景，難點就從「能不能取到資料」升級為「不同模態是否能被對齊、排序、融合並保持可驗證性」 [來源：Ask in Any Modality: A Comprehensive Survey on Multimodal Retrieval-Augmented Generation, 2025]。
  • 對本主題而言，研究價值不在於再找一個更大的視覺語言模型，而是理解跨域推理為何在文件、圖表與多來源資料環境中失真，以及怎樣的系統設計最能兼顧正確率、延遲與可部署性。這也是本報告把焦點放在三條主線上的原因：多模態檢索、視覺 token 效率、以及跨模態融合與 reranking。
  • 第一，多模態外部記憶已經從加分項變成基線能力。MuRAG 在 2022 年就證明，把圖文混合內容放進外部非參數記憶，能讓模型在需要同時檢索影像與文字證據的 WebQA 與 MultimodalQA 任務上，相對競品提升超過 10% 準確率 [來源：MuRAG: Multimodal Retrieval-Augmented Generator, 2022]。到 2025 年，RAG-Anything 更直接指出，現實知識庫本來就是多模態的，如果系統仍假設所有知識都能被扁平化成純文字，長文件與複雜文件的檢索就會系統性漏失 [來源：RAG-Anything: All-in-One RAG Framework, 2025]。這與 ACL 的總覽互相印證：多模態 RAG 的價值，不只是讓模型「看到圖片」，而是讓它能在推理時真正引用跨模態證據 [來源：Ask in Any Modality: A Comprehensive Survey on Multimodal Retrieval-Augmented Generation, 2025]。
