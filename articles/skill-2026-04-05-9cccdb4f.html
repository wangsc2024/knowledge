關鍵洞察

  1. SKILL.md 可以是引導型文件：不一定要包含完整實作細節，可以引導使用者使用更成熟的工具（如 Python 實作）
  2. Draft 狀態的 Skill 需要定期檢視：本次審查發現 status="draft" 但實際功能已整合，需要同步更新狀態
  3. cache-ttl 語義分析：品質評估、健康檢查等動態資料應設 "0min"；靜態內容（如習慣提示）可設 "N/A"
  4. 實作方式演化需同步文件：當實作方式從 Bash 演化為 Python 工具時，SKILL.md 應同步更新，避免誤導

---

後續建議

  1. 定期審查 draft 狀態 Skill：建議每月執行一次 grep -r 'status: "draft"' skills/ 檢視哪些 Skill 尚未完成整合
  2. 建立 Skill 演化追蹤機制：當 Skill 實作方式改變時（如 Bash → Python），自動提醒更新 SKILL.md
  3. cache-ttl 規範化：建立 cache-ttl 設定指引，明確哪些類型 Skill 應設 "0min"、"N/A" 或具體時間
