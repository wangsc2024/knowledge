KV Cache 壓縮與長上下文推理優化

主題概述

隨著 LLM 上下文窗口擴展至百萬 token 級別，KV Cache 已成為推理階段最大的記憶體瓶頸。以 70B 參數模型處理 200K token 為例，KV Cache 單獨即消耗 40-80 GB GPU 記憶體（佔總推理記憶體 70%）。2026 年湧現多項突破性技術，涵蓋五大優化方向，本文系統整理最新進展與工程實踐。

---

一、五大 KV Cache 優化方向（2026 分類框架）

根據最新綜述論文（arXiv:2603.20397），KV Cache 優化可分為：

1. Cache Eviction（快取驅逐）

  • 核心思路：依注意力分數選擇性丟棄低重要性 token
  • 代表方法：

- H2O（Heavy-Hitter Oracle）：追蹤累積注意力分數，辨識並保留「重擊手」token，達 5-10× 記憶體縮減，吞吐量提升最高 29×

- SnapKV：在編碼階段透過投票機制識別關鍵 token，同時保留周圍上下文，達 8.2× 記憶體縮減、3.6× 吞吐量提升

2. Cache Compression（快取壓縮）

  • 量化壓縮：

- KIVI：非對稱 2-bit 量化，峰值記憶體減少 2.6×，準確度損失 <2%

- TurboQuant（Google）：結合 W&B 評估的 KV Cache 壓縮流水線

  • 低秩壓縮：

- MLA（Multi-Head Latent Attention）：DeepSeek-V2 首創，將完整 K/V 向量壓縮至低維潛在向量後快取，再按需解壓。KV Cache 縮減 93.3%（Absorb 模式達 98.6%），生成吞吐量提升 5.76×

- MHA2MLA-VLM（2026 新）：將既有視覺語言模型轉換為 MLA 架構的參數高效框架，無需昂貴重訓

  • 語義分塊壓縮：

- ChunkKV：以語義區塊（非孤立 token）為壓縮單位，保留完整語言結構與上下文完整性

- KVTC（KV Cache Transform Coding）：達 20× 壓縮（特定場景 40×+），同時維持推理與長上下文準確度

3. Hybrid Memory Solutions（混合記憶體）

  • PagedAttention（vLLM）：將 KV Cache 視為虛擬記憶體分頁，支援多序列高效共享，無損準確度
  • ShadowKV：低秩 Key Cache 留 GPU + Value Cache 卸載至 CPU，配合即時重建最小稀疏 KV 對
  • TailorKV：逐層自適應量化 + 稀疏策略，GPU 記憶體減少 73.8%，吞吐量提升 8-18×

4. Dynamic Sparse Attention（動態稀疏注意力）

  • DHSA（Dynamic Hierarchical Sparse Attention）：

- 依資料驅動自適應分割序列為變長區塊，與輸入注意力模式匹配

- Prefill 延遲降低 20-60%，峰值記憶體減少 35%

- 相較靜態 block sparse，準確度提升 6-18%

  • RocketKV（兩階段壓縮）：

- 第一階段：SnapKV 粗粒度永久驅逐（pooling kernel=63）

- 第二階段：混合稀疏注意力（HSA）在序列和注意力頭兩個維度同時細粒度篩選

- 壓縮比最高 400×，端到端加速 3.7×，解碼峰值記憶體減少 32.6%

- 自適應壓縮分配公式：r = min(0.2 + 0.06 × log₂(c), 0.8)

- RocketKV-MT 變體：跨多輪對話保留完整 KV 歷史

  • FAST-Prefill：首個 FPGA 加速器，針對長上下文 prefill 階段的動態稀疏注意力

5. Novel Attention Mechanisms（新型注意力機制）

  • KIMI Linear：混合注意力方法，百萬 token 序列吞吐量提升 6×
  • 線性注意力/對數線性注意力：替代 softmax，降低計算複雜度

三、2026 年研究趨勢與未來方向

  1. 自適應多階段流水線：根據工作負載特徵動態組合多種優化技術，無單一技術在所有場景佔優
  2. 硬軟體協同設計：利用 CSD（計算型儲存裝置）、FPGA 等專用硬體進行近儲存計算
  3. MLA 架構擴散：從 DeepSeek 系列擴展至視覺語言模型（MHA2MLA-VLM），降低遷移成本
  4. 百萬 token 級專用優化：1-2M token 窗口已成標準，壓縮與快取是關鍵創新點
  5. 多輪對話場景：RocketKV-MT 等方案解決永久驅逐策略在多輪場景的侷限

---

四、工程實踐啟示

  • 部署選型：無單一萬能方案，需依硬體限制、延遲要求、上下文長度選擇最適技術組合
  • 量化 + 稀疏雙管齊下：TailorKV 證明逐層結合量化與稀疏可達最佳性價比
  • MLA 為新模型首選：新訓練的模型應優先考慮 MLA 架構，既有模型可透過 MHA2MLA 轉換
  • FlashAttention 相容性：RocketKV、Dynamic Sparse Flash Attention 等均已與 FlashAttention 無縫整合
