GitHub Scout：Agent 邊界與護欄專案觀測（2026-04-07）

專案觀測摘要

  • OpenHands/OpenHands：70.7k stars，最新 release 2026-03-30；強項是 agent runtime、CLI 與 skills 共存，適合作為多工具代理的主控參考。
  • browser-use/browser-use：86.2k stars，最新 release 2026-04-02；代表瀏覽器工具 API 邊界，提醒 runtime sandbox 與 browser API 必須分層描述。
  • e2b-dev/E2B：11.6k stars，最新 release 2026-04-02；核心價值是 secure sandbox 與 capability-based execution boundary。
  • guardrails-ai/guardrails：6.6k stars，最新 release 2026-04-03；顯示 guardrails 最適合以輸入/輸出 validator 與政策鏈來落地。
  • langfuse/langfuse：24.4k stars，最新 release 2026-03-31；可借鏡 trace/eval/prompt governance 的審計做法。

改進提案

  1. P0：為 github_scout 的 research-registry probe 增加 PowerShell fallback，避免 uv 權限失敗時失去去重與前處理可觀測性。
  2. P1：規劃 operator_override、tool_boundary 與 run_metadata 的結構化 trace linkage，補齊 guardrail 決策與 rollback 稽核。

審查結論

  • P0 提案可行性 9/10、穩定性 9/10，風險低，已直接落實。
  • P1 提案可行性 7/10、穩定性 7/10，但會影響日誌/schema 消費端，需人工審核後再推進。
  • 本輪主題命中 Agent 安全、Tool Use、Guardrails 與 Sandbox，因此逐案補齊了 guardrail_surface、tool_boundary、operator_override。

已落實項目

  • 已修改 emplates/auto-tasks/github-scout.md 與 skills/github-scout/SKILL.md，加入 uv 失敗時的 PowerShell registry fallback 與 preprocess 紀錄要求。

待人工審核項目

  • docs/plans/github-scout-20260407-operator-override-trace-linkage.md：是否允許未來擴充結構化 override/trace 事件鏈。

專案證據與限制

  • 本次於 2026-04-07 直接觀測 GitHub repo 頁；stars 與最近更新以 repo 頁顯示的 stars 與 Latest release 為準。
