一、專案概述

Crawl4AI 是目前 GitHub 上最熱門的 LLM 友善網頁爬蟲開源專案，截至 2026 年 2 月已累積 61,000+ Stars、6,200+ Forks，最新版本 v0.8.0，採 Apache-2.0 授權。它於 2024 年中啟動，不到一年即登上 GitHub Trending 第一名，定位為「Scrapy for LLMs」。

核心價值：將任意網頁轉換為 LLM 可直接消化的 Clean Markdown，無需外部 API 呼叫，完全本地運行。這解決了 AI Agent 和 RAG 管線中最關鍵的資料擷取瓶頸——從「如何抓取網頁」轉變為「如何讓網頁內容對 LLM 有意義」。

二、技術架構（四層分層設計）

2.1 架構總覽

Crawl4AI 採用清晰的四層架構：


┌─────────────────────────────────────────┐
│  Layer 4: API & MCP 整合層              │
│  REST API / MCP Server / Docker         │
├─────────────────────────────────────────┤
│  Layer 3: 資料提取策略層                │
│  LLM / CSS+XPath / Clustering / BM25   │
├─────────────────────────────────────────┤
│  Layer 2: 內容處理層                    │
│  Markdown 生成 / Fit Markdown / 分塊    │
├─────────────────────────────────────────┤
│  Layer 1: 瀏覽器管理層                  │
│  Playwright / Browser Pool / 反偵測     │
└─────────────────────────────────────────┘

2.2 瀏覽器管理層（Layer 1）

  • Managed Browser Pooling：非同步瀏覽器池管理，支援並行爬取
  • Remote CDP 支援：透過 Chrome DevTools Protocol 連接遠端瀏覽器
  • Session 持久化：跨請求維護登入狀態和 Cookie
  • Stealth Mode：內建反機器人偵測規避機制
  • 多瀏覽器相容：Chromium、Firefox、WebKit

2.3 內容處理層（Layer 2）

  • Clean Markdown：將 HTML 轉為結構化 Markdown，保留語義層級
  • Fit Markdown：啟發式過濾，僅保留頁面核心內容（去除導航列、頁尾等）
  • BM25 內容過濾：依查詢相關性排序段落，僅保留相關內容
  • 引用格式化：自動將頁面連結轉為引用格式

2.4 資料提取策略層（Layer 3）

三種提取策略可組合使用：

| 策略 | 需 LLM | 適用場景 |

|------|--------|----------|

| CSS/XPath Schema | 否 | 結構化 HTML（表格、列表） |

| RegexExtraction | 否 | 快速匹配（Email、URL） |

| LLM Extraction | 是 | 複雜版面、自由文本 |

| Clustering | 否 | 跨頁面模式識別 |

Adaptive Intelligence（v0.8 核心特色）：

  • 爬蟲會「學習」可靠的 CSS 選擇器
  • 信心分數隨爬取次數提升
  • 自動偵測版面變更並調整策略
  • 實現「零配置高品質提取」的目標

2.5 API 與整合層（Layer 4）

  • REST API：Docker 部署後提供標準 HTTP 端點
  • MCP Server：原生支援 Model Context Protocol

- 透過 SSE 或 stdio 連接 Claude Code、Cursor 等 AI 工具

- 暴露 4 個工具：scrape、crawl、crawl_site、crawl_sitemap

- 整合指令：claude mcp add --transport sse c4ai-sse http://localhost:11235/mcp/sse

  • MCP + RAG：社群已有 crawl4ai-mcp-server 與 mcp-crawl4ai-rag 整合 Supabase 的方案

三、功能特色（與同類工具差異化）

3.1 與 Firecrawl 比較

| 面向 | Crawl4AI | Firecrawl |

|------|----------|----------|

| 定位 | 本地優先的開源爬蟲 | API 優先的雲端服務 |

| 部署 | 自託管（完全掌控） | SaaS + 開源版 |

| 語言 | Python 原生 | Python/JS SDK |

| 核心特色 | Adaptive Intelligence | FIRE-1 導航 Agent |

| 成本 | 完全免費 | 付費（開源版有限制） |

| 效能 | 中高（Python 瓶頸） | 中（雲端延遲） |

| LLM 整合 | 本地模型（Ollama）+ 雲端 API | 主要雲端 API |

3.2 與 Spider 比較

Spider（Rust 引擎）在靜態頁面吞吐量上是 Crawl4AI 的 9.5 倍，但在 JS 密集頁面差距縮小（瀏覽器為瓶頸）。Crawl4AI 的優勢在於 Python 生態整合和 Adaptive Intelligence。

3.3 獨特賣點

  1. 完全本地執行：資料不離開你的伺服器，適合資料主權敏感場景
  2. Adaptive Intelligence：爬蟲越爬越聰明，自動學習最佳選擇器
  3. LLM 友善輸出：Markdown 格式直接餵給 RAG 管線或 Agent
  4. Docker 一鍵部署：內建監控儀表板和互動式 Playground
  5. MCP 原生支援：直接成為 Claude Code 的工具

四、快速上手

4.1 安裝


pip install -U crawl4ai
crawl4ai-setup       # 安裝瀏覽器依賴
crawl4ai-doctor      # 驗證安裝

4.2 基本爬取


import asyncio
from crawl4ai import AsyncWebCrawler

async def main():
    async with AsyncWebCrawler() as crawler:
        result = await crawler.arun(url="https://example.com")
        print(result.markdown)     # Clean Markdown
        print(result.fit_markdown) # 過濾後的核心內容

asyncio.run(main())

4.3 LLM 提取


from crawl4ai.extraction_strategy import LLMExtractionStrategy

strategy = LLMExtractionStrategy(
    provider="ollama/llama3.2",  # 本地模型
    instruction="Extract product names and prices"
)

result = await crawler.arun(
    url="https://shop.example.com",
    extraction_strategy=strategy
)

4.4 Docker 部署


docker run -d -p 11235:11235 unclecode/crawl4ai:latest
# 存取 http://localhost:11235 查看 Playground

4.5 RAG 管線整合（Milvus 範例）


from crawl4ai import AsyncWebCrawler
from pymilvus import MilvusClient

# 步驟 1：爬取並轉為 Markdown
async with AsyncWebCrawler() as crawler:
    result = await crawler.arun(url="https://docs.example.com")

# 步驟 2：分塊 + 向量化
chunks = chunk_text(result.fit_markdown)
embeddings = embed(chunks)

# 步驟 3：存入向量資料庫
client = MilvusClient(uri="http://localhost:19530")
client.insert(collection_name="docs", data=embeddings)

六、潛在應用（Daily-Digest-Prompt 專案）

6.1 新聞擷取強化

目前 fetch-news.md 和 fetch-hackernews.md 使用 curl + API。Crawl4AI 可以：

  • 爬取無 API 的新聞網站（如地方新聞、技術部落格）
  • 自動將新聞頁面轉為 Clean Markdown 供 LLM 摘要
  • Fit Markdown 過濾廣告和導航列

6.2 研究任務自動化

templates/auto-tasks/ 中的研究任務（tech-research、ai-deep-research 等）可以：

  • 用 Crawl4AI 爬取 GitHub README + 技術文件
  • 取代手動 WebFetch，獲得更乾淨的 Markdown
  • 結合 BM25 過濾只保留研究相關段落

6.3 MCP 整合

透過 Crawl4AI MCP Server：

  • 在 Claude Code 中直接呼叫 scrape 和 crawl 工具
  • 無需手動寫 curl 指令，AI Agent 自動決定何時爬取
  • 與現有的 api-cache Skill 結合，快取爬取結果

6.4 知識庫自動擴充

結合 knowledge-query Skill：

  • Crawl4AI 爬取 → Markdown 轉換 → 自動匯入知識庫
  • 實現「發現新內容 → 爬取 → 去重 → 匯入」的自動管線

七、優缺點評估

優點

  1. Stars 數量驚人（61K+），社群信任度高
  2. 完全開源免費，Apache-2.0 授權
  3. Adaptive Intelligence 是真正的差異化特色
  4. MCP 原生支援，與 Claude Code 無縫整合
  5. 本地優先，不依賴外部 API，資料主權有保障
  6. Python 生態整合，pip 安裝即用
  7. Docker 支援完善，含監控和 Playground

缺點

  1. 效能不如 Rust 方案（Spider 快 9.5 倍在靜態頁面）
  2. 瀏覽器依賴：Playwright 安裝增加部署複雜度
  3. 記憶體消耗：並行爬取時瀏覽器池佔用大量 RAM
  4. 反爬蟲對抗：Stealth Mode 非萬能，Cloudflare 等進階防護仍有挑戰
  5. 版本仍在 0.x：API 可能不穩定，breaking changes 風險
  6. Windows 支援：部分功能在 Windows 上表現不穩定（Linux/macOS 最佳）
