Vite 遊戲門戶網站開發經驗

專案背景

使用 Vite 建置遊戲門戶網站，部署至 Cloudflare Pages。包含 8 個 HTML5 遊戲，使用 ES6 模組動態生成遊戲卡片。

關鍵問題與解決方案

1. ES6 模組導入/導出問題

問題： 遊戲卡片完全不顯示，打包後 getAllGames() 未定義

根因：

  • 使用 CommonJS 的
  • 未正確導入模組，依賴全域變數檢查
  • Vite 打包時，CommonJS 導出在瀏覽器環境無法正確運作

解決方案：

關鍵要點：

  • Vite 是 ES6 模組優先的建置工具
  • 瀏覽器環境必須使用 ES6
  • CommonJS 的 僅適用於 Node.js 環境

2. 遊戲目錄結構規範

問題： blockPuzzle 遊戲路徑錯誤，無法正常載入

根因：

  • blockPuzzle 路徑為 4 層：
  • 其他遊戲都是 2 層：
  • 路徑不一致造成導航和資源載入問題

解決方案： 統一所有遊戲為 2 層結構

最佳實踐：

  • 所有遊戲保持相同的目錄深度
  • 避免超過 3 層的深層嵌套
  • 路徑一致性有助於維護和擴展

3. 相對路徑返回連結

問題： 遊戲內的「返回」連結指向錯誤的 URL

根因：

  • 使用 只能回到 目錄
  • 遊戲在 ，需要回到根目錄

解決方案：

相對路徑規則：

  • = 上一層目錄
  • = 上兩層目錄
  • = 根目錄（絕對路徑）

  1. 新增遊戲時：

- 確保 2 層目錄結構 - 檢查返回連結使用 - 更新 - 執行 並本地測試

- ES6 模組正確導入/導出 - 所有路徑使用相對路徑 - 本地 測試 - Git 提交訊息清晰
