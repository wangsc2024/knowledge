Chat 對話系統規劃報告 v1.1

日期：2026-03-17 | 狀態：規劃中（待確認後實作）

核心目標

在現有 my-gun-relay（Gun.js + SEA E2E 加密）基礎上，建構具備 Function Calling（工具調用）能力的 Chat 對話系統。使用者透過自然語言與 LLM 對話，LLM 自主決定呼叫後端工具並整合結果回覆。

設計原則

  • 漸進式升級：在現有 Gun.js 基礎上疊加，不重寫
  • Skill-First：工具系統復用 daily-digest-prompt 的 Skills 生態
  • SSE + WebSocket 混合：LLM Streaming 用 SSE，即時聊天用 WebSocket
  • Human-in-the-Loop：敏感工具呼叫需使用者確認
  • 本地優先：所有元件本地部署

技術棧

| 層級 | 技術 |

|------|------|

| 後端 | FastAPI (Python, uv) |

| 前端 | React + Vite + TypeScript + Tailwind CSS |

| LLM | Groq llama-3.3-70b（主）+ Claude（備） |

| Streaming | SSE (Server-Sent Events) |

| 資料庫 | SQLite (aiosqlite) |

| 即時聊天 | 保留 Gun.js WebSocket |

API 架構

核心端點

  • POST /api/chat — SSE Streaming 對話（支援 tool_call 事件）
  • GET /api/conversations — 列出對話
  • GET /api/conversations/{id} — 完整歷史
  • GET /api/tools — 列出工具
  • POST /api/chat/confirm — Human-in-the-Loop 確認

SSE 事件類型

conversation_created → message_saved → thinking → tool_call → tool_result → token (逐字) → usage → done

Function Calling 工具系統

工具定義格式（相容 OpenAI）

每個工具含：name, description, parameters (JSON Schema), requires_confirmation, category, timeout_seconds

內建工具（第一階段 9 個）

| 工具 | 分類 | 確認 |

|------|------|------|

| knowledge_query | query | 否 |

| knowledge_import | action | 是 |

| todoist_list_tasks | query | 否 |

| todoist_create_task | action | 是 |

| todoist_complete_task | action | 是 |

| pingtung_news | query | 否 |

| web_search | query | 否 |

| get_current_time | system | 否 |

| calculate | system | 否 |

工具執行流程

使用者訊息 → LLM 推理 → tool_calls? → 解析並行呼叫 → 確認機制 → 執行 → 結果回傳 LLM → 再次推理（遞迴上限 5 次）→ 最終 SSE Streaming 回覆

工具註冊機制（Plugin 式）

ToolRegistry 類別支援動態註冊與發現，list_tools() 輸出 OpenAI 格式供 LLM 使用，execute() 含超時控制。

前端介面

  • 佈局：Sidebar（對話列表）+ Main Area（訊息流 + 輸入框）
  • 訊息類型：使用者、助手、工具呼叫卡片（可摺疊）、確認請求、錯誤、系統
  • 風格：Glassmorphism（Deep navy + Indigo + Emerald），延續現有 index.html
  • 響應式：768px 斷點

LLM 整合

  • 雙模型路由：一般用 Groq，複雜推理用 Claude
  • 上下文管理：滑動窗口 50 條 + 摘要壓縮（超 30 條自動摘要舊訊息）
  • 標題自動生成：Groq 8B 快速生成 ≤20 字標題

資料模型（SQLite）

3 張表：conversations、messages（含 tool_calls/tool_call_id/tool_name）、tool_executions（執行日誌）

安全設計

  • Prompt Injection：工具輸入驗證 + 結果截斷 + System Prompt 強化
  • 工具濫用：requires_confirmation + 頻率限制 + 最大迭代上限
  • XSS：DOMPurify
  • 認證：Bearer Token + sessionStorage + CORS localhost
  • SSRF：工具 URL 白名單

實作階段

  • Phase 1 MVP（2-3天）：FastAPI + SQLite + 2 工具 + 極簡 React
  • Phase 2 核心（2-3天）：多對話 + 完整工具集 + 確認機制 + 雙模型
  • Phase 3 優化（1-2天）：自動標題 + 搜尋 + 工具管理 + Token 統計
  • Phase 4 進階（可選）：Gun.js 橋接 + MCP + 自訂工具 + 語音

Token 成本

日常以 Groq 免費額度為主，月成本約 $15（僅複雜推理 fallback Claude）。

補充設計

  • 錯誤碼：11 個標準化錯誤碼（400-503）
  • 可觀測性：JSONL 結構化日誌 + 健康檢查端點含依賴狀態
  • 快取：知識庫搜尋 TTLCache 5 分鐘
  • Gun.js 橋接：FastAPI 輪詢 Bot API pending 任務，不動加密通道
