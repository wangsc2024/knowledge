Pong v1.1 優化心得：AI 難度系統與 localStorage 排行榜實作

概述

將 Neon Pong 從 v1.0 升級到 v1.1，新增三級 AI 難度選擇（Easy/Normal/Hard）與本地排行榜系統。

AI 難度設計

三級參數差異

| 參數 | Easy | Normal | Hard |

|------|------|--------|------|

| AI 速度 | 3.0 | 4.2 | 5.8 |

| AI 誤差範圍 | 70px | 40px | 15px |

| 初始球速 | 4.5 | 5.5 | 6.5 |

| 最大球速 | 7 | 10 | 12 |

| 球速遞增 | 0.2 | 0.3 | 0.4 |

設計原則

  1. AI 誤差範圍是最影響難度的參數 — 決定 AI 追蹤球的精確度
  2. 球速遞增影響長回合的緊張感 — Hard 模式球速飆升更快
  3. Easy 模式 AI 故意「懶散」，讓新手也能獲勝
  4. Hard 模式 AI 幾乎完美追蹤，但保留 15px 誤差避免「不可能贏」

localStorage 排行榜

資料結構


{
  easy: { wins: 0, streak: 0, bestStreak: 0 },
  normal: { wins: 0, streak: 0, bestStreak: 0 },
  hard: { wins: 0, streak: 0, bestStreak: 0 }
}

連勝機制

  • 玩家勝利：streak++，若超過 bestStreak 則更新
  • 玩家敗北：streak = 0（重置當前連勝）
  • 每個難度獨立追蹤，跨難度不影響

錯誤處理

  • try/catch 包裝 localStorage 操作，避免隱私模式或配額滿時崩潰
  • 讀取失敗回傳預設值

選單 UI 改進

  • 三個霓虹風格的圓角按鈕，各自對應難度顏色（綠/黃/紅）
  • 鍵盤 ← → 或觸控點擊切換
  • 選中按鈕有發光邊框 + 三角指示器
  • 下方即時顯示當前難度的勝利統計

技術要點

  • DIFFICULTY 常量物件集中管理所有參數，易於調整
  • currentDiff 變數在 startGame() 時依選擇設定，全遊戲共用
  • Game Over 畫面顯示難度標籤和統計，強化成就感
  • 遊戲中左上角淡顯難度標示，不干擾遊玩
