Agent 架構新版本觀測報告（2026-04-24）

本次主題

Agent 架構新版本：microsoft/agent-framework v1.0 + AgentScope 顯式狀態 + Temporal signal HITL

專案觀測摘要

microsoft/agent-framework（7.1k★）

  • v1.0 於 2026-04-02 發佈（AutoGen + Semantic Kernel 統一 SDK）
  • runtime_model: 圖式工作流 + A2A 跨框架 agent-to-agent 通訊 + .NET/Python 雙語言
  • state_strategy: workflow checkpoint + time-travel 回放 + pause/resume 長期流程
  • human_review_model: interrupt-point 模式：HITL approval gates + AG-UI operator UI + time-travel 重跑
  • 新增 A2A 協議原生支援，可與 LangGraph、CrewAI 等跨框架協作

agentscope-ai/agentscope（19.6k★）

  • 2026-02 v1.1.0 重大重構（FastAPI 直接繼承）
  • runtime_model: Message-Passing + Actor 模型；HiClaw 提供 Matrix room-based 協作 OS
  • state_strategy: 每 Agent 顯式狀態（非黑盒）；ReMe kit 管理長期記憶
  • human_review_model: HiClaw Matrix room HITL - 使用者可隨時觀看並介入，無黑盒
  • agentscope-runtime 提供 Agent-as-a-Service + sandbox lifecycle 管理

temporalio/temporal（19.8k★）

  • 耐久執行平台，2026 年 agent workflow 整合快速成長
  • runtime_model: 耐久執行引擎；event sourcing；失敗後自動從中斷點繼續
  • state_strategy: 事件溯源 + 決定性重播；Temporal Service 持久化全部狀態；Saga 補償內建
  • human_review_model: signal-based 異步 HITL（workflow 阻塞等待外部 signal）；適合高延遲異步審核

  • 區分 interrupt-point（同步）vs signal-based（異步）HITL 兩種模式

兩個 P1 低風險方案均通過審查並直接落實。主要發現：2026 年 Agent 架構分化為三大路線（圖式 + HITL gates / 顯式狀態 + Matrix HITL / 耐久執行 + signal HITL），本系統可從各路線擷取模式補強。
