Context Engineering 整合實作 2026

Tool RAG 化、Routing 動態化與 Context-Compressor 自動觸發的 Synthesis

日期：2026-04-28

研究類型：insight_briefing（synthesis 深化）

摘要

2026 年 Context Engineering（CE）三大核心實作模式收斂：Tool RAG 化（Progressive Disclosure 三層載入，解決 90+ 工具 >50K token 前置成本）、Context Routing 混合策略（rule-based 粗篩 + LLM 細分的 hybrid routing）、Context Compressor 自動觸發（LangChain Deep Agents 三段觸發機制）。arxiv 2601.07190 的 Active Context Compression 研究確認：agent 主動控制壓縮（22.7% token 節省，精確度維持）優於外部強制觸發。最重要洞見：65% 企業 AI 失敗源於 Context Drift（目標漂移），而非 token 耗盡。

核心洞見

1. Tool RAG 化 — Progressive Disclosure 三層載入

  • Discovery：僅名稱+一句話描述（100-300 tokens）
  • Activation：任務相關工具完整指令（2,000-5,000 tokens）
  • Execution：完整 schema 與範例
  • 90+ 工具 >50,000 tokens，Progressive Disclosure 可大幅降低前置成本
  • 本系統 Gap：Agent Pool coordinator.py 全量注入，routing.yaml 靜態路由未達 RAG 化

2. Context Routing 混合策略

  • Hybrid = rule-based（O(1)）+ LLM classifier（語義）
  • Pre-Route 輕量 metadata 分析 → SkewRoute 檢索分數分佈代理 query hardness
  • 本系統：llm_classifier.py + llm_router.yaml 已是 hybrid，但研究子類型粒度可細化

3. Context Compressor 三段觸發（LangChain Deep Agents）

  • Tool result > 20,000 tokens → filesystem offload + 10 行 preview
  • Context > 85% capacity → older args offload
  • Offload 仍不足 → LLM 結構化摘要（session intent + artifacts + next steps）
  • 壓力測試建議從 10-20% capacity 觸發

4. Active Context Compression（arxiv 2601.07190）

  • 22.7% token 節省（14.9M → 11.5M tokens），精確度維持（60%）
  • Focus Agent：slime mold 啟發，agent 主動決定壓縮時機
  • Knowledge block 保留學習成果，raw history 逐步清除

5. Context Drift 優先於 Token 耗盡

  • 65% 企業 AI 失敗源於 drift/memory loss
  • 防 Drift：保留 error traces + structured summaries（三部分）+ needle-in-haystack 驗證

建議行動

| 優先 | 建議 | 涉及檔案 |

|------|------|----------|
