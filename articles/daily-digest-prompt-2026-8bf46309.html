Daily-Digest-Prompt 專案資安優化報告（2026-03-08）

資安優化系統設計架構決策hooks路徑遍歷競態條件敏感資訊防護

優化概述

對 daily-digest-prompt 專案執行三路並行安全審查（Python hooks / PowerShell 腳本 / 配置與 bot），識別出 3 Critical + 7 Important 問題，完成 7 項修復。

Critical 修復（P0）

P0-1：路徑遍歷防護強化

  • 檔案：hooks/pre_write_guard.py
  • 問題：_check_path_traversal 僅在路徑含 .. 時檢查，攻擊者可用絕對路徑（如 C:\Windows\System32\file）直接逃逸專案目錄
  • 修復：移除 .. 捷徑判斷，對所有路徑執行 os.path.abspath() 解析後與 project_root + os.sep 比對
  • 關鍵設計：使用 startswith(norm_root + os.sep) 而非 startswith(norm_root) 防止前綴相似目錄（如 project-evil/）繞過

P0-2：檔案鎖競態條件修復

  • 檔案：hooks/hook_utils.py 的 file_lock 類別
  • 問題：Windows LK_NBLCK（非阻塞鎖）在鎖已被佔用時拋出 OSError，但與 ImportError 一起被靜默吞掉，導致團隊並行模式下 read-modify-write 無鎖
  • 修復：改用 LK_LOCK（阻塞等待），並分離 ImportError（模組不可用）和 OSError（鎖定失敗）的處理邏輯

P0-3/P0-4：敏感檔案排除版控

  • bot/data/keypair.json：Gun.js SEA 私鑰（priv + epriv）明文存在磁碟上但未被 .gitignore 排除
  • context/behavior-patterns.json：post_tool_logger 累積的行為模式中包含 .env 讀取命令片段

Important 修復（P1）

P1-1：攔截日誌敏感資訊消毒

  • 檔案：hooks/hook_utils.py 的 log_blocked_event()
  • 問題：被攔截的命令（最可能包含敏感 token）原樣寫入 JSONL 日誌
  • 修復：summary 參數經過 sanitize_sensitive_data() 消毒後再寫入

P1-2：Loop-state 路徑修復

  • 檔案：hooks/post_tool_logger.py
  • 問題：loop-state 使用相對路徑 state/loop-state-*.json，團隊並行模式下 CWD 漂移導致迴圈偵測失效
  • 修復：改用 os.path.dirname(os.path.dirname(os.path.abspath(__file__))) 取得絕對路徑

P1-3：Token 記憶體清理

  • 檔案：hooks/on_stop_alert.py
  • 問題：token.json 完整 dict 載入記憶體，exception path 可能洩漏 access_token、client_secret
  • 修復：使用 pop() 取出唯一需要的 refresh_token 後立即 clear() + del 清除

測試驗證

  • 新增 7 個回歸測試（5 個路徑遍歷 + 2 個日誌消毒）
  • 全套 682 個測試通過，零失敗

安全審查方法論

  • 三路並行子 Agent 審查：hooks Python / PowerShell 腳本 / 配置與 bot
  • 僅回報 HIGH/MEDIUM 信心度問題
  • 分類：Critical（直接可利用）> Important（需特定條件）> 觀察（理論風險）
