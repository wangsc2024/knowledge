OpenClaw — 2026 年最熱門開源 AI 助理專案

OpenClaw：病毒式爆紅的個人 AI 助理

專案概述

OpenClaw（暱稱 "Molty"）是 2026 年最熱門的開源專案，由 PSPDFKit 創始人 Peter Steinberger 創建。專案原名 Clawdbot（2025 年 11 月發布），2026 年 1 月改名為 OpenClaw 後爆紅，在約 60 天內從 9,000 星增長到超過 25 萬星，成為 GitHub 史上成長最快的開源專案之一。

GitHub 星數：250,000+（2026 年 3 月）

創建者：Peter Steinberger（奧地利開發者）

核心特性

1. 完全本地運行

OpenClaw 是一個本地運行的個人 AI 助理，所有數據都保存在本機，永不上傳雲端。這提供了：

  • 數據隱私保護
  • 不依賴外部服務的穩定性
  • 低延遲、高安全性

2. 50+ 整合服務

專案支援超過 50 個整合，涵蓋：

  • 通訊平台：WhatsApp、Telegram、Slack、Discord、Signal、iMessage、Microsoft Teams、Matrix、IRC、Feishu、LINE、Mattermost、Nextcloud Talk、Nostr、Synology Chat、Twitch、WeChat、QQ
  • AI 模型：Ollama（本地 LLM）、Claude、GPT、Mistral
  • 生產力工具：自動化工作流、智能家居控制
  • 多平台支援：macOS、iOS、Android、Windows、Linux

3. 自我改進能力（類 AGI）

許多用戶稱 OpenClaw 具備「自我改進」特性，類似 AGI（通用人工智慧）：

  • 自主撰寫程式碼以創建新技能
  • 主動自動化任務
  • 長期記憶使用者偏好
  • 持續學習與優化

4. 全功能能力

  • 網頁瀏覽與資料抓取
  • 表單填寫自動化
  • Shell 命令執行
  • 程式碼撰寫與執行
  • 智能家居設備控制
  • 跨平台訊息整合（多通道收件匣）

架構設計

OpenClaw 的架構由三個主要組件組成：

1. Gateway（閘道）

  • 長期駐留的守護程序（daemon process）
  • 單一控制平面，管理所有：

- 訊息通道（messaging channels）

- 會話（sessions）

- 工具（tools）

- 事件（events）

  • 始終在線，作為人類、訊息通道與 Agent 執行之間的邊界

2. Agent 執行層

  • 處理任務執行與決策
  • 自主撰寫與執行程式碼
  • 工具調用與整合

3. 記憶與狀態管理

  • 長期記憶儲存
  • 使用者偏好追蹤
  • 會話狀態持久化

授權與成本

  • 完全開源：無訂閱費用
  • BYOK（Bring Your Own API Key）：使用者自備 API 金鑰
  • 無供應商鎖定
