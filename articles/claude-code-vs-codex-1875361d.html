三工具核心哲學

Claude Code（Anthropic）：以 CLI 為核心，優化「生產力感受」。Token 消耗大但體驗流暢（子 Agent 並行、動畫、即時計數器）。本質上同時是行銷工具——功能設計優先考慮 Twitter 截圖效果（如 pet mode、sub-agent 視覺化、Lo-fi radio）。模型改進停滯（Opus 4.5→4.7 被認為是退步），靠大量新功能彌補。內部員工使用 Mythos 模型和自訂版本，外部用戶得到的是閹割版。桌面 App 體驗極差（登入流程、Safari 內嵌認證崩壞）。

Codex（OpenAI）：以 App 為核心，注重實際生產力。Token 效率高（GPT-5.5 用一半 token 達更好分數）。功能追求實用而非華麗（鍵盤快捷截圖、diff 標記設定、雙擊 Command 截圖傳 context）。Computer Use 能力搭配專用 Mac Mini 驗證改動效果。OpenAI 內部千人每日使用同一版本——「你得到的就是他們用的」。開源 CLI 及 App Server 鼓勵第三方整合（T3 Code 即基於此建構）。

Cursor：Cloud Runner 遠超競爭對手——啟動完整圖形化 Linux 沙箱，Agent 可用 Computer Use 測試 UI 改動並錄影回報。適合團隊協作（Slack 整合：at cursor bot 觸發修復 + 影片回報）。本地 IDE / CLI 投資較少，ACP 綁定更新慢，SDK 最近才推出。定位：面向未來的「無本地機器」企業級解決方案。

關鍵觀點

  • Anthropic 策略：模型不夠聰明就多燒 token → 更多 sub-agent → 更多酷炫功能
  • OpenAI 策略：讓模型更有效率 → 加 Computer Use 驗證 → 減少無謂 token 消耗
  • Cursor 策略：押注遠期「雲端代理」願景 → 全圖形化沙箱 + webhook 整合
  • 模型進展不對稱：OpenAI 4 代（5.2→5.5）大幅進步，Anthropic 同期 Opus 退步兩次
  • 選擇建議：需動力/新手→Claude Code；資深工程師要實用→Codex；團隊級雲端→Cursor
