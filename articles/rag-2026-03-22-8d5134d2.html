一、執行摘要

本次針對 daily-digest-prompt 專案進行 RAG 整合與快取策略的深度優化分析。結合 2026 年業界最新研究（RAGFlow「RAG-to-Context」年度回顧、kapa.ai 100+ 團隊實踐、arXiv Agentic Plan Caching 論文），聚焦三大優化方向：（1）知識庫分層架構與二階段檢索、（2）Agentic Plan Caching 任務級快取、（3）漂移偵測與品質計分強化。

三、優化建議（6 項，P0-P2 分級）

P0-A：知識庫分層架構（Tiered Knowledge Base）

現況問題：單一 KB 索引服務所有查詢，核心文件與即時資料混合，TTL 設定無法兼顧兩者。

業界最佳實踐（kapa.ai, RAGFlow）：

  • 核心層：版本化文件（SKILL.md、CLAUDE.md、ADR），低頻更新，長 TTL（24h+）
  • 即時層：API 回應、新聞、任務狀態，短 TTL（15-120min）
  • 各層獨立 SLA 與快取策略

專案適用方案：

  1. cache-policy.yaml 新增 tier 欄位（core / realtime）
  2. 核心層查詢走 /api/search/semantic（概念匹配），即時層走 /api/search/keyword（精確詞）
  3. cache/status.json 按 tier 分別計算命中率

預期效益：核心層命中率提升至 90%+，即時層保持低延遲

P0-B：二階段檢索（Search → Retrieve 解耦）

現況問題：/api/search/hybrid 同時承擔定位與上下文組裝，大量 chunk 降低精確度。

業界最佳實踐（RAGFlow 2025 回顧）：

  • Search 階段：精確定位最相關的 3-5 個 chunk（高 precision）
  • Retrieve 階段：以定位結果為種子，擴展完整上下文段落（高 recall）

專案適用方案：

  1. 去重查詢：hybrid topK=5 → 取 noteId → /api/notes/:id 全文
  2. 研究策略分析：semantic topK=10 minScore=0.4 → 按 score 分群 → 高分群直接用、低分群摘要化
  3. templates/shared/kb-depth-check.md 修改為兩階段流程

P1-A：Agentic Plan Caching（任務級執行計畫快取）

現況問題：每次 Todoist 自動任務都從零開始規劃執行步驟，相似任務重複計算。

學術研究（arXiv 2506.14852）：

  • 從成功執行中抽取結構化 Plan 模板
  • 用關鍵詞匹配新請求與已快取計畫
  • 輕量模型適配到具體任務
  • 效益：成本降低 50.31%、延遲降低 27.28%、維持 96.61% 性能

專案適用方案：

  1. 新建 cache/plan-templates/ 目錄，按 task_type 存放成功執行的 Plan 摘要
  2. run-todoist-agent-team.ps1 Phase 1 新增 Plan Cache 查詢步驟
  3. 命中時跳過 Plan 生成，直接進入執行
  4. 與現有 context/continuity/ 連續記憶互補（continuity = 主題記憶，plan cache = 流程記憶）

預期效益：Todoist 自動任務平均執行時間縮短 20-30%

P1-B：快取健康自動修復（Cache Auto-Heal）

現況問題：cache/knowledge.json 已 9 天未更新，其他源也多數超期，但系統未自動修復。

專案適用方案：

  1. check-health.ps1 [快取效率] 區塊新增自動修復觸發：當 critical cache（knowledge）超過 2× TTL 時，自動執行 health check + 重新快取
  2. config/cache-policy.yaml 新增 auto_heal 欄位：{enabled: true, trigger_multiplier: 2, max_retries: 3}
  3. run-agent-team.ps1 Phase 0 新增快取健康前置檢查

P1-C：漂移偵測（Drift Detection）

現況問題：知識庫內容隨時間增長，但無機制偵測查詢模式變化或知識庫覆蓋度漂移。

業界最佳實踐（DEV Community 2025）：

  • 統計分佈比較：定期對比查詢 embedding 分佈與歷史基線
  • 聚類分析：發現新出現的查詢類型
  • 性能監控：準確率下降時觸發自動策展

專案適用方案：

  1. hooks/post_tool_logger.py 已記錄所有 KB 查詢，新增每週統計摘要
  2. check-health.ps1 新增 [KB 覆蓋度] 區塊：比較過去 7 天 KB 查詢主題分佈與現有 tags 覆蓋度
  3. 低覆蓋度主題自動加入 context/improvement-backlog.json

P2-A：Ingestion Pipeline 語義增強

現況問題：KB 匯入僅存 contentText 原文，未做語義增強（摘要、實體抽取、候選問題）。

業界最佳實踐（RAGFlow TreeRAG）：

  • 匯入時同步生成：文件摘要、關鍵實體、候選問題
  • 階層式文件摘要分離細粒度搜尋與粗粒度閱讀

專案適用方案：

  1. /api/import 後觸發輕量後處理：Groq relay 生成 3 句摘要 + 5 個關鍵詞
  2. 摘要存入 metadata，提升 semantic search 精確度
  3. 與現有 kb-content-scoring.yaml 四維度評分整合

七、結語

daily-digest-prompt 專案的 RAG 整合已達高成熟度（357 個 API 引用、59 個 Skill），但快取效率（4/10）是當前最大瓶頸。本次提出的 6 項優化建議中，P0-A（分層知識庫）和 P1-B（快取自動修復）可在 3 小時內完成，預期將系統健康度從 7.5 提升至 8.5/10。P1-A（Agentic Plan Caching）是最具創新價值的方案，來自 2025 年最新學術研究，可望大幅降低自動任務的重複計算成本。
