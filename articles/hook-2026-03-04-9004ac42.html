專案優化報告：Hook 工具模組品質提升

優化概述

針對 daily-digest-prompt 專案的 Hooks 機器強制層進行深度分析與優化，重點改善程式碼品質、安全性與測試覆蓋率。

分析方法

四維並行深度分析：

  1. Hooks 程式碼品質審查（代碼重複、安全風險、複雜度）
  2. 配置檔案一致性檢查（數值矛盾、命名規範）
  3. PowerShell 腳本品質審查
  4. 測試覆蓋缺口分析

優化項目

1. safe_load_json() 共用工具函數（hook_utils.py）

  • 問題：5+ 個模組重複實作 JSON 安全載入（try/except FileNotFoundError/JSONDecodeError）
  • 方案：抽取 safe_load_json(filepath, default) 至 hook_utils.py，統一錯誤處理
  • 影響：behavior_tracker.py、agent_guardian.py、post_tool_logger.py 三處引用
  • 效益：減少約 30 行重複程式碼，統一容錯邏輯

2. _find_token_usage_file() 絕對路徑修正（post_tool_logger.py）

  • 問題：函數包含相對路徑 state/token-usage.json 作為 fallback，團隊並行模式下 5 路 Phase 1 Agent 的 CWD 可能不同，導致寫入不同檔案的競態條件
  • 方案：移除相對路徑 fallback，始終使用基於 __file__ 的絕對路徑
  • 影響：每日 44 次 Agent 執行 × 20-100 次工具呼叫的 Token 統計準確性
  • 安全性：消除團隊模式下的競態條件

3. _YAML_AVAILABLE 模組層級旗標（hook_utils.py）

  • 問題：_load_yaml_config() 在 hot path 中使用 try: import yaml except ImportError，每次呼叫都觸發例外處理
  • 方案：模組載入時一次性偵測 PyYAML 可用性，設定 _YAML_AVAILABLE 布林旗標
  • 影響：每次 Hook 呼叫都會經過 YAML 載入路徑
  • 效益：消除 hot path 中的 try/except 開銷，改用 if not _YAML_AVAILABLE 分支判斷

4. behavior_tracker.py 測試覆蓋（新增 test_behavior_tracker.py）

  • 問題：behavior_tracker.py 是每次 PostToolUse 都會呼叫的模組，但僅有 17 個測試（只覆蓋 sanitize + signature），track()、load/save、cleanup、report 等核心函數完全無測試
  • 方案：擴展至 45 個測試，覆蓋全部 7 個功能類別
  • 新增測試類別：

- TestLoadSavePatterns: 4 個測試（JSON 損壞/檔案缺失/roundtrip/目錄建立）

- TestCleanupStale: 3 個測試（過期清理/近期保留/時間戳更新）

- TestTrack: 10 個測試（新模式/信心遞增/上限/錯誤追蹤/IO 統計/溢位淘汰/靜默失敗/消毒/去重/截斷）

- TestReport: 3 個測試（空模式/有資料/低信心）

- TestConstants: 4 個測試（常量合理性驗證）

5. safe_load_json 測試（test_hook_utils.py 新增 8 個）

  • 覆蓋：有效 JSON/缺失檔案/損壞 JSON/空檔案/Unicode/陣列/巢狀結構/預設值

驗證結果

  • 語法檢查：4 個修改檔案全部通過
  • 測試套件：619 個測試全部通過（從 556 增加 63 個）

- behavior_tracker: 17 → 45（+28）

- hook_utils: 33 → 41（+8）

- 其餘 27 個模組的 546 個既有測試全部維持通過

  • 品質自評：無 P0/P1 問題

技術決策

  • safe_load_json 保留 ImportError fallback：確保獨立模組（如 agent_guardian.py 可脫離 hook_utils 運作）
  • _YAML_AVAILABLE 使用模組層級變數：一次性偵測，避免 hot path 開銷
  • _find_token_usage_file 移除 fallback：強制使用絕對路徑，犧牲微小的靈活性換取並行安全

影響範圍

  • 修改：4 個 Python 檔案（hook_utils.py、post_tool_logger.py、agent_guardian.py、behavior_tracker.py）
  • 新增/修改：2 個測試檔案（test_behavior_tracker.py、test_hook_utils.py）
  • 向後相容：所有修改保持向後相容（ImportError fallback）
