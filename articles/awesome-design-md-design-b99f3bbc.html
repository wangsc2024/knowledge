awesome-design-md — DESIGN.md 驅動 Coding Agent 建構一致性 UI

概述

awesome-design-md 是由 VoltAgent 組織維護的開源專案（29.2K Stars / 3.6K Forks，MIT 授權），提供 58 個從知名網站萃取的 DESIGN.md 設計系統文件。開發者只需將 DESIGN.md 放入專案根目錄，AI Coding Agent 即可讀取並產出與目標品牌視覺一致的 UI。

此概念源自 Google Stitch 所提出的 DESIGN.md 格式 — 一種純 Markdown 設計系統文件，無需 Figma 匯出、JSON Schema 或特殊工具鏈，LLM 可原生理解。

核心問題與解決方案

問題

傳統設計交接依賴 Figma 匯出、JSON 設計 Token、分散文件等格式，這些格式與 LLM 的資訊處理方式不對齊，導致 AI Coding Agent 產出的 UI 缺乏視覺一致性。

解決方案

以 Markdown 作為設計系統的載體（因為「Markdown 是 LLM 最擅長閱讀的格式」），將完整視覺語言編碼為九大標準區段，讓任何 AI Agent 都能讀取並實作。

---

DESIGN.md 九大標準區段

| # | 區段 | 內容 |

|---|------|------|

| 1 | Visual Theme & Atmosphere | 整體視覺氛圍與風格定義 |

| 2 | Color Palette & Roles | 語義命名、Hex 色碼、功能用途 |

| 3 | Typography Rules | 字型家族、層級表 |

| 4 | Component Stylings | 按鈕、卡片、輸入框的各狀態樣式 |

| 5 | Layout Principles | 間距尺度、網格、留白哲學 |

| 6 | Depth & Elevation | 陰影系統、層面階層 |

| 7 | Design Do's and Don'ts | 設計守則與禁忌 |

| 8 | Responsive Behavior | 斷點、觸控目標、收合策略 |

| 9 | Agent Prompt Guide | 專為 AI Agent 撰寫的提示指引 |

每個設計系統包含三個檔案：DESIGN.md（規格）、preview.html（亮色預覽）、preview-dark.html（暗色預覽）。

使用流程

  1. 複製 — 從 repo 中選取目標網站的 DESIGN.md，放入專案根目錄
  2. 指示 — 告訴 AI Agent「使用 DESIGN.md 建構 UI」
  3. 產出 — Agent 讀取設計規格，產出視覺一致的介面

---

技術特色與設計哲學

  • 純 Markdown：無需解析器或額外配置，LLM 原生可讀
  • 設計 Token 來源：從生產網站公開可見的 CSS 值萃取
  • Google Stitch 相容：遵循 Google Stitch 提出的 DESIGN.md 標準格式
  • 無所有權聲明：明確聲明不擁有任何網站的視覺識別，僅供 AI Agent 參考
  • 社群驅動：接受貢獻（改善現有檔案、回報問題、請求新網站的 DESIGN.md）

與本系統的關聯性分析

文件驅動架構的共鳴

本專案（Daily Digest Prompt）採用「Prompt 薄層 + YAML 外部配置 + Markdown 模板」的文件驅動架構（ADR-002），而 awesome-design-md 證明了同樣的原則適用於 UI 設計：用 Markdown 文件作為 Agent 與設計系統之間的契約。

潛在借鏡

  1. DESIGN.md 的九大區段結構可參考用於標準化本系統的 SKILL.md 格式
  2. preview.html 視覺預覽的概念可應用於 Skill 執行結果的可視化
  3. Agent Prompt Guide 區段是一種「為 Agent 量身打造的文件」思維，與 Skill-First 策略一致
