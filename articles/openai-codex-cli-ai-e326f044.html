OpenAI Codex CLI — 開源終端 AI 程式助手深度研究

概述

OpenAI Codex CLI 是 OpenAI 推出的開源終端 AI 程式助手，以 Apache 2.0 授權釋出。截至 2026 年 4 月，已累積 67K GitHub Stars、9K Forks、400+ 貢獻者，是目前最熱門的 AI 開發者工具之一。最新版本 v0.116.0（2026-03-19）已發布超過 640 個標記版本，展現極高的迭代速度。

核心定位：本地執行的終端原生 AI 程式代理（coding agent），無需瀏覽器或 IDE 外掛，直接在 shell 中運行。

---

技術架構

程式碼組成

| 組件 | 語言 | 用途 |

|------|------|------|

| codex-rs | Rust（95.6%） | 核心執行引擎，快速且自包含 |

| codex-cli | TypeScript | 命令列介面實作 |

| sdk | Python | 程式化嵌入（CI/CD 整合） |

| 建構系統 | Bazel | monorepo 建構管線 |

| 套件管理 | pnpm | workspace 配置 |

部署形式

  1. CLI 介面：codex 命令，終端全螢幕 UI
  2. IDE 整合：VS Code、Cursor、Windsurf 擴展
  3. 桌面應用：codex app 或 ChatGPT app 整合
  4. Web 版：chatgpt.com/codex
  5. 自動化：codex exec 子命令，非互動式執行（stdout 輸出，可串接 CI/CD）

安裝方式


# npm
npm install -g @openai/codex
# Homebrew（macOS）
brew install --cask codex
# 直接下載二進位（macOS ARM/Intel, Linux x86_64/ARM64）

核心功能

1. 三層核准模式（Approval Modes）

| 模式 | 行為 | 適用場景 |

|------|------|----------|

| Auto（預設） | 可讀取、編輯、執行工作目錄內命令 | 日常開發 |

| Read-only | 僅瀏覽檔案，修改需逐一核准 | 程式碼審查 |

| Full Access | 不受限的機器與網路存取 | 信任環境 |

可透過 /permissions 在會話中切換，所有操作皆保留 transcript 供回滾。

2. 沙箱與安全機制

  • Shell 升級控制：內建 shell escalation controls
  • 網路代理策略：可配置 allowlist 控制出站請求
  • 自訂 CA 憑證：企業防火牆環境可透過 SSL_CERT_FILE 環境變數配置根 CA bundle
  • 所有操作可追溯：保留完整 transcript

3. 會話恢復（Conversation Resumption）

  • 會話本地儲存
  • resume 子命令可重開先前工作
  • 保留 transcript 歷史與先前核准記錄

4. 多模態輸入

  • 支援 PNG、JPEG 圖片輸入
  • 命令列引數或剪貼簿貼上
  • 多圖片以逗號分隔

5. MCP 整合

  • 支援 STDIO 或 HTTP MCP Server
  • 在設定檔中配置，Codex 自動啟動並暴露工具
  • 與內建工具並列使用

6. Web 搜尋

  • 預設使用快取（索引）結果（安全考量）
  • --search 或 web_search = "live" 啟用即時搜尋

7. 認證方式

  • ChatGPT 方案整合：Plus/Pro/Team/Edu/Enterprise 訂閱直接使用
  • API Key 認證：獨立 API key
  • 裝置碼登入（v0.116.0 新增）：無需手動輸入 token

推薦模型

官方推薦使用 gpt-5.4，結合前沿程式效能、強化推理、原生電腦操控與更廣泛的專業工作流。可透過 /model 命令或 --model gpt-5.4 參數在會話中切換。

與 Claude Code 的比較分析

| 維度 | Codex CLI | Claude Code |

|------|-----------|-------------|

| 開源授權 | Apache 2.0 | 專有 |

| 核心語言 | Rust 95.6% | TypeScript |

| Stars | 67K | 102K |

| 核准模式 | 3 層（Auto/Read-only/Full） | 逐指令權限 |

| 沙箱 | 內建 shell + network 控制 | Hook 系統 + 手動配置 |

| MCP 支援 | 原生（STDIO/HTTP） | 原生 |

| 多模態 | 圖片輸入 | 圖片輸入 |

| 會話恢復 | 本地儲存 + resume | 內建 |

| 認證 | ChatGPT 訂閱 / API Key | Anthropic API Key |

| 部署形式 | CLI + IDE + Desktop + Web | CLI |

| 企業功能 | CA cert / sandbox / Python SDK | Team 方案 |

| 自動化 | codex exec（非互動） | claude -p（pipe 模式） |

關鍵差異

  1. 開源 vs 專有：Codex CLI 完整開源（Apache 2.0），Claude Code 雖有 npm 套件但核心為專有
  2. Rust 核心：Codex CLI 以 Rust 為主（95.6%），啟動速度與記憶體效率優勢顯著
  3. 多平台部署：Codex CLI 同時支援 CLI/IDE/Desktop/Web 四種介面
  4. Hook 系統：兩者都支援 prompt hook，但 Codex 的 User Prompt Hook 更偏向企業 policy enforcement

---

對 daily-digest-prompt 專案的啟發

  1. 核准模式設計：三層核准模式可借鏡到本專案的 Agent 權限控制（目前用 Hook 攔截）
  2. 會話恢復：Codex 的 resume 機制與本專案的 context/continuity/ 連續記憶有相似設計思路
  3. User Prompt Hook：企業級 context injection 可對標本專案的 preamble.md 共用前言機制
  4. exec 子命令：非互動式執行模式對應 claude -p pipe 模式，可做自動化基準比較
  5. MCP 整合：Codex 原生 MCP 支援提供了 Agent↔工具 標準化的參考實作
