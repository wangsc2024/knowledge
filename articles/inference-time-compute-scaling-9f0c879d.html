Inference-Time Compute Scaling 完整指南

從 Test-Time Compute、推理 Token 最佳化到 2026 開源推理模型生態的技術全景與效能基準

---

一、概述

Inference-Time Compute Scaling（推論時計算擴展，又稱 Test-Time Compute、Inference Scaling）是 2026 年 LLM 領域最重要的技術範式轉移之一。核心概念是：在推論階段投入更多計算資源，而非持續增大模型參數，以提升回答品質與準確度。

為什麼這很重要？

  • 訓練前擴展放緩：隨著模型參數規模趨近實用極限，單純增大模型帶來的邊際效益遞減
  • 推論成本已超越訓練成本：成為實際部署的主要瓶頸
  • 更靈活的計算分配：可依任務難度動態調整推論計算預算
  • Pareto 最優：小模型 + 進階推論演算法，在成本與效能上常優於大模型直接推論

二、推論時擴展的四大類別

2.1 推理型方法（Reasoning-Based）

| 技術 | 原理 | 適用場景 |

|------|------|----------|

| Chain-of-Thought（CoT） | 引導模型逐步推理，生成中間步驟 | 數學、邏輯、程式設計 |

| Tree-of-Thought（ToT） | 探索多條推理路徑，選擇最佳分支 | 複雜規劃、創意任務 |

| ReAct | 交替進行推理與行動（工具呼叫） | Agentic 工作流 |

| Interleaved Reasoning | 推理與工具呼叫交錯執行 | 多步驟 Agent 任務 |

關鍵特徵：CoT 額外開銷約 3-5 倍 token 使用量（例如數學題從 300 token 增至 1,200 token）。

2.2 取樣型方法（Sampling-Based）

| 技術 | 原理 | 適用場景 |

|------|------|----------|

| Best-of-N | 並行生成 N 個獨立回答，選最佳者 | 品質敏感任務 |

| Self-Consistency | 多次取樣後多數決 | 有明確答案的推理任務 |

| Majority Voting | 統計最常出現的答案 | 數學、邏輯推理 |

關鍵特徵：計算量線性增長（N 倍），但可完全並行化。

2.3 搜尋型方法（Search-Based）

| 技術 | 原理 | 適用場景 |

|------|------|----------|

| Beam Search | 保留 K 個最佳候選路徑 | 程式碼生成、翻譯 |

| Monte Carlo Tree Search（MCTS） | 模擬 + 回溯選擇最優路徑 | 複雜推理、博弈 |

| Test-Time Search | 動態探索多條推理路徑並由評估器篩選 | o3 系列模型的核心機制 |

關鍵特徵：需要評估器（Critic / Verifier）來區分候選路徑的品質。

2.4 驗證型方法（Verification-Based）

| 技術 | 原理 | 適用場景 |

|------|------|----------|

| Process Reward Model（PRM） | 逐步驟評分，獎勵正確推理過程 | 數學證明、多步驟推理 |

| Outcome Reward Model（ORM） | 僅評估最終結果 | 有明確答案的任務 |

| Verification CoT | 驗證器生成驗證推理鏈 | 高可靠性需求場景 |

關鍵特徵：PRM 比 ORM 更精確但需要步驟級標註資料。

三、推理模型的內部架構

3.1 OpenAI o3 系列

  • 架構：Dense Transformer + Test-Time Search
  • 訓練：大規模強化學習（RL），在驗證過的推理序列上訓練
  • 推論機制：生成數百/數千條候選推理路徑 → 評估器檢查計算和邏輯錯誤 → 選擇最佳路徑
  • 隱藏 CoT：推理過程不對使用者公開（安全考量 + 刻意對齊）
  • Thinking Budget：reasoning_effort 參數（low/medium/high），從 low 到 high 準確度提升 10-30%
  • 效能：AIME 2024 96.7%、MATH-500 ~97%、Codeforces Elo 2727
  • 訓練成本：約 120 萬 A100 GPU 小時

3.2 DeepSeek R1

  • 架構：Mixture-of-Experts（MoE），671B 總參數 / 37B 活躍參數
  • 四階段訓練流程：

1. SFT：~1,000 筆高品質推理樣本微調

2. GRPO RL：Group Relative Policy Optimization，規則式獎勵（準確性、格式、語言一致性）

3. Rejection Sampling：篩選 60 萬輸出 + 20 萬通用領域樣本的 SFT

4. 多樣 RL：結合規則式和 LLM 式獎勵模型，涵蓋 100+ 任務類別

  • 透明 CoT：推理過程完全可見
  • 效能：MATH-500 97.3%、AIME 2024 79.8%
  • 訓練成本：約 266 萬 H800 GPU 小時

3.3 Anthropic Claude Thinking 模式

  • Extended Thinking：在生成回覆前進行延伸思考
  • Budget Token：可設定思考 token 上限
  • Streaming 支援：思考過程可串流顯示
  • 應用：Claude Opus 4.6 支援 1M token 上下文 + 延伸思考

  2. 蒸餾民主化：8B 蒸餾模型在 AIME 達到 87.5%，可在消費級硬體運行
  3. Agent 原生設計：MiniMax-M2.1 等模型專為多步驟 Agent 工作流最佳化
  4. 雙模式切換：Qwen3 系列支援速度模式與深度推理模式動態切換
  5. 多 Token 預測：Qwen3-Next 等模型在訓練時採用多 Token 預測，加速推論

五、成本與效能的權衡

5.1 推論成本結構

| 策略 | 額外計算倍數 | 準確度提升 | 適用場景 |

|------|-------------|-----------|----------|

| 基礎推論 | 1x | 基準 | 簡單查詢 |

| CoT 提示 | 3-5x | +10-20% | 中等推理 |

| Best-of-N (N=8) | 8x | +15-25% | 品質關鍵任務 |

| Test-Time Search | 10-100x | +20-40% | 複雜推理、競賽 |

5.2 計算預算分配策略

  • 動態分配：依任務難度自動調整思考深度（o3 的 reasoning_effort）
  • 提前終止：簡單問題快速回答，不浪費推理 token
  • 混合策略：Easy 問題用小模型直答，Hard 問題用大模型 + 深度推理

5.3 Pareto 前沿發現

關鍵發現：小模型 + 進階推論演算法的組合，在成本-效能 Pareto 前沿上常優於大模型直接推論。例如 Ministral 14B 在消費級 GPU 上達到 5-10 倍大模型的推理效能。

---

六、2026 前沿方向

  1. 推論效率最佳化：降低延遲、減少不必要的推理 token，讓推理更精準而非更長
  2. 多模態推理擴展：文字推理延伸至圖像、音頻（Gemini 多模態推理）
  3. Agentic 推理：推理 + 行動交錯的 Agent 架構成為主流（ReAct → Interleaved Reasoning）
  4. 驗證器訓練民主化：PRM 標註成本降低，開源 PRM 資料集增加
  5. 推論即服務（Reasoning-as-a-Service）：雲端 API 暴露計算預算參數，用戶按需付費
  6. 邊緣推理：INT4 量化 + 超稀疏 MoE 讓複雜推理在消費級硬體可行

七、實踐建議

何時使用推論時擴展？

| 場景 | 推薦策略 | 原因 |

|------|---------|------|

| 數學/邏輯推理 | CoT + Self-Consistency | 步驟明確，多數決有效 |

| 程式碼生成 | Best-of-N + Test Cases | 可自動驗證正確性 |

| 開放式問答 | Extended Thinking | 需要深度思考但無明確驗證 |

| Agent 任務 | Interleaved Reasoning | 推理與工具呼叫交錯 |

| 成本敏感 | 小模型 + PRM | Pareto 最優解 |

| 延遲敏感 | 基礎推論 + 快取 | 避免額外推理開銷 |

選擇模型的決策框架

  1. 預算充足 + 最高準確度 → o3 high reasoning / Claude Extended Thinking
  2. 開源 + 高效能 → Qwen3-235B / DeepSeek-V3.2
  3. 邊緣部署 + 消費級硬體 → Ministral 14B / R1-Distill-8B
  4. Agent 工作流 → MiniMax-M2.1 / GLM-4.7
  5. 極致成本效率 → MiMo-V2-Flash（Claude 2.5% 成本）
