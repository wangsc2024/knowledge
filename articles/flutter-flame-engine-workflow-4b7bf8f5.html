一、Flame Engine 核心架構

1.1 概述

Flame 是基於 Flutter 的模組化 2D 遊戲引擎，輕量、文件完善、活躍維護。最大優勢是一套程式碼部署至 iOS、Android、Web、桌面，且支援 Flutter 的 Hot Reload 即時預覽。

1.2 核心元件

| 元件 | 說明 |

|------|------|

| FlameGame | 遊戲主類別，管理 Game Loop（update/render）和元件樹 |

| Component System (FCS) | 類似 Flutter Widget 的模組化設計，所有遊戲物件皆為 Component |

| World + CameraComponent | World 容納遊戲物件，Camera 控制視角與縮放 |

| SpriteComponent | 渲染 2D 圖片的基礎元件 |

| Input Mixins | TapCallbacks、DragCallbacks、KeyboardHandler 等模組化輸入處理 |

| Collision Detection | 內建 Hitbox 碰撞系統（CircleHitbox、RectangleHitbox、PolygonHitbox） |

1.3 基本結構範例


class MyGame extends FlameGame {
  @override
  Future<void> onLoad() async {
    // 載入素材、初始化元件
    final player = SpriteComponent()
      ..sprite = await loadSprite('player.png')
      ..size = Vector2(64, 64)
      ..position = Vector2(100, 100);
    world.add(player);
  }

  @override
  void update(double dt) {
    super.update(dt);
    // 遊戲邏輯，dt 為幀間隔（秒）
  }
}

| 套件 | 用途 | 安裝指令 |

|------|------|----------|

| flame | 核心引擎 | flutter pub add flame |

| flame_network_assets | 網路素材載入 | flutter pub add flame_network_assets |

二、OpenMoji — 免費開源 SVG 表情圖示

2.1 專案資訊

  • 網站：https://openmoji.org/
  • GitHub：https://github.com/hfg-gmuend/openmoji
  • 授權：CC BY-SA 4.0（可商用，需署名 + 相同方式分享）
  • 數量：4,000+ 個表情符號，統一風格設計
  • 來源：德國 HfG Schwäbisch Gmünd 設計大學 50+ 名學生與教授

2.2 可用格式

| 格式 | 規格 | 適用場景 |

|------|------|----------|

| SVG Color | 向量，無限縮放 | 遊戲 UI、道具圖示（搭配 flame_svg） |

| SVG Black | 向量，單色 | 線框風格遊戲 |

| PNG Color 72×72 | 點陣 | 小型遊戲精靈 |

| PNG Color 618×618 | 點陣 | 高解析度顯示 |

| OpenMoji Fonts | 字型（Beta） | 實驗性用途 |

2.3 在 Flame 中整合 OpenMoji

方法 A：使用 flame_svg 載入 SVG


// pubspec.yaml 中註冊 assets/openmoji/
final emoji = await Svg.load('openmoji/1F525.svg'); // 火焰
final component = SvgComponent(svg: emoji, size: Vector2(48, 48));
world.add(component);

方法 B：使用 PNG 精靈圖


// 將 OpenMoji PNG 放入 assets/images/emoji/
final sprite = await loadSprite('emoji/1F525.png');
final component = SpriteComponent(sprite: sprite, size: Vector2(48, 48));

2.4 署名要求

All emojis designed by OpenMoji — the open-source emoji and icon project. License: CC BY-SA 4.0

建議放在遊戲的「關於」或「致謝」頁面。

三、ZapSplat — 免費音效與音樂資源庫

3.1 專案資訊

  • 網站：https://www.zapsplat.com/
  • 規模：160,000+ 音效、650+ SFX 包、800+ 免版稅音樂曲目
  • 用戶：260 萬+ 創作者（影片製作者、遊戲開發者、Podcaster）
  • 授權：免費會員需署名；Premium 會員免署名

3.2 遊戲相關分類

| 分類 | 說明 |

|------|------|

| Game Sounds | 勝利、獎勵、升級、金幣收集、失敗 |

| Game Music & Loops | 可循環播放的背景音樂，各種風格 |

| UI Sounds | 按鈕點擊、選單切換、確認音 |

| Ambient | 環境音效（森林、城市、水流） |

3.3 格式與品質

  • MP3：壓縮格式，檔案小，適合行動裝置
  • WAV：無損格式，音質最佳
  • 免費下載，即拖即用

3.4 在 Flame 中整合音效


// 使用 flame_audio 套件
import 'package:flame_audio/flame_audio.dart';

// 預載入音效（放在 assets/audio/）
await FlameAudio.audioCache.loadAll([
  'sfx/coin_collect.wav',
  'sfx/jump.wav',
  'bgm/level1_loop.mp3',
]);

// 播放音效
FlameAudio.play('sfx/coin_collect.wav');

// 播放循環背景音樂
FlameAudio.bgm.play('bgm/level1_loop.mp3');

// 停止背景音樂
FlameAudio.bgm.stop();

五、遊戲開發 Workflow 完整流程

Phase 0：概念與規劃


遊戲概念定義
├── 遊戲類型選擇（平台、射擊、RPG、解謎）
├── 核心機制設計（玩家操作、勝負條件）
├── 目標平台確認（iOS/Android/Web/Desktop）
└── 素材需求清單
    ├── 視覺：角色、場景、UI、特效
    ├── 音效：動作音、環境音、UI 音
    └── 音樂：背景音樂、勝利/失敗曲

Phase 1：專案初始化


# 建立 Flutter 專案
flutter create --template=app my_game
cd my_game

# 安裝核心套件
flutter pub add flame
flutter pub add flame_audio
flutter pub add flame_svg   # 如需使用 OpenMoji SVG

# 建立素材目錄結構
mkdir -p assets/images/sprites
mkdir -p assets/images/ui
mkdir -p assets/images/emoji    # OpenMoji
mkdir -p assets/audio/sfx        # ZapSplat 音效
mkdir -p assets/audio/bgm        # ZapSplat 背景音樂
mkdir -p assets/tiles            # Tiled 地圖

pubspec.yaml 配置：


flutter:
  assets:
    - assets/images/sprites/
    - assets/images/ui/
    - assets/images/emoji/
    - assets/audio/sfx/
    - assets/audio/bgm/
    - assets/tiles/

Phase 2：素材準備


素材管線（Asset Pipeline）
├── 1. 蒐集素材
│   ├── OpenMoji → 下載 SVG/PNG，選取遊戲需要的表情
│   ├── ZapSplat → 下載 WAV/MP3 音效與循環音樂
│   ├── Kenney.nl → 下載角色精靈圖與地圖磚塊
│   └── 自製素材 → Aseprite（像素畫）/ Inkscape（向量）
│
├── 2. 素材處理
│   ├── 精靈圖集（Sprite Sheet）打包 → TexturePacker
│   ├── SVG 尺寸標準化（統一畫布大小）
│   ├── 音效格式轉換（建議 WAV 用於 SFX，MP3 用於 BGM）
│   └── 循環音樂確認無縫接軌（loop point）
│
└── 3. 匯入專案
    ├── 放入對應 assets/ 子目錄
    ├── 更新 pubspec.yaml
    └── 執行 flutter pub get

Phase 3：核心開發（迭代循環）


開發迭代循環
├── Step 1：建立 FlameGame 主類別
│   ├── onLoad() 預載入所有素材
│   ├── 設定 World + CameraComponent
│   └── debugMode = true（開發階段）
│
├── Step 2：實作遊戲元件
│   ├── Player（SpriteAnimationComponent + 輸入 Mixin）
│   ├── Enemy（行為邏輯 + AI 狀態機）
│   ├── Collectible（碰撞觸發 + 音效播放）
│   └── UI Overlay（Flutter Widget 疊加 — 計分、生命值）
│
├── Step 3：物理與碰撞
│   ├── 加入 Hitbox（Circle/Rectangle/Polygon）
│   ├── 實作 CollisionCallbacks
│   └── 可選：flame_forge2d 完整物理模擬
│
├── Step 4：音效整合
│   ├── FlameAudio.play() 用於一次性音效
│   ├── FlameAudio.bgm.play() 用於背景音樂
│   └── 音量控制與靜音選項
│
└── Step 5：Hot Reload 測試
    ├── 修改 → 儲存 → 即時預覽
    ├── 用 debugMode 檢視 Hitbox 範圍
    └── 效能監控（維持 60 FPS）

Phase 4：效能優化

| 技巧 | 說明 |

|------|------|

| Sprite Batching | 合併多精靈至單一紋理，減少 Draw Call |

| Texture Atlas | 使用 TexturePacker 打包精靈圖集 |

| Object Pooling | 重用記憶體密集物件（子彈、粒子） |

| Viewport Culling | 跳過螢幕外物件的渲染 |

| Delta Time | 所有移動乘以 dt，確保不同裝置速度一致 |

| Asset Preloading | 在 onLoad() 一次載入所有素材 |

Phase 5：測試與發布


品質保證與發布
├── 多平台測試
│   ├── Android（模擬器 + 實機）
│   ├── iOS（模擬器 + 實機）
│   ├── Web（Chrome + Firefox）
│   └── Desktop（Windows/macOS/Linux）
│
├── 整合 Flutter Casual Games Toolkit
│   ├── 廣告（AdMob）
│   ├── 應用內購買
│   ├── 排行榜與成就（Google Play Games / Game Center）
│   └── 分析（Firebase Analytics）
│
└── 發布
    ├── flutter build appbundle  （Android）
    ├── flutter build ios        （iOS）
    ├── flutter build web        （Web）
    └── flutter build windows    （Desktop）
