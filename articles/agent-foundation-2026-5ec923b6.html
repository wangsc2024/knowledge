Agent 執行品質閘門與輸出驗證管線 — 深度研究洞察簡報

系列: agent-quality-gates | 階段: foundation

摘要

本報告建立 Agent 執行品質閘門與輸出驗證管線的核心概念框架（foundation 階段）。關鍵發現：品質閘門已從單純阻擋機制演進為智能化、風險導向的決策系統；Agent 輸出驗證需採用三層架構（middleware → context → client）；多層驗證框架（CoV + ASVCA）可確保事實準確性、用戶安全性與可驗證溯源性。

5 個核心洞察

1. 品質閘門核心模式（2026）

品質閘門具備 6 個關鍵特性：policy-driven、observable、automatable、composable、latency-bound、governance-aware。部署位置演進：CI 早期（靜態檢查）→中期（整合測試）→晚期（金絲雀閘門）→運行時（SLO-based 動態閘門）。風險導向轉型：從「阻擋一切」轉向「聚焦真正影響用戶、安全或業務功能的問題」。

2. Agent 輸出驗證管線三層架構

來源: Microsoft Agent Framework


[Agent Middleware] → [Context Layer] → [Chat Client Layer]
  • Agent Middleware：攔截所有 run() 呼叫，可檢查或修改輸入/輸出
  • Context Layer：管理對話歷史、注入額外上下文
  • Chat Client Layer：處理 LLM 通訊

輸出驗證二階段：Pydantic validation → Custom output validators

3. 多層驗證框架（CoV + ASVCA）

Chain-of-Verification (CoV)：分層驗證管線，獨立驗證器評估不同面向（事實/邏輯/安全/來源對齊）。

ASVCA 三維度評分：事實準確性、用戶安全性、可驗證溯源性。分數控制釋出許可、仲裁優先級、審計標記。

4. JSON Schema 驗證最佳實踐

核心原則：Be specific、Use the Right Version（$schema）、Leverage Built-in Formats、Keep it DRY（$ref）。

運行時驗證策略：TypeScript 類型（開發時）+ JSON Schema（運行時）雙重驗證，API middleware 整合，入口驗證。

效能陷阱：additionalProperties: false 成本高，複雜 oneOf 效能差，必須 benchmark。

5. 格式漂移偵測與自動修復

偵測策略：持續驗證、漂移度量、趨勢分析（7/30 天驗證失敗率）。

自動修復機制（分級）：

  • Tier 1：缺失預設值自動補上
  • Tier 2：命名不一致自動正規化
  • Tier 3：結構性錯誤標記 format_failed
