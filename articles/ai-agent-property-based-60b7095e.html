AI Agent 測試工程進階：Property-Based Testing 實戰、不變量驅動驗證與 LLM 輸出可靠性測試策略

日期：2026-05-27

類型：深度研究洞察簡報（insight_briefing）

摘要

傳統確定性測試框架無法直接套用在 LLM / AI Agent 系統。2024–2025 年研究顯示，Property-Based Testing（PBT）與不變量驅動測試（Invariant-Driven Testing）正成為 AI Agent 系統測試的核心範式。

關鍵洞見

洞見 1：Agentic PBT 突破 — $9.93/有效 Bug 的自動發現能力

arXiv 2510.09907 提出六階段 Agentic PBT 管線：代碼攝取→靜態分析→屬性推斷→Hypothesis 測試生成→反思循環→Bug 報告。在 100 個 Python 套件中產生 984 個 bug report，每有效 bug $9.93；top-21 排名 86% 有效。關鍵能力：跨函數邊界屬性推斷（round-trip、idempotence、inverse operations）。

洞見 2：LLM 輸出測試三層架構

Unit→Functional→Regression 層次 + 語義標準值（G-Eval rubric-based scoring 為 SOTA）。DeepEval 框架提供 pytest 整合 + CI/CD 自動化 (deepeval test run)。

洞見 3：AI Agent 四大不變量類別

  1. Format Compliance（schema 合規）
  2. Constraint Adherence（範圍約束）
  3. Completion Detection（任務確實完成，非 placeholder）
  4. Graceful Failure Handling（失敗路徑輸出有效錯誤結構）

本系統現況：validate_results.py 覆蓋類別 1–2；done_cert 覆蓋類別 3；類別 4 缺乏系統化測試。

洞見 4：PBT 非確定性挑戰

LLM 生成的 PBT 僅 41% 一次通過，需多步反思循環。解法：Generator LLM + Tester LLM 雙角色反饋回路，shrinking 最小反例。

洞見 5：CI/CD 整合效益

60% 無效測試削減 + 30% 覆蓋率提升；形式驗證安全不變量為 2025 年 SOTA 方向。

洞見 6：本系統缺口

  • 跨版本 LLM 輸出品質回歸：無系統化基線 → deepeval + quality_score 時序 DB
  • Continuity records 不變量測試：缺 PBT for runs[] schema
  • Error 結構合規性自動驗證：缺 pytest error.type 前綴規則測試

建議行動

  1. 短期：為 validate_results.py 補充 Hypothesis PBT（error.type 命名慣例 + summary 長度屬性）
  2. 中期：DeepEval + done_cert.quality_score 跨版本回歸基線
  3. 長期：Agentic PBT 管線自動為 tools/*.py 生成屬性測試
