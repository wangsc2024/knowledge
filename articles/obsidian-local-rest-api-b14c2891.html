Obsidian Local REST API 設定指南

概述

Obsidian Local REST API 是由 coddingtonbear 開發的社群插件，透過本機 REST API 讓外部程式（如 AI Agent、自動化腳本）安全地讀寫 Obsidian Vault 筆記。本專案（daily-digest-prompt）已建立完整的 Skill 支援（obsidian-api、obsidian-memory），但缺少實際的插件安裝與 API Key 設定。

安裝步驟

1. 安裝 Local REST API 插件

  1. 開啟 Obsidian → Settings → Community Plugins
  2. 關閉 Safe Mode（若尚未關閉）
  3. 點擊 Browse，搜尋「Local REST API」
  4. 找到 coddingtonbear 的版本，點擊 Install → Enable

2. 取得 API Key

  • 安裝後進入 Settings → Local REST API
  • 插件首次載入時會自動生成 256-bit API Key（SHA-256 hash，64 字元十六進位字串）
  • 在設定頁面可直接複製 API Key

3. 網路配置

| 設定項 | 預設值 | 說明 |

|--------|--------|------|

| HTTPS Port | 27124 | 加密端口（預設啟用） |

| HTTP Port | 27123 | 未加密端口（預設停用） |

| Binding Host | 127.0.0.1 | 僅本機存取 |

| Enable HTTPS | true | 使用自簽憑證 |

| Enable HTTP | false | 需手動啟用 |

4. 憑證管理

  • 自簽 X.509 憑證，RSA 2048-bit，有效期 365 天
  • curl 需加 -k（--insecure）跳過憑證驗證
  • 可從 https://127.0.0.1:27124/obsidian-local-rest-api-certificate.crt 下載憑證並信任

專案整合設定（daily-digest-prompt）

.env 設定


OBSIDIAN_API_KEY=your-64-char-hex-key-here

依賴配置（config/dependencies.yaml）

專案已配置完成：

  • skills.obsidian_api.api.base_url: https://127.0.0.1:27124
  • skills.obsidian_api.api.base_url_http: http://127.0.0.1:27123
  • skills.obsidian_api.api.auth_env: OBSIDIAN_API_KEY
  • skills.obsidian_api.memory_folder: daily-digest

可用性檢查（每次使用前必做）


OBSIDIAN_BASE=$(uv run python -X utf8 -c \
  "import yaml; d=yaml.safe_load(open('config/dependencies.yaml')); \
   print(d['skills']['obsidian_api']['api']['base_url'])" 2>/dev/null \
  || echo "https://127.0.0.1:27124")
OBSIDIAN_KEY=$(grep '^OBSIDIAN_API_KEY=' .env 2>/dev/null | cut -d= -f2 | tr -d '\r')
OBSIDIAN_OK=$(curl -sk --max-time 3 "$OBSIDIAN_BASE/" \
  -H "Authorization: Bearer $OBSIDIAN_KEY" \
  | grep -c '"ok":true' 2>/dev/null || echo "0")

API 端點速查

| 端點 | 方法 | 用途 |

|------|------|------|

| / | GET | 健康檢查（無需 auth） |

| /vault/{path} | GET/PUT/POST/PATCH/DELETE | 筆記 CRUD |

| /vault/{dir}/ | GET | 列出目錄 |

| /search/simple/ | POST | 全文搜尋 |

| /search/ | POST | DQL/JsonLogic 搜尋 |

| /tags/ | GET | 所有標籤與使用次數 |

| /commands/ | GET | 可用命令列表 |

| /commands/{id}/ | POST | 執行命令 |

| /active/ | GET/PUT/PATCH/POST/DELETE | 當前開啟的筆記 |

認證方式

所有端點（除 GET /）需 Bearer Token：


Authorization: Bearer <your-api-key>

進階設定（隱藏選項）

需在 Settings → Local REST API 中開啟「Show advanced settings」：

  • 自訂 HTTPS/HTTP 端口
  • 修改 API Key
  • 配置憑證 SAN（Subject Alternative Names）
  • 修改 Authorization Header 名稱
  • 修改 Binding Host（0.0.0.0 開放網路存取，有安全風險）

相關整合方案

  • MCP 整合：使用 mcp-obsidian（MarkusPfundstein）或 Claudesidian MCP 將 Obsidian 接入 Claude
  • Dataview Plugin：安裝後可使用 DQL 進階查詢（本專案 obsidian-memory Skill 使用）
  • Obsidian MCP Server Enhanced：提供更全面的 MCP 存取功能

故障排除

| 問題 | 解決方式 |

|------|----------|

| 連線拒絕 | 確認 Obsidian 桌面 App 已開啟 |

| 401 Unauthorized | 檢查 .env 中 OBSIDIAN_API_KEY 是否正確 |

| 憑證錯誤 | curl 加 -k 或信任自簽憑證 |

| HTTP 端口無回應 | 需在插件設定中手動啟用 HTTP Server |

| DQL 查詢失敗 | 安裝 Dataview plugin 或改用 /search/simple/ |
