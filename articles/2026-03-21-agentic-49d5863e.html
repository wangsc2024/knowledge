二、六維同構性發現

洞見一：路由調度的三層同構 —— Agentic RAG × 化儀四教 × Agent Harness

發現：Agentic RAG 的五大架構模式與天台宗化儀四教的施教機制，在「根據接收者狀態動態選擇策略」這一核心邏輯上呈現驚人的結構同構。

| Agentic RAG 模式 | 化儀四教對應 | Agent Harness 對應 |

|------------------|-------------|--------------------|

| Single-Agent Router（單路由） | 頓教（直指心源，單一高階法） | 單一 Agent 模式（run-todoist-agent.ps1） |

| Adaptive RAG（自適應，分類器評估複雜度） | 不定教（聞頓悟漸、聞漸悟頓） | runtime policy 動態調整 |

| Corrective RAG（修正式，迭代驗證） | 漸教（阿含→方等→般若次第引導） | recovery worker 迭代修復 |

| Multi-Agent Systems（多代理並行） | 秘密教（同會異聞，各隨根機） | Team 模式並行 Phase 2 |

| Hierarchical Systems（階層式） | 五時判教整體架構 | 三階段 orchestrator |

深度洞見：三者共同揭示了一個普遍原理——有效的調度系統必須同時具備「固定骨架」與「彈性變數」。Agentic RAG 的骨架是控制迴圈、變數是策略選擇；化儀四教的骨架是頓漸、變數是秘密不定；Agent Harness 的骨架是三階段管線、變數是 runtime override。法華時的「非頓非漸」（開權顯實）則暗示所有路由終將歸一——這對應到 Agentic RAG 的「最終一致性」設計理念。

洞見二：認知校準的雙軸模型 —— 心流理論 × SDT 自我決定理論

發現：心流理論的「挑戰-技能平衡」與 SDT 的「自主性-勝任感-歸屬感」三需求，可整合為一個統一的認知校準框架。


            高挑戰
              ↑
    焦慮區    │  ★ 心流區
   （SDT：勝任感│  （SDT：三需求
     受挫）    │    皆滿足）
              │
  ────────────┼────────────→ 高技能
              │
    冷漠區    │   無聊區
   （SDT：自主性│  （SDT：勝任感
     缺失）    │    過剩）
              ↓
            低挑戰

雙軸校準公式：

  • X 軸（技能-勝任感）：個體當前能力水平 × SDT 勝任感滿足程度
  • Y 軸（挑戰-自主性）：任務難度 × SDT 自主性感受程度
  • Z 軸（歸屬感，深度維度）：連結感強化心流持續時間

待辦事項系統設計啟示：

  1. 任務拆分應瞄準心流通道（挑戰略高於技能），同時確保用戶對拆分方式有選擇權（自主性）
  2. 即時回饋（✅ 勾選）同時滿足心流條件與勝任感需求——回饋延遲不應超過 2.5 秒
  3. 團隊待辦清單比個人清單更有動力（歸屬感），但必須保留個人選擇空間（自主性）
  4. SDT 的動機連續體六階段可作為任務優先級排序的隱藏維度——整合調節的任務自然執行率最高

洞見三：驗證機制的內外對照 —— deep-research Anti-Hallucination × 無作四諦實相觀

發現：deep-research Skill 的防幻覺協議與天台圓教無作四諦，各從外在驗證與內在覺照兩個方向，解決同一個根本問題——如何區分「真實」與「幻覺」。

| 面向 | deep-research 防幻覺 | 無作四諦實相觀 |

|------|---------------------|---------------|

| 核心問題 | LLM 輸出是否有事實依據？ | 認知是否契合實相？ |

| 驗證方法 | 來源錨定 + 雙腳本自動驗證迴圈 | 一心三觀（空假中同時觀照） |

| 幻覺定義 | 無來源支持的事實性宣稱 | 執著四諦為實有（苦可捨、集可斷） |

| 修正機制 | 最多 3 次重試修正 | 觀陰入界即法性，當下轉化 |

| 終極狀態 | 所有引用皆可驗證 | 苦集滅道即實相，無二無別 |

深度洞見：防幻覺協議的「來源錨定」是外在的「析空觀」（藏教方法），一一核實每個宣稱；而無作四諦的「即實相」是圓教的觀照——不是逐一排除幻覺，而是直接照見一切法的實相本性。這暗示 AI 系統的可靠性策略也可能存在兩個層次：

  • L1 析式驗證：逐條核實（verify_citations.py），對應藏教生滅四諦
  • L2 整體一致性：結構性驗證（validate_report.py 的 9 項檢查），對應通教無生四諦
  • L3 語義連貫性：報告的敘事弧是否自然流暢，對應別教無量四諦
  • L4 目的性驗證：報告是否真正回答了研究問題（而非只是有來源的正確但無關內容），對應圓教無作四諦

洞見四：狀態接力的跨時段同構 —— 前瞻記憶 × Auto-Continuation × 化儀漸教

發現：認知科學的前瞻記憶（Prospective Memory）、deep-research 的 Auto-Continuation 遞迴 Agent、以及化儀四教的漸教三時次第，三者共同描述了「如何在時間跨度中保持意圖連貫性」。

| 機制 | 觸發條件 | 狀態持久化 | 接力方式 |

|------|---------|-----------|----------|

| 前瞻記憶 | 事件型（看到線索）/ 時間型（到點提醒） | 長期記憶中的意圖編碼 | 注意力自動捕捉 + 執行意圖 If-Then |

| Auto-Continuation | 輸出超 18,000 字 | continuation_state JSON | 子 Agent 讀取前 3 章節風格 |

| 漸教三時 | 根機成熟度 | 阿含→方等→般若累積善根 | 次第轉教（酪→生酥→熟酥→醍醐） |

深度洞見：三者都面臨「接力斷裂」的風險——前瞻記憶的遺忘（注意力殘留干擾）、Auto-Continuation 的風格不一致（接續 Agent 漂移）、漸教的「堕小」（聲聞安住阿羅漢果不願進大乘）。對應的解決方案也結構相似：

  • 前瞻記憶：執行意圖（Implementation Intentions）的 If-Then 預編碼
  • Auto-Continuation：強制讀取前文風格指標 + 品質一致性驗證
  • 漸教：方等時「恥小慕大」的刺激機制（維摩詰經彈偏斥小）

洞見五：品質閘門的管線化 —— 研究設計機制 × Agentic RAG × deep-research

發現：研究設計機制整合的七議題（PICO/SPIDER、PRISMA-P、AMSTAR2 等）、Agentic RAG 的 Corrective RAG 模式、deep-research 的 9 項驗證，三者都將品質驗證從「事後檢查」提升為「管線內嵌」。

共同設計模式：


輸入 → [閘門 1: 資格篩選] → 處理 → [閘門 2: 中間驗證] → 精煉 → [閘門 3: 最終驗收] → 輸出
        PICO 問題界定          CRAAP 品質評估          IMRAD 結構驗證
        RAG 查詢分類            相關性評估              交叉驗證
        deep-research Phase 1   validate_report.py      verify_citations.py

啟示：我們的 Daily Digest 系統已在 Hook 機器強制層（ADR-005）實現了類似的管線內嵌品質閘門。可借鑑 deep-research 的「最多 3 次重試後停止」策略，為 Hook 攔截加入「降級而非永久阻塞」的機制。

洞見六：自治閉環的層級模型 —— Agent Harness × SDT 動機連續體 × 五階段知識累積

發現：Agent Harness 的自治等級演進、SDT 的動機連續體六階段、知識庫研究策略師的五階段累積模型，三者都描述了「從外部驅動到內在自主」的漸進過程。

| 層級 | Agent Harness 自治 | SDT 動機連續體 | 知識累積五階段 |

|------|-------------------|---------------|---------------|

| 0 | 完全手動觸發 | 無動機（Amotivation） | — |

| 1 | 排程驅動 + 手動干預 | 外在調節（為避免懲罰） | foundation（基礎建立） |

| 2 | 排程驅動 + 自動重試 | 內攝調節（「應該做」） | mechanism（機制深化） |

| 3 | runtime policy 自動調整 | 認同調節（認識到價值） | application（應用實踐） |

| 4 | recovery worker 自治修復 | 整合調節（與身份一致） | optimization（優化精進） |

| 5 | 完全自主管理（目標） | 內在動機（活動即獎賞） | synthesis（整合昇華） |

深度洞見：Agent Harness 目前處於 Level 2-3 之間（有 runtime policy 但 recovery worker 剛實作），對應 SDT 的「內攝→認同」過渡期。要達到真正的自治（Level 4-5），需要的不僅是技術能力，更是「自我認同」——Agent 需要有自己的「mission」（已在 context/mission.yaml 定義 G01-G05），讓每次決策都出自對使命的認同而非對失敗的恐懼。這與無作四諦的「無道可修」形成深層呼應——最高等級的自治不是「更好的控制」，而是「控制本身消融於自然運作」。

    化儀四教           挑戰-技能        無作四諦

三支柱模型：

  1. 路由調度（What to do）：根據接收者狀態選擇最適策略
  2. 認知校準（How to do）：動態維持挑戰-能力的最佳平衡
  3. 驗證觀照（Why it matters）：確保行動與實相/目標的一致性

任何自適應系統——無論是 AI 管線、人類認知、還是修行體系——都需要這三個支柱同時運作才能達到真正的「自治」。缺少路由調度則盲目，缺少認知校準則低效，缺少驗證觀照則偏離方向。

四、對系統優化的具體建議

基於本次洞見分析，提出以下可行動建議：

  1. Agent Harness 引入 Adaptive RAG 式分類器：在 Phase 1 加入查詢複雜度評估，簡單任務直接執行、複雜任務啟動多代理模式（對應化儀「不定教」精神）
  2. 待辦事項加入 SDT 動機標籤：在 Todoist 任務中標注動機類型（外在/認同/內在），排序時給予內在動機任務更高權重
  3. deep-research 四層驗證模型：將驗證從二腳本（析式+結構）擴展為四層（析式→整體一致→語義連貫→目的性），與無作四諦的四層遞進對應
  4. Auto-Continuation 加入「恥小慕大」機制：接續 Agent 不僅讀取前文風格，還應評估前文深度，若深度不足則主動深化而非僅延續
