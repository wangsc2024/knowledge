一、專案概述

FastMCP（PrefectHQ/fastmcp）是由 Prefect 維護的開源 Python 框架，專門用於快速構建 Model Context Protocol（MCP）伺服器、客戶端與應用程式。截至研究日已累積 23,200+ Stars、1,800+ Forks、197 位貢獻者，採用 Apache-2.0 開源授權。最新版本 v3.0.2 於 2026 年 2 月 22 日發布。

核心定位

FastMCP 自稱為「The fast, Pythonic way to build MCP servers and clients」，其口號為「Move fast and make things」。框架的核心價值在於將 MCP 協議的複雜性（schema 生成、驗證、傳輸協商、協議生命週期管理）全部封裝，讓開發者只需關注業務邏輯。

里程碑級影響力

  • FastMCP 1.0（2024）被正式併入官方 MCP Python SDK
  • 每天超過 100 萬次下載
  • 某版本的 FastMCP 為 70% 的 MCP 伺服器提供動力（跨所有語言）
  • 被 GitHub 官方部落格列為 Top 10 開源 AI 專案

二、技術架構

2.1 三大支柱（Three Pillars）

FastMCP 的架構建立在三個核心概念上：

| 支柱 | 用途 | 說明 |

|------|------|------|

| Servers | 暴露工具/資源/提示 | 透過裝飾器將 Python 函數轉換為 MCP 相容的工具 |

| Clients | 連接 MCP 伺服器 | 支援本地/遠端、程式化/CLI 連接，完整協議支援 |

| Apps | 互動式 UI | 直接在對話中渲染互動界面 |

2.2 三原語架構（v3.0 核心設計）

FastMCP 3.0 從「功能堆疊」轉變為「設計框架」，提出「Context Applications」概念——管理資訊流向 Agent 的自適應系統。架構由三個基本原語構成：

  1. Components（元件）：MCP 的原子單位

- Tools：可執行操作（函數呼叫）

- Resources：唯讀資料存取

- Prompts：可重用的訊息模板

  1. Providers（提供者）：回答「元件從何而來？」

- LocalProvider：經典 @tool 裝飾器體驗

- FileSystemProvider：目錄掃描 + 熱重載（reload=True）

- SkillsProvider：將指令檔案暴露為 MCP 資源

- OpenAPIProvider：REST API 自動轉換

  1. Transforms（變換）：修改 Provider 行為的中介層

- VersionFilter：版本控制與相容性管理

- AuthMiddleware：認證中介層

- 自定義變換：不改原始碼即可修改行為

2.3 傳輸協議支援

| 傳輸方式 | 用途 | 官方 SDK 支援 |

|---------|------|-------------|

| stdio | 本地 CLI 工具 | 有 |

| SSE | 即時通訊（Server-Sent Events） | 有限 |

| HTTP/WebSocket | 網路服務 | 有限 |

| Memory | 單元測試/嵌入式環境 | 無 |

FastMCP 的代理模式（Proxy）可解鎖傳輸自由度——不再受限於後端伺服器的原始傳輸方式。

三、功能特色（v3.0 新增）

3.1 元件版本控制


@mcp.tool(version="1.0")
def search(query: str) -> list:
    """搜尋（v1.0）"""
    return basic_search(query)

@mcp.tool(version="2.0")
def search(query: str, filters: dict = None) -> list:
    """搜尋（v2.0，支援過濾）"""
    return advanced_search(query, filters)

自動暴露最高版本，透過 VersionFilter 保持向下相容。

3.2 細粒度認證與授權

  • 每元件 auth 參數：@tool(auth=require_admin)
  • AuthMiddleware：群組級別的防禦縱深
  • 延續 v2 的 OAuth 整合，新增細粒度控制

3.3 OpenTelemetry 原生整合

內建標準化追蹤，覆蓋工具呼叫、資源讀取、提示渲染等關鍵路徑，實現生產級可觀測性。

3.4 背景任務（SEP-1686 整合）

透過 Docket 啟動長時間任務，避免超時，維持回應性——這對生產環境至關重要。

3.5 開發者體驗

  • 熱重載：fastmcp dev server.py 自動監測檔案變更
  • 裝飾器即函數：@tool 裝飾的函數仍可直接匯入和測試
  • 同步函數自動 threadpool：無需手動 async 包裝

四、與官方 MCP SDK 的比較

| 維度 | 官方 MCP Python SDK | FastMCP |

|------|-------------------|---------|

| API 簡潔度 | 中等（需手動配置） | 極高（裝飾器一行搞定） |

| 開發效率 | 中等 | 極高 |

| 傳輸支援 | stdio, SSE | stdio, SSE, HTTP, WS, Memory |

| FastAPI 整合 | 無 | 內建支援 |

| OpenAPI 轉換 | 無 | 自動生成 MCP 伺服器 |

| 代理模式 | 無 | 完整支援 |

| 測試工具 | 基本 | Memory 傳輸 + 內建測試支援 |

| 客戶端使用 | 中等 | 極簡 |

| 版本控制 | 無 | 元件級版本管理 |

| 認證 | 基本 | 細粒度 per-component + middleware |

核心差異：官方 SDK 提供底層協議支援，FastMCP 提供高層抽象——兩者互補而非競爭（FastMCP 1.0 已被併入官方 SDK）。

五、使用方式（快速上手）

安裝


uv pip install fastmcp

建立工具伺服器


from fastmcp import FastMCP

mcp = FastMCP("我的 MCP 伺服器")

@mcp.tool
def add(a: int, b: int) -> int:
    """加法計算"""
    return a + b

@mcp.resource("config://app")
def get_config() -> str:
    """取得應用配置"""
    return "debug=true"

@mcp.prompt
def review_code(code: str) -> str:
    """程式碼審查提示模板"""
    return f"請審查以下程式碼：\n{code}"

if __name__ == "__main__":
    mcp.run()

客戶端連接


from fastmcp import Client

async with Client("http://127.0.0.1:8001/sse") as client:
    result = await client.call_tool("add", {"a": 5, "b": 3})
    print(result)  # 8

從 OpenAPI 自動生成


from fastmcp import FastMCP

mcp = FastMCP.from_openapi("https://api.example.com/openapi.json")
mcp.run()

七、潛在應用（Daily-Digest-Prompt 專案）

7.1 知識庫 MCP 化

目前 daily-digest-prompt 的知識庫 API（localhost:3000）使用 curl 直接呼叫。可用 FastMCP 封裝為標準 MCP 伺服器：


from fastmcp import FastMCP
import httpx

mcp = FastMCP("KB MCP Server")

@mcp.tool
async def search_kb(query: str, top_k: int = 5) -> list:
    """搜尋知識庫"""
    async with httpx.AsyncClient() as client:
        resp = await client.post("http://localhost:3000/api/search/hybrid",
                                 json={"query": query, "topK": top_k})
        return resp.json()["results"]

@mcp.tool
async def import_note(title: str, content: str, tags: list[str]) -> dict:
    """匯入筆記到知識庫"""
    async with httpx.AsyncClient() as client:
        resp = await client.post("http://localhost:3000/api/import",
                                 json={"notes": [{"title": title, "contentText": content,
                                                   "tags": tags, "source": "import"}],
                                        "autoSync": True})
        return resp.json()

7.2 Skill 系統 MCP 橋接

利用 SkillsProvider 將 23 個 SKILL.md 暴露為 MCP 資源，讓任何 MCP 相容的 LLM 都能查詢 Skill 索引。

7.3 自動任務 MCP 代理

18 個自動任務的 Todoist 查詢、路由、計分邏輯可封裝為 MCP 工具，實現跨框架可組合性。

7.4 OpenTelemetry 可觀測性

FastMCP 3.0 的原生 OTel 整合可替代或補充現有的 post_tool_logger.py JSONL 日誌系統，提供標準化的分散式追蹤。

八、優缺點評估

優點

  1. 極低上手門檻：裝飾器 API 讓新手 5 分鐘內建立第一個 MCP 伺服器
  2. 設計哲學成熟：v3.0 的三原語架構（Components/Providers/Transforms）展現清晰的關注點分離
  3. 生態影響力巨大：70% 市佔率意味著社群資源豐富、問題解答快速
  4. 生產就緒：版本控制、認證、OTel、背景任務等企業級功能齊全
  5. 傳輸靈活性：代理模式解鎖傳輸協議自由切換
  6. 測試友善：Memory 傳輸讓單元測試無需啟動真實伺服器
  7. OpenAPI 自動轉換：現有 REST API 一行程式碼變 MCP 伺服器

缺點

  1. Python 限定：TypeScript 版由不同作者（punkpeye/fastmcp）維護，API 不完全一致
  2. 與官方 SDK 的定位模糊：v1 被合併但 v2/v3 獨立發展，新手可能困惑該用哪個
  3. Prefect 綁定傾向：免費託管（Prefect Horizon）和背景任務（Docket）依賴 Prefect 生態
  4. MCP 協議本身仍在演進：協議變更可能導致破壞性更新
  5. 企業功能文件有限：v3 的部分進階功能（如 VersionFilter 具體用法）文件仍待完善
