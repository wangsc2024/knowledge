mergekit — 開源 LLM 模型合併工具集深度研究

專案概述

mergekit 是由 Arcee AI 開發的開源 LLM（大型語言模型）模型合併工具集，專注於解決一個核心問題：如何在不需要額外訓練的前提下，將多個預訓練或微調模型的能力合併到單一模型中。

  • GitHub：https://github.com/arcee-ai/mergekit
  • Stars：6,822（截至 2026-03-01）
  • Forks：669

核心價值

模型合併（Model Merging）是一種強大的技術，能在不增加推論成本的前提下：

  1. 組合多個專精模型：將數學、程式碼、對話等各領域微調模型合為一體
  2. 無需訓練資料：直接在權重空間操作，無需存取原始訓練資料
  3. 零額外推論成本：合併後的模型大小與單一模型相同，不像 ensemble 需要多次推論
  4. 創意組合：透過創造性的模型組合產生新的能力

這與傳統的模型集成（Ensemble）截然不同——集成需要同時執行多個模型，而合併後是單一模型。

技術架構

1. Out-of-Core 計算引擎

mergekit 的最大技術亮點是其 out-of-core 架構設計：

  • 延遲載入（Lazy Loading）：使用 safetensors 的 memory-mapped I/O，不將整個模型載入記憶體
  • DAG 任務圖：核心引擎基於 networkx 建構有向無環圖（DAG），將合併操作分解為可排程的任務節點
  • 串流處理：逐層處理權重張量，峰值記憶體遠低於模型總大小
  • 最低硬體需求：可在純 CPU 環境執行，或僅需 8GB VRAM 的 GPU 加速

這意味著即使在消費級硬體上，也能合併 70B 參數的大型模型。

4. YAML 驅動的配置系統

所有合併操作透過 YAML 配置文件描述，具備階層化參數覆寫：


merge_method: ties
base_model: meta-llama/Llama-3-8B
parameters:
  density: 0.5     # 全域參數
slices:
  - sources:
      - model: codellama/CodeLlama-7b
        parameters:
          weight: 0.7  # 每層/每模型參數覆寫

參數優先級：slices.sources.parameters > slices.parameters > models.parameters > parameters

16 種合併演算法

mergekit 實作了業界最完整的模型合併演算法集：

基礎方法

| 方法 | 核心思想 | 模型數 | 適用場景 |

|------|---------|--------|--------|

| Linear | 加權平均 | >=2 | Model Soup、多 checkpoint 平均 |

| SLERP | 球面線性插值 | 2 | 兩模型間平滑過渡 |

| NuSLERP | 增強版 SLERP | 2 | 更靈活的權重控制 |

| Multi-SLERP | 重心球面插值 | >=2 | 多模型球面平均 |

| Karcher Mean | 黎曼重心 | >=2 | 幾何上最正確的平均 |

Task Vector 方法（核心創新區）

| 方法 | 核心思想 | 特色 |

|------|---------|------|

| Task Arithmetic | 任務向量加法 | 技能轉移與組合的基礎 |

| TIES | 稀疏化 + 符號共識 | 減少多模型干擾 |

| DARE | 隨機剪枝 + 重縮放 | 更穩健的技能保留 |

| DELLA | 自適應幅度剪枝 | 優先保留重要變化 |

| Model Breadcrumbs | 離群值移除 | 精煉任務向量 |

| SCE | 方差自適應權重 | 動態層級加權 |

特殊方法

| 方法 | 用途 |

|------|------|

| Model Stock | 幾何權重計算（>=3 模型） |

| Nearswap | 相似區域選擇性合併 |

| Arcee Fusion | 動態閾值特徵融合 |

| Passthrough | Frankenmerging 層堆疊 |

| RAM | 張量局部 RAM+ 變體（2026-01 新增） |

進階功能

1. Mixture of Experts (MoE) 建構（mergekit-moe）

將多個 Dense 模型組合成 MoE 架構（如 Mixtral 格式）：


base_model: NousResearch/Hermes-2-Pro-Mistral-7B
gate_mode: hidden  # 用隱藏狀態初始化 gate
experts:
  - source_model: BioMistral/BioMistral-7B
    positive_prompts: ['As a doctor of medicine,']
  - source_model: deepseek-ai/deepseek-coder-7b
    positive_prompts: ['Write a Python function']

三種 Gate 初始化：hidden（品質最好）、cheap_embed（低硬體需求）、random（用於後續訓練）。

支援 Mixtral、Deepseek MoE、Qwen MoE 輸出架構。

2. 演化合併搜索（mergekit-evolve）

使用 CMA-ES 演化演算法自動搜索最佳合併參數：

  • 整合 EleutherAI 的 lm-evaluation-harness 作為評估函數
  • 支援 Ray 分散式叢集執行
  • 自動最佳化權重、密度等超參數
  • 靈感來自 SakanaAI 的演化模型合併論文

3. LoRA 萃取（mergekit-extract-lora）

從微調模型萃取 PEFT 相容的低秩近似（LoRA adapter）：


mergekit-extract-lora --model finetuned_model \
  --base-model base_model --out-path lora_output

4. Tokenizer 移植（mergekit-tokensurgeon）

使用正交匹配追蹤（OMP）等方法，在模型間移植 Tokenizer：

  • 10 種近似策略（OMP、PCA、子詞拆解等）
  • 無需額外訓練
  • 適用於推測解碼（Speculative Decoding）的草稿模型建構
  • 2026 年發表相關論文

5. 多階段合併（mergekit-multi）

支援在單一 YAML 中定義多步驟合併工作流，後步可引用前步輸出。

6. Raw PyTorch 合併（mergekit-pytorch）

支援非 HuggingFace 格式的純 PyTorch 模型（.pt / .safetensors）合併。

使用方式

安裝


git clone https://github.com/arcee-ai/mergekit.git
cd mergekit
pip install -e .          # 基礎安裝
pip install -e .[evolve]  # 含演化搜索
pip install -e .[vllm]    # 含 vLLM 加速評估

需求：Python >= 3.10、PyTorch >= 2.0

基礎合併範例


# config.yml - TIES 合併三個模型
merge_method: ties
base_model: meta-llama/Llama-3-8B
parameters:
  density: 0.5
  weight: 1.0
models:
  - model: codellama/CodeLlama-7b
    parameters:
      weight: 0.7
      density: 0.6
  - model: WizardLM/WizardLM-7B
    parameters:
      weight: 0.3
dtype: bfloat16


mergekit-yaml config.yml ./merged-model --cuda

上傳到 HuggingFace


huggingface-cli login
huggingface-cli upload username/my-merged-model ./merged-model .

潛在應用與自身專案連結

1. 本地模型客製化管線

結合 Daily-Digest-Prompt 專案的 Unsloth 微調研究，可建構完整的模型客製化管線：


Unsloth 微調 → mergekit 合併 → Ollama/vLLM 部署

例如：將針對「屏東政策分析」微調的模型與「佛學知識問答」微調的模型合併，產出一個多能力的本地模型。

2. 模型能力組合

使用 TIES 或 DARE 方法，可將不同領域的開源模型合併：

  • 基礎模型（如 Llama 3）+ 中文增強模型 + 程式碼模型
  • 產出一個同時擅長中文對話、程式撰寫的合併模型

3. LoRA 萃取與輕量部署

使用 mergekit-extract-lora 從大型微調模型萃取 LoRA，減少部署所需的儲存空間和記憶體。

4. 演化搜索最佳化

mergekit-evolve 可自動搜索最佳合併參數，搭配自定義評估任務（如知識庫問答正確率），找出最適合特定用途的合併配方。

5. MoE 建構實驗

使用 mergekit-moe 建構個人版 Mixture of Experts 模型，為不同任務類型分配不同專家模型。

優缺點評估

優點

  1. 演算法最完整：16 種合併方法，涵蓋從簡單平均到演化搜索
  2. 極低硬體門檻：Out-of-core 架構讓消費級硬體也能合併大模型
  3. 學術嚴謹：EMNLP 論文發表，每種方法都有學術論文支撐
  4. 生態完善：10 個 CLI 工具覆蓋合併、LoRA 萃取、MoE 建構、Tokenizer 移植等
  5. YAML 驅動：聲明式配置，可版本控制、可重現
  6. 廣泛模型支援：Llama、Mistral、Qwen、GPT-NeoX、GLM4 等主流架構
  7. 社群活躍：HuggingFace 上數千個合併模型使用 mergekit 產生

缺點

  1. 主要開發者集中：247 commits 中大部分來自單一開發者（cg123），存在巴士因子風險
  2. 文件偏技術：缺乏「何時選擇哪種方法」的決策指南（需要自己實驗）
  3. 評估整合有限：mergekit-evolve 依賴 lm-eval-harness，自定義評估需要較多設定
  4. 缺乏視覺化：無內建的合併效果視覺化或比較工具
  5. Open Issues 偏多：254 個未關閉議題，部分長期未回覆
