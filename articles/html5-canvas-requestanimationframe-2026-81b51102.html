研究日期：2026-02-27
研究動機：daily-digest-prompt 專案的創意遊戲優化自動任務（Pac-Man v1.1、Frogger v2.1）大量使用 HTML5 Canvas 渲染引擎，需要建立系統性的效能優化知識，確保遊戲在各種設備上流暢運行。

---

一、技術概述

HTML5 Canvas 是瀏覽器原生的 2D 繪圖 API，透過 JavaScript 操作像素級渲染。在遊戲開發中，Canvas 2D Context 需要每幀重繪所有可見元素，因此效能優化直接決定遊戲的流暢度。以 60 FPS 為目標，每幀預算僅 16.67ms，包含物理更新、碰撞偵測、渲染繪製和垃圾回收。現代瀏覽器的 GPU 加速雖減輕部分負擔，但 Canvas 2D 的 draw call 開銷、狀態機切換成本和記憶體管理仍是效能瓶頸。

---

二、核心概念與原理

2.1 requestAnimationFrame 遊戲迴圈

requestAnimationFrame (rAF) 是瀏覽器提供的動畫排程 API，相較 setTimeout/setInterval 有三大優勢：

  1. 與顯示器刷新同步：60Hz 螢幕每 16.67ms 回調一次，120Hz 每 8.33ms
  2. 背景標籤頁自動節流：不可見時暫停回調，節省電池
  3. 避免 macrotask 佇列衝突：不像 setTimeout 會與事件處理競爭

#### 固定時間步長（Fixed Timestep）


const TARGET_FPS = 60;
const FRAME_INTERVAL = 1000 / TARGET_FPS; // 16.67ms
let previousTime = 0;

function gameLoop(currentTime) {
  requestAnimationFrame(gameLoop);
  const deltaTime = currentTime - previousTime;

  if (deltaTime >= FRAME_INTERVAL) {
    // 關鍵：校正累計誤差，防止「死亡螺旋」
    previousTime = currentTime - (deltaTime % FRAME_INTERVAL);
    updatePhysics(FRAME_INTERVAL);
  }
  // 渲染可以每幀都執行（插值平滑）
  draw();
}
requestAnimationFrame(gameLoop);

死亡螺旋（Spiral of Death）防護：若某幀因 GC 或複雜運算延遲，不校正的話後續所有幀都會偏移。previousTime = currentTime - (deltaTime % FRAME_INTERVAL) 用模運算「倒轉時鐘」，確保幀排程不會累計漂移。

#### 輸入處理模式


// 正確：追蹤按鍵狀態，在遊戲迴圈中讀取
const pressedKeys = new Set();
document.addEventListener('keydown', (e) => pressedKeys.add(e.key));
document.addEventListener('keyup', (e) => pressedKeys.delete(e.key));

function updatePhysics() {
  if (pressedKeys.has('ArrowLeft')) player.moveLeft();
  if (pressedKeys.has('ArrowRight')) player.moveRight();
}

2.2 Canvas 渲染管線

Canvas 2D Context 是一個狀態機：每次修改 fillStyle、strokeStyle、globalAlpha 等屬性都有開銷。渲染管線的優化核心是減少 draw call 和狀態切換。

三、最佳實踐與常見陷阱

3.1 渲染優化（七大技巧）

#### 技巧 1：離屏 Canvas 預渲染

對複雜但不常變化的圖形（如精靈動畫幀），先繪製到離屏 Canvas 快取，再用 drawImage() 貼到主畫布。


// 預渲染精靈到離屏 Canvas
const spriteCache = document.createElement('canvas');
spriteCache.width = 64;
spriteCache.height = 64;
const spriteCtx = spriteCache.getContext('2d');
drawComplexSprite(spriteCtx); // 一次性繪製

// 遊戲迴圈中直接貼圖（極快）
function draw() {
  ctx.drawImage(spriteCache, player.x, player.y);
}

注意：離屏 Canvas 尺寸要緊貼內容，過大會浪費 GPU 傳輸頻寬。

#### 技巧 2：多層 Canvas 分離

將遊戲場景分為靜態背景層、動態遊戲層、UI 覆蓋層：


<div id="stage">
  <canvas id="bg-layer" style="z-index:1"></canvas>
  <canvas id="game-layer" style="z-index:2"></canvas>
  <canvas id="ui-layer" style="z-index:3"></canvas>
</div>

背景層每關只繪製一次，遊戲層每幀更新，UI 層僅在分數/生命變化時重繪。這大幅減少每幀的像素操作量。

#### 技巧 3：批次繪製呼叫


// 錯誤：每條線獨立 stroke — N 次 draw call
for (let i = 0; i < points.length - 1; i++) {
  ctx.beginPath();
  ctx.moveTo(points[i].x, points[i].y);
  ctx.lineTo(points[i+1].x, points[i+1].y);
  ctx.stroke();
}

// 正確：合併為單一路徑 — 1 次 draw call
ctx.beginPath();
for (let i = 0; i < points.length - 1; i++) {
  ctx.moveTo(points[i].x, points[i].y);
  ctx.lineTo(points[i+1].x, points[i+1].y);
}
ctx.stroke();

#### 技巧 4：最小化狀態機切換


// 錯誤：交替切換 fillStyle
for (let i = 0; i < 100; i++) {
  ctx.fillStyle = (i % 2) ? 'red' : 'blue';
  ctx.fillRect(i * 10, 0, 10, 480);
}

// 正確：按顏色分組繪製
ctx.fillStyle = 'red';
for (let i = 0; i < 50; i++) ctx.fillRect(i * 20, 0, 10, 480);
ctx.fillStyle = 'blue';
for (let i = 0; i < 50; i++) ctx.fillRect(i * 20 + 10, 0, 10, 480);

#### 技巧 5：整數座標避免子像素渲染

瀏覽器對浮點數座標會進行抗鋸齒插值運算，消耗額外效能：


// 錯誤：浮點座標觸發子像素渲染
ctx.drawImage(sprite, 10.3, 20.7);

// 正確：取整（位元運算最快）
ctx.drawImage(sprite, (10.3 + 0.5) | 0, (20.7 + 0.5) | 0);

#### 技巧 6：關閉不需要的 Alpha 通道


// 不需要透明度時關閉 alpha，讓瀏覽器跳過合成運算
const ctx = canvas.getContext('2d', { alpha: false });

#### 技巧 7：避免昂貴操作

  • shadowBlur：陰影模糊是最耗效能的屬性之一，遊戲中盡量用預渲染陰影替代
  • 文字渲染：fillText() 比 drawImage() 慢 5-10 倍，頻繁更新的分數建議預渲染為圖片
  • getImageData()：像素級讀取觸發 GPU 到 CPU 回傳，極其昂貴

3.2 記憶體管理

#### 物件池模式（Object Pool）

遊戲中頻繁建立/銷毀的物件（粒子、子彈、道具）會觸發 GC 暫停，造成卡頓。物件池預分配固定數量物件，用完歸還而非銷毀。


class ParticlePool {
  constructor(size) {
    this.pool = Array.from({ length: size }, () => ({
      x: 0, y: 0, vx: 0, vy: 0, life: 0, active: false
    }));
  }

  acquire() {
    const particle = this.pool.find(p => !p.active);
    if (particle) particle.active = true;
    return particle;
  }

  release(particle) {
    particle.active = false;
  }

  update(dt) {
    for (const p of this.pool) {
      if (!p.active) continue;
      p.x += p.vx * dt;
      p.y += p.vy * dt;
      p.life -= dt;
      if (p.life <= 0) this.release(p);
    }
  }
}

// 預分配 200 個粒子，整場遊戲零 GC
const particles = new ParticlePool(200);

#### Dirty Rectangles（髒矩形）

只重繪變化的區域，而非每幀清除整個 Canvas：


function updateAndDraw() {
  const prevBounds = { x: entity.x - 2, y: entity.y - 2, w: entity.w + 4, h: entity.h + 4 };
  entity.update();
  const newBounds = { x: entity.x - 2, y: entity.y - 2, w: entity.w + 4, h: entity.h + 4 };

  ctx.clearRect(prevBounds.x, prevBounds.y, prevBounds.w, prevBounds.h);
  ctx.clearRect(newBounds.x, newBounds.y, newBounds.w, newBounds.h);
  entity.draw(ctx);
}

適用場景：物件數量少且移動幅度小的遊戲。物件密集的動作遊戲中，髒矩形重疊計算的開銷可能超過全屏重繪。

3.3 高 DPI 螢幕適配


function setupHiDPI(canvas) {
  const dpr = window.devicePixelRatio || 1;
  const rect = canvas.getBoundingClientRect();
  canvas.width = rect.width * dpr;
  canvas.height = rect.height * dpr;
  const ctx = canvas.getContext('2d');
  ctx.scale(dpr, dpr);
  canvas.style.width = `${rect.width}px`;
  canvas.style.height = `${rect.height}px`;
}

五、程式碼範例：完整優化遊戲迴圈


class Game {
  constructor(canvas) {
    this.ctx = canvas.getContext('2d', { alpha: false });
    this.targetFps = 60;
    this.frameInterval = 1000 / this.targetFps;
    this.previousTime = 0;
    this.particles = new ParticlePool(200);
    this.pressedKeys = new Set();
    this.fps = 0;
    this.frameCount = 0;
    this.lastFpsUpdate = 0;
    this.bindInput();
  }

  bindInput() {
    document.addEventListener('keydown', (e) => this.pressedKeys.add(e.key));
    document.addEventListener('keyup', (e) => this.pressedKeys.delete(e.key));
  }

  start() {
    requestAnimationFrame((t) => this.loop(t));
  }

  loop(currentTime) {
    requestAnimationFrame((t) => this.loop(t));
    const delta = currentTime - this.previousTime;

    this.frameCount++;
    if (currentTime - this.lastFpsUpdate >= 1000) {
      this.fps = this.frameCount;
      this.frameCount = 0;
      this.lastFpsUpdate = currentTime;
    }

    if (delta >= this.frameInterval) {
      this.previousTime = currentTime - (delta % this.frameInterval);
      this.update(this.frameInterval / 1000);
    }
    this.draw();
  }

  update(dt) {
    this.handleInput();
    this.updateEntities(dt);
    this.particles.update(dt);
    this.checkCollisions();
  }

  draw() {
    this.ctx.clearRect(0, 0, this.ctx.canvas.width, this.ctx.canvas.height);
    this.drawBackground();
    this.drawEntities();
    this.drawParticles();
    this.drawUI();
  }
}
