改進提案

P0 已落實：雙層 tool boundary 契約

  • 靈感來源: browser-use/browser-use + e2b-dev/E2B
  • 變更檔案:

- config/github-scout-review-policy.yaml

- templates/auto-tasks/github-scout.md

- prompts/team/todoist-auto-github_scout.md

  • 核心改動: 若專案同時涉及 runtime sandbox 與瀏覽器/桌面工具 API，tool_boundary 必須拆為 execution_environment_boundary 與 tool_api_boundary。
  • 審查分數: feasibility 9 / stability 9
  • rollback/降級: 還原三個契約檔案或將雙層欄位降級為建議。

P1 待審：雙層 boundary 遙測 sidecar

  • 靈感來源: e2b-dev/E2B + guardrails-ai/guardrails
  • 計畫檔: docs/plans/github-scout-20260403-dual-layer-tool-boundary-telemetry.md
  • 核心構想: 以 sidecar 或 feature flag，把 execution_environment_boundary、tool_api_boundary、operator override 與 rollback 結果對齊到 trace。
  • 審查分數: feasibility 7 / stability 7
  • rollback/降級: 維持 review-only，不直接碰核心 logger/error schema。

審查結論

  • 採納低風險方案，原因是現有單一 tool_boundary 容易把 sandbox control plane 與 browser API 信任面混寫。
  • 高風險方案暫不直接實裝，原因是會牽動 post_tool_logger、error schema 與下游分析契約。
  • 本輪研究限制：2026-04-03 僅使用 GitHub repo/release/org 搜尋快照，未直接做即時 GitHub REST 觀測；stars 與最近更新需後續再複核。

已落實項目

  • 已把 execution_environment_boundary 與 tool_api_boundary 納入 github_scout 的 review policy、auto-task template 與 team prompt 契約。
