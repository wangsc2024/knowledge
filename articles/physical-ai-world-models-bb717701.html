Physical AI 與 World Models：多模態 Agent 從感知走向行動的技術路線：深度技術研究報告

執行摘要

本研究聚焦 Physical AI 與 world models 如何讓多模態 Agent 從「看得懂」走向「做得到」。綜合 NVIDIA Cosmos、Google DeepMind Genie 2、Physical Intelligence pi0 與 OSWorld 可見，產業正收斂到世界生成或預測、合成資料擴充、再到 vision-language-action 後訓練的閉環。關鍵不只是模型變大，而是補上可模擬未來、回放錯誤與安全評測的世界模型層；這對個人 Agent 系統最直接的啟發，是先建立任務沙盒、回放資料與閉環評測。

研究背景與動機

多模態 Agent 在過去兩年快速從文字互動走向看圖、用工具、操作電腦與控制機器人，但這個演進很快遇到同一個問題：感知能力提升不等於行動能力成熟。純語言模型可以解釋畫面、推論指令，卻不一定知道下一步操作會造成什麼狀態變化，也缺乏足夠安全且低成本的試錯環境。Physical AI 與 world model 因此成為關鍵交會點。前者關注 Agent 如何在現實或擬真的物理世界中感知、規劃並執行任務；後者則提供一種可預測環境演化、生成新場景並支援訓練與評測的表示層。

從產業角度看，NVIDIA、Google DeepMind 與 Physical Intelligence 分別代表三條互補路線：一條偏重 world foundation model 與 synthetic data 工廠，一條偏重可互動世界生成器，一條偏重 generalist policy 與 vision-language-action 執行器。這些路線共同指向一件事：多模態 Agent 若要真正從感知走向行動，需要閉環，而不是單點模型升級。

核心技術解析

第一個核心主張是，Physical AI 要補上的不是更大的感知模型，而是能模擬未來狀態的 world model 層。NVIDIA Cosmos 明確把 world foundation model 描述為 physical AI 的底座，並以 Predict、Transfer、Reason 三條能力線支撐環境生成、模擬轉寫實與物理推理 [來源：Physical AI with World Foundation Models | NVIDIA Cosmos, 2025]。NVIDIA Research 也把 physical AI 描述成需要同時具備 policy model 與 world model 的雙數位分身 [來源：Cosmos World Foundation Model Platform for Physical AI | NVIDIA Research, 2025]。Google DeepMind 的 Genie 2 雖不直接面向量產機器人，但同樣把 world model 定位為可生成可互動 3D 世界、讓 embodied agents 在幾乎無限課程中訓練與評估的引擎 [來源：Genie 2: A large-scale foundation world model | Google DeepMind, 2024]。這些證據共同顯示，world model 的價值在於把『未來會發生什麼』轉為可訓練的中介層。

第二個核心主張是，當前前沿系統正收斂到『世界生成或預測 + 合成資料 + VLA/政策後訓練』的閉環。Cosmos 官方頁與研究頁都把 synthetic data、資料整理、WFM 預訓練與下游後訓練串成完整工具鏈 [來源：Physical AI with World Foundation Models | NVIDIA Cosmos, 2025] [來源：Cosmos World Foundation Model Platform for Physical AI | NVIDIA Research, 2025]。中文官方文章更揭露這個平台以極大規模資料與 WFM 為核心，將先行模擬、合成資料與強化學習視為實體 AI 的核心用途 [來源：NVIDIA 開放 Cosmos 世界基礎模型給實體 AI 開發者社群使用, 2025]。另一方面，pi0 與其論文將 vision-language-action flow model 視為 general robot control 的主要執行形式，代表語言理解與連續動作輸出正在被統一到同一框架中 [來源：Our First Generalist Policy | Physical Intelligence, 2024] [來源：$π_0$: A Vision-Language-Action Flow Model for General Robot Control, 2024]。因此，閉環並不是抽象口號，而是資料工廠、世界模型與控制模型的實際分工。

第三個核心主張是，多模態 Agent 的主瓶頸仍是 grounding、operational knowledge 與泛化。OSWorld 在真實電腦環境 benchmark 中指出，即使是具備多模態能力的先進 agent，實際完成開放任務時仍容易卡在環境定位、工具操作與程序知識 [來源：OSWorld: Benchmarking Multimodal Agents for Open-Ended Tasks in Real Computer Environments, 2024]。Genie 2 的路線說明了為何需要大量新課程與新世界來提升泛化 [來源：Genie 2: A large-scale foundation world model | Google DeepMind, 2024]；pi0 則用跨 embodiment 資料與 action expert 來補上控制泛化 [來源：Our First Generalist Policy | Physical Intelligence, 2024]。這說明問題早已不是語言模型能否理解指令，而是 Agent 是否真的能把理解可靠地對接到環境狀態變化。

第四個核心主張是，數位 Agent 與 physical agent 的路線正在收斂。OSWorld 所代表的真實電腦操作，其實就是一種數位 embodied environment；它暴露的 grounding 與 closed-loop evaluation 問題，與機器人在物理世界的挑戰相互映照 [來源：OSWorld: Benchmarking Multimodal Agents for Open-Ended Tasks in Real Computer Environments, 2024]。Cosmos 把資料工廠與後訓練視為核心，pi0 把感知與動作壓進同一 policy，這些做法對個人 Agent 系統的可借鏡之處，不在硬體，而在於是否有環境回放、任務沙盒、失敗可重演與策略可修正的訓練閉環 [來源：Physical AI with World Foundation Models | NVIDIA Cosmos, 2025] [來源：Our First Generalist Policy | Physical Intelligence, 2024]。

值得注意的分歧在於 world model 的直接角色並未完全統一。NVIDIA Cosmos 把它放在資料與模擬基礎設施的中心；DeepMind Genie 2 偏向把它視為訓練與評測世界的生成器；而 Physical Intelligence 的 pi0 更強調把 vision-language 與 action 直接整合成可泛化 policy。這些差異不代表彼此衝突，而是反映不同團隊對瓶頸位置的判斷不同。若從工程落地來看，前兩者偏向先把世界建立好，再訓練 agent；後者偏向先把 agent 執行器做強，再用資料擴展泛化。

關鍵洞見

  • 已驗證✅（支持來源 4）Physical AI 與 world model 的真正交會點，是把「預測未來」變成可訓練、可評估、可回放的中介層，而不只是更好的感知模型。
  • 已驗證✅（支持來源 3）多模態 Agent 若沒有環境生成與環境評測能力，從 chat agent 升級到 action agent 會很快撞上資料稀缺與安全測試瓶頸。
  • 已驗證✅（支持來源 4）VLA 是把語言理解落地成連續控制的執行層，但它本身不解決資料成本，因此必須和模擬/合成資料管線配套。
  • 已驗證✅（支持來源 3）digital agent 與 physical agent 的技術路線開始收斂：兩者都需要更強的 grounding、planning 與 closed-loop evaluation。
  • 待驗證⚠️（支持來源 2）對個人 Agent 專案而言，最值得借鏡的不一定是造機器人，而是先建立「世界模型化」的任務沙盒與回放資料集。
  • 待驗證⚠️（支持來源 2）未來競爭焦點可能會從單模型能力，轉向資料工廠、模擬器、後訓練與安全護欄的一體化堆疊。

若把這條技術路線放回工程現場，可以看到一個很實際的轉折。過去很多 Agent 專案把重點放在 prompt、工具數量或模型選型，但 Physical AI 的研究提醒我們，真正限制自治能力的往往是環境互動資料不足、錯誤無法重演，以及系統無法在部署前先經歷大量低成本試錯。這也是為什麼 world model、simulator 與 synthetic data 會突然變得重要：它們不是附屬元件，而是讓 Agent 真正能學會行動的訓練與驗證基礎設施。

對個人專案而言，這意味著可以把『世界模型』採用更務實的定義來落地。它不一定是巨型影片生成模型，也可以是任務狀態機、頁面截圖序列、工具呼叫日誌與結果標註所構成的近似世界。只要這一層能幫助系統預測某個動作之後最可能出現的狀態、辨識偏差並回放修正，它就已經在扮演 world model 的工程角色。從這個角度看，Physical AI 並不是離 daily-digest-prompt 很遠的前沿名詞，而是對 Agent 工程方法的一次重新校準。

實務應用建議

  1. 為個人 Agent 建立任務沙盒與回放系統。先把瀏覽器操作、檔案整理、摘要生成等工作流程錄成可重放軌跡，讓 Agent 有可觀察的狀態轉移。
  2. 導入 closed-loop evaluation。不要只看最終答案正確與否，也要追蹤每一步工具呼叫、狀態判斷與失敗恢復是否合理。
  3. 建立 synthetic task generation。參考 world model 的精神，用模板與歷史任務生成更多變體測試集，提升 Agent 對新情境的穩健性。
  4. 把 planning 與 execution 分層。語言模型負責任務理解與策略草擬，執行層負責可驗證的動作，避免單一模型同時承擔所有責任。
  5. 為高風險動作加上安全護欄與人工接管點。Physical AI 強調先模擬再部署，個人 Agent 系統同樣應先在沙盒驗證，再放到真實工作流。

與現有知識的連結

這份研究與既有 AI Agent 自治度評測框架研究可以直接銜接。前一份研究處理的是能力 benchmark、過程指標與治理 KPI；本研究補上的是另一個更底層的問題：Agent 需要怎樣的世界表示與環境生成機制，才能讓那些 KPI 真正有提升空間。換句話說，自治度評測回答『怎麼量』，Physical AI 與 world models 回答『能力從哪裡長出來』。對 daily-digest-prompt 而言，這表示未來可以把研究與自動任務系統升級為有沙盒、有回放、有 synthetic task、有 closed-loop traces 的 learning system，而不只是靜態 pipeline。

未來展望

未來三個方向值得持續追蹤。第一，world model 是否會從研究概念進一步產品化為通用 agent simulator。第二，VLA 與 generalist policy 是否能在更少示範資料下保持跨場景泛化。第三，安全與治理工具是否能跟上閉環 agent 的速度，讓世界生成、後訓練與真實部署之間有可審計的邊界。若這三者同步成熟，下一代多模態 Agent 的競爭重心就會從單模型指標，轉向資料工廠、模擬器、控制器與治理層的整體堆疊。
