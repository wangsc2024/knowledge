專案觀測摘要

本次搜尋主題：Agent Eval 框架（deepeval/agentevals-dev/promptfoo）

1. confident-ai/deepeval（~14k★）

  • 類型：Python LLM 評估框架，50+ 指標
  • Agent 專屬指標（6 項）：task_completion / tool_correctness / step_efficiency / plan_adherence / plan_quality / argument_correctness
  • LLM-as-judge：支援，可自訂 judge 模型
  • CI/CD：pytest 風格，可整合 GitHub Actions
  • 與本系統整合路徑：done_cert.quality_score 可以 6 指標加權平均計算；tools/quality_gate.py 可引入 tool_correctness/task_completion 作為通過門檻

2. promptfoo/promptfoo（13.2k★，2026-03 被 OpenAI 收購）

  • 類型：CLI 評估 + red-teaming，declarative YAML config
  • 主要功能：50+ 漏洞類型（prompt injection/PII/tool misuse）+ 模型比較
  • CI/CD：promptfoo-action GitHub Action 官方支援
  • 注意：OpenAI 收購後中立性需評估，暫不作為主要整合目標

3. agentevals-dev/agentevals（~500★）

  • 類型：OTel trace 驅動，框架無關，no-re-run eval
  • 核心優勢：從既有 OTel spans 評估，無需重跑 LLM 呼叫（降低評估成本）
  • 相容：LangChain / Strands / Google ADK / 任何 OTel 框架
  • 格式：Jaeger JSON / OTLP
  • 與本系統整合路徑：post_tool_logger.py JSONL → OTel span JSON 對映（trace_id/span_id/gen_ai.* 欄位）
