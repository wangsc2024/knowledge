claude-deep-research-skill 深度研究

概述

199-biotechnologies/claude-deep-research-skill 是一個企業級深度研究 Skill，專為 Claude Code 設計。透過 8 階段結構化管線（Pipeline）產出引用驗證的研究報告，搭配來源可信度評分、多搜尋引擎整合與自動化品質驗證機制。

授權：MIT，純 Python 標準函式庫（基本功能無需 pip install）。

---

核心架構：8 階段研究管線

Phase 1: SCOPE（研究範圍界定）

  • 分解研究問題為核心組件
  • 識別利害關係人觀點
  • 定義範圍邊界（in/out scope）
  • 建立成功標準與待驗證假設
  • 使用 Ultrathink（延伸推理）探索多種問題框架

Phase 2: PLAN（策略制定）

  • 識別一級與二級來源
  • 建立知識依賴圖（knowledge dependency graph）
  • 制定搜尋查詢策略與變體
  • Graph-of-Thoughts：分支為多條潛在研究路徑，收斂為最佳策略
  • 定義品質門檻（quality gates）

Phase 3: RETRIEVE（並行資訊收集）

  • Step 0：取得當前日期（防止使用訓練資料年份）
  • 分解為 5-10 個獨立搜尋角度
  • 並行執行：單一訊息發出所有搜尋（5-10 concurrent），加 2-3 個聚焦子 Agent
  • 支援三種搜尋工具：WebSearch（內建）、Exa MCP（語義搜尋）、search-cli（多提供者聚合）

Phase 4: TRIANGULATE（三角驗證）

  • 跨來源交叉比對事實
  • 至少 2 個來源確認同一事實才視為可靠

Phase 4.5: OUTLINE REFINEMENT（大綱精煉）

  • 根據收集到的證據調整報告結構

Phase 5: SYNTHESIZE（整合）

  • 將證據整合為連貫敘事
  • 散文優先（>=80% prose）

Phase 6: CRITIQUE（批判審查）

  • 多人格紅隊（Red-Teaming）：以不同觀點審視研究
  • 可回溯到 Phase 3 填補證據缺口（loop-back）

Phase 7: REFINE（精煉）

  • 回應批判回饋，強化弱論證

Phase 8: PACKAGE（打包交付）

  • 產出三種格式：Markdown、HTML（McKinsey 風格）、PDF
  • 輸出至 ~/Documents/[Topic]_Research_[YYYYMMDD]/

---

四種研究模式

| 模式 | 階段 | 耗時 | 適用場景 |

|------|------|------|----------|

| Quick | 3 個 | 2-5 分鐘 | 初步探索 |

| Standard | 6 個 | 5-10 分鐘 | 一般研究（預設） |

| Deep | 8 個 | 10-20 分鐘 | 關鍵決策 |

| UltraDeep | 8+ 個 | 20-45 分鐘 | 全面嚴謹審查 |

來源可信度評分系統

使用 scripts/source_evaluator.py 自動評估來源可信度（0-100 分）。依據來源域名、出版類型、作者權威性評分。低於閾值的來源在報告中降權。品質標準：至少 10+ 來源，每個重大主張至少 3 個來源佐證。

---

引用驗證機制

scripts/verify_citations.py 執行三類自動檢查：

  1. DOI 驗證：確認學術來源 DOI 可解析
  2. URL 驗證：確認連結有效（非 404）
  3. 幻覺偵測：標記無來源支撐的主張

驗證迴圈：驗證 -> 修復 -> 重試，最多 3 輪循環。

---

品質門檻（Quality Gates）

報告驗證 9 項自動檢查

  1. 執行摘要長度（200-400 字）
  2. 必要章節齊備
  3. 引用格式正確
  4. 參考書目與引用對應
  5. 無佔位文字（TBD, TODO）
  6. 字數合理（500-10,000）
  7. 至少 10 個來源
  8. 無斷裂內部連結
  9. 散文佔比 >= 80%

Anti-Hallucination 協議

  • 每個事實主張必須即時引用來源
  • 區分事實與綜合分析
  • 明確標記推測
  • 不確定時承認未找到來源而非捏造引用

---

參考書目要求（零容忍）

每個引用都必須有完整書目項目。格式：作者/機構 (年份). 標題. 出版物. URL (Retrieved: 日期)。禁止佔位符、範圍引用、截斷。

---

磁碟持久化引用（sources.json）

引用存於 sources.json，能夠在 context compaction 和 continuation agents 之間存活。報告超過 18K 字時自動啟動遞迴 Agent 機制。

---

與本專案的整合價值

  1. 結構化程度：8 階段管線提供比現有 web-research Skill 更嚴謹的研究框架
  2. 來源評分：自動來源可信度評分可與現有 A/B/C/D 分級對應
  3. 引用驗證：自動化 DOI/URL 驗證補強現有交叉驗證規則
  4. 品質閘道：9 項自動檢查確保報告品質一致性
  5. Anti-Hallucination：系統性防幻覺協議提升研究可信度

使用方式

安裝：git clone https://github.com/199-biotechnologies/claude-deep-research-skill ~/.claude/skills/deep-research

觸發：在 Claude Code 中輸入 Use deep research to [研究問題]
