QA System 安全優化報告 #21

日期：2026-03-01 | 品質分數：5/5 | 第二十一次掃描

優化方向

auth.py 6 處 raw SQL（user_role INSERT + user_department INSERT/SELECT）→ ORM relationship 操作

安全掃描結果

全部綠燈，無新安全問題。已確認：SECRET_KEY 環境變數化、SQL 注入防護（auth.py raw SQL 已全部移除）、XSS 防護、CORS 限制、Cookie 安全、JWT 集中解碼、python-jose CVE 修復、錯誤訊息零洩漏。

已修復問題（6 項品質）

品質修復

  1. 新用戶角色關聯：raw SQL INSERT INTO user_role → user.roles.append(staff_role)，由 SQLAlchemy 自動管理關聯表
  2. 新用戶部門關聯：raw SQL INSERT INTO user_department → user.departments.append(department)
  3. 既有用戶部門查詢：raw SQL SELECT * FROM user_department → department not in user.departments（ORM 自動載入）
  4. 既有用戶部門新增：raw SQL INSERT INTO user_department → user.departments.append(department)
  5. 移除 from sqlalchemy import text：auth.py 不再需要 raw SQL 模組
  6. auth.py 行數：228 行 → 210 行（-18 行，-8%）

ORM 遷移里程碑

至此整個 QA System 主要模組全部完成 ORM 遷移：

  • questions.py：close(#17) → edit(#18) → create(#19) → fetch/list(#20)
  • auth.py：sso_login user_role + user_department(#21)
  • 全專案零 raw SQL 字串操作
