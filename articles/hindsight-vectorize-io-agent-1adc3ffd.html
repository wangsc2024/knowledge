一、專案概述

Hindsight（vectorize-io/hindsight）是一套開源的 AI Agent 記憶系統，核心理念是讓 Agent 的記憶「像人類記憶一樣運作」——具備上下文感知、時間感知、且能形成並更新信念（beliefs）。專案以 MIT 授權釋出，2026 年 3 月登上 GitHub Trending，目前 4.3K Stars。

二、核心架構：四網路記憶模型

Hindsight 將 Agent 記憶組織為四個邏輯網路，區別於傳統的「對話日誌」或「單一向量庫」方式：

2.1 World（世界事實）

  • 儲存關於環境的客觀事實（如「爐子會很燙」）
  • 經由 LLM 抽取結構化事實、實體與關係
  • 自動進行實體解析（Entity Resolution），將不同表述統一為標準形式

2.2 Experience（經驗記憶）

  • 儲存 Agent 自身的互動經歷（如「我碰了爐子，很痛」）
  • 帶有時間戳和上下文元資料
  • 支援多 session 持久化

2.3 Mental Models（心智模型）

  • 由 Reflect 操作自動生成的「活文件」
  • 從原始記憶和經驗中合成高層理解
  • 自動更新——新記憶進入時，相關心智模型自動修訂
  • 範例：「我的技術棧偏好」會隨對話累積自動演進

2.4 Opinion（信念/觀點）

  • 帶有信賴度分數（confidence scores）的推論
  • 明確區分「證據」與「推論」——提供認識論透明度
  • 開發者可審計 Agent 為何如此回答

設計哲學：將記憶視為推理的一級基質（first-class substrate），而非薄薄的檢索層。

---

三、三元操作管線：Retain → Recall → Reflect

3.1 Retain（保留）

  • 將互動轉為結構化、時間感知的記憶格式
  • LLM 抽取關鍵事實、時序資料、實體與關係
  • 正規化為標準形式後存入記憶庫
  • 非同步處理：事實抽取和索引在背景執行，記憶需數秒才可搜尋

3.2 Recall（回憶）

  • 四路並行檢索策略：

1. 語義搜尋（向量相似度）

2. 關鍵字搜尋（BM25）

3. 圖形走訪（實體/時序連結）

4. 時序過濾（時間範圍篩選）

  • 結果經由 Reciprocal Rank Fusion 合併
  • 再通過 Cross-Encoder Reranker 精排
  • 自動 Token 管理：超出 context window 時自動裁剪低相關度記憶

3.3 Reflect（反思）

  • 對既有記憶進行 LLM 推理，形成新連結與洞察
  • 支援風險分析、模式辨識、問答等進階用例
  • 可建立和更新心智模型（Mental Models）
  • 產出帶有信賴度的觀點，並可追溯推理依據

五、MCP 原生整合

Hindsight 提供 MCP（Model Context Protocol）端點，可直接作為 Claude Desktop / Claude Code / Cursor 等工具的記憶伺服器：

設定方式


{
  "mcpServers": {
    "hindsight": {
      "type": "http",
      "url": "http://localhost:8888/mcp/your_bank_id/"
    }
  }
}

Claude Code CLI


claude mcp add --transport http hindsight http://localhost:8888/mcp/your_bank_id/

Memory Banks 隔離

  • URL 路徑控制隔離：每個 bank_id 獨立
  • 適用場景：不同專案、團隊共享知識、多 Agent 系統

六、可借鑑的技術項目（對 Daily-Digest-Prompt 系統的啟示）

6.1 四路並行檢索 → 知識庫搜尋升級

  • 目前知識庫使用 hybrid search（語義 + BM25 RRF）
  • Hindsight 新增了圖形走訪和時序過濾兩個維度
  • 借鑑：為知識庫新增時間衰減因子和實體關係索引，提升「最近修改過的相關筆記」的召回率

6.2 Retain-Recall-Reflect 三元模型 → Agent 記憶升級

  • 目前系統的 digest-memory.json 和 continuity/*.json 是扁平 JSON 記錄
  • Hindsight 的 Reflect 操作能自動從原始記憶合成高層洞察
  • 借鑑：為 continuity 機制新增 reflect 步驟——每 N 次執行自動合成趨勢報告，取代人工閱讀歷史記錄

6.3 證據與推論分離 → 研究品質提升

  • Hindsight 的 Opinion 網路明確區分「事實」和「推論」
  • 借鑑：研究筆記模板新增「證據層」和「推論層」兩個區塊，讓讀者能獨立評估資料來源和分析結論

6.4 Cross-Encoder Reranking → 搜尋精度

  • 目前 RRF 合併後直接取 top-K
  • 借鑑：在 hybrid search 後加入輕量級 cross-encoder 重排（可用 Groq 快速推理），特別對長查詢和模糊查詢有顯著提升

6.5 Mental Models 自動更新 → 知識庫活文件

  • Hindsight 的心智模型隨新記憶自動修訂
  • 借鑑：知識庫每日洞見報告可設計為「活文件」模式——新研究成果自動合併到對應主題的總覽筆記中

6.6 Memory Banks 多租戶隔離 → 多 Agent 記憶分區

  • 目前所有 Agent 共用同一套 continuity 檔案
  • 借鑑：可依 Agent 類型（研究類 / 任務類 / 審查類）建立獨立記憶分區，避免跨類型記憶污染

七、技術實作細節

後端架構

  • 儲存：PostgreSQL + pgvector
  • 索引：稀疏/稠密向量雙重表示
  • 部署：Docker 容器化（API port 8888 + Admin UI port 9999）
  • SDK：Python (pip install hindsight-client) + TypeScript (npm install @vectorize-io/hindsight-client)
  • 嵌入式模式：pip install hindsight-all（無需獨立伺服器）

最小整合範例


from hindsight_client import Hindsight

client = Hindsight(base_url="http://localhost:8888")
client.retain(bank_id="my-bank", content="Alice works at Google")
result = client.recall(bank_id="my-bank", query="Where does Alice work?")
insight = client.reflect(bank_id="my-bank", query="Tell me about Alice")

九、結論與展望

Hindsight 代表了 Agent Memory 從「對話日誌 + 向量搜尋」進化到「結構化知識圖譜 + 多路檢索 + 反思推理」的範式轉移。其四網路架構和 Retain-Recall-Reflect 三元操作為 Agent 記憶系統提供了清晰的設計藍圖。

對於 Daily-Digest-Prompt 系統而言，最具價值的借鑑是：

  1. 四路並行檢索提升知識庫搜尋品質
  2. Reflect 機制讓 continuity 記憶從被動記錄升級為主動合成洞察
  3. MCP 原生整合可直接作為 Claude Code 的持久化記憶層
