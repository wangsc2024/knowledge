kb_system_optimize 2026-05-27：post_tool_logger 監控盲區修補（kb_hybrid_cached cache-read 標籤缺失）

kb_system_optimizemonitoringcachepost_tool_logger技術修補

問題診斷

cache_hit_ratio = 0.202 假警報根因：kb_hybrid_cached.py 透過 Bash 呼叫時，classify_bash() 只加 python tag，未加 cache-read（命中）或 api-call（未命中），導致 compute_system_metrics.py 的公式分子分母皆缺失。

公式：cache_hit_ratio = cache_reads / (cache_reads + cacheable_api_calls)

  • cache_reads：計 cache-read tag 的事件
  • cacheable_api_calls：計 api-call tag 且 summary 含 /api/search/hybrid 的事件

7 天滾動視窗混入前幾日大量原始 API 呼叫，使比例失真；單日 5/26、5/27 實際比例均為 1.000。

修補方案（Plan B）

新增 _apply_kb_hybrid_cached_tags(cmd, tool_output, tags, summary) helper（hooks/post_tool_logger.py），在 main() Bash 分支呼叫 classify_bash 後緊接呼叫：

  • status=hit 或 status=miss_stale_fallback → 加 cache-read（計入分子）
  • status=miss → 加 api-call + summary 補 [/api/search/hybrid]（計入分母，classify_api_call 辨識為 cacheable_kb_search）

驗證結果

  • uv run pytest tests/hooks/test_post_tool_logger.py -q：137/137 PASS（+5 新增迴歸測試）
  • hooks/validate_config.py --check-auto-tasks：96 個自動任務一致
  • Plan A（tests/console 5 errors）：已由 tests/console/conftest.py 的 pytest_ignore_collect 在 5/24 前修復，無需重複處理
