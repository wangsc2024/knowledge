Tetris v2.1 優化心得：成就解鎖系統與遊戲統計追蹤實作

優化日期：2026-02-25

遊戲路徑：D:\Source\game\tetris

優化方向

社交元素 + 玩法機制：為經典俄羅斯方塊加入成就解鎖系統和遊戲統計追蹤面板，提升遊戲粘性與成就感。

技術實作重點

1. 成就系統架構

  • 定義 12 個成就，涵蓋消行、Tetris、Combo、分數、等級五大維度
  • 每個成就有 check(sessionStats) 函式，接收當前 session 統計數據判定是否解鎖
  • 成就狀態持久化到 localStorage（key: tetris_achievements），跨遊戲保留
  • 使用 session-scoped 追蹤（sessionStats），每次 startGame() 重置

2. 成就觸發時機

checkAchievements() 在三個關鍵時機呼叫：

  1. 消行後（checkLines）：追蹤消行數、Tetris 次數、分數、Combo
  2. 升級後（checkLines 中 level up 段落）：追蹤等級
  3. 硬降後（hardDrop）：分數可能在硬降時跨越閾值

3. Toast 通知佇列

  • 使用 achToastQueue 陣列管理多個成就同時解鎖的情況
  • 每個 Toast 顯示 2.5 秒後消失，400ms 間隔後顯示下一個
  • CSS transition 實現平滑的滑入/滑出動畫
  • 不干擾遊戲操作（pointer-events: none）

4. 統計資料追蹤

  • tetris_stats localStorage key 儲存跨遊戲累計數據
  • 8 個統計指標：總遊戲次數、總分數、總消行、Tetris 次數、最高分、最高等級、最佳消行、最大 Combo
  • endSessionStats() 在遊戲結束時一次性寫入

5. UI 整合

  • 開始畫面新增「統計」按鈕，進入統計 & 成就總覽
  • 遊戲結束畫面新增：消行數、最大連擊、本局解鎖的成就列表
  • 成就卡片：已解鎖顯示彩色 icon + 金色邊框，未解鎖灰色鎖頭
  • 成就解鎖有專屬 8-bit 音效（升序五音符 C5-E6 + 延音）

設計決策

  1. Session-scoped 成就判定：成就在單局遊戲中判定（如「一次消4行」），而非跨遊戲累計。這樣每局遊戲都有機會觸發成就，保持激勵感。
  2. 一次解鎖永久保留：成就只需解鎖一次，符合傳統遊戲成就系統設計。
  3. 12 個成就的難度階梯：從「首次消行」（新手秒解鎖）到「消行王者 200 行」（需要高水準長時間遊玩），覆蓋全技能範圍。
