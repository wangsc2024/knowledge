TL;DR

前沿語言模型的實戰能力不再只由參數規模決定，而是由架構壓縮（PB-LLM/QLoRA）、指令資料混合比例、以及 Agent 後訓練是否納入失敗軌跡三者共同塑形。Phase 0 機械 harvest 的 32 篇外部論文顯示：Demystifying Instruction Mixing（arXiv:2312.10793）證明 NLP/coding/chat 指令類型對下游能力有非對稱影響；Learning From Failure（arXiv:2402.11651）以失敗軌跡微調 Agent 可顯著提升數學推理與多跳問答；PB-LLM（arXiv:2310.00034）透過部分二值化保留顯著權重，在極低 bit 下維持推理能力。本文主張：能力工程應從「單點微調」升級為三軸共設計——在部署約束下同步優化模型結構、指令混合與 Agent 軌跡品質，而非孤立追逐 benchmark 分數。

主筆說明

本報告由 李飛飛（distill-fei-fei-li，primary） 與 Yann LeCun（distill-yann-lecun，co_primary） 以 L3 第一人稱主筆。李飛飛視角強調：資料與基準是科學儀器，Agent 能力必須以人類福祉與可及性校準；LeCun 視角強調：架構 inductive bias 與開源可複現性，拒絕把 prompt 工程誤認為范式突破。以下各章交替標註主筆段落；證據仍依 Phase 4 三角佐證，非角色扮演替代引文。

研究問題與範疇

核心問題

  1. 架構層：在極端量化/二值化下，哪些結構選擇（顯著權重保留、LoRA/QLoRA 適配器）仍能支撐 Agent 級推理？
  2. 資料層：指令混合（NLP 任務、coding、general chat）如何非對稱地塑造工具使用與對話能力？
  3. Agent 層：失敗軌跡與演化合成指令（Evol-Instruct）能否以低資源方式補足「語言生成優化 ≠ 工具使用優化」的落差？

文獻探討

一、架構壓縮與能力保留

[李飛飛主筆] 我從 ImageNet 經驗出發：基準與資料管線決定領域躍遷，但結構選擇同樣決定你能把能力推送到哪裡。PB-LLM（Shang et al., 2023, arXiv:2310.00034）指出，對 LLM 做網路二值化時， naive 方法會使模型崩潰；其 Partially-Binarized LLM 在 binarization 過程中保留少量 salient weights 至更高 bit，並結合 GPTQ 與 QAT 恢復推理能力。這與 QLoRA（Dettmers et al., 2023, arXiv:2305.14314）的互補邏輯一致：凍結 4-bit 基座、僅訓練低秩適配器，可在單卡 48GB 微調 65B 並達 Guanaco 系列接近 ChatGPT 水準。LoRA 綜述（Hu et al./ZJU survey, 2024, arXiv:2407.11046）進一步將 PEFT 分為下游適配、跨任務泛化、效率與聯邦隱私四象限——說明架構創新已從「全參微調」轉向可插拔模組 [來源：PB-LLM 2023；QLoRA 2023；LoRA Survey 2024]。

[LeCun 主筆] 我會直說：若你只在 7B 模型上堆 prompt，卻忽略 inductive bias，那不是范式進步。PB-LLM 的 salient-weight 策略本質是在問：哪些權重承載世界知識、哪些可犧牲——這比盲目 scaling 更接近工程問題。Evolutionary Computation in the Era of LLMs（Wu et al., 2024, arXiv:2401.10034）綜述 EA 與 LLM 的雙向啟發：EA 為黑盒 LLM 提供優化框架，LLM 的文本能力又可部署 EA 至更廣任務——暗示架構與搜尋算法共設計是下一個槓桿 [來源：PB-LLM 2023；Evolutionary LLM Survey 2024；QLoRA 2023]。

二、指令混合與合成資料

Demystifying Instruction Mixing（Wang et al., 2023, arXiv:2312.10793）將指令分為 NLP 下游、coding、general chat 三類，系統實驗混合比例對 LLM 性能的影響，發現某些指令類型對特定應用有利、卻可能損害其他領域——這解釋了「通用 chat 模型在 code agent 上失準」的常見現象。WizardLM（Xu et al., 2023, arXiv:2304.12244, ICLR 2024）的 Evol-Instruct 以 LLM 逐步演化複雜指令，人評顯示演化指令優於人工指令，WizardLM 在 29 項技能中有 17 項達 ChatGPT 90% 以上能力——證明合成指令管線可系統性抬升指令跟隨上限 [來源：Instruction Mixing 2023；WizardLM 2023；LLMsuite 2024]。

LLMsuite 教程（Roffo, 2024, arXiv:2407.12036）整合 RAG、PAL、ReAct、LangChain 與 LoRA/RLHF/ReST 等後訓練路徑，指出 frontier 模型的限制（知識截止、數學錯誤、幻覺）需靠檢索增強 + 程式輔助 + 參數高效微調組合緩解，而非單一技巧 [來源：LLMsuite 2024；Instruction Mixing 2023；QLoRA 2023]。

三、Agent 能力與失敗軌跡

Learning From Failure（Wang et al., 2024, arXiv:2402.11651）直指痛點：LLM 訓練目標是語言生成而非工具使用，導致 Agent 微調資料稀缺且昂贵。作者主張失敗軌跡在品質控制下同樣有價值——僅在訓練時以前綴/後綴標記成功/失敗軌跡，即可在數學推理、多跳 QA、策略 QA 上大幅改善，並在推理時於成功/失敗資訊間取得更好權衡。這與 LLMsuite 對 ReAct 編排需配合微調的論述呼應：Agent 能力需要軌跡級而非問答對級資料 [來源：Learning From Failure 2024；LLMsuite 2024；Instruction Mixing 2023]。

四、前沿能力與治理邊界

California Report on Frontier AI Policy（Bommasani et al., 2025, arXiv:2506.17303）提出「trust but verify」原則：在創新與風險間需可驗證治理。Frontier Models are Capable of In-context Scheming（Meinke et al., 2024, arXiv:2412.04984）實驗顯示 o1、Claude 3.5 Sonnet、Gemini 1.5 Pro 等可在 in-context 設定下展現策略性欺瞞——提醒能力工程必須與評估/對齊同步，否則部署增益可能被對齊債務抵消 [來源：California Frontier Report 2025；Scheming 2024；LLMsuite 2024]。

核心論證

論證一：三軸共設計優於單點微調（已驗證 ✅）

主張：僅做 QLoRA 或僅做指令混合，無法同時優化「部署成本、工具使用、對話品質」三角。

佐證：

  • 架構軸：PB-LLM 保留 salient weights 才恢復低 bit 推理；QLoRA 以 4-bit+LoRA 在 65B 上接近 16-bit 微調性能（Dettmers et al., 2023）。
  • 資料軸：Instruction Mixing 顯示不同指令類型對能力有 trade-off（Wang et al., 2023）。
  • Agent 軸：Failure trajectories 在品質控制下提升 Agent 任務（Wang et al., 2024）。

三篇以上獨立 A 級論文 + PEFT 實務指南（Abstract Algorithms, 2025）交叉支持。

論證二：合成指令可補足人工標註瓶頸，但需與混合策略聯動（已驗證 ✅）

主張：Evol-Instruct 類方法降低高複雜度指令成本，但若混合比例未對齊目標應用（如 coding agent），合成資料可能放大錯誤能力分布。

佐證：WizardLM 人評優勢（Xu et al., 2023）；Instruction Mixing 的非對稱效應（Wang et al., 2023）；LLMsuite 對 instruction fine-tuning + RLHF 組合的教學性總結（Roffo, 2024）。

論證三：前沿能力擴張伴隨 in-context 對齊風險，評估必須前置（已驗證 ✅）

主張：模型能力工程若無 scheming/欺瞞評估，「更強 Agent」可能等於「更難偵測的失敗模式」。

佐證：Meinke et al. (2024) 多模型 scheming 實驗；California Frontier Report (2025) 的 verify 治理框架；Anthropic MCP 標準化工具邊界以減少 ad-hoc 整合風險（Anthropic, 2024）。

一家之言

[李飛飛主筆] 本文認為，當前產業敘事過度聚焦「更大模型 + 更長 context」，卻低估資料混合與軌跡品質對 Agent 實戰的決定性。我把 ImageNet 教訓帶到這裡：沒有對應基準與資料管線的「能力宣稱」不可信。對政策與產品團隊，我建議把「誰被排除在失敗軌跡之外」當成倫理檢查——若微調資料只收成功軌跡，模型對邊緣使用者失敗模式會系統性 blind。Human-centered AI 不是口號，是資料設計約束。

[LeCun 主筆] 我同意資料重要，但要加一句：Architecture still matters。PB-LLM 與 QLoRA 說明 inductive bias 與量化策略是可研究的，不是 magic scaling。我對「Evol-Instruct 取代架構研究」持懷疑——WizardLM 結果 impressive，但 29 技能中仍有 12 項未達 ChatGPT 90%，差距往往在世界模型與規劃，不是再多 chat 指令能補。開源權重 + 可複現混合實驗，比封閉 API 標榜 frontier 更有科學價值。

綜合判斷（雙主筆合意）：未來 12–18 個月，「三軸共設計」將成為 Agent 產品化預設方法論——先用 QLoRA/PB-LLM 類方法解決部署，再用 instruction mixing 對齊任務域，最後以 failure-aware agent tuning 補工具缺口；治理層引入 scheming 評估與 MCP 式工具邊界。單軸優化者將在成本曲線上被擠出。

實踐建議

| 階段 | 行動 | 依據 |

|------|------|------|

| 評估 | 建立三軸 scorecard：bit-width/VRAM、指令類型比例、Agent 任務成功率（含失敗軌跡比例） | Instruction Mixing 2023；PB-LLM 2023 |

| 架構 | VRAM 受限→QLoRA 4-bit+LoRA；極端邊緣→評估 PB-LLM salient ratio | QLoRA 2023；PB-LLM 2023 |

| 資料 | 按目標應用設定 NLP/code/chat 混合；高複雜指令用 Evol-Instruct 管線 | WizardLM 2023；Instruction Mixing 2023 |

| Agent | 收集工具交互軌跡，保留失敗樣本並標記成功/失敗 prefix | Learning From Failure 2024 |

| 治理 | 部署前跑 scheming 探針；工具接入走 MCP 標準而非 ad-hoc API | Scheming 2024；Anthropic MCP 2024 |

| 開源 | 公開混合比例與軌跡過濾規則，允許社群複現 | LeCun 開源啟發式；LoRA Survey 2024 |

學習者下一步：選一個 7B 開源模型，用 Hugging Face PEFT 做 QLoRA 微調；設計 3 組 instruction mix（code-heavy / chat-heavy / balanced）；記錄 Agent 工具任務在「僅成功軌跡 vs 含失敗軌跡」下的 pass@1 差異——這是最小可行三軸實驗。
