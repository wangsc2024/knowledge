Speculative Decoding（推測解碼）— 2026 年 LLM 推理加速核心技術

概述

Speculative Decoding（推測解碼）是 2026 年最重要的 LLM 推理優化技術之一。由 Google Research 於 2022 年提出，借鑑 CPU 推測執行（Speculative Execution）的概念，透過「小模型草擬、大模型驗證」的雙模型協作模式，在不損失輸出品質的前提下實現 2-3 倍推理加速。截至 2026 年 3 月，該技術已從學術研究進入生產標準，被 vLLM、SGLang、TensorRT-LLM 等主流服務框架原生支援，Google Search AI Overviews 等大規模產品已在生產環境中部署。

---

核心機制：Draft-Verify 模式

基本原理

LLM 推理的根本瓶頸在於自迴歸生成的序列性：每個 token 必須等待前一個 token 完成才能生成，但現代 GPU 的計算能力遠超記憶體頻寬限制（memory-bound），導致大量算力閒置。Speculative Decoding 的核心洞察是：

  1. Token 難度不均勻：部分 token（如常見詞、重複模式）容易預測，部分需要複雜推理
  2. 小模型可近似大模型：對於簡單 token，小模型（Draft Model）與大模型（Target Model）的預測分佈高度一致
  3. 驗證比生成更快：大模型可以在單次前向傳播中同時驗證多個 token

執行流程（五步迴圈）


1. Draft Phase：小模型（如 Llama-3.2-1B）快速生成 γ 個候選 token
2. Verify Phase：大模型（如 Llama-3.2-3B）在單次前向傳播中驗證所有候選 token
3. Accept：大模型接受最長的正確前綴（h 個 token）
4. Correct：大模型自行生成第 (h+1) 個 token
5. Repeat：從新位置重複步驟 1

關鍵參數

| 參數 | 符號 | 說明 |

|------|------|------|

| 接受率 | α (alpha) | 大模型接受草擬 token 的機率，α ≥ 0.6 時效果顯著 |

| 草擬長度 | γ (gamma) | 每步草擬的 token 數量，γ = 4-5 為常見設定 |

| 平均接受長度 | τ (tau) | 每輪平均接受的 token 數：τ = (1 - α^(γ+1)) / (1 - α) |

數學保證

透過 Speculative Sampling 演算法，輸出分佈與標準自迴歸解碼完全一致（identical output distribution），不存在品質損失。這是區別於其他近似加速方法（如量化、剪枝）的核心優勢。

2026 年前沿變體

EAGLE-3（2026 最新）

  • 加速比：6.5× over baseline，1.4× over EAGLE-2
  • 創新：改進的特徵級推測，而非 token 級，減少草擬模型與目標模型的分佈差異
  • 適用場景：通用文本生成、程式碼補全

PEARL Framework（ICLR 2025）

  • 加速比：2.5× latency reduction（A100 80G，bf16）
  • 創新：自適應草擬長度動態調整（Adaptive Draft Length），根據即時接受率自動優化 γ 值
  • 驗證環境：Llama 2 70B + Qwen 2.5 7B/72B

Speculative Speculative Decoding / SSD（ICLR 2026）

  • 核心突破：將推測與驗證操作完全並行化
  • 機制：在驗證進行的同時，草擬模型預測可能的驗證結果並預先準備下一輪推測
  • 效果：若實際驗證結果在預測集合中，可立即返回推測結果，完全消除草擬開銷

DFlash

  • 加速比：4.8× throughput（LLaMA3.1-8B，NVIDIA Blackwell 6000 Pro）
  • 特色：硬體感知的快閃注意力融合推測解碼

Ouroboros

  • 加速比：2.4× over traditional speculative decoding，3.9× over vanilla
  • 特色：自回饋式草擬，草擬模型利用目標模型的部分隱藏狀態

Mirror-SD

  • 加速比：最高 5.8×（14B-66B 模型）
  • 適用：語音 LLM，使用聲學相似性群組（Acoustic Similarity Groups）進行粗粒度驗證

Saguaro

  • 加速比：2× over existing baselines，5× over autoregressive
  • 創新：多 token 串流（Multi-token Streaming），草擬模型每步可發出多個 token

Intel-Weizmann 異質協作

  • 加速比：2.8× faster inference
  • 突破：解耦詞彙表對齊限制，任意小模型可加速任意大模型

Google 生產案例

Google Search 的 AI Overviews 功能使用 Speculative Decoding 加速推理，在維持回答品質的同時顯著降低延遲。這是該技術在全球最大規模搜尋引擎中的實際部署。

部署考量

  1. Draft Model 選擇：同家族小模型效果最佳（如 Llama-3.2-1B → 3B），接受率更高
  2. γ 值調優：需根據 GPU 配置和模型組合實測，Multi-GPU（TP=2）需更謹慎調整
  3. 記憶體開銷：需同時載入兩個模型，但小模型佔用極低
  4. 批次處理：高併發場景下需平衡推測開銷與吞吐量

與相關技術的關係

| 技術 | 目標 | 品質影響 | 與 Speculative Decoding 關係 |

|------|------|---------|-----------------------------|

| 量化（Quantization） | 減少記憶體 | 輕微損失 | 可組合使用 |

| 知識蒸餾（Distillation） | 小模型學大模型 | 有損失 | 可用於訓練更好的 Draft Model |

| KV Cache 優化 | 減少重複計算 | 無損失 | 互補：SGLang 的 RadixAttention |

| Test-Time Compute | 花更多計算提升品質 | 提升品質 | 方向相反但可互補 |

| Diffusion LLMs | 平行生成範式 | 近似無損 | 替代方案，非序列生成 |

未來發展方向

  1. 多模態擴展：從文本擴展到影像和語音生成的推測解碼
  2. 自推測（Self-Speculative）：單一模型同時充當草擬和驗證角色，消除額外模型開銷
  3. 分散式推測：跨多節點的推測與驗證分離
  4. 硬體協同設計：NVIDIA Blackwell 系列針對推測解碼的專用硬體加速
