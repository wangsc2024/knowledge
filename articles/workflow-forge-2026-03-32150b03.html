產物設計規格

1. error-object-schema.json（JSON Schema）

必填欄位（5 個）：

  • type（string, uri）：錯誤類型 URI，如 config/error-types.yaml#timeout
  • title（string, ≤100 chars）：錯誤標題（簡短描述）
  • detail（string, ≤500 chars）：詳細錯誤訊息（含 context）
  • retryable（boolean）：是否可重試（true = 暫時性錯誤，false = 永久性錯誤）
  • instance（string, uri）：錯誤發生的具體執行實例（如 logs/structured/20260330_050332.jsonl#line156）

選填欄位（5 個）：

  • trace_id：追蹤 ID（對應 PowerShell $DIGEST_TRACE_ID）
  • task_key：任務鍵（對應 frequency-limits.yaml）
  • phase：失敗的執行階段（team mode）
  • timestamp：錯誤發生時間（ISO 8601）
  • caused_by：根因錯誤（遞迴結構，用於錯誤鏈）

參考標準：RFC 7807 Problem Details for HTTP APIs

範例：


{
  "type": "config/error-types.yaml#timeout",
  "title": "AI 深度研究任務超時",
  "detail": "任務 ai_deep_research 在 Phase 2 執行超過 1440 秒，被 PowerShell 強制終止",
  "retryable": true,
  "instance": "logs/structured/20260330_050332.jsonl#line156",
  "trace_id": "abc123-def456",
  "task_key": "ai_deep_research",
  "phase": "phase2",
  "timestamp": "2026-03-30T05:03:32+08:00"
}

2. error-types.yaml（錯誤類型定義）

7 大分類：

  • timeout：執行逾時
  • api_failure：API 呼叫失敗
  • validation：格式驗證失敗
  • dependency：依賴缺失
  • resource：資源不足
  • config：配置錯誤
  • external：外部服務不可用

17 種錯誤類型（摘錄）：

  • timeout：執行逾時（retryable: true, priority: 3）
  • api_failure：API 呼叫失敗（retryable: true, priority: 4）
  • validation_error：格式驗證失敗（retryable: false, priority: 3）
  • skill_not_found：Skill 不存在（retryable: false, priority: 4）
  • context_overflow：Context Window 溢出（retryable: false, priority: 4）
  • config_semantic_error：配置語義錯誤（retryable: false, priority: 4）

關鍵決策

  1. 採用 RFC 7807 Problem Details：業界標準，與 HTTP API 錯誤格式對齊
  2. error.type 使用 YAML URI：config/error-types.yaml#timeout，避免硬編碼，集中管理
  3. 必填 5 欄位：type, title, detail, retryable, instance — 平衡簡潔與診斷力
  4. 支援錯誤鏈：caused_by 遞迴結構，追蹤根因錯誤
  5. 不修改 PowerShell 核心邏輯：僅新增 Schema 與定義，不改執行腳本
