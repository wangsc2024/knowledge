Test-Time Compute Scaling（測試時計算擴展）

從預訓練擴展到推理時擴展的 LLM 新範式、核心方法論與 2025-2026 年最新進展

---

一、技術概述

Test-Time Compute Scaling（測試時計算擴展，又稱 Inference-Time Scaling 或推理時擴展）是 2024-2026 年大型語言模型（LLM）領域最重大的範式轉移之一。傳統的 Scaling Laws（Chinchilla Laws, Kaplan Laws）聚焦於預訓練階段的計算擴展——透過增加模型參數量、訓練資料量與訓練算力來提升效能。而 Test-Time Compute Scaling 則揭示了一個互補的新維度：在推理階段投入更多計算資源，讓模型「思考更久」，從而在不改變模型權重的情況下顯著提升輸出品質。

這一範式的核心洞察是：模型不需要在預訓練時就「知道所有答案」，而是可以在遇到困難問題時，透過額外的推理計算來「推導出答案」。這類似於人類面對簡單問題可以快速回答（System 1），但面對複雜問題需要深思熟慮（System 2）的雙系統認知模型。

代表性系統：OpenAI o1/o3/o4-mini（2024-2025）、DeepSeek-R1（2025-01）、Claude Opus 4.6 Extended Thinking（2025）、Google Gemini 2.5 Pro（2025）、Qwen QwQ（2025）。

---

二、核心原理與創新點

2.1 兩大擴展機制

根據 Snell et al.（2024, arXiv:2408.03314）的開創性研究，Test-Time Compute 的擴展主要透過兩個機制：

機制一：搜尋式擴展（Search-Based Scaling）

  • 生成多個候選回應，使用 Process Reward Model（PRM）逐步驗證
  • 常見實現：Best-of-N 採樣、Beam Search、Monte Carlo Tree Search（MCTS）
  • PRM 相比 Outcome Reward Model（ORM）更有效，因為能提供逐步的密集獎勵信號
  • 關鍵發現：搜尋效能與問題難度高度相關——中等難度問題收益最大

機制二：自適應分佈修正（Adaptive Distribution Refinement）

  • 在測試時根據具體 prompt 動態調整模型的輸出分佈
  • 讓模型根據問題特性「思考」不同時間長度
  • 實現方式：extended thinking tokens、chain-of-thought 延長、反思與回溯

2.2 Compute-Optimal 策略

Snell et al. 的核心貢獻是提出 Compute-Optimal Test-Time Scaling 策略：

  • 不同難度的問題需要不同量級的推理計算
  • 對簡單問題過度推理反而浪費算力甚至降低品質
  • 最優策略是根據問題難度自適應分配推理計算
  • 這一策略相比 Best-of-N 基線提升 4 倍以上效率
  • 關鍵結論：在 FLOPs 匹配的條件下，較小模型配合最優測試時計算，可以超越 14 倍大的模型

2.3 Budget Forcing（預算強制）

s1 模型（2025, arXiv:2501.19393）提出了極簡的測試時擴展方法：

  • 僅用 1,000 條精選推理軌跡（s1K 資料集）微調 Qwen2.5-32B
  • Budget Forcing 機制：強制終止或延長模型的思考過程
  • 當模型試圖結束推理時，追加「Wait」token 迫使其繼續思考、二次檢查
  • 結果：s1-32B 在數學競賽題上超越 o1-preview 高達 27%
  • 從 50% 外推到 57%（AIME24），展示了超越基礎效能的擴展能力

2.4 強化學習驅動的推理湧現

DeepSeek-R1（2025-01, arXiv:2501.12948）證明了另一條路徑：

  • 透過純強化學習（RL）激勵推理能力，無需人類標注的推理軌跡
  • RL 框架促進了進階推理模式的自發湧現：自我反思、驗證、動態策略適應
  • 在數學、程式設計競賽、STEM 領域超越傳統監督學習
  • 大規模模型的湧現推理模式可以系統性地蒸餾到小模型

三、核心方法分類

3.1 串行思考（Sequential Thinking）

| 方法 | 描述 | 代表 |

|------|------|------|

| Chain-of-Thought (CoT) | 逐步推理展開 | GPT-4, Claude |

| Extended Thinking | 內部長鏈推理 token | OpenAI o1, Claude Opus 4.6 |

| Budget Forcing | 強制延長/截斷思考 | s1-32B |

| Pause Tokens | 插入特殊 token 增加計算深度 | Goyal et al. (2023) |

3.2 並行搜尋（Parallel Search）

| 方法 | 描述 | 代表 |

|------|------|------|

| Best-of-N | 生成 N 個候選選最優 | 通用方法 |

| Repeated Sampling | 大量重複採樣 | Large Language Monkeys |

| Beam Search + PRM | 束搜索 + 過程獎勵 | Snell et al. |

| MCTS | 蒙特卡洛樹搜索 | AlphaProof, rStar |

3.3 反思與修正（Reflection & Refinement）

| 方法 | 描述 | 代表 |

|------|------|------|

| Self-Refinement | 模型自我修正輸出 | Madaan et al. |

| Verification | 生成後驗證正確性 | DeepSeek-R1 |

| Backtracking | 發現錯誤後回溯 | 推理模型通用 |

| Atom of Thoughts | 馬可夫化推理鏈 | AoT (2025) |

四、最新進展（2025-2026）

4.1 Atom of Thoughts（AoT, 2025-02, arXiv:2502.12018）

  • 利用馬可夫過程的無記憶性，將推理分解為自包含的原子單元
  • 最小化對歷史上下文的依賴，降低冗餘計算
  • 可與樹搜索、反思精煉等方法無縫整合
  • 隨計算預算增加，效能持續超越既有基線

4.2 「1B 能否超越 405B？」（2025-02, arXiv:2502.06703）

  • 系統分析策略模型、PRM 與問題難度對 TTS 的影響
  • 提出重新思考 Compute-Optimal 測試時擴展策略
  • 探索極小模型配合大量推理計算的可能性邊界

4.3 推理行為的四個習慣（2025-03, arXiv:2503.01307）

  • 識別出有效自我改進的四個認知行為：驗證、回溯、子目標設定、反向推導
  • Qwen 自然展現這些行為而 Llama 缺乏，解釋了 RL 訓練效果差異
  • 關鍵發現：推理行為的存在比答案的正確性更重要
  • 用錯誤答案但正確推理模式訓練，效果等同正確答案訓練

4.4 推理對抗魯棒性（2025-01, arXiv:2501.18841）

  • 測試時計算增加可提升對抗性攻擊的魯棒性
  • 多數攻擊的成功率隨推理計算增長趨近零
  • 無需對抗訓練即可獲得魯棒性提升

五、與既有技術的比較

| 維度 | 預訓練擴展（Train-Time） | 測試時擴展（Test-Time） |

|------|------------------------|------------------------|

| 計算時機 | 訓練階段（一次性） | 推理階段（每次查詢） |

| 成本模式 | 固定成本，推理邊際成本低 | 固定成本低，推理成本按需擴展 |

| 適用場景 | 通用知識與能力 | 特定困難問題深度推理 |

| 擴展上限 | 受限於資料與算力 | 受限於推理時算力預算 |

| 效能提升 | 對數線性（power law） | 可超越 14x 大模型（特定難度） |

| 靈活性 | 模型發布後固定 | 可根據問題動態調整 |

| 代表定律 | Chinchilla Scaling Laws | Test-Time Compute Scaling Laws |

與 RAG 的比較

  • RAG 透過外部知識檢索增強，解決的是知識缺口
  • Test-Time Scaling 透過額外計算增強，解決的是推理能力缺口
  • 兩者互補：RAG 提供事實基礎，TTS 提供深度推理

與 Fine-Tuning 的比較

  • Fine-Tuning 修改模型權重，永久改變行為
  • Test-Time Scaling 不修改權重，每次推理動態調整
  • TTS 的優勢：無需訓練資料、即時生效、可動態控制計算預算

六、應用場景與影響

6.1 直接應用

  • 數學推理：競賽級數學問題（AIME、IMO），o3 + TTS 已接近人類頂尖水準
  • 程式碼生成：複雜演算法設計與 debug（SWE-bench 基準）
  • 科學研究：STEM 領域假說驗證、實驗設計
  • 形式化證明：數學定理自動證明（AlphaProof + MCTS）

6.2 產業影響

  • 推理成本經濟學轉變：從「訓練一次、推理便宜」轉向「按需付費的推理計算」
  • 模型尺寸策略：小模型 + 多推理計算可替代大模型，降低部署門檻
  • API 定價模式：OpenAI 的 o1/o3 按思考 token 計費，開創新商業模式
  • 邊緣部署：小模型在本地設備配合充足推理時間，可達到雲端大模型水準

6.3 對 AI 安全的啟示

  • 推理模型的「思考過程」可審計，提升透明度
  • 更多推理計算可提升對抗攻擊的魯棒性
  • 但也帶來新風險：「思考 token」可能包含欺騙性推理（unfaithful CoT）

---

七、開放問題與未來方向

  1. 最優分配問題：如何自動判斷問題難度並分配最優推理計算量？
  2. Scaling Law 統一：預訓練擴展與測試時擴展的統一理論框架尚未建立
  3. 推理效率：長思考鏈的 token 消耗巨大，如何壓縮推理過程？
  4. 多模態擴展：Vision-Language 模型的 TTS 研究尚處早期
  5. 推理忠實性：如何確保模型的「思考過程」真實反映其推理邏輯？
  6. 冷啟動問題：哪些初始認知行為決定了模型是否能從 RL 中自我改進？
