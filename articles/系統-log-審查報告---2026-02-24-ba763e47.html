發現問題

🔴 嚴重問題 1：自動任務 key 格式不一致（已修復）

  • 根因：Phase 1 查詢 Agent（LLM）產出的 task key 格式不穩定
  • YAML 定義：log_audit（底線分隔）
  • LLM 變體：logaudit（無分隔）、gitpush（無分隔）、tech-research（連字號）、ai-deep-research（連字號）、ai-github（縮寫+連字號）
  • 影響：PS 字典查詢失敗 → 任務被標記 skipped → 浪費 API 費用和執行時間
  • 受影響記錄：

- 2026-02-24 14:30: 3/4 任務跳過（tech-research, ai-deep-research, ai-github） - 2026-02-24 11:30: 2/4 任務跳過（logaudit, gitpush） - 2026-02-24 15:30: 可能再次受影響（當前正在執行）

  • 修復方案：run-todoist-agent-team.ps1 加入 key 正規化（連字號→底線 + 別名映射表）

🔴 嚴重問題 2：Phase 1 查詢失敗（3 次）

  • 2026-02-23 16:30: phase 1 failed (640s)

  • 根因：LLM 呼叫超時或 API 不穩定
