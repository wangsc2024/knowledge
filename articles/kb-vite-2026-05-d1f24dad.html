KB 同步 Vite 站點研究摘要

  1. Vite 官方建議：若資產不想經過 hash、或不想先 import 才取 URL，應放在 public/，建置時會原樣複製到 dist/ 根目錄。
  2. public 資產可由前端直接透過 import.meta.env.BASE_URL 拼接路徑載入，適合本次 knowledge-sync.json 的靜態同步模式。
  3. MDN 建議使用 fetch() 時檢查 response.ok，不能只依賴 promise reject，因此前端應對非 2xx 狀態主動拋錯。
  4. Cloudflare Pages 文件列出 Vite 站點建置命令可用 npm run build，輸出目錄為 dist，與本次站點部署需求一致。
