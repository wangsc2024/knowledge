專案優化報告：Hooks 代碼品質與配置一致性修正

日期

2026-03-03

概述

對 daily-digest-prompt 專案進行深度分析與優化，以資訊安全合規為優先，修正 7 項代碼品質與配置一致性問題。全部 556 個測試通過。

修正項目

1. frequency-limits.yaml 註解摘要修正

  • 問題：底部每日上限摘要註解聲稱合計 47 次/日，但多個任務實際 daily_limit 為 0（停用），實際啟用合計僅 17 次/日
  • 修正：將每個群組和合計數字更新為與實際 daily_limit 值一致

2. post_tool_logger.py 冗餘 import sys 移除

  • 問題：第 429 行在 except 區塊內有 import sys，但頂層已匯入
  • 修正：移除冗餘的 import 語句

3. post_tool_logger.py import datetime 衝突修正

  • 問題：import datetime as _dt 在函數內匯入完整模組，與頂層 from datetime import datetime 造成命名混淆
  • 修正：改為 from datetime import timedelta

4. hook-rules.yaml sensitive-env-powershell 規則修正

  • 問題：pattern 中包含 =，導致合法的環境變數賦值也會被攔截
  • 修正：從匹配組移除 =，僅攔截管道/串接操作

5. on_stop_alert.py _rotate_logs() UnboundLocalError 修正

  • 問題：kept 變數在 try 區塊內定義，但 except ImportError 的 fallback 直接引用 kept
  • 修正：新增 if kept 預檢查，並將 UnboundLocalError 改為 NameError

6. frequency-limits.schema.json daily_limit 最小值修正

  • 問題：JSON Schema 要求 daily_limit 最小值為 1，但 daily_limit=0 表示停用是合法語義
  • 修正：minimum 從 1 改為 0

7. dedup-policy.schema.json 新增缺失屬性

  • 問題：dedup-policy.yaml 新增了 strategy_search 和 series_tracking，但 schema 設了 additionalProperties: false
  • 修正：在 schema 中新增這兩個屬性的完整定義

測試結果

全部 556 個測試通過（hooks 529 + skills 27）
