Sub-500ms Voice Agent 全端實作

摘要

開發者從零打造延遲低於 500ms 的語音 Agent，使用 Groq STT（語音轉文字）→ LLM 推理 → TTS（文字轉語音）管線。在 Hacker News 獲得 397 分、120 則討論。

技術亮點

  • 端到端延遲 < 500ms：關鍵在於使用 Groq 的超快推理速度
  • 完整管線：語音輸入 → STT 轉錄 → LLM 生成回應 → TTS 語音輸出
  • 從零建構：不依賴第三方語音 Agent 框架
  • 實時對話體驗：接近人類自然對話的回應速度

與本專案關聯

本專案已整合 Groq Relay（bot/groq-relay.js），此案例展示 Groq 在語音場景的延遲優勢，可作為未來語音互動功能的參考架構。
