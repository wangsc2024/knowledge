RAG-Anything — All-in-One 多模態 RAG 框架深度研究

專案概述

RAG-Anything 是香港大學數據科學實驗室（HKUDS）開發的開源多模態 RAG（Retrieval-Augmented Generation）框架，建構於 LightRAG 之上，旨在解決傳統 RAG 系統僅支援純文字的根本限制。專案於 2025 年 6 月創建，截至 2026 年 3 月已累積 14,000+ Stars、1,671 Forks，採 MIT 授權。

核心價值主張：現代文件包含交錯的文字、圖片、表格、數學公式與圖表，RAG-Anything 提供單一統一框架處理所有模態，無需拼裝多個專用工具。

技術報告：arXiv:2510.12323

技術架構：五階段多模態處理管線

第一階段：文件解析（Document Parsing）

  • 核心引擎：MinerU（高保真度文件解析器）
  • 支援格式：PDF、Office 文件（.docx/.pptx/.xlsx，需 LibreOffice）、圖片
  • 自適應內容分解：保留原始版面結構與元素層級關係
  • 解析品質是整個管線的基石 — 改善 OCR/表格/公式辨識的收益往往大於更換 LLM

第二階段：內容分析與路由（Content Analysis & Routing）

  • 自動分類識別內容類型（文字/圖片/表格/公式/圖表）
  • 並行多管線架構：文字管線與多模態管線同時處理
  • 智慧路由決定每段內容的最佳分析策略

第三階段：專用分析器（Specialized Analyzers）

| 分析器 | 功能 | 技術特點 |

|--------|------|----------|

| Visual Content Analyzer | 圖片描述、空間關係提取 | 整合 VLM（視覺語言模型）|

| Structured Data Interpreter | 表格分析、統計模式識別 | 行列語義、數值趨勢 |

| Mathematical Expression Parser | LaTeX 公式解析 | 數學符號語義化 |

| Extensible Modality Handler | 自訂內容類型 | 插件式擴展架構 |

第四階段：多模態知識圖譜建構（Multimodal Knowledge Graph）

這是 RAG-Anything 最核心的創新 — 雙圖融合架構（Dual-Graph Construction）：

  1. 跨模態關係圖：提取跨越不同模態的實體與關係（如：圖片中的數據與表格中的數字的對應關係）
  2. 文字語義圖：傳統的文字實體與關係提取

兩張圖在統一表徵空間中融合，建立「證據鏈」（Evidence Chains）— 例如將圖片中的趨勢線連結到表格中的具體數值再連結到文字中的結論。

第五階段：模態感知檢索（Modality-Aware Retrieval）

  • 向量-圖譜混合檢索：結合向量相似度搜尋（語義模糊匹配）與知識圖譜遍歷（結構化精確推理）
  • 自適應排序機制：根據查詢類型動態調整向量 vs 圖譜的權重
  • VLM-Enhanced Query Mode：查詢涉及視覺內容時，自動將相關圖片傳入 VLM 進行聯合分析

功能特色與差異化

與傳統 RAG 的根本區別

| 面向 | 傳統 RAG | RAG-Anything |

|------|----------|---------------|

| 內容支援 | 純文字 | 文字+圖片+表格+公式+圖表 |

| 知識表徵 | 向量嵌入 | 向量 + 知識圖譜雙重表徵 |

| 檢索策略 | 向量相似度 | 向量-圖譜混合 |

| 跨模態理解 | 無 | 自動建立跨模態證據鏈 |

| 長文件表現 | 顯著下降 | 圖譜結構維持連貫性 |

與同類框架比較

  • vs LightRAG（父專案）：LightRAG 專注文字知識圖譜 RAG，RAG-Anything 擴展至多模態
  • vs Microsoft GraphRAG：GraphRAG 使用社群摘要策略，RAG-Anything 使用跨模態實體關係
  • vs RAGFlow：RAGFlow 以 DeepDoc 為解析核心，RAG-Anything 以 MinerU 為解析核心並加入知識圖譜層
  • vs NaiveRAG：純向量分塊檢索，在長文件和多模態場景遠不及 RAG-Anything

特殊功能

  1. Direct Content List Insertion：繞過文件解析，直接注入預處理的結構化內容
  2. VLM-Enhanced Query：查詢時動態整合視覺分析
  3. Context-Aware Processing Module：智慧上下文整合
  4. 可擴展模態處理器：支援自訂新的內容類型

快速上手

安裝


# PyPI 安裝（推薦）
pip install raganything
pip install 'raganything[all]'  # 含所有可選功能

# 從原始碼安裝（使用 uv）
git clone https://github.com/HKUDS/RAG-Anything.git
cd RAG-Anything
uv sync
uv run python examples/raganything_example.py

基本使用


from raganything import RAGAnything, RAGAnythingConfig

config = RAGAnythingConfig(
    working_dir="./rag_storage",
    parser="mineru",
    enable_image_processing=True,
    enable_table_processing=True,
    enable_equation_processing=True
)

rag = RAGAnything(config)

# 處理文件
await rag.ainsert("research_paper.pdf")

# 多模態查詢
result = await rag.aquery(
    "圖表中的趨勢與表格數據是否一致？",
    mode="hybrid"  # 向量 + 圖譜混合檢索
)

系統需求

  • Python 3.10+
  • LibreOffice（Office 文件處理）
  • VLM API（如 OpenAI GPT-4V 或本地 VLM）

潛在應用場景

直接應用於 daily-digest-prompt 專案

  1. 知識庫多模態化：目前 localhost:3000 知識庫僅支援純文字（contentText + Tiptap JSON），可借鑑 RAG-Anything 的多模態索引概念，未來支援圖表混排筆記的語義搜尋
  2. 研究報告增強：自動任務產出的研究報告若包含圖表引用，可用 RAG-Anything 建立跨模態知識圖譜
  3. Podcast/Video 素材管理：article-to-podcast 和 article-to-video 管線可整合多模態 RAG 做素材檢索

其他應用

  • 學術論文分析：含大量公式和圖表的論文，跨模態問答
  • 財務報告分析：表格數據與文字結論的交叉驗證
  • 技術文件知識庫：工程圖紙 + 規格書的統一檢索
  • 專利分析：圖示 + 權利要求 + 技術說明的關聯搜尋

優缺點評估

優點

  1. 真正的多模態統一：不是把圖片轉文字再做 RAG，而是建立原生的多模態知識圖譜
  2. 雙圖融合架構創新：向量語義 + 圖譜結構的互補檢索，在長文件場景優勢明顯
  3. 基於 LightRAG 的成熟基礎：站在巨人肩上，文字 RAG 品質有保障
  4. MIT 授權：商業友善
  5. uv 支援：現代 Python 工具鏈，與本專案技術棧一致
  6. 模態可擴展：插件架構支援新增自訂內容類型

缺點

  1. 掃描品質依賴：手寫文件或低品質掃描的表現會顯著下降
  2. 延遲較高：VLM/LLM 依賴導致即時系統面臨延遲挑戰
  3. 圖譜驗證需求：法律/合規場景中，自動提取的關係需人工驗證
  4. 資源消耗：多模態處理管線的計算和記憶體需求較純文字 RAG 高出數倍
  5. 生態成熟度：相比 LangChain/LlamaIndex 等老牌框架，社群生態尚在建設中
