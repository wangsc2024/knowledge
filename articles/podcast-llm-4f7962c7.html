曉晨（解說）：大家好，今天我們要聊的是 LLM 的觀測性與 trace，特別聚焦在最近的幾個開源平台。

云哲（提問）：觀測性聽起來很高深，能簡單說明一下什麼是 LLM 觀測性嗎？

曉晨（解說）：簡單來說，就是在 LLM 執行過程中，記錄關鍵資料點，如 prompt、回覆、計算資源使用，以及模型內部的推理路徑，讓開發者可以追蹤、除錯與優化整體系統。

云哲（提問）：那目前有哪些常見的觀測平台可以使用呢？

曉晨（解說）：主要有三個：Langfuse、Arize Phoenix，以及 OpenLLMetry。它們都支援 OpenTelemetry 標準，提供 trace、評估與資料視覺化功能。

云哲（提問）：能多說說 Langfuse 的特點嗎？

曉晨（解說）：Langfuse 是開源的 LLM 工程平台，最近釋出 v4 Python SDK，全面 OTel‑native，支援 tracing、LLM‑as‑judge 評估、prompt 管理與資料集管理，已被逾兩千家公司使用，每月處理上千億筆觀測。

云哲（提問）：Arize Phoenix 又有什麼優勢呢？

曉晨（解說）：Phoenix 為 AI Observability 與 Evaluation 平台，採用 OTLP‑native 設計，支援 LLM‑as‑judge 與人工標註工作流，能為 agents、tools 與 guardrails 各自產生獨立的 OTel span，與 GenAI Agentic Conventions 完美對齊。

云哲（提問）：那 OpenLLMetry 呢？它聚焦什麼功能？

曉晨（解說）：OpenLLMetry 提供 OTel 擴充套件，包含針對 Ollama、LangChain、OpenAI、MCP 等儀器包，新增 MCP 伺服器供查詢 trace，並遵循 gen_ai.* semconv 命名規範，已在本系統的 daily_digest 中引用。

曉晨（解說）：本次提出了三項改進：P1‑a 為 ollama span alignment 已落實；P1‑b 為在 GitHub Scout SKILL.md 新增十個觀測觸發詞，同樣已落實；P2 為在 crosswalk 補充 OTel GenAI Agentic semconv 設定，已完成。
