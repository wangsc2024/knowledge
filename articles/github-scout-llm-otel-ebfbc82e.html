改進提案

P1: done_cert quality_score 推送至 Langfuse Score API（低→中風險）

  • 來源: langfuse/langfuse (28k★)
  • 方案: 每次自動任務完成後，新增 tools/langfuse_quality_push.py，透過 POST /api/public/scores 推送 done_cert.quality_score，以 trace_id 關聯 PE step checkpoint
  • 效益: 建立 longitudinal quality 追蹤 dashboard；trace_id 跨回合可對齊
  • 風險: 中風險（新外部依賴 Langfuse server）；kill switch DDP_LANGFUSE_PUSH；rollback = 停用該 post-flight hook
  • feasibility: 8, stability: 8

P1: run_metadata.operation_name 對齊 OTel gen_ai.operation.name（低風險）

  • 來源: traceloop/openllmetry (7.1k★)
  • 方案: config/github-scout-review-policy.yaml audit_trace_crosswalk 補充 gen_ai.agent.* 實驗草案屬性採納時程；新 github_scout 回合 operation_name 改用小寫蛇形（daily_digest.github_scout.search）
  • 效益: OTel 後端相容性；forward compatibility with Langfuse/Phoenix
  • 風險: 低風險（僅配置文件文件性更新）；rollback = 還原 YAML

P2: agentevals OTLP trace-based no-re-run eval 可行性評估（觀察期）

  • 來源: agentevals-dev/agentevals (135★)
  • 方案: 評估 post_tool_logger.jsonl → OTel span 轉換 → agentevals 評分路徑
  • 目前狀態: 135★ 屬新興框架；建議觀察至 2026-07-09 後再決策

審查結論

  • P1 Langfuse 推送：採納，但因需外部 Langfuse 服務，標為中風險。需確認本地 Langfuse 是否已部署（localhost:3100）或使用 Langfuse Cloud
  • P1 OTel 對齊：採納，低風險，直接在 config YAML 補充文件性欄位
  • P2 agentevals：deferred 到下一輪決策
