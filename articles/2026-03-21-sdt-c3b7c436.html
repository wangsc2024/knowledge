二、核心洞見：「外在排程 vs 內在驅動」的四維同構

2.1 第一維：SDT 動機品質階梯 ↔ 知識庫研究成熟度五階段

SDT 將動機分為六階段連續體（Deci & Ryan, 2000）：


無動機 → 外在調節 → 內攝調節 → 認同調節 → 整合調節 → 內在動機
（被動）   （獎懲驅動） （內疚驅動） （價值驅動） （身份驅動） （興趣驅動）

知識庫的五階段研究模型驚人地對應：


foundation → mechanism → application → optimization → synthesis
（概念認識）  （原理理解）  （實作應用）    （精進優化）    （跨域整合）

同構對應：

  • foundation ≈ 外在調節：因為任務要求而蒐集基礎概念，驅動力來自「系統排程」
  • mechanism ≈ 認同調節：開始理解「為什麼重要」，從被動蒐集轉向主動理解
  • application ≈ 整合調節：知識與實踐融為一體，「這就是我這樣的系統會做的事」
  • synthesis ≈ 內在動機：為了跨域連結本身的智識樂趣而研究，不需要外部任務驅動

佐證：昨日楞嚴經達到 synthesis 完結篇——「從破妄顯真到圓通成就的現代禪修完整路徑」。這不是因為某個 Todoist 任務要求它完成，而是因為知識累積到臨界質量後，整合自然發生。這正是 SDT 所說的「自成目的性」（autotelic experience）。

2.2 第二維：心流通道模型 ↔ 研究去重與飽和機制

心流理論的核心是挑戰—技能平衡：

  • 挑戰太低 → 無聊（Boredom）
  • 挑戰太高 → 焦慮（Anxiety）
  • 恰好匹配 → 心流（Flow）

知識庫的去重機制（dedup-policy.yaml）實際上是一個認知負荷優化器：

  • topic_cooldown_days: 3 → 防止同一主題重複研究（防無聊——挑戰太低）
  • saturation_threshold: 3 → 建議換方向（提供新鮮挑戰——保持在心流通道）
  • 五階段模型的階段推進 → 自動提升挑戰難度（從 foundation 到 synthesis）

深層洞見：Agent Harness 在昨日的多篇研究中顯示飽和跡象（by_type 計數偏高），而認知科學領域剛剛進入 foundation。去重機制引導系統從飽和領域轉向新鮮領域，恰好維持了心流理論所說的「挑戰—技能平衡」。去重不只是避免重複——它是系統級的心流通道維護器。

2.3 第三維：事件驅動架構(EDA) ↔ SDT 動機連續體的架構映射

昨日新增的 EDA 研究揭示了一個架構演進方向：

| 排程模式 | SDT 對應 | 特徵 |

|----------|---------|------|

| Cron 定時排程 | 外在調節 | 固定時間執行，不管有無意義事件，「因為時間到了」 |

| 輪詢(Polling) | 內攝調節 | 反覆檢查「是否該做了」，帶有焦慮性質的主動檢測 |

| Webhook 事件觸發 | 認同調節 | 識別有意義的事件並回應，「因為這件事值得回應」 |

| Pub/Sub 事件串流 | 整合調節 | 多消費者各取所需，事件處理已融入系統身份 |

| Event Sourcing | 內在動機 | 事件本身即是知識，記錄一切，為了理解本身 |

Daily-Digest 系統目前以 Cron 為核心（每小時 Todoist、每日摘要），處於「外在調節」階段。EDA 研究為系統自身的演進提供了路線圖——從「因為排程到了」到「因為有意義的事情發生了」。

關鍵引用：FlowFuse 2026 技術博客指出：「EDA flips the traditional model entirely — instead of systems asking questions on a schedule, they listen for events and react the moment something happens, eliminating the gap between occurrence and response.」這與 SDT 從外在調節到內在動機的轉變完全同構。

2.4 第四維：CRAAP Test 來源品質 ↔ SDT 勝任感(Competence)的元層次

昨日的 CRAAP Test 研究與 deep-research Skill 的 Anti-Hallucination 機制，揭示了系統正在發展元勝任感（Meta-Competence）：

  • SDT 的勝任感 = 個體感受到自己能有效完成任務
  • 系統的元勝任感 = 系統學會評估自己研究品質的能力

當系統引入 CRAAP Test（五維度來源評估）和 SIFT Method（橫向閱讀快速查核），它不只是在做研究——它在學會如何判斷研究做得好不好。這是 SDT 勝任感需求在系統層面的遞迴實現。

四、化儀四教的啟示：因機施教即是因需而動

昨日化儀四教從 mechanism 進入 application 階段（「從理論到實修的根機診斷與修行路徑選擇」），為上述模型提供了東方智慧的補充：

  • 頓教 ≈ 內在動機者——根機利鈍已具足，不需漸進階梯
  • 漸教 ≈ 外在調節→整合調節——需要循序漸進的引導
  • 秘密教 ≈ 個性化事件驅動——同一事件，不同人接收到不同的觸發
  • 不定教 ≈ 自適應 RAG——根據反饋動態調整策略

化儀四教的核心洞見是：最好的教學不是統一排程，而是因應學習者的內在狀態動態調整。這與 SDT 的「自主性支持」環境設計、EDA 的「事件反應式」架構，以及 Agentic RAG 的「自適應控制迴圈」形成四重共鳴。
