Skill 審查優化 foundation 深化研究

研究範疇：AI Agent 系統設計中的 Skill 品質審查機制（Foundation 階段）。

1. 研究背景與動機

本研究旨在補齊 skill_audit 系列在「Foundation（基礎）」階段的知識缺口。隨著 daily-digest-prompt 系統的運行，相關主題已在 ai_deep_research、tech_research 及 insight_briefing 等自動化任務中頻繁出現，但缺乏統合性的架構化文獻。

1.1 研究邏輯架構

2. 核心內容分析：證據深度解構

2.1 證據 A：Skill 品質審查機制 (KB id: `27f426b2-dfa`)

  • 核心主題：Skill 品質深度審查之基礎理論。
  • 關鍵技術點：

* AI Agent 系統設計：探討 Skill 從研究階段轉向生產環境（Production）的轉型路徑。

* 共通機制萃取：從多個實際採用案例中提煉出標準化的審查指標。

* 落地路徑：定義了從基礎研究到實踐 Playbook 的完整鏈條。

2.2 證據 B：哲學與系統論的跨領域映射 (KB id: `fd63342a-d17`)

  • 核心主題：以《法華經·藥草喻品》之「一味之雨」喻示系統的「普適性」與「差異化」。
  • 系統設計啟示：

* 一味之雨 (Universal Mechanism)：指代系統的核心底層邏輯或通用 Prompt 機制，應具備遍覆全域的普適性。

* 草木差別 (Adaptive Output)：指代系統應根據不同任務（根性、精進、利鈍之異）提供差異化的輸出結果。

* 設計原則：追求「底層邏輯一致（一味），執行層面適配（隨類得益）」的架構設計。

2.3 證據 C：AI 系統開發與修證體系 (綜合分析)

  • 核心主題：結合 AI Agent 架構設計與修證體系（如禪那、大定）的邏輯對照。
  • 關鍵概念：

* 架構設計深度：探討從簡單任務執行到複雜思維鏈（CoT）的演進。

* 穩定性研究：研究如何透過結構化的流程（如四聖諦、八正道之邏輯模型）提升 AI 決策的穩定性與正確性。

3. 跨證據綜合分析：Skill 審查框架

透過對上述證據的整合，我們提煉出 Skill 審查的三個核心維度：

  1. 一致性 (Consistency)：底層機制是否如「一味之雨」般穩定，確保所有 Skill 在基礎邏輯上不偏離預設目標。
  2. 適配性 (Adaptability)：系統是否具備「隨類得益」的能力，能針對不同複雜度的任務（小根小莖 vs 大根大莖）調整其處理深度。
  3. 結構化路徑 (Structured Path)：審查流程必須符合 OODA 循環，從觀察（Observe）到行動（Act）需有明確的邏輯支撐。

4. 實踐應用 Playbook (初步框架)

| 步驟 | 行動項目 | 關鍵指標 (KPI) |

| :--- | :--- | :--- |

| Step 1: Observe | 收集 Skill 運行日誌與錯誤樣本 | 錯誤覆蓋率、異常頻率 |

| Step 2: Orient | 對比既有機制，定位偏差原因 | 偏差度、機制匹配度 |

| Step 3: Decide | 決定優化策略（增加 Prompt 限制或調整 Workflow） | 預期提升幅度 |

| Step 4: Act | 部署優化後的 Skill 並進行 A/B Test | 成功率提升、延遲變化 |

5. 風險與權衡 (Risks & Trade-offs)

  • 過度泛化風險：追求「一味」的通用性可能導致在特定複雜任務上的適配度下降。
  • 複雜度成本：引入過多層次的審查機制會增加系統延遲（Latency）。
  • 維護成本：隨著 Skill 種類增加，維持「機制一致性」的工程成本將呈指數級增長。

6. 延伸研究方向

  • 自動化審查 Agent：開發專門用於審查其他 Skill 品質的 Meta-Agent。
  • 動態適配算法：研究如何根據任務難度自動切換不同的 Skill 深度（對應「隨類得益」的自動化實現）。
  • 長效穩定性評估：針對長期運行中的 daily-digest-prompt 進行漂移（Drift）檢測。
