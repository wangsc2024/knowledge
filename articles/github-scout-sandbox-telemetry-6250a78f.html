專案觀測摘要

  • guardrails-ai/guardrails（6.6k stars，最近更新 2026-03-16）

- 網址：https://github.com/guardrails-ai/guardrails

- 相關性：以 rails 拆分 guardrail surface，適合作為本專案研究輸出的控制面基線。

- Tool boundary：偏政策/驗證層，提醒 guardrail 不等於 sandbox。

  • snyk/agent-scan（2k stars，最近更新 2026-03-25）

- 網址：https://github.com/snyk/agent-scan

- 相關性：把 agent、skills、MCP servers 納入 inventory 與掃描報告，很適合補足 operator override 與 supply-chain 觀測。

- 證據限制：原查詢落在 invariantlabs-ai/mcp-scan，GitHub 目前 redirect 至 snyk/agent-scan。

  • agent-infra/sandbox（3.4k stars，最近更新 2025-11-12）

- 網址：https://github.com/agent-infra/sandbox

- 相關性：把 Browser/Shell/File/MCP 收斂到同一 sandbox，直接映射到本專案的工具邊界分析。

- 證據限制：repo 首頁未直接露出可機讀 commit 日期，本輪以 latest release 當最近更新代理。

改進提案

  1. P0 / 已落實：補強 github_scout 契約範例

- 將 projects[] 範例補上 evidence_snapshot

- 將 execution_contract.run_metadata 範例改為物件格式

- 在 template / team prompt 補上 redirect、rename、release-date 代理證據的註記要求

  1. P1 / plan_ready：規劃 sandbox telemetry 與 operator override 稽核

- 先輸出高風險計畫，不直接觸碰 hooks 或 scheduler

- 預計後續以 feature flag 驗證 telemetry schema 與降級路徑

審查結論

  • 已落實提案審查分數：feasibility 9 / stability 9

- 原因：只修改技能與提示契約，屬低風險文檔層修補

- 主要風險：若只改其中一份文件，仍可能再次漂移

- Rollback：還原三個檔案即可

  • 高風險提案審查分數：feasibility 7 / stability 7

- 原因：概念清楚，但一旦實作容易擴散到 hooks、structured logs 與 runtime state

- 降級方式：保留文字化 verification_outcome，等人工審核後再小範圍 rollout

  • templates/auto-tasks/github-scout.md：新增 repo redirect / latest release 代理證據的 evidence_snapshot.evidence_note 要求
  • prompts/team/todoist-auto-github_scout.md：同步補上 run_metadata 物件化提醒
