QA System 安全優化報告 #17

日期：2026-02-27 | 品質分數：5/5 | 第十七次掃描

優化方向

close_question raw SQL 改 ORM 操作（消除 Enum 不一致）+ datetime timezone-aware + zeep 版本升級

安全掃描結果

全部綠燈，無新安全問題。已確認：SECRET_KEY 環境變數化、SQL 注入防護、XSS 防護、CORS 限制、Cookie 安全、JWT 集中解碼、python-jose CVE 修復、錯誤訊息零洩漏。

已修復問題（4 項品質）

品質修復

  1. close_question raw SQL → ORM 操作：端點原本用 UPDATE questions SET status = 'closed' raw SQL 寫入字串，與 ORM Enum(QuestionStatus) 不一致。改為 question_obj.status = QuestionStatus.CLOSED，消除 ORM refresh 時的 Enum KeyError 風險。
  1. close_question datetime.now() → datetime.now(timezone.utc)：結案時間改為 timezone-aware，與全專案 #5 datetime 遷移一致。
  1. test_close_question_api 改用 ORM 驗證：原本因 raw SQL/Enum 不一致被迫用 raw SQL SELECT 驗證狀態（workaround），現改為 expire_all() + ORM query + assert status == QuestionStatus.CLOSED，同時驗證 summary 和 closed_date。
  1. zeep 版本升級 4.2.1 → 4.3.2：消除 cgi 模組棄用警告（Python 3.13 已移除 cgi 模組）。

測試結果

103/103 全數通過（zero failures），延續 #16 里程碑。

剩餘議題

  • questions.py 進一步模組拆分（仍 884 行）
  • edit_question 端點仍用 raw SQL 寫入 status 字串（同類問題）
