Ollama — 開源本地 LLM 推論框架深度研究

概述

Ollama 是 2026 年最具影響力的本地 LLM 推論框架，GitHub 166K+ Stars、15.1K Forks，Q1 2026 月下載量達 5,200 萬次（三年增長 520 倍）。專案以 Go 語言構建伺服器層，底層整合 llama.cpp 推論引擎，提供 OpenAI 相容 REST API，讓開發者在消費級硬體上以零邊際成本運行大型語言模型。

核心架構（三層設計）

Layer 1：Go Server（控制平面）

  • 模型管理：下載、版本控制、量化格式轉換（GGUF）
  • GPU 記憶體分配與多模型排程
  • OpenAI 相容 HTTP API（/api/chat、/api/generate）
  • 並行請求處理（OLLAMA_NUM_PARALLEL 配置）

Layer 2：llama.cpp 推論引擎（資料平面）

  • Georgi Gerganov 主導的 C/C++ 推論核心（73K Stars）
  • 4-bit 量化（Q4_K_M）：模型體積縮減 75%
  • CUDA Graph 支援與 Operator Fusion 優化
  • Apple Metal、NVIDIA CUDA、AMD ROCm 多後端

Layer 3：模型層

  • GGUF 格式標準化（HuggingFace 上 135,000+ GGUF 模型）
  • 支援模型：Llama 4、Qwen 2.5/3.5、DeepSeek-R1、Gemma 3、Mistral、Kimi-K2.5、GLM-5、MiniMax 等
  • Modelfile 自定義（類似 Dockerfile 的模型配置語法）

效能基準（2026 Q1）

| 硬體 | 模型 | 推論速度 | 備註 |

|------|------|---------|------|

| RTX 4090 (24GB) | 8B 參數 | 145 tok/s | 最高吞吐 |

| RTX 4090 (24GB) | 32B 參數 | ~35 tok/s | VRAM 上限 |

模型品質

  • Qwen 2.5 32B：MMLU 83.2%（接近 GPT-4 的 86.4%）
  • Qwen 3.5 7B：MMLU 76.8%（1/4 參數量，3 倍推論速度）
  • 與雲端 API 品質差距：前沿推理任務仍有 5-15% 差距

零成本推論經濟學

以每日 50,000 請求為基準：

| 方案 | 月成本 | 備註 |

|------|--------|------|

| 雲端 API（GPT-4o 等級）| ~$2,250/月 | 按 token 計費 |

| Mac Studio M4 Max (128GB) | ~$139/月 | 36 個月攤提 |

| RTX 4090 自組 PC | ~$55/月 | 36 個月攤提 |

| 電費 | ~$15/月 | 額外成本 |

損益兩平點：約 1,000 請求/日；超過此量後本地推論成本優勢呈指數級放大。

安裝與部署


# macOS/Linux
curl -fsSL https://ollama.com/install.sh | sh

# Windows PowerShell
irm https://ollama.com/install.ps1 | iex

# Docker
docker run -d -v ollama:/root/.ollama -p 11434:11434 ollama/ollama

# 基本使用
ollama pull qwen2.5:32b
ollama run qwen2.5:32b

生產部署配置


OLLAMA_NUM_PARALLEL=4        # 並行請求數
OLLAMA_MAX_LOADED_MODELS=2   # 同時載入模型數
OLLAMA_KEEP_ALIVE=5m         # 閒置卸載時間

整合生態系統

  • 開發工具：VS Code、Emacs、Claude Code、Codex、OpenCode
  • AI 平台：Open WebUI（124K Stars）、Dify、n8n
  • SDK 相容：透過 OpenAI 相容 API，僅需變更 base_url 即可對接現有 OpenAI SDK
  • 2026 新增：Windows ARM64 原生支援、多模態模型（Vision + Text）、Web Search 整合

與 daily-digest-prompt 專案的關聯

本專案的 config/llm-router.yaml 已有 Groq 路由機制，Ollama 可作為：

  1. 本地 fallback 層：Groq API 不可用時降級至本地推論
  2. 開發/測試環境：免費無限制的本地模型測試
  3. 隱私敏感任務：不需將資料送至外部 API 的場景
  4. 成本優化：高頻低複雜度任務（如分類、摘要）可路由至本地

侷限與風險

  1. 品質上限：前沿推理任務仍落後雲端 5-15%
  2. 硬體門檻：70B 模型需 128GB 以上記憶體
  3. 維運成本：需自行管理模型更新、GPU 驅動相容性
  4. 多模態限制：Vision 能力仍不及 GPT-4o/Claude
