AI Agent 互通協定比較 — MCP、A2A、ACP、ANP 四大協定的技術架構與生態演進

研究日期：2026-03-28 | 標籤：AI, Agent Protocol, MCP, A2A, ACP, ANP, 互通性

概述

2025–2026 年 AI Agent 生態系迎來「協定標準化」浪潮。四大協定各自解決不同層次的問題，形成互補的多層協定棧（Protocol Stack）。2025 年 12 月，Linux Foundation 成立 Agentic AI Foundation（AAIF），由 OpenAI、Anthropic、Google、Microsoft、AWS、Block 六家共同創立，統一治理 MCP 與 A2A。

四大協定技術比較

1. MCP（Model Context Protocol）— Agent ↔ Tool

  • 發起者：Anthropic（2024 年底開源，2025-12 捐贈 AAIF）
  • 定位：標準化 AI Agent 如何連接外部工具、資料源與服務（垂直整合）
  • 架構：Client-Server 模型，使用 JSON-RPC 2.0 作為通訊格式
  • 核心資源類型：

- Resources：唯讀資料源（檔案、資料庫記錄、API 回應）

- Tools：可執行動作（查詢、寄信、建檔）

- Prompts：可重用提示模板

- Sampling：Server 可向 Client 請求 LLM 完成

  • 傳輸層：stdio（本地）、SSE（遠端）、Streamable HTTP（生產環境雙向串流）
  • 採用規模：2026-02 達 9,700 萬次月下載（Python + TypeScript SDK），5,800+ 個公開 MCP Server
  • 關鍵特性：工具抽象化、結構化 Schema 發現、解耦可擴展

2. A2A（Agent-to-Agent Protocol）— Agent ↔ Agent

  • 發起者：Google（2025-04 發布，2025-06 捐贈 Linux Foundation）
  • 定位：標準化 AI Agent 之間的發現、溝通與協作（水平整合）
  • 架構：Client-Remote 模型，使用 JSON over HTTPS
  • Agent 發現：透過 Agent Cards（JSON manifest），部署於 /.well-known/agent.json

- 包含名稱、描述、URL、支援能力、認證需求、技能清單

  • Task 生命週期：submitted → working → input-required → completed / failed / canceled
  • 串流支援：長時間任務使用 SSE 即時進度更新
  • 合作夥伴：50+ 家發布合作夥伴，2026-02 超過 100 家企業加入
  • 關鍵特性：動態發現、多步驟任務狀態管理、加密身份標準、複雜協商模式

3. ACP（Agent Communication Protocol）— Agent ↔ Agent（REST 風格）

  • 發起者：IBM Research
  • 定位：開放的 Agent 互通協定，基於 REST 原則
  • 架構：Client-Server REST，HTTP 端點定義明確，支援同步與非同步互動
  • 核心特性：

- 無狀態 RESTful 設計，相容 cURL/Postman

- YAML 元資料被動發現

- MIME 多模態訊息支援

- 框架無關（BeeAI、CrewAI、LangChain、LangGraph 皆可）

  • 現狀：2025-08 已合併入 A2A，ACP 團隊停止獨立開發，技術貢獻整合至 A2A
  • 遺產：REST 哲學與 Agent Registry 概念被 A2A 吸收

4. ANP（Agent Network Protocol）— 跨組織網際網路級

  • 定位：網際網路規模的跨組織 Agent 通訊
  • 架構：基於標準 Web 協定（HTTPS、DNS），使用 JSON-LD 語義標記
  • 三步流程：

1. 發現：透過可預測 URL 的能力描述檔

2. 認證：使用 W3C Decentralized Identifiers（DIDs）

3. 通訊：帶語境標記的 JSON-LD 訊息

  • 關鍵特性：端到端加密、去中心化身份驗證、語義發現
  • 適用場景：跨企業 Agent 安全協作、跨組織商業場景

互補式多層協定棧

四大協定並非互斥競爭，而是各自佔據不同層次：

| 層次 | 協定 | 職責 |

|------|------|------|

| Layer 1：Context & Tools | MCP | Agent 存取工具與資料 |

| Layer 2：Messaging | A2A（含原 ACP） | Agent 間任務委派與協作 |

| Layer 3：Networking | ANP | 跨組織發現與加密通訊 |

| Layer 4：Transactions | AP2（Agent Payment Protocol） | 自主支付與交易授權 |

成熟的企業 AI 基礎設施將同時使用多層協定。

治理結構

  • AAIF（Agentic AI Foundation）：2025-12 成立，統一治理 MCP + A2A
  • 六大共同創辦者：OpenAI、Anthropic、Google、Microsoft、AWS、Block
  • ACP 已合併入 A2A（2025-08），不再獨立維護
  • 開放治理確保無單一廠商鎖定

對本專案的啟示

  1. MCP 已是事實標準：本專案已採用 MCP（skills 即 MCP-like 工具抽象），方向正確
  2. A2A 值得關注：當 Agent Team 需要跨服務協作時，Agent Cards 發現機制可參考
  3. ANP 的 DID 認證：未來跨組織 Agent 通訊的安全模型值得追蹤
  4. 多層棧思維：設計 Agent 架構時應區分「工具存取」與「Agent 協作」兩個正交維度
