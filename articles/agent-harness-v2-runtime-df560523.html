Agent Harness 自主管理機制 v2

日期：2026-03-20

本輪完成

  • 將 tools/autonomous_harness.py 從單純 dry-run supervisor 擴充為真正的自治控制面。
  • 新增 fairness、token budget、scheduler heartbeat 三類 gate。
  • 新增 state/autonomous-runtime.json，輸出 normal / degraded / recovery 模式與對應執行政策。
  • run-todoist-agent-team.ps1 在 Phase 2 前讀取 runtime policy，會依模式降低 auto-task 平行度，並過濾重型/研究型任務。

控制迴路

  1. Observe：讀取 run-fsm、scheduler-state、failed-auto-tasks、api-health、auto-task-fairness-hint、token-budget-state、scheduler-heartbeat
  2. Evaluate：判斷 stale run、重複失敗、open circuit、任務飢餓、token 告警、heartbeat stale
  3. Gate：推導 normal / degraded / recovery
  4. Act：輸出 autonomous-harness-plan.json、autonomous-recovery-queue.json、autonomous-runtime.json
  5. Adapt：排程層根據 runtime policy 自動降載與限制重型任務

關鍵設計

  • rebalance_tasks：當飢餓任務超過閾值時觸發，優先降低平行度並補跑低頻任務。
  • scale_down_workload：當 token budget 當日已告警時觸發，禁止高成本工作負載。
  • scheduler-heartbeat gate：heartbeat 遺失或逾時時進入 recovery 模式。
  • recovery queue 去重：避免相同 pending action 持續重複入列。

已知限制

  • agent -p 本機仍回 [internal]，Cursor CLI 無法作為本輪可用執行後端。
  • pytest 受本機 ACL 影響，session cleanup 被暫存目錄權限阻斷；需在可寫暫存環境重跑。
  • 目前只有 run-todoist-agent-team.ps1 已讀取 runtime policy，run-agent-team.ps1 尚未接上。
