一、專案概述

Firecrawl（firecrawl/firecrawl）是一套將網站轉換為 LLM 可讀資料的 Web Data API 平台，由 Mendable（Y Combinator 支持）團隊開發，定位為「Turn entire websites into LLM-ready markdown or structured data」。截至 2026 年 3 月已累積 87,652 Stars、6,211 Forks，採用 AGPL-3.0 授權，是目前 GitHub 上 AI 資料擷取領域最具影響力的開源專案。

專案定位

Firecrawl 解決了 AI/LLM 應用中一個核心痛點：如何可靠地從網頁取得乾淨、結構化的資料。傳統爬蟲面臨 JavaScript 渲染、反爬蟲機制、動態內容等挑戰，Firecrawl 透過多引擎架構和 AI 輔助提取，提供業界領先的 >80% 覆蓋率基準測試成績。

二、核心架構：五層 Web Data 管線

Firecrawl 採用 Monorepo 架構，核心 API 位於 apps/api，配合多個獨立服務協同運作。

2.1 系統組件架構

根據 docker-compose.yaml 分析，完整部署包含五個核心服務：


[Client] --> [API Server (Node.js, port 3002)]
                  |
                  +--> [Redis (Queue & Rate Limit)]
                  |
                  +--> [RabbitMQ (Message Broker)]
                  |
                  +--> [Playwright Service (port 3000)]
                  |
                  +--> [PostgreSQL (nuq-postgres)]

| 服務 | 技術 | 用途 |

|------|------|------|

| API Server | Node.js / TypeScript | HTTP API + Worker 處理 |

| Redis | Redis Alpine | 任務佇列（Bull Queue）+ 速率限制 |

| RabbitMQ | RabbitMQ 3 Management | 訊息代理（跨 Worker 通訊） |

| Playwright Service | TypeScript + Playwright | JavaScript 渲染、瀏覽器自動化 |

| PostgreSQL | PostgreSQL | 持久化儲存（認證、日誌） |

2.2 多引擎爬取架構（Engine Layer）

Firecrawl 的核心競爭力在於其多引擎爬取策略。apps/api/src/scraper/scrapeURL/engines/ 目錄揭示了五類引擎：

| 引擎 | 用途 | 特點 |

|------|------|------|

| fetch | 基礎 HTTP 請求 | 最快、資源最少，適合靜態頁面 |

| playwright | 瀏覽器渲染 | 處理 JavaScript SPA、動態內容 |

| fire-engine | 進階爬取（雲端） | IP 輪替、反偵測、Robot 繞過 |

| pdf | PDF 文件解析 | 支援 LlamaParse 整合 |

| document | 非 HTML 文件 | DOCX、圖片 OCR 等 |

引擎選擇邏輯位於 engines/index.ts（18,393 bytes），根據 URL 特性、用戶配置和可用資源自動選擇最適合的引擎。

2.3 轉換器層（Transformer Layer）

transformers/ 目錄包含七個核心轉換器：

| 轉換器 | 功能 |

|--------|------|

| llmExtract.ts | LLM 驅動的結構化資料提取（52,280 bytes，最大模組） |

| agent.ts | Agent 模式轉換（自主資料蒐集） |

| diff.ts | 內容變更追蹤（Change Tracking） |

| performAttributes.ts | 屬性萃取（品牌色彩、字型等） |

| removeBase64Images.ts | Base64 圖片清理 |

| sendToSearchIndex.ts | 搜尋索引發送 |

| uploadScreenshot.ts | 截圖上傳 |

2.4 後處理器層（Postprocessor Layer）

目前包含 YouTube 特化處理器（youtube.ts），負責 YouTube 影片頁面的結構化解析。

三、五大核心功能

3.1 Scrape（單頁爬取）

將任意 URL 轉換為七種輸出格式：

  • markdown：LLM 最友善的格式，清除廣告/導航等雜訊
  • html：清理後的 HTML
  • rawHtml：原始 HTML
  • screenshot：Base64 編碼截圖
  • links：頁面內所有連結
  • json：LLM 驅動的結構化 JSON（支援 Pydantic Schema 或自然語言 Prompt）
  • branding：品牌識別（色彩、字型、排版）

from firecrawl import Firecrawl
from pydantic import BaseModel

app = Firecrawl(api_key="fc-YOUR_API_KEY")

class CompanyInfo(BaseModel):
    company_mission: str
    is_open_source: bool

result = app.scrape(
    'https://example.com',
    formats=[{"type": "json", "schema": CompanyInfo.model_json_schema()}]
)

3.2 Actions（互動式爬取）

在提取資料前執行瀏覽器操作：點擊、輸入、滾動、等待、截圖。適合需要登入或互動才能存取的頁面：


doc = app.scrape(
    url="https://example.com/login",
    actions=[
        {"type": "write", "text": "user@example.com"},
        {"type": "press", "key": "Tab"},
        {"type": "write", "text": "password"},
        {"type": "click", "selector": 'button[type="submit"]'},
        {"type": "wait", "milliseconds": 2000}
    ]
)

3.3 Search（網路搜尋 + 爬取）

搜尋網路並可選地爬取結果頁面完整內容：


results = firecrawl.search(
    "firecrawl web scraping",
    limit=3,
    scrape_options={"formats": ["markdown", "links"]}
)

自托管版本支援 SearXNG 作為搜尋後端。

3.4 Crawl（全站爬取）

從起始 URL 遞歸爬取整個網站，支援深度限制、URL 過濾、並發控制：


docs = app.crawl("https://docs.example.com", limit=100)
for doc in docs.data:
    print(doc.metadata.source_url, doc.markdown[:100])

3.5 Agent（AI 自主資料蒐集）

v2.8.0 引入的革命性功能 -- 描述你需要的資料，AI Agent 自動搜尋、導航、提取：


result = app.agent(
    prompt="Find the pricing plans for Notion",
    model="spark-1-pro"  # 或 spark-1-mini（預設，便宜 60%）
)

Spark 模型家族：

| 模型 | 定位 | Recall | 成本 |

|------|------|--------|------|

| Spark 1 Fast | 即時檢索 | -- | 最低 |

| Spark 1 Mini | 日常提取（預設） | ~40% | 標準的 40% |

| Spark 1 Pro | 複雜跨域研究 | ~50% | 標準 |

Agent 支援結構化輸出（Pydantic Schema）、Webhook 回呼、指定 URL 聚焦、並行執行（v2.8.0 新增）。

3.6 Map（URL 發現）

快速發現網站所有 URL，支援搜尋過濾：


result = app.map("https://example.com", search="pricing")

四、技術特色與創新

4.1 開發流程

根據 CLAUDE.md，Firecrawl 採用端到端測試優先的開發方法（E2E-first），稱為「snips」：

  • 先寫 E2E 測試（happy path + failure path）
  • 實作功能通過測試
  • 透過 pnpm harness jest 運行（harness 自動啟動 API + Worker）
  • CI 跑完整測試套件

4.2 資源管理

  • CPU/RAM 閾值：Worker 在 CPU/RAM 超過 80% 時自動拒絕新任務
  • 並發控制：NUM_WORKERS_PER_QUEUE（預設 8）、CRAWL_CONCURRENT_REQUESTS（預設 10）、MAX_CONCURRENT_JOBS（預設 5）
  • 瀏覽器池：BROWSER_POOL_SIZE（預設 5）控制 Playwright 並發頁面數

4.3 多 AI 後端支援

  • OpenAI API：預設 LLM 後端（JSON 提取、Agent 功能）
  • Ollama：實驗性支援本地 LLM（如 deepseek-r1:7b）
  • OpenAI 相容 API：任何 OpenAI 相容端點
  • LlamaParse：PDF 解析專用

4.4 Change Tracking（變更追蹤）

diff.ts 轉換器提供網站內容變更監控功能，適合：

  • 價格監控
  • 競爭對手分析
  • 合規性監測

4.5 MCP 整合生態

Firecrawl 提供官方 MCP Server（firecrawl/firecrawl-mcp-server，5,639 Stars），支援 Claude Code、Cursor、OpenCode 等 AI 編輯器直接使用 Firecrawl 進行網頁爬取。

4.6 Skill 安裝機制


npx skills add firecrawl/cli

一鍵為 AI Agent 安裝 Firecrawl 能力，重啟後即可自主使用。

六、自托管架構

Firecrawl 支援完整的自托管部署，透過 Docker Compose 一鍵啟動：


docker compose up -d

自托管限制

| 功能 | 雲端版 | 自托管版 |

|------|--------|--------|

| 基礎爬取（fetch + Playwright） | 完整 | 完整 |

| Fire-engine（進階反偵測） | 完整 | 不可用 |

| AI 功能（JSON/Agent） | 完整 | 需自備 OpenAI Key 或 Ollama |

| Search | Google | 需配置 SearXNG |

| 多架構支援 | -- | amd64 + arm64（v2.8.0+） |

| SOC2 Type2 合規 | 內建 | 自行負責 |

適用場景

  • 資料安全合規要求嚴格的組織
  • 需要客製化爬取邏輯
  • 內網資料爬取
  • 學習與貢獻目的

八、與 Daily-Digest-Prompt 系統的借鑑意義

8.1 直接應用價值

  1. 新聞爬取增強：目前 fetch-news.md 透過 MCP JSON-RPC 查詢屏東新聞，若未來需要爬取新聞全文或跨來源新聞，Firecrawl 的 Search + Scrape 組合是理想方案
  2. HackerNews 深度分析：fetch-hackernews.md 目前只取摘要，可透過 Firecrawl 爬取原始文章全文供 Groq 翻譯/摘要
  3. 知識庫內容擴充：Firecrawl Agent 可自主搜尋+提取研究主題的完整資料，提升 KB 內容品質

8.2 架構借鑑

  1. 多引擎降級策略：Firecrawl 的 fetch -> playwright -> fire-engine 自動降級模式，可參考用於本專案的 API 快取降級機制
  2. E2E 測試優先：snips 測試方法論（先寫 E2E happy/failure path）可應用於本專案的 Todoist API 測試
  3. 資源閾值管理：CPU/RAM 80% 閾值自動拒絕任務的模式，可參考應用於 team mode 的並行 Agent 資源控制
  4. Monorepo + Harness 模式：pnpm harness 自動啟動依賴服務後跑測試，類似本專案的 run-*-team.ps1 多階段管線

8.3 MCP 整合可能性

Firecrawl MCP Server（5,639 Stars）可直接整合到本專案的 Claude Code Agent 中，讓自動任務具備網頁深度爬取能力。安裝方式：


npx skills add firecrawl/cli

九、優缺點評估

優點

  1. 業界領先的覆蓋率：>80% 基準測試覆蓋率，超越所有其他測試提供者
  2. 完整 AI 整合：從 JSON Schema 提取到自主 Agent，原生支援 LLM 工作流
  3. 多引擎架構：自動選擇最適引擎，兼顧效能與可靠性
  4. 豐富的輸出格式：7 種格式覆蓋所有 AI 應用場景
  5. 強大的 SDK 生態：Python/JS/Rust/Go/CLI 全覆蓋
  6. 活躍的社群：87K+ Stars、6K+ Forks、每日持續更新
  7. 自托管友善：Docker Compose 一鍵部署，支援 Ollama 本地 LLM
  8. MCP 整合：與 Claude Code/Cursor/OpenCode 無縫整合

缺點

  1. AGPL-3.0 授權：商業使用需注意授權要求（修改後的衍生作品必須開源）
  2. Fire-engine 不開源：進階反偵測功能僅雲端版可用，自托管受限
  3. AI 功能依賴外部 LLM：JSON 提取和 Agent 需要 OpenAI/Ollama，增加部署複雜度
  4. 資源需求較高：完整部署需要 Redis + RabbitMQ + PostgreSQL + Playwright，記憶體建議 8GB+
  5. 自托管維護成本：需自行處理代理、反偵測、搜尋引擎配置
