Mixture-of-Agents (MoA) — 多 LLM 協作推理架構

概述

Mixture-of-Agents (MoA) 是由 Together AI 研究團隊（Wang et al., 2024）提出的多模型協作推理框架。其核心洞見在於 LLM 的「協作性」（Collaborativeness）現象：當模型接收其他模型的輸出作為參考時，即使那些模型能力較弱，仍能生成更優質的回應。MoA 利用此特性，透過分層架構將多個 LLM 組織為協作團隊，達成超越任何單一模型的推理品質。

核心架構設計

分層結構

MoA 採用多層（通常 3 層）架構，每層包含 n 個 LLM 代理：

  1. 第一層：各 LLM 獨立回應使用者查詢，生成多元初始回應
  2. 中間層：每個代理將前層所有輸出作為輔助資訊，迭代精煉回應
  3. 最終層：聚合者綜合所有回應，產出單一高品質最終答案

角色分工：Proposer 與 Aggregator

| 角色 | 功能 | 特性 | 代表模型 |

|------|------|------|----------|

| Proposer（提案者） | 生成多元參考回應 | 擅長產出有價值的獨特觀點 | WizardLM-2-8x22b、Mixtral-8x22B |

| Aggregator（聚合者） | 整合多模型輸出為統一答案 | 擅長綜合分析與品質把控 | Qwen1.5-110B-Chat、GPT-4o |

關鍵發現：GPT-4o、Qwen1.5、LLaMA-3 在兩角色均表現優異；WizardLM 作為 Proposer 卓越但聚合能力較弱。

性能基準

Self-MoA：單模型自集成的挑戰

OpenReview 研究發現

「Rethinking Mixture-of-Agents: Is Mixing Different Large Language Models Beneficial?」提出 Self-MoA 概念：

  • Self-MoA 僅使用單一頂級模型的多次抽樣（而非多個不同模型）
  • 在 AlpacaEval 2.0 上，Self-MoA 較 MoA 改進 6.6%
  • 跨 MMLU、CRUX、MATH 等基準，平均改進 3.8%

啟示：模型多樣性的價值可能被高估，單一強模型的多次推理加聚合可能更具成本效益。但這一結論仍有爭議，取決於具體任務類型和模型組合。

實作考量

優勢

  • 零微調部署：完全依賴提示工程，無需訓練
  • 模組化組合：可靈活替換任意層的模型
  • 開源超越閉源：純開源配置超越 GPT-4o
  • 互補專長：不同模型的錯誤模式不相關，集成更穩健

劣勢

  • 延遲增加：多層串行推理，TTFT 顯著上升
  • 成本倍增：6 個模型 × 3 層 = 18 次推理調用
  • 回應冗長：聚合傾向保留更多內容
  • API 依賴：需要同時存取多個模型端點

成本優化策略

  • MoA-Lite：兩層配置，成本削減 50%+ 仍超越 GPT-4 Turbo
  • Proposer 數量調整：3 個 vs 6 個的邊際效益遞減
  • 模型大小混合：小模型作 Proposer，大模型作 Aggregator

對 Agent 系統設計的啟示

  1. 多 Agent 協作的理論基礎：MoA 為「多個 AI Agent 合作優於單一 Agent」提供了實證支持
  2. 角色專業化：不同模型適合不同角色，與 Agent Team 的分工模式一致
  3. 異質集成優於同質：使用不同架構/訓練資料的模型組合，錯誤相關性更低
  4. 可應用於 RAG：在檢索增強生成中，多模型交叉驗證可提高事實準確性
