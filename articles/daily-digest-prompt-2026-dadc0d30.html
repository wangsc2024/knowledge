優化概述

針對專案失敗統計（state/failure-stats.json）進行根因分析，識別並修復兩個高頻失敗問題，預期可減少 15.5% 的 Phase Failure（45/291 次）。

問題分析

系統健康狀態

  • 當前成功率：67.8%（來源：state/health-brief-latest.json）
  • 主要問題：逾時、Phase失敗
  • Phase Failure 統計（2026-03-18 至 2026-04-17）：

- 總計：291 次

- 細分：assembly_side_effects_missing（34次）、result_file_missing（11次）、agent_crash（11次）、timeout（21次）

根因識別

#### 問題 1：assembly_side_effects_missing（34次，11.7%）

症狀：Phase 3 組裝 Agent 聲稱完成，但預期副作用未產生。

根因：

  1. PowerShell 腳本僅檢查 context/continuity/todoist-hourly.json 的 mtime
  2. LLM 可能「幻覺完成」— 輸出看似合理但實際未執行關鍵步驟
  3. 缺少全面的副作用驗證機制

影響範圍：run-todoist-agent-team.ps1:8162-8190

#### 問題 2：result_file_missing（11次，3.8%）

症狀：Phase 2 執行完成但未產生預期的結果檔案。

根因：

  1. Agent 執行過程中崩潰或異常
  2. LLM 跳過 Fail-Safe 佔坑步驟
  3. 結果檔案驗證不足

實施優化

優化 1：強化 Phase 3 副作用驗證機制

檔案：prompts/team/todoist-assemble.md

變更內容：

  • 新增「步驟 6.5：執行後自我驗證」（73行）
  • 建立 6 項強制檢查清單：ntfy 通知、排程連續記憶、warned_labels、closed_task_ids、計數器、history

預期效果：防止 LLM「幻覺完成」，提供詳細失敗診斷，失敗時保留檔案供除錯。

優化 2：改進結果檔案生成機制

檔案：templates/shared/preamble.md

變更內容：

  • 強化 Fail-Safe 佔坑（+23行）：明確「不可跳過」，新增驗證步驟
  • 強化結果驗證（+14行）：詳細化驗證流程，明確必填欄位

預期效果：確保所有任務都有基本結果檔案（即使崩潰），提升格式正確性。
