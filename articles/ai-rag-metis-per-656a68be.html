深度研究洞察簡報：複合 AI 系統的在線 RAG 配置學習

METIS Per-Query Profile 動態建立機制與多目標最佳化

日期：2026-04-24

系列：LLM 路由與聯合優化（第 4 期，mechanism 深化）

摘要

複合 AI 系統（Compound AI System）中的 RAG 配置選擇長期存在「優化延遲」或「優化品質」的二選一困境。2025-2026 年的研究突破展示了三條並行路徑：Per-Query Profile 驅動的聯合排程（METIS）、全局對齊局部獎勵的 Prompt 協同優化（OPTIMAS）、多目標貝葉斯優化的 Agent 團隊配置（MALBO），以及聲明式工作流 + 自適應調度（Murakkab）。核心洞見：配置空間剪枝 + 在線學習 + 多目標 Pareto 前沿搜尋，才是系統級效率優化的正確姿勢。

核心洞見

洞見 1：METIS 聯合優化（SOSP 2025）

METIS（arXiv 2412.10543）首次在 RAG 系統中實現聯合排程與配置適應。

機制：

  • Query Profile 估計：用輕量 LLM 快速分析每個 query 的資訊需求輪廓（需要幾個 chunk？是否需要跨段落聯合推理？）
  • 配置空間剪枝：從全量配置空間中剪掉無效方案，大幅縮小搜尋空間
  • 聯合排程：依據可用資源與 query pool 特徵，同時決定配置選擇與執行順序

結果：4 個主流 RAG-QA 資料集，延遲降低 1.64-2.54×，品質零損失。

洞見 2：OPTIMAS 局部-全局獎勵對齊（ICLR 2026）

OPTIMAS（arXiv 2507.03041）解決複合 AI DAG 中「局部改進損害全局」問題。

解法：維護全局對齊的局部獎勵函數（Globally-Aligned Local Reward Function），每次迭代用少量真實數據輕量調整局部獎勵，使局部改進與全局目標同向。

結果：在 5 個真實複合 AI 系統上，相比基準方法平均提升 +11.92%。

洞見 3：MALBO 多目標 Bayesian 優化

MALBO（arXiv 2511.11788）將 LLM Agent 團隊組成正式化為多目標優化問題。

方法：在連續 LLM 特徵空間中搜尋，用獨立高斯過程代理模型估計 accuracy/cost，以 EHVI 引導樣本採集，產出 Pareto 前沿。

結果：配置成本降低 > 45%，異質專業化團隊可實現高達 65.8% 成本減少。

洞見 4：Murakkab 聲明式工作流（HotOS 2025）

Murakkab（arXiv 2501.16634）提出聲明式工作流 + 自適應調度 runtime。

結果：工作流完成時間加速 ~3.4×，能源效率提升 ~4.5×。

洞見 5：Per-Query Profile 的共通性

跨四篇論文的共識：細粒度 profiling 是所有優化的基礎；配置空間剪枝是計算效率的核心；在線迭代學習優於一次性離線優化。

建議行動

短期：為 auto-task 引入 task profiling（參考 METIS per-query profile）；OODA Decide 引入雙目標指標（成功率 + token 成本）。

中期：聲明式自動任務排程（參考 Murakkab）；局部-全局獎勵對齊（self_heal 與 arch_evolution 協同優化方向）。
