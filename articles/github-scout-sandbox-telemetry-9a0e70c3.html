GitHub Scout：Sandbox telemetry feature flag 與 operator override rollout - 2026-03-29

專案觀測摘要

  • 主題：Sandbox telemetry feature flag 與 operator override rollout。
  • agentscope-ai/agentscope-runtime：GitHub repo 顯示 670 stars，README NEWS 於 2026-02 提到架構重構與 Distributed Interrupt Service，顯示其 sandbox/runtime 已把人工中斷與狀態恢復視為正式能力。
  • QwenLM/Qwen-Agent：GitHub repo 顯示 15.8k stars，README News 於 2026-02-16 提到 Qwen3.5 agent demo，且 README 明示 code interpreter 與 MCP 是主要能力，也提醒部分執行器僅適合本機測試。
  • NVIDIA-NeMo/Guardrails：GitHub repo 顯示 5.9k stars，repo releases 顯示 v0.21.0 發佈於 2026-03-12，代表 guardrail surface 仍在活躍演進。
  • snyk/agent-scan：GitHub repo 顯示 2k stars，原 invariantlabs-ai/mcp-scan 會 redirect 到 snyk/agent-scan，release 頁最新版本為 2025-11-21，代表蒐證時必須保留 redirect/release 類證據，否則 stars 與最近更新會失真。

改進提案

  • P0 / 已落實：把 evidence_snapshot 納入 github_scout 契約，要求每個專案記錄 repo_observed_at、source_type、freshness_note，用來說明 stars/最近更新來自 repo page、releases page 或 search snapshot，降低 redirect 與 release-only 專案造成的證據漂移。
  • P1 / 待人工審核：建立 sandbox audit rollout 計畫，以 feature flag 方式把 sandbox decision、tool boundary 與 operator override 記錄接到執行層；此案會碰 audit schema 與 runtime/hook 邏輯，不宜自動落地。
  • P1 / 草案：後續可把 tool_boundary 的審查模板擴充為「允許本機測試 / 禁止 production 直開」的覆寫矩陣，對齊 Qwen-Agent 在 README 中對 code interpreter 的風險提示。

審查結論

  • 已採納 evidence_snapshot 契約層改進。可行性 9/10、穩定性 9/10，因為僅修改 config/template/prompt/skill，不碰 scheduler、核心攔截 hooks 與 run 腳本。
  • 採納原因：本輪觀測到 redirect repo、release 頁與 repo 頁資訊來源不同，若沒有證據快照，stars 與最近更新無法精準追溯。
  • 主要風險：只增加欄位，低風險；若後續流程未依新欄位填寫，最多造成審查資料不完整，不影響執行邏輯。
  • rollback / 降級方式：移除 config/github-scout-review-policy.yaml 中的 evidence_snapshot 要求，並還原 templates/auto-tasks/github-scout.md、prompts/team/todoist-auto-github_scout.md、skills/github-scout/SKILL.md 的對應文字。
  • 工具邊界 / 護欄影響：本次已落實項目只加強研究證據契約，不改工具調用或護欄策略；高風險計畫則明確要求 feature flag 關閉時回退到既有 review-only 流程。

已落實項目

  • 更新 config/github-scout-review-policy.yaml：新增 project_evidence.evidence_snapshot_required_fields 與說明。
  • 更新 templates/auto-tasks/github-scout.md：安全主題研究時，要求 projects[] 補齊 evidence_snapshot。
  • 更新 prompts/team/todoist-auto-github_scout.md：把 evidence_snapshot 納入本輪研究與輸出要求。
  • 更新 skills/github-scout/SKILL.md：在 projects schema 補上 evidence_snapshot 範例與定義。

待人工審核項目

  • docs/plans/github-scout-20260329-sandbox-audit-rollout.md：高風險 sandbox audit rollout 計畫，需人工決定是否把 audit schema 與 feature flag 接入執行層。
