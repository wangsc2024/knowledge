Claude Code Plugin 生態系統完整指南 — Marketplace 架構、LSP 整合、Git Worktree 隔離與企業級治理（2026-02）

主題概述

Claude Code 在 2026 年初已從單一 CLI 工具演化為擁有 9,000+ 插件的完整開發平台。Plugin 系統是這個生態的核心骨架，它將 Slash Commands、Subagents、MCP Servers、Hooks 和 LSP Servers 五大擴展點統一封裝為可安裝、可分享、可治理的插件單元。本文深入解析 Plugin 架構的技術規格、Marketplace 分發機制、LSP 語言服務整合、Git Worktree 隔離，以及企業級管控策略。

---

一、Plugin 架構五大擴展點

Claude Code Plugin 是五種擴展機制的統一封裝：

1.1 五大組件

| 擴展點 | 用途 | 範例 |

|--------|------|------|

| Slash Commands / Skills | 自定義 / 指令，用 SKILL.md 定義行為 | /review、/deploy、/fix-issue 123 |

| Subagents | 專用 Agent，支援 context: fork 隔離執行 | security-reviewer、compliance-checker |

| MCP Servers | 透過 Model Context Protocol 連接外部工具與資料源 | 資料庫查詢、CI/CD 整合 |

| Hooks | 在工具生命週期關鍵點自定義行為 | PreToolUse 驗證、PostToolUse 日誌 |

| LSP Servers | Language Server Protocol 即時程式碼智能 | 型別檢查、go-to-definition、diagnostics |

1.2 Plugin 目錄結構


my-plugin/
├── .claude-plugin/
│   └── plugin.json          # 插件清單（必須）
├── skills/
│   └── review/
│       └── SKILL.md          # Skill 定義
├── agents/
│   └── security-reviewer.md  # Subagent 定義
├── scripts/
│   └── validate.sh           # Hook 腳本
└── servers/
    └── db-server              # MCP/LSP 伺服器

1.3 plugin.json Schema


{
  "name": "review-plugin",
  "description": "Adds a /review skill for quick code reviews",
  "version": "1.0.0",
  "author": { "name": "DevTools Team", "email": "dev@example.com" },
  "commands": ["./commands/core/", "./commands/experimental/preview.md"],
  "agents": ["./agents/security-reviewer.md"],
  "hooks": {
    "PostToolUse": [{
      "matcher": "Write|Edit",
      "hooks": [{ "type": "command", "command": "${CLAUDE_PLUGIN_ROOT}/scripts/validate.sh" }]
    }]
  },
  "mcpServers": {
    "enterprise-db": {
      "command": "${CLAUDE_PLUGIN_ROOT}/servers/db-server",
      "args": ["--config", "${CLAUDE_PLUGIN_ROOT}/config.json"]
    }
  },
  "lspServers": { ... }
}

關鍵變數：${CLAUDE_PLUGIN_ROOT} 在 Hook 和 Server 配置中引用插件安裝目錄（因為插件安裝時會被複製到 ~/.claude/plugins/cache）。

二、Marketplace 分發架構

2.1 Marketplace 本質

Marketplace 是一個 Git 倉庫，根目錄下有 .claude-plugin/marketplace.json 索引檔。支援 GitHub、GitLab、Bitbucket 或自建 Git 服務託管。

2.2 marketplace.json Schema


{
  "name": "company-tools",
  "owner": { "name": "DevTools Team", "email": "devtools@example.com" },
  "metadata": {
    "description": "Internal development tools",
    "version": "1.0.0",
    "pluginRoot": "./plugins"
  },
  "plugins": [
    {
      "name": "code-formatter",
      "source": "./plugins/formatter",
      "description": "Automatic code formatting",
      "version": "2.1.0",
      "category": "productivity",
      "tags": ["formatting", "code-quality"],
      "keywords": ["lint", "format"]
    },
    {
      "name": "deploy-tools",
      "source": { "source": "github", "repo": "company/deploy-plugin", "ref": "v2.0.0" }
    }
  ]
}

2.3 五種 Plugin Source 類型

| Source | 格式 | 適用場景 |

|--------|------|----------|

| 相對路徑 | "./plugins/my-plugin" | 同倉庫內的插件 |

| GitHub | { "source": "github", "repo": "owner/repo", "ref": "v2.0", "sha": "abc123..." } | GitHub 託管 |

| Git URL | { "source": "url", "url": "https://gitlab.com/team/plugin.git" } | 任意 Git 服務 |

| npm | { "source": "npm", "package": "@acme/plugin", "version": "^2.0.0" } | npm 分發 |

| pip | { "source": "pip", "package": "my-plugin" } | Python pip 分發 |

2.4 CLI 操作


# 新增 Marketplace
/plugin marketplace add owner/repo
/plugin marketplace add https://gitlab.com/company/plugins.git
/plugin marketplace add ./local-marketplace

# 安裝插件
/plugin install review-plugin@my-marketplace

# 更新 Marketplace
/plugin marketplace update

# 驗證
claude plugin validate .
/plugin validate .

三、Skill 系統深度解析

3.1 SKILL.md Frontmatter 完整欄位

| 欄位 | 用途 |

|------|------|

| name | Skill 名稱，成為 /name 指令 |

| description | Claude 自動匹配的描述 |

| argument-hint | 自動完成提示，如 [issue-number] |

| disable-model-invocation | true = 僅手動觸發（如 /deploy） |

| user-invocable | false = 僅 Claude 自動觸發（背景知識） |

| allowed-tools | 限制 Skill 可用的工具 |

| model | 指定執行模型 |

| context | fork = 在隔離 subagent 中執行 |

| agent | Subagent 類型（Explore / Plan / general-purpose） |

| hooks | Skill 生命週期 Hooks |

3.2 動態上下文注入

!\command\`` 語法在 Skill 送出前執行 Shell 命令，結果內嵌到 prompt 中：


---
name: pr-summary
context: fork
agent: Explore
---
## PR 上下文
- PR diff: !`gh pr diff`
- 變更檔案: !`gh pr diff --name-only`

3.3 字串替換變數

| 變數 | 說明 |

|------|------|

| $ARGUMENTS | 所有引數 |

| $ARGUMENTS[N] / $N | 第 N 個引數（0-indexed） |

| ${CLAUDE_SESSION_ID} | 當前 Session ID |

3.4 Skill 優先級

Enterprise > Personal (~/.claude/skills/) > Project (.claude/skills/) > Plugin（命名空間 plugin-name:skill-name）。

四、Git Worktree 隔離

4.1 核心概念

Git Worktree 讓多個 Agent 在同一 repo 上並行工作，互不干擾。每個 Agent 擁有獨立的工作目錄和分支。

4.2 使用方式


# CLI 啟動隔離 worktree
claude --worktree
claude -w

# 指定 worktree 名稱
claude --worktree feature-auth

# 搭配 tmux 背景執行
claude --worktree --tmux

4.3 Subagent 宣告式隔離

在 Agent 定義的 frontmatter 中加入：


---
isolation: worktree
---

Agent 自動在獨立 worktree 中執行，完成後根據是否有變更決定清理策略：

  • 無變更 → 自動刪除 worktree 和分支
  • 有變更/提交 → 提示用戶保留或移除

4.4 平行開發場景

  • Agent A 在 worktree-1 重寫 src/auth.ts
  • Agent B 在 worktree-2 用不同方案重寫同檔案
  • 開發者審查兩個分支，選擇最佳方案

4.5 非 Git VCS 支援

透過 WorktreeCreate 和 WorktreeRemove Hooks 自定義 worktree 建立/清理邏輯，支援 SVN、Perforce、Mercurial 等。

五、LSP Server 整合

5.1 支援語言

11 種語言：Python、TypeScript、Go、Rust、Java、C/C++、C#、PHP、Kotlin、Ruby、HTML/CSS。

5.2 提供的功能

  • Go-to-definition：跳轉到定義位置
  • Find references：查找所有引用
  • Type checking：即時型別檢查
  • Diagnostics：錯誤與警告診斷
  • Symbol documentation：符號型別資訊與文件

5.3 配置方式

在 plugin.json 中設定 lspServers，指定伺服器執行檔、通訊協議、支援的副檔名和語言標識。

六、企業級治理

6.1 Managed Marketplace 限制

管理員透過 strictKnownMarketplaces 控制可安裝的 Marketplace：


{
  "strictKnownMarketplaces": [
    { "source": "github", "repo": "acme-corp/approved-plugins" },
    { "source": "hostPattern", "hostPattern": "^github\\.example\\.com$" }
  ]
}

| 值 | 行為 |

|----|------|

| 未定義 | 無限制，可新增任何 Marketplace |

| 空陣列 [] | 完全鎖定，不可新增 |

| 來源清單 | 僅允許匹配的 Marketplace |

6.2 團隊自動安裝

在專案 .claude/settings.json 中配置：


{
  "extraKnownMarketplaces": {
    "company-tools": {
      "source": { "source": "github", "repo": "your-org/claude-plugins" }
    }
  },
  "enabledPlugins": {
    "code-formatter@company-tools": true
  }
}

協作者信任專案時自動提示安裝。

6.3 Release Channel（穩定/最新）

透過兩個 Marketplace 指向同一 repo 的不同 ref，實現穩定版和最新版分軌分發，透過 Managed Settings 分配給不同用戶群組。

6.4 Strict Mode

| 值 | 行為 |

|----|------|

| true（預設） | plugin.json 為權威，Marketplace 可補充 |

| false | Marketplace 為完整定義，plugin 不得有衝突 |

6.5 私有倉庫驗證

| 平台 | 環境變數 |

|------|----------|

| GitHub | GITHUB_TOKEN / GH_TOKEN |

| GitLab | GITLAB_TOKEN / GL_TOKEN |

| Bitbucket | BITBUCKET_TOKEN |

6.6 ConfigChange Hook

新增 ConfigChange Hook 事件，在 Session 中配置檔變更時觸發，支援企業安全審計和選擇性阻止設定變更。

七、與 daily-digest-prompt 專案的對比洞察

| 維度 | Claude Code Plugin 系統 | daily-digest-prompt |

|------|------------------------|---------------------|

| 擴展封裝 | 五合一 Plugin（Skills + Agents + MCP + Hooks + LSP） | Skills 自包含 + Hooks 獨立 |

| 分發機制 | Marketplace + npm/pip/GitHub | 專案內 skills/ 目錄 |

| 版本管理 | plugin.json version + ref/sha 鎖定 | SKILL.md frontmatter version |

| 隔離模式 | Git Worktree 原生支援 | PowerShell Start-Job 進程隔離 |

| 企業治理 | strictKnownMarketplaces + Managed Settings | Hook 規則 + YAML Schema 驗證 |

| 動態載入 | Skill 描述預載 → 觸發時才載全文 | SKILL_INDEX.md 索引 → 按需 Read |

可借鑑之處

  1. Plugin 化封裝：將 22 個 Skills + 6 個 Hooks 打包為 Plugin，支援跨專案複用
  2. Git Worktree：Phase 2 並行執行 Agent 可改用 worktree 隔離取代 Start-Job
  3. LSP 整合：為 Hook Python 腳本提供即時型別檢查和 diagnostics
  4. Marketplace 分發：將本專案的 Skills 發布為公開 Marketplace
