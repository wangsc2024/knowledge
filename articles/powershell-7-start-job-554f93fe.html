PowerShell 7 非同步作業管理：Start-Job 殭屍防護、Timeout 守衛與 Process Tree Kill

技術概述

PowerShell 7 的非同步作業管理（Start-Job / System.Diagnostics.Process）在 Windows 排程環境中存在三個主要失效模式：殭屍進程殘留、stdout/stderr Pipe 死結、以及 Kill 後子進程未終止。掌握這三類陷阱與對應的防護模式，是建立可靠 Windows 排程系統的基礎工程。

核心概念與原理

1. uv 孤兒問題（已有 issue 確認）

uv.exe 在 Windows 上不使用 Job Object 綁定子進程生命週期（[來源：astral-sh/uv #11817, 2025]），因此：

  • Kill($uvProcess) → uv.exe 終止，但 python.exe 子進程成為孤兒
  • 必須使用 $proc.Kill($true) (entireProcessTree) 或 taskkill /F /T /PID
  • SIGINT/Ctrl+C 傳播也不可靠（[來源：astral-sh/uv #8654, 2024]）

三角佐證：uv #11817（確認）+ uv #8654（SIGINT 不可靠）+ puppetlabs PR #202（相同根因模式）

2. WaitForExit(int) 的兩階段完成語義

WaitForExit(int milliseconds) 返回 true 後，非同步輸出 handler 可能尚未清空 [來源：Microsoft Learn Process.WaitForExit, .NET 10]。

正確模式（需兩步）：


$exited = $proc.WaitForExit(60000)  # 帶 timeout 等待進程退出
if ($exited) {
    $proc.WaitForExit()  # 無參數版：等 BeginOutputReadLine handler 清空
}

若跳過第二步，讀到的 JSON 可能截斷（[來源：PS issue #25262, 2025]）。

三角佐證：.NET 10 文件（明確說明）+ PS issue #25262（實際截斷案例）+ PSScriptAnalyzer #1297（相同症狀）

3. Kill(entireProcessTree:true) 的 Exited 事件 Bug

Kill(entireProcessTree:$true) 在 Windows 上不觸發 Exited 事件（[來源：dotnet/runtime #63328, 2022]），必須改用輪詢：


$proc.Kill($true)
$proc.WaitForExit(5000) | Out-Null  # 輪詢等待，非事件驅動
$proc.Close()

三角佐證：dotnet/runtime #63328（已確認 bug）+ .NET Kill() 文件（說明 Kill 是非同步的）+ phantom-killer（生產環境繞過方案）

4. Pipe 死結防護（Raymond Chen 定律）

OS pipe buffer 預設 ~4KB（Windows）。若父進程在 WaitForExit() 中阻塞而不排空 pipe，當子進程輸出超過 4KB 時雙方死結（[來源：Raymond Chen / The Old New Thing, 2011]）。

唯一可靠修法：BeginOutputReadLine() + BeginErrorReadLine() 必須在 WaitForExit() 之前呼叫：


$proc.BeginOutputReadLine()   # 非同步排空 stdout
$proc.BeginErrorReadLine()    # 非同步排空 stderr
$proc.WaitForExit(60000)      # 現在安全等待

三角佐證：Raymond Chen 2011（根因分析）+ Icinga2 #32（生產修復案例）+ PS issue #25262（現代 PS 7 確認仍有此問題）

5. Start-Job 清理的必要步驟序列

Stop-Job 發信號但不等待 pwsh.exe 終止（[來源：Microsoft Learn about_Jobs, 2025]）。完整清理序列：


Stop-Job $job -ErrorAction SilentlyContinue
Wait-Job $job -Timeout 10 | Out-Null  # 確保 pwsh.exe 已終止
Remove-Job $job -Force

最佳實踐與陷阱

禁止模式

  • & uv run python tools/xxx.py 無 timeout — 會殭屍
  • Start-Process -Wait — PS #25262 確認可無限 hang
  • 順序呼叫 ReadToEnd() 兩次（stdout 後 stderr）— Pipe 死結
  • Kill() 後不呼叫 WaitForExit() — Kill 是非同步的
  • Stop-Job 後不接 Wait-Job — pwsh.exe 仍在運行

必用模式

  • System.Diagnostics.Process + WaitForExit(ms) + Kill(entireProcessTree:$true)
  • BeginOutputReadLine() / BeginErrorReadLine() 在 WaitForExit() 前
  • Two-stage timeout：WaitForExit(primary) → kill → WaitForExit(5000) → Close()
  • 每個 PS wrapper 的 finally block 確保 Kill+Close

與本專案的關聯

daily-digest-prompt 的 Invoke-UvRunWithTimeout（run-todoist-agent-team.ps1）實作了此模式的最佳實踐，解決了 2026-04-19 殭屍事件與 2026-04-20 wrapper hang 的根因。TimeoutSec 建議值：YAML 解析 30s、小工具 45-60s、中任務 60s、大型 workflow 900s。

程式碼範例

Invoke-UvRunWithTimeout 完整實作


function Invoke-UvRunWithTimeout {
    param([string[]]$Arguments, [int]$TimeoutSec = 60, [string]$Label = 'uv-run', [string]$WorkDir = $PSScriptRoot)
    $psi = [System.Diagnostics.ProcessStartInfo]::new()
    $psi.FileName = 'uv'; $psi.Arguments = ($Arguments | ForEach-Object { "'$_'" }) -join ' '
    $psi.WorkingDirectory = $WorkDir; $psi.RedirectStandardOutput = $true
    $psi.RedirectStandardError = $true; $psi.UseShellExecute = $false; $psi.CreateNoWindow = $true
    $proc = [System.Diagnostics.Process]::new(); $proc.StartInfo = $psi
    $stdout = [System.Text.StringBuilder]::new(); $stderr = [System.Text.StringBuilder]::new()
    $proc.add_OutputDataReceived({ param($s,$e) if ($null -ne $e.Data) { $stdout.AppendLine($e.Data) } })
    $proc.add_ErrorDataReceived({  param($s,$e) if ($null -ne $e.Data) { $stderr.AppendLine($e.Data) } })
    $proc.Start() | Out-Null
    $proc.BeginOutputReadLine(); $proc.BeginErrorReadLine()  # 防 pipe 死結
    $exited = $proc.WaitForExit($TimeoutSec * 1000)
    if (-not $exited) {
        Write-Warning "[$Label] Timeout ${TimeoutSec}s — killing tree"
        try { $proc.Kill($true) } catch { }
        $proc.WaitForExit(5000) | Out-Null; $proc.Close(); return $null
    }
    $proc.WaitForExit(); $proc.Close()  # 無參數：清空 async handler
    if ($proc.ExitCode -ne 0) { return $null }
    try { return $stdout.ToString() | ConvertFrom-Json } catch { return $stdout.ToString() }
}
