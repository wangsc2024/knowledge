三、核心洞見：研究品質的四層覺醒

洞見 1：從「找資料」到「評資料」— CRAAP/SIFT 的品質基礎設施革命

觀察：2026-03-20 新增的「來源可信度評估框架完整指南」是一篇超過 5000 字的系統性研究，整合了三大來源評估框架：

  1. CRAAP Test（2004, CSU Chico）：五維度學術評估（Currency、Relevance、Authority、Accuracy、Purpose）
  2. SIFT Method（2017, Mike Caulfield）：四步驟快速事實查核（Stop、Investigate、Find、Trace）
  3. AI 多代理自動化評估管線（2025, Frontiers in AI）：三階段順序管線（Content Analyzer → Fact Verifier → Evidence Assessor）

深層意義：這不僅是一篇研究筆記，而是 Daily-Digest 系統研究元能力的基礎設施升級。系統原本的 web-research Skill 僅有 A/B/C/D 四級粗略分級，現在有了理論基礎來建構更精細的品質評估管線。

跨領域連結：

  • CRAAP 的「縱向深入」vs SIFT 的「橫向比對」，恰好對應天台教觀的「止」（定心一境）與「觀」（正智抉擇）——兩者都是單一方法不足以應對複雜判斷
  • AI 多代理管線的五維加權公式 Score = (Eq×0.4) + (Sc×0.25) + (Ec×0.15) + (Er×0.1) + (Sp×0.1) 與本系統的 config/scoring.yaml 六因子計分有結構同構性，均將多維度評估降維為單一可排序的分數

洞見 2：從「單次搜尋」到「自適應管線」— Agentic RAG 的典範轉移

觀察：「Agentic RAG 深度研究」記錄了從靜態 RAG 管線到自適應控制迴圈的演化五大架構模式。傳統 RAG 是「查詢→檢索→生成」的線性管線，Agentic RAG 則引入了：

  • 查詢改寫與路由（Query Rewriting & Routing）：Agent 評估查詢品質，必要時改寫後再檢索
  • 檢索結果自我評估（Self-Evaluation）：Agent 判斷檢索結果是否足以回答問題
  • 多步推理（Multi-Step Reasoning）：自動拆解複雜問題為子問題序列
  • 工具選擇（Tool Selection）：依問題特性選擇最適檢索策略

與現有系統的映射：

| Agentic RAG 概念 | Daily-Digest 現有對應 | 潛在升級方向 |

|------------------|---------------------|-------------|

| 查詢改寫 | web-research Skill 多角度搜尋 | 自動評估首輪結果品質，動態調整搜尋策略 |

| 自我評估 | 品質自評 JSON（research_depth 判定） | 檢索結果不足時自動追加搜尋 |

| 多步推理 | kb-research-strategist 五階段模型 | 將階段判定自動化，減少人工標記 |

| 工具選擇 | knowledge-query Skill 六種 API | 依查詢類型自動路由到最適 API |

深層意義：Agentic RAG 的核心不是「更好的檢索」，而是「Agent 知道自己不知道什麼」——這正是元認知（metacognition）的定義。系統從「給我答案」演化為「我來判斷這個答案夠不夠好」。

洞見 3：從「驅動力」到「動機品質」— SDT 揭示的研究深度瓶頸

觀察：「自我決定理論（SDT）」筆記提出了動機連續體模型，從無動機（amotivation）到內在動機（intrinsic motivation）共六個等級。核心洞見是：動機的「類型」比「強度」更重要。

對研究系統的啟示：

| 動機類型 | 對應的研究行為 | 產出品質 |

|---------|-------------|--------|

| External regulation（外部調控） | 因排程觸發而執行，最低限度完成 | 淺層摘要，無原創洞見 |

| Introjected regulation（內攝調控） | 因「應該學」的罪惡感驅動 | 篇幅足夠但缺乏深度 |

| Identified regulation（認同調控） | 認識到主題的價值而主動投入 | 有見解但侷限於單一角度 |

| Intrinsic motivation（內在動機） | 純粹的智識好奇心驅動 | 跨領域洞見，原創連結 |

自我指涉的發現：本系統的自動任務中，部分研究任務確實呈現「external regulation」特徵——因 frequency-limits.yaml 的排程驅動，產出品質參差。解決方案不是「增加排程頻率」（提高動機強度），而是「讓 Agent 發現主題之間的內在連結」（提升動機品質）。

這與心流理論（Csikszentmihalyi）的觀點一致：心流狀態需要「挑戰與技能的動態平衡」。過簡單的研究任務（已有大量資料的主題）或過困難的任務（完全陌生的領域）都無法觸發深度研究狀態。

洞見 4：從「事後檢查」到「結構內建品質」— 無作四諦的品質哲學

觀察：「無作四諦」筆記記錄了天台圓教的核心觀點：苦集滅道四諦當體即是中道實相，無需外部修造。「無作」的意思是：系統本自圓成，品質是結構的內在屬性，不是事後檢查的外加屬性。

與 deep-research Skill 品質門檻的對照：

| 品質保證策略 | deep-research 對應 | 佛學對應 | 品質內建程度 |

|------------|-------------------|---------|------------|

| 事後人工審查 | 紅隊多角色審查 | 漸教（次第修行） | 低（外加） |

| 結構性檢查 | validate_report.py 9 項結構檢查 | 別教（歷劫修因） | 中（半內建） |

| 表述即驗證 | Anti-Hallucination「來源錨定」原則 | 圓教無作四諦 | 高（完全內建） |

核心洞見：Anti-Hallucination 協議中「每個事實宣稱在同一句中立即引用來源」的設計，與無作四諦「苦不待斷而自空」的邏輯完全一致——最優的品質保證不是事後檢查（有作），而是結構本身就蘊含正確性（無作）。

實踐建議：將「來源錨定」原則推廣到所有研究模板，讓每個知識點在產出的瞬間就帶著來源標記，而非事後補引用。

四、元認知演化的統一模型

綜合四個洞見，提出「研究品質覺醒四階段」模型：


階段 1：無覺察（Pre-reflective）
  系統只知「找資料」，不評估來源品質
  對應：基礎 RAG（retrieve-augment-generate）
  對應：SDT 的 amotivation

      ↓ 觸發：研究品質不穩定，產出參差

階段 2：外部規範（Rule-based）
  引入品質分級（A/B/C/D）、去重策略、冷卻天數
  對應：CRAAP Test 的規則式檢查
  對應：SDT 的 external/introjected regulation
  Daily-Digest 現狀：已達此階段（web-research Skill、dedup-policy.yaml）

      ↓ 觸發：規則難以涵蓋所有情境，需要判斷力

階段 3：自適應判斷（Agentic）
  Agent 能自我評估檢索結果品質，動態調整策略
  對應：Agentic RAG 的自適應控制迴圈
  對應：SIFT Method 的橫向比對
  對應：SDT 的 identified regulation
  Daily-Digest 進行中：Agentic RAG 研究已完成，待整合

      ↓ 觸發：內在好奇心驅動，超越規則的智慧

階段 4：結構性品質（Intrinsic Quality）
  品質是結構的內在屬性，不需外部檢查
  對應：Anti-Hallucination 的「來源錨定」
  對應：無作四諦「苦集滅道當體即實相」
  對應：SDT 的 intrinsic motivation
  Daily-Digest 目標：讓每個研究產出天然帶有品質證據

五、行動建議

短期（1-2 週）

  1. 來源錨定模板化：修改所有研究模板，要求每個事實宣稱在同一行附帶來源等級標記（如 [A]、[B]）
  2. CRAAP 評分整合：在 web-research Skill 的品質自評 JSON 中新增 craap_scores 欄位，細化 A/B/C/D 分級為五維度數值
  3. Agentic RAG 路由原型：在 knowledge-query Skill 中實驗「依查詢複雜度自動選擇 API 端點」的路由邏輯

中期（1 個月）

  1. SDT 動機品質指標：在 research-quality.json 中追蹤每次研究的「動機類型」，用產出品質（來源數×交叉驗證數×原創洞見數）反推動機品質，作為任務排程的調整依據
  2. SIFT 橫向驗證工作流：研究任務完成後，自動觸發一個輕量級「交叉查核」子 Agent，對關鍵宣稱進行橫向比對

長期（季度）

  1. Anti-Hallucination 全系統化：將「結構內建品質」原則推廣到所有產出文件——不僅研究報告，洞見報告、系統審查報告等也應天然帶有自驗證結構
