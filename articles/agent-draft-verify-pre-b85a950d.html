Agent 執行品質保證機制 — Draft-Verify、Pre-Mortem 與 Post-Mortem 三階段防護策略

產出日期：2026-03-27

研究主題：Agent 執行品質保證的三階段防護機制整合

研究深度：adequate（16 個來源，A 級 6 篇、B 級 8 篇、C 級 2 篇）

---

摘要

本簡報整合業界最佳實踐，提出 Agent 執行品質保證的三階段防護策略：Draft-Verify（預防性驗證）、Pre-Mortem（主動失敗預測）與 Post-Mortem（系統化事後分析）。研究發現，三階段整合運作可形成完整防護鏈：Draft-Verify 提升即時準確率達 30%+，Pre-Mortem 的 FMEA 方法能早期偵測缺陷，Blameless Post-Mortem 促進持續學習與心理安全。關鍵整合點在於自動化工具支援、平衡混合防護層級，以及建立「What/How」為主的無責問句框架。

---

關鍵洞見

1. Draft-Verify 模式：即時驗證的雙 Agent 協作與 Reflection 機制

  • 雙 Agent 協作：Agent A（Writer）產出草稿 → Agent B（Editor）進行驗證
  • Reflection 模式：Agent 自我批判可將 HumanEval 表現從 80% 提升至 91%
  • Verification-Aware Planning：將檢查變成一等任務（first-class tasks）

2. Pre-Mortem / FMEA：假設失敗的主動風險管理

  • 假設失敗，向後推導：團隊假設專案失敗，識別潛在原因
  • FMEA：結構化腦力激盪、風險評級與排序，早期偵測缺陷
  • 從被動到主動：轉變為主動風險管理策略

3. Blameless Post-Mortem：系統聚焦與自動化根因分析

  • 無責假設：假設每個人都有良好意圖
  • 系統性分析：多重貢獻因子而非單一根因
  • 問句框架：What/How > Why（避免追責）
  • 自動化工具：Google、Rootly 等已自動化 postmortem 建立

4. 三階段整合：Preventive、Detective、Corrective 的平衡混合

  • Preventive Controls：Draft-Verify、Pre-Mortem（執行前與執行中驗證）
  • Detective Controls：Circuit Breaker、Guardian 系統（執行中異常偵測）
  • Corrective Controls：Post-Mortem（執行後修復與預防）
  • 平衡混合：即使一層失效，其他層仍能降低風險

---

建議行動

短期（1-2 週）

  1. 在高風險 Agent 任務中引入 Reflection 步驟，目標準確率提升 15-30%
  2. 執行 FMEA 分析，建立 Pre-Mortem 檢查清單（config/pre-mortem-checklist.yaml）

中期（1 個月）

  1. 開發 tools/postmortem_generator.py，自動化 Post-Mortem 流程
  2. 整合 agent_guardian.py 為 Detective 控制層

長期（2-3 個月）

  1. 建立三階段整合儀表板，視覺化 Preventive/Detective/Corrective 指標
  2. 進入 mechanism 階段深化，建立 skills/quality-gate-orchestrator/SKILL.md
