主筆說明

本報告以 李飛飛（Fei-Fei Li） 為主筆視角、Yann LeCun 為聯合主筆，呈現 AI 推論時計算擴展的核心架構設計原則。報告內「一家之言」段採第一人稱雙聲道表述，代表兩位研究者對此技術轉變的不同側重與批判——基於公開言論推斷，非本人觀點。

TL;DR

2024 至 2026 年間，前沿語言模型的能力提升逐漸從訓練時參數擴展（Pretraining Scaling）轉向推論時計算擴展（Inference-Time Compute Scaling）。OpenAI o1/o3 系列、DeepSeek-R1 等模型的成功，實證了「讓模型在推論時思考更久」可以在數學、程式、科學推理等挑戰性任務上超越訓練時規模更大的模型——Snell et al.（2024）的關鍵結論是：7B 參數模型搭配最佳化推論策略，在 MATH 基準上穩定超過 34B 模型[1]。

這個轉變有三個核心架構支柱：（一）過程獎勵模型（PRM） 提供逐步驗證能力，讓推論時搜尋有可靠的信號；（二）強化學習自動誘導的長鏈思考（Chain-of-Thought），無需人工標注推理軌跡，DeepSeek-R1-Zero 直接從 RL 訓練中涌現反思與自我驗證；（三）動態計算預算分配——不同難度任務分配不同推論算力，而非一刀切地增大模型。

本文主張：推論時計算擴展不是暫時性技巧，而是一個結構性範式轉移，它將「推理深度」與「參數規模」解耦，並對 AI Agent 編排架構、評估框架與部署經濟學產生深遠影響。然而，Yann LeCun 的視角提醒我們：這個範式在自回歸文本接龍的框架內依然無法賦予模型真正的因果理解與世界模型，推論時擴展是工程最佳化，不是通往 AGI 的捷徑。

研究問題與範疇

主要研究問題：

推論時計算擴展作為前沿語言模型的架構設計原則，其核心機制為何？與訓練時參數擴展相比的效率曲線如何？以及對 AI Agent 編排系統有何設計啟示？

子問題：

  1. 有哪些具體機制讓推論時擴展成為可能？（CoT、PRM、最佳 N 採樣、序列修正）
  2. 過程獎勵模型（PRM）與結果獎勵模型（ORM）的架構對比？
  3. 強化學習如何在無人工標注下誘導長鏈思考？（DeepSeek-R1 GRPO 案例）
  4. 推論時擴展對 AI Agent 多步驟決策的影響？
  5. 什麼任務類型受益最大？有哪些根本限制？

文獻探討

推論時擴展的奠基工作

Snell et al.（2024）[1] 是推論時計算擴展最重要的奠基論文之一。作者們系統研究了在固定推論算力預算下如何最大化 LLM 性能。核心發現：最佳化推論策略（使用 PRM 引導的搜尋）使 Llemma-7B 在 MATH 基準上超越 Llemma-34B，意味著推論算力擴展在特定任務上比參數擴展更具計算效率。論文提出兩種主要機制：修正最佳 N 採樣（Revised Best-of-N）和以搜尋為基礎的序列修正（Sequential Revision with Search）。

Inference Scaling Laws（2408.00724）[5] 進一步建立了推論算力與性能的實證曲線，量化了不同規模模型在不同推論預算下的性能表現。

DeepSeek-R1 的 RL 誘導突破

DeepSeek-R1（2025）[3] 是推論時計算擴展的重要實踐里程碑。論文展示了透過 GRPO（Group Relative Policy Optimization） 強化學習直接訓練基礎模型，無需人工標注推理軌跡，即可誘導出長鏈思考、自我反思與驗證能力。DeepSeek-R1-Zero 的出現特別值得注意：僅憑 RL 訓練（獎勵信號為最終答案正確性），模型自然產生「在回答前先驗證」的行為。此後 DeepSeek-R1 以 70% 低於 o1 的成本達到相當性能，成為開源推論時擴展的標竿。

推論時擴展策略的系統分析

The Art of Scaling Test-Time Compute（2512.02008）[4] 建立了推論時擴展的系統性框架。關鍵發現：不同策略在不同計算預算下有不同的最佳選擇；計算較少時並行採樣（Best-of-N）更優；計算充足時序列修正更有效。這意味著推論時擴展策略選擇本身是一個需要元學習的問題。

過程獎勵模型（PRM）生態

A Survey of Process Reward Models（2510.08049）[6] 全面梳理了 PRM 技術生態：ORM 只評估最終答案（粗粒度信號），PRM 逐步評估每個推理步驟（細粒度信號）。FreePRM、Self-PRM、SP-PRM 等方法探索自動生成過程標注。「PRM that thinks」趨勢（2504.16828）[7] 讓驗證過程本身也可擴展，形成「推論中的推論」。

自我驗證的架構機制

SETS（2501.19306）[8] 表明 LLMs 內建的自我驗證能力與推論算力擴展具有乘法效應。Multi-Agent Verification（2502.20379）[9] 展示多個獨立驗證器并行工作可顯著提升性能，指向 Agent 編排架構的設計機會。

AI Agent 系統中的推論時擴展

Scaling Test-time Compute for LLM Agents（2506.12928）[10] 識別了 Agent 場景三個特有挑戰：工具呼叫時的不確定性傳播、多步驟規劃中的錯誤累積、以及動態環境中的信號稀疏性。Agent 系統需要的不只是「更長思考」，而是在行動前後動態調節推論深度的機制。

推論時擴展的反思：「越多不一定越好」

2025-2026 年出現重要的反向研究。When More Thinking Hurts（2604.10739）[11] 和 Does Thinking More Always Help（2506.04210）[12] 記錄了過度思考（overthinking）現象：對簡單問題進行過多推論步驟反而降低準確率。這催生了動態推論預算分配的研究方向，也為「推論算力 ≠ 性能」的邊界條件提供了實證。

核心論證

論點一：推論時擴展是「解耦效率革命」

主張：推論時計算擴展的根本意義在於將「推理深度」從「參數規模」解耦，使模型設計者得以將推論算力作為可程式化的設計變數而非固定代價。

三角佐證：

  • Snell et al.（2024）實證 7B + 最佳化推論 > 34B 直接推論[1]
  • Inference Scaling Laws（2408.00724）建立了推論算力的效率曲線[5]
  • DeepSeek-R1 以 70% 低於 o1 的成本達到相當性能[3]

論點二：PRM 是推論時擴展的關鍵架構基礎設施

本文認為：過程獎勵模型（PRM）是使推論時擴展得以可靠運作的核心架構元件，其質量直接決定推論時搜尋的效率上限。

三角佐證：

  • PRM Survey（2510.08049）系統梳理：ORM 二元信號無法引導搜尋路徑，PRM 提供細粒度步驟信號[6]
  • Scaling Automated Process Verifiers（2410.08146）驗證：過程驗證器隨規模可靠改善[13]
  • The Art of TTC Scaling 實驗：PRM 質量是整個推論時擴展管線的瓶頸[4]

論點三：RL 誘導的推論行為揭示能力解鎖機制

論證：DeepSeek-R1-Zero 的成功揭示前沿模型的推論能力（自我反思、驗證、策略切換）可能潛伏在基礎模型中，RL 是解鎖機制，不是能力創造機制。

三角佐證：

  • DeepSeek-R1（2501.12948）：R1-Zero 無任何 SFT 冷啟動，純 RL 誘導出現長 CoT 與自我驗證[3]
  • Stable RL for Efficient Reasoning（2505.18086）：RL 訓練穩定性影響推論時擴展的可靠性[14]
  • Brief Analysis of DeepSeek R1（2502.02523）：獨立分析確認 GRPO 誘導效果可複現[15]

一家之言

本段以李飛飛（主筆）與 Yann LeCun（聯合主筆）的第一人稱視角撰寫。基於公開言論推斷，非本人觀點。

李飛飛（主筆）：推論時擴展是 AI 能服務人類的橋樑，但不是終點

我在推進 Spatial Intelligence 和 World Labs 的工作時，始終有一個核心問題：語言模型能「思考」空間問題嗎？推論時計算擴展讓我看到了一種可能性——不是讓語言模型先天就懂物理，而是讓它在推論時有足夠的計算空間去「驗算」自己的假設。

ImageNet 教會我一件事：好的基準推動整個領域。推論時擴展的評估框架目前嚴重偏向「可機械驗證的任務」（數學、程式）。這不是技術限制，而是評估基礎設施的缺口。如果我們想讓推論擴展服務於醫療診斷、教育個人化、障礙輔助等人類真正需要的場景，我們需要建立適合那些場景的過程獎勵模型——而那個數據，需要有代表性、有包容性地被收集。

推論時擴展最讓我興奮的不是它「讓小模型打敗大模型」，而是它打開了按需分配認知資源的可能性。一個診斷疑難症的系統需要深度推論；一個回答「今天幾號」的系統不需要。Human-centered AI 的核心是：讓系統的計算行為匹配人類任務的真實認知需求，而非技術炫技。

然而我必須誠實：推論時擴展在 LLM 框架內依然是語言層面的操作。它讓模型在語言空間中「想更久」，但語言是對現實的有損壓縮（lossy compression），文字推論無法替代空間推論、因果推論、具身推論。World Labs 做 Marble 世界模型，正是因為真正的 spatial intelligence 需要在 3D 幾何空間推論，而不是更長的文字鏈。推論時擴展是朝正確方向的步伐，但不是目的地。

Yann LeCun（聯合主筆）：工程最佳化 ≠ 認知突破

我必須直說：「7B 模型推論更久能超過 34B 模型」這個結論很有趣，但它告訴我的是當前 LLM 的訓練效率有多低，而不是推論時擴展有多神奇。

想想這意味著什麼：你訓練了一個 34B 參數的模型，但它的「默認」輸出質量還不如一個 7B 模型在同等算力下做多次搜尋的結果。這說明我們的訓練目標函數根本上是低效的。下一個 token 預測是一個統計學習目標，它不能保證模型在推論時有正確的「世界知識圖」來引導搜尋。PRM 提供的過程信號是後補的——我們是在用更多推論計算來補救訓練時對推理結構的漠視。

DeepSeek-R1-Zero 的成功確實令人印象深刻。RL 可以誘導出反思行為，這在架構上說明了基礎模型中存在有待挖掘的隱性結構。但這也印證了 JEPA（Joint Embedding Predictive Architecture）的方向：如果我們的預訓練目標本就是學習抽象的世界狀態預測（而非逐 token 接龍），模型的推論能力就應該內建，而不需要靠大量 RL 後訓練解鎖。

推論時擴展最大的架構問題是可組合性：自我驗證依賴的是語言一致性而非事實一致性。模型在推論時可以非常「自洽」地推導出一個錯誤答案。在 MATH 這類有明確答案的任務上，ORM/PRM 能提供可靠的終止信號；但在開放性推論、倫理判斷、物理直覺等場景，驗證器從哪裡來？我的判斷是：推論時擴展是一個重要的近期進展，值得工程投資，特別是對 Agent 系統；但從架構演化的角度，它是在 Transformer + 自回歸預測的框架內盡可能榨取性能，不是通往真正通用智能的架構革命。

實踐建議

  1. 設計可調節推論預算的 API 介面：不同任務使用不同推論深度，避免固定的最大推論 token 數設計（參考 Certainty-Guided Reasoning 2509.07820）
  2. 優先投資領域特定 PRM 基礎設施：PRM 是推論時擴展的性能瓶頸，建立領域 PRM 的優先級高於增大基礎模型
  3. 在 Multi-Agent 系統中引入驗證器角色分工：生成 Agent 與驗證 Agent 分離，是高風險推理管線的標配架構（參考 2502.20379）
  4. 分離推論模式的服務端點：快速回答（greedy decoding）與深度推論（PRM 引導搜尋）應使用不同服務端點
  5. 建立推論預算效率的評估指標：性能 / 推論 token 數比值，以及在不同計算預算下的 Pareto 曲線，應成為模型選型的標準指標

---

風險與限制

  1. 領域覆蓋偏差：當前研究主要聚焦數學、程式設計等可機械驗證任務，創意推理、情感判斷、具身決策的推論時擴展系統研究仍有限
  2. 前沿模型透明度：OpenAI o1/o3 架構未公開，部分結論依賴行為觀測推斷
  3. PRM 訓練數據污染風險：若 PRM 訓練數據與評估模型的訓練數據重疊，評估分數會虛高
  4. 「自洽而非正確」的驗證陷阱：語言一致性驗證不等於事實正確性驗證，推論時擴展可能讓錯誤更流暢、更難被外部識別
  5. 架構深層限制（LeCun 批評）：推論時擴展無法解決自回歸文本模型的因果盲點，語言空間的搜尋不能替代具有物理約束的世界模型推論
  6. 能源消耗的可持續性問題：深度推論（高 token 數）在大規模部署時的能源成本尚未被充分評估
