以 Rust 重寫 RAG 知識庫系統

研究日期：2026-03-19 | 目標：仿 RAG_Skill（Node.js/Express）架構，以 Rust 實現高效能知識庫系統，port 8020，整合 LlamaIndex 等效功能與 OpenAI Function Calling

---

一、概述與動機

現有 RAG_Skill 系統採用 Node.js + Express + SQLite + Qdrant + HuggingFace Embedding 架構，port 3000。本研究規劃以 Rust 重寫，目標：

  1. 效能提升：Rust 的零成本抽象與記憶體安全，搜尋延遲可降低 2-5 倍
  2. 資源節省：無 GC 停頓，記憶體占用更低
  3. 類型安全：編譯期捕獲更多錯誤
  4. OpenAI 模型：LLM 採 gpt-4o-mini、Embedding 採 text-embedding-3-small
  5. Function Calling：實現 LlamaIndex 等效的工具呼叫能力

3.2 啟動流程


#[tokio::main]
async fn main() -> Result<()> {
    tracing_subscriber::init();
    let config = Config::from_env()?;
    let note_store = NoteStore::new(&config.data_dir).await?;
    let qdrant = QdrantService::new(&config.qdrant_url, &config.collection).await?;
    let openai = async_openai::Client::new();
    
    let state = AppState { note_store, qdrant, openai, config };
    
    let app = Router::new()
        .nest("/api", api_routes())
        .layer(CorsLayer::permissive())
        .with_state(state);
    
    let addr = SocketAddr::from(([0, 0, 0, 0], 8020));
    axum::serve(TcpListener::bind(addr).await?, app).await?;
    Ok(())
}

四、核心服務實作

4.1 Embedding 服務（OpenAI text-embedding-3-small）


use async_openai::types::{CreateEmbeddingRequestArgs, EmbeddingInput};

pub async fn embed_texts(client: &Client<OpenAIConfig>, texts: &[String]) -> Result<Vec<Vec<f32>>> {
    let request = CreateEmbeddingRequestArgs::default()
        .model("text-embedding-3-small")
        .input(EmbeddingInput::StringArray(texts.to_vec()))
        .build()?;
    let response = client.embeddings().create(request).await?;
    Ok(response.data.into_iter().map(|e| e.embedding).collect())
}

4.2 Qdrant 向量操作


use qdrant_client::prelude::*;
use qdrant_client::qdrant::vectors_config::Config;
use qdrant_client::qdrant::{CreateCollection, VectorParams, Distance};

pub async fn create_collection(client: &QdrantClient, name: &str) -> Result<()> {
    client.create_collection(&CreateCollection {
        collection_name: name.to_string(),
        vectors_config: Some(VectorsConfig {
            config: Some(Config::Params(VectorParams {
                size: 1536, // text-embedding-3-small 維度
                distance: Distance::Cosine.into(),
                ..Default::default()
            })),
        }),
        ..Default::default()
    }).await?;
    Ok(())
}

4.3 Function Calling 實作（LlamaIndex 等效）

async-openai 支援 OpenAI 的 tool use / function calling 機制：


use async_openai::types::{
    ChatCompletionRequestMessage, ChatCompletionToolArgs,
    ChatCompletionToolType, FunctionObjectArgs,
    CreateChatCompletionRequestArgs,
};

// 定義工具
let tools = vec![
    ChatCompletionToolArgs::default()
        .r#type(ChatCompletionToolType::Function)
        .function(
            FunctionObjectArgs::default()
                .name("search_knowledge_base")
                .description("搜尋知識庫中的相關筆記")
                .parameters(serde_json::json!({
                    "type": "object",
                    "properties": {
                        "query": { "type": "string", "description": "搜尋查詢詞" },
                        "top_k": { "type": "integer", "description": "返回結果數量" }
                    },
                    "required": ["query"]
                }))
                .build()?
        )
        .build()?,
    ChatCompletionToolArgs::default()
        .r#type(ChatCompletionToolType::Function)
        .function(
            FunctionObjectArgs::default()
                .name("get_note_by_id")
                .description("根據 ID 取得筆記完整內容")
                .parameters(serde_json::json!({
                    "type": "object",
                    "properties": {
                        "note_id": { "type": "string" }
                    },
                    "required": ["note_id"]
                }))
                .build()?
        )
        .build()?,
];

// Agent 迴圈
let request = CreateChatCompletionRequestArgs::default()
    .model("gpt-4o-mini")
    .messages(messages.clone())
    .tools(tools.clone())
    .build()?;

let response = client.chat().create(request).await?;
let choice = &response.choices[0];

if let Some(tool_calls) = &choice.message.tool_calls {
    for tool_call in tool_calls {
        let result = match tool_call.function.name.as_str() {
            "search_knowledge_base" => {
                let args: SearchArgs = serde_json::from_str(&tool_call.function.arguments)?;
                search_kb(&qdrant, &openai, &args.query, args.top_k).await?
            }
            "get_note_by_id" => {
                let args: GetNoteArgs = serde_json::from_str(&tool_call.function.arguments)?;
                get_note(&note_store, &args.note_id).await?
            }
            _ => "未知工具".to_string(),
        };
        // 將工具結果加入 messages 繼續對話
        messages.push(ChatCompletionRequestMessage::Tool { tool_call_id, content: result });
    }
    // 再次呼叫 LLM 取得最終回應
}

六、Rust RAG 框架比較

6.1 Rig 框架

  • 特色：trait 抽象 LLM Provider，支援 OpenAI/Cohere/Anthropic 一行切換
  • 向量整合：In-memory、MongoDB、SQLite、Qdrant（via rig-contrib）
  • RAG 管線：EmbeddingsBuilder + context_rag_agent() 自動處理檢索
  • 限制：Function Calling 尚未完全實作（部分 todo!()）
  • 適用：快速原型、多 Provider 切換場景

6.2 Swiftide 框架

  • 特色：Rust 原生 RAG 管線，串流式處理
  • 索引管線：Pipeline::from_loader().then_in_batch(Embed).then_store_with(vectorstore)
  • 查詢管線：支援 RAGAS 評估、LanceDB/Qdrant 整合
  • 適用：大量文件索引、需要評估框架的場景

6.3 自行組合（推薦本專案）

  • 理由：完全控制架構，與 RAG_Skill API 對齊，Function Calling 自行實作
  • 組合：axum + async-openai + qdrant-client + rusqlite
  • 優勢：無框架學習成本，API 端點一一對應

七、文件分塊策略


pub fn chunk_text(text: &str, chunk_size: usize, overlap: usize) -> Vec<String> {
    let paragraphs: Vec<&str> = text.split("\n\n").filter(|p| !p.trim().is_empty()).collect();
    let mut chunks = Vec::new();
    let mut current = String::new();
    
    for para in paragraphs {
        if current.len() + para.len() > chunk_size && !current.is_empty() {
            chunks.push(current.clone());
            // 保留 overlap
            let overlap_text = &current[current.len().saturating_sub(overlap)..];
            current = overlap_text.to_string();
        }
        if !current.is_empty() { current.push_str("\n\n"); }
        current.push_str(para);
    }
    if !current.is_empty() { chunks.push(current); }
    chunks
}

八、混合搜尋 (Hybrid Search) 實作

RRF（Reciprocal Rank Fusion）結合語義與關鍵字搜尋：


pub fn rrf_merge(semantic: Vec<SearchResult>, keyword: Vec<SearchResult>, k: f32) -> Vec<SearchResult> {
    let mut scores: HashMap<String, f32> = HashMap::new();
    for (rank, result) in semantic.iter().enumerate() {
        *scores.entry(result.note_id.clone()).or_default() += 1.0 / (k + rank as f32 + 1.0);
    }
    for (rank, result) in keyword.iter().enumerate() {
        *scores.entry(result.note_id.clone()).or_default() += 1.0 / (k + rank as f32 + 1.0);
    }
    // 排序並返回
    let mut merged: Vec<_> = scores.into_iter().collect();
    merged.sort_by(|a, b| b.1.partial_cmp(&a.1).unwrap());

}
