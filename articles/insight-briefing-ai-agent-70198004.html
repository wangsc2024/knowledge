AI Agent I/O 預算工程：per-call Context 控制、量化監控與動態壓縮策略（2026）

日期：2026-05-23

任務：insight_briefing

主題：AI Agent Token 效率工程——per-call I/O 預算控制、量化監控與動態壓縮策略 2026

驅動因素：本系統 avg_io_per_call=20,109（超標準值 12,000 約 67%）；cache_hit_ratio=0.283（低於 0.4 警戒線）

摘要

AI Agent 系統中，Context 膨脹是隱性成本的主要來源：每次呼叫的輸入 token 成本隨 Context 累積線性增長，而工具輸出、歷史對話、系統提示三者合計往往佔用 70% 以上的 per-call 預算。2025–2026 年業界已發展出五條可量化的干預路徑：（1）工具輸出過濾（67% 削減）、（2）階層式記憶壓縮（60–80%）、（3）提示快取（50–90% 成本降幅）、（4）語義快取（73% 高重複工作流）、（5）動態閾值監控（三級預警 70%→85%→90% 觸發自動緊縮）。

一、問題根因：Context 膨脹的三個驅動因素

1.1 工具輸出冗餘

工具回應是 per-call 預算最大的殺手。若 Agent 使用 46 個工具、每個工具回應平均 150 token，單次工具輸出即佔 6,900 token；過濾至 15 個相關工具後降至 2,250 token（削減 67%）。關鍵原則：在接收工具回應時立即過濾，而非推遲到壓縮階段。

1.2 對話歷史累積

無優化的 20 輪對話累積 5,000–10,000 token；加入 BufferWindow 滑動窗口（保留近 5 輪完整）+ 歷史壓縮（3:1 至 5:1 比例）後降至 500–1,000 token（削減 90%）。

1.3 靜態系統提示未快取

靜態內容本應達到 95%+ 快取命中率；而本系統 cache_hit_ratio=0.283 顯示快取利用率嚴重不足。

二、五項核心洞見

洞見 1：工具輸出過濾效益最大，應為第一優先

  • 生產建議壓縮比：工具輸出 10:1 到 20:1
  • 工具數量管理：每次呼叫注入不超過 15 個相關工具
  • avg_io_per_call=20,109；若工具輸出佔 60%，僅此項即可降至 ~12,000

洞見 2：三層記憶架構是 Agent 長時間運行的必需基礎設施

  • 短期（最近 3–5 輪）：完整保留逐字內容
  • 中期（3–10 輪前）：摃桿萃取式摘要壓縮（3:1 至 5:1）
  • 長期（10 輪前）：只保留提取的事實與決策
  • Auto Compaction 觸發後平均縮小 60–80%

洞見 3：提示快取是 cost/call 最高 ROI 的優化項

  • Prompt Cache 命中：成本降低 50–90%
  • 靜態段落可達 95%+ 命中率
  • 快取命中率 40% 目標：增加 static prefix 長度 + 減少 session 間靜態內容變動

洞見 4：動態閾值監控（三級預警）優於靜態 hard-cap

  • 70%：警示 → 85%：強烈警示 → 90%：自動緊縮
  • 建議 P95 per-call 超過 15,000 → warning；超過 20,000 → 觸發 context-compressor Skill

洞見 5：語義快取對高重複查詢有 73% 成本降幅

  • 語意等效查詢命中快取，毫秒 vs 秒回應
  • 高重複工作流（health-check / log-query）成本削減 73%

三、Context Utilization Rate：被忽略的核心指標

  • Context Utilization Rate 目標：60–80%（低於 60% = 過度配置；高於 80% = 容量風險）
  • 模型實際有效利用率僅 30–60%（「lost in the middle」問題）
  • 量化指標組合：avg_io_per_call + P95 per-call + cache_hit_ratio + context_utilization_rate

四、建議行動

| 優先級 | 行動 | 預期效果 | 難度 |

|--------|------|---------|------|

| P0 | 工具回應接收時立即截斷 | avg_io_per_call -40~60% | 中 |

| P0 | 靜態系統 prompt 前綴固定化 | cache_hit_ratio +20~40% | 低 |

| P1 | 實施 BufferWindow（最近 5 輪完整 + 歷史摘要） | 長時間 session token -70% | 中 |

| P1 | avg_io_per_call 趨勢告警（15k→warning, 20k→auto-compact） | 預防性干預 | 低 |

| P2 | 語義快取層（高重複 health-check / log-query） | per-query cost -73% | 高 |

| P2 | 三層記憶架構 | 長期 Agent cost -60~80% | 高 |
