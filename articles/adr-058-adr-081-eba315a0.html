📚 主要發現：

  1. 編號碰撞：context/adr-registry.json 內 ADR-20260324-058／081 為公平排程／WRR；本任務的 058／081 為 docs/adr 文件序號，已在 ADR 正文與 registry 新條目標明並分開。
  2. 旗標策略：在無即時遠端開關需求下，環境變數 + 集中 features 模組 比 LaunchDarkly／Unleash 更省依賴與維運；FEATURE_NEW_UI 僅 === 'true' 才啟用，避免跨平台字串歧義。
  3. 日誌：pino 10.3.1（npm 與 npm 套件頁 佐證）在 Node LTS 上成熟，JSON 至 stdout 利於容器收集；搭配 redact + remove: true 處理授權標頭等敏感路徑。
  4. 測試：透過清除 features／routes 的 require 快取，整合測試能對齊當前 process.env.FEATURE_NEW_UI；CI 以 三次 vitest（預設、true、false）滿足矩陣。
  5. 知識庫 API：已在本機對 http://localhost:3000/api/notes 執行 Invoke-RestMethod（依 knowledge-query 流程）；回應無錯誤但無可展示之筆記資料，未進行知識庫寫入。

💾 知識庫：未寫入外部知識庫；可自行以關鍵字 ADR-058 feature flag、ADR-081 pino structured logging、FEATURE_NEW_UI 檢索本 repo 之 docs/adr/、bot/src/。
