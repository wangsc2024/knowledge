Daily Digest Hooks 系統優化報告 (2026-02-24)

專案優化hookspython程式碼品質測試daily-digest

Hooks 系統深度優化報告

優化概覽

執行日期：2026-02-24

影響範圍：5 個 Python 模組 + 2 個測試檔

測試結果：495 個測試全數通過（從 496 降為 495 因合併重複測試案例）

O1: cjk_guard.py 重複映射清理

  • 問題：CJK_CORRECTIONS 列表有重複條目（0x8FBA 出現兩次）、3 個 no-op 條目（jp == tc）、以及運行時過濾的開銷
  • 修正：將 list + dict comprehension 重構為直接 dict 定義，移除重複與 no-op，刪除未使用的 import re
  • 效益：減少 ~30 行程式碼，消除運行時過濾開銷，映射從 15 → 12 個有效條目

O2: validate_config.py DRY 優化

  • 問題：_apply_add_field 中 dict 分支和 list 分支有 ~40 行完全相同的 value_strategy 邏輯
  • 修正：抽取 _resolve_field_value(item, index, transformation) 共用函數
  • 效益：消除重複邏輯，未來新增 value_strategy 只需改一處

O3: on_stop_alert.py reserved_bytes 修正

  • 問題：build_alert_message 呼叫 _get_skill_diff() 時未傳入 reserved_bytes，可能導致 ntfy 訊息超過 4096 byte 上限
  • 修正：在 info 級別和 warning/critical 兩條路徑都正確計算已用位元組數並傳入
  • 效益：防止 ntfy 訊息被截斷，確保 SKILL.md 修改 diff 資訊完整顯示

O4: agent_guardian.py 正則快取整合

  • 問題：ErrorClassifier 的 _extract_http_status、_extract_retry_after、_is_whitelisted 每次都重新編譯正則
  • 修正：整合 hook_utils.get_compiled_regex（含獨立 fallback），3 處 re.search 改用快取版本
  • 效益：每日 44 次 Agent 執行 × 20-100 次工具呼叫，正則編譯從 O(n) 降至 O(1)（快取命中後）

O5: post_tool_logger.py 字串匹配精確化

  • 問題：classify_bash 使用 bare in 匹配（如 curl in command），可能誤匹配子字串（如 curly、procure）
  • 修正：新增 _cmd_has_word 輔助函數，檢查詞邊界（command 開頭或空格/管道/分號後）
  • 效益：提升日誌標籤準確度，避免非 API 呼叫被標記為 api-call

測試修正

  • test_cjk_guard.py：更新 import 與測試案例以適配新的 CORRECTIONS dict 結構
  • test_on_stop_alert.py：修正 test_trims_old_session_summaries 使用相對日期（timedelta），避免日期過期導致測試失敗

技術洞察

  • Python dict 覆蓋語義：重複 key 的 dict 定義合法但最後值生效，清理重複可避免混淆
  • 正則快取策略：模組層級快取比 lru_cache 更適合跨函數共享同一 pattern
  • ntfy byte limit：4096 bytes 是 ntfy 正文上限，含 UTF-8 多位元組字元時需用 encode utf-8 精確計算
  • pytest 日期陷阱：硬編碼日期會在超過 retention_days 後導致測試失敗，應用 datetime.now + timedelta
