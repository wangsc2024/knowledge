專案分析

1. Langfuse (25,200 stars)

GitHub: https://github.com/langfuse/langfuse

描述: 開源 LLM 工程平台，提供可觀測性、prompt 管理、評估與除錯功能

核心特性:

  • 統一平台整合 observability、prompt mgmt、evals、datasets、playground
  • Session-based tracing，追蹤跨多輪對話的 context
  • Prompt 版本控制與 A/B testing
  • LLM-as-judge evaluations，自動化品質評估
  • 支援 self-hosted（Docker/K8s），資料留在本地

Guardrails: 輸入層 prompt 版本控制與審查，輸出層評估與標記

Tool Boundary: SDK 追蹤 LLM 呼叫，信任開發者主動埋點，無強制攔截

Operator Override: Web UI 手動標記與評估介面（review-only mode），版本控制允許回退

2. Arize Phoenix (9,400 stars)

GitHub: https://github.com/Arize-ai/phoenix

描述: 開源 AI 可觀測性平台，基於 OpenTelemetry 標準提供追蹤、評估與實驗管理

核心特性:

  • OpenTelemetry-native instrumentation，產出標準 OTEL traces/spans
  • 明確支援 Claude Agent SDK（與本系統高度相關）
  • 內建 trace 可視化 UI，查看 LLM 呼叫鏈路
  • 整合 evaluation framework，支援自訂 evaluators
  • 完全開源且 self-hostable

Runtime Model: OpenTelemetry trace/span 模型，事件驅動追蹤

State Strategy: trace context 跨 span 傳遞，dataset 與 experiment 持久化儲存

Human Review: Playground 互動式測試與評估標記介面

Tool Boundary: OTel instrumentation 追蹤 LLM 與工具呼叫，信任框架層埋點正確

3. AgentGateway (2,400 stars)

GitHub: https://github.com/agentgateway/agentgateway

描述: Linux Foundation 專案，提供 Agent 通訊的安全、可觀測性與治理代理層

核心特性:

  • LLM Gateway：統一 API 路由（OpenAI、Anthropic、Gemini、Bedrock）
  • MCP Gateway：透過 Model Context Protocol 連接工具
  • A2A Gateway：安全的 agent-to-agent 通訊
  • 多層 Guardrails：regex、OpenAI moderation、AWS Bedrock、custom webhooks
  • OpenTelemetry 整合，JWT/API key 認證，RBAC with CEL policy engine

Guardrails: 輸入層（regex、moderation）+ 規劃層（RBAC、rate limiting）多層防護

Tool Boundary (雙層):

  • 執行環境邊界: Proxy 不控制 Agent 執行環境，信任上游 Agent 遵守協定
  • 工具 API 邊界: MCP Gateway 控制工具存取，OAuth + CEL policy 細粒度授權

Operator Override: CEL policy engine 配置檔與 API（deny mode），policy 版本控制允許回退
