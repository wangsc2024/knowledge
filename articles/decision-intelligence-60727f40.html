決策智慧（Decision Intelligence）— 從認知偏誤去偏到結構化決策的系統方法

一、概述

決策智慧（Decision Intelligence, DI）是由 Google 前首席決策科學家 Cassie Kozyrkov 在 2017-2018 年間正式提出並推廣的跨領域方法論。其核心定義為：「將資訊轉化為更好行動的藝術與科學，適用於任何情境與任何規模」。DI 整合了資料科學、人工智慧與人類判斷力，強調倫理實踐，避免過度依賴演算法。

2025-2026 年，Gartner 將決策智慧列為組織智慧化的關鍵趨勢，IBM、Google 等企業均推出 DI 平台。Kozyrkov 現為 Kozyr 公司 CEO，為 NASA、Gucci、Salesforce 等機構提供 AI 決策諮詢。

---

二、Kozyrkov 決策智慧四步框架

步驟 1：問題框定（Problem Framing）

  • 定義決策情境、預設行動、所需資訊和風險容忍度
  • 防止「第三類錯誤」（Type Three Error）：精準地解決了錯誤的問題
  • 關鍵問句：「如果我什麼都不做，會怎樣？」

步驟 2：選項生成（Option Generation）

  • 使用探索性資料分析（EDA）在獨立的靈感數據集上進行
  • 平衡創意與嚴謹，避免確認偏誤
  • 技巧：腦力激盪 + 結構化分解（MECE 原則）

步驟 3：評估（Evaluation）

  • 依據預先定義的成功指標進行統計推論或機器學習模型評估
  • 區分三種決策類型：

- Type 1（不可逆）：需深度分析，如組織重組

- Type 2（可逆）：快速決策即可，如 A/B 測試

- Type 0（無需決策）：已有明確最優選項

步驟 4：監控（Monitoring）

  • 決策實施後持續驗證與績效追蹤
  • 建立回饋迴路，確保決策品質隨時間改善
  • 方法論優先於技術產物（模型準確率不等於業務影響力）

三、認知偏誤去偏技術（Cognitive Debiasing）

認知偏誤是系統性偏離理性判斷的模式。去偏技術分為三大類（Croskerry et al., 2013）：

3.1 教育策略（Educational Strategies）

| 技術 | 說明 |

|------|------|

| 推理理論訓練 | 理解決策模型與認知偏誤機制 |

| 偏誤接種（Bias Inoculation） | 預先教導特定偏誤，建立檢測工具 |

| 認知輔導系統 | 電腦化回饋，追蹤個人決策模式 |

| 情境模擬訓練 | 在安全環境中練習辨識與修正認知錯誤 |

3.2 工作場域策略（Workplace Strategies）

| 技術 | 說明 |

|------|------|

| 結構化資料蒐集 | 使用檢查清單確保全面評估 |

| 後設認知與反思 | 刻意檢驗初始印象，延緩判斷 |

| 情緒去偏（Affective Debiasing） | 管理情緒對判斷的影響 |

| 群體決策 | 複雜情境諮詢同事，避免個人盲點 |

| 減速策略（Slowing Down） | 延遲結論，允許更深入考慮 |

| 暴露控制 | 限制過早取得他人意見，避免錨定效應 |

3.3 強制函數（Forcing Functions）

| 技術 | 說明 |

|------|------|

| 統計/臨床預測規則 | 演算法判斷優於直覺的場景 |

| 認知強制策略 | 針對特定偏誤場景的系統化實踐變更 |

| 考慮相反面（Consider the Opposite） | 尋找反駁初始印象的證據 |

| 排除最壞情境（ROWS） | 系統性考慮最嚴重可能 |

| 檢查清單（Checklists） | 源自航空業，現廣泛用於醫療與工程 |

| 停止規則（Stopping Rules） | 判定何時已有足夠資訊做最優決策 |

四、結構化分析技術（Structured Analytic Techniques）

源自美國情報分析界（CIA/DIA），結構化分析技術（SAT）是一套定性分析不確定性的原則與程序：

核心方法

  1. 競爭假說分析（ACH）：列出所有可能假說，逐項評估每個證據對各假說的支持/反對程度
  2. 魔鬼代言人（Devil's Advocacy）：指定團隊成員挑戰主流觀點
  3. 紅隊分析（Red Teaming）：模擬對手思維，預測反應
  4. 關鍵假設檢查（Key Assumptions Check）：識別並驗證分析中的隱含假設
  5. 預驗屍法（Pre-mortem Analysis）：假設決策已失敗，回溯可能原因

SAT 的去偏效果

  • 迫使分析師考慮更廣泛的選項與替代方案
  • 使推理過程透明化
  • 減少確認偏誤、過度自信與群體迷思

五、實務整合：DI + 去偏 + SAT 的決策流程


問題框定 → 假設生成 → 偏誤檢查 → 證據評估 → 決策 → 監控回饋
   ↓           ↓           ↓           ↓         ↓        ↓
  DI步驟1    DI步驟2    去偏技術     SAT/ACH   DI步驟3  DI步驟4
                      (後設認知)   (競爭假說)          (持續驗證)

日常決策應用建議

  1. 小決策（Type 2）：快速執行，設定回滾機制
  2. 中決策：使用 MECE 分解 + 考慮相反面
  3. 大決策（Type 1）：完整走 DI 四步 + ACH + Pre-mortem
  4. 持續改善：建立決策日誌，定期回顧，教育直覺

避免的常見陷阱

  • 第三類錯誤：精準解決錯誤問題（框定失敗）
  • 確認偏誤：只尋找支持既有信念的證據
  • 過度自信：高估自身判斷準確度
  • 沉沒成本謬誤：因已投入成本而繼續錯誤決策
  • 可得性偏誤：以容易回想的例子代替全面分析
