groq 審查結果

Frontmatter 規範性：✅ 優秀

  • ✅ name 與目錄名一致
  • ✅ description 含明確 "Use when:" 觸發條件
  • ✅ allowed-tools: [Bash, Read, Write]
  • ✅ cache-ttl: 5min 已驗證合理性：

- Groq Relay 自有快取 5 分鐘（llm-router.yaml 的 cache_ttl_min: 5）

- Skill 層級的 cache-ttl 是指「本地快取檔案 TTL」（api-cache Skill 的概念）

- Groq 主要用於即時前處理（摘要/翻譯），快取 5 分鐘平衡了即時性與 API 配額

- 結論：5min 是合理的

  • ✅ depends-on: config/dependencies.yaml
  • ✅ triggers: 9 個

內容品質：✅ 優秀

  • ✅ 四模式說明清晰（summarize/translate/classify/extract）
  • ✅ 有完整 curl 範例和 JSON 格式定義
  • ✅ 降級策略完善（Relay 不可用時跳過並記錄）
  • ✅ 速率限制和快取機制說明清楚

路由一致性：⚠️ 發現不一致

問題：「與 llm-router.yaml 的關係」章節與實際配置不一致

修正前（SKILL.md 表格）：

  • news_summary → groq
  • en_to_zh → groq

實際配置（llm-router.yaml）：

  • news_summary → cursor_cli（Composer 2 Fast）
  • en_to_zh → groq_direct（直呼 Groq API，tools/groq_direct.py）

修正內容

更新「與 llm-router.yaml 的關係」章節，使其與實際配置一致：


| routing_rule | 實際 Provider | 說明 |
|-------------|--------------|------|
| `news_summary` | cursor_cli | 新聞快速摘要（Composer 2 Fast） |
| `en_to_zh` | groq_direct | 英文翻譯（直呼 Groq API，tools/groq_direct.py） |
| `topic_classify` | groq | 主題分類（透過 Relay） |
| `quick_extract` | groq | 結構化萃取（透過 Relay） |
| `policy_analysis` | claude | 政策深度解讀（保留給 Claude） |
| `code_review` | claude | 程式碼審查（保留給 Claude） |

並新增說明：

  • groq（本 Skill）：透過 bot/groq-relay.js 代理，避免 API KEY 暴露
  • groq_direct：直接呼叫 Groq API（tools/groq_direct.py），金鑰讀 bot/.env，用於批次任務

總評：9.0/10

路由說明不一致是唯一問題，已在本次審查中修正。整體架構設計優秀，降級機制完善。
