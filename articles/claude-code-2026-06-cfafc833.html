【Claude Code Has Quietly Evolved — 三大核心轉變】

半年前的使用指引有很多已過時，Claude Code 在三個維度靜默升級：

① 記憶系統

• 出廠幾乎無用（.memory.md 幾乎不捕捉任何內容；recall 依賴精確關鍵字）

• 社群插件修補：MemArch（存儲+注入+語義召回 2 行安裝）、GBrain（多人場景）、Hermes（注入優化）

• 關鍵認知：記憶最大弱點不是存，是 recall——需語義向量而非關鍵字搜索

② 計畫層 / 推理分配

• /effort 設定推理投入（low → ultra code）；預設 auto 對複雜任務偏低

• Ultra Code = 讓模型自己設計 workflow 並派出多 agent 分工驗收，人只需提供品牌脈絡與完成條件

③ 自主排程

• /goal：設定「完成條件」讓 agent 自驅執行直到達標，Haiku 在每轉後評估是否滿足

• /routines（/schedule）：設定每日定時 agent，搭配 /goal 實現全自主週期性任務

• 核心主張：不是全自主，而是「反覆且有明確終止條件的任務讓 Claude 跑，決策點留給人」
