一、專案概述

Browser Use 是 2025-2026 年 GitHub 上最熱門的 AI 瀏覽器自動化開源框架，截至 2026 年 2 月已累積 78.9K Stars、9.3K Forks、293 位貢獻者、8,673 次 Commit。最新版本 v0.11.13（2026-02-25），已發布 116 個版本，採 MIT 授權。

核心定位：讓 AI Agent 能夠像人類一樣操作瀏覽器，透過自然語言指令完成網頁任務。

---

二、技術架構（DOM + Vision 雙軌）

2.1 核心處理管線


使用者輸入 → LLM 處理 → DOM/Vision 分析 → 動作規劃 → Playwright 執行 → 狀態更新 → 回饋迴圈
  1. DOM 擷取：掃描網頁，擷取所有互動元素（按鈕、輸入框、連結、表單），建立結構化索引
  2. 螢幕截圖：同步擷取頁面截圖，LLM 同時分析 DOM 結構與視覺佈局
  3. LLM 決策：基於 DOM + 視覺雙重資訊，決定下一步最佳動作
  4. Playwright 執行：透過 WebSocket 協議控制瀏覽器執行動作
  5. 狀態回饋：觀察結果、更新狀態，回傳 LLM 進行下一輪規劃

2.2 DOM 蒸餾（DOM Distillation）

將完整 DOM 精簡為 LLM 可理解的結構，標記可互動元素的索引編號，大幅減少 Token 消耗。這是 Browser Use 優於純視覺方案的關鍵技術。

2.3 自適應錯誤恢復

不依賴顯式錯誤處理規則，而是透過回饋迴圈實現自我修正：當動作未產生預期結果時，LLM 接收更新的狀態資訊，自動調整策略。

三、支援的 LLM 與模型策略

| 模型 | 適用場景 | 成本 |

|------|---------|------|

| ChatBrowserUse | 官方優化模型，最佳效能 | $0.20/百萬 input tokens |

| GPT-4o | 複雜推理與導航 | 中高 |

| GPT-4o-mini | 經濟型網頁爬取 | 低 |

| Claude (Anthropic) | 高品質理解與分析 | 中高 |

| Gemini (Google) | 多模態視覺整合 | 中 |

| Llama 3.1 (Ollama) | 本地部署、隱私優先 | 免費（需 GPU） |

模型無關（Model Agnostic） 是核心設計原則，開發者可自由選擇最適合場景的模型。

五、核心功能

  1. 多分頁支援：可同時在多個分頁操作（如：分別搜尋後比較結果）
  2. 持久化 Session：支援真實瀏覽器 Profile，保持登入狀態
  3. 自訂工具框架：@tools.action(description='...') 裝飾器模式擴充功能
  4. MCP 整合：透過 Model Context Protocol 連接外部服務（檔案系統、資料庫等）
  5. CLI 介面：命令列直接控制瀏覽器
  6. 雲端部署：Browser Use Cloud 提供託管版本（含隱匿瀏覽、CAPTCHA 處理）
  7. Claude Code 整合：可作為 Claude Code 的 Skill 使用

八、生產部署注意事項

  • 無頭模式：生產環境建議啟用 headless 模式
  • 解析度：建議 1920x1080 以確保元素偵測準確
  • 超時設定：預設 60 秒 action timeout
  • 安全性：API Key 透過環境變數管理，遵守 robots.txt
  • 基礎設施：開源版需自行管理基礎設施；生產環境建議搭配 Browserbase 或 Firecrawl

九、對 daily-digest-prompt 的啟發

  1. DOM 蒸餾思路：類似本專案的 Context Window 保護策略，精簡資訊再餵入 LLM
  2. 回饋迴圈架構：與本專案的品質自評迴圈（quality-gate）異曲同工
  3. 模型無關設計：值得參考的架構模式，讓核心邏輯不綁定特定 LLM
  4. MCP 整合：Browser Use 已支援 MCP，可探索整合至本專案的 Web 研究流程
