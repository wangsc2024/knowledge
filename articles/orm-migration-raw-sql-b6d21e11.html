ORM Migration 最佳實踐 — 從 Raw SQL 安全遷移至 ORM

技術概述

ORM（Object-Relational Mapping）Migration 是將散落在程式碼中的原始 SQL 查詢，逐步替換為 ORM 抽象層的重構過程。這不是簡單的語法轉換，而是涉及型別安全、交易管理、效能調優與零停機部署的系統性工程。本文以 SQLAlchemy 2.0+ 為主軸，結合 timezone-aware datetime 處理與資料庫遷移安全策略，提供可操作的遷移指南。

---

核心概念與原理

1. ORM vs Raw SQL 的決策矩陣

| 場景 | 建議 | 原因 |

|------|------|------|

| CRUD 基本操作 | ORM | 型別安全、自動參數化、跨資料庫相容 |

| 複雜報表/CTE/Window Function | Raw SQL | ORM 表達力不足，vendor-specific 語法 |

| 批量更新（>1000 筆） | Raw SQL | 單一 SQL 取代數百次 ORM row mutation |

| Migration 腳本 | Raw SQL | 靜態、可審計、不依賴 ORM model 當前狀態 |

| 含使用者輸入的查詢 | ORM 或 text() 參數化 | 防止 SQL Injection |

核心原則：ORM 用於例行領域邏輯，Raw SQL 用於查詢密集或資料庫原生任務。

2. SQLAlchemy 2.0 關鍵變更


# SQLAlchemy 1.x（已棄用）
result = engine.execute("SELECT * FROM users")  # 移除於 2.0

# SQLAlchemy 2.0+（必須使用 Connection）
from sqlalchemy import text
with engine.connect() as conn:
    result = conn.execute(text("SELECT * FROM users WHERE id = :id"), {"id": 42})
    conn.commit()

重點：2.0 強制使用明確的 Connection 物件和交易管理，不再有隱式自動提交。

3. Enum 一致性問題（實戰案例）

這是本專案 QA System 今日修復的核心問題：


# 問題：raw SQL 寫入字串值
conn.execute(text("UPDATE questions SET status = 'closed' WHERE id = :id"), {"id": qid})

# ORM refresh 時觸發 Enum KeyError — 因為 ORM 期望 QuestionStatus.CLOSED
db.refresh(question)  # ❌ KeyError: 'closed'

# 修正：使用 ORM 的 Enum 常量
question.status = QuestionStatus.CLOSED
db.commit()  # ✅ ORM 與 DB 一致

教訓：混用 raw SQL 與 ORM 時，Enum、DateTime 等型別的序列化/反序列化不一致是最常見的陷阱。

Timezone-Aware DateTime 處理

全域策略：UTC 儲存，本地顯示


from datetime import datetime, timezone
from sqlalchemy import DateTime
from sqlalchemy.orm import Mapped, mapped_column

class Event(Base):
    __tablename__ = 'events'
    id: Mapped[int] = mapped_column(primary_key=True)
    # 使用 timezone=True 確保 DB 層支援時區
    created_at: Mapped[datetime] = mapped_column(
        DateTime(timezone=True),
        server_default=func.now()
    )

常見陷阱與修正


# ❌ 錯誤：naive datetime（無時區資訊）
closed_date = datetime.now()  # 系統可能用本地時區解讀

# ✅ 正確：明確使用 UTC
closed_date = datetime.now(timezone.utc)

# ✅ 更佳：使用 server_default 讓 DB 管理時間戳
# mapped_column(DateTime(timezone=True), server_default=func.now())

SQLite TypeDecorator（無原生時區支援時）


from sqlalchemy import TypeDecorator, DateTime

class TZDateTime(TypeDecorator):
    impl = DateTime
    cache_ok = True

    def process_bind_param(self, value, dialect):
        if value is not None:
            if value.tzinfo is None:
                value = value.replace(tzinfo=timezone.utc)
            return value.astimezone(timezone.utc)
        return value

    def process_result_value(self, value, dialect):
        if value is not None and value.tzinfo is None:
            return value.replace(tzinfo=timezone.utc)
        return value

安全遷移策略

1. 增量轉換（非一次性重寫）

遷移順序：

  1. 識別：找出所有 raw SQL 位置（grep text( 或 execute(）
  2. 分類：標記每個查詢為「可轉 ORM」或「保留 raw SQL」
  3. 隔離：將 raw SQL 集中到 repository 層
  4. 包裝：建立 wrapper 返回 ORM 相容物件
  5. 替換：逐一轉換，每次轉換後執行完整測試

2. 零停機資料庫遷移（Alembic）

兩階段部署：

| 操作類型 | 順序 |

|---------|------|

| 新增欄位 | 先 DB migration → 後部署應用 |

| 移除欄位 | 先部署應用（停止使用）→ 後 DB migration |

| 改名欄位 | 7 步漸進法（見下） |

欄位改名的 7 步漸進法：

  1. 新增新欄位
  2. 部署雙寫（寫入新舊兩欄）
  3. 資料一致性驗證
  4. 遷移既有資料
  5. 讀取切換至新欄位
  6. 移除舊欄位寫入
  7. 刪除舊欄位

3. Migration 腳本安全規則


# Alembic migration 最佳實踐
def upgrade():
    # ✅ 使用 Raw SQL，不依賴 ORM model
    op.execute(sa.text("""
        ALTER TABLE questions
        ALTER COLUMN closed_date TYPE TIMESTAMP WITH TIME ZONE
        USING closed_date AT TIME ZONE 'UTC'
    """))

    # ❌ 不要在 migration 中 import ORM model
    # from models import Question  # 如果 model 變了，migration 會壞

程式碼範例：完整遷移模式


# === 遷移前：raw SQL 混用 ===
def close_question_v1(db, question_id, summary):
    db.execute(text("""
        UPDATE questions SET status = 'closed',
        summary = :summary, closed_date = :now
        WHERE id = :id
    """), {"id": question_id, "summary": summary, "now": datetime.now()})
    db.commit()

# === 遷移後：ORM 型別安全 ===
def close_question_v2(db: Session, question_id: int, summary: str):
    question = db.query(Question).filter(Question.id == question_id).one()
    question.status = QuestionStatus.CLOSED
    question.summary = summary
    question.closed_date = datetime.now(timezone.utc)
    db.commit()
    db.refresh(question)  # 安全：ORM Enum 一致
    return question
