QA System 安全優化報告 #14

優化方向

reports 路由測試覆蓋補全（21 個測試案例，覆蓋全部 3 個端點）+ get_reports 既有 Bug 修復

安全掃描結果

修復 1 個紅色問題，其餘全部綠燈：

已修復（安全）

  • get_reports 端點 Bug（紅色）：使用不存在的 question.department_id 屬性，導致 AttributeError → 500 Internal Server Error。修復：改為遍歷 question.report_departments 和 question.answer_departments，與 create_report/update_report 一致。

仍有效（綠燈）

  • SECRET_KEY 環境變數化
  • SQL 注入防護（參數化 text() + bind params）
  • XSS 防護（無 innerHTML/document.write/|safe）
  • CORS 限制來源、Cookie httponly/secure/samesite
  • JWT 集中解碼 + 30 分鐘過期
  • python-jose >=3.5.0 CVE 修復
  • 錯誤訊息洩漏全專案零殘留

已修復問題

品質修復

新增 test_reports_api.py，21 個測試案例覆蓋 reports 路由全部 3 個端點：

| 端點 | 測試數 | 涵蓋場景 |

|------|--------|----------|

| POST /reports/{question_id} | 7 | 認證、權限、404、成功建立、部門匹配、已結案拒絕、無部門存取拒絕 |

| GET /reports/{question_id} | 7 | 認證、404、空列表、序列化 Bug 記錄、部門匹配、無部門存取拒絕、無權限拒絕 |

| PUT /reports/{report_id} | 7 | 認證、404、創建者更新、管理員更新、非創建者拒絕、已結案拒絕、無權限拒絕 |

安全修復

get_reports 端點使用 question.department_id（Question model 沒有此屬性），導致：

  • 所有非管理員存取一律 500 Internal Server Error
  • 錯誤堆疊可能洩漏內部資訊

修復後改為遍歷多對多關聯表（與 create_report、update_report 一致）。
