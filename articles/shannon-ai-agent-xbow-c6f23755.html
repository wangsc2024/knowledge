1. 專案概述

Shannon 是由 Keygraph 開發的全自主 AI 滲透測試工具，專為 Web 應用程式和 API 設計。其核心理念是解決「開發團隊透過 AI 工具持續交付程式碼，但滲透測試通常一年僅做一次」的安全缺口。Shannon 能在每次建置或發布時按需執行安全測試，將滲透測試從週期性人工服務轉變為持續自動化流程。

核心定位：白箱（source-available）AI 滲透測試，結合靜態原始碼分析與動態漏洞利用驗證，只報告有可重現 PoC（Proof-of-Concept）的真實漏洞。

---

2. 技術架構

2.1 三階段多 Agent 管線

Shannon 採用多階段管線架構，各階段並行處理：

  1. 偵察階段（Reconnaissance）：整合 Nmap（埠掃描）、Subfinder（子網域列舉）、WhatWeb（技術指紋識別）、Schemathesis（API Fuzzing）等工具，收集目標應用程式資訊與端點
  2. 漏洞分析階段（Vulnerability Analysis）：多個並行 Agent 各自負責特定攻擊領域（SQL Injection、XSS、SSRF、認證繞過、授權失敗等），結合原始碼分析引導攻擊策略
  3. 漏洞利用階段（Exploitation）：對發現的漏洞執行實際攻擊驗證，並行攻擊多個漏洞類別，產生可複製貼上的 PoC 程式碼

2.2 技術棧

| 組件 | 技術選擇 | 用途 |

|------|---------|------|

| 語言 | TypeScript/Node.js | 核心框架 |

| AI 引擎 | Anthropic Claude（Agent SDK） | 推理與決策 |

| 容器化 | Docker + Docker Compose | 隔離執行環境 |

| 工作流引擎 | Temporal | 任務編排、中斷恢復、檢查點 |

| 瀏覽器自動化 | Playwright | Web 應用互動與漏洞利用 |

| 監控 | Temporal Web UI（localhost:8233） | 即時進度追蹤 |

2.3 Shannon Pro 進階功能（商業版）

  • CPG 資料流分析（SAST）：追蹤使用者輸入從 source 到 sink 的路徑，上下文感知消毒驗證
  • 商業邏輯測試：自動化不變量發現 + Fuzzer 生成
  • SCA 可達性分析：判斷脆弱函式是否實際可從入口點觸及
  • 祕密偵測：模式匹配 + LLM 偵測 + 活性驗證（確認憑證是否仍有效）
  • 靜態-動態關聯：SAST 發現與動態測試結果自動關聯

4. 使用方式

快速啟動


git clone https://github.com/KeygraphHQ/shannon.git
cd shannon
export ANTHROPIC_API_KEY="your-key"
./shannon start URL=https://your-app.com REPO=your-repo

執行成本與時間

  • 單次完整掃描：約 1-1.5 小時
  • API 費用：約 $50 USD（使用 Claude 3.5 Sonnet）
  • 支援 AWS Bedrock 和 Google Vertex AI 作為替代 AI 提供者

進階配置

  • 認證測試：支援表單登入、SSO、TOTP 2FA 自動化
  • 工作區管理：命名工作區支援中斷恢復、跳過已完成階段
  • 自訂規則：YAML 配置焦點區域和排除路徑
  • 多倉庫支援：單一 Repo、Monorepo、多 Repo（前後端分離）

5. 架構設計啟示

5.1 對 Agent 系統設計的借鑑

  1. 白箱 + 動態融合：靜態分析引導動態測試方向，大幅減少誤報。這種「分析→行動→驗證」循環是高品質 Agent 的核心模式
  2. Temporal 工作流引擎：提供企業級可靠性（中斷恢復、檢查點），值得任何長時間 Agent 任務參考
  3. 並行 Agent 分工：各攻擊類別獨立 Agent 並行，與 Daily-Digest-Prompt 的團隊並行模式異曲同工
  4. 僅報告有 PoC 的漏洞：「證據驅動」而非「可能性報告」，大幅提升安全報告的可操作性

5.2 安全考量

  • Shannon 是攻擊性工具，僅應用於授權範圍內的應用測試
  • AGPL-3.0 授權要求修改後的程式碼也必須開源
  • 執行成本（$50/次）使其適合重要發布前的安全驗證，而非每次 commit
