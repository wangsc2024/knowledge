Memory and Dreaming for Self-Learning Agents (Anthropic Official)

youtuberesearchClaudememorydreamingauto

摘要

Anthropic API 團隊 Ravi 介紹 Claude Managed Agents 的兩大新功能：Memory（記憶）與 Dreaming（做夢）。Memory 讓 Agent 在任務間學習與傳遞知識，以檔案系統為儲存模型（讓 Claude 用它擅長的 Bash/文件操作），支援多 Agent 共享記憶庫（readonly/readwrite 分層）、樂觀並發控制防止寫入衝突、版本控制與審計追蹤。Dreaming 是解耦於 Agent 主迴圈的離線反饋程序，分析跨 Session 的 transcript 找出重複錯誤與低效模式，自動整理並優化 Memory 內容。Rocketin 報告首次通過錯誤降低 97%，Harvey 法律基準完成率提升 6 倍。

關鍵發現

  • Memory 三層架構：Storage Layer → Structure（檔案系統優化格式）→ Processing（Agent 邊做邊記）
  • Dreaming 完全去耦，可 ad-hoc / 夜間 / hourly / event-triggered，分析跨 session 跨 agent transcript
  • Opus 4.7 在 file-system-based memory 為 SOTA
  • 獨立 Memory API 支援 CRUD、匯出、編修
  • Demo：SRE on-call agent 用共享記憶協調跨 session 修復

適用情境

Agent 架構設計、Memory 系統設計、多 Agent 協作、自我改進迴圈
