專案觀測摘要

OpenHands/OpenHands

  • GitHub repo 觀測到 71.5k stars，最新 release 為 1.6.0（2026-03-30）。
  • 文件將 sandbox 分成 Docker、Process、Remote 三類；Docker 為預設且隔離較佳，Process 明確標示無容器隔離。
  • Settings UI 與 RUNTIME、SANDBOX_VOLUMES 等環境變數是主要 operator override 入口。
  • 來源：、、

browser-use/browser-use

  • GitHub repo 觀測到 88.7k stars；搜尋快照顯示 repo 最近更新至 2026-04-15，repo page 顯示最新 release 為 0.12.6（2026-04-02）。
  • Cloud quickstart 強調 stealth browsers、CAPTCHA solving、residential proxies、managed infrastructure。
  • sessions.create()/run() 與 browsers.create() 暴露 profile_id、proxy、keep_alive、workspace_id 等控制面，屬高能力 browser tool API。
  • 來源：、

e2b-dev/E2B

  • GitHub repo 觀測到 11.8k stars，最新 release 為 @e2b/python-sdk@2.20.0（2026-04-02）。
  • E2B 把安全主軸放在 execution sandbox：E2B_SANDBOX metadata、lifecycle、kill/pause/autoResume、Sandbox.connect()。
  • 適合承接 execution environment boundary，但不直接提供高階 tool approval semantic。
  • 來源：、、

openai/openai-agents-python

  • GitHub repo 觀測到 23.1k stars，最新 release 為 v0.14.2（2026-04-18）。
  • README 將 guardrails、sandbox agents、human in the loop、tracing 並列為核心概念。
  • Guardrails 文件明寫：function-tool guardrails 不等於 handoff / hosted tool / built-in execution tool coverage；HITL approval surface 為 run-wide，且可序列化 RunState 後再 resume。
  • 來源：、、

改進提案

P1：把 guardrail coverage gap 納入 github_scout 研究契約

  • 來源專案：openai/openai-agents-python
  • 重點：研究與審查時，不再把 agent-level guardrail 誤寫成全工具覆蓋；需明確寫出 handoff、delegation、hosted tool、built-in execution 的 coverage 與 fallback。
  • 本輪已落實於 config/github-scout-review-policy.yaml 與 templates/auto-tasks/github-scout.md。

P0：建立統一 Tool Boundary Coverage Gate + Operator Override Registry

  • 來源專案：OpenAI Agents SDK + OpenHands + browser-use + E2B 綜合觀測
  • 重點：把 execution environment boundary 與 tool API boundary 拆開，並把 allow/deny/review-only 變成可執行 gate。
  • 風險：需要動到 pre_*_guard.py、approval queue、trace/rollback，屬高風險，僅出計畫不實作。

P2：補上 last_updated 的活動來源正規化

  • 來源專案：browser-use / E2B
  • 重點：目前 last_updated 常混用 release 日期、repo 活動快照、頁面觀測；建議後續新增 last_updated_source 或等價欄位，降低新鮮度誤讀。

審查結論

  • P1 採納：低風險、可立即提升研究輸出的安全精度；已通過 config 驗證。
  • P0 延後：價值高，但會碰核心攔截 hooks 與人審恢復流程，需人工審核後再分階段導入。
  • P2 保留：有價值，但牽涉結果 consumer 與既有 schema，先進 backlog 追蹤。

已落實項目

  • 更新 config/github-scout-review-policy.yaml：新增 guardrail_surface_recommended_facets、guardrail_gap_expectations、coverage gap 審查要求。
  • 更新 templates/auto-tasks/github-scout.md：要求安全主題逐案標明 guardrail coverage/exclusions、approval scope、handoff 與 built-in execution gap。

待人工審核項目

  • docs/plans/github-scout-20260420-tool-boundary-coverage-gate.md
  • 是否將 run-wide approval / serialized RunState 引入本系統 approval queue
  • 是否把 tool boundary gate 接到核心 pre_*_guard.py
