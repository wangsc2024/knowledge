Daily Digest Hooks 系統品質優化報告

優化概覽

針對 daily-digest-prompt 專案的 Hooks 機器強制層（7 個 Python 模組）進行深度品質審查與優化，共完成 5 項程式碼改進，測試套件 532/532 全通過。

優化項目

1. hook_utils.py — 消除重複開檔 + sys import 提升

  • 問題：load_yaml_rules() 和 filter_rules_by_preset() 各自獨立開啟 hook-rules.yaml，同一次 hook 呼叫中重複讀取
  • 修正：提取 _load_yaml_config() 共用函式，將 import sys 從函式內部提升至模組頂層
  • 效益：每次 hook 呼叫減少 1 次檔案 I/O，每日 44 次 Agent 執行 × 20-100 次工具呼叫的效能改善

2. post_tool_logger.py — 錯誤偵測誤判強化

  • 問題：BENIGN_PATTERNS 不足，agent_guardian 的 error_category 欄位、post_tool_logger 的 error_classification 變數名、JSONL 的 error- 標籤前綴、"denied": false JSON 布林值會被誤判為錯誤
  • 修正：新增 5 個 BENIGN_PATTERNS 條目
  • 效益：減少假陽性 error tag，提升 session 健康狀態判斷準確度

3. on_stop_alert.py — Gmail OAuth 狀態檔路徑安全性

  • 問題：check_gmail_token_expiry() 使用相對路徑 os.path.join("key", "token.json")，當 hook 的 cwd 與專案根目錄不同時找不到檔案
  • 修正：改用 __file__ 推算專案根目錄的絕對路徑
  • 效益：無論 cwd 為何，都能正確定位 token.json 和 gmail-oauth-state.json

4. validate_config.py — timeouts.yaml 巢狀 schema 驗證

  • 問題：timeouts.yaml 驗證僅檢查 version 頂層鍵，缺少對 daily_digest_team.phase1_timeout 等巢狀必要鍵的驗證
  • 修正：新增 nested_required schema 機制，驗證 daily_digest_team、todoist_team、audit_team 各自的必要鍵
  • 效益：配置修改後可即時偵測缺少關鍵 timeout 設定，避免 Agent 執行時因缺少 timeout 值而失敗

5. agent_guardian.py — CircuitBreaker 競態條件防護

  • 問題：_update_state() 的 read-modify-write 操作無鎖保護，團隊模式多個並行 Agent 可能同時讀寫 api-health.json 導致狀態覆蓋
  • 修正：新增 _atomic_update() 方法，使用 msvcrt/fcntl 跨平台檔案鎖保護狀態更新
  • 效益：團隊模式下的 CircuitBreaker 狀態更新保證原子性

測試結果

  • 基線：532 個測試
  • 修改後：532/532 全通過（6.81 秒）
  • 受影響測試更新：on_stop_alert 測試加 __file__ monkeypatch、validate_config 最小配置測試更新

影響範圍

  • 修改檔案：5 個 Python hooks + 2 個測試檔
  • hooks/hook_utils.py、hooks/post_tool_logger.py、hooks/on_stop_alert.py、hooks/validate_config.py、hooks/agent_guardian.py
  • tests/hooks/test_on_stop_alert.py、tests/hooks/test_validate_config.py

技術決策

  • CircuitBreaker 鎖機制採用 graceful degradation：Windows 用 msvcrt、Linux 用 fcntl、無鎖可用時退化為無鎖模式
  • BENIGN_PATTERNS 採用子字串匹配而非正則，平衡精確度與效能
  • 路徑安全性使用 __file__ 而非環境變數，確保 hook 的可攜性
