Frogger v2.0 品質優化實踐

概要

將 Frogger 從 v1.0 升級至 v2.0，重點解決六大品質問題，並加入三大留客機制。

修復的品質問題

1. Fixed Timestep 遊戲循環

  • 問題：v1.0 使用 dt（秒）直接乘以速度，不同裝置行為不一致
  • 修復：採用 accumulator 模式，FIXED_DT = 1000/60，物理更新與渲染分離
  • 程式碼模式：
    • 
let accumulator = 0;
function gameLoop(timestamp) {
  requestAnimationFrame(gameLoop);
  let dt = timestamp - lastTime;
  lastTime = timestamp;
  if (dt > MAX_FRAME_DT) dt = MAX_FRAME_DT;
  accumulator += dt;
  while (accumulator >= FIXED_DT) {
    update();
    accumulator -= FIXED_DT;
  }
  render();
}

2. 物件池（Object Pool）

  • 問題：粒子和浮動文字使用 push/splice，造成 GC 壓力
  • 修復：預建立固定大小池（粒子 200、文字 30），用 active 旗標管理生命週期
  • 效益：零 GC 壓力，穩定 60FPS

3. 計分系統修復

  • 問題：v1.0 前進計分的 if 區塊為空（永遠不加分）
  • 修復：追蹤 furthestRow，每前進一格 +10 分，搭配浮動文字回饋

4. 全螢幕響應式 Canvas

  • 問題：v1.0 用固定尺寸 Canvas + CSS 縮放
  • 修復：Canvas 大小 = 視窗大小，所有座標通過 sx()/sy() 縮放函數轉換
  • 手機支援：虛擬 D-Pad（四個圓形按鈕）+ 滑動手勢 fallback

新增留客機制

Combo 系統

  • 連續進家不死亡 → combo 遞增，3 秒內（180 幀）須再進家
  • combo x2 起額外加分：50 * (combo - 1)
  • combo x3 顯示「連續跳躍!」、x5 額外 +200 且顯示「完美節奏!」
  • 死亡重置 combo

成就系統

  • 12 個成就，localStorage 持久化
  • 涵蓋里程碑（初渡、關卡、分數）、技巧（速度、combo）、挑戰（無死亡通關）
  • Game Over 畫面展示新解鎖成就

方向感知瞳孔

  • 青蛙眼睛瞳孔隨移動方向偏移，增加角色生動感

學到的設計原則

  1. Fixed Timestep 是基本功：任何涉及物理的遊戲必須用固定時間步
  2. 物件池 = 穩定幀率：頻繁建立/銷毀物件是效能殺手
  3. 即時回饋 = 成癮性：每個動作都應有視覺+音效回饋（浮動文字、粒子、音效）
  4. 連鎖機制 = 留客：Combo 系統讓玩家追求「再多一把」
  5. 成就 = 長期目標：給玩家超越單局的追求動力
