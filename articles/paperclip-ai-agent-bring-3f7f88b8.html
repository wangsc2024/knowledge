一、專案概述

Paperclip（paperclipai/paperclip）是一款開源 AI Agent 編排平台，定位為「零人力公司的基礎設施層」。不同於傳統 Agent 框架專注於單一 Agent 能力或工作流設計，Paperclip 將整個「公司組織結構」建模為 Agent 的運行環境——組織圖、預算、治理、審批閘門、目標對齊一應俱全。

截至 2026 年 3 月，專案已累積 7,700+ Stars、876 Forks、564 commits，採用 MIT 授權，是 2026 年 3 月 Trendshift 排行榜上增長最快的專案之一。

核心定位

「If you think of an individual AI agent (like OpenClaw or Claude) as an employee, Paperclip is the company.」

Paperclip 明確不是：聊天機器人介面、工作流視覺化編輯器、Prompt 管理工具、程式碼審查工具。它是組織基礎設施——管理自主 Agent 團隊的編排層。

二、技術架構

2.1 技術棧

| 層級 | 技術選型 | 說明 |

|------|---------|------|

| 語言 | TypeScript（97.1%） | 型別安全的全棧開發 |

| 後端 | Node.js 20+ | 單一進程部署模式 |

| 前端 | React | 管理儀表板 UI |

| 資料庫 | PostgreSQL | 本地開發用嵌入式 PG，生產環境接外部 PG |

| 套件管理 | pnpm 9.15+ | 高效依賴管理 |

| 授權 | MIT | 完全開源 |

2.2 核心架構模型：Bring-Your-Own-Agent（BYOA）

Paperclip 採用「自帶 Agent」架構，不限制 Agent 的實現方式。支援的 Agent 類型包括：

  • OpenClaw：個人 AI 助手（265K Stars，2026 年增長最快的開源專案）
  • Claude Code：Anthropic 的程式碼 Agent
  • Codex / Cursor：OpenAI 和 Cursor 的 Agent
  • Bash 腳本：任何可執行的 shell 命令
  • HTTP Webhooks：任何支援 HTTP 回呼的服務

唯一要求：Agent 能接收 Heartbeat 信號。

2.3 Heartbeat 排程機制

Agent 透過排程的 Heartbeat（心跳信號）被喚醒：

  1. Agent 收到 Heartbeat → 檢查待辦工作
  2. 取出任務（原子化 checkout，防止雙重執行）
  3. 執行工作，更新狀態
  4. 在下一個 Heartbeat 恢復上下文繼續執行

關鍵特性：Agent 的狀態在 Heartbeat 之間持久化，不會因重啟而丟失上下文。

---

三、五大核心特性

3.1 組織圖驅動的任務流

  • 建模公司層級結構：公司 → 部門 → 角色 → Agent
  • 每個 Agent 有明確的職位描述、彙報線和職責範圍
  • 任務委派沿組織圖上下流動：上級 Agent 可委派子任務，下級 Agent 可向上回報
  • 目標繼承鏈：Task → Project → Goal → Company Mission

3.2 原子化預算治理

| 功能 | 說明 |

|------|------|

| 月度預算 | 每個 Agent/部門設定月度 Token 上限 |

| 原子扣款 | 任務 checkout 時即扣除預算，防止超支 |

| 80% 警告 | 接近預算上限時自動告警 |

| 100% 暫停 | 預算耗盡自動暫停，需 Board 覆寫才能繼續 |

| 多維追蹤 | 支援 per-agent、per-task、per-project 成本分析 |

實際成本參考：典型 AI Agent 編排成本為 $200-$2,000/月，視使用量而定。

3.3 治理與審批閘門

使用者作為「董事會」角色，擁有完整控制權：

  • 招聘審批：Agent 無法自行新增 Agent，需 Board 核准
  • 策略審批：重大配置變更需經審批閘門
  • 版本化配置：所有配置變更自動版本化，支援回滾
  • 即時干預：可隨時暫停、恢復、覆寫、重分配或終止任何 Agent

3.4 不可變審計日誌

  • Append-only 設計，不可修改或刪除
  • 記錄每個 Agent 的每次指令、回應、工具呼叫
  • 完整追溯：誰做了什麼、為什麼、結果如何
  • 多租戶隔離：每個「公司」的審計記錄完全獨立

3.5 Runtime Skill Injection

  • Agent 可在運行時動態學習新工作流，無需重新訓練
  • 專案上下文隨任務注入，Agent 理解當前工作的背景
  • 未來路線圖：Clipmart 市場提供預建組織模板（交易桌、內容工廠、開發代理商）

四、實際案例與成效

案例 1：Felix（Nat Eliason 的 AI Agent）

  • 一個 AI Agent 自主運營，處理內容創作、研究和商務開發
  • 已創造超過 $100,000 美元營收，目標 $1,000,000
  • 展示了單一 Agent 在 Paperclip 框架下的商業可行性

案例 2：Aaron Sneed 的 Agent 議會

  • 15 個自定義 GPT Agent 組成的協作團隊
  • 節省超過 每週 20 小時的人工時間
  • 每個 Agent 有明確定義的職責和交接流程

案例 3：OpenClaw CEO 模式

  • 一個 OpenClaw Agent 擔任 CEO，管理完整的 Paperclip 公司
  • 下轄多個專業部門：內容、銷售、分析、競標自動化
  • 每日營運成本僅 $57 AUD（約 $37 USD）

五、與其他 Agent 框架的定位比較

| 維度 | Paperclip | OWL (CAMEL-AI) | Dify | n8n |

|------|-----------|---------------|------|-----|

| 核心定位 | 組織治理層 | 多 Agent 協作框架 | AI 工作流平台 | 通用自動化平台 |

| Agent 模型 | BYOA（自帶 Agent） | Planner-Coordinator-Worker | 內建 Agent 節點 | 內建 AI 節點 |

| 預算控制 | 原子化，多維度 | 無 | 基本 Token 計量 | 無 |

| 組織結構 | 完整組織圖 | 任務導向 | 工作流導向 | 節點導向 |

| 治理機制 | 董事會模式 | 無 | 基本權限 | 基本權限 |

| 審計 | Append-only 完整追蹤 | 基本日誌 | 執行日誌 | 執行日誌 |

| 適用場景 | 20+ Agent 自主運營 | 複雜任務分解 | 可視化工作流 | 跨系統整合 |

關鍵差異：Paperclip 處於「組織管理」層級，其他框架處於「任務執行」層級。Paperclip 可以包裝其他框架的 Agent 作為組織中的「員工」。

六、部署與上手

本地開發（一鍵啟動）


npx paperclipai onboard --yes

自動建立嵌入式 PostgreSQL、初始化公司配置、啟動 Dashboard。

生產部署

  • 指向外部 PostgreSQL 實例
  • 部署至 Vercel、Docker 或任何 Node.js 托管平台
  • 多租戶支援：一次部署可運行多家「公司」

七、對 Daily-Digest-Prompt 系統的借鑑啟示

  1. Heartbeat 模式：Paperclip 的排程心跳機制與本專案的 Windows Task Scheduler 排程異曲同工，但 Paperclip 的持久化狀態管理更完善
  2. 預算治理：Token 預算的原子化扣款概念可借鑑至本專案的 state/token-budget-state.json
  3. 組織圖概念：將 daily-digest 的多個子 Agent（todoist、digest、research）視為「部門」，可參考 Paperclip 的層級委派模式
  4. 審計日誌：Paperclip 的 Append-only 審計與本專案的 hooks/post_tool_logger.py JSONL 日誌理念一致，但粒度更細
  5. BYOA 架構：「自帶 Agent」模式驗證了本專案「Claude Code CLI + PowerShell 編排」的架構方向

---

八、限制與注意事項

  1. 非萬能替代：「Even in a 'zero-human company,' there is always a human setting strategy」— 人類治理仍不可或缺
  2. 錯誤放大效應：多 Agent 環境中錯誤會跨系統傳播，需要更嚴格的監控
  3. ROI 時間成本：投資回報通常在第 2-3 個月才顯現，非即時效益
  4. 流程前置：「If you cannot explain the steps a task requires, an AI agent cannot do it reliably」— 流程定義先於自動化
  5. 成本控制：缺乏預算控制的 Agent 可能快速累積高額費用（曾有 $200+ 單次事件報導）

九、未來展望

  • Clipmart 市場：預建組織模板（交易桌、內容工廠、開發代理商）的上線將降低使用門檻
  • 零人力公司趨勢：2026 年已有多位創業者驗證此模式，小團隊 + AI Agent 可在速度、成本和一致性上超越大型傳統團隊
  • Agent 編排標準化：Paperclip 的治理模式可能成為多 Agent 系統的組織標準
