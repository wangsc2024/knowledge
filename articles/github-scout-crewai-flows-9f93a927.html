GitHub Scout 分析報告：多代理編排框架 2026

搜尋日期：2026-04-25

搜尋主題：多代理編排（CrewAI Flows / LangGraph / AutoGen）

專案觀測摘要

CrewAI (49.8k★)

  • URL: https://github.com/crewAIInc/crewAI
  • 架構模式：雙模式 — Crews（角色驅動自治）+ Flows（事件驅動 @listen decorator）
  • 狀態管理：Flows 跨任務 secure state，支援 and_/or_ 邏輯觸發條件
  • HITL 模式：interrupt-point（同步短延遲），Crew 執行時可插入 human input 步驟
  • 特點：完全獨立於 LangChain，比 LangGraph 快 5.76x（QA 任務），Fortune 500 60%+ 採用

LangGraph (30.3k★)

  • URL: https://github.com/langchain-ai/langgraph
  • 架構模式：圖式編排（Pregel 啟發），節點-邊有向狀態圖，支援條件邊與循環
  • 狀態管理：checkpoint 機制（durable execution），短期工作記憶 + 跨 session 長期記憶
  • HITL 模式：interrupt-point 模式（interrupt() API），圖執行遇 breakpoint 暫停後等待人工 approve/edit/skip
  • 特點：低層級控制，支援長時間運行的有狀態 Agent 系統

AutoGen (57.4k★) — ⚠️ 維護模式

  • URL: https://github.com/microsoft/autogen
  • 架構模式：對話式多代理，三層架構（Core/AgentChat/Extensions）
  • 狀態管理：會話歷史式記憶，缺乏內建流程概念
  • HITL 模式：支援 human-in-the-loop 對話插入，但無明確 interrupt-point 或 signal API
  • ⚠️ 狀態警示：已進入維護模式，官方建議遷移至 Microsoft Agent Framework

改進提案

P1-A：interrupt-point vs signal-based HITL 分類標準化 ✅ 已落實

  • 來源：LangGraph interrupt() 與 Temporal signal 業界分類
  • 目標：config/autonomous-harness.yaml → human_review_gates
  • 內容：新增 hitl_gate_classification 區分兩種 HITL 模式

- interrupt_point：同步短延遲（self_heal、github_scout P0 確認）

- signal_based：異步高延遲（arch_evolution、constraint_evolve 跨日審核）

  • 風險：low（純新增欄位）

P1-B：CrewAI Flows 觸發詞補齊 github-scout SKILL ✅ 已落實

  • 來源：CrewAI Flows 架構分析
  • 目標：skills/github-scout/SKILL.md v2.2.0 → v2.3.0
  • 內容：新增 9 個觸發詞（CrewAI Flows, crewai, multi-agent orchestration 2026 等）
  • 風險：low（純新增 triggers）

P2-A：AutoGen 維護模式警示

  • 內容：技術選型決策資訊——未來不建議依賴 AutoGen，優先評估 CrewAI Flows 或 Microsoft Agent Framework

審查結論
