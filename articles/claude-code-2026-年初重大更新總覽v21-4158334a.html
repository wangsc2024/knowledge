Claude Code 2026 年初重大更新總覽

概述

2026 年 1-2 月，Anthropic 為 Claude Code 帶來多項重大更新，包括 Claude Opus 4.6 模型支援、Agent Teams 多智能體協作、自動記憶持久化、Skills 熱重載等核心功能。版本從 v2.1.0 迭代至 v2.1.39，累計超過 1,096 個 commits，標誌著 Claude Code 從「編程助手」升級為「生產級基礎設施」。

核心新功能

1. Claude Opus 4.6 模型（v2.1.32）

  • 1M token 上下文窗口（Beta）—— Opus 級別模型首次支援百萬 token
  • 改進的編程技能：更長時間執行 Agent 任務、大型代碼庫更可靠
  • 更好的代碼審查與調試能力
  • Fast Mode 支援（v2.1.36）：高速推理模式可用於 Opus 4.6

2. Agent Teams 多智能體協作（研究預覽）

  • 將大型任務拆分給多個智能體並行處理
  • 每個智能體負責自己的部分並直接協調
  • 需啟用環境變數：CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1
  • 支援 Task(agent_type) 語法限制可生成的子 Agent 類型

3. 自動記憶持久化

  • Claude 自動記錄並回憶工作中的記憶
  • 支援 memory frontmatter 字段：user、project、local 三種範圍
  • 跨會話保持上下文連續性

4. Skills 系統強化（v2.1.0）

  • 熱重載：Skills 修改後即時生效
  • Forked Context：Skills 可擁有獨立上下文
  • Custom Agent 支援：可定義自訂 Agent
  • 斜線命令調用：/skill-name 快速啟動
  • .claude/skills/ 目錄自動載入（包含 --add-dir 指定的目錄）
  • Skill 字符預算隨上下文窗口動態調整（佔上下文 2%）

重要改進

PDF 處理增強（v2.1.30）

  • pages 參數：可讀取特定頁面範圍（如 pages: "1-5"）
  • 大型 PDF（>10 頁）@提及時返回輕量級引用，不內聯到上下文

會話管理

  • Summarize from here：部分對話摘要功能
  • Compression：自動壓縮早期上下文，支援更長對話
  • /rewind：復原代碼變更
  • /teleport：將會話傳送至 claude.ai/code

終端與 UI

  • Shift+Enter 換行：無需設定即可使用
  • 可自訂快捷鍵：/keybindings 配置
  • 寬字元改進：Emoji、CJK 字元渲染優化
  • 可點擊文件路徑：支援 OSC 8 的終端可直接點擊開啟

權限與安全

  • 萬用字元權限：如 Bash(*-h*) 允許所有 -h 結尾命令
  • Sandbox 模式修復：防止 .claude/skills 目錄寫入
  • Heredoc 解析改進：防止命令注入

使用方式

啟用 Agent Teams

export CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1
claude  # 啟動後可使用多智能體協作

常用新命令

/keybindings   # 自訂快捷鍵
/debug         # 排錯當前會話
/release-notes # 查看發行說明
/model         # 切換模型（可選 Opus 4.6）
/rewind        # 復原代碼變更

Skills 定義位置

.claude/skills/       # 專案級 Skills
~/.claude/skills/     # 用戶級 Skills

企業採用

  • Allianz 合作：成為 2026 年首批大型企業客戶，全員啟用 Claude Code 並建立多步驟工作流 Agent
  • Claude Cowork：面向非技術用戶的版本，可編輯文件、起草報告
