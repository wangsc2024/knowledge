改進提案

P1：guardrail_surface_layers annotation（guardrails-ai 借鑑）

借鑑 guardrails-ai 多層驗證模型，在 autonomous-harness.yaml 新增 guardrail_surface_annotations 區塊，標明現有各 HITL gate 覆蓋防護層次（input/planning/execution/output）。低風險，純文件標注。

P1：execution_sandbox_contract（E2B 借鑑）

E2B 為每個 session 定義顯式沙箱契約（timeout/allowed-domains/resource-limits）。本系統目前超時配置散落在 timeouts.yaml，缺乏集中的沙箱邊界聲明文件。在 autonomous-harness.yaml 新增 execution_sandbox_contract 聚合記錄。低風險，純文件聚合。

P2：bash curl 出站域名白名單（browser-use 借鑑）

browser-use 以 allowed_domains/exclude_urls 限制可達域名。本系統 pre_bash_guard.py 缺乏 curl 出站白名單。高風險（核心攔截層），需完整計畫後人工審核。

P3：Docker sandbox config（OpenHands 借鑑）

長期架構方向，評估為 claude -p 子進程提供容器隔離。觀察期 3 個月。

審查結論

  • P1 兩項（低風險）：可直接在 autonomous-harness.yaml 新增文件欄位，feasibility=9/stability=9，已落實
  • P2（高風險）：輸出計畫至 docs/plans/，待人工審核
  • P3（低優先）：登錄 backlog，觀察期後重評

已落實項目

  • autonomous-harness.yaml 新增 execution_sandbox_contract 區塊

工具邊界分析小結

E2B：執行環境邊界（microVM）+ 工具 API 邊界（SDK primitives）完全分離，default_mode=deny，最嚴格。

OpenHands：規劃層有 operator forbidden actions；Docker 提供執行隔離；review-only gate。

browser-use：無執行環境隔離，工具 API 邊界靠 URL 白名單，default_mode=allow，最寬鬆。

guardrails-ai：專注 LLM API 邊界的輸入/輸出驗證，不覆蓋執行環境。
