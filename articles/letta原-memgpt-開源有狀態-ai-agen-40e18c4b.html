Letta（原 MemGPT）— 開源有狀態 AI Agent 平台深度研究

專案概述

Letta（前身為 MemGPT）是一個開源的有狀態 AI Agent 建構平台，專注於讓 AI Agent 具備持久記憶與自我學習能力。與傳統聊天機器人在每次對話後遺忘所有上下文不同，Letta Agent 能記住、學習並隨時間持續改進。

核心問題與解決方案

問題：Agent 的「金魚腦」困境

當前大多數 AI Agent 框架面臨以下痛點：

  1. 無狀態對話：每次會話從零開始，喪失上下文
  2. 上下文窗口限制：即使模型支持長上下文，也無法無限記住所有歷史
  3. 知識不可累積：Agent 無法從過去的互動中學習與改進
  4. 記憶碎片化：不同會話間的記憶無法串聯

Letta 的解決方案：三層記憶架構 + 自編輯機制

Letta 借鑑了作業系統的虛擬記憶管理概念（源自 MemGPT 研究論文），實現了 AI Agent 的「虛擬無限記憶」。

技術架構：三層記憶模型

Letta 的記憶系統分為三層，類比電腦的 RAM 與磁碟儲存：

第一層：核心記憶（Core Memory / In-Context Memory）

  • 類比：RAM（隨機存取記憶體）
  • 特點：始終存在於 LLM 上下文窗口中，Agent 每次推理都能直接存取
  • 實作：以可編輯的記憶區塊（Memory Blocks）形式存在，類似可寫入的系統提示
  • 典型區塊：human（使用者偏好）、persona（Agent 人格）、goals（目標）、preferences（偏好）
  • 關鍵特性：Agent 可透過工具呼叫自主編輯核心記憶內容
# 核心記憶區塊範例
memory_blocks = [
    {"label": "human", "value": "使用者喜歡正體中文回覆，偏好技術深度分析"},
    {"label": "persona", "value": "我是一個專注於 AI 開源研究的助手"}
]

第二層：回憶記憶（Recall Memory）

  • 類比：磁碟上的交易日誌
  • 特點：保存完整的對話歷史，自動持久化到磁碟
  • 存取方式：透過搜尋工具（語意搜尋、全文搜尋、混合搜尋）按需檢索
  • 用途：當 Agent 需要回顧過去的對話時，從回憶記憶中檢索相關片段

第三層：檔案記憶（Archival Memory）

  • 類比：外部資料庫 / 長期儲存
  • 特點：儲存經過處理的結構化知識，不同於原始對話歷史
  • 儲存格式：支援向量資料庫、圖資料庫等多種後端
  • 存取方式：Agent 透過專用工具查詢與寫入
  • 用途：跨會話的長期知識積累

自編輯記憶機制（Self-Editing Memory）

這是 Letta 最具革命性的設計——Agent 自主管理自己的記憶：

  1. 主動寫入：Agent 判斷重要資訊時，自動寫入核心記憶或檔案記憶
  2. 主動搜尋：Agent 意識到需要額外資訊時，主動搜尋回憶或檔案記憶
  3. 主動更新：Agent 發現記憶過時或需要修正時，自主編輯核心記憶區塊
  4. 遞歸摘要：對話過長時，Agent 自動壓縮歷史對話為摘要，保留關鍵資訊

這種設計讓 Agent 不再被動接收記憶管理，而是像人類一樣主動決定「記住什麼、忘記什麼、何時回想」。

Agent File（.af）開放格式

Letta 提出了 Agent File（.af），一種開放標準的 Agent 序列化格式：

包含內容

組件說明
模型配置上下文窗口限制、模型名稱、嵌入規格
訊息歷史完整對話記錄
系統提示核心行為指令
記憶區塊可編輯的人格與使用者資訊
工具規則工具排序與約束定義
環境變數工具執行配置
工具定義完整原始碼與 JSON Schema

核心價值

  • 可攜性：在不同系統間遷移 Agent
  • 版本控制：追蹤 Agent 配置隨時間的變化
  • 安全性：匯出時自動將密鑰設為 null，防止憑證外洩
  • 協作：團隊成員間共享訓練過的 Agent
# Agent File 匯入範例
from letta_client import Letta
client = Letta(base_url="http://localhost:8283")
agent_state = client.agents.import_file(
    file=open("/path/to/agent.af", "rb")
)

Letta Code：記憶優先的程式開發 Agent

Letta Code 是基於 Letta 平台的程式開發助手，在 TerminalBench 基準測試中排名 #1 模型無關開源編碼 Agent：

關鍵差異化

  • 持久 Agent：不同於傳統無狀態編碼助手，跨會話保持記憶
  • 程式碼庫記憶化：/init 命令深度分析現有程式碼庫，形成記憶
  • 技能學習：將重複任務模式轉化為可重用技能（Markdown 檔案）
  • 跨會話搜尋：/search 命令支援向量、全文、混合搜尋歷史對話
  • 模型無關：支援 Claude、GPT、Gemini 等多種模型後端

記憶基準測試：檔案系統就夠了？

Letta 團隊使用 LoCoMo 基準測試研究記憶效能：

  • 測試結果：使用 GPT-4o mini + 檔案操作工具達到 74.0% 準確率，超越專門記憶工具
  • 核心洞察：「記憶更多關乎 Agent 如何管理上下文，而非具體的檢索機制」
  • 實務啟示：Agent 天生擅長檔案系統操作（grep、search_files），因為訓練資料中大量出現這類工具使用
  • 結論：Agent 能力比特定工具更重要；專門的記憶解決方案若 Agent 不會用，反而表現更差

潛在應用：與 daily-digest-prompt 專案的關聯

1. 記憶架構升級借鑑

目前 daily-digest-prompt 使用 digest-memory.json 作為跨次記憶，這是一種手動管理的扁平化記憶。Letta 的三層記憶模型提供了升級路徑：

  • 核心記憶 → 現有的 digest-memory.json（使用者偏好、連續天數等）
  • 回憶記憶 → 現有的 state/scheduler-state.json（執行歷史）
  • 檔案記憶 → 現有的知識庫（localhost:3000）

差距在於：Letta 的 Agent 會自主決定何時讀寫記憶，而目前我們是Prompt 硬編碼讀寫時機。

2. Agent File 格式的啟發

.af 格式的「Agent 可攜性」概念，可以啟發：

  • 將 Skill + 記憶 + 配置打包為可分享的 Agent 定義檔
  • 支援 Agent 版本控制與回滾
  • 團隊成員間共享調試過的 Agent 配置

3. 自編輯記憶的可行路徑

在不引入 Letta 框架的前提下，可以借鑑其「Agent 自管理記憶」的概念：

  • 在 Skill 中增加「記憶更新建議」機制，讓 Agent 主動提議更新 digest-memory.json
  • 為研究任務增加「自動摘要」機制，對話結束前壓縮關鍵發現

4. 遊戲開發 Agent 的記憶

遊戲設計任務（D:\Source\game）可以借鑑 Letta Code 的「技能學習」機制：

  • Agent 在修改遊戲程式碼時累積「設計模式記憶」
  • 跨次遊戲優化保持風格一致性

優缺點評估

優點

  1. 記憶架構領先：三層記憶 + 自編輯是目前最完整的 Agent 記憶方案
  2. 學術根基紮實：源自 MemGPT 研究論文，不是純工程拼湊
  3. Agent File 開放標準：.af 格式有潛力成為 Agent 生態的通用格式
  4. Letta Code 表現優異：TerminalBench #1 模型無關開源 Agent
  5. 模型無關：不綁定特定 LLM 供應商
  6. 社群活躍：158 位貢獻者、173 個版本、Discord 活躍社群
  7. 視覺化開發環境：ADE 提供即時事件歷史、記憶區塊視覺化

缺點

  1. 學習曲線陡峭：三層記憶模型需要理解新的心智模型
  2. 生態仍在早期：.af 格式目前僅 Letta 實作，跨框架支持有限
  3. 效能開銷：自編輯記憶需要額外的工具呼叫，增加延遲與成本
  4. .af 限制：目前不支持檔案記憶段落、資料來源附件、MCP 配置、多 Agent 檔案
  5. Python 為主：雖有 TypeScript SDK，但核心框架仍以 Python 為主
  6. 雲端依賴：Letta Cloud 功能需要 app.letta.com API Key

安裝與快速上手

CLI 安裝

npm install -g @letta-ai/letta-code

SDK 安裝

# Python
pip install letta-client

# TypeScript
npm install @letta-ai/letta-client

Hello World 範例（Python）

from letta_client import Letta

client = Letta(token="your-api-key")

agent = client.agents.create(
    model="openai/gpt-4o",
    memory_blocks=[
        {"label": "human", "value": "使用者偏好與背景"},
        {"label": "persona", "value": "AI 研究助手人格"}
    ],
    tools=["web_search", "run_code"]
)

response = client.agents.messages.create(
    agent_id=agent.id,
    messages=[{"role": "user", "content": "你好，記住我喜歡深度技術分析"}]
)
