專案觀測摘要

本次主題：agent_memory（長期記憶層框架）。觀測 4 個 GitHub 高 stars 專案。

1. mem0ai/mem0 — 59,728★（REST 2026-06-30）

  • memory_architecture：混合存儲三層（user/session/agent scope）+ intelligent layer
  • storage_backends：Qdrant、Redis、SQLite、managed cloud
  • retrieval_strategy：語義向量 + 智能優先級過濾
  • 本系統關聯：memory-policy.yaml continuity 層 quality_score 政策已引用此模型，但執行層 preamble.md 尚未強制過濾

2. getzep/graphiti — 28,152★（REST 2026-06-27）

  • memory_architecture：時態知識圖（bi-temporal: valid_time + system_time），每條 edge 含有效期
  • storage_backends：Neo4j（主）、FalkorDB（備選）
  • retrieval_strategy：混合（語義向量 + BM25 + 圖遍歷），支援時態查詢
  • 本系統關聯：continuity.finding_expires_days [STALE] 標記機制靈感來源，但 preamble.md 尚未實作自動 stale check

3. topoteretes/cognee — 25,714★（REST 2026-06-29）

  • memory_architecture：知識圖 + ECL Pipeline（Extract→Cognify→Load）+ RDF/OWL 本體驗證
  • storage_backends：向量、圖、SQL（可自托管），支援多種 vector store
  • retrieval_strategy：GraphRAG + 本體多跳查詢（HotPotQA 0.93）
  • 本系統關聯：cognee-mcp 提供 Claude Code MCP 整合點，可作為 continuity archival_tier 圖查詢後端（高風險）

4. letta-ai/letta — 23,586★（REST 2026-06-26）

  • memory_architecture：OS 式三層（core/recall/archival），agent 自主決策何者值得記憶
  • storage_backends：SQLite、PostgreSQL、多種 vector store
  • retrieval_strategy：精確（core memory context window）+ 向量（archival）+ SQL（recall）
  • 本系統關聯：continuity.archival_tier compression_rule 尚未實作 agent self-edit 壓縮邏輯
