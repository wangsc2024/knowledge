TodoWrite 工具概述

TodoWrite 是 Claude Code 的進度追蹤工具，讓 Agent 建立、更新 todo 清單來組織多步驟任務。

何時該用 TodoWrite（效益正面）

  • 互動式長時間任務：20+ 步驟的開發任務，人類可即時看到進度
  • 多檔案重構：需要追蹤哪些檔案已修改、哪些待處理
  • VSCode 互動模式：開發者直接與 Claude 對話，TodoWrite 提供可視化進度
  • 複雜除錯：需要系統性排查多個可能原因

何時該禁止 TodoWrite（效益負面或零）

  • -p pipe 模式（所有排程 Agent）：沒有人在看輸出，TodoWrite 的進度條無人觀看
  • 時間受限的 Agent（<5 分鐘）：每次 TodoWrite 觸發一個完整 LLM 回合（8-10 秒），浪費寶貴預算
  • 步驟已明確寫在 prompt 中：Agent 不需要用 todo 重新自我提醒已知步驟
  • Phase 1 fetch agents：單一明確任務（查詢 API → 寫入結果），無需追蹤

實測數據（2026-02-15 todoist-team）

在 180 秒預算的 todoist-query Agent 中：

  • Agent 自動建立 4 次 TodoWrite，內容為自我組織的工作步驟
  • 4 次 TodoWrite 觸發 4 個不必要的 LLM 回合，共浪費約 92 秒（佔預算 51%）
  • 直接導致 Phase 1 超時失敗（連續 11 次 100% 失敗率）
  • 修復方式：在 prompt 中加入「禁止使用 TodoWrite」效率規則

Prompt 中禁止 TodoWrite 的範例

## 效率規則（必遵守）


    

  • 禁止使用 TodoWrite：本 Agent 有 N 秒時限，TodoWrite 每次浪費 10 秒
    • 

  • 最小工具呼叫：減少不必要的 Bash/Read 呼叫

設計原則

TodoWrite 的價值 = 可視化進度 × 觀眾存在 × 任務複雜度 ÷ 時間成本。當任何一個因子為零時，TodoWrite 的淨效益就是負的。
