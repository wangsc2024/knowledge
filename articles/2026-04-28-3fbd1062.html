方案 A：MigrationWatch PSObject null-guard

目標：修復每輪 todoist-team 執行時 [MigrationWatch] Exception: The property 'task_key' cannot be found on this object

根本原因：state/quality-trend.json 有聚合摘要條目（date/avg_score 格式，無 task_key），Where-Object { $_.task_key -eq $watchTask } 在 PS7 中遇到無此屬性的 PSCustomObject 時拋出異常。

修改：run-todoist-agent-team.ps1 L9154/L9162

  • Before: Where-Object { $_.task_key -eq $watchTask }
  • After: Where-Object { $null -ne $_ -and $_.PSObject.Properties['task_key'] -ne $null -and $_.task_key -eq $watchTask }
  • 同步：L9190 MigrationWatch ntfy URL https://ntfy.sh → https://ntfy.sh/wangsc2025

驗收：✓ 消除每輪 ERROR log；✓ 57任務一致性通過；✓ 1278 tests 通過（4 pre-existing 失敗非本次引入）

方案 B：ntfy runaway session 告警 HTTP=400 修復

目標：修復 run-token-efficiency.ps1 runaway session 偵測時 ntfy 告警 HTTP=400 靜默失效

根本原因：curl URL 為 https://ntfy.sh（無 topic 路徑），即使 JSON payload 含 topic 欄位，ntfy.sh 根端點回傳 HTTP=400。

修改：run-token-efficiency.ps1 L209/L236/L303

  • Before: "https://ntfy.sh"
  • After: "https://ntfy.sh/$NtfyTopic"

驗收：✓ URL 明確帶 topic；✓ runaway/caveman drift/podcast guard 三路告警均修復
