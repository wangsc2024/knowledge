AI Agent 長期記憶架構 2026：Mem0、Zep、Letta 三大系統的設計哲學、效能比較與生產部署實踐

執行摘要

2026 年，AI Agent 記憶系統已從「可選功能」演化為「一等公民架構組件」。以 Mem0（48K GitHub stars）、Zep（時序知識圖）、Letta/MemGPT（OS 比喻的三層記憶）為代表的三大框架，各自凝結了截然不同的設計哲學：Letta 讓 LLM 自主管理記憶如同作業系統；Zep 透過 Graphiti 引擎保留時間關係，在 LongMemEval benchmark 超越 Mem0 達 14.8 個百分點（63.8% vs 49.0%）；Mem0 以混合存儲（向量+知識圖+KV）達成 80% token 節省，聚焦消費者個性化。

生產部署標準已收斂至雙層異步架構：Hot Path（即時 context + 摘要圖狀態）加上 Cold Archive（語義相似搜尋），記憶寫入強制非同步以避免 latency 惡化。LangGraph 整合方面，checkpointer（thread-level）搭配 external store（cross-thread）已有 Redis、MongoDB 成熟方案。

最新研究（MemMachine，2026-04）顛覆了「提取優先」的主流思路：原始情節保存（ground-truth preserving）配合 retrieval-stage 優化，在 LoCoMo benchmark 達 SOTA 0.9169，同時節省 78% token。本報告整合 11 篇來源，提供可直接應用於 daily-digest-prompt 系統的記憶架構建議。

三大架構哲學的本質差異

Letta（MemGPT）採用 LLM 作業系統模型：Core Memory（始終在 context，可自編輯）、Archival Memory（外部向量存儲）、Recall Memory（對話歷史）。Agent 透過工具呼叫自主決定何時讀寫記憶。DMR Benchmark MemGPT 93.4%。

Zep 採用時序知識圖架構：Graphiti 引擎動態合成對話數據與結構化商業數據，保留時間關係。DMR Benchmark 94.8%；LongMemEval +18.5% accuracy；latency 降低 90%；LongMemEval 63.8%。

Mem0 採用混合存儲：三層記憶範圍（user/session/agent），混合存儲（向量+知識圖+KV），80% token reduction，48K GitHub stars，LongMemEval 49.0%。

生產部署的雙層異步架構

三路徑設計：Hot Path（即時 context + 摘要圖）→ 回應；異步寫入 Warm Memory（結構化事實，高速資料庫）；異步寫入 Cold Archive（壓縮 session 日誌）。非同步記憶寫入（async by default）；每次記憶寫入等於一次 embedding 調用，是最高頻 GPU 操作；外部記憶只適合多 session 持久任務。

LangGraph 外部記憶整合最佳實踐

Short-term checkpointer：SqliteSaver、RedisSaver、AsyncRedisSaver，thread-level state 持久化。Long-term store：RedisStore、MongoDB Store，cross-thread 共享記憶，以 namespace + key 組織。成本優化：僅在邏輯斷點 checkpoint；message trimming + summary nodes；Mem0 處理語義記憶，LangGraph checkpointer 處理執行狀態。

記憶衝突與智慧遺忘機制

記憶衝突：時間加權（指數時間衰減懲罰）、語義合併（arbiter agent 生成時間界限調和陳述）、多 Agent 訪問控制（隔離層級）。智慧遺忘：TTL 分層（永久/7-30天）、Refresh-on-Read（使用時重置衰減計時器）、Importance Scoring（FadeMem）。「無限保留是架構缺陷」已成共識。

MemMachine：Retrieval-First 新範式（2026-04，SOTA）

儲存原始情節（ground-truth preserving）+ 句子級索引，最小化 LLM 依賴。LoCoMo 0.9169（SOTA），LongMemEvalS 93%，Token 比 Mem0 少 78%。反直覺發現：GPT-4.1-mini 配合優化提示詞效能超越 GPT-4.1。Retrieval stage 優化（+4.2%）遠大於 ingestion 優化（+0.8%）。

關鍵洞見

  1. 架構哲學決定適用場景（已驗證）：Zep→時序推理，Mem0→消費者個性化，Letta→自主複雜代理
  2. 非同步是生產必需（已驗證）：async mode 已成 2026 預設
  3. LongMemEval 成為記憶 benchmark 標準（已驗證）：Zep 63.8% > Mem0 49.0%
  4. 「無限保留是架構缺陷」已成共識（已驗證）：TTL 分層 + Refresh-on-Read + Importance Scoring
  5. LangGraph 記憶整合成熟（已驗證）：Redis/MongoDB 均有生產案例
  6. MemMachine retrieval-first 範式（待驗證）：比 Mem0 節省 78% token

實務應用（daily-digest-prompt 視角）

continuity/ 記憶層可加入 TTL 分層與 Refresh-on-Read。research-registry.json 可增 importance_score 欄位延長高品質研究主題 TTL。LangGraph cutover 時推薦 RedisSaver + RedisStore。多 Agent 並行記憶衝突可參考 Zep arbiter agent 模式。

未來展望

Retrieval-first 取代 Extraction-first；記憶走向 managed service；多模態記憶整合（Memvid）。新興系統：EverMemOS、MAGMA、MemOS。
