不完全資訊博弈中的 AI 決策

日期：2026-03-02 | 研究觸發：矮人礦坑桌遊分析 + Test-Time Compute Scaling 研究
核心問題：當 AI 面對隱藏資訊、對手意圖不確定時，如何做出最優決策？

---

一、技術概述

不完全資訊博弈（Imperfect Information Game）是指參與者無法完全觀察到遊戲狀態的博弈，與圍棋、象棋等完全資訊遊戲形成鮮明對比。典型案例包括撲克牌（看不到對手手牌）、矮人礦坑桌遊（不知道誰是破壞者）、狼人殺/間諜等社交推理遊戲。

這類問題的核心挑戰在於：(1) 資訊不對稱 — 每位玩家持有私有資訊（private information），決策必須在不確定性下進行；(2) 策略欺騙 — 最優策略可能需要虛張聲勢（bluffing）或隱藏真實意圖；(3) 博弈樹爆炸 — 由於資訊集（information set）的存在，狀態空間遠大於同規模的完全資訊遊戲。在 AI 研究中，不完全資訊博弈被視為比圍棋更困難的挑戰，因為 AlphaGo 式的 MCTS（蒙地卡羅樹搜尋）無法直接套用 — 你不能假裝知道對手的手牌來做搜尋。

---

二、核心概念與原理

2.1 資訊集（Information Set）

在完全資訊遊戲中，每個節點代表一個確定狀態。在不完全資訊遊戲中，玩家可能無法區分多個可能的遊戲狀態 — 這些不可區分的狀態集合稱為「資訊集」。例如在德州撲克中，你知道自己的手牌和公共牌，但對手的手牌有多種可能，所有這些可能性構成一個資訊集。

AI 的策略必須在「資訊集」層級做決策（而非單一狀態），這是與完全資訊遊戲 AI 的根本差異。

2.2 納許均衡與 Exploitability

在雙人零和不完全資訊遊戲中，最優策略是納許均衡（Nash Equilibrium）— 即使對手知道你的策略也無法利用你。衡量策略品質的指標是 exploitability：偏離納許均衡的程度，以可被利用的期望損失計算。

2.3 Counterfactual Regret Minimization（CFR）

CFR 是解決不完全資訊博弈的核心演算法，由 Martin Zinkevich 等人於 2007 年提出。核心思想：

  1. 反事實後悔值（Counterfactual Regret）：在每個資訊集中，計算「如果當時選了另一個動作，期望收益會增加多少」
  2. 累積後悔值：透過自我對弈（self-play）反覆迭代，累積每個動作的後悔值
  3. 策略更新：使用 regret-matching — 下一輪的動作機率正比於正向累積後悔值
  4. 收斂保證：平均策略在迭代次數趨近無窮時收斂到納許均衡

CFR 虛擬碼：


def cfr(history, player, reach_probabilities):
    if is_terminal(history):
        return utility(history, player)
    
    info_set = get_information_set(history, current_player)
    strategy = regret_matching(info_set.cumulative_regret)
    
    action_values = {}
    node_value = 0
    
    for action in legal_actions(history):
        new_reach = update_reach(reach_probabilities, strategy[action])
        action_values[action] = cfr(history + action, player, new_reach)
        node_value += strategy[action] * action_values[action]
    
    if current_player == player:
        for action in legal_actions(history):
            regret = action_values[action] - node_value
            info_set.cumulative_regret[action] += reach_opponent * regret
    
    return node_value

2.4 CFR 變體演進

| 變體 | 年份 | 關鍵改進 | 代表應用 |

|------|------|---------|----------|

| Vanilla CFR | 2007 | 原始版本，遍歷完整遊戲樹 | 小型撲克 |

| MCCFR | 2009 | 蒙地卡羅取樣，不需遍歷全樹 | 中型撲克 |

| CFR+ | 2014 | 負後悔值歸零（而非累積），收斂更快 | 完美解決 Heads-up Limit Hold'em |

| Deep CFR | 2019 | 用深度神經網路近似後悔值和策略 | 大規模撲克 |

| DCFR | 2019 | 折扣 CFR — 對舊迭代施加衰減權重 | 實務訓練加速 |

| Dream | 2020 | 深度後悔最小化 + 經驗回放 | 多玩家遊戲 |

三、里程碑系統深度分析

3.1 DeepStack（2017）— 首個擊敗職業玩家的撲克 AI

  • 架構：結合 CFR 自我博弈 + 深度神經網路（價值估計）+ 有限前瞻搜尋
  • 關鍵創新：「持續再解算（continual re-solving）」— 不預計算完整策略，而是在每個決策點即時求解子賽局
  • 深度直覺網路：訓練 7 層前饋神經網路，輸入公共牌+下注歷史，預估反事實價值，取代 CFR 對深層子樹的遍歷
  • 成就：在 Heads-up No-Limit Texas Hold'em 中擊敗多位職業玩家

3.2 Libratus（2017）— CMU 的撲克 AI 冠軍

  • 三階段架構：

1. 預計算（Blueprint Strategy）：用 MCCFR 在抽象化遊戲樹上訓練數百萬核心小時，得到藍圖策略

2. 子博弈即時求解：遊戲進行中，用 unsafe subgame solving 技術在真實（未抽象化）子博弈中求解精確策略

3. 自我改進：每晚分析白天對局，修補被對手利用的策略弱點

  • 成就：20 天對局中以 +14.7 bb/100 的統計顯著優勢擊敗四位頂級職業玩家

3.3 Pluribus（2019）— 首個多人撲克超人 AI

  • 核心突破：從雙人零和擴展到六人遊戲（非零和），納許均衡理論不再直接適用
  • 技術方案：

1. 修改版 MCCFR 訓練藍圖策略（512 CPU 核心，8 天，每日 CPU 成本約 $150）

2. Depth-Limited Search：不搜尋到遊戲結束，而是用藍圖策略估計葉節點價值

3. Real-Time Search with Modified Strategy：搜尋時假設所有對手使用藍圖策略的修改版本（而非最優回應），避免多人環境的策略發散

  • 效能：對戰包含多位世界冠軍在內的頂級職業玩家，具統計顯著優勢
  • 計算效率：推論僅需 2 CPU + 128GB RAM，無需 GPU，展示大規模商業應用的可行性

3.4 ReBeL（2020）— Meta AI 的通用化框架

  • 核心思想：Regularized Nash Dynamics (RNaD) + Belief-based Learning，將 AlphaZero 範式推廣到不完全資訊遊戲
  • Public Belief State (PBS)：將不完全資訊遊戲轉化為完全資訊遊戲 — PBS 是所有玩家手牌機率分布的向量，從公共角度可完全觀測
  • Self-Play + Value Network：類似 AlphaZero，用自我對弈 + 神經網路學習 PBS 的價值和策略
  • 通用性：同一框架適用於撲克、Liar's Dice 等多種遊戲，不需遊戲特定的手工抽象化

3.5 Student of Games（2023）— DeepMind 統一框架

  • 野心：建立同時處理完全資訊和不完全資訊遊戲的單一演算法
  • 結合 MCTS + CFR：在搜尋樹中，機會節點用 CFR 處理，確定節點用 MCTS 處理
  • 在國際象棋、圍棋、撲克等遊戲上展現競爭力，但未在任何單一遊戲上達到 SOTA

四、LLM 在社交推理遊戲中的應用

4.1 狼人殺 / Werewolf with LLM

2023-2025 年，多項研究探索用 LLM 玩社交推理遊戲：

  • 自然語言推理：LLM 可以用自然語言進行「誰是狼人」的推理、投票辯論、偽裝無辜
  • Theory of Mind（心智理論）：LLM 展現出初步的心智理論能力 — 推測其他玩家的信念和意圖
  • 挑戰：

- 策略一致性：LLM 傾向過於誠實，不擅長長期欺騙策略維持

- 記憶與追蹤：上下文視窗限制使長局遊戲的資訊追蹤困難

- 多回合規劃：LLM 偏好短視決策，缺乏多回合策略規劃能力

4.2 Avalon/Resistance 與 LLM Agent

  • MIT/Meta 等團隊的研究顯示，GPT-4 / Claude 等 LLM 在 Avalon 遊戲中：

- 可進行基本的社交推理（誰投了反對票 → 可能是間諜）

- 但在需要「計算精確機率」的場景下明顯不如 CFR 方法

- 混合架構最有前景：LLM 處理自然語言溝通 + CFR/博弈論引擎處理策略計算

4.3 Test-Time Compute 在社交推理中的潛力

結合今日研究的 Test-Time Compute Scaling：

  • 推理時計算擴展 讓 LLM 在決策前「深入思考」，可能改善社交推理遊戲中的策略品質
  • Process Reward Model 可以引導 LLM 的推理鏈，避免邏輯跳躍
  • Budget Forcing 確保足夠的思考深度，而非衝動決策

五、最佳實踐與常見陷阱

最佳實踐

  1. 抽象化是關鍵：大型遊戲必須先做資訊抽象（合併相似的資訊集）和動作抽象（離散化連續動作空間），否則遊戲樹不可處理
  2. 自我對弈收斂：CFR 類演算法需要大量自我對弈迭代，但收斂速度可透過 CFR+/DCFR 等變體加速 10-100 倍
  3. 即時搜尋比預計算更重要：Libratus/Pluribus 的經驗顯示，遊戲進行中的即時子博弈求解品質決定了實戰表現
  4. 不要追求完美均衡：在多人遊戲中，近似均衡 + 利用對手弱點的混合策略更實用
  5. 計算效率設計：Pluribus 證明了 CPU-only 的方案可以達到超人水準，不需昂貴 GPU 推論

常見陷阱

  1. 直接套用 MCTS：在不完全資訊遊戲中直接用 MCTS 會導致「策略透明化」— 你的搜尋隱含假設你知道對手的牌
  2. 忽略資訊洩漏：子博弈求解如果不考慮「安全性」（safe subgame solving），可能產生比藍圖策略更差的結果
  3. 過度抽象化：抽象化層級過粗會丟失關鍵的策略細節，Libratus 的成功部分歸功於在子博弈中恢復精細粒度
  4. 多人擴展陷阱：雙人零和的理論保證不適用於多人環境，需要新的均衡概念和訓練方法

六、與本專案的關聯

6.1 遊戲開發應用（D:\\Source\\game）

本專案的 creative-game-optimize 自動任務負責優化復古遊戲。若未來開發含 AI 對手的桌遊（如數位版矮人礦坑），可應用：

  • CFR-Lite：簡化版 CFR 用於小規模隱藏角色遊戲的 AI 決策
  • Belief Tracking：追蹤 AI 對其他玩家身分的信念分布，隨遊戲進展更新
  • 行為多樣性：用混合策略避免 AI 行為過於可預測

// 簡化版 Belief Tracking 範例（矮人礦坑場景）
class SaboteurAI {
  constructor(playerCount) {
    this.beliefs = {};
    const saboteurPrior = 3 / playerCount;
    for (let i = 0; i < playerCount; i++) {
      this.beliefs[i] = saboteurPrior;
    }
  }

  updateBelief(playerId, action) {
    const likelihood = this.actionLikelihood(action, 'saboteur');
    const priorSab = this.beliefs[playerId];
    const evidence = likelihood * priorSab + 
                     this.actionLikelihood(action, 'miner') * (1 - priorSab);
    this.beliefs[playerId] = (likelihood * priorSab) / evidence;
  }

  actionLikelihood(action, role) {
    if (action.type === 'block_path' && role === 'saboteur') return 0.7;
    if (action.type === 'block_path' && role === 'miner') return 0.1;
    if (action.type === 'extend_path' && role === 'miner') return 0.8;
    if (action.type === 'extend_path' && role === 'saboteur') return 0.3;
    return 0.5;
  }
}

6.2 Agent 系統設計啟示

daily-digest-prompt 的多 Agent 團隊模式可借鑒不完全資訊博弈的思想：

  • 資訊隔離：Phase 1 的五路並行 Agent 各自擁有不同的資訊片段，Phase 2 組裝 Agent 需要「融合」這些部分觀測
  • 不確定性下的決策：當 API 失敗時，Agent 需在不完全資訊下決策（用快取？跳過？重試？），類似不完全資訊博弈的決策邏輯
  • 信念追蹤：Todoist 路由中的 TaskSense 計分本質上是一個信念系統 — 根據標籤、描述、截止時間等觀測信號推斷任務的真實優先級

6.3 Test-Time Compute 的交叉應用

今日另一項研究「Test-Time Compute Scaling」與不完全資訊博弈的交集：

  • DeepStack 的「持續再解算」本質上就是 test-time compute — 在推論時投入更多計算來求解更精確的策略
  • Pluribus 的 real-time search 也是同樣概念：訓練時用粗略藍圖，推論時用精確搜尋
  • 這種「訓練粗 + 推論精」的模式也適用於 LLM 的推理時計算擴展

七、前沿趨勢（2025-2026）

  1. LLM + 博弈論混合架構：用 LLM 處理自然語言互動，用 CFR 引擎處理策略計算，兩者透過介面整合
  2. 多模態不完全資訊：結合視覺（表情識別）、語音（語調分析）的多模態社交推理
  3. 可解釋的策略推理：讓 AI 的社交推理過程可解釋
  4. Real-Time Adaptation：從固定均衡策略轉向即時適應對手風格的動態策略
  5. 大規模多人環境：從 2-6 人撲克擴展到 MMO 經濟、外交協商等大規模不完全資訊場景
