Browser-Use：讓 AI Agent 自動操控瀏覽器的開源框架

專案概述

Browser-Use 是一個開源 Python 框架，核心目標是讓 AI Agent 能像人類一樣操控網頁瀏覽器。它解決了 LLM 雖然擁有推理能力，卻無法直接與網頁互動的痛點。透過 Playwright 瀏覽器自動化引擎與 LLM 的結合，Browser-Use 讓 AI Agent 能夠填寫表單、網路購物、搜尋資訊、操作 SaaS 工具等，實現真正的 Web 自動化。

技術架構

核心模組分析

Browser-Use 採用模組化設計，主要元件：

模組職責
agent/AI Agent 核心邏輯（service.py 為主入口、prompts.py 定義系統提示、message_manager/ 管理對話歷史、judge.py 任務完成判定）
browser/Playwright 瀏覽器封裝（頁面管理、元素互動、截圖）
dom/DOM 解析與元素定位（將網頁結構轉為 LLM 可理解的格式）
controller/Agent 動作控制器（點擊、輸入、滾動、導航等）
llm/LLM 提供者抽象層（支援多家模型）
tools/自定義工具擴展系統
mcp/Model Context Protocol 整合
sandbox/雲端沙箱執行環境
skill_cli/CLI 命令列工具（持久化瀏覽器控制）
integrations/第三方整合（如 Claude Code Skill）

設計決策

  1. Playwright 為底層引擎：選擇 Playwright 而非 Selenium，因其更好的現代瀏覽器支援、自動等待機制、多頁面管理。
  2. DOM 結構化提取：將網頁 DOM 轉為帶索引的互動元素列表，LLM 只需指定元素索引即可操作。
  3. ChatBrowserUse 專用模型：針對瀏覽器自動化場景專門優化的 LLM，號稱比通用模型快 3-5 倍。
  4. Actor 模式：agent/actor 分離，Agent 負責決策，Actor 負責執行。
  5. Variable Detector：自動偵測 Agent 產出的變數（如搜尋結果、價格等），便於後續流程使用。

獨特功能

  1. CLI 持久化瀏覽器：browser-use open/state/click/type/screenshot/close — 瀏覽器在命令間保持開啟，快速迭代。
  2. Sandbox 裝飾器：@sandbox() 一行程式碼即可將 Agent 部署到雲端隔離環境。
  3. Templates 系統：uvx browser-use init --template default/advanced/tools 快速產生範例。
  4. Claude Code Skill：官方提供 SKILL.md，可直接安裝到 Claude Code 使用。
  5. Judge 系統：自動判定任務是否完成，減少無限迴圈。

使用方式

快速上手（5 步驟）

from browser_use import Agent, Browser, ChatBrowserUse
import asyncio

async def example():
    browser = Browser()
    llm = ChatBrowserUse()
    agent = Agent(
        task="Find the number of stars of the browser-use repo",
        llm=llm,
        browser=browser,
    )
    history = await agent.run()
    return history

asyncio.run(example())

自定義工具擴展

from browser_use import Tools

tools = Tools()

@tools.action(description='Save data to database.')
def save_to_db(data: str) -> str:
    # 自定義邏輯
    return f"Saved: {data}"

agent = Agent(task="Your task", llm=llm, browser=browser, tools=tools)

潛在應用（Daily-Digest-Prompt 系統）

直接應用場景

  1. 增強 WebFetch 能力：目前 daily-digest-prompt 的 WebFetch 只能抓取靜態頁面。Browser-Use 可處理需要 JavaScript 渲染、登入驗證的動態頁面。
  2. Todoist Web UI 操作：某些 Todoist 進階功能（如看板視圖、時間線）只有 Web 才有，Browser-Use 可自動化這些操作。
  3. 自動化研究流程：目前研究任務依賴 WebSearch + WebFetch，Browser-Use 可實現更複雜的多步驟研究（如登入學術網站、下載 PDF、截圖保存）。
  4. GitHub 操作增強：自動化 PR 審查、Issue 分類等 Web UI 操作。
  5. 表單填寫自動化：自動填寫政府報表、線上申請等。

優缺點評估

優點

  • MIT 授權，完全開源免費
  • Python 原生，與 daily-digest-prompt 技術棧一致
  • 極高的開發活躍度（每週多次發版）
  • 支援多家 LLM（不限定 OpenAI）
  • MCP 原生整合，與 Claude Code 生態無縫銜接
  • @sandbox 和 CLI 降低部署門檻
  • 自定義工具系統靈活

缺點

  • 依賴 LLM API 呼叫，每次瀏覽器操作都產生費用
  • ChatBrowserUse 專用模型需要付費（$0.20/1M input, $2.00/1M output）
  • 瀏覽器啟動耗資源（Chromium 記憶體佔用大）
  • 版本迭代過快，API 穩定性可能有風險
  • 246 個開放 Issues 暗示仍有不少 edge case
  • 隱私風險：Agent 操控瀏覽器意味著可能觸及敏感資料

風險評估

  • 成本控制：每次任務可能消耗大量 token（DOM 結構可能非常長），需設定 token 預算
  • 安全性：需要在沙箱環境中運行，避免 Agent 誤操作（如誤刪資料、誤發訊息）
  • 維護成本：版本更新頻繁，需定期跟進升級
