聚類分析基礎概念

聚類分析（Clustering Analysis）是無監督學習的重要分支，目標是將相似的數據點分組，而將不相似的點分離。與分類（Classification）不同，聚類不需要預先標註的訓練資料。

聚類方法分類

聚類算法主要分為四大類 [GeeksforGeeks, 2026; CodeSignal, 2026]：

  1. 基於劃分（Partitioning）：如 K-Means，需預先指定簇數
  2. 層次聚類（Hierarchical）：如 Agglomerative Clustering，建立樹狀結構
  3. 密度聚類（Density-based）：如 DBSCAN、HDBSCAN，識別高密度區域
  4. 基於網格（Grid-based）：將空間劃分為網格單元

本文聚焦於密度聚類，因其在異常偵測領域的特殊優勢。

DBSCAN 演算法原理

什麼是 DBSCAN？

DBSCAN（Density-Based Spatial Clustering of Applications with Noise）是由 Martin Ester 等人於 1996 年提出的密度聚類演算法 [Wikipedia, 2026; DataCamp, 2026]。其核心思想是：簇（Cluster）是數據空間中由低密度區域分隔的高密度連續區域 [Domino Data Lab, 2026]。

核心概念與參數

DBSCAN 需要兩個主要參數 [DataCamp, 2026; GeeksforGeeks, 2026; scikit-learn, 2026]：

  • ε (epsilon)：鄰域半徑，定義「鄰近」的最大距離
  • MinPts：形成密集區域所需的最小點數（scikit-learn 中稱為 min_samples）

基於這兩個參數，DBSCAN 將數據點分為三類 [DataCamp, 2026; GeeksforGeeks, 2026]：

  1. 核心點（Core Points）：在半徑 ε 內至少有 MinPts 個鄰居的點
  2. 邊界點（Border Points）：在核心點的 ε 鄰域內，但自身鄰居數 < MinPts
  3. 噪音點（Noise Points）：既非核心點也非邊界點，標記為 -1

演算法執行步驟

  1. 隨機選取一個未訪問的點 p
  2. 若 p 的 ε-鄰域內點數 ≥ MinPts → p 是核心點，建立新簇
  3. 將 p 的所有密度可達（density-reachable）點加入該簇
  4. 重複 1-3 直到所有點被訪問
  5. 未被分配到任何簇的點標記為噪音 [Baeldung, 2026; Built In, 2026]

密度可達（Density-Reachability）：若能透過一系列核心點的鄰域「跳躍」從點 p 到達點 q，則 q 是 p 的密度可達點 [CodeSignal, 2026]。

DBSCAN 的優勢

相較於 K-Means 等中心式聚類，DBSCAN 具有以下優勢 [DataCamp, 2026; KDnuggets, 2020]：

  • 不需預先指定簇數：適合探索性數據分析
  • 可發現任意形狀簇：不限於凸形或球形簇
  • 自動識別噪音：內建異常偵測能力
  • 對初始值不敏感：結果穩定性高

HDBSCAN：DBSCAN 的進化版

HDBSCAN 的核心改進

HDBSCAN（Hierarchical DBSCAN）是 DBSCAN 的強化版，由 Campello 等人提出 [Medium, Amit Yadav, 2026; Daily Dose of DS, 2026]。其核心改進包括：

#### 1. 處理變密度簇

DBSCAN 假設局部密度均勻，對變密度數據表現不佳 [Medium, Amit Yadav, 2026]。HDBSCAN 透過探索多種密度尺度（varying epsilon values）解決此問題，能同時識別稀疏和稠密的簇 [Daily Dose of DS, 2026]。

#### 2. 參數簡化

DBSCAN 需要兩個參數（ε 和 MinPts），而 ε 的選擇極為關鍵且困難 [Medium, Amit Yadav, 2026]。HDBSCAN 僅需 min_cluster_size 單一參數，大幅簡化調參難度 [Daily Dose of DS, 2026; stataiml, 2026]。

#### 3. 層次結構與自動選簇

HDBSCAN 建立簇的層次結構，透過穩定性分析自動確定最優簇數 [Medium, Amit Yadav, 2026; Daily Dose of DS, 2026]。

#### 4. 新點預測能力

DBSCAN 不支持新點預測，而 HDBSCAN 可基於已訓練模型預測新點的簇歸屬 [Medium, Amit Yadav, 2026; hdbscan.readthedocs.io, 2026]。

#### 5. 穩定性

HDBSCAN 對執行次數和子採樣具有良好穩定性，參數選擇更穩健 [Medium, Amit Yadav, 2026]。

DBSCAN vs HDBSCAN 效能權衡

執行速度：DBSCAN 明顯快於 HDBSCAN。在某些數據集上，DBSCAN 約 18 秒，而 HDBSCAN 需 11 分鐘 [GitHub Issue #504, scikit-learn-contrib/hdbscan, 2026]。

適用場景 [stataiml, 2026]：

  • 使用 DBSCAN：均勻密度數據、效能優先、簡單場景
  • 使用 HDBSCAN：變密度數據、複雜真實世界數據、需減少手動調參

scikit-learn 實作指南

DBSCAN 實作


from sklearn.cluster import DBSCAN
import numpy as np

# 建立 DBSCAN 模型
clustering = DBSCAN(eps=3, min_samples=2)

# 擬合數據
clustering.fit(data)

# 獲取簇標籤（-1 表示噪音）
labels = clustering.labels_

# 核心樣本索引
core_samples_mask = np.zeros_like(labels, dtype=bool)
core_samples_mask[clustering.core_sample_indices_] = True

[scikit-learn 官方文件, 2026; DataCamp, 2026]

關鍵屬性

  • labels_：每個樣本的簇標籤，-1 表示噪音點
  • core_sample_indices_：核心樣本的索引
  • components_：核心樣本的副本

參數選擇建議

eps 選擇：最關鍵參數，可使用 k-distance 圖法 [DataCamp, 2026; GeeksforGeeks, 2026]：

  1. 計算每個點到第 k 近鄰的距離（k = min_samples）
  2. 繪製 k-distance 圖並排序
  3. 圖中的「肘點」即為建議 eps 值

min_samples 選擇 [DataCamp, 2026]：

  • 值越高 → 越稠密的簇（較少噪音）
  • 值越低 → 越稀疏的簇（較多小簇）
  • 經驗法則：從 2×維度數 開始嘗試

聚類於異常偵測的應用

為什麼聚類適合異常偵測？

聚類於異常偵測的核心邏輯是：異常值應屬於小且稀疏的簇，或遠離其所屬簇的中心 [GitHub, xbeat/Machine-Learning, 2026; Medium, CBLOF, 2026]。

DBSCAN 的天然異常偵測能力

DBSCAN 的噪音點（labels_ == -1）即為異常值 [GeeksforGeeks, 2026; ProjectPro, 2026]。這使得 DBSCAN 成為無監督異常偵測的理想選擇 [Analytics Vidhya, 2023]。

其他聚類異常偵測算法

  1. Local Outlier Factor (LOF)：密度異常偵測，與 DBSCAN 相似 [ProjectPro, 2026]
  2. Cluster-Based Local Outlier Factor (CBLOF)：結合局部距離與簇大小 [Medium, Chris Kuo, 2026]
  3. Isolation Forest：樹狀結構異常偵測 [ProjectPro, 2026]

Python 函式庫

PyOD（Python Outlier Detection）：多變量異常偵測函式庫，支援監督與無監督學習 [GitHub, xbeat/Machine-Learning, 2026]。
