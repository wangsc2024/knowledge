daily-digest-prompt優化validate_config資訊安全timeouts

範圍

本輪針對 daily-digest-prompt：配置驗證正確性（避免誤報／漏同步）與 timeout 單一真相對齊。未改動業務 prompt／頻率規則語意，僅修 hook 路徑推算與 YAML 缺鍵。

發現與根因

  1. AutoResearch 工具鏈「偽缺失」

hooks/validate_config.py 內 _validate_autonomous_prompt_toolchain 使用 os.path.dirname(config_dir) 當 repo 根。整合測試 TestSyntheticConfigs.test_valid_minimal_config 將 YAML 寫在 pytest tmp_path 根目錄時，dirname 指到臨時目錄上層而非倉庫根，導致誤判 tools/prompt_format_lint.py 等五檔缺失（實際檔案於倉庫內存在）。

  1. timeout 交叉驗證失敗（真實配置）

config/frequency-limits.yaml 已含 sef_evolution_full.timeout_seconds: 900，但 config/timeouts.yaml → todoist_team.phase2_timeout_by_task 缺同名鍵，check_timeout_sync（hooks/validate_config.py）回報缺失（TestTimeoutSync.test_real_config_synced）。

已完成變更（路徑）

| 項目 | 路徑 | 說明 |

|------|------|------|

| Repo 根推算 | hooks/validate_config.py _validate_autonomous_prompt_toolchain | 改以 __file__ 所在 hooks/ 之上一層為倉庫根，與 validate_config(None) 預設 --/config 路徑語意一致，合成 config 目錄測試不再誤報。 |

| Phase2 逾時 | config/timeouts.yaml | 新增 sef_evolution_full: 900，與 frequency-limits 900s 對齊。 |

測試證據

  • uv run pytest tests/hooks/test_validate_config.py::TestSyntheticConfigs::test_valid_minimal_config tests/hooks/test_validate_config.py::TestTimeoutSync::test_real_config_synced → 2 passed（約 0.5s）。
  • uv run pytest tests/hooks -q → 1005 passed（約 10.6s，本機 Win32 / Python 3.11.13）。

資訊安全與治理（含外部參考）

  • 本專案將 frequency-limits.timeout_seconds 與 timeouts.phase2_timeout_by_task 強制交叉檢查，屬於組態層 單一真相＋防止靜默漂移；與 OWASP 供應鏈／建置防線中「在建置與部署前驗證產物與設定完整性」之方向一致（參考：、CI/CD 風險類別 ）。
  • AutoResearch 工具鏈存在性檢查可避免管線在缺檔時靜默繼續；修正路徑推算後，錯誤訊息僅在 真實缺檔 時才出現，降低誤判造成的「警報疲勞」（不利於安全營運）。

剩餘／未涵蓋

  • 全套 pytest（>5k cases）未於本輪跑滿；本輪證據以 hooks 全套件為主。
