二、深度分析发现

🔴 **P0-Critical：Token 用量危机**

#### 问题描述

  • 今日总用量：15.19M billable tokens / 164.11 USD
  • cache_read_tokens 异常高：103.59M tokens（vs 仅 18.6K input tokens，比例 5560:1）
  • 超预算比例：314%（1.5M 日预算）

#### 根因分析

通过 behavior-patterns.json 分析，发现以下文件被超高频读取：

| 文件 | 读取次数 | 总字符数 | 影响 |

|------|---------|---------|------|

| preamble.md | 3390次 | 271,383 | 每个 agent 都读取共用前言 |

| auto-tasks-today.json | 2723次 | 218,151 | 频率追踪文件反复读取 |

| todoist.json (cache) | 2172次 | 150,477 | 快取文件反复验证 |

| continuity/todoist-hourly.json | 1750次 | 158,084 | 连续性追踪 |

| research-registry.json | 1601次 | 149,144 | 研究去重检查 |

累计影响：仅这 5 个文件就贡献了超过 1M chars 的反复读取，导致大量 cache_read_tokens。

#### 优化建议

方案 A：Phase 0 预读机制（推荐）


# 在 run-todoist-agent-team.ps1 Phase 0 加入：
$PreloadContext = @{
    preamble = Get-Content "templates/shared/preamble.md" -Raw
    auto_tasks = Get-Content "context/auto-tasks-today.json" -Raw | ConvertFrom-Json
    research_reg = Get-Content "context/research-registry.json" -Raw | ConvertFrom-Json
}
# 通过环境变量传递给 agent，减少重复读取

预期效果：

  • 减少 3390 + 2723 + 1601 = 7714 次读取
  • 节省约 638KB 反复读取字符数
  • 预计 Token 用量降低 40-50%

方案 B：引用计数优化

  • 在 post_tool_logger.py 加入引用计数警告
  • 当单一文件单日读取 >100 次时触发 ntfy 告警
  • 配合 check-health.ps1 显示 Top 10 高频读取文件

🔴 **P0-Critical：plan parse error 连续失败**

#### 问题描述

从 logs/todoist-team_20260415_043004.log：


[Phase1] WARN: LLM job 完成但未產出 plan 檔案，疑似對話性回應
[ERROR] Failed to parse plan: The property 'tasks' cannot be found
  • 连续 2 次尝试均失败
  • LLM 输出对话性回应而非标准 JSON plan

#### 根因分析

  1. prompts/team/todoist-query.md 可能输出格式约束不够明确
  2. LLM 可能因 context 过载而回退到对话模式
  3. 缺少输出格式 Schema 验证

#### 优化建议

方案 A：加强 Prompt 输出格式约束

在 todoist-query.md 加入：


## ⚠️ 输出格式（强制要求）

**你必须严格输出以下 JSON 格式，不可有任何对话性文字：**

\```json
{
  "tasks": [
    {
      "task_id": "string",
      "content": "string",
      "labels": ["string"],
      "priority": "p1|p2|p3|p4"
    }
  ],
  "total_daily_cap": 65,
  "remaining_slots": 10
}
\```

**禁止输出**：任何解释、对话、提示文字。只输出上述 JSON。

🟡 **P1-High：快取策略优化**

#### 问题描述

  • hackernews.json：过期 1941 分钟（TTL 180 分钟）
  • pingtung-news.json：过期 1936 分钟（TTL 360 分钟）
  • semantic-cache-kb.json：9 天未更新
  • 快取命中率仅 26.3%（目标 40%）

#### 根因分析

  1. Phase 0 快取更新逻辑可能被跳过
  2. API 调用失败时未正确更新快取时间戳
  3. 语义快取未被充分利用（9 天未更新表示很少使用）

#### 优化建议

方案 A：Phase 0 强制快取刷新


# 在 Phase 0 加入强制刷新逻辑
$CacheFiles = @('hackernews', 'pingtung-news', 'gmail')
foreach ($cache in $CacheFiles) {
    $CachePath = "cache/$cache.json"
    $CacheAge = (Get-Date) - (Get-Item $CachePath).LastWriteTime
    $TTL = (Get-Content "config/cache-policy.yaml" | ConvertFrom-Yaml).$cache.ttl_minutes

    if ($CacheAge.TotalMinutes -gt $TTL) {
        Write-Host "[Cache] $cache 已过期 ($($CacheAge.TotalMinutes) > $TTL)，触发刷新"
        # 调用对应 API 刷新...
    }
}
