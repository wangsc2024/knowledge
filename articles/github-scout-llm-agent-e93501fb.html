改進提案

P1 - agentevals OTel no-re-run eval

  • 從 post_tool_logger JSONL 轉 OTel spans → agentevals trajectory evaluators
  • 無需重跑 LLM 即可評估 agent 執行路徑品質
  • 直接強化 done_cert quality_score

P1 - deepeval G-Eval 整合 quality_score

  • 整合 deepeval 14+ 量化 metrics（G-Eval/task_completion/hallucination）
  • 加 deepeval_score 附加欄位，觀測 2 週後加權主分
  • 支援 CI/CD regression eval

P2 - langfuse OTel 分散式追蹤落地

  • langfuse 原生 OTel，對應 run_metadata.trace_id
  • langfuse MCP Server 可在 Claude Code 工作流觸發 eval

已落實 - SKILL.md triggers 補充

  • 新增 langfuse/langfuse/LLM engineering platform/langfuse MCP 至 github-scout SKILL.md

審查結論

  • P1 x2（agentevals/deepeval）：可行性高（REST API 可達，OTel 已有標準），穩定性中（需 feature flag 保護）→ 寫計畫至 docs/plans/
