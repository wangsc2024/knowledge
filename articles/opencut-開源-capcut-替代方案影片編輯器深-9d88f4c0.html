OpenCut — 開源 CapCut 替代方案影片編輯器深度研究

專案概述

OpenCut 是一款免費、開源的影片編輯器，定位為 CapCut 的開源替代方案，支援 Web、桌面與行動平台。專案於 2025 年 6 月創建，短短 8 個月內突破 45,000 Stars，成為 2025-2026 年增長最快的開源專案之一。

  • GitHub: OpenCut-app/OpenCut
  • Stars: 45,892（截至 2026-02-18）
  • Forks: 4,689
  • 授權: MIT License
  • 語言: TypeScript (99.2%)、CSS (0.8%)
  • 創建日期: 2025-06-22
  • 官網: https://opencut.app

核心價值主張

  1. 隱私優先：影片處理完全在本地裝置執行，不上傳雲端
  2. 免費功能：CapCut 逐漸將基本功能付費化，OpenCut 提供免費替代
  3. 簡單易用：延續 CapCut 證明的「簡單好用」哲學
  4. 無浮水印：不強制添加品牌浮水印
  5. 無訂閱制：完全免費使用

核心架構特色

1. 瀏覽器端影片處理（FFmpeg WASM）

  • 使用 @ffmpeg/ffmpeg 在瀏覽器中執行 FFmpeg，無需伺服器端處理
  • 影片編碼、轉檔、合併全部在客戶端完成
  • 零隱私風險：影片資料永不離開用戶裝置

2. Zustand 狀態管理架構

  • timeline-store.ts：時間軸狀態（多軌道、播放頭位置、縮放）
  • editor-store.ts：編輯器全局狀態
  • panel-store.ts：面板佈局管理
  • assets-panel-store.tsx：素材面板狀態
  • keybindings-store.ts：快捷鍵配置

3. 核心引擎（core/managers/）

  • Manager 模式封裝複雜邏輯，與 UI 解耦
  • 渲染管線正在重構為「binary rendering approach」（二進制渲染）

4. 客戶端 AI（Hugging Face Transformers.js）

  • 使用 @huggingface/transformers 在瀏覽器端執行 AI 模型
  • 可能用於自動字幕生成、語音識別等功能
  • 完全客戶端推論，不需 API 呼叫

功能特色

已實現功能

  • Timeline 編輯：多軌道時間軸、拖放操作、縮放
  • 即時預覽：編輯時即時預覽效果
  • 多軌道支援：影片、音訊、字幕等獨立軌道
  • 文字特效：文字疊加、字幕編輯
  • 音效管理：sounds-store.ts 管理音效資源
  • 貼紙系統：stickers-store.ts 貼紙面板
  • 鍵盤快捷鍵：完整的快捷鍵系統 + 自訂配置

重構中功能

  • 預覽面板：字體、貼紙、特效正在以新的二進制渲染方式重構
  • 匯出功能：匯出管線重構中

使用方式

快速開始

# 1. Fork + Clone
git clone https://github.com/YOUR_USERNAME/OpenCut.git
cd OpenCut

# 2. 啟動資料庫（可選，僅後端功能需要）
docker-compose up -d

# 3. 安裝依賴
cd apps/web
bun install

# 4. 環境變數
cp .env.example .env.local

# 5. 資料庫遷移（若使用 Docker）
bun run db:migrate

# 6. 啟動開發伺服器
bun run dev
# → http://localhost:3000

關鍵環境變數

  • DATABASE_URL: PostgreSQL 連接字串
  • BETTER_AUTH_SECRET: 認證密鑰
  • UPSTASH_REDIS_REST_URL: Redis 端點

潛在應用

對本專案（Daily-Digest-Prompt）的啟發

  1. Monorepo 架構參考：Turborepo + packages/ 的組織方式值得借鑒，若未來擴展 daily-digest 為多套件結構
  2. Zustand 狀態管理：輕量級狀態管理模式，可用於未來的 Web Dashboard
  3. WASM 本地處理：FFmpeg WASM 的模式展示了「隱私優先」的工程實踐，可應用於本地 AI 推論
  4. Biome 替代 ESLint：Biome 作為 Linter/Formatter 的全合一方案，大幅簡化工具鏈
  5. 客戶端 AI（Transformers.js）：瀏覽器端執行 AI 模型的模式，可能適用於知識庫的本地語義搜尋

一般應用場景

  • 內容創作者：免費替代 CapCut/Premiere 進行短影片製作
  • 教育：開放源碼的影片編輯工具用於教學
  • 企業：自架部署確保影片資料不外洩
  • 開發者：基於 OpenCut 構建客製化影片處理工具

優缺點評估

優點

  • 隱私保護卓越：FFmpeg WASM 確保影片永不上傳
  • 技術棧現代：Next.js 15 + React 19 + TypeScript 5.8，走在前沿
  • 社群增長驚人：8 個月 45K Stars，證明市場需求
  • MIT 授權：完全自由使用和修改
  • Vercel 贊助：基礎設施穩定
  • Bun + Biome：開發體驗極佳，工具鏈精簡

缺點

  • 渲染引擎重構中：預覽面板和匯出功能尚在重構，功能不完整
  • 單人主導：77% commits 來自一位開發者，Bus Factor 風險
  • 僅 Web 平台：桌面和行動原生版尚未推出
  • 瀏覽器性能限制：WASM FFmpeg 相比原生 FFmpeg 仍有性能差距
  • 功能豐富度不足：相比 CapCut 的豐富特效庫，OpenCut 仍在早期
  • 文件較少：架構文件不夠詳細（AGENTS.md 存在但 README 較簡略）

架構亮點與設計決策

  1. Manager 模式（core/managers/）：將複雜業務邏輯封裝在 Manager 類中，與 React 元件解耦，提高可測試性
  2. Store 層分離（Zustand）：每個功能域獨立 Store（timeline、editor、panel、keybindings），避免單一大 Store
  3. 二進制渲染重構：從 Canvas/DOM 渲染轉向二進制渲染管線，目標是接近原生性能
  4. Monorepo 共享套件：packages/ui 和 packages/env 支援未來多平台擴展
