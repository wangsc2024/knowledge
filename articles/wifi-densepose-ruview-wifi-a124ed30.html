一、專案概述

WiFi DensePose（又名 RuView，ruvnet/wifi-densepose）是一個開源系統，能將日常 WiFi 訊號轉化為即時人體姿態估計、生命徵象監控與存在偵測——完全不需要攝影機、影像或穿戴式裝置。專案以 Rust 語言重寫核心引擎，實現了相對於 Python 原型 810 倍的端對端加速。

二、學術源頭：CMU DensePose From WiFi

本專案的理論基礎源自卡內基美隆大學（CMU）2023 年發表的論文「DensePose From WiFi」（arXiv:2301.00250），作者為 Jiaqi Geng、Dong Huang 與 Fernando De la Torre。

核心貢獻

  • 開發深層神經網路，將 WiFi 訊號的相位與振幅映射至 24 個人體區域的 UV 座標
  • 僅使用 WiFi 訊號作為輸入，即可估計多人稠密姿態，效果媲美基於影像的方法
  • 克服攝影機系統的遮蔽、照明限制，且無需 LiDAR 或雷達等昂貴硬體

三、技術架構：八階段訊號處理管線

WiFi DensePose 的核心是一條精心設計的八階段處理管線：

3.1 訊號擷取層

  1. WiFi 傳輸與散射：WiFi 訊號穿過環境，遇人體產生反射、吸收、繞射
  2. CSI 擷取：透過 ESP32 網格或研究級網卡（Intel 5300 / Atheros AR9580）捕捉通道狀態資訊
  3. 多頻融合：3 通道 x 56 子載波 = 168 虛擬子載波，形成高維度環境視圖

3.2 訊號處理層

  1. 多基站融合：注意力加權的跨視角嵌入，減少遮蔽效應
  2. 相干性閘門（Coherence Gating）：確保量測穩定性
  3. 訊號演算法：Hampel 濾波器、SpotFi、Fresnel 模型、BVP 頻譜圖等 6 種 SOTA 演算法

3.3 AI 推論層

  1. RuVector 引擎：11 個自包含 crate，涵蓋 HNSW 搜尋、注意力機制、GNN 圖網路、時序壓縮、最小切割、求解器
  2. Signal-Line CRV 協議：6 階段格式塔處理，含 Poincare 球嵌入與 GNN 拓撲分析

3.4 關鍵演算法

| 演算法 | 功能 | ADR 編號 |

|--------|------|----------|

| 對比式 CSI 嵌入 | 從原始 CSI 自監督學習 | ADR-024 |

| MERIDIAN 跨環境泛化 | 梯度反轉的領域不變訓練 | ADR-027 |

| 多基站感測 | TDM 協議 + 相干性閘門 | ADR-029/030/031 |

| QUIC 網格安全 | TLS 1.3 AEAD + 重放保護 | ADR-032 |

| Signal-Line CRV | Poincare 球嵌入 + GNN | ADR-033 |

五、感測能力矩陣

5.1 人體姿態估計

  • 17 身體關鍵點即時偵測
  • 多人追蹤：每個 AP 支援 3-5 人，多 AP 線性擴展
  • 穿牆感測：最深可達 5 公尺（混凝土、瓦礫皆可穿透）

5.2 生命徵象監控

  • 呼吸偵測：6-30 次/分鐘
  • 心率偵測：40-120 次/分鐘
  • 非接觸式：無需穿戴任何裝置

5.3 自學習機制

  • 無標註自學習：從原始 WiFi 資料自舉（bootstrap），無需人工標註訓練資料
  • 跨域泛化：MERIDIAN 架構使一個房間訓練的模型可在不同環境中使用
  • 持久場模型：透過 SVD 分析建立房間電磁特徵結構

八、隱私安全的雙刃劍分析

8.1 隱私優勢（設計面）

  • 無視覺資料：管線中完全不產生影像或影片
  • GDPR 友善：消除影像相關的合規要求
  • HIPAA 相容：無影像資料=無影像法規負擔
  • 身份無關：擷取的是身體姿態幾何，非身份或外觀

8.2 隱私風險（攻擊面）

DEV Community 的深入分析指出 CSI 技術存在嚴重隱私隱憂：

| 風險維度 | 說明 |

|---------|------|

| 完全被動式攻擊 | 攻擊者只需「聆聽」無線電波，不傳輸任何訊號 |

| 低成本 | 一塊 ESP32-S3（~$30 USD）即可執行攻擊 |

| 加密無效 | WPA3 等所有 WiFi 加密對此無效——這是物理層問題，非密碼學問題 |

| 遠端存取 | 可從隔壁房間或建築物外部執行 |

| 難以偵測 | 被動監聽幾乎無法被目標察覺 |

8.3 安全對策（RuView 實作）

  • QUIC 網格安全（ADR-032）：TLS 1.3 AEAD 傳輸、HMAC 認證信標、SipHash 框架完整性、重放保護
  • 對抗式偵測：場模型可識別異常環境入侵
  • 領域正規化：硬體變異正規化防止晶片組指紋識別
  • 1,031+ 測試：含獨立能力審計（ADR-028）

---

九、技術限制與挑戰

  1. 環境敏感：在受控環境中效果最佳，高度動態空間或大群聚效能下降
  2. 校準需求：需策略性放置路由器並校準系統
  3. 多人上限：單 AP 3-5 人，需多 AP 擴展
  4. CSI 可用性：一般消費級 WiFi 設備不暴露 CSI 資料，需特定硬體
  5. 訓練資料：初始模型仍需 WiFi-姿態配對資料集

十、對軟體開發者的啟示

10.1 Rust 改寫模式

WiFi DensePose 展示了一個典型的「Python 原型 → Rust 生產」模式：

  • 保留 Python 訓練管線（靈活性優先）
  • 核心推論與訊號處理改用 Rust（效能優先）
  • 810x 加速證明了系統語言的重要性

10.2 自包含 Crate 架構

RuVector 的 11 個 vendored crate 展示了模組化設計：

  • 每個 crate 獨立可測試
  • 自包含避免外部依賴地獄
  • ADR 文件化每個架構決策

10.3 ADR 驅動開發

專案使用大量 ADR（Architecture Decision Records）：

  • ADR-024 至 ADR-033 記錄了核心技術決策
  • 每個 ADR 含問題、選項、決策、結果
  • 與 Daily-Digest-Prompt 專案的 ADR 實踐可互相借鑑
