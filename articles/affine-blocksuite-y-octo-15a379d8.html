一、專案概述

AFFiNE（發音 [ə'fain]，toeverything/AFFiNE）是由 TOEVERYTHING 團隊開發的開源次世代知識庫平台，定位為「There can be more than Notion and Miro」——將文件撰寫、白板規劃與資料庫管理統一在同一個平台上。截至 2026 年 3 月已累積 65,200+ Stars、4,600+ Forks，採用 MIT 授權（社區版），是目前 GitHub 上最具影響力的 Notion 開源替代方案之一。

二、核心技術創新

2.1 BlockSuite 協作編輯器

BlockSuite 是 AFFiNE 的編輯器核心，一個完全自研的協作式區塊編輯器框架：

  • 60+ 區塊類型：段落、標題、清單、程式碼、資料庫、嵌入網頁、形狀、投影片等
  • 16+ Widget 套件：工具列、拖曳控制、連結文件等 UI 覆蓋層
  • 雙模式渲染：Page Mode（傳統文件）與 Edgeless Mode（無邊界畫布）共存
  • Specification Pattern 插件架構：透過 Spec 擴展核心 Schema，支援自訂 Block、View、Store、Service

Block State (Y.Doc) → Store (Reactive) → Widgets → View (Page/Edgeless)

2.2 y-octo：Rust 原生 CRDT 引擎

y-octo 是 AFFiNE 自研的高效能 CRDT 實作，以 Rust 編寫，與 YJS 格式完全相容：

  • 效能提升：相比 JavaScript 版 YJS，在 CRDT 操作和文件解析上快 10-100 倍
  • 執行緒安全：原生支援多執行緒併發操作
  • 核心 API：

- Doc::apply_update() — 合併二進制更新到文件狀態

- Doc::encode_state_as_update() — 序列化變更為二進制格式

- Doc::merge_updates() — 高效合併多個更新

- Doc::get_state_vector() — 取得時鐘向量用於同步協商

2.3 Edgeless 無邊界畫布

AFFiNE 最具差異化的特色——同一文件可在「文件模式」與「無邊界畫布模式」之間無縫切換：

  • 在畫布上放置任何建構區塊：富文本、便利貼、嵌入網頁、多視圖資料庫、連結頁面、形狀、投影片
  • 從結構化文件轉為腦力激盪白板，再將便利貼轉回結構化文件
  • 這是 Notion（文件）+ Miro（白板） 的真正統一，非簡單並列

三、系統架構（五層分離）

第一層：客戶端層

六個平台特定應用程式共享核心業務邏輯：

| 平台 | 套件 | 程式碼複用率 |

|------|------|----------|

| Web | @affine/web（Next.js） | 100% |

| Desktop | @affine/electron | 90% + 原生 |

| iOS | @affine/ios（Capacitor） | 95% + 插件 |

| Android | @affine/android（Capacitor） | 95% + 插件 |

| Admin | @affine/admin | 80% + 管理 UI |

第二層：共享前端核心

  • @affine/core（重要度 221）：應用程式邏輯，含 WorkbenchService、DocService、EditorService
  • @affine/component：基於 Radix UI + Vanilla Extract CSS 的 UI 元件庫
  • @blocksuite/affine（重要度 142）：協作編輯器
  • @toeverything/infra（重要度 121）：依賴注入框架
  • 狀態管理：Jotai（React 原子化狀態）

第三層：後端服務

  • NestJS（v11）+ Apollo GraphQL（1,600+ 行 Schema）
  • Prisma ORM + PostgreSQL（56 張表）
  • BullMQ 作業佇列
  • Socket.io 即時同步
  • Redis 快取/佇列
  • S3 相容儲存 檔案管理
  • Elasticsearch 搜尋

第四層：原生效能層

  • @affine/native：NAPI-RS 綁定，用於 Electron 桌面功能和媒體擷取
  • affine_server_native：後端 Rust 模組，檔案類型偵測和文件解析
  • affine_mobile_native：UniFFI 綁定，iOS/Android 原生功能

第五層：資料持久化

| 儲存 | 用途 |

|------|------|

| PostgreSQL | 結構化資料（56 表） |

| Redis | 快取、佇列 |

| S3 相容 | 檔案（Blob） |

| Elasticsearch | 搜尋 |

| IndexedDB | 客戶端本地儲存（CRDT 格式） |

四、CRDT 同步協議

文件同步管線（Local-First → Cloud Sync）

  1. 客戶端儲存：文件以 Y.js CRDT 格式存於 IndexedDB
  2. 即時傳輸：WebSocket 連線經 Socket.io 傳送 Y-protocol 更新
  3. 伺服器閘道：NestJS SpaceSyncGateway 接收更新並廣播給訂閱者
  4. 持久化：更新寫入 PostgreSQL updates 表；背景作業建立快照
  5. 離線支援：Service Worker 啟用 PWA 功能，支援離線操作

WebSocket 事件協議

| 事件 | 方向 | 用途 |

|------|------|------|

| space:join | Client → Server | 訂閱工作區更新 |

| space:push-doc-update | Client → Server | 發送 CRDT 變更（二進制 Uint8Array） |

| space:broadcast-doc-update | Server → Clients | 廣播變更給已連線客戶端 |

| space:load-doc | Client → Server | 請求文件狀態進行同步 |

OctoBase：Local-First 資料引擎

OctoBase 是 AFFiNE 的開源資料庫引擎（Rust 編寫），專為 Local-First 協作設計：

  • 輕量級、可擴展的資料引擎
  • 本地優先但支援協作
  • 處理 CRDT 更新的合併、快照、歷史版本

五、AI Copilot 架構

多供應商 LLM 系統

透過 CopilotProviderFactory 路由到不同 LLM 供應商：

| 供應商 | 用途 |

|--------|------|

| OpenAI（GPT） | 文本生成、對話 |

| Anthropic Claude | 文本分析、摘要 |

| Google Gemini | 多模態能力 |

| FAL AI | 圖像生成 |

| Perplexity | 語義搜尋 |

AI 功能

  • AI 聊天助手：與文件內容對話
  • Inline Actions：選取文字後即時改寫、翻譯、摘要
  • 文件生成：從提示生成完整文件
  • 投影片建立：AI 驅動的簡報製作
  • 心智圖摘要：自動將文件轉為心智圖
  • 語義搜尋：基於向量嵌入的相似度搜尋

AFFiNE 的獨特優勢

  1. 三模態統一：文件（Notion）+ 白板（Miro）+ 資料庫（Airtable）首次在開源專案中真正統一
  2. Edgeless Mode：結構化文件↔無邊界白板的雙向轉換，業界唯一
  3. y-octo Rust CRDT：自研高效能 CRDT，而非直接使用 JavaScript YJS
  4. 95%+ 跨平台複用：同一份 TypeScript 核心驅動 Web、Desktop、Mobile

  1. CRDT 同步模式：AFFiNE 的 Local-First → Cloud Sync 管線與本專案的「PowerShell 預計算 → Agent 讀取」模式異曲同工——先本地處理，再同步到中央
