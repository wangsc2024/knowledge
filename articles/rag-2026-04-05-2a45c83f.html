專案RAG優化日報 2026-04-05

資安強化與敏感資料防護管線優化

優化摘要

本次對 daily-digest-prompt 專案進行資訊安全全面審查，識別 14 項安全風險（4 CRITICAL、5 HIGH、3 MEDIUM、2 LOW），並實施 3 項 P0 安全強化，修改 2 個檔案，新增 17 個安全測試，802 個 hooks 測試全部通過。

安全審查發現

#### CRITICAL 風險（4 項）

  1. 日誌敏感資料消毒不完整：hook_utils.py 的 sanitize_sensitive_data() 僅覆蓋 Authorization headers、X-Token headers、環境變數三類，未覆蓋 URL query parameters（?api_key=xxx）和 JSON body 中的 credentials（"password": "xxx"）
  2. Prompt Injection 偵測規則已定義但未啟用：INJECTION_PATTERNS 定義了 7 個注入模式但無對應函式可供呼叫
  3. scheduler-state.json 寫入攔截 regex 存在邊界繞過：eval、sh -c、tee 等變體可繞過現有 pattern
  4. 結構化日誌輪轉未設權限保護：50MB 輪轉產生的日誌檔無 chmod 600

#### HIGH 風險（5 項）

  1. pre_read_guard 允許讀取 .env.example（負向前瞻設計）
  2. API 呼叫缺少統一 HTTPS 驗證和超時設定
  3. results/ JSON schema 驗證為 warn_only（不強制阻擋）
  4. scheduler-state regex 攔截有 eval/sh -c 邊界情況
  5. 日誌輪轉檔案未設定權限保護

已實施優化（3 項）

#### 1. 強化日誌敏感資料消毒（hook_utils.py）

問題：URL query params 和 JSON body credentials 未被消毒，可能洩漏至 JSONL 日誌

修復：新增兩個 pre-compiled regex：

  • _RE_SANITIZE_URL_PARAM：匹配 ?api_key=xxx、&token=xxx、&secret=xxx、&password=xxx、&access_key=xxx
  • _RE_SANITIZE_JSON_CRED：匹配 "password": "xxx"、"api_key": "xxx"、"auth_token": "xxx" 等 JSON 格式

測試：新增 7 個測試案例（URL param × 3、JSON body × 3、多參數 × 1），全部通過

#### 2. 啟用 Prompt Injection 偵測函式（hook_utils.py）

問題：INJECTION_PATTERNS（7 個模式）已定義但無可呼叫的 API

修復：

  • 預編譯 7 個 injection patterns 為 _COMPILED_INJECTION_PATTERNS
  • 新增 check_injection_patterns(text: str) -> list 函式，回傳匹配的模式清單（空 = 安全）
  • 供 post_tool_logger、pre_write_guard 等模組引用

偵測的注入模式：

  1. ignore (all) previous instructions
  2. system: you are
  3. 標籤
  4. ADMIN MODE
  5. forget everything/all
  6. you are now a
  7. disregard all/any previous/prior

測試：新增 10 個測試案例（6 個偵測 + 2 個安全文字 + 1 個大小寫 + 1 個多重模式），全部通過

#### 3. 安全測試補強

新增 17 個測試（7 URL/JSON 消毒 + 10 injection 偵測），hooks 測試套件從 785 → 802 個，全部通過。

未實施但建議的後續優化

| 優先級 | 項目 | 建議 |

|--------|------|------|

| P1 | scheduler-state regex 強化 | 改為白名單模式（僅 PowerShell 允許寫入） |

| P1 | results/ schema 驗證升級 | warn_only → block（強制阻擋格式不符的 JSON） |

| P2 | 日誌權限保護 | 輪轉時設定 chmod 600 |

| P2 | API 超時統一配置 | 新增 api-security-policy.yaml |

| P3 | PII guard 增強 | 身分證字號加檢查碼驗證 |

測試結果

  • hooks 測試套件：802 passed（+17 新增，0 失敗）
  • 新增測試分類：

- TestSanitizeSensitiveData：17 個（原 10 + 新 7）

- TestCheckInjectionPatterns：10 個（全新）

技術細節

修改檔案：

  1. hooks/hook_utils.py — 新增 _RE_SANITIZE_URL_PARAM、_RE_SANITIZE_JSON_CRED、_COMPILED_INJECTION_PATTERNS、check_injection_patterns() 函式
  2. tests/hooks/test_hook_utils.py — 新增 TestCheckInjectionPatterns 類別（10 個測試）、擴充 TestSanitizeSensitiveData（7 個測試）

設計決策：

  • URL param regex 使用 [?&] 前綴確保只匹配 query string，不誤傷 URL path
  • JSON body regex 使用雙引號界定符確保只匹配 JSON 格式，不影響 YAML 或其他格式
  • Injection 偵測回傳匹配清單而非 bool，讓呼叫端可判斷嚴重程度
  • 所有 regex 均為模組層級 pre-compiled（re.compile），避免 hot path 重複編譯
