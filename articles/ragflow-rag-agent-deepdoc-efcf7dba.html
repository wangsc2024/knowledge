RAGFlow — 開源 RAG + Agent 引擎深度研究

專案概述

RAGFlow（infiniflow/ragflow）是由 InfiniFlow 團隊開發的開源 RAG 引擎，結合先進的 RAG 技術與 Agent 能力，為 LLM 提供卓越的上下文層。截至 2026 年 3 月擁有 75.6K Stars、8.5K Forks，是 GitHub Top 20 AI 專案之一。

核心定位：「Deep document understanding + grounded citations = reduced hallucinations」— 透過深度文件理解與有根據的引用來減少 LLM 幻覺。

---

五層微服務架構

RAGFlow 採用清晰的五層分離架構：

1. Client Layer（客戶端層）

  • React 19 SPA 前端
  • Python SDK
  • HTTP REST API

2. API Gateway（API 閘道）

  • Quart 非同步框架（port 9380）
  • JWT 認證
  • 非同步請求處理

3. Service Layer（服務層）

  • DialogService：對話管理
  • DocumentService：文件處理
  • KnowledgebaseService：知識庫管理
  • CanvasService：Agent 工作流

4. Worker Layer（工作層）

  • Redis queue consumer
  • 5 個併發執行緒的任務執行器
  • 透過 Redis streams + consumer groups 解耦長時間文件操作
  • 支援水平擴展

5. Storage Layer（儲存層）

  • 文件引擎：Elasticsearch 或 Infinity（可插拔切換）
  • 中繼資料：MySQL
  • 物件儲存：MinIO（S3 相容）
  • 快取/佇列：Redis

---

七階段文件處理管線（Document Processing Pipeline）

  1. Upload：檔案驗證 + MinIO 物件儲存
  2. Task Queuing：Redis stream 發布，consumer group 確認
  3. Parsing：可選解析器（DeepDoc、MinerU、Docling、naive）
  4. Chunking：12+ 種模板式分塊（語義邊界感知）
  5. Enrichment：LLM 輔助關鍵字提取、問題生成、中繼資料合成
  6. Embedding：批次向量編碼（512-1024 維度可配置）
  7. Indexing：批量插入文件儲存 + BM25 tokenization

DeepDoc 視覺處理

  • ONNX-based OCR
  • PDF 版面辨識
  • 表格結構識別
  • 位置中繼資料提取（頁碼、邊界框）

---

12+ 分塊策略（Chunking Strategies）

| 類型 | 說明 |

|------|------|

| 模板式 | 依文件類型（論文、書籍、法律、QA、表格、簡報）使用 regex 模式 |

| 語義式 | 內容感知邊界偵測 |

| 位置感知 | 保留頁碼、邊界框、層級結構 |

| 上下文附加 | 將父區段標題和表格附加到分塊 |

| Parent-Child | 父子分塊策略，支援階層式檢索 |

| TOC 結構 | 自動生成文件目錄，減緩因分塊不精確導致的上下文遺失 |

分塊中繼資料欄位：page_num_int、top_int（位置）、important_kwd（關鍵字）、available_int（啟用旗標）。

---

混合檢索與重排序（Hybrid Search + Re-ranking）

混合搜尋

  • 向量匹配：HNSW 索引，密集向量搜尋（q_N_vec 欄位）
  • BM25 關鍵字：全文搜尋（content_ltks tokenized 欄位）
  • 融合：可配置權重的分數組合

重排序

  • 可選的 LLMBundle.rerank() 交叉編碼器評分
  • 結果格式化包含引用和分塊歸因
  • 有根據的引用（grounded citations）減少幻覺

---

Agent Canvas 工作流系統

  • DSL：JSON 格式的領域特定語言定義元件圖
  • 內建元件：LLM、檢索、程式碼執行、記憶操作
  • 串流：WebSocket 即時輸出
  • 狀態管理：變數作用域 + 輸入/輸出綁定
  • API：Canvas API（port 9381）管理工作流 CRUD 和執行編排
  • MCP 整合：透過 rag/llm/tool_calling.py 支援 Model Context Protocol

| Session Memory | 對話歷史快取於 Redis，持久化至 MySQL dialog 表 |

| Long-term Memory | 知識庫索引於文件儲存，動態檢索相關分塊 |

2026 路線圖重點

  • ContextEngine as filesystem：將上下文引擎實作為檔案系統介面

  • EraRAG：圖譜式 RAG 架構（補充 RAPTOR/GraphRAG）
  • Agent workflow checkpoints：Agent 執行狀態儲存/恢復
