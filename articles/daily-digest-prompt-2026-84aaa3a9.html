Daily-Digest-Prompt 專案優化報告

日期：2026-02-26

任務 ID：6fxqhGvW6Qpv9MHX

測試基線：496 → 532（全數通過）

優化項目摘要

1. benchmark.yaml test_count 門檻更新

  • 問題：目標門檻 ≥200 已過時，實際測試數量 532
  • 修正：更新為 ≥450，反映真實系統成熟度
  • 影響：system-insight Skill 基準線比對更準確

2. LoopDetector 白名單擴充

  • 問題：state/loop-state-*.json 正常讀寫被 LoopDetector 誤判為迴圈
  • 修正：新增 r"state/loop-state-[0-9a-f]{8}\.json$" 白名單 pattern
  • 影響：消除 PostToolUse hook 的假陽性迴圈偵測

3. .gitignore 安全性強化

  • 問題：缺少密碼學金鑰檔案 pattern（.key, .pem, .crt, .p12, *.pfx）
  • 修正：新增 5 個密碼學檔案排除 pattern
  • 影響：防止意外將私鑰/憑證提交到版本控制

4. check-health.ps1 Circuit Breaker 儀表板

  • 問題：無法一覽 API 健康狀態，需手動查看 api-health.json
  • 修正：新增 52 行 PowerShell 區塊，讀取 api-health.json 顯示彩色狀態
  • 功能：顯示每個 API 的 state/failures/cooldown，用顏色區分健康狀態
  • 影響：一條指令即可掌握所有 API 斷路器狀態

5. post_tool_logger.py 迴圈偵測錯誤記錄

  • 問題：except Exception: pass 靜默吞掉所有錯誤，難以排查
  • 修正：改為 except Exception as e: 並輸出到 stderr
  • 影響：迴圈偵測失敗時可在 stderr 日誌追蹤根因

6. benchmark.schema.json 補齊 output_format

  • 問題：benchmark.yaml 包含 output_format 欄位但 schema 未定義
  • 修正：新增 output_format 為可選 object 屬性
  • 影響：validate_config.py 驗證時不再靜默忽略未知欄位

技術亮點

  • 三路並行分析：同時啟動 3 個 Explore Agent 掃描配置/Hooks/腳本
  • 零迴歸：532 個測試全數通過，無任何退化
  • 資安導向：.gitignore 強化防止金鑰外洩，LoopDetector 白名單減少誤判
