二、核心功能摘要

四種研究模式

| 模式 | 階段數 | 耗時 | 適用場景 |

|------|--------|------|----------|

| Quick | 3 | 2-5 min | 初步探索 |

| Standard | 6 | 5-10 min | 一般研究（預設） |

| Deep | 8 | 10-20 min | 複雜主題、關鍵決策 |

| UltraDeep | 8+ | 20-45 min | 全面報告、最高嚴謹度 |

8 階段管線

  1. SCOPE — 研究範圍界定
  2. PLAN — 搜尋策略規劃
  3. RETRIEVE — 並行搜尋（5-10 concurrent searches + 2-3 sub-agents）
  4. TRIANGULATE — 來源交叉驗證

4.5. OUTLINE REFINEMENT — 大綱精煉

  1. SYNTHESIZE — 綜合分析
  2. CRITIQUE — 多角色紅隊審查（Skeptical Practitioner、Adversarial Reviewer、Implementation Engineer）
  3. REFINE — 基於審查修正
  4. PACKAGE — 產出報告（MD + HTML + PDF）

關鍵特性

  • First Finish Search：依模式自適應品質閾值
  • Critique loop-back：Phase 6 可回到 Phase 3 補查缺口
  • Disk-persisted citations：sources.json 在 context compaction 後仍存活
  • Auto-continuation：>18K 字報告自動遞迴生成
  • McKinsey 風格 HTML：專業報告模板

三、與 Daily-Digest-Prompt 專案的整合評估

互補性分析

| 面向 | 現有系統 | deep-research 補強 |

|------|---------|--------------------|

| 研究流程 | web-research Skill（搜尋→篩選→摘要→KB匯入） | 8 階段管線 + 來源可信度評分 + 引用驗證 |

| 品質保證 | 人工審查 | validate_report.py（9 項自動檢查）+ verify_citations.py |

| 報告格式 | Markdown 筆記 | MD + HTML（McKinsey 風格）+ PDF |

| 來源追蹤 | research-registry.json（7 天滾動） | citation_manager.py + source_evaluator.py（可信度評分） |

| 長文處理 | 無 | auto-continuation（>18K 字自動分段） |

適用的自動任務

以下自動任務可受益於 deep-research Skill：

  1. ai_deep_research — 直接替換為 deep 或 ultradeep 模式
  2. tech_research — standard 模式即可
  3. ai_github_research — 搭配 search-cli 的 GitHub 搜尋
  4. 佛學研究系列（fahua、jingtu、shurangama）— deep 模式確保引用嚴謹

整合注意事項

  1. 耗時控制：deep 模式 10-20 min，ultradeep 20-45 min，需與 config/timeouts.yaml 的 phase2_timeout 對齊
  2. 輸出路徑：預設輸出到 ~/Documents/，非專案內 results/ — 需調整或後處理
  3. Token 預算：深度研究消耗較多 Token，需與 config/budget.yaml 的 80%/100% 門檻配合
  4. search-cli 可選：基本功能用 WebSearch 即可，search-cli 提供多引擎但需 API key

使用方式

在 Claude Code 中直接輸入：


deep research on [研究主題]
deep research in deep mode: [複雜主題]

四、品質標準

  • 10+ 來源，每個主要論點至少 3 個來源支持
  • Executive Summary 200-400 字
  • 每個 Finding 600-2,000 字，散文為主（≥80%）
  • 完整參考文獻（含 URL，無佔位符）
  • 自動驗證迴圈：validate → fix → retry（最多 3 次）
