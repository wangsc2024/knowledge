Mamba 與 State Space Models（SSM）深度研究

後 Transformer 時代的線性複雜度序列建模架構

---

一、概述

Mamba 是由 Tri Dao（Princeton）和 Albert Gu（CMU）於 2023 年提出的神經網路架構，建構於 State Space Models（SSM）之上，被譽為「第一個在自回歸大語言模型領域能與 Transformer 競爭的替代架構」。其核心突破在於以 O(L) 線性時間複雜度取代 Transformer Attention 的 O(L²) 二次複雜度，同時保持甚至超越同等參數規模 Transformer 的效能表現。

截至 2026 年 3 月，Mamba 架構已從學術研究走向生產部署，包括 Mistral 的 Codestral Mamba、AI21 的 Jamba 系列、IBM 的 Granite 4.0 等，混合 Transformer-Mamba 架構更成為新一代 LLM 設計的主流趨勢。

---

二、數學基礎

2.1 連續時間 SSM

SSM 使用兩個核心方程描述動態系統：

  • 狀態方程：h'(t) = A·h(t) + B·x(t) — 基於輸入 x(t) 和矩陣 A、B 更新隱藏狀態 h(t)
  • 輸出方程：y(t) = C·h(t) + D·x(t) — 從狀態透過矩陣 C、D 推導輸出 y(t)

2.2 離散化（Zero-Order Hold）

為了處理離散序列（如文字 token），需要將連續 SSM 離散化：

  • 輸入值在下一個離散信號到達前保持不變
  • 引入可學習的步長（Δ）控制輸入解析度
  • 將函數對函數映射轉換為序列對序列映射（x_k → y_k）

2.3 HiPPO 初始化

HiPPO（High-order Polynomial Projection Operators）用於初始化矩陣 A：

  • 將所有輸入信號壓縮為係數向量
  • 利用 Legendre 多項式追蹤歷史信號
  • 強調近期 token 同時衰減較舊資訊
  • 使 SSM 能維持長序列的上下文依賴

---

三、Selective SSM（S6）— Mamba 的核心創新

3.1 線性時不變性的問題

傳統 SSM 存在根本限制：矩陣 A、B、C 對所有輸入保持不變（Linear Time Invariance），無法進行「內容感知推理」。在選擇性複製（selective copying）和歸納頭（induction heads）等合成任務上表現不佳。

3.2 輸入依賴參數

Mamba 的突破性解決方案：讓 B、C 和 Δ 變為 input-dependent：

  • 矩陣 B 和 C 根據每個 token 的內容動態生成 → 選擇性壓縮資訊
  • 步長 Δ：較小值忽略特定詞彙；較大值優先當前輸入而非上下文
  • 等效於讓 SSM 具備類似 Attention 的「內容感知」能力，但保持 O(L) 複雜度

3.3 Mamba Block 結構

  1. 線性投影：擴展輸入嵌入維度
  2. 1D 卷積層：捕捉局部模式（防止 token 獨立處理）
  3. Selective SSM：遞迴離散化 + HiPPO 初始化 + 選擇性掃描
  4. 門控機制：SSM 輸出與門控路徑的逐元素乘法
  5. 殘差連接：保留原始輸入

---

四、Hardware-Aware Parallel Scan

4.1 選擇性掃描演算法

利用乘法和加法的結合律（associative property）將序列分解為可並行計算的部分，迭代組合部分結果，在保持遞迴結構的同時支援 GPU 並行執行。

4.2 核心融合與 SRAM/DRAM 優化

靈感來自 Tri Dao 的 FlashAttention：

  • 將離散化、選擇性掃描和 C 乘法融合為單一 GPU 核心
  • 中間結果不寫回慢速 DRAM，全在 SRAM 中完成
  • 反向傳播時重新計算中間狀態（而非儲存），更高效

4.3 訓練 vs 推理模式

  • 訓練：使用可並行化的卷積表示
  • 推理：使用高效的遞迴表示
  • 這種雙模式設計是 Mamba 同時兼顧訓練效率和推理速度的關鍵

五、Mamba-2 與 State Space Duality（SSD）

Mamba-2（2024 年 5 月）帶來重大理論突破：

5.1 State Space Duality

證明 SSM 和 Attention 屬於同一數學家族，這意味著：

  • SSM 可以用 Attention 的語言描述，反之亦然
  • 為混合架構提供了理論基礎
  • 開啟了跨架構知識遷移的可能性

5.2 架構改進

| 改進項目 | 說明 |

|---------|------|

| 簡化實作 | Selective SSM 僅需約 25 行程式碼 |

| 並行參數生成 | B 和 C 透過複製而非專用線性層產生 |

| 更大隱藏維度 | 支援更具表達力的模型 |

| Multi-Head 變體 | 類似 Transformer 的多頭注意力機制 |

| 速度提升 | 比原版 Mamba 快 2-8 倍 |

六、效能比較：Mamba vs Transformer

6.1 計算複雜度

| 指標 | Transformer | Mamba |

|------|-------------|-------|

| 序列長度複雜度 | O(L²) | O(L) |

| 推理 FLOPs（2.8B 參數） | 基準 | 約 0.67x（少 33%） |

| 推理吞吐量 | 基準 | 最高 5x 提升 |

| 記憶體使用 | 隨序列長度二次增長 | 線性增長 |

| 可處理序列長度 | 受限（通常 128K-1M） | 理論上無限 |

6.2 基準測試表現

在同等參數規模下，Mamba 在標準語言模型基準測試（MMLU、HellaSwag 等）上匹配或超越 Transformer 模型，在長上下文評估中優勢更為明顯。

七、混合架構：Transformer + Mamba

7.1 Jamba（AI21 Labs）

全球首個生產級 Mamba 混合模型：

  • 架構：Attention 與 Mamba 層以 1:7 比率交替，每兩個 block 加入 MoE 層
  • 參數：398B 總參數 / 94B 活躍參數
  • 上下文：支援 256K token
  • 效能：長上下文吞吐量是 Mixtral 8x7B 的 3 倍
  • NVIDIA RULER 長上下文基準測試達到頂級表現

7.2 其他混合實現

| 模型 | 架構特點 |

|------|----------|

| Codestral Mamba（Mistral AI） | 純 Mamba 程式碼模型 |

| Granite 4.0（IBM） | 混合 SSM + Attention |

| TransMamba | Transformer Encoder + Mamba Decoder |

7.3 混合架構的設計原理

為什麼不完全替換 Transformer？

  1. Attention 擅長精確回溯：需要精確引用先前內容時（如 in-context learning），Attention 仍有優勢
  2. SSM 擅長高效壓縮：處理長序列時的全局上下文壓縮，SSM 更高效
  3. 互補組合：少量 Attention 層（1/8）處理精確引用，大量 SSM 層處理高效推理

八、應用領域

Mamba/SSM 的應用已超越 NLP：

| 領域 | 應用 |

|------|------|

| 自然語言處理 | 語言建模、程式碼生成、長文件理解 |

| 基因組學 | DNA 序列建模（百萬級 token 長度） |

| 語音生成 | 音頻序列處理 |

| 電腦視覺 | Vision Mamba（ViM）影像分類 |

| 金融 | 高頻交易市場微結構建模 |

| 科學計算 | PDE（偏微分方程）求解 |

九、局限性與挑戰

  1. 生態系統成熟度：Transformer 有十年的優化歷史和工具鏈，Mamba 仍在追趕
  2. 訓練方式差異：input-dependent 參數使得無法使用卷積捷徑，需要不同的訓練方法
  3. 精確回溯能力：純 SSM 在需要精確引用先前 token 的任務上仍不如 Attention
  4. 硬體生態：當前 GPU 和 TPU 主要針對 Transformer 優化，SSM 的硬體協同設計仍在發展中
  5. In-Context Learning：純 Mamba 在少樣本學習任務上可能不如 Transformer

十、2026 展望與趨勢

10.1 混合架構成為主流

2026 年的趨勢清晰指向「混合架構」而非「替換 Transformer」：

  • AI21 的 Jamba 系列已驗證大規模混合部署的可行性
  • IBM Granite 4.0 採用混合設計
  • 學術界（TransMamba）持續探索最佳混合比例

10.2 關鍵研究方向

  • 最佳 Attention:SSM 比例：目前 1:7（Jamba）是否為最優？
  • 硬體協同設計：針對 SSM 優化的專用加速器
  • 跨架構知識蒸餾：利用 SSD 理論在 Transformer 和 SSM 間遷移知識
  • 長序列推理：百萬 token 級別的實際應用場景
