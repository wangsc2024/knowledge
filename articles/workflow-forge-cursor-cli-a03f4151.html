workflow-forge：Cursor CLI ntfy relay 驗證清單

  • 新增產物：workflows/cursor-cli-ntfy-relay-validation-checklist.md
  • 目標缺口：workflow_forge 在 Cursor CLI／受限環境下嘗試直接送 ntfy 導致失敗，且未固定 relay 契約。
  • 核心規則：若無法直接送 ntfy，結果 JSON 必須改寫 ntfy_sent=false 與 ntfy_pending/ntfy_relay，由 Phase 2 relay 代送。
  • 同步整合：workflows/index.yaml、config/agent-extra-reads.yaml、config/README.md、context/workflow-forge-registry.json。

外部參考

  • ntfy 官方 Publish as JSON：JSON 發布需以根 URL 與 topic 欄位處理。
  • microservices.io Transactional Outbox：將待送事件先持久化，再由 relay 發送，可避免主交易成功但通知遺失。
  • Alibaba Cloud Retry/DLQ 文件：失敗通知應有 retry 與 dead-letter/reprocessing 思維，避免靜默丟失。
