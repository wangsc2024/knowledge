專案概述（約 220 字）

OpenHands 是社群驅動的「AI 驅動軟體開發」平台，前身為 OpenDevin；目標是把可修改程式碼、執行終端機、瀏覽網頁與呼叫 API 等開發者工作，封裝成可重複、可觀測的代理管線，並以 MIT 授權（enterprise/ 目錄除外）對外開放 [來源: OpenHands README, 2026]。官方 README 將 Software Agent SDK 定位為所有產品線的引擎，並提供 CLI、本機 GUI、雲端與企業自建路徑 [來源: OpenHands README, 2026]。中文圈翻譯文件亦強調「少編碼、多創造」與 uvx／Docker 兩種本機啟動路徑，並警告單一使用者本機設計不適合多租戶 [來源: readme-i18n 中文翻譯頁, 2025]。

技術架構（約 420 字）

V0 時期採單體、預設全面沙箱、EventStream pub/sub 傳遞事件；團隊自述在爆紅期間累積了執行緒／async 順序保證與維運複雜度，且 Docker 映像與 CLI 依賴過重 [來源: The Path to OpenHands v1, 2025]。V1 以 OpenHands Software Agent SDK 為核心重做：論文與 README 對齊描述 openhands.sdk／openhands.tools／openhands.workspace／openhands.agent_server 四套件分層，並以「可選隔離、預設本機單行程」取代強制沙箱，以對齊 MCP 假設與本機憑證存取 [來源: arXiv:2511.03690, 2025；OpenHands README, 2026]。狀態面採 事件溯源（event sourcing）：不可變事件 append、單一 ConversationState 為可變真相來源，支援持久化與重播；工具鏈採 Action–Execution–Observation，並把 MCP 工具映射為同一型別安全路徑 [來源: arXiv:2511.03690, 2025]。LLM 層透過 LiteLLM 支援多供應商，並內建 RouterLLM、非原生 function-calling 模型的文字工具呼叫降級路徑 [來源: arXiv:2511.03690, 2025]。生產資料宣稱 V1 相對 V0 系統可歸因失敗下降 61%、事件持久化開銷可忽略 [來源: arXiv:2511.03690, 2025]——此為論文量化主張，非本 repo 量測。

功能特色與差異化（約 280 字）

與 SWE-agent（研究導向、ACI 極簡介面）及商用 Devin（封閉模型與 SaaS）相比，OpenHands 主打 模型無鎖、可自架、SDK 化組裝；第三方比較文宣稱在 SWE-Bench Verified 類基準上 OpenHands 與 SWE-agent 可達較高區間，而 Devin 偏向易用整合 [來源: ToolHalla 比較文, 2026]。相較 README 單方敘述，以下表綜合 README（產品矩陣）與比較部落格（定位／定價敘述）：

使用方式（約 180 字）

官方 README 建議以 SDK 文件（docs.openhands.dev/sdk）與 CLI 文件為入口 [來源: OpenHands README, 2026]。中文翻譯頁提供 uvx 一鍵啟動 GUI 或 CLI 的示例（需 Python 3.12）：


uvx --python 3.12 --from openhands-ai openhands serve
# 或
uvx --python 3.12 --from openhands-ai openhands

Docker 路徑需掛載 docker.sock 與資料目錄，並注意強化安裝指南以降低暴露面 [來源: readme-i18n 中文翻譯頁, 2025]。

優缺點評估（約 280 字）
