TL;DR

當前多語言前沿語言模型（Multilingual LLM）的技術格局呈現一個深層矛盾：模型規模不斷擴張，但語言覆蓋的公平性赤字從未根本解決。MMLU-ProX 基準評測顯示，即便是表現最佳的 Qwen2.5-72B，英語與史瓦希里語（Swahili）之間的準確率差距仍達 30.2%；BenchMAX 跨語言評測套件更記錄了最高 24.3% 的多語言性能差距。這不是資料量的問題，而是架構選擇的問題。

本報告論證，多語言能力的真正躍遷需要在三個維度上進行同步架構重構：（1）語言感知分詞器設計——解決高資源語言分詞粒度優勢造成的系統性懲罰；（2）動態語言資料課程——以課程學習取代固定溫度採樣，並加速收斂 18–45%；（3）跨語言感知對齊協議——Aya 系列研究覆蓋 101 語言的指令微調實踐，顯示多語言對齊不能只靠翻譯注水。

從 BLOOM（2022）、XGLM（2021）、Jais（2023）到 Aya Expanse（2024），現有文獻的集體軌跡指向同一結論：多語言公平性必須在預訓練架構層就被設計進去，而非事後補丁。

  3. 預訓練資料課程策略如何影響多語言能力的湧現路徑？
  4. 跨語言感知的對齊協議如何有別於簡單翻譯注水？

研究範疇

本報告聚焦於 2021–2025 年主要多語言 LLM 架構設計文獻，涵蓋：開放式多語言基礎模型（BLOOM、XGLM、Jais、Aya 系列）、分詞器跨語言轉移技術、多語言預訓練資料課程研究、以及跨語言評測基準（MMLU-ProX、BenchMAX、Crosslingual Capabilities）。排除範圍：機器翻譯系統（非 LLM 架構）、語音語言模型、RAG 架構多語言應用。

文獻探討

3.1 多語言大型語言模型的奠基工作

XGLM（Meta AI, 2021） [1] 是首個系統性探索多語言少樣本學習的大型生成模型，基於 30 種語言的平衡語料庫進行預訓練。最大規模的 7.5B 參數版本在多語言常識推理超越同等規模 GPT-3 達 +7.4%（0-shot）及 +9.4%（4-shot），並在 FLORES-101 機器翻譯基準中 182 個翻譯方向的 171 個超越 GPT-3。XGLM 的核心啟示在於：跨語言能力可以通過預訓練語料的語言平衡設計來系統性提升，這是後來 BLOOM 和 Aya 系列的理論基礎。

BLOOM（BigScience Workshop, 2022） [2] 是第一個真正規模化的開放多語言語言模型，由 392 位研究者協作完成。176B 參數版本在 ROOTS 語料庫上訓練，涵蓋 46 種自然語言及 13 種程式語言（共 59 種），並在多項基準上達到競爭性能。BLOOM 的架構選擇——decoder-only Transformer、多語言詞彙表設計——及開放授權（Responsible AI License）成為此後多語言 LLM 研究的重要基準點。BigScience 計畫明確了多語言模型需要分散化治理的原則，影響了後來 Aya 計畫的社群組織模式。

Jais（MBZUAI, 2023） [3] 是阿拉伯語中心開源 LLM，基於 GPT-3 解碼器架構，在阿拉伯語與英語混合語料上進行預訓練。13B 參數版本在阿拉伯語知識與推理能力上，顯著優於任何現有開放阿拉伯語及多語言模型。Jais 的重要貢獻在於語言特定架構（language-centric architecture）的可行性論證：通過刻意強化特定語言的訓練資料比例，可以在不犧牲英語性能的前提下大幅提升目標語言的能力，直接挑戰了「通用多語言模型比語言特定模型更優」的主流假設。

3.2 指令微調與多語言對齊

Aya Dataset（2024） [4] 通過眾包收集了 101 種語言的指令-回應對，其中超過 50% 的語言被視為低資源語言。Aya Model（2024） [5] 在 mT5 基礎上進行多語言指令微調，在多數任務上超越 mT0 和 BLOOMZ，同時覆蓋雙倍數量的語言。研究發現，多語言指令微調的資料配比對性能有決定性影響，且非線性——更多翻譯資料並不自動帶來更好的多語言能力。Aya 23（2024） [6] 和 Aya Expanse（2024） [7] 延續深化此路線，共同結論是：多語言對齊的瓶頸並非語言數量，而是各語言之間指令品質的均衡程度。

3.3 分詞器設計的跨語言影響

Trans-Tokenization（2024） [8] 提出跨語言詞彙轉移策略，通過語義相似詞嵌入的加權平均初始化目標語言 token 嵌入，驗證了「轉移分詞化 LLM」（Tweeties）和「多頭 LLM」（Hydra LLMs）的可行性。Parallel Tokenizers（2025） [9] 提出並行分詞器架構，在 13 種低資源語言上預訓練，證明在情感分析、仇恨言論偵測和情緒分類任務上超越傳統多語言基線。Model-Aware Tokenizer Transfer（2025） [10] 指出多語言模型分詞器系統性地偏向高資源語言，導致低資源語言詞彙份額不足、準確率下降且處理速度變慢，分詞器轉移是實際可行的緩解方案。

3.4 預訓練資料課程策略

Revisiting Multilingual Data Mixtures（2025） [11] 系統性重新審視語言資料混合策略，發現傳統溫度採樣導致高資源語言反復訓練，動態課程策略能更好地平衡語言覆蓋。Beyond Random Sampling（2025） [12] 發現課程學習在訓練早期和中期一致加速收斂，將達到基線性能所需的訓練步驟減少 18–45%。Enhancing Multilingual LLM Pretraining（2025） [13] 提出基於模型回饋的動態資料選取策略。Rethinking Multilingual Continual Pretraining（2025） [14] 探索持續預訓練的跨語言資料混合問題，提出在現有英語中心模型基礎上有效進行跨語言持續預訓練的策略。

3.5 跨語言評測框架

MMLU-ProX（2025） [15] 涵蓋 29 種語言、每種語言 11,829 個相同問題，評測顯示 Qwen2.5-72B 在英語（70.3%）與史瓦希里語（40.1%）之間存在 30.2% 差距，25 個最先進模型均呈現顯著性能差距。BenchMAX（2025） [16] 記錄了最高 24.3% 的多語言 LLM 性能差距，所有模型在 MCQ 以英語呈現時一致表現更好。Crosslingual Capabilities and Knowledge Barriers（2024） [17] 指出即便模型規模持續增大，多語言性能差距依然存在且是系統性的。

核心論證

本文主張：當前多語言前沿語言模型的架構設計，以「英語中心預訓練 + 多語言後期微調」為主流路線，本文論證此策略導致低資源語言在推理、工具使用與指令跟隨上出現系統性能力赤字——MMLU-ProX 最佳模型（Qwen2.5-72B）英語與史瓦希里語之間仍存在 30.2% 的差距正是佐證。本文主張，跨語言能力的真正平等化需要在三個架構維度上同時進行重構：（1）語言感知的分詞器設計以解決高生育率懲罰（high fertility penalty）問題；（2）動態語言採樣課程（dynamic language curriculum）取代固定溫度採樣；（3）跨語言感知的對齊協議而非翻譯注水。闡明此三軸架構框架，是構建真正多語言公平 AI 系統的先決條件。

4.1 論證一：分詞器是隱藏的語言公平性瓶頸（三角佐證）

多語言 LLM 的第一個系統性偏見存在於分詞器層。現代 LLM 普遍採用 BPE 或 SentencePiece 分詞，在詞彙表構建上天然偏向高資源語言。高生育率懲罰（high fertility penalty）的機制：高資源語言（英語、德語、法語）的生育率低，低資源語言（史瓦希里語、孟加拉語、祖魯語）的生育率高——用低資源語言表達相同內容，消耗更多計算資源，模型理解深度卻更差，是雙重懲罰。

Trans-Tokenization（2024）[8] 提供具體技術路線：通過跨語言詞彙轉移，用語義相似的源語言嵌入初始化目標語言 token 嵌入，在保持主幹模型參數不變的前提下顯著提升低資源語言初始化品質。Parallel Tokenizers（2025）[9] 更進一步，提出並行分詞器架構，讓不同語言使用各自最優的分詞策略，再在嵌入空間中對齊，從而消除單一詞彙表對低資源語言的系統性歧視。Model-Aware Tokenizer Transfer（2025）[10] 從公平性角度論證分詞器轉移的必要性。三篇獨立研究均從不同技術角度確認分詞器是多語言能力差距的架構根源之一。

4.2 論證二：資料課程策略決定跨語言能力的湧現效率（三角佐證）

多語言 LLM 的第二個架構瓶頸是預訓練資料的組合策略。傳統固定溫度採樣無法令人滿意：α = 1 高資源語言占優，α = 0 低資源語言過擬合，中間值選取缺乏理論依據。課程學習提供了動態解決方案。

Beyond Random Sampling（2025）[12] 顯示課程學習加速收斂，減少訓練步驟 18–45%。Revisiting Multilingual Data Mixtures（2025）[11] 指出語言資料混合策略的最優解是動態的：訓練早期廣覆蓋策略更有效，後期高品質優先。Enhancing Multilingual LLM Pretraining（2025）[13] 提出基於模型回饋的動態資料選取，本質上是將課程設計從靜態超參數空間轉移到動態模型狀態空間。三項獨立研究均以實驗方式確認動態課程策略優於靜態溫度採樣。

4.3 論證三：多語言性能差距在當前架構下是系統性的、非線性縮小的（三角佐證）

MMU-ProX（2025）[15] 對 25 個最先進模型的全面評測揭示：即便是最大規模的模型，高資源語言與低資源語言之間的能力差距依然顯著，且模型規模增大無法線性消除差距。BenchMAX（2025）[16] 與 Crosslingual Capabilities（2024）[17] 共同確認，所有 LLM 在 MCQ 以英語呈現時的表現系統性優於同一問題以目標語言呈現，表明模型的「思考」仍然以英語為主要媒介。

這一現象的根本原因在於：當前多語言 LLM 的訓練目標是多語言生成能力，而非語言無關概念理解。模型學到的是「如何用各種語言表達」，而非「如何用語言無關方式理解概念再翻譯表達」。這是深層的架構限制，無法通過簡單增加低資源語言訓練資料來克服。Jais（2023）[3] 提供了反例：語言特定架構（Arabic-centric pretraining）在阿拉伯語任務上的表現，顯著優於同等規模通用多語言模型，表明語言特定設計的架構承諾比語言無差別的通用多語言訓練更有效。

一家之言

以下為主筆李飛飛與共同主筆 Yann LeCun 的觀點，基於各自公開著作與演講推斷，非本人直接授權。

李飛飛（主筆）：語言多樣性是 AI 公平性的科學前線

我在 Stanford HAI 長期推動 Human-Centered AI，但我要坦白說：AI 社群至今未能認真對待語言公平性。MMLU-ProX 顯示 30% 的英語-史瓦希里語差距，這不只是個「技術問題」，這代表著幾億非英語使用者在我們聲稱為「通用」的 AI 系統面前，實際上面對的是一個系統性偏見的機器。

我們在 ImageNet 時代學到一件事：數據是科學儀器，數據的構成決定你能看到什麼、看不到什麼。當我們設計一個以英語語料為主體的詞彙表，我們其實是在科學儀器上對非英語世界戴上了一副濾鏡。分詞器的「高生育率懲罰」不只是工程細節——它是一種制度性的語言歧視（institutional language discrimination）被編碼進模型架構之中。

我同意這篇報告的三軸論點。但我要強調，這三個架構維度背後的驅動力必須是語言使用者，而非只是研究者。Aya 計畫在這一點上做得最好：它將 101 種語言的語言社群納入指令資料的眾包創建，而非僅靠翻譯 API 注水。這是「Human-Centered AI」的真實實踐——不是把 AI 帶給人，而是讓人的語言進入 AI 的設計過程。

然而，我也必須誠實面對一個根本性的矛盾：現有的多語言評測框架（MMLU-ProX、BenchMAX 等）本質上是用英語邏輯設計的問題，翻譯成各語言。這本身就是一種文化偏見——我們用英語思維衡量各語言的「智能」，而非讓各語言文化自我定義「智能」的意涵。真正的語言公平性需要每種語言都有文化原生的評測基準（culturally-native benchmarks），而不只是翻譯自 MMLU 的試題。

Jais 的阿拉伯語中心路線給了我信心：語言特定的架構承諾是可行的，也是必要的。但它同時也提醒我們，單純把多語言能力問題交給規模化（scaling）解決，是一種懶惰的策略。這個領域需要更多像 Jais 這樣的語言文化嵌入式研究，而非僅靠更大的通用模型順帶帶出更多語言。

最後，我想說的是：技術必須服務人類的繁榮（technology must serve human flourishing）。而當前的多語言 AI 格局是：世界上 70% 的人口使用的語言，在我們最先進的模型中是二等公民。這不只是技術失敗，這是想像力的失敗——我們未能想像出一個真正以人類多樣性為出發點的 AI 架構設計範式。Aya 的 101 語言承諾、BLOOM 的社群協作精神、Jais 的語言文化嵌入路線，這三者的集體啟示是：多語言 AI 的未來，不是由更大的英語中心模型帶動的，而是由更多語言的在地化設計承諾塑造的。

Yann LeCun（共同主筆）：分詞器是個訊號，但真正的問題更深

我以 Yann LeCun 視角和你聊，基於公開言論推斷，非本人觀點。

我一直說「LLMs are a detour」，但在多語言問題上，我需要說得更精確：多語言 LLM 揭示了 Transformer + token 架構的根本性限制，而這限制在語言多樣性的壓力下被放大到不可忽視的程度。

分詞器偏見問題是真實的，但它只是症狀。根本問題是：現有的 LLM 架構在概念層面沒有語言無關性（language-agnostic conceptual representation）。當我提出 JEPA（Joint Embedding Predictive Architecture）和世界模型時，核心訴求是：智能系統應該建立在對世界的概念理解上，而語言只是這個概念理解的一種表達媒介。但現有的 LLM——包括多語言 LLM——把語言本身作為概念的載體，而非概念的表達。

這導致了一個根本性的架構困境：你要訓練一個真正「思考」概念、「說多種語言」的系統，還是一個「在多種語言中模式匹配」的系統？MMLU-ProX 顯示的 30% 語言差距，我的解讀是：這些模型確實是在「英語裡想，再翻譯」——而不是在「概念裡想，用目標語言表達」。這是根本性的架構問題，三軸框架是正確的工程方向，但不要把工程最優化誤認為架構突破。

對於開源社群，我要強調：Llama 和 BLOOM 的開源路線是正確的。封閉多語言模型只服務有錢的公司，開源模型讓 Jais 這樣的語言特定架構研究成為可能。開放的重要性不只在於可複現性，而在於讓語言社群能夠直接參與自己語言的 AI 架構設計。

我也要為課程學習策略的架構地位正名。從架構角度看，課程學習不只是個訓練技巧，而是對「語言能力湧現路徑」的顯式設計。Beyond Random Sampling 顯示的 18–45% 收斂加速，背後是一個深刻的結論：語言知識的獲取有其自然順序，就像人類學語言一樣，從簡單到複雜的漸進曝光比隨機曝光更有效。這呼應了我長期強調的 inductive bias 的重要性——不同語言的表達結構不同，通用的隨機採樣策略忽略了這一點。

長遠看，我倡導第四個維度：世界模型的多語言延伸。如果我們能建立真正的概念基礎世界模型，「多語言」這個問題就會變成表達層問題，而非理解層問題——就像人類用中文思考和用英文思考本質上是同一個概念系統的不同表達。這才是長遠解法。短期的分詞器修復和課程策略是必要的工程，但不要讓工程的局部最優遮蔽了架構層的根本問題。

---

實踐建議

對模型開發者

分詞器稽核優先：在多語言模型開發之前，為每種目標語言計算生育率（fertility）統計，將高生育率語言（>5 tokens/word）列為高優先級分詞器優化標的。Trans-Tokenization 和 Model-Aware Tokenizer Transfer 提供了具體的技術路線。

實施動態語言課程：放棄固定溫度採樣，改採課程學習策略。建議預訓練前三分之一採用廣覆蓋、均等採樣策略；後三分之二逐步加大高品質語料的比重。Beyond Random Sampling 的實驗顯示此策略能減少 18–45% 的訓練步驟，節省資源的同時提升最終性能。

學習 Aya 的對齊路線：多語言指令微調不能只靠翻譯資料。Aya 的眾包方式（語言社群原生創作指令-回應對）應成為標準做法，尤其對高複雜度語言（阿拉伯語、泰語、梵語等形態豐富語言）更為關鍵。

對評測研究者

引入文化原生基準：MMLU-ProX 和 BenchMAX 是重要進步，但仍是翻譯自英語的測試集。各語言文化應有自己的原生知識評測（如阿拉伯古典學術、中國四書五經推理、斯瓦希里詩歌文化脈絡）。

將能力差距（capability gap）作為標準報告指標：在技術報告中，高資源語言與低資源語言的性能差距應與整體準確率並列報告。以 ΔEN-X（英語與語言 X 的準確率差距）為標準化指標。

對政策制定者

語言公平性作為 AI 治理原則：在 AI 系統採購或評估標準中，應明確要求報告多語言能力差距指標。政府研究機構應填補語言特定架構研究的資金缺口，就像 GENCI 支持 BLOOM 的模式。

---

風險與限制

安全與多語言的取捨：多語言能力的提升可能帶來安全對齊新風險。現有 RLHF 和安全訓練大多集中在英語資料，低資源語言版本的安全護欄尚不完善，提升低資源語言能力的同時需要同步投資多語言安全評測。

評測通脹（benchmark inflation）：MMLU-ProX 等基準的廣泛使用可能導致模型針對基準過擬合，而非真正的多語言能力提升。

低資源語言的資料品質：提升多語言預訓練中低資源語言資料比例，會引入更多低品質資料（網路爬取的低資源語言資料品質普遍低於英語資料）。Rethinking Multilingual Continual Pretraining 的研究提醒，資料品質的優先級應高於語言覆蓋的廣度。

研究限制：本報告以英語和中文資料庫為主要搜索來源，對阿拉伯語、印地語、韓語等非英語語言學術社群的原生研究文獻可能有遺漏。多語言 LLM 領域演進極快，部分結論可能在報告完成後已有更新。
