gemini-cli 專案深度研究與架構洞察

專案概述

Google 官方開源 Gemini AI CLI 工具，v0.30.0-nightly，Apache 2.0 授權。技術棧：Node.js 20+ / TypeScript / React (Ink) / esbuild / Vitest。規模：4,664 commits、1,449 TypeScript 檔案、6 個 monorepo packages。

Monorepo 結構

  • packages/cli：前端 UI（React/Ink 終端機渲染）
  • packages/core：後端邏輯（API 編排、工具執行、排程器）含 agents/scheduler/tools/hooks/skills/policy/safety/routing 等模組
  • packages/sdk：程式化 SDK（GeminiCliAgent 類別）
  • packages/a2a-server：實驗性 Agent-to-Agent 伺服器

關鍵機制

1. Agent 編排

結構化 AgentDefinition（promptConfig + modelConfig + runConfig + toolConfig + outputConfig），支援 local/remote Agent，Agent Registry 集中管理，LocalAgentExecutor 執行 Agent Loop。5 種終止模式：GOAL/ERROR/TIMEOUT/MAX_TURNS/ABORTED。

2. Hook 系統

11 種事件（BeforeTool/AfterTool/BeforeAgent/AfterAgent/SessionStart/SessionEnd/Notification/PreCompress/BeforeModel/AfterModel/BeforeToolSelection），5 種決策（ask/block/deny/approve/allow），四層配置優先級（Project > User > System > Extensions）。

3. Skill 系統

SKILL.md frontmatter（name + description）+ Markdown body，與 daily-digest-prompt 格式高度一致。載入優先級：Built-in → Extension → User → Workspace。ActivateSkillTool 工具啟用 Skill。

4. Policy Engine

TOML 策略配置、規則匹配（工具名稱 + 參數 pattern + Approval Mode）、通配符支援、Shell 安全分析。SafetyChecker 協議含 protocolVersion 版本化。

5. 重試機制

retryWithBackoff：指數退避 + 隨機 jitter（初始 5s、最大 30s）。錯誤分類：TerminalQuotaError（不可重試）、RetryableQuotaError（可重試）、ValidationRequiredError（需使用者動作）。支援 AbortSignal、模型自動降級。

6. 模型路由

CompositeStrategy 策略鏈：FallbackStrategy → OverrideStrategy → ClassifierStrategy → NumericalClassifierStrategy → DefaultStrategy。

可移植技術（P0）

  1. 指數退避重試：可直接適配到 PowerShell 腳本
  2. Safety Checker 協議版本化：在 hook JSON 中加入 protocol_version
  3. Agent 終止模式分類：擴展 scheduler-state.json 的 result 欄位為 5 種

與 daily-digest-prompt 比較

Gemini CLI 是程式碼驅動的企業級平台（1,449 TS 檔）；daily-digest-prompt 是文件驅動的精簡系統（改配置不改 prompt）。Skill 格式高度一致，證明設計方向正確。本專案獨有優勢：研究去重、自動任務輪轉、ntfy 通知、跨次記憶持久化。
