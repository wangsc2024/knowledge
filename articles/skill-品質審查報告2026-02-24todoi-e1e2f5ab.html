審查摘要

todoist-task-creator 審查結果

品質評分提升：3/5 → 4/5

發現問題 3 項：

  1. 標籤速查表引用 deep-research（非已有 Skill 名稱），應為 web-research
  2. 標籤查詢 API 回應格式假設為直接陣列，但 Todoist API v1 回傳 { "results": [...] } 格式
  3. routing.yaml 中同樣有 3 處 deep-research 引用（未在本次修改，因規範限制不動 config/）

修正內容：

  • 替換所有 deep-research → web-research（4 處）
  • 修正標籤查詢 Python 解析邏輯，相容 dict 和 list 格式
  • 版本升級為 1.1.0

game-design 審查結果

品質評分提升：3/5 → 4/5

發現問題 4 項：

  1. 缺少結構化執行步驟（只有品質標準，沒有「如何執行」的流程）
  2. 遊戲工作目錄 D:\Source\game 未在 SKILL.md 中提及
  3. 缺少錯誤處理/降級機制
  4. version 1.0.0 建立至今未更新

修正內容：

  • 新增「遊戲工作目錄」段落（含目錄結構樹）
  • 新增「執行步驟」段落（6 步驟：定位分析→KB 查詢→實施修改→品質驗證→部署→KB 回寫）
  • 新增「錯誤處理與降級」表格（5 種狀況與處理方式）
  • 新增 Cloudflare Pages 部署命令範例
  • 版本升級為 1.1.0

待處理項目

  1. routing.yaml 中 `deep-research` 引用：3 處需改為 web-research（不在本次 Skill 審查範圍，需另行處理）
