一、專案概述

Browser Use 是由 Magnus Muller 和 Gregor Zunic 創建的開源 Python 框架，目標是讓 AI Agent 能像人類一樣操作網頁瀏覽器。專案於 2024 年創立，從零快速增長至 79K+ Stars，成為 GitHub 史上增長最快的瀏覽器自動化專案之一，被 Fortune 500 企業（Airbnb、Amazon、Google、Meta、Microsoft、Stripe 等）廣泛採用。

核心價值主張

  • 將網站轉化為 AI Agent 可操作的介面
  • 用自然語言描述任務，AI 自動規劃並執行瀏覽器操作
  • 模型無關（Model Agnostic）：支援 OpenAI、Anthropic、Google Gemini、本地 Ollama 等

二、技術架構（三層設計）

2.1 LLM 整合層（Intelligence Layer）

  • 多模型支援：透過 LangChain 整合 5+ LLM 供應商
  • 自建模型 ChatBrowserUse：專為瀏覽器自動化優化的自研模型，任務完成速度比通用模型快 3-5 倍
  • 對話歷史管理：跨導航步驟維持上下文，Agent 能理解多步驟任務的前後關聯
  • 自主任務規劃：LLM 分析當前頁面狀態並決定下一步操作

2.2 DOM 蒸餾引擎（DOM Distillation Engine）

這是 Browser Use 最核心的技術創新：

  • 頁面簡化：取得原始 DOM 後，去除不相關元素（廣告、裝飾性 CSS 等）
  • 互動元素標記：自動識別並標籤所有可互動元素（按鈕、輸入框、連結等）
  • Token 壓縮：經蒸餾後的 DOM 大幅減少 Token 消耗，降低 LLM API 成本
  • 雙模式理解：同時提取 DOM 結構 + 頁面截圖，結合結構化與視覺化理解

2.3 瀏覽器控制引擎（Browser Control Engine）

  • 基於 Playwright：使用 Microsoft Playwright 跨瀏覽器自動化框架
  • WebSocket 通訊：比 HTTP 輪詢更快的瀏覽器通訊方式
  • 多分頁支援：同時操作多個分頁處理並行工作流
  • 無頭/視覺模式：支援 headless 生產環境和可視化除錯模式

|------|------------|----------|-----------|--------|

| 語言 | Python | Python/JS | TypeScript | Python |

五、快速上手

安裝（需 Python 3.11+）


uv init && uv add browser-use && uv sync

基本使用


from browser_use import Agent, Browser, ChatBrowserUse
import asyncio

async def main():
    browser = Browser()
    agent = Agent(
        task="在 GitHub 上搜尋 browser-use 並回報 star 數量",
        llm=ChatBrowserUse(),
        browser=browser
    )
    result = await agent.run()
    print(result)

asyncio.run(main())

使用其他 LLM


from langchain_openai import ChatOpenAI
from langchain_anthropic import ChatAnthropic

# OpenAI
agent = Agent(task="...", llm=ChatOpenAI(model="gpt-4o"))

# Anthropic Claude
agent = Agent(task="...", llm=ChatAnthropic(model="claude-sonnet-4-5-20250929"))

七、潛在應用場景（與 Daily-Digest-Prompt 的連結）

7.1 每日摘要增強

  • 自動化網頁資料擷取：目前系統使用 curl + API 擷取資料（Todoist、新聞、HN）。Browser Use 可處理需要 JavaScript 渲染的網站，例如動態載入的政府公告頁面
  • 屏東新聞備用方案：當 MCP 端點不可用時，Browser Use 可直接從縣府網站爬取新聞

7.2 Todoist 任務自動化

  • 複雜網頁任務執行：某些 Todoist 任務涉及網頁操作（如「比較三個產品價格」），Browser Use 可作為子 Agent 工具
  • 研究任務增強：auto-task 的 AI 研究任務可用 Browser Use 深入網站取得更豐富資料

7.3 系統審查擴展

  • UI 測試自動化：系統審查（system-audit）的品質維度可加入 UI 自動化測試
  • 競品分析：定期掃描類似工具的官網，追蹤功能更新

7.4 遊戲開發輔助

  • 遊戲測試自動化：若遊戲有 Web 版本，可用 Browser Use 自動化功能測試

八、優缺點評估

優點

  1. SOTA 表現：WebVoyager 89.1% 成功率，業界最高
  2. 模型無關：不綁定特定 LLM，彈性極高
  3. DOM 蒸餾技術：大幅降低 Token 消耗，比視覺方法更經濟
  4. 社群龐大：79K+ stars、295 contributors、活躍開發
  5. MIT 授權：商業使用無限制
  6. 完整生態：CLI、Web UI、雲端服務、技能市場
  7. LangChain 整合：與主流 Agent 框架無縫銜接

缺點

  1. 自管基礎設施：開源版需自行處理代理、擴展和反偵測
  2. Python 限定：僅支援 Python，TypeScript 開發者需用 Stagehand
  3. LLM API 成本：大量任務的 API 呼叫費用可能很高
  4. 需 Python 3.11+：版本要求較高
  5. 雲端功能另計：隱身瀏覽、CAPTCHA 解決等進階功能需付費
  6. DOM 依賴：高度依賴 DOM 結構，面對非標準網頁可能表現下降
