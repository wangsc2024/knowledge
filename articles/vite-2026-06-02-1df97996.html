Vite 靜態知識庫同步與部署要點

研究日期：2026-06-02

核心結論

  1. vite.dev/config/shared-options 明確指出 publicDir 內檔案會在 dev 期間以網站根路徑提供，build 時原樣複製到輸出目錄根層，適合放 knowledge-sync.json 這類部署時必須保留原始檔名的靜態資料。
  2. vite.dev/guide/static-deploy 指出 Vite 部署到 GitHub Pages 時要正確設定 base；若採相對部署可用 ./。本專案 game_web/vite.config.js 已採 base: './'，可避免子路徑站點抓不到 JSON。
  3. MDN Fetch 文件指出 fetch() 不會因 404 自動 throw，前端必須先檢查 response.ok 再呼叫 response.json()。因此同步站前端應保留 ok 檢查與錯誤訊息分支。
  4. Cloudflare Pages build configuration 文件指出 monorepo 需明確指定 root directory，React (Vite) / Vue 類站常見 build command 為 npm run build、輸出目錄為 dist。本專案若將 game_web/ 獨立掛到 Pages，應把 root directory 指向該子目錄。

實作決策

  • 同步資料維持輸出到 game_web/public/knowledge-sync.json，不改成前端直連 API。
  • 前端持續用 fetch(import.meta.env.BASE_URL + 'knowledge-sync.json')。
  • 新增熱門標籤統計，直接由同步腳本產生 stats.topTags，前端純渲染。
