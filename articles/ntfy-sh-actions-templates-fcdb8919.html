ntfy.sh 推播通知進階實踐完整指南（2026）

技術概述

ntfy（發音「notify」）是一個基於 HTTP 的開源推播通知服務，透過簡單的 PUT/POST 請求即可發送通知到手機或桌面。它是 daily-digest-prompt 專案的核心通知基礎設施，每日處理數十則自動化告警與任務摘要。本文深入探討 ntfy 的進階功能、自託管部署配置，以及與 AI Agent 的整合最佳實踐。

---

核心概念與架構

發布/訂閱模型

ntfy 採用簡潔的 topic-based 發布/訂閱架構：

  • 發布者透過 HTTP PUT/POST 發送訊息到指定 topic
  • 訂閱者透過 HTTP、WebSocket、SSE 或行動 App 接收
  • 無需註冊帳號即可使用公共 ntfy.sh 服務
  • 支援自託管（Go 語言編寫，單一二進制檔）

訊息格式


# 最簡形式
curl -d "Hello" ntfy.sh/mytopic

# JSON 格式（推薦用於複雜訊息）
curl -d '{"topic":"mytopic","title":"Alert","message":"Details","priority":4}' ntfy.sh

---

進階發布功能

1. 優先級系統（Priority）

五級優先級控制通知行為：

| 等級 | 名稱 | 行為 |

|------|------|------|

| 5 | max/urgent | 長時間震動、彈出通知、螢幕亮起 |

| 4 | high | 長震動、預設音效、彈出 |

| 3 | default | 標準震動與音效 |

| 2 | low | 靜音，需下拉查看 |

| 1 | min | 靜音，歸類為「其他通知」 |


curl -H "Priority: urgent" -d "伺服器 CPU 使用率超過 95%" ntfy.sh/alerts
curl -H "Priority: low" -d "每日備份完成" ntfy.sh/status

2. 標籤與表情符號（Tags）

以逗號分隔的標籤，支援 emoji short code 自動轉換：


curl -H "Tags: warning,server01" -d "磁碟空間不足" ntfy.sh/ops
# warning → ⚠️ 自動轉換

curl -H "Tags: white_check_mark,backup" -d "備份成功" ntfy.sh/status
# white_check_mark → ✅

常用對應：warning → ⚠️、rotating_light → 🚨、white_check_mark → ✅、x → ❌、robot → 🤖

3. 排程訊息（Scheduled Delivery）

延遲發送或定時投遞：


# 相對時間
curl -H "In: 30m" -d "提醒：會議即將開始" ntfy.sh/reminders
curl -H "In: 2 hours" -d "記得喝水" ntfy.sh/health

# 絕對時間
curl -H "At: tomorrow, 10am" -d "早安報告" ntfy.sh/daily
curl -H "At: Tuesday, 7am" -d "週二例會提醒" ntfy.sh/meetings

# Unix 時間戳
curl -H "At: 1639194738" -d "精確排程" ntfy.sh/scheduled

限制：最短 10 秒，最長 3 天。

4. 互動式操作按鈕（Actions）

每則通知最多附加 3 個操作按鈕，支援四種類型：

#### View Action（開啟連結）


curl -H "Actions: view, 開啟監控面板, https://grafana.example.com" \
  -d "系統異常" ntfy.sh/alerts

#### HTTP Action（觸發 API）


curl -H "Actions: http, 關閉告警, https://api.example.com/alerts/ack, method=POST" \
  -d "需要確認的告警" ntfy.sh/ops

#### Broadcast Action（Android Intent）


curl -H "Actions: broadcast, 拍照, extras.cmd=pic, extras.camera=front" \
  -d "遠端拍照指令" ntfy.sh/phone

#### Copy Action（複製到剪貼簿）


curl -H "Actions: copy, 複製驗證碼, 123456" \
  -d "您的 OTP 驗證碼" ntfy.sh/auth

#### JSON 格式（複雜操作）


curl -d '{
  "topic": "home",
  "message": "門已開啟",
  "actions": [{
    "action": "http",
    "label": "鎖門",
    "url": "https://api.example.com/lock",
    "method": "PUT",
    "body": "{\"action\": \"lock\"}"
  }, {
    "action": "view",
    "label": "查看監視器",
    "url": "https://cameras.example.com/front"
  }]
}' ntfy.sh

5. 附件（Attachments）


# 上傳本地檔案（PUT）
curl -T screenshot.png -H "Filename: screenshot.png" ntfy.sh/reports

# 引用外部 URL
curl -H "Attach: https://example.com/report.pdf" \
  -H "Filename: monthly-report.pdf" \
  -d "月報已產出" ntfy.sh/reports

限制：單檔 15MB、每訪客 100MB、3 小時後自動清理。

6. 自訂圖示（Icons）


curl -H "Icon: https://example.com/logo.png" \
  -H "Title: 系統狀態" \
  -d "所有服務正常運作" ntfy.sh/status

7. 訊息更新與刪除

Dead Man's Switch 模式（心跳監控）：


# 建立排程訊息（30 分鐘後投遞）
curl -H "X-ID: heartbeat-001" -H "In: 30m" \
  -d "⚠️ 系統心跳中斷！" ntfy.sh/monitoring

# 每次心跳時更新（重設計時器）
curl -H "X-ID: heartbeat-001" -H "In: 30m" \
  -d "⚠️ 系統心跳中斷！" ntfy.sh/monitoring

# 若程式停止，30 分鐘後自動收到告警

8. 模板系統（Templates）

#### 預建模板（GitHub、Grafana、Alertmanager）


# 直接接收 GitHub webhook
curl -d '{"action":"opened","pull_request":{"title":"feat: new feature"}}' \
  'https://ntfy.sh/github-events?template=github'

#### 自訂模板


# /etc/ntfy/templates/deploy.yml
title: |
  {{if eq .status "success"}}✅ 部署成功{{else}}❌ 部署失敗{{end}}
message: |
  環境：{{ .environment }}
  版本：{{ .version }}
  耗時：{{ .duration }}
priority: |
  {{if eq .status "failed"}}5{{else}}3{{end}}

#### 行內模板


curl -d '{"host":"web01","cpu":95}' \
  'https://ntfy.sh/alerts?tpl=yes&t={{.host}} CPU: {{.cpu}}%'

自託管部署

快速啟動


# Docker
docker run -p 80:80 binwiederhier/ntfy serve

# 二進制
ntfy serve

# Windows 服務
ntfy serve --windows-service

核心配置（server.yml）


base-url: "https://ntfy.example.com"
listen-http: ":8080"
cache-file: "/var/lib/ntfy/cache.db"
cache-duration: "24h"
attachment-cache-dir: "/var/lib/ntfy/attachments"
attachment-file-size-limit: "15M"
attachment-total-size-limit: "5G"
log-level: "info"
log-format: "json"
behind-proxy: true

認證與 ACL


# 建立管理員
ntfy user add --role=admin admin_user

# 建立一般用戶
ntfy user add reader_user

# 設定 ACL
ntfy access reader_user alerts ro    # 唯讀
ntfy access writer_bot alerts wo     # 只寫
ntfy access '*' public-topic rw      # 匿名讀寫

# 預建 Access Token
ntfy token add --label="CI Bot" ci_user

速率限制


visitor-request-limit-burst: 60         # 初始配額
visitor-request-limit-replenish: "5s"   # 每 5 秒補充 1 次
visitor-message-daily-limit: 1000       # 每日上限
global-topic-limit: 15000               # 全域 topic 上限

Email 整合


# 出站 SMTP（通知轉寄 email）
smtp-sender-addr: "smtp.gmail.com:587"
smtp-sender-user: "bot@gmail.com"
smtp-sender-pass: "app-password"
smtp-sender-from: "ntfy@example.com"

# 入站 SMTP（email 轉通知）
smtp-server-listen: ":25"
smtp-server-domain: "ntfy.example.com"

監控


enable-metrics: true
metrics-listen-http: ":9090"  # Prometheus 相容

與本專案的關聯（daily-digest-prompt）

現有整合模式

本專案透過 ntfy 實現三類通知：

  1. 每日摘要推播（run-agent-team.ps1）：Phase 2 組裝完成後推送
  2. Todoist 任務結果（run-todoist-agent-team.ps1）：Phase 3 組裝後推送
  3. Session 異常告警（on_stop_alert.py）：Hook 自動偵測並告警

可改進方向

#### 1. 引入優先級分級


# 當前：所有通知同一優先級
# 改進：依告警等級設定
# - 攔截 ≥3 次 → priority: 5 (urgent)
# - 有錯誤 → priority: 4 (high)
# - 正常摘要 → priority: 3 (default)
# - SKILL.md 修改 → priority: 2 (low)

#### 2. Action 按鈕整合


# 錯誤告警加上快速操作
curl -H "Actions: view, 查看日誌, http://localhost:3000/logs; http, 重新執行, http://localhost:8080/retry, method=POST" \
  -d "Todoist Agent 執行失敗" ntfy.sh/wangsc2025

#### 3. Dead Man's Switch 監控


# 在 run-agent-team.ps1 開頭設定心跳
curl -H "X-ID: daily-digest-heartbeat" -H "In: 20m" \
  -d "⚠️ 每日摘要未在預期時間內完成" ntfy.sh/wangsc2025

# 成功完成後刪除排程訊息
curl -X DELETE ntfy.sh/wangsc2025/daily-digest-heartbeat

#### 4. 模板化通知格式


# 建立自訂模板供 webhook 直接使用
# /etc/ntfy/templates/digest.yml
title: |
  {{if eq .status "success"}}✅{{else}}❌{{end}} {{ .agent }}
message: |
  {{ .summary }}
priority: |
  {{if eq .status "failed"}}5{{else}}3{{end}}
tags: |
  {{if eq .status "success"}}white_check_mark{{else}}rotating_light{{end}}

#### 5. UnifiedPush 整合

ntfy 可作為 UnifiedPush 分發器，替代 Google FCM，在去 Google 化的 Android 設備（如 GrapheneOS）上接收推播通知。

實用模式與最佳實踐

腳本整合模式


# 命令成功/失敗分別通知
long-running-command \
  && curl -H "Tags: white_check_mark" -H "Priority: low" -d "任務完成" ntfy.sh/topic \
  || curl -H "Tags: x" -H "Priority: high" -d "任務失敗" ntfy.sh/topic

磁碟空間監控


#!/bin/bash
avail=$(df / | awk 'NR==2 {print int($4/1024/1024)}')
if [ "$avail" -lt 10 ]; then
  curl -H "Priority: urgent" -H "Tags: warning" \
    -d "磁碟剩餘 ${avail}GB" ntfy.sh/alerts
fi

SSH 登入告警（PAM 整合）


#!/bin/bash
if [ "${PAM_TYPE}" = "open_session" ]; then
  curl -H "Priority: high" -H "Tags: warning" \
    -d "SSH 登入：${PAM_USER}@${PAM_RHOST}" ntfy.sh/security
fi

CI/CD 管線通知


# GitHub Actions
- name: Notify ntfy
  if: always()
  run: |
    curl -H "Title: Build ${{ job.status }}" \
      -H "Tags: ${{ job.status == 'success' && 'white_check_mark' || 'x' }}" \
      -H "Priority: ${{ job.status == 'success' && '3' || '5' }}" \
      -d "${{ github.repository }} @ ${{ github.sha }}" \
      ntfy.sh/${{ secrets.NTFY_TOPIC }}

AI Agent MCP 整合

ntfy-me-mcp 專案提供 MCP Server，讓 AI Agent 透過 Model Context Protocol 直接發送 ntfy 通知：

  • 支援 token 認證存取自託管伺服器
  • 可透過 npx 或 Docker 部署
  • 適用於監控 CI/CD 管線、自動化工作流告警

---

安全注意事項

  1. Topic 命名安全：公共 ntfy.sh 的 topic 是公開的，使用隨機字串或自託管避免資訊洩露
  2. Token 認證：自託管環境務必啟用認證，使用 Access Token 而非明文密碼
  3. 速率限制：合理配置防止濫用，特別是附件功能
  4. TLS 加密：透過反向代理（nginx/caddy）終止 TLS，不在 ntfy 層處理
  5. 訊息快取：敏感訊息可用 ?cache=no 避免伺服器端快取
