OpenClaw — 2026 年 GitHub 史上最速爆紅的本地 AI 助理平台

執行摘要

OpenClaw 是由 PSPDFKit 創辦人 Peter Steinberger 開發的開源個人 AI 助理，於 2026 年 1 月底爆紅，72 小時內衝上 60,000+ GitHub Stars，截至 3 月已成為 GitHub 上最受關注的 AI Agent 專案之一。其核心定位是「本地優先的 AI Gateway」，讓使用者在自己的設備上運行 AI 助理，透過 WhatsApp、Telegram、Slack、Discord、LINE 等 25+ 通訊平台與 AI 互動，並支援 5,400+ 社群 Skill 插件。然而，其快速成長伴隨嚴重安全隱憂：CVE-2026-25253（CVSS 8.8）遠端程式碼執行漏洞、13.5 萬+暴露實例，以及 ClawHub 技能註冊表中 12% 惡意 Skill 的問題，使其成為「高潛力但須謹慎採用」的代表性專案。

研究背景

2026 年 AI Agent 開源生態進入爆發期。GitHub Octoverse 2025 報告顯示平台上已有 430 萬+ AI 相關 repo，LLM 專案年增 178%。在此背景下，OpenClaw 以「本地 AI 助理」定位脫穎而出，與 Claude Code（終端編碼 Agent）、OpenHands（AI 驅動開發）、goose（Block 的自主 Agent）形成差異化競爭 [來源：GitHub Octoverse, 2025] [來源：DigitalOcean, 2026]。

核心架構分析

Hub-and-Spoke Gateway 架構

OpenClaw 採用 hub-and-spoke 架構，核心是單一 Gateway 控制面（src/gateway/server.ts），運行在 Node.js 22+ 上，使用 WebSocket 綁定 127.0.0.1:18789。每台主機只允許一個 Gateway 實例，負責協調所有通訊平台連接、Session 狀態管理、健康監控與 Cron 排程 [來源：OpenClaw Architecture, Substack, 2026]。

Agent Runtime 四階段迴圈

PiEmbeddedRunner 執行 AI 迴圈，每輪包含四個階段：

  1. Session Resolution — 將訊息映射到對應 Session（main/dm/group）
  2. Context Assembly — 載入歷史、組建系統 prompt、執行語義記憶搜尋
  3. Model Streaming — 呼叫語言模型並攔截 tool call
  4. State Persistence — 將對話狀態持久化到磁碟

典型延遲預算：存取控制 <10ms、Session 載入 <50ms、Prompt 組建 <100ms、首 token 200-500ms、工具執行 100ms-3s [來源：OpenClaw Architecture, Substack, 2026]。

記憶系統

多層記憶架構：

  • SQLite + 向量嵌入：存於 ~/.openclaw/memory/.sqlite，混合搜尋（向量相似度 + BM25 關鍵字）
  • MEMORY.md：策展式長期事實記憶
  • daily log：memory/YYYY-MM-DD.md 每日活動日誌
  • Session 壓縮：自動摘要舊對話回合，防止 context overflow
  • 嵌入優先順序：本地模型 > OpenAI > Gemini API [來源：OpenClaw Architecture, Substack, 2026]

Skill 生態系統

Skill 位於 skills//SKILL.md，採選擇性注入而非全量載入，避免 prompt 膨脹。ClawHub 是社群 Skill 註冊表，目前收錄 5,400+ Skill。特色機制包括：

  • Self-Improving Agent：記錄錯誤、學習與偏好到記憶中，讓 Agent 隨使用越來越聰明
  • Foundry：元擴展（meta-extension），可自行擴展新能力、提取關鍵字做模式匹配、計算成功率並將高價值模式結晶為專屬工具 [來源：VoltAgent/awesome-openclaw-skills, GitHub] [來源：lekt9/openclaw-foundry, GitHub]

安全性深度分析（重要警示）

CVE-2026-25253：遠端程式碼執行

  • CVSS 8.8（高危）
  • 攻擊向量：WebSocket origin header 繞過
  • 影響：攻擊者可在任何暴露的 OpenClaw 實例上執行任意程式碼
  • 公網掃描發現 135,000+ 暴露實例，其中 50,000+ 直接受此漏洞影響 [來源：Security Research, 2026]

ClawHub 惡意 Skill 問題

  • 審計結果：約 2,857 個 Skill 中有 341 個（12%）含惡意程式碼
  • 類型：資料外洩程式碼、prompt injection payload
  • 總共發現 512 個漏洞，其中 8 個被評為 Critical [來源：ClawHub Security Audit, 2026]

與 Claude Code 安全模型對比

| 面向 | OpenClaw | Claude Code |

|------|----------|-------------|

| 權限模型 | 本機高權限，root 存取 | 逐指令請求權限，沙盒隔離 |

| Prompt Injection 防禦 | Context 隔離 + 結構化工具結果包裝 | 嚴格分離對話上下文與系統存取 |

| 供應鏈安全 | 12% 惡意 Skill | 封閉生態，官方審核 |

| Docker 隔離 | DM/Group session 可用 | 不適用（終端模式） |

與 daily-digest-prompt 的借鏡

| OpenClaw 特性 | 本專案對應 | 啟發 |

|--------------|-----------|------|

| SKILL.md 選擇性注入 | skills/SKILL_INDEX.md + 按需 Read | 機制高度相似，可參考其 prompt 組合策略 |

| MEMORY.md + daily log | context/digest-memory.json + continuity | 已有類似設計 |

| Gateway 控制面 | run-agent-team.ps1 Phase 0→1→2 | OpenClaw 的事件驅動可參考 |

| Self-Improving Agent | system-insight + arch-evolution | 閉環自改進理念一致 |

| ClawHub 12% 惡意 Skill | Hook 機器強制層 | 提醒外部 Skill 引入需審計 |

實務應用建議

  1. 不建議直接暴露到公網：必須走 SSH tunneling 或 Tailscale，絕不開放 18789 埠
  2. Skill 審計：從 ClawHub 安裝前應人工審查 SKILL.md 內容
  3. 本專案可借鏡：其 Session 壓縮機制（自動摘要舊 context）與 Foundry 自延展模式
  4. 模型自由度：支援 25+ LLM 提供者，一行 config 切換，適合成本敏感場景

知識缺口與未來方向

  • OpenClaw 的 multi-agent routing（不同 channel 路由到獨立 Agent）機制值得深入
  • Foundry 自延展 Skill 的品質治理機制尚不明確
  • CVE-2026-25253 修補後的安全架構改進待追蹤
