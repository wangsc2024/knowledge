Vite SSG 靜態網站生成與知識庫驅動內容管線

一、概述

本研究探討以 Vite 為核心的靜態網站生成（SSG）技術生態系統，重點聚焦於如何將 RAG 知識庫作為內容資料源，透過自動化管線生成高效能靜態知識分享網站。研究涵蓋三大框架比較（VitePress / Astro Starlight / Vite-SSG）、內容管線設計模式、增量同步策略，以及與 Cloudflare Pages 等邊緣平台的部署整合。

研究動機

知識庫（如 RAG 系統、Obsidian、Notion）中積累的知識若僅停留在 API 查詢階段，其可及性（Accessibility）和可發現性（Discoverability）受限。將知識庫轉化為靜態網站，能同時達成：

  • SEO 友善：搜尋引擎可索引所有內容
  • 離線存取：靜態 HTML 不依賴後端服務
  • 高效能：CDN 邊緣快取，TTFB < 50ms
  • 低成本：Cloudflare Pages / Netlify 免費方案即可承載

---

二、Vite SSG 生態系統三大方案

2.1 VitePress（推薦用於 Vue 生態 + 文件站）

架構：Vite + Vue 3 的 SPA/SSG 混合體

  • 初次造訪為純靜態 HTML（SEO + 快速 LCP）
  • 後續導航自動升級為 SPA（client-side routing）
  • Markdown 即 Vue SFC：100% HTML 語法相容
  • HMR 延遲僅 ~100ms

核心特色：

  • 自動搜尋：內建 MiniSearch 全文搜尋（無需額外配置）
  • 多語系：原生 i18n 支援
  • 主題系統：預設主題 + CSS Variables 自定義
  • Vue 元件嵌入：Markdown 中直接使用
