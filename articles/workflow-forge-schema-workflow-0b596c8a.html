Workflow 設計

觸發時機（3 個）

  1. Phase 2 自動任務完成後（即時驗證）

- 執行者：run-todoist-agent-team.ps1 Phase 2 completion hook

- 執行方式：uv run python tools/validate_results.py --file results/todoist-auto-$taskKey.json

- 優先級：high

- 作用：快速失敗（fail-fast），避免等到 Phase 3 才發現格式錯誤

  1. Phase 3 組裝開始前（批次驗證）

- 執行者：prompts/team/todoist-assemble.md 步驟 0

- 執行方式：uv run python tools/validate_results.py --batch results/

- 優先級：high

- 作用：最後一道防線，確保所有結果檔案格式正確，跳過無效檔案

  1. 手動驗證（開發/除錯時）

- 執行者：開發者透過 CLI

- 優先級：medium

驗證步驟（5 步）

  1. 檔案存在性檢查：驗證 results/todoist-auto-{task_key}.json 存在
  2. JSON 語法解析：用 json.load() 解析檔案
  3. Schema 驗證：對照 config/schemas/results-auto-task-schema.json 驗證必填欄位（agent, task_key, status）、型別、值域
  4. 交叉驗證：task_key 與檔名一致性
  5. 生成驗證報告：彙整結果寫入 results/validation-report.json

驗證報告格式


{
  "validated_at": "ISO-8601 timestamp",
  "total_files": 3,
  "valid_count": 2,
  "invalid_count": 1,
  "missing_count": 0,
  "invalid_files": [
    {
      "file": "results/todoist-auto-test.json",
      "errors": ["missing field: task_key"],
      "warnings": []
    }
  ],
  "valid_files": [
    "results/todoist-auto-skill_forge.json",
    "results/todoist-auto-workflow_forge.json"
  ]
}

整合計畫

Phase 2 整合（run-todoist-agent-team.ps1）


foreach ($taskKey in $selectedTaskKeys) {
    $resultFile = "results/todoist-auto-$taskKey.json"
    if (Test-Path $resultFile) {
        Write-Host "[驗證] 驗證結果檔案：$resultFile" -ForegroundColor Cyan
        uv run python tools/validate_results.py --file $resultFile
    }
}

Phase 3 整合（prompts/team/todoist-assemble.md）

在步驟 0（前置檢查）新增：


## 步驟 0：結果檔案驗證（前置檢查）

**第一步**：執行批次驗證。

uv run python tools/validate_results.py --batch results/



**第二步**：讀取驗證報告。

cat results/validation-report.json



**第三步**：檢查驗證結果。
- 若 `invalid_count > 0`：記錄無效檔案清單，在步驟 1 讀取結果時跳過這些檔案
- 若 `invalid_count == 0`：繼續執行步驟 1
