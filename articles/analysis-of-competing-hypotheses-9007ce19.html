競爭假設分析法（Analysis of Competing Hypotheses, ACH）完整指南

概述

競爭假設分析法（Analysis of Competing Hypotheses, ACH）是由 CIA 資深情報分析官 Richards J. Heuer, Jr. 於 1970 年代開發的結構化分析方法論，旨在評估多個相互競爭的假設以解釋觀察到的數據。ACH 的核心理念是以反證法（falsification）取代驗證法（verification）——不是尋找支持偏好假設的證據，而是系統性地尋找能排除假設的不一致證據。

ACH 已成為情報分析、商業決策、政策分析、AI 安全評估等領域最廣泛使用的結構化分析技術（SAT）之一。2010 年 Heuer 與 Randolph Pherson 合著的《Structured Analytic Techniques for Intelligence Analysis》進一步將 ACH 納入 55 種結構化分析技術的完整分類體系中。

「ACH 的設計目的是幫助分析師克服或至少最小化那些使前瞻性情報分析難以實現的認知限制。」— Richards Heuer

一、創始人背景：Richards J. Heuer, Jr.（1927-2018）

1.1 生平與貢獻

  • 學歷：Williams College 哲學學士（1950）、University of Southern California 國際關係碩士
  • CIA 生涯：45 年（1951-1995），歷任行動處（Directorate of Operations, 24 年）、情報處（Directorate of Intelligence, 4 年），1979 年退休後以約聘身份持續服務至 1995 年
  • 轉折點：Yuri Nosenko 案——蘇聯 KGB 叛逃者真偽判定的極度爭議案件，促使 Heuer 深入研究認知偏見對情報判斷的影響
  • 學術啟發：1977 年 ISA 會議後接觸 Kahneman 與 Tversky 的認知心理學研究，將行為科學系統性地引入情報分析

1.2 核心著作

| 著作 | 年份 | 重要性 |

|------|------|--------|

| 《Psychology of Intelligence Analysis》 | 1999 | CIA 出版，情報分析心理學奠基之作，首次系統闡述 ACH |

| 《Structured Analytic Techniques for Intelligence Analysis》（與 Pherson 合著） | 2010/2015 | 55 種 SATs 完整分類，ACH 為核心技術 |

二、ACH 的認知科學基礎

2.1 ACH 對抗的認知偏見

| 認知偏見 | 定義 | ACH 如何對抗 |

|---------|------|-------------|

| 確認偏誤（Confirmation Bias） | 傾向尋找、解讀和記憶支持既有信念的資訊 | 強制考慮所有假設，不允許只驗證偏好假設 |

| 錨定效應（Anchoring） | 過度依賴第一個接觸到的資訊 | 要求同時列出所有假設，避免被首個假設錨定 |

| 可得性偏誤（Availability Heuristic） | 容易回想的事件被認為更可能發生 | 系統性列舉所有證據，不依賴記憶可及性 |

| 過度自信（Overconfidence） | 對自己判斷的正確性過於自信 | 矩陣強制量化不一致性，暴露隱含的不確定性 |

| WYSIATI（What You See Is All There Is） | Kahneman 提出，人傾向只考慮眼前可見的資訊 | 矩陣系統性揭示證據缺口和資訊空白 |

| 群體思維（Groupthink） | 團體成員追求一致性而壓制異議 | 結構化流程使異議合法化，每個假設都被平等對待 |

2.2 反證法 vs 驗證法

傳統分析的致命缺陷是驗證偏誤：分析師先選定「最可能」的假設，然後尋找支持證據。ACH 翻轉這個邏輯：


傳統方法（驗證法）：
  選定假設 H1 → 搜尋支持 H1 的證據 → 確認 H1
  問題：忽略了同樣的證據可能也支持 H2、H3

ACH 方法（反證法）：
  列出所有假設 H1..Hn → 列出所有證據 E1..Em
  → 對每條證據問「這與哪些假設不一致？」
  → 排除最不一致的假設 → 最不被排除的即為最可能

這與 Karl Popper 的科學哲學（可否證性原則）一脈相承：好的假設必須是「可以被反駁的」，而非「可以被確認的」。

三、ACH 七步流程（標準方法論）

步驟 1：辨識所有可能假設

  • 召集具有不同觀點的分析師進行腦力激盪
  • 關鍵原則：不預設「可能」或「不可能」，將所有邏輯上可行的假設都列入
  • 建議 3-7 個假設（過少失去分析價值，過多難以管理）
  • 避免互斥疲勞：假設之間可以有重疊，但應盡量涵蓋所有可能解釋

步驟 2：列出證據與論據

  • 列出所有已知的證據、論據、假定和邏輯推論
  • 包含支持和反對各假設的資訊
  • 關鍵：同時列出「不存在的證據」（Evidence of Absence）——如果某假設為真，我們預期看到什麼證據但實際上沒看到？

步驟 3：建立診斷矩陣（最核心步驟）

建立「假設 vs 證據」的矩陣，這是 ACH 的核心工具：


              H1(假設1)  H2(假設2)  H3(假設3)  H4(假設4)
E1(證據1)      C           I           C           N
E2(證據2)      I           C           I           C
E3(證據3)      C           C           I           I
E4(證據4)      N           I           C           C
E5(缺失證據)   I           C           C           I

C = Consistent（一致）
I = Inconsistent（不一致）
N = Not Applicable（不適用）

診斷性（Diagnosticity）：並非所有證據對區分假設都同樣有用。如果一條證據對所有假設都一致（或都不一致），它的診斷性為零。高診斷性的證據是那些對某些假設一致但對其他假設不一致的證據。

橫向思考（Working Across）vs 縱向思考（Working Down）：

  • 縱向（傳統）：逐個假設檢視所有證據 → 容易產生確認偏誤
  • 橫向（ACH 推薦）：逐條證據對比所有假設 → 發現診斷性差異

步驟 4：精煉矩陣

  • 審視分析結果，辨識資訊缺口
  • 收集額外證據以消除更多假設
  • 重新評估假設的相互排斥性和完備性

步驟 5：評估不一致性

  • 計算每個假設的「不一致性分數」
  • 不一致性越高的假設越不可能為真
  • 排除最不一致的假設
  • 注意：矩陣產生的數學結果不應凌駕分析師的判斷

步驟 6：敏感性分析

  • 測試結論的穩健性：如果關鍵證據是錯誤的、具誤導性的或有不同解讀，結論會如何改變？
  • 識別「支柱」（Linchpin）證據——結論最依賴的關鍵假定
  • 雙重檢查關鍵證據的有效性和重要論據的一致性

步驟 7：結論、報告與里程碑

  • 向決策者提供結論，附上被考慮和被排除的替代方案及排除理由
  • 識別未來分析的里程碑和指標
  • 記錄推理過程以供審計和回顧

四、ACH 矩陣實戰範例

情境：某企業 AI 系統出現異常行為

假設集合：

  • H1：訓練資料污染（Data Poisoning）
  • H2：模型過擬合導致分佈外行為
  • H3：對抗性攻擊（Adversarial Attack）
  • H4：正常的罕見事件（False Alarm）

證據收集：

  • E1：異常僅出現在特定輸入模式下
  • E2：訓練資料已通過品質檢查
  • E3：生產環境有外部存取記錄
  • E4：其他類似模型未出現相同行為
  • E5：異常行為具有系統性規律（非隨機）

ACH 矩陣：


              H1(資料污染)  H2(過擬合)  H3(對抗攻擊)  H4(正常罕見)
E1(特定模式)     C           C           CC          I
E2(品質OK)       I           C           N           C
E3(外部存取)     N           N           CC          N
E4(獨特行為)     C           I           C           I
E5(系統規律)     C           I           CC          II

不一致計數：     1           3           0            3

CC = 強一致, C = 一致, I = 不一致, II = 強不一致, N = 不適用

分析結論：H3（對抗性攻擊）不一致性最低且有多條強一致證據，是最可能的解釋。H2 和 H4 各有 3 個不一致，應優先排除。

敏感性分析：如果 E3（外部存取記錄）是誤判，H1 變為最可能解釋——這是支柱證據，需雙重驗證。

五、ACH 的優勢與局限

5.1 優勢

| 優勢 | 說明 |

|------|------|

| 可審計性 | 完整的推理軌跡，決策者可追溯從證據到結論的每一步 |

| 認知去偏 | 結構化流程迫使分析師考慮所有假設，而非只看偏好假設 |

| 團隊協作 | 矩陣提供共同語言，促進多元觀點的整合 |

| 透明度 | 假定和推理都顯性化，容易被檢視和挑戰 |

| 教學價值 | 培養分析思維，適合訓練新手分析師 |

5.2 局限與批評

| 局限 | 批評者 | 說明 |

|------|--------|------|

| 耗時 | — | 建構和維護矩陣需要大量時間 |

| 離散判斷過多 | Tim van Gelder | 分析師被要求做太多細粒度判斷，許多對辨識最佳假設貢獻甚微 |

| 證據-假設關係過於簡化 | Tim van Gelder | 假設證據「本身」與假設一致或不一致，忽略了情境依賴性 |

| 假設集合扁平化 | Tim van Gelder | 將假設視為平面列表，無法處理假設之間的層次關係 |

| 無法表示從屬論證 | Tim van Gelder | 不能展示支撐某條證據的次級論證 |

| 假設生成偏見 | 社會建構論者 | ACH 未充分解決假設初始形成的問題性——文化和身份等社會建構因素可能限制哪些假設被考慮 |

| 靜態快照 | — | 矩陣中的證據是靜態的，無法反映動態變化 |

| 欺騙脆弱性 | Elsaesser & Stech | 對手（尤其在情報領域）可能蓄意產生誤導性資訊 |

六、ACH 的演進與增強型變體

6.1 結構化競爭假設分析（SACH）

SACH 允許分析師將一個假設拆分為多個子假設，提供更細緻的估計。例如：

  • 初始：「伊拉克有大規模殺傷性武器」vs「沒有」
  • 拆分：「WMD 在巴格達」/「WMD 在摩蘇爾」或「生物 WMD」/「化學 WMD」/「核 WMD」

6.2 貝葉斯增強型 ACH

Valtorta 等人將貝葉斯分析加入 ACH，開發了 CACHE（Collaborative ACH Environment），引入「貝葉斯社群」概念，支持分散式分析師團隊的協作。

6.3 主觀邏輯增強（Subjective Logic）

Pope 與 Josang 使用主觀邏輯——一種明確處理不確定性的數學方法論，形成 Sheba 技術，應用於 Veriluma 的情報評估軟體。

6.4 假設映射（Hypothesis Mapping）

Tim van Gelder 提出以假設映射（類似論證映射 Argument Mapping）取代 ACH 矩陣，解決扁平化和從屬論證的問題。

6.5 圖論方法

Akram 與 Wang 應用圖論範式，將假設和證據表示為圖結構，支持更複雜的關係建模。

七、ACH 在 AI 時代的應用（2025-2026）

7.1 AI 安全評估

  • 模型行為異常診斷：當 AI 系統出現非預期行為時，ACH 矩陣可系統性地區分訓練錯誤、對抗攻擊、過擬合等多種可能原因
  • Sleeper Agent 檢測：Microsoft 2026 年的研究使用類似 ACH 的方法論評估模型是否被植入後門
  • 對齊評估（Alignment Assessment）：評估 AI 系統是否真正「理解」其目標，而非僅在表面上符合

7.2 AI 輔助的 ACH 自動化

  • LLM 生成假設：使用 LLM 進行假設腦力激盪，突破人類分析師的認知限制
  • 自動證據提取：NLP 從大量文件中自動識別和分類相關證據
  • 動態矩陣更新：傳統 ACH 矩陣是靜態的，AI 可實現持續更新
  • 診斷性自動計算：機器學習計算每條證據的診斷性權重

7.3 商業策略分析

  • 競爭對手意圖分析：企業用 ACH 評估競爭對手的戰略意圖
  • 市場預測：評估多個市場走向假設的相對可能性
  • 風險評估：系統性地識別和排除低風險假設，聚焦高風險情境

7.4 與知識庫已有框架的互補關係

| 已有框架 | 互補方式 |

|---------|----------|

| 溯因推理（Abductive Reasoning） | ACH 是溯因推理的結構化實作，將「最佳解釋推理」流程化 |

| 貝葉斯思維（Bayesian Thinking） | 增強型 ACH 整合貝葉斯機率更新，解決定性判斷的局限 |

| OODA Loop | ACH 可嵌入 OODA 的 Orient（定向）階段，作為評估工具 |

| 預驗屍法（Pre-Mortem） | Pre-Mortem 生成失敗假設，ACH 評估這些假設的相對可能性 |

| RPD（識別啟動決策） | RPD 適用於時間壓力下的直覺決策；ACH 適用於高風險需深度分析的情境。兩者互補 |

| 系統思考 | ACH 處理線性因果假設；系統思考處理反饋迴路和涌現行為。結合使用可覆蓋更複雜的問題 |

九、ACH 實踐指南：快速入門

適用情境

  • 高風險決策（結果的代價很大）
  • 多種合理解釋並存
  • 需要向他人解釋推理過程
  • 團隊中有不同意見需要整合

不適用情境

  • 時間極度緊迫（改用 RPD）
  • 問題定義明確、無需區分假設
  • 資訊極度稀缺（連假設都難以建立）

五分鐘 ACH 快速版

  1. 寫下 3 個可能解釋（假設）
  2. 列出 5 條最重要的證據
  3. 對每條證據標記 C/I/N
  4. 找出不一致最多的假設並暫時排除
  5. 對存活假設做敏感性檢查：「如果 X 證據是錯的呢？」
