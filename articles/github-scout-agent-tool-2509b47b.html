改進提案

P0 已落實：operator_override 固定觀測要點

  • 來源專案: openai/openai-agents-python, guardrails-ai/guardrails, kubernetes-sigs/agent-sandbox
  • 改進內容: 將安全主題下的 operator_override 從自由文字收斂為固定觀測要點，至少包含覆寫入口、預設模式（allow/deny/review-only）與 rollback/降級方式。
  • 落點檔案:

- config/github-scout-review-policy.yaml

- templates/auto-tasks/github-scout.md

- prompts/team/todoist-auto-github_scout.md

P1 待審：Sandbox / Guardrail Control Plane Taxonomy

  • 來源專案: kubernetes-sigs/agent-sandbox
  • 改進內容: 為 orchestration plane / guardrail plane / sandbox plane 建立統一 taxonomy 與 feature flag rollout。
  • 計畫檔案: docs/plans/github-scout-20260329-sandbox-control-plane-taxonomy.md

審查結論

  • P0 審查分數: feasibility 9 / stability 8
  • P0 審查說明: 只修改研究契約層，不碰核心 hooks、scheduler-state 或 run 腳本；rollback 為還原 config/template/prompt 備份。工具邊界/護欄影響僅限研究欄位更具體，不改實際攔截策略。
  • P1 審查分數: feasibility 6 / stability 6
  • P1 審查說明: 若直接接入執行層，會影響 sandbox 與護欄決策路徑，因此僅輸出計畫並保留人工審核。降級方式為維持 review-only 流程。

已落實項目

  • 已把 operator_override 的固定觀測要點寫入 review policy、auto-task template 與 team prompt。

待人工審核項目
