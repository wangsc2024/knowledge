關鍵決策

  1. 選擇 Bash 萃取而非 offset/limit：

- improvement-backlog.json 是結構化 JSON，需篩選 status=pending

- Read offset/limit 無法做條件篩選，仍會讀取大量無關資料

- Bash + Python 可精確萃取必要欄位，輸出最小化

  1. 不修改 arch_evolution.md：

- 已有完整保護機制（第 48-74 行）

- 萃取時還做了 description 截斷（150 字元）

- 有錯誤處理機制（FileNotFoundError / Exception）

- 無需重複優化

  1. 清理殭屍快取而非調整 TTL：

- knowledge.json 16 天未更新，可能是服務中斷期間遺留

- TTL 已從 120 → 240 分鐘（4h），符合 low volatility 策略

- 清理後下次查詢將重建，恢復正常快取循環
