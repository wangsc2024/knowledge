OpenClaw — 2026 年 GitHub 史上增長最快的開源個人 AI Agent

概述

OpenClaw 是由 PSPDFKit 創辦人 Peter Steinberger 開發的開源個人 AI 助理，2025 年 11 月以 Clawdbot 之名發布，2026 年 1 月 30 日正式更名為 OpenClaw。專案在四個月內從 9,000 Stars 飆升至 250,000+ Stars，超越 React 成為 GitHub 史上增長最快的非聚合類專案。2026 年 2 月 14 日，創始人宣布加入 OpenAI，專案將移交開源基金會管理。

---

一、核心架構：本地優先閘道器（Local-First Gateway）

1.1 設計哲學

OpenClaw 採用「本地閘道器」架構，在使用者自有設備上運行，作為 LLM 與系統資源之間的橋梁。核心理念是隱私優先 — 所有設定資料與對話歷史儲存在本地 Markdown 文件中。

1.2 架構組成

| 層級 | 功能 | 說明 |

|------|------|------|

| 閘道器層 | LLM 橋接 | 連接任意 AI 模型（Claude、GPT、DeepSeek、本地模型） |

| 執行層 | 系統存取 | 讀寫檔案、執行 shell 命令、瀏覽器自動化 |

| 整合層 | 通訊橋接 | 50+ 整合（WhatsApp、Telegram、Slack、Discord、LINE、iMessage 等） |

| 記憶層 | 持久記憶 | 本地 Markdown 文件儲存偏好與上下文 |

| 技能層 | Skills 系統 | 100+ 預配置 AgentSkills，支援自我擴展 |

1.3 模型不可知（Model Agnostic）

  • 自帶 API Key：使用者提供自己的 API 金鑰
  • 完全免費：開源，僅有 AI 模型 API 使用費
  • 支援本地模型：可與 Ollama 等本地推論引擎搭配

---

二、Skills 系統與自我進化

2.1 Skills 架構

OpenClaw 的 Skills 系統與 Claude Code 的 SKILL.md 架構高度相似：

  • Skills 以目錄形式儲存，每個目錄包含 SKILL.md 元資料檔
  • 100+ 預配置 AgentSkills 涵蓋常見工作流
  • 自主擴展：Agent 可自動撰寫程式碼建立新 Skills

2.2 核心能力

  • 持久記憶：跨 session 保留使用者偏好與上下文
  • 檔案系統存取：讀寫檔案、執行 shell 命令（支援沙箱模式）
  • 瀏覽器控制：自動化表單填寫、資料擷取
  • 主動自動化：可設定觸發條件自動執行任務

三、安全爭議與產業回應

3.1 安全疑慮

  • Gartner：稱 OpenClaw 設計「預設不安全」（insecure by default）
  • Cisco 分析師：標記為「安全噩夢」
  • 風險點：自主執行程式碼、存取敏感資料、跨平台操作

3.2 Nvidia NemoClaw 回應

Nvidia CEO Jensen Huang 稱 OpenClaw 為「可能是有史以來最重要的軟體發布」，並推出 NemoClaw 強化方案：

| 元件 | 功能 |

|------|------|

| Nemotron 模型 | Nvidia 自研 LLM |

| OpenShell Runtime | 網路隔離、隱私路由、策略強制執行 |

| 安全合作夥伴 | CrowdStrike、Cisco、Google、Microsoft |

3.3 產業定位

Huang 將 AI 演進分為三代：

  1. 生成（ChatGPT）：對話式 AI
  2. 推理（o1/o3）：深度思考
  3. 代理（OpenClaw）：自主任務執行

Nvidia 預估 2027 年因 Agentic AI 推論需求，算力市場將超過 1 兆美元。
