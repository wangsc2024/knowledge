OpenClaw — 史上成長最快的開源個人 AI 助手

概述

OpenClaw 是 2026 年 GitHub 上成長最快的開源專案，由 PSPDFKit 創辦人 Peter Steinberger 開發。專案在 2026 年 1 月底爆紅後，10 天內從 9,000 飆升至 210,000+ Stars，截至 2026 年 3 月初已達 247,000 Stars、47,700 Forks。採用 MIT 授權，完全開源。

OpenClaw 的核心定位是「在你自己設備上運行的個人 AI 助手」，透過本地 Gateway 連接 AI 模型與 50+ 第三方整合，資料永遠不離開你的機器。

核心架構：Hub-and-Spoke Gateway

Gateway 控制平面

OpenClaw 採用 Hub-and-Spoke 架構，核心是一個長期運行的 Gateway 進程，作為控制平面（Control Plane）：

  • WebSocket Server：連接所有訊息平台與控制介面
  • Session 管理：管理使用者會話、頻道、工具與事件
  • Provider 連接：擁有所有 AI 供應商的連線，其他元件透過 WebSocket 連接
  • 訊息路由：將每個路由後的訊息分派到 Agent Runtime

Agent Runtime 執行模型

Agent Runtime 以 序列化迴圈（Serialized Loop） 運行，每個 Session 一次只執行一個 Run：

  1. Intake：接收輸入
  2. Context Assembly：組裝上下文
  3. Model Inference：模型推理
  4. Tool Execution：工具執行
  5. Streaming Replies：串流回覆
  6. Persistence：持久化

多代理路由（Multi-Agent Routing）

支援將不同的入站頻道/帳號/對話路由到隔離的 Agent（獨立 Workspace + per-agent Sessions），實現多用戶/多情境的隔離管理。

50+ 訊息平台整合

OpenClaw 支援的通訊頻道涵蓋主流平台：

| 類別 | 平台 |

|------|------|

| 即時通訊 | WhatsApp、Telegram、Signal、LINE、Zalo |

| 企業協作 | Slack、Discord、Microsoft Teams、Google Chat、Mattermost |

| 社群 | IRC、Matrix、Nostr、Twitch |

| Apple 生態 | iMessage（legacy）、BlueBubbles |

| 其他 | Feishu、Nextcloud Talk、Synology Chat、Tlon、WebChat |

| 原生應用 | macOS App、iOS/Android App、CLI |

Skills 系統

OpenClaw 使用目錄式 Skills 系統：

  • 每個 Skill 是一個目錄，包含 SKILL.md 元資料與使用說明
  • 三層優先級：Workspace Skills > Global Skills > Bundled Skills
  • 可自主學習新 Skills：Agent 能自動撰寫程式碼建立新 Skill
  • 100+ 預配置 AgentSkills

核心工具能力

  • 檔案系統：讀寫檔案、管理目錄
  • Shell 執行：運行命令列指令
  • 瀏覽器控制：自動化網頁操作、表單填寫、資料擷取
  • 排程自動化：Cron 任務、Canvas、Nodes
  • API 控制：發送郵件、管理行事曆、智慧家居控制

本地優先隱私模型

資料主權

  • 記憶儲存：以本地 Markdown 檔案儲存，不依賴雲端
  • 模型無關：支援雲端 API（自帶 API Key）或本地模型部署
  • 無訂閱制：費用僅來自模型 API 使用量
  • 沙箱模式：可選擇 sandbox 或完整系統存取

Heartbeat Daemon（心跳守護進程）

OpenClaw 內建自主排程的心跳守護進程，可在無使用者提示的情況下主動執行任務——這是大多數 AI Agent 不具備的特性。

與其他 AI Agent 的定位差異

| 維度 | OpenClaw | 傳統 AI Agent |

|------|----------|---------------|

| 資料位置 | 本地 Markdown | 雲端資料庫 |

| 授權 | MIT 開源 | 多為商業/受限 |

| 自主性 | Heartbeat 主動行動 | 被動回應 |

| 整合廣度 | 50+ 平台原生支援 | 通常 5-10 個 |

| Skills 擴展 | 自主學習+建立 | 手動配置 |

| 模型綁定 | 模型無關 | 通常綁定單一供應商 |

借鏡價值（與 daily-digest-prompt 專案對照）

  1. Skills 系統同構：OpenClaw 的 SKILL.md 目錄式設計與本專案的 Skill-First 策略高度相似
  2. Heartbeat 守護進程：類似本專案的 scheduler-state + PowerShell 排程機制
  3. 本地 Markdown 記憶：與 digest-memory.json 的跨次記憶理念一致
  4. Multi-Agent 路由：與本專案的 routing.yaml 三層路由有異曲同工之處
  5. Gateway 架構：Hub-and-Spoke 模式可借鏡於 LINE Bot 與 Gun Relay 的訊息路由設計
