mem0ai/mem0（57,735★）

  • memory_architecture: 混合存儲三層（user/session/agent scope）：語義向量 + key-value + 知識圖譜並行，intelligent layer 自動決定 add/update/delete 操作，去重合併相似記憶
  • storage_backends: Qdrant（向量）、Redis（快取/KV）、SQLite（本地，可插拔 PostgreSQL/MongoDB）
  • retrieval_strategy: 語義向量 + BM25 + entity boosting 並行融合評分；跨 scope 查詢；不支援時態精確查詢

getzep/graphiti（27,010★）

  • memory_architecture: 時態知識圖（bi-temporal model）：facts 帶 valid_from/invalid_from，支援查詢特定時刻有效的事實；即時增量更新無需批次重建
  • storage_backends: Neo4j（圖），可選 PostgreSQL（關係型 fallback）
  • retrieval_strategy: 圖遍歷 + 混合向量/BM25 + 時態過濾；精確查詢 T 時刻有效 facts

microsoft/graphrag（33,458★）

  • memory_architecture: 圖式 RAG 兩層索引：全局 Community Summary + 局部 Entity Subgraph；批次離線建圖
  • retrieval_strategy: Global Search（廣泛問題）vs Local Search（精確問題）雙模搭配

letta-ai/letta（23,147★）

  • memory_architecture: OS 式三層記憶：core memory（LLM context 有限工作區）、archival memory（外部無上限向量儲存）、recall memory（最近對話/事件）
  • storage_backends: core: LLM context；archival: Qdrant/Pinecone；recall: SQLite/PostgreSQL
  • retrieval_strategy: 分層路由：精確匹配（recall）、語義向量搜索（archival）；LLM 自主呼叫 memory_search 工具；memory paging 避免 context overflow
