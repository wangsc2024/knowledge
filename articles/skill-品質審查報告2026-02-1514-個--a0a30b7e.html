發現的問題與改善建議

P1：pingtung-policy-expert 非標準欄位

問題：description 中包含 Must pair with: pingtung-news，這是非標準的 frontmatter 格式。

建議：

  • 從 description 移除 Must pair with: 行
  • 新增正式的 depends-on: [pingtung-news] 欄位
  • 版本升級至 1.0.1

P2：8 個 Skill 缺少 cache-ttl 明確標注

問題：以下 Skill 未在 frontmatter 中標注 cache-ttl，雖然對非 API Skill 這不是錯誤，但會造成理解模糊。

受影響 Skill：skill-scanner、pingtung-policy-expert、ntfy-notify、scheduler-state、atomic-habits、learning-mastery、digest-memory、api-cache

建議：為所有非 API Skill 添加 cache-ttl: N/A 明確標示不需要快取。

P3：缺乏正式的依賴宣告機制

問題：Skill 間的依賴關係（如 pingtung-policy-expert 必搭 pingtung-news）僅在 description 或 SKILL_INDEX.md 中文字描述，缺少結構化的 depends-on 欄位。

已識別依賴：

  • pingtung-policy-expert → pingtung-news（必搭）
  • game-design → knowledge-query（建議搭配）
  • api-cache → 所有 API Skill（包裝關係）

建議：在 frontmatter 規範中新增可選的 depends-on 陣列欄位。
