一、CRAAP Test — 學術界標準的五維度來源評估

1.1 起源與發展

CRAAPTest 由 Sarah Blakeslee 及加州州立大學奇科分校（CSU Chico）圖書館團隊於 2004 年開發，是資訊素養（Information Literacy）教育中最廣泛採用的來源評估工具。

1.2 五維度評估框架

Currency（時效性）

  • 資訊發布或最後更新的日期
  • 該主題是否有重要的新發展
  • 連結是否仍然有效
  • 適用問題：「這份資料是否反映了最新的研究成果？」

Relevance（相關性）

  • 資訊是否直接回答你的研究問題
  • 內容的深度與廣度是否適當
  • 是否支持你的論點
  • 目標受眾的知識水準是否匹配

Authority（權威性）

  • 作者或組織的資歷與專業背景
  • 域名類型暗示（.edu、.gov、.org、.com）
  • 是否經過編輯審查流程
  • 內容是否可能被未授權修改

Accuracy（準確性）

  • 主張是否有充分證據支持
  • 資訊是否可透過其他來源交叉驗證
  • 是否經過同儕審查
  • 是否承認替代觀點的存在
  • 紅旗警示：抄襲、誇大、偏見、邏輯謬誤、文法錯誤

Purpose（目的性）

  • 資訊存在的原因（教育、告知、娛樂、說服）
  • 是否有商業、政治或宗教動機
  • 廣告是否清楚標示

1.3 局限性與批評

2020 年，Stanford 的 Sam Wineburg 等人在工作論文中指出，CRAAP 方法使學生「容易受到錯誤資訊影響」（susceptible to misinformation）。主要批評：

  • 表面特徵偏見：學生傾向於評估網站的外觀專業度而非實質內容
  • 缺乏橫向閱讀：CRAAP 鼓勵縱向深入單一來源，而非橫向比對多來源
  • 對精心包裝的假資訊無效：設計精良的偽科學網站可能通過所有五項測試

二、SIFT Method — 數位時代的快速事實查核

2.1 起源與設計理念

SIFT 由數位素養專家 Mike Caulfield 於 2017 年提出，原名「The 4 Moves and a Habit」。其核心理念是：重新脈絡化（Recontextualization） — 還原網路內容通常被剝離的背景脈絡。

2.2 四步驟框架

S — Stop（停下來）

  • 在閱讀或分享內容之前暫停
  • 評估你是否認識這個來源及其聲譽
  • 雙重用途：初始評估 + 過程中的重新聚焦
  • 關鍵原則：「快速而淺層的調查將構成我們在網路上大多數的行為」

I — Investigate the Source（調查來源）

  • 花約 60 秒了解出版者
  • 檢視作者的專業資歷
  • 了解來源的議程或利益關係
  • 查看準確性和公正性的歷史記錄
  • Caulfield 推薦「Just add Wikipedia」策略：用維基百科快速了解網站背景

F — Find Better Coverage（尋找更好的報導）

  • 不接受單一文章的框架
  • 搜尋多個受信任的來源探討相同主張
  • 識別專家共識
  • 理解更廣泛的脈絡和歷史

T — Trace Claims（追溯原始脈絡）

  • 追蹤引用、影片、圖片到原始來源
  • 網路內容常因以下原因失去脈絡：選擇性剪輯、誤導性標題、錯誤呈現的研究結果
  • 還原原始脈絡以判斷你看到的版本是否被準確呈現

2.3 SIFT vs CRAAP 的關鍵差異

| 面向 | CRAAP Test | SIFT Method |

|------|-----------|-------------|

| 閱讀策略 | 縱向（深入單一來源） | 橫向（跨來源比對） |

| 速度 | 較慢（系統性評估） | 較快（60 秒初步判斷） |

| 適用場景 | 學術研究、論文寫作 | 日常網路瀏覽、快速事實查核 |

| 抗偽資訊能力 | 較弱（可被表面特徵欺騙） | 較強（橫向閱讀揭露偽裝） |

| 重點 | 來源本身的品質屬性 | 外部對來源的評價與共識 |

三、AI 多代理自動化可信度評估管線

3.1 架構概述（Frontiers in AI, 2025）

一篇 2025 年發表在 Frontiers in Artificial Intelligence 的研究提出了三階段順序管線：

  1. Content Analyzer Agent（內容分析代理）

- 主張提取與分類

- 使用 NLP 技術：命名實體辨識、依存句法分析、語義分析

  1. Scientific Fact Verifier Agent（科學事實驗證代理）

- 從權威來源檢索證據

- 使用 Serper API 即時從 WHO、CDC、PubMed Central、Cochrane 等預定義權威域名檢索

- 證據在內部合成而非原始附加到提示中，降低幻覺風險

  1. Health Evidence Assessor Agent（健康證據評估代理）

- 可信度評分與五級分類

3.2 五維度加權評分公式


Score = (Eq × 0.4) + (Sc × 0.25) + (Ec × 0.15) + (Er × 0.1) + (Sp × 0.1)

| 維度 | 權重 | 說明 |

|------|------|------|

| Evidence Quality（證據品質） | 40% | 支持研究的方法論嚴謹度 |

| Scientific Consensus（科學共識） | 25% | 權威來源間的一致性 |

| Evidence Consistency（證據一致性） | 15% | 多項研究間的吻合度 |

| Evidence Recency（證據時效性） | 10% | 與當前理解的對齊程度 |

| Scientific Plausibility（科學合理性） | 10% | 與既有原理的一致性 |

3.3 五級分類系統

| 等級 | 分數範圍 | 判定標準 |

|------|---------|----------|

| Highly Likely | 81-100 | 壓倒性科學證據與共識 |

| Likely | 61-80 | 充分支持證據，小幅限制 |

| Moderate | 41-60 | 混合證據，共識發展中 |

| Unlikely | 21-40 | 有限支持，大量矛盾發現 |

| Highly Unlikely | 0-20 | 直接與既有科學共識矛盾 |

3.4 與人類專家的對比評估

  • 平均絕對誤差：6.25 分（0-100 量表）
  • 加權 Cohen's κ：0.68（95% CI: 0.52-0.84）— 「實質性一致」
  • 精確等級一致率：70%
  • 相鄰等級一致率：95%
  • 處理速度：每主張 <7 秒 vs 人工審查 2-4 小時
  • 已知偏差：保守偏差（+3.25 分），「高度不可能」類別召回率 0%

四、對 Daily-Digest 研究型任務的實踐啟示

4.1 整合建議：CRAAP + SIFT 雙層評估


第一層：SIFT 快速篩選（~60 秒）
  Stop → 是否認識此來源？
  Investigate → Wikipedia 快查來源背景
  Find → 是否有其他可信來源報導相同內容？
  Trace → 原始資料在哪裡？
  ↓
通過 SIFT 的來源進入第二層
  ↓
第二層：CRAAP 深度評估（研究級來源）
  Currency → 2 年內？
  Relevance → 直接回答研究問題？
  Authority → .edu/.gov/知名機構？
  Accuracy → 可交叉驗證？有同儕審查？
  Purpose → 教育目的而非商業推廣？

4.2 映射到 web-research Skill 的來源品質分級

| web-research 等級 | CRAAP 對應 | SIFT 對應 |

|-------------------|-----------|----------|

| A（官方文件、學術論文） | 5 項全高分 | 來源廣受認可，多來源引用 |

| B（知名技術部落格） | 權威性+準確性高 | 來源可查，有專業背景 |

| C（一般部落格、論壇） | 部分維度合格 | 需額外交叉驗證 |

| D（未知來源） | 多維度不合格 | SIFT 第一步即觸發警告 |

4.3 自動化可信度評估的可行路徑

借鑑 Frontiers 論文的三代理管線，可為 Daily-Digest 設計簡化版：

  1. Source Classifier：基於域名白名單 + URL 模式匹配快速分級（A/B/C/D）
  2. Content Verifier：對 B/C 級來源用 KB hybrid search 交叉驗證核心主張
  3. Consensus Scorer：統計同一主張被多少獨立來源支持，計算共識分數
