Skill 品質深度審查 foundation 深化研究

文件狀態：知識庫深化研究 (Foundation Phase)
核心目標：整合 skill_quality_audit 系列既有證據，建立 AI Agent Skill 品質審查的結構化知識基底，並對接 OODA 決策循環。
技術背景：由 LangGraph research_langgraph_v2 工作流合成，旨在將分散於 ai_deep_research、tech_research 等自動化任務中的碎片化資訊，轉化為具備高資訊熵的實踐指南。

1. 研究背景與架構設計

1.1 研究動機

隨著 AI Agent 系統從實驗室走向生產環境，Skill（技能/能力模組）的品質穩定性成為核心瓶頸。現有系統（如 daily-digest-prompt）雖已具備自動化任務能力，但缺乏統一的品質審查標準與機制。本研究旨在補齊「Foundation」階段的理論與實踐缺口。

1.2 論述邏輯：OODA 循環對接

本文採用「機制 $\rightarrow$ 應用 $\rightarrow$ 風險」的遞進結構，對應決策循環：

  • Observe / Orient (觀察與定向)：透過核心內容分析建立概念模型。
  • Decide (決策)：透過「實踐應用 Playbook」提供具體執行路徑。
  • Act (行動)：落實於具體工作流，並透過「風險與權衡」進行反饋修正。

---

2. 核心機制分析：以《楞嚴經》修證體系為隱喻的品質審查模型

透過對證據 1（KB id: 63e5f188-49b）的深度解構，我們發現高品質 Skill 的開發與審查，可借鑒「圓頓止觀」的修證邏輯，建立一套從「穩定性」到「智慧化」的層次化框架。

2.1 三位一體的品質維度表

| 品質層次 | 對應修法 | 核心目標 (Objective) | 對治問題 (Countermeasures) | 理想狀態 (Target State) |

| :--- | :--- | :--- | :--- | :--- |

| 第一層：穩定性 | 奢摩他 (止) | 建立基礎穩定性，確保輸出不偏離預期。 | 對治散亂、隨機性過高、指令漂移。 | 心得安住：輸出穩定、符合規範。 |

| 第二層：精準度 | 三摩 (觀) | 強化對上下文與邏輯的洞察力，實現精準判斷。 | 對治昏沉、無記、邏輯空洞、缺乏深度。 | 智慧明照：能看破幻象，精準捕捉關鍵資訊。 |

| 第三層：綜合力 | 禪那 (止觀等持) | 實現穩定性與精準度的動態平衡。 | 對治「止多觀少」（枯定）或「觀多止少」（掉舉）。 | 定慧圓融：具備高度適應性與複雜問題處理能力。 |

2.2 品質失衡的風險預警

  • 過度追求穩定 (止多觀少)：Skill 表現為「土木金石」狀態，即輸出極度穩定但內容僵化、缺乏洞察力，無法處理複雜變量。
  • 過度追求靈活性 (觀多止少)：Skill 表現為「風中之燈」狀態，即邏輯看似豐富但極不穩定，容易產生幻覺（Hallucination）或隨機跳躍。

3. 實踐應用 Playbook (初步框架)

基於上述機制，Skill 品質審查應遵循以下操作步驟：

3.1 建立「如來藏」式底層架構 (Foundation Setup)

  • 確立本體：在設計 Skill 時，必須先定義其「不生不滅」的核心邏輯（即不可變的指令集與約束條件），而非僅從生滅的 Prompt 表面進行修補。
  • 入口選擇：參考「耳根圓通」法門，尋找最能觸發 Agent 核心能力的輸入介面（如特定的 API 結構或 Context 注入點）。

3.2 漸次優化路徑 (Iterative Optimization)

  1. 除其助因 (Stabilization)：透過負向約束（Negative Prompting）與格式強制，解決散亂問題。
  2. 建立觀照 (Observation)：引入檢核機制（Verifier/Critic Agent），對輸出進行邏輯與事實的對比。
  3. 圓頓集成 (Integration)：透過多 Agent 協作或長鏈工作流，實現止觀平衡的最終狀態。

4. 待研究議題與檢索關鍵詞

4.1 後續研究方向

  • AI Agent 穩定性量化指標：如何將「止」與「觀」的程度轉化為可測量的數據（如 Entropy, Perplexity, Success Rate）。
  • 動態權衡機制：研究在不同任務場景下，如何自動調整 Skill 的「止」與「觀」比例。
