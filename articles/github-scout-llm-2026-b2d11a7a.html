改進提案

P1-A：對齊 OTel GenAI 語義約定（post_tool_logger.py）

  • 目標：將 post_tool_logger.py 的 JSONL 欄位升級至 gen_ai.input.messages / gen_ai.output.messages（OTel v1.38.0 新屬性）
  • 價值：直接接入 Phoenix/Langfuse 可視化，無需轉換層
  • 風險：medium（需漸進遷移，保留舊欄位向後相容）
  • rollback：保留舊欄位 6 個月，v flag 切換

P1-B：整合 agentevals 做 no-re-run eval

  • 目標：讀取 post_tool_logger.py 的 JSONL，轉換為 OTel spans，用 agentevals 評分 done_cert quality_score
  • 價值：現有 quality_score 是 LLM self-assessment，agentevals 提供獨立 Task Completion / Tool Correctness 驗證
  • 風險：low（唯讀分析，不改現有流程）
  • rollback：feature flag 關閉即可

P2：traceloop/openllmetry 儀器化

  • 目標：在 tools/ollama_local.py 的 claude API 呼叫層加裝 openllmetry 儀器，自動產出 gen_ai.system=ollama 標準 spans
  • 風險：low（decorator 方式，不改業務邏輯）
