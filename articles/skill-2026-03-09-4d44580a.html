修正內容（3 項）

#### 1. 新增 WebSearch/WebFetch 具體範例

在「工具選擇策略」後新增段落，提供 3 輪搜尋策略：

第一輪：領域全景搜尋


WebSearch: "{領域名稱} 核心概念 定義 體系"
WebSearch: "{領域名稱} history development evolution"
WebSearch: "{領域名稱} 主要分支 分類 taxonomy"

第二輪：深度概念搜尋


WebSearch: "{概念名稱} 詳細說明 原理 mechanism"
WebSearch: "{概念名稱} 實例 案例 example application"
WebSearch: "{概念名稱} vs {相近概念} 比較 差異"

第三輪：權威來源


WebSearch: "{領域名稱} 經典書籍 推薦 必讀"
WebSearch: "{領域名稱} 權威論文 seminal papers"

#### 2. 新增錯誤處理與降級策略

增加完整段落「錯誤處理與降級策略」，包含：

8 種錯誤情境表格：

  • WebSearch 失敗或無結果 → 改用 Claude 內建知識
  • WebFetch 超時或 404 → 跳過該來源
  • Phase 2.5 素材不足 1500 字 → 自動補充「詳細說明」區塊
  • build_vault.sh 腳本不存在 → 改用內建 Python 建置邏輯
  • Obsidian 配置檔案生成失敗 → 跳過 .obsidian/
  • 筆記名稱重複 → 自動加序號
  • JSON 解析失敗 → 清理格式後重試
  • 磁碟空間不足 → 中止建置

4 個品質閘門：

  • Phase 1：研究報告 ≥ 3000 字，≥ 10 個核心概念，≥ 3 個權威來源
  • Phase 2：結構符合規範，筆記數 15-50，名稱唯一
  • Phase 2.5：每則 ≥ 1500 字，必填欄位完整，連結目標存在
  • Phase 3：所有檔案成功建立，雙向連結驗證通過，Dataview 語法正確

#### 3. 修正 build_vault.sh 依賴問題

在 Phase 3-5 增加降級方案：

優先方案：使用 scripts/build_vault.sh（若存在）

降級方案：若腳本不存在，使用內建建置邏輯

  • 讀取 outline.md 解析目錄結構
  • 讀取 content_materials.json 載入筆記素材
  • 使用 Bash mkdir -p 建立所有目錄
  • 使用 Write 工具逐一生成知識筆記、MOC、配置
  • 驗證雙向連結完整性

修正後評分：4.5/5
