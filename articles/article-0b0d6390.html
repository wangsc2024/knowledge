多模態模型最新進展 深度研究報告

執行摘要

本報告依 context/ai-research-plan.json 所列四項研究問題，綜合學術調查、官方 API 文件、產業部落格與中文場域案例，說明多模態大語言模型（MLLM／VLM）在 2025–2026 年的技術主軸與落地風險。共識上，社群正從「對比式雙編碼器＋檢索」轉向「以強大語言模型為中樞的指令微調與輕量橋接（projector／adapter／LoRA）」[來源：Vision Language Models: A Survey of 26K Papers, 2025]；同時，原生多模態（文字／影像／音訊／影片統一建模）在 Gemini 2.5 等產品線已進入可對話、可串流、可工具呼叫的實用階段 [來源：Advanced audio dialog and generation with Gemini 2.5, 2025]。評測面，MMMU 指出大學科層級、異質圖像與專家推理仍大幅拉開模型差距，GPT‑4V 類系統在基準上僅約 56% 準確率，顯示「感知＋知識＋推理」三合一仍未解 [來源：MMMU benchmark site, 2024]。HEMM 進一步把失敗案例對應到「多步推理、外部知識、醫療／科學細粒度對齊」等維度 [來源：HEMM: Holistic Evaluation of Multimodal Foundation Models, 2024]。在工程實務上，OpenAI 官方文件釐清了影像 token 計費、detail 層級與解析度上限，是成本與品質折衷的單一真相來源 [來源：Images and vision — OpenAI API docs, 2025]；而中文產業媒體則記錄 Phi‑4‑multimodal 以約 56 億參數在裝置端整合語音／影像／文字，並在 ASR WER 上對標 Whisper 系列 [來源：iThome 報導 Phi‑4‑multimodal, 2025]。台灣場域中，資策會案例顯示多模態摘要可補強僅有逐字稿時的長照訪視缺口 [來源：新通訊／資策會專欄, 2026]。整體結論：（1）架構典範轉向 LLM 中心＋PEFT；（2）原生多模態把產品體驗推到即時互動；（3）評測揭露高難度領域仍是短板；（4）上線關鍵在於解析度、OCR／vision 路由與幻覺治理；（5）中小模型＋在地資料成為邊緣與法遵情境的可行路徑。

研究背景與動機

單模態 LLM 已無法涵蓋企業與公共部門累積的圖表、掃描件、醫學影像、UI 截圖與音訊紀錄。多模態模型的目標，是把「可計算的特徵抽取」與「可推理的語言介面」縫合成同一決策介面。然兩大缺口仍在：其一，學術社群需要可橫向比較的基準與失敗分類（MMMU、HEMM）；其二，工程團隊需要把示範級 demo 轉成可觀測、可降級、可稽核的生產管線。若缺乏第二點，即使模型版本快速迭代，運維成本與錯誤外溢仍會吞噬收益。本研究因此同時追蹤「文獻中的典範轉移」與「文件中的 API／token 行為」，並以中文案例對照國際產品節奏，回答計畫中的四個研究問題。

核心技術解析

子題 A：從對比預訓練到指令式 VLM／MLLM

2023–2025 年頂會論文文本挖掘顯示，VLM／LLM 相關摘要占比由約 16% 升至約 40%，且訓練目標從純對比式 InfoNCE 轉為更多交叉熵、排序與蒸餾混合；同時，LLaVA 家族在摘要中的能見度快速上升，反映「以對話與指令資料微調強大 LLM」成為主流配方 [來源：Vision Language Models: A Survey of 26K Papers, 2025]。結構上，典型 MLLM 仍由視覺編碼器、語言模型與銜接模組（如 Q‑Former、MLP projector）組成，但創新重心已前移到資料管線（高品質多模態對話合成）與參數高效調適，而非重新訓練整個視覺骨干。

子題 B：原生多模態與即時音訊／視訊互動

Google DeepMind 對 Gemini 2.5 的說明強調模型自底層即為多模態，能在低延遲下進行語音對話、風格控制、工具呼叫、背景語音判斷與音訊－視訊串流理解 [來源：Advanced audio dialog and generation with Gemini 2.5, 2025]。與「先 ASR 再 LLM」的級聯式系統相比，原生路徑可減少模態轉換造成的語用與韵律資訊流失，但換來的是更重的安全與合成內容標記（SynthID 等）責任邊界。對開發者而言，Vertex／AI Studio 通道意味著多模態能力與雲端配額、區域法遵綁定，架構設計需把「模型能力」與「部署邊界」分開治理。

子題 C：評測、難例與「專家級 AGI」敘事

MMMU 以約 1.15 萬道大學程度題目、跨 30 科目與 30 餘種異質圖像類型，測試 LMM 的感知、知識與推理；官方網站指出 GPT‑4V 早期僅約 56% 準確率，並持續更新排行榜與 MMMU‑Pro 等更嚴格變體 [來源：MMMU benchmark site, 2024]。GitHub 上的 mmmu-benchmark/mmmu 儲存庫提供可重現評估程式，利於把論文宣稱與本地環境對齊 [來源：mmmu-benchmark/mmmu, 2024]。HEMM 則以 30 個資料集從「基礎技能、資訊流、真實場域用例」三層評估，指出醫療、HCI、科學子集合平均表現顯著低於多媒體娛樂向任務 [來源：HEMM, 2024]。兩套框架共同支持一個工程判斷：垂直領域上線前，必須自建與 MMMU／HEMM 類似的分層測試，而非只跑公開總分。

子題 D：OpenAI 視覺 API 的計費與限制作為系統約束

官方文件將影像輸入支援格式、每請求大小上限、detail 參數（low／high／original／auto）以及 patch／tile 與 token 換算寫死為可計算規則，並明列醫學影像不適用、非拉丁文字、小字、圖表線條樣式、全景變形等限制 [來源：Images and vision — OpenAI API docs, 2025]。這些條款直接影響 OCR 替代方案、前處理解析度與人機審查門檻，是撰寫 SLO 與錯誤預算時的硬約束。

關鍵洞見

  1. ✅ 典範轉移有量化證據：VLM 論文占比與訓練目標的統計變化，支持「社群資源移向 MLLM＋指令微調」的論述；支持來源包含大規模文獻計量與模型族譜表 [來源：Survey of 26K Papers, 2025]（≥3 類佐證：論文統計、模型表、趨勢圖敘述）。
  2. ✅ 原生多模態把 UX 推到「連續互動」：低延遲語音、工具呼叫與串流視訊理解，使產品形態接近即時助理；支持來源為官方技術部落格與開發者入口說明 [來源：Gemini 2.5 blog, 2025]。
  3. ⚠️ 「分數高」不等於「你的文件可解」：MMMU／HEMM 顯示醫療、化學結構、長尾圖像類型仍集體失分；單一排行榜不足以驗證落地 [來源：MMMU site, 2024；HEMM, 2024]。
  4. ✅ 成本與延遲由「解析度×token 規則」主導：部落格與官方文件交錯指出：高解析度使視覺編碼器成為瓶頸，且 token 近似随面积超線性成長；必須以實測決定降採樣策略 [來源：tianpan.co 生產經驗文, 2026；OpenAI vision docs, 2025]。
  5. ⚠️ 幻覺在劣化掃描上呈「自信錯誤」：生產部落格描述模型在無法清楚閱讀時仍輸出合理數字，與 OpenAI「準確性有限」的自述形成互補警示 [來源：tianpan.co, 2026；OpenAI limitations, 2025]。反方觀點：部分廠商行銷材料可能淡化長尾風險——因此需以內部對抗樣本集驗證。

實務應用建議

  1. 建立「解析度—延遲—成本」矩陣：以官方 patch／tile 規則估算最壞 token，並在 staging 重放真實相機／掃描器解析度分佈 [來源：OpenAI vision docs, 2025]。
  2. 實作 OCR／Vision 二階路由器：乾淨表單走 OCR＋結構化抽取；版面混亂或手寫走多模態 LLM；記錄分歧案例以供再訓練 [來源：tianpan.co, 2026]。
  3. 垂直領域基準先於上線：從 MMMU 題型與 HEMM 場域集合各抽樣固定題組，作為回歸測試；避免只看總分 [來源：MMMU；HEMM, 2024]。
  4. 裝置端與邊緣場景評估 SLM：參考 Phi‑4‑multimodal 的參數量、上下文長度與 DPO／RLHF 管線，評估是否足以在離線或內網環境取代雲端大模型 [來源：iThome, 2025]。
  5. 治理 Deepfake 與資料最小化：公共部門案例顯示多模態摘要可提升行政效率，但同場域討論亦點出隱私與真偽風險；需搭配存取稽核與輸出浮水印政策 [來源：新通訊專欄, 2026；Gemini SynthID 說明, 2025]。

與現有知識的連結

本專案 daily-digest-prompt 以 PowerShell 管線、結構化結果 JSON 與 ntfy 通知為核心，與多模態研究的交集在於：任何自動化 digest 若引入影像／音訊理解，都必須複用本 repo 已存在的 token／逾時／降級語意（見 CLAUDE.md 與 skills/knowledge-query 對 KB 匯入的規範）。因此多模態能力的採用應落在「可觀測子系統」而非裸模型呼叫。

未來展望

仍未完全解決的問題包含：（1）如何在公開評測與企業機密文件之間建立可遷移的壓力測試；（2）多語言（尤其中日韓）小字與直排文件的穩定 OCR／vision 協議；（3）跨模態對齊的可解釋性與稽核軌跡。建議下一輪研究聚焦「ColPali／頁級視覺檢索＋RAG」與本專案 tools/ 機械化管線的整合評估。
