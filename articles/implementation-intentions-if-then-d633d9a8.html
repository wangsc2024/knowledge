執行意圖（Implementation Intentions）— 從 If-Then 計畫、意圖-行動落差到 WOOP 策略的行為科學完整框架

概述

知識工作者每天面對大量待辦事項，最大的挑戰往往不是「不知道該做什麼」，而是「知道卻做不到」。這個現象在行為科學中被稱為意圖-行動落差（Intention-Action Gap）。研究顯示，人們僅有 53% 的機率能將良好意圖轉化為實際行動，意圖只能解釋 18-23% 的行為變異量。

Peter Gollwitzer 於 1999 年提出的執行意圖（Implementation Intentions）理論，透過結構化的 If-Then 計畫，將目標追求從依賴意志力的「頂層控制」轉變為情境觸發的「策略性自動化」，是目前最有實證支持的意圖-行動落差解法。結合 Gabriele Oettingen 的心理對比（Mental Contrasting），發展出 MCII/WOOP 整合框架，提供從願望到行動的完整路徑。

---

一、意圖-行動落差（Intention-Action Gap）

1.1 定義與量化

意圖-行動落差指個人的價值觀、態度或意圖與實際行為之間的不一致。又稱為：

  • 價值-行動落差（Value-Action Gap）
  • 知識-態度-實踐落差（Knowledge-Attitudes-Practice Gap）

關鍵數據：

  • 計畫行為理論（TPB）研究：意圖僅解釋 18-23% 的行為變異量
  • 人們將良好意圖轉化為行動的機率僅約 53%
  • 新年決心的失敗率高達 80% 以上

1.2 根因分析：COM-B 模型

Michie 等人的 COM-B 模型將行為的決定因素分為三大類：

| 因素 | 說明 | 待辦事項情境 |

|------|------|-------------|

| Capability（能力） | 心理與物理能力 | 是否具備完成任務的知識、技能、認知資源 |

| Opportunity（機會） | 外部促進/阻礙因素 | 時間、空間、工具、社會環境是否支持 |

| Motivation（動機） | 有意識與無意識的驅動 | 短期慾望、習慣、社會規範的干擾 |

1.3 啟動失敗與脫軌的六大障礙

啟動階段（Failing to Get Started）：

  1. 遺忘行動：認知負荷導致意圖從工作記憶中消退
  2. 錯失良機：未能辨識或抓住適當的行動時機
  3. 初始抗拒：短期成本（不適、努力）壓過長期收益

執行階段（Getting Derailed）：

  1. 注意力被劫持：競爭性誘惑自動吸引注意力
  2. 習慣性反應覆蓋：舊有行為模式壓制新意圖
  3. 負面情緒脫軌：焦慮、沮喪、疲勞消耗自我控制資源

二、執行意圖（Implementation Intentions）核心理論

2.1 定義與結構

執行意圖是一種 If-Then 格式的從屬計畫（subordinate plan），將預期的關鍵情境與目標導向的行為反應連結起來：

「如果情境 Y 出現，那麼我就執行行為 Z，以達成目標 X！」
（If situation Y is encountered, then I will initiate behavior Z in order to reach goal X!）

與目標意圖的區別：

| 維度 | 目標意圖 | 執行意圖 |

|------|---------|--------|

| 格式 | 「我要達成 X」 | 「如果 Y，那麼 Z」 |

| 指定內容 | What（什麼） | When, Where, How（何時、何地、如何） |

| 控制方式 | 頂層控制（需意志力） | 策略性自動化（情境觸發） |

| 認知需求 | 高（持續監控與決策） | 低（自動化反應） |

2.2 理論基礎：Rubicon 行動階段模型

Heckhausen 和 Gollwitzer 的行動階段模型將目標追求分為四個階段：

  1. 審思階段（Deliberative）：權衡各種願望的可行性與期望性
  2. 計畫階段（Implemental）：形成執行意圖，規劃何時、何地、如何行動
  3. 行動階段（Actional）：執行目標導向的行為
  4. 評估階段（Evaluative）：評估結果，決定繼續或調整

執行意圖在「計畫階段」形成，跨越從意圖到行動的 Rubicon（不歸路），確保後續階段的順暢執行。

2.3 雙重心理機制

執行意圖透過兩個互補機制發揮作用：

機制一：增強線索可及性（Enhanced Cue Accessibility）

  • 形成 If-Then 計畫會提升關鍵情境的心理表徵強度
  • 使人更快辨識出與行動相關的情境線索
  • 即使在認知負荷高的情況下也能偵測到觸發條件

機制二：策略性自動化（Strategic Automaticity）

  • 情境與反應之間建立強烈的心理連結
  • 觸發條件出現時，行為自動啟動，無需有意識的審思
  • 具備習慣的自動化特徵（即時性、效率性、無意識性），但由意圖而非重複經驗形成
核心洞見：執行意圖本質上是「一次性習慣形成」— 人們透過有意識的計畫，將控制權「委派」給預選的情境線索，實現自動化目標追求。

2.4 解決的四大自我調節問題

| 問題 | 機制 | 效果量 |

|------|------|-------|

| 啟動目標追求 | 情境觸發自動啟動行為 | d = 0.61 |

| 屏蔽干擾 | 預設的 If-Then 計畫壓制競爭性反應 | d = 0.77 |

| 脫離失敗路徑 | 預設退出條件，避免沉沒成本陷阱 | 顯著 |

| 保存自我控制資源 | 自動化減少意志力消耗 | 顯著 |

三、實證研究：後設分析結果

3.1 Gollwitzer & Sheeran（2006）後設分析

  • 94 項獨立研究，超過 8,000 名參與者
  • 整體效果量：d = 0.65（中到大效果）
  • 在健康行為、學業成就、環境行為等多領域均有顯著效果

3.2 MCII 後設分析（Cross & Sheffield, 2021）

  • 24 項獨立研究，15,907 名參與者
  • 整體效果量：g = 0.336（小到中效果）
  • 關鍵調節變項：面對面引導（g = 0.465）> 文件式引導（g = 0.277）

3.3 環境行為後設分析（2025）

  • 執行意圖對永續行為的效果量：d = 0.781（大效果）
  • 僅計入實驗研究時：d = 0.473（中等效果）

3.4 意圖強度的四大特徵

研究發現，強意圖具有：

  1. 更高的行為預測力
  2. 更強的時間穩定性
  3. 更大的改變抗性
  4. 資訊處理偏向性（傾向搜尋支持行動的資訊）

四、MCII 與 WOOP：整合框架

4.1 心理對比（Mental Contrasting）

Gabriele Oettingen 提出的心理對比策略要求個人：

  1. 先想像理想未來（激發動機）
  2. 再面對現實障礙（啟動問題解決）

單獨的正面幻想反而降低行動力（能量已在幻想中「消耗」），心理對比透過「現實衝擊」將幻想能量轉化為行動承諾。

4.2 MCII = 心理對比 + 執行意圖

Mental Contrasting with Implementation Intentions：

  • 心理對比先將願望轉化為有承諾的目標
  • 識別出的障礙成為 If-Then 計畫的「If」部分
  • 克服障礙的行為成為「Then」部分
  • 兩者組合的效果優於任一策略單獨使用

4.3 WOOP 實踐工具

| 步驟 | 英文 | 操作 | 待辦事項應用 |

|------|------|------|-------------|

| W | Wish | 明確一個重要且可行的願望 | 「今天完成 API 重構」 |

| O | Outcome | 想像達成後的最佳結果 | 想像重構完成後的程式碼品質 |

| O | Obstacle | 識別內在的主要障礙 | 「會被 Slack 訊息打斷」 |

| P | Plan | 形成 If-Then 計畫 | 「如果 Slack 通知響起，我就設定勿擾模式 2 小時」 |

WOOP 的優勢：

  • 可在 5 分鐘內完成的日常練習
  • 有書籍、影片、手機 App 等多種載體
  • 無需正式訓練即可產生效果
  • 適用於健康、學業、關係、個人發展等多領域

五、待辦事項執行的實踐應用

5.1 有效 If-Then 計畫的設計原則

  1. 具體化情境線索：使用精確的時間、地點、內在狀態

- 差：「如果有時間」→ 好：「如果下午 2 點吃完午餐回到座位」

  1. 工具性反應：指定具體的第一個動作

- 差：「就去運動」→ 好：「就穿上跑鞋出門慢跑回家」

  1. 條件式語法：使用 If-Then 而非順序語句

- 差：「我先做 A 再做 B」→ 好：「如果 A 完成，那麼立即開始 B」

  1. 非重疊性：多個 If-Then 計畫的觸發條件不應重複

5.2 結合 Todoist 的 If-Then 範本


任務：撰寫技術報告
WOOP 流程：
  W：今天完成報告初稿
  O：報告完成後可以安心進行下一個專案
  O：可能會因為找資料而分心瀏覽
  P：如果發現自己在瀏覽無關網頁，就立即關閉瀏覽器回到文件

配套 If-Then 計畫：
  1. 如果上午 9:00 打開電腦 → 立即開啟報告文件（啟動儀式）
  2. 如果寫到不確定的部分 → 先用 [TODO] 標記，繼續寫下一段（防止卡住）
  3. 如果 Pomodoro 計時器響起 → 站起來伸展 5 分鐘後繼續（精力管理）
  4. 如果感到疲倦想刷手機 → 閉眼深呼吸 30 秒後回到寫作（誘惑屏蔽）

5.3 與已有知識的整合

| 已有知識庫筆記 | 與執行意圖的關係 |

|---------------|----------------|

| Deep Work 與認知負荷理論 | If-Then 計畫透過自動化降低認知負荷，為深度工作創造條件 |

| Time Boxing 與 Parkinson's Law | 時間框是 If-Then 的觸發線索（「如果 9:00」），兩者互補 |

| 元認知（Metacognition） | PME 循環的「計劃」階段可嵌入 If-Then 計畫設計 |

| 原子習慣 | 執行意圖是「習慣堆疊」的科學基礎（情境-反應連結） |

六、邊界條件與注意事項

  1. 前提條件：執行意圖的效果建立在「已有強烈目標意圖」之上。沒有目標承諾，If-Then 計畫無法生效
  2. 計畫品質：模糊的計畫（「如果有機會」）效果遠不如精確計畫（「如果下午 3 點」）
  3. 面對面引導更有效：後設分析顯示引導式介入（g=0.465）優於自助式（g=0.277）
  4. 不適合所有目標：對於已經自動化的習慣行為，If-Then 計畫的邊際效益較低
  5. 需要定期更新：情境改變時，舊的 If-Then 計畫可能失效，建議每週反思調整
