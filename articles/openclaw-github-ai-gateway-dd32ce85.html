一、專案概述

OpenClaw（前身 ClawdBot → Moltbot → OpenClaw，openclaw/openclaw）是 2026 年 GitHub 上最具話題性的開源專案，以「Your own personal AI assistant. Any OS. Any Platform.」為定位，10 天內累積 210,000+ Stars，創下 GitHub 史上最快增長紀錄。截至 2026 年 2 月底已超過 220K Stars、20K+ Forks，採用 MIT 授權。

核心數據

| 指標 | 數值 |

|------|------|

| GitHub Stars | 220,000+ |

| Forks | 20,000+ |

| ClawHub Skills | 5,705+（社群貢獻） |

| 支援通訊管道 | 15+ 個（WhatsApp/Telegram/Slack/Discord 等） |

| 授權 | MIT |

| 語言 | TypeScript（Node.js） |

| 增長速度 | 10 天 210K Stars |

核心定位

OpenClaw 與 ChatGPT、Claude 等雲端 AI 的根本差異在於三個設計哲學：

  1. 完全開源（MIT）：所有程式碼公開，可自行審計與修改
  2. 本地優先（Local-First）：記憶以 Markdown 檔案存於本機，你完全掌控 Agent 知道什麼
  3. 自主排程（Heartbeat Daemon）：無需人工觸發，Agent 定時主動檢查並行動

二、核心架構

OpenClaw 的架構可拆解為五個核心層：

2.1 Gateway（閘道器）

  • 單一 Node.js 進程作為控制平面
  • 管理 sessions、channels、tools、events
  • 支援 daemon 安裝（背景持續運行）
  • 所有通訊管道（WhatsApp、Telegram、Slack 等）統一收束到 Gateway

2.2 Agent Loop（代理迴圈）

遵循經典 Observe → Plan → Act 模式，翻譯為平台術語：

  1. Intake：從各通訊管道接收訊息
  2. Context Assembly：組裝上下文（記憶 + 對話歷史 + 相關 Skill）
  3. Model Inference：呼叫 LLM（支援 Claude、DeepSeek、GPT 等多模型）
  4. Tool Execution：執行工具呼叫（Skill 定義的 API 互動）
  5. Streaming Replies：串流回應到來源管道
  6. Persistence：持久化記憶與狀態

Agent 運行佇列化、按 session key 序列化，並橋接到事件串流，使工具輸出與助手回應可即時串流。

2.3 Memory System（記憶系統）

設計哲學：Markdown 就是真相源（Source of Truth）

  • 記憶以純 Markdown 檔案存於 ~/clawd/ 工作區
  • 模型只記得寫入磁碟的內容
  • 可選搜尋後端：QMD（BM25 + 向量 + 重排序）
  • QMD 僅用於檢索，Markdown 始終是權威來源

與 Daily-Digest 的對比啟示：

  • 本專案的 context/digest-memory.json 類似 OpenClaw 的記憶系統
  • OpenClaw 選擇 Markdown 而非 JSON，因為 LLM 對 Markdown 理解最自然
  • QMD 搜尋後端類似本專案的知識庫 hybrid search

2.4 Heartbeat Daemon（心跳守護進程）

  • 預設每 30 分鐘觸發一次
  • 每次心跳載入 Agent 當前上下文
  • 讀取 HEARTBEAT.md 中的檢查清單
  • 只在需要行動時才發訊息（不是每 30 分鐘都發）
  • 無需人工觸發即可主動執行任務

與 Daily-Digest 的對比啟示：

  • 本專案的 HEARTBEAT.md 排程元資料概念與 OpenClaw 不謀而合
  • 差異：本專案用 Windows Task Scheduler 外部觸發，OpenClaw 內建 daemon
  • OpenClaw 的「只在需要時行動」模式值得借鑑

2.5 Multi-Agent Routing（多代理路由）

  • 可將不同通訊管道/帳號/對話對象路由到隔離的 Agent
  • 每個 Agent 有獨立工作區和 session
  • 支援 Voice Wake 和 Talk Mode（macOS/iOS/Android，使用 ElevenLabs）

三、ClawHub Skill 生態系統

3.1 Skill 架構

OpenClaw 的 Skill 系統與本專案高度相似：

  • Skill 定義於 ~/clawd/skills//SKILL.md
  • 每個 SKILL.md 包含與 API 互動或執行工作流的指引
  • Skill 檔案就是 Markdown — LLM 直接理解

五、安全風險與治理啟示

OpenClaw 的爆發性增長帶來嚴重安全隱患，這對所有 AI Agent 系統都是警示：

5.1 已知 CVE

| CVE 編號 | CVSS | 類型 | 修復版本 |

|----------|------|------|----------|

| CVE-2026-25253 | 8.8 | WebSocket 跨站劫持 → RCE | 2026.1.29 |

| CVE-2026-24763 | — | 命令注入 | — |

| CVE-2026-25157 | — | 命令注入 | — |

| CVE-2026-28363 | — | 遠端程式碼執行 | — |

5.2 系統性風險

  1. API Key 明文存儲：配置、記憶、聊天日誌中的 API key 以明文存放，RedLine/Lumma 竊取軟體已將 OpenClaw 檔案路徑加入目標清單
  2. Skill 供應鏈攻擊：5,700+ 社群 Skill 中 17% 被標記為惡意
  3. 全系統存取權：OpenClaw 需要完整系統存取，安全專家稱為「致命三角」
  4. Prompt Injection：開放式 Agent 易受提示注入攻擊

5.3 對本專案的安全治理啟示

| OpenClaw 風險 | Daily-Digest 對應防護 |

|---------------|----------------------|

| API Key 明文 | .env + pre_read_guard.py 攔截 |

| 惡意 Skill | scan-skills.ps1 + Cisco AI Defense 掃描 |

| 命令注入 | pre_bash_guard.py 13 條規則攔截 |

| Prompt Injection | todoist-query + research-task 消毒指引 |

| 全系統存取 | Hook 機器強制層限制檔案/路徑存取 |

本專案的 Hook 安全層（ADR-005）在設計上已預防了 OpenClaw 暴露的大部分風險。

七、對 Daily-Digest-Prompt 系統的借鑑意義

7.1 架構共通性

  1. Skill-First 策略：OpenClaw 的 SKILL.md 與本專案的 Skill 系統理念相同
  2. Heartbeat 排程：兩者都用 HEARTBEAT.md 管理排程元資料
  3. Local-First 記憶：Markdown/JSON 檔案系統持久化
  4. Gateway 模式：OpenClaw 單進程 Gateway ≈ 本專案的 PowerShell 腳本調度器

7.2 可借鑑的改進方向

  1. 動態 Skill 發現：從 SKILL_INDEX.md 靜態索引演進為 ClawHub 式動態搜尋
  2. Markdown 記憶：考慮將 digest-memory.json 轉為 Markdown 格式，提升 LLM 理解度
  3. 條件式心跳：「只在需要時行動」模式可減少無效執行
  4. 多管道擴展：目前僅 ntfy 單一通知管道，可參考 OpenClaw 多管道架構

7.3 需警惕的教訓

  1. 安全優先：OpenClaw 的 CVE 危機證明高速增長不等於高品質安全
  2. Skill 供應鏈：社群 Skill 必須經過安全審核（本專案的 scan-skills.ps1 已做到）
  3. 成本控制：Agent 主動模式的 API 成本可能失控

八、結論

OpenClaw 是 2026 年 AI Agent 領域的標誌性事件：它證明了「個人 AI 助手」已從概念走向主流。其 Skill-First 設計哲學、Markdown 記憶系統、Heartbeat 排程機制與本專案（Daily-Digest-Prompt）的設計理念高度共通，驗證了文件驅動架構的正確性。

然而，OpenClaw 也是一面鏡子——展示了安全治理跟不上增長速度的風險。本專案的 Hook 機器強制層（pre_bash_guard + pre_write_guard + pre_read_guard）、Skill 安全掃描、以及結構化日誌系統，恰好預防了 OpenClaw 暴露的核心安全問題。

核心啟示：好的 AI Agent 架構 = 透明（Markdown 記憶）+ 可控（Skill 隔離）+ 安全（機器強制層）+ 自主（Heartbeat 排程）。
